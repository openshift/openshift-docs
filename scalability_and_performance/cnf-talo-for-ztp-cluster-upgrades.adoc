= Topology Aware Lifecycle Operator for ZTP cluster upgrades
include::_attributes/common-attributes.adoc[]
:context: cnf-topology-aware-lifecycle-operator

toc::[]

You can use the Topology Aware Lifecycle Operator (TALO) to manage the software lifecycle of multiple single node OpenShift (SNO) clusters. TALO uses {rh-rhacm-first} policies to perform changes on the target clusters.
