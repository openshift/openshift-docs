== Configuring User Managed Certificates for the Kubernetes API

External clients access the Kubernetes API at `api.<cluster_name>.<base_domain>` hostname,
as defined by the values specified in the `install-config.yaml` file during installation.

The OpenShift Container Platform automatically provides and manages the TLS certificates used
to serve secure traffic when using the Kubernetes API.

You can specify custom certificates for securing the Kubernetes API traffic through the
`spec.servingCerts` section of the `APIServer` cluster configuration resource:

. First, create a kubernetes.io/tls type secret containing the custom TLS info in the
`openshift-config` namespace:
+
---
$ oc --namespace openshift-config create secret tls my_custom_cert --certFile /path/to/tls.crt --keyFile /path/to/tls.key
---
. Next, add a reference to the TLS secret to the `APIServer/cluster` configuration resource:
+
---
spec:
  servingCerts:
    namedCertificates:
    - names:
      - api.<cluster_name>.<base_domain>
      servingCertificate:
        name: <user_managed_cert_secret_name>
---
