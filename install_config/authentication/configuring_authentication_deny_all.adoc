== Overview

You can configure the {product-title} master configuration file to use Deny All All authentication in your cluster.


== Configuring Deny All Authentication

Set `DenyAllPasswordIdentityProvider` in the `identityProviders` stanza to
deny access for all user names and passwords.


To configure the Deny All identity provider:

include::install_config/authentication/configuring_authentication_allow_all.adoc[tag=configuring_authentication_common_steps1]

. Specify the `mappingMethod` parameter to determine how identities are mapped to users.
include::install_config/authentication/configuring_authentication_allow_all.adoc[tag=configuring_authentication_common_steps2]

. Specify the following values to configure Deny All authentication:
+
.Master Configuration Using `DenyAllPasswordIdentityProvider`
====

----
oauthConfig:
  ...
  identityProviders:
  - challenge: true
    login: true
    mappingMethod: claim
    name: my_deny_provider <1>    
    provider:
      apiVersion: v1
      kind: DenyAllPasswordIdentityProvider<2>
----
<1> This provider name is prefixed to provider user names to form an identity
name.
<2> Specify *_DenyAllPasswordIdentityProvider_* as the provider kind.
====
