= Tutorial: {product-title} with AWS STS explained

include::_attributes/attributes-openshift-dedicated.adoc[]
:context: cloud-experts-rosa-sts-explained

toc::[]

[role="_abstract"]
This tutorial outlines the two options for allowing {product-title} to interact with resources in a user's Amazon Web Service (AWS) account. It details the components and processes that {product-title} with Security Token Service (STS) uses to obtain the necessary credentials. It also reviews why {product-title} with STS is the more secure, preferred method.

[NOTE]
====
This content currently covers {product-title} Classic with AWS STS.
ifdef::openshift-rosa-hcp[]
For {product-title} with hosted control planes (HCP) with AWS STS, see xref:../../welcome/cloud-experts-rosa-hcp-sts-explained.adoc#cloud-experts-rosa-hcp-sts-explained[AWS STS and {product-title} with HCP explained].
endif::openshift-rosa-hcp[]
====

This tutorial will:

* Enumerate two of the deployment options:
** {product-title} with IAM Users
** {product-title} with STS
* Explain the differences between the two options
* Explain why {product-title} with STS is more secure and the preferred option
* Explain how {product-title} with STS works
