You might require a Web Application Firewall (WAF) in front of workloads running on OpenShift (ROSA). The WAF itself must not run on OpenShift to ensure that OpenShift resources do not experience denial of service due to handling the WAF.

[id="cloudfront-method_{context}"]
== CloudFront method

[NOTE]
====
This is the recommended method. It works with most third party WAF systems that act as a reverse proxy.
====

The CloudFront method uses a custom domain, custom route, Let's Encrypt (LE) SSL certificate, AWS CloudFront, and WAF.

For more information, see xref:../cloud_experts_tutorials/rosa-mobb-using-cloudfront-and-waf.adoc#rosa-mobb-using-cloudfront-and-waf[Using CloudFront and WAF].

[id="alb-method_{context}"]
== Application Load Balancer method

This method installs the Application Load Balancer (ALB) Operator and uses the ALB to route via WAF. There can only be one ALB for each application.

For more information, see xref:../cloud_experts_tutorials/rosa-mobb-using-alb-and-waf.adoc#rosa-mobb-using-alb-and-waf[Using ALB and WAF].
