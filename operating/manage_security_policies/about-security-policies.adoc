[role="_abstract"] 

{product-title-short} helps you secure your container infrastructure using a robust policy engine and an admission controller that allow you to establish appropriate security guardrails, inform on policy violations, and selectively enforce policies to prevent insecure situations. 

Specifically, {product-title-short} can help you fail the CI pipeline block deployments in the build stage, reject deployments from being deployed in the deployment stage, and shut down running deployments that violate runtime policies. {product-title-short} also provides policy management and violation management tools described in the following sections.

You can also create your own custom security policies by using the {product-title-short} portal or defining them in custom resources (CR) and using them with your Kubernetes continuous delivery (CD) tool.

[id="policy-enforcement_{context}"]
== Policy enforcement

{product-title-short} allows you to enforce security policies during all phases of the development lifecycle:

* Build stage enforcement: {product-title-short} checks the build by using the the `roxctl image check` and `roxctl deployment check` commands. You can configure your CD to fail the build if a problem is detected.
* Deploy stage enforcement: {product-title-short} detects violations and uses the Admission Controller to reject the deployment attempt or scale deployments to zero.
* Runtime stage enforcement: {product-title-short} can monitor unsafe Kubernetes requests and selectively block them by using the API; for example, it can block attempted access to config maps or secrets.

[id="share-security-policies_{context}"]
== Sharing security policies

Beginning from {product-title} version 3.0.44,  you can share your security policies between different Central instances, by exporting and importing policies.
Sharing policies helps you enforce the same standards for all your clusters.
To share policies, you export them as JSON files, and then import them back into another Central instance.

[NOTE]
====
Currently, you cannot export multiple security policies at once by using the {product-title-short} portal.
However, you can use the API for exporting multiple security policies.
In the {product-title-short} portal, go to *Help* -> *API reference* to see the API reference.
====
