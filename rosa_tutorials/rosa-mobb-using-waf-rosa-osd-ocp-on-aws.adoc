You might require a Web Application Firewall (WAF) in front of workloads running on OpenShift (ROSA). The WAF itself must not run on OpenShift to ensure that OpenShift resources do not experience Denial of Service through handling the WAF.

[id="cloudfront-method_{context}"]
== CloudFront method

This is the preferred method and can also work with most third party WAF systems that act as a reverse proxy.

This method uses a custom domain, custom route, Let's Encrypt (LE) SSL certificate, AWS CloudFront, and WAF.
