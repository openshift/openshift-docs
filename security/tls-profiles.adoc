= Enabling TLS profiles
include::modules/common-attributes.adoc[]

toc::[]

Cluster administrators can choose which cipher suites to use for the control plane. This provides a convenient way to ensure that {product-title} control plane components use cryptographic libraries that do not allow known insecure protocols, ciphers, or algorithms.

A TLS profile defines the TLS ciphers that the Kubernetes API server must use when connecting with the kubelet to protect communication between the kubelet and the Kubernetes API server.

You can separately configure TLS profiles for the Ingress controllers, the Kubernetes control plane (Kube API server, controller manager, and scheduler), and the kubelet when it acts as an HTTP server.
