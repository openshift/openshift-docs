= Enabling TLS profiles
include::modules/common-attributes.adoc[]

toc::[]

Cluster administrators can choose which cipher suites to use for their {product-title} cluster. A TLS profile defines the TLS ciphers that various {product-title} components  must use when connecting with other components throught the cluster. This provides a convenient way to ensure that the {product-title} components use cryptographic libraries that do not allow known insecure protocols, ciphers, or algorithms.

You can separately configure TLS profiles for the Ingress controllers, the Kubernetes control plane (Kubernetes API server, controller manager, and scheduler), and the kubelet when it acts as an HTTP server for the Kubernetes API server.
