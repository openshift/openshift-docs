= Enabling TLS security profiles
include::modules/common-attributes.adoc[]

toc::[]

Cluster administrators can choose which cipher suites to use for their {product-title} cluster. A TLS security profile defines the TLS ciphers that various {product-title} components  must use when connecting with other components throught the cluster. This provides a convenient way to ensure that the {product-title} components use cryptographic libraries that do not allow known insecure protocols, ciphers, or algorithms.

You can configure a TLS security profile for each of the following components:

// TODO: links?

* the Ingress controller
* the Kubernetes control plane (Kubernetes API server, Kubernetes controller manager, and Kubernetes scheduler)
* the kubelet, when it acts as an HTTP server for the Kubernetes API server
