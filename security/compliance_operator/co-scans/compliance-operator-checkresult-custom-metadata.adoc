= Propagating custom metadata to ComplianceCheckResult objects
include::_attributes/common-attributes.adoc[]
:context: compliance-operator-checkresult-metadata

toc::[]

Starting in Compliance Operator 1.9.0, you can add your own labels and annotations to `Rule` and `CustomRule` objects so that matching metadata appears on generated `ComplianceCheckResult` objects after a scan. This helps downstream tools, dashboards, and ticketing workflows consume business-specific identifiers without maintaining a separate mapping.
