The {zero-trust-full} leverages Secure Production Identity Framework for Everyone (SPIFFE) and the SPIFFE Runtime Environment (SPIRE) to provide a comprehensive identity management solution for distributed systems. {zero-trust-full} supports SPIRE version 1.12.4 running as an operand.

These release notes track the development of {zero-trust-full}.

== {zero-trust-full} 0.2.0 (Technology Preview)

Issued: 2025-09-08

There are no advisories for this release.

[id="zero-trust-manager-features-enhancements_{context}"]
=== New features and enhancements

*Support for managed OIDC Routes*

With this release, the Operator exposes the SPIRE OIDC Discovery Provider endpoints through OpenShift Routes under the domain `*.apps.<cluster-domain>` for the selected default installation.

With this release, the managedRoute` and  `externalSecretRef` fields have been added to the `spireOidcDiscovetProvider` spec.

With this release, the `managedRoute` field is boolean and is set to `true` by default. If set to `false`, the Operator stops managing the route and the existing route is not automatically deleted. If set back to 'true', the Operater resumes managing the route. If a route does not exist, the Operator creates a new one. If a route already exists, the operator forces an update to resolve any conflicts.

With this release, the `externalSecretRef` references an externally managed Secret that contians the TLS certificate for the `oidc-discovery-provider` Route host. When provided, this populates the route's `.Spec.TLS.ExternalCertificate` field.

*Enable Manual User Configurations*

With this release, the Operator controller switches to `create-only` mode once the `ztwim.openshift.io/create-only=true` annotation is present on the Operator’s APIs. This allows resource creation while skipping the updates. A user can update the resources manually to test their configuration. This annotation needs to be added to the following APIs:

* SpireServer

* ZeroTrustWorkloadIdentityManager

* SpireAgents

* SpireOIDCDiscoveryProvider

With this release, the annotation needs to be added on the {zero-trust-full} resource for static resources such as Role-Based Access Control (RBAC), Switched Virtual Circuit (SVC), and SA resources.

With this release, the annotation should be added to the SpireServer, Agent, OIDC, and the Container Storage Interface (CSI) resource APIs.

With this release, once the annotation is removed and the pod restarts, the operator tries to come back to the desired state. The annotation is applied only once during start up or a restart.

[id="cert-manager-operator-0-2-0-known-issues_{context}"]
=== Known issues

* The `JwtIssuer` field for both the `SpireServer` and the OpenID Connect (OIDC) Discovery Provider needs to be a URL. If the `JwtIssuer` field is not in this format, an error displays.

*Enabling the custom Certificat Authority Time-To-Live for the SPIRE bundle*

With this release, the following Time-To-Live (TTL) fields have been added to the `SpireServerSpec` API for SPIRE Server certificate management:

* `CAValidity` (default: 24h)

* `DefaultX509Validity` (default: 1h)

*  `DefaultJWTValidity` (default: 5m)

The hard-coded values can be replaced in the server configuration with user-configurable options that provide users with the flexibility to customize certificate and {svid-full} lifetimes based on their security requirements.

This initial release of {zero-trust-full} is a Technology Preview. This version has the following known limitations:

* Support for SPIRE federation is not enabled.

* Key manager supports only the `disk` storage type.

* Telemetry is supported only through Prometheus.

* High availability (HA) configuration for SPIRE Servers or the OpenID Connect (OIDC) Discovery provider is not supported.

* External datastore is not supported. This version uses the internal `sqlite` datastore deployed by SPIRE.

* This version operates using a fixed configuration. User-defined configurations are not allowed.

* The log level of operands are not configurable. The default value is `DEBUG``.
