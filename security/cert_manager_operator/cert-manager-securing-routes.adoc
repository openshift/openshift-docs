= Securing OpenShift routes with cert-manager
include::_attributes/common-attributes.adoc[]
:context: cert-manager-securing-routes

toc::[]

In the {product-title} 4.16 release, the route API is extended to provide a configurable option to reference TLS certificates via secrets. With the "Configure routes with externally managed certificate" feature enabled, you can minimize errors from manual intervention, streamline the certificate management process, and enable the {product-title} router to promptly serve the referenced certificate.
