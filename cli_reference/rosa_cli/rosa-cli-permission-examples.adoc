= Required minimum permissions for ROSA CLI commands
:context: rosa-cli-permission-examples
toc::[]

In certain situations, you might want to create roles with policies that only contain the required minimum permissions needed to perform specific actions using commands within the {product-title} (ROSA) CLI. Doing so adheres to the "principals of least privilege", where users that are assigned these roles are limited only to the action they need to perform, with no other permissions assigned to them that are outside the scope of the specific action.
While the examples included in this topic listed do not address every possible action that can be achieved using commands within the ROSA CLI, several of the more commonly-used commands are covered.
For more information about configuring permissions, policies and roles within the AWS console, see link:https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies.html[AWS Identity and Access Management] in the AWS documentation.
