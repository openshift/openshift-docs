= Setting up your environment for Red{nbsp}Hat build of Karpenter
:context: rosa-nodes-autonode-setup

toc::[]

[role="_abstract"]
Before creating your {product-title} cluster that uses {autonode}, you must create the Amazon Web Services (AWS) Identity and Access Management (IAM) resources that grant the Karpenter controller the permissions it needs to provision and manage compute nodes. Then, you must enable the service. Enabling AutoNode installs the Karpenter controllers and creates the default node classes that {autonode} uses to provision compute nodes.
 
{autonode} requires a dedicated IAM policy and role with permissions to create and terminate Amazon Elastic Compute Cloud (Amazon EC2) instances, manage security groups, and create instance profiles. This IAM role is assumed by the Karpenter controller through IAM Roles for Service Accounts (IRSA).

[NOTE]
====
{autonode} requires a cluster created in the private preview shard with properly tagged network resources. After enabling {autonode}, you can verify that the Karpenter custom resource definitions (CRDs) are present and ready.
====

.Prerequisites

* You have installed and configured the latest {rosa-cli-first} version 1.2.57 or later.
* You have installed and configured the latest AWS command-line interface (CLI).
* You have been enrolled in the {autonode} private preview program.
* You have access to a {product-title} cluster version 4.19 or later.
* You have the required AWS IAM permissions to create policies and roles.
