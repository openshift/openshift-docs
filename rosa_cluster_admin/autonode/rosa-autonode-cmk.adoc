= Encrypt root volumes with a customer-managed key
:context: rosa-autonode-cmk

toc::[]

[role="_abstract"]
You can use a customer-managed key (CMK) to encrypt the root volumes attached to worker nodes that the {autonode} provisions. By default, root volumes are encrypted using the default AWS KMS key for EBS encryption. To use a different symmetric encryption KMS key, you must configure the KMS key policy and create an `OpenshiftEC2NodeClass` with the desired key.
