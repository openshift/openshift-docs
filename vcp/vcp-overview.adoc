= Understanding virtualized control planes
:context: vcp-overview

toc::[]

[role="_abstract"]
A virtualized control plane deployment is an {product-title} cluster that runs its control plane nodes as virtual machines (VMs) on a hosting cluster with {VirtProductName}.

:FeatureName: KubeVirt Redfish
include::snippets/technology-preview.adoc[]

Using a virtualized control plane is useful in the following example scenarios:

* You must meet regulatory requirements that mandate VM-level isolation for control plane components.
* You want to reduce hardware costs by consolidating multiple cluster control planes on shared infrastructure.
* You need faster provisioning of new clusters compared to physical bare metal.

You can use installation workflows such as Agent-based Installer or {ztp-first}, to deploy virtualized control planes exactly like physical servers with baseboard management controllers (BMCs).

[NOTE]
====
Virtualized control planes differ from {hcp-capital}.
With virtualized control planes, the control plane runs as VMs with hypervisor-level isolation.
With {hcp-capital}, the control plane runs as pods with container-level isolation.
====
