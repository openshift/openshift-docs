= Booting VMs via Redfish
:context: vcp-booting-vms

toc::[]

[role="_abstract"]
The Redfish API operations described in this section are used during virtualized control plane deployment to boot VMs from installation media.
These operations are typically performed by the installation tooling (agent-based installer or ZTP), but understanding them helps with troubleshooting and manual intervention.

[id="vcp-insert-boot-media_{context}"]
== Insert boot media

Attach the installation ISO to the VM's virtual CD drive:

[source,terminal]
----
$ REDFISH_URL=$(oc get route kubevirt-redfish -n kubevirt-redfish -o jsonpath='{.spec.host}')
$ SYSTEM_ID="openshift-install.master-0"

$ curl -X POST -sk -u "$USER:$PASS" \
  "https://$REDFISH_URL/redfish/v1/Systems/$SYSTEM_ID/VirtualMedia/Cd/Actions/VirtualMedia.InsertMedia" \
  -H "Content-Type: application/json" \
  -d '{
    "Image": "http://<http_server>/agent.x86_64.iso",
    "Inserted": true,
    "WriteProtected": true
  }'
----

[id="vcp-set-boot-override_{context}"]
== Set boot source override

Configure the VM to boot from the virtual CD on the next power-on:

[source,terminal]
----
$ curl -X PATCH -sk -u "$USER:$PASS" \
  "https://$REDFISH_URL/redfish/v1/Systems/$SYSTEM_ID" \
  -H "Content-Type: application/json" \
  -d '{
    "Boot": {
      "BootSourceOverrideTarget": "Cd",
      "BootSourceOverrideEnabled": "Once"
    }
  }'
----

[NOTE]
====
To use the `BootSourceOverrideEnabled` field, you must enable the `RebootPolicy` feature gate on the hosting cluster.
====

[id="vcp-power-on-vm_{context}"]
== Power on the VM

Start the VM to begin the installation:

[source,terminal]
----
$ curl -X POST -sk -u "$USER:$PASS" \
  "https://$REDFISH_URL/redfish/v1/Systems/$SYSTEM_ID/Actions/ComputerSystem.Reset" \
  -H "Content-Type: application/json" \
  -d '{"ResetType": "On"}'
----
