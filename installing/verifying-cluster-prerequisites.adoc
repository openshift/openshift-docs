[role="_abstract"]
Before you install the Loki Operator or Red Hat OpenShift Logging Operator, verify that your cluster can offer persistent storage for `LokiStack` components. Missing storage prerequisites are one of the most common causes of silent deployment failures.

== Why verify prerequisites first

`LokiStack` uses `PersistentVolumeClaims` (PVCs) for ingester write-ahead logging (WAL) and temporary storage. Without a working default `StorageClass`:

* Operators install successfully.
* The `LokiStack` shows `Ready`.
* Pods remain `Pending` and no logs are collected or stored.

Completing these checks now prevents hours of troubleshooting later.
