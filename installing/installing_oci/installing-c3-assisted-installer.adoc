= Installing a cluster on {oci-c3-no-rt} by using the {ai-full}
:context: installing-c3-assisted-installer

toc::[]

You can use the {ai-full} to install a cluster on {oci-c3-no-rt}, so that you can run cluster workloads on on-premise infrastructure while still utilizing {oci-first} services. *copied from agent-based - ok?* This method is recommended for most users, and requires an internet connection.

If you want to set up the cluster manually or using other automation tools, or if you are working in a disconnected environment, you can use the Red Hat Agent-based Installer for the installation. For details, see xref:../../installing/installing_oci/installing-oci-agent-based-installer.adoc#installing-oci-agent-based-installer[Installing a cluster on {oci-first-no-rt} by using the Agent-based Installer].

For more details and the complete instructions, see link:https://app.slack.com/client/E030G10V24F/D05CCFA8UVB[OpenShift Cluster Setup with Assisted Installer on Compute Cloud@Customer]. 

== Main steps

The procedure is broken down into the following sections:

. Installing Terraform and OCI CLI on a C3 Bastion Server 

. Terraform Script Execution – Part 1  (Run Script via Home region) 

. OpenShift Image Preparation to OCI 

. Terraform Script Execution – Part 2  (“Run Script via C3 Region”) 

. Install the Cluster using the RH Assisted Installer UI

== Bastian installation

. Create a Compartment in your OCI Home Region that will be managing this deployment. 
. Create an Instance from that Compartment.
. SSH into the bastion Instance using the Public or Private IP Address.

For details, see link: TBD

== Terraform installation

Download the Terraform binary AMD64 zip file onto your laptop. Unzip the file and move it to the  `bin` directory.
