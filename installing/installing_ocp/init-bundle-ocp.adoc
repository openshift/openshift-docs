= Generating and applying an init bundle or cluster registration secret for RHACS on {osp}
include::modules/common-attributes.adoc[]
:context: init-bundle-ocp
:topic-helm:
:topic-operator:

toc::[]

[role="_abstract"]

{product-title-short} uses a special artifact during installation that allows Central to communicate securely with secured clusters that you are adding. Before release 4.7, {product-title-short} used _init bundles_ for this communication. In 4.7, {product-title-short} provides an alternative to init bundles called a _cluster registration secret_.

:FeatureName: Cluster registration secret
include::snippets/technology-preview.adoc[]
:!FeatureName:

Currently, using an init bundle or a CRS during installation is supported. However, {product-title-short} does not yet support using the portal to create a CRS. You must create the CRS by using the `roxctl` CLI.

With the addition of CRSes, the following system behaviors occur:

* When your secured cluster initially connects using the CRS certificate, it retrieves multiple certificates and stores them as Kubernetes secrets on the secured cluster.
* The deployment scripts used when installing {product-title-short} by using the `roxctl` CLI have been changed and by default use CRS instead of init bundles when deploying a new secured cluster. To change the default behavior, set `ROX_DEPLOY_SENSOR_WITH_CRS` to `false`.

Before you install the `SecuredCluster` resource on a cluster, you must create an init bundle or CRS. The cluster that has `SecuredCluster` installed and configured then uses this bundle or CRS to authenticate with Central. You can create an init bundle by using either the {product-title-short} portal or the `roxctl` CLI. You can use the `roxctl` CLI to create a CRS. You can then apply the init bundle or the CRS by using the {ocp} web console or the `oc` CLI, running the `helm install` command, or by using the `roxctl` CLI.
