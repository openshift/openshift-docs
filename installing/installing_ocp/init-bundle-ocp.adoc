= Generating and applying an init bundle or cluster registration secret for RHACS on {osp}
include::modules/common-attributes.adoc[]
:context: init-bundle-ocp
:topic-helm:
:topic-operator:

toc::[]

[role="_abstract"]

{product-title-short} uses a special artifact during installation that allows Central to communicate securely with secured clusters that you are adding. Before release 4.7, {product-title-short} used _init bundles_ exclusively for initiating the secure communication channel. Beginning with 4.7, {product-title-short} provides an alternative to init bundles called _cluster registration secrets_ (CRSes).

:FeatureName: Cluster registration secrets
include::snippets/technology-preview.adoc[]

include::snippets/cluster-registration-secrets.adoc[]

Currently, you can use either an init bundle or a cluster registration secret (CRS) during installation. However, {product-title-short} does not yet provide a way to create a CRS by using the portal. Therefore, you must create the CRS by using the `roxctl` CLI.

You can use CRSes when you are installing {product-title-short} by using the Operator or Helm installation methods. The scripts in the manifest installation method have been updated to use CRSes.

Before you set up a secured cluster, you must create an init bundle or CRS. The secured cluster then uses this bundle or CRS to authenticate with Central. You can create an init bundle by using either the {product-title-short} portal or the `roxctl` CLI. You must use the `roxctl` CLI to create a CRS. You can then apply the init bundle or the CRS by using the {ocp} web console, by using the `oc` or `kubectl` CLI, or by running the `helm install` command. If you are using the manifest installation method, the init bundle or CRS is included in the generated cluster bundle.
