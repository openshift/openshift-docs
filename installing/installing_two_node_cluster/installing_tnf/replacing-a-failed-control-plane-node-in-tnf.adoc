= Replacing a failed control-plane node in TNF
include::_attributes/common-attributes.adoc[]
:context: install-tnf-replacing-control-plane-node

toc::[]

[role="_abstract"]
In a Two-Node with Fencing (TNF) cluster environment, recovering a failed control plane node requires a specific workflow orchestrated by the Cluster Etcd Operator (CEO). Unlike standard high-availability clusters, TNF environments leverage deep integration between Pacemaker and CEO automation.

[NOTE]
====
This procedure differs from the standard etcd member replacement. The standard procedure uses manual `etcdctl member remove/add` commands, while TNF uses CEO-orchestrated automation with pacemaker integration.
====
