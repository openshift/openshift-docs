= Failed control plane node replacement in two-node OpenShift cluster with fencing
include::_attributes/common-attributes.adoc[]
:context: failed-control-plane-node-replacement-in-tnf

toc::[]

[role="_abstract"]
In a two-node OpenShift cluster with fencing (TNF) environment, recovering a failed control plane node requires a specific workflow orchestrated by the Cluster etcd Operator (CEO). Unlike standard high-availability clusters, TNF environments use deep integration between Pacemaker and CEO automation.

[NOTE]
====
This procedure differs from the standard etcd member replacement. The standard procedure uses manual `etcdctl member remove/add` commands and TNF uses CEO-orchestrated automation with pacemaker integration.
====
