= Pulling from private registries with delegated authentication

[role="_abstract"]
To pull images from private registries that delegate authentication to a separate service in {product-title}, you can create pull secrets for both the authentication server and the registry endpoint. Use the `oc create secret docker-registry` command to create separate secrets for each service.

.Procedure

. Create a secret for the delegated authentication server by entering the following command:
+
[source,terminal]
----
$ oc create secret docker-registry \
    --docker-server=sso.redhat.com \
    --docker-username=developer@example.com \
    --docker-password=******** \
    --docker-email=unused \
    redhat-connect-sso
----

. Create a secret for the private registry by entering the following command:
+
[source,terminal]
----
$ oc create secret docker-registry \
    --docker-server=privateregistry.example.com \
    --docker-username=developer@example.com \
    --docker-password=******** \
    --docker-email=unused \
    private-registry
----
