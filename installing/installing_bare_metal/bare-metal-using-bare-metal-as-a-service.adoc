= Using bare metal as a service
include::_attributes/common-attributes.adoc[]
:context: bare-metal-using-bmaas

toc::[]

[role="_abstract"]
To run specialized workloads that require direct hardware access or high-performance computing, you can use Bare Metal as a Service (BMaaS) to provision and manage hosts. BMaaS uses the Metal^3^ API and the Bare Metal Operator (BMO) to integrate these external resources with your cluster, providing full lifecycle management for independent bare-metal systems.

These hosts, external to the {product-title} cluster, can run workloads that might not be suitable for containerization or virtualization. For example, workloads such as applications that require direct hardware access, conduct high-performance computing tasks or are legacy applications. BMaaS has the following capabilities:

* Provisioning of bare-metal hosts, including initial configuration.
* Lifecycle management such as power management, firmware updates, and decommissioning by using the BMO.

As standalone systems, these hosts operate independently of the {product-title} cluster and support diverse workloads by integrating bare-metal resources with containerized and virtualized applications. BMaaS can run other operating systems, but only {op-system-base-full} and CentOS Stream 9 were tested.
