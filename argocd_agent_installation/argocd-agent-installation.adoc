= Installing Argo CD Agent
:context: argocd-agent-installation

toc::[]

The Argo CD Agent consists of two components, *Principal* and *Agent* that work together to synchronize applications between the control plane and workload clusters in a hub-and-spoke configuration. This guide provides instructions for installing the required Argo CD Agent components. You can install the Principal component by using Argo CD resources and the Agent component by using Helm charts.

For an architectural overview and more details about Argo CD Agent, see the `Understanding Argo CD Agent architecture` section.

.Prerequisites

* You have installed the {gitops-title} Operator on two {OCP} clusters.
* You have installed the OpenShift CLI (`oc`).
* You have installed the {gitops-title} `argocd` CLI.
* You have logged in to the default Argo CD instance.
* You have installed a `cluster-scoped Argo CD` instance in your defined namespace.
* You have configured `Apps in any namespace` feature on the Hub cluster.
* You have installed `argocd-agentctl` CLI tool on your {OCP} clusters.
* You have installed `Helm CLI` for your Agent setup.
