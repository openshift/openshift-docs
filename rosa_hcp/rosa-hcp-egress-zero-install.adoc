= Creating {egress-zero-title}
include::_attributes/attributes-openshift-dedicated.adoc[]
:context: rosa-hcp-egress-zero-install

toc::[]

[role="_abstract"]
You can create a {product-title} cluster with {egress-zero} to pull Red{nbsp}Hat container images from a regional Amazon Elastic Container Registry (ECR) instead of from the public internet. This enhances cluster stability and security.

Your cluster first tries to pull the images from Quay, and when they are not reached, it instead pulls the images from the image registry in the local region.

Clusters with {egress-zero} pull Red{nbsp}Hat container images from an ECR in the cluster's local region instead of from registries on the public internet. ECR provides storage for OpenShift release images as well as Red{nbsp}Hat Operators. All requests for ECR are kept within your AWS network by serving them over a VPC endpoint within your cluster.

{egress-zero-title} use AWS ECR to provision your clusters without the need for public internet. Because necessary cluster lifecycle processes occur over AWS private networking, AWS ECR serves as a critical service for core cluster platform images.

You can create a fully operational cluster that does not require a public egress by configuring a virtual private cloud (VPC) and using the `--properties zero_egress:true` flag when creating your cluster.

[NOTE]
====
Clusters created in restricted network environments cannot use certain {product-title} features, such as {red-hat-lightspeed} and Telemetry, and might experience failures for workloads that require public access to registries such as `quay.io`. With {egress-zero}, you can install Red{nbsp}Hat-owned Operators from the software catalog, but only the default Operator channel is mirrored. For a complete list of Red{nbsp}Hat-owned Operators, see the Red{nbsp}Hat Ecosystem Catalog.
====

[IMPORTANT]
====
* You can use {egress-zero} on all supported {product-title} versions. Red{nbsp}Hat recommends using the latest available z-stream release for each {ocp} version.

* Due to an upstream issue with the internal image registry in disconnected environments, clusters with {egress-zero} cannot fully use all platform components. You can restore these features by upgrading or installing your cluster with the latest {ocp} version.
====
