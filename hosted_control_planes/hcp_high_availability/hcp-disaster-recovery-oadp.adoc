= Disaster recovery for a hosted cluster by using {oadp-short}
:context: hcp-disaster-recovery-oadp

toc::[]

[role="_abstract"]
By using the {oadp-first} Operator for disaster recovery, you can restore hosted cluster namespaces from object storage instead of manually rebuilding every cluster. In addition, you back up etcd as part of the control plane backup, and you can back up hosted clusters independently.

You can use the {oadp-short} Operator to perform disaster recovery for {hcp} on {aws-first} and bare metal.

The disaster recovery process with {oadp-first} involves the following steps:

. Preparing your platform, such as {aws-full} or bare metal, to use {oadp-short}
. Backing up the data plane workload
. Backing up the control plane workload
. Restoring a hosted cluster by using {oadp-short}
