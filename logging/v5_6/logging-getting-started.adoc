= Getting started with logging

This overview of the logging deployment process is provided for ease of reference. It is not a substitute for reviewing the full documentation.

.Prerequisites
* {product-title} 4.10+
* LogStore preference: *Elasticsearch* or *LokiStack*
* Output pipeline selection

.Procedure
. Install the Operator for the logstore you'd like to use. For *Elasticsearch*, install the *OpenShift Elasticsearch Operator*. For LokiStack, install the *Loki Operator*.

. Install the *Red Hat OpenShift Logging Operator*.
+

include::snippets/logging-elastic-dep-snip.adoc[]
+
. Create a `ClusterLogging` custom resource (CR) instance.
.. Select a Collector Implementation. Refer to the xref:../logging-5-6-reference.adoc[Logging reference] to review the differences between Fluentd and Vector implementations.
+

include::snippets/logging-fluentd-dep-snip.adoc[]
+
. Create a `ClusterLogForwarder` custom resource (CR) instance. Refer to the xref:../logging-5-6-reference.adoc[Logging reference] to review the available output options for logging.
.. Create a secret for selected output pipeline.
