= Build container images for your application

include::_attributes/common-attributes.adoc[]
:context: using-builds

toc::[]

[role="_abstract"]
Build container images by using build strategies directly on your OpenShift cluster, eliminating the need for external build infrastructure.
Container images package your application code, dependencies, and runtime environment into a portable unit that runs consistently across different environments.
You can build images from Dockerfiles by using the Buildah strategy, compile source code with Source-to-Image (S2I), or use Cloud Native Buildpacks for automatic language detection and best practices.
{builds-shortname} supports various source inputs including Git repositories and OCI artifacts.
If you are working in fully connected environments or network-restricted clusters with proxy configurations, you can build secure, optimized container images for your applications.
For applications requiring Red Hat Enterprise Linux (RHEL) packages, you can also build images with RHEL subscription entitlements.
