For accessing the binding metadata from a workload, the metadata has to be projected into it from the backing service. After the backing service exposes the binding metadata, {servicebinding-title} Operator automatically projects this set of metadata into your workload in the following methods:

. By default, as files.
. As environment variables, after you configure the `.spec.bindAsFiles` parameter from the `ServiceBinding` resource.
