= Using the SR-IOV plugin
include::_attributes/attributes-microshift.adoc[]
:context: microshift-SRIOV-plugin

toc::[]

When you run network-intensive workloads on MicroShift, you need a simplified and declarative method to configure and access high-performance networking interfaces. By supporting the SR-IOV device plugin and CNI provider using an optional installation, The SR-IOV device plugin exposes SR-IOV devices as resources and uses NetworkAttachment Definitions (NADs) to access Virtual Functions (VFs). As a result, you do not need to manually configure SR-IOV through the operating system. {microshift-short} allows you to treat Virtual Functions as resources that can be reliably mapped to your containerized applications. These resources reduce manual error and ensure consistent, low-latency networking for critical applications.
