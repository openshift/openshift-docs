= Overview of cluster updates with {rh-rhacm} and {cgu-operator}
include::_attributes/common-attributes.adoc[]
:context: core-cluster-upgrades-overview

toc::[]

[role="_abstract"]
You can use {rh-rhacm-first} and {cgu-operator-full} ({cgu-operator}) to perform z-stream, y-stream, and EUS-to-EUS updates on spoke clusters managed from a hub cluster.
This section describes the policy-based update workflow where you define update policies on the {rh-rhacm} hub and {cgu-operator} orchestrates their enforcement across target clusters.

If you are managing a single cluster or troubleshooting a specific cluster directly, see xref:../../../post_installation_configuration/day_2_core_cnf_clusters/updating/update-welcome.adoc#update-welcome[Updating an {product-title} cluster] for manual updates of individual clusters.
