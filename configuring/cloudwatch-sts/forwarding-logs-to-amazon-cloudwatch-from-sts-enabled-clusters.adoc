= Forwarding logs to Amazon CloudWatch from STS enabled clusters

:context: forwarding-logs-to-amazon-cloudwatch-from-sts-enabled-clusters
toc::[]

Amazon CloudWatch is a service that helps administrators observe and monitor resources and applications on {aws-first}.

You can forward logs from {loggingproductname} to CloudWatch creating a ClusterLogForwarder.

To configure CLF to forward logs to CloudWatch you need:

{aws-first} {sts-first} lets {ocp-product-title} components and workloads assume {aws-short} Identity and Access Management (IAM) roles and obtain short-lived, limited-privilege credentials.

How it works:

* The log collector requests temporary {aws-short} credentials from {sts-fist} by presenting its service account token to the OIDC provider in AWS. 
* {aws-short} validates the token. Afterwards, depending on the trust policy, {aws-short} issues short-lived, temporary credentials (access key ID, secret access key, and session token) for the log collector to use.




The logging system intelligently parses the secret and extracts a valid role ARN string, even if it's nested under a 'credentials' key with other data present. This change simplifies the secret creation process, especially when using tools like the CCO utility that might include extra information.

== Prerequisites

* Your cluster supports {sts-first}.
