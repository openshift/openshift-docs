= Configuring private connections
include::../includes/attributes-openshift-dedicated.adoc[]
:context: rosa-configuring-private-connections


Private cluster access can be implemented to suit the needs of your {product-title} (ROSA) environment.

.Procedure
. Access your ROSA AWS account and use one or more of the following methods to establish a private connection to your cluster:

- xref:dedicated-aws-peering[Configuring AWS VPC peering]: Enable VPC peering to route network traffic between two private IP addresses.

- xref:dedicated-aws-vpn[Configuring AWS VPN]: Establish a Virtual Private Network to securely connect your private network to your Amazon Virtual Private Cloud.

- xref:dedicated-aws-dc[Configuring AWS Direct Connect]: Configure AWS Direct Connect to establish a dedicated network connection between your private network and an AWS Direct Connect location.

. link:https://access.redhat.com/documentation/en-us/openshift_container_platform/4.16/html-single/install_rosa_classic_clusters/#rosa-private-cluster[Configure a private cluster on ROSA].
