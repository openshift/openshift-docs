= {oadp-short} 1.4 performance
include::_attributes/common-attributes.adoc[]
:context: oadp-performance-1-4

toc::[]

The objective of this performance testing was to run regression test cases for backup and restore performance of {oadp-first} 1.4 using the plugin types Kopia and CSI.

The majority of CSI and Kopia flows showed comparable performance with the previous {oadp-short} 1.3, including the Velero Built-in DataMover. In the majority of cases tested, the main performance-related difference for this release was related to Kopia flows, which showed a significant improvement in the duration of results compared to {oadp-short} versions 1.3.

| Test | Result (hh:mm:ss) |
| ---- | ----------- |
| Restore a namespace with 33K secrets | * Restore without `update` flag took 11 minutes compared to 55 minutes when using {oadp-short} 1.3.0 * Restore with `update` flag took 6 minutes compared to 28 minutes when using {oadp-short} 1.3.0 |
| Backup using Kopia filesystem (FS) of 4TiBs  logical unit number (LUN) of which, 3TiBs is utilized data on CephRBD storage class. | The backup took 37 inutes and 04 seconds compared to 2 hours 22 minutes and 12 seconds on {oadp-short} 1.3.1. |
