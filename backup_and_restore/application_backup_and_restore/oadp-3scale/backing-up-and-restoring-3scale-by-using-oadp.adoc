= Backing up and restoring 3scale by using OADP

With Red Hat 3scale API Management, you can manage your APIs for internal or external users. 
You can deploy 3scale API Management components on-premise, in the cloud, as a managed service, or in any combination based on your requirements.

With {oadp-first}, you can safeguard 3scale deployments by backing up application resources, persistent volumes, and configurations. You can configure OADP to perform the following operations with 3scale: 

* Create a backup of 3scale components by following the steps from xref:../../../backup_and_restore/application_backup_and_restore/oadp-3scale/backing-up-3scale-by-using-oadp.adoc#backing-up-3scale-by-using-oadp[Backing up 3scale].
* Restore the components to scale up the 3scale Operator and deployment by following the steps from xref:../../../backup_and_restore/application_backup_and_restore/oadp-3scale/restoring-3scale-by-using-oadp.adoc#restoring-3scale-by-using-oadp[Restoring 3scale].

[NOTE]
====
In this example, the non-service affecting approach is used to back up and restore 3scale on-cluster storage by using the {oadp-first} Operator.
Additionally, ensure that you are restoring 3scale on the same cluster where it was backed up from. If you want to restore 3scale on a different cluster, ensure that both clusters are using the same custom domain.
====
