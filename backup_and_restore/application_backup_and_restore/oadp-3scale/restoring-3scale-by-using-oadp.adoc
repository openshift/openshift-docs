= Restoring 3scale by using OADP
include::_attributes/common-attributes.adoc[]
:context: restoring-3scale-by-using-oadp

toc::[]

You can restore 3scale components by restoring the backed up 3scale Operator resources. You can also restore databases such as MySQL and Redis. 

Once the data has been restored, you can scale up the 3scale Operator and deployment. 

.Prerequisites

* You installed and configured Red Hat 3scale. For more information, see link:{Link3scaleInstalling3scale}#install-threescale-on-openshift-guide[Installing 3scale API Management on OpenShift] and link:{Link3scaleInstalling3scale}[Red Hat 3scale API Management].

* You backed up the 3scale Operator, and databases such as MySQL and Redis.

* Ensure that you are restoring 3scale on the same cluster where it was backed up from. If you want to restore 3scale on a different cluster, ensure that both clusters are using the same custom domain.
