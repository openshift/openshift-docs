= Lifecycle policies and planning

To plan an upgrade, review the link:https://docs.openshift.com/rosa/rosa_policy/rosa-life-cycle.html[lifecycle policies]. This information includes how many versions are supported, the CVE process, mandatory upgrade policies, and so on.

[id="rosa-upgrading_{context}"]
= Upgrading a cluster
There are three methods to upgrade a {product-title} (ROSA) cluster:

* Manually through the `rosa` CLI
* Manually through the OpenShift Cluster Manager (OCM) console
* Scheduling automatic upgrades through the OpenShift Cluster Manager (OCM) console
