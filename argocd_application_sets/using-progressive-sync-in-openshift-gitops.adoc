[role="_abstract"]
Progressive Sync is an advanced feature that supports application rollout strategies by managing synchronization across multiple clusters. You can configure `ApplicationSet` strategies to control how applications are created, updated, and deleted, as well as how to enable and use the *Progressive Sync* status panel in the *Application details* view.

You can configure creation strategies (AllAtOnce or RollingSync) to control application deployment patterns and deletion strategies to manage application removal order. When you enable Progressive Sync and configure a `RollingSync` strategy, the {gitops-title} console displays real-time visibility into sync waves, application health, and deployment progress.

[NOTE]
====
To monitor Progressive Sync status in the {gitops-shortname} Web UI, ensure that the feature is enabled in your Argo CD controller and your ApplicationSet uses a `RollingSync` strategy.
====

[id="prerequisites_{context}"]
== Prerequisites

* You have logged in to the {OCP} cluster as an administrator.
* You have installed the {gitops-shortname} Operator on your {OCP} cluster.
* You can access the default Argo CD instance in the `openshift-gitops` namespace.
* You have created an Argo CD ApplicationSet resource in your defined namespace, for example, `openshift-gitops`.
