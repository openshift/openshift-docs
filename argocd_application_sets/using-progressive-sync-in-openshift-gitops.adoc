[role="_abstract"]
Progressive Sync is an advanced feature that improves application rollout strategies by managing synchronization stages across multiple clusters. {gitops-shortname}, based on Argo CD, introduces a *Progressive Sync* status panel in the *Application details* view. Use this feature to visually monitor rollout waves, synchronization status, and application health during deployments managed by an `ApplicationSet` controller with a `RollingSync` strategy.

In addition, you can use the {gitops-title} console to display real-time Progressive sync status, improve operational awareness, and simplify troubleshooting during staged rollouts.

[id="prerequisites_{context}"]
== Prerequisites

* You have logged in to the {OCP} cluster as an administrator.
* You have installed the {gitops-shortname} Operator on your {OCP} cluster.
* You can access the default Argo CD instance in the `openshift-gitops` namespace.
* You have created an Argo CD ApplicationSet resource in your defined namespace, for example, `openshift-gitops`.
* You have created a few Argo CD applications using the above Argo CD ApplicationSet resources, in your defined namespace, for example, `openshift-gitops`.
