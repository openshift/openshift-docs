[role="_abstract"]
{product-title} 3.0.59 includes feature enhancements, bug fixes, scale improvements, and other changes.

*Release date:* April 28, 2021

[id="important-bug-fixes"]
== Important bug fixes

* *ROX-6696*: {product-title} includes updates to AWS Security Hub integration to respect Amazon's rate limits on payload size to prevent errors.
* *ROX-5299*, *ROX-6496*, *ROX-6856*, *ROX-6718*, and *ROX-6472*:{product-title} includes updated Sensor which fixes multiple data race conditions, which resulted in errors.

[id="resolved-in-version-30591"]
=== Resolved in version 3.0.59.1

Release date: May 4, 2021

* *ROX-7053*: Previously, reusing a cluster name might cause autogenerated registry integrations to get duplicated.

[id="resolved-in-version-30592"]
=== Resolved in version 3.0.59.2

Release date: May 13, 2021

* *ROX-7154*: Previously, using standalone pods with attached services would sometimes result in unstable deployments and crash loops.

[id="important-system-changes"]
== Important system changes

* *ROX-7016*: Product is now rebranded as {product-title}.
* *ROX-6909*: The *Curl in image* and *Wget in image* policies are now disabled by default to reduce default violations.

[id="api"]
=== API

You can now use the *GetUpgradeStatus* endpoint `/v1/centralhealth/upgradestatus` which provides Central roll-back related information.

[id="roxctl-cli"]
=== roxctl CLI

The default value of the `--json-fail-on-policy-violations` option for the `roxctl image check` command is now `true`.
