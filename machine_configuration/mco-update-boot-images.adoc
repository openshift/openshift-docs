= Updated boot images

toc::[]

The Machine Config Operator (MCO) uses a boot image to scale up {op-system-first} nodes. For {gcp-first} and {aws-first} clusters, the Machine Config Operator (MCO) manages and updates the boot image in the machine sets in your cluster. This means that, by default, the MCO updates the boot images whenever you upgrade your cluster.

For all other platforms, the MCO does not update the boot image with each cluster update.
