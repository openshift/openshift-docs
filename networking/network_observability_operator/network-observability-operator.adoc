[role="_abstract"]
Network Observability Operator provides network flow monitoring, visualization, and analysis capabilities for {product-title} clusters.

Network Observability Operator enables you to monitor network traffic flows between pods, services, and nodes in your cluster. You can use this data to troubleshoot connectivity issues, analyze traffic patterns, detect security threats, and optimize network performance.

Key capabilities include:

* Collecting network flow data using eBPF technology
* Visualizing traffic flows in the {product-title} web console
* Filtering and analyzing flows by namespace, pod, service, or port
* Exporting flow data to external analysis tools

For complete documentation about installing, configuring, and using Network Observability Operator, see the Network Observability documentation in the Observability section.
