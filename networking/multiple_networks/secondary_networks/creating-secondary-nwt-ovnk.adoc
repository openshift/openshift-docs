= Creating secondary networks on OVN-Kubernetes
include::_attributes/common-attributes.adoc[]
:context: configuring-additional-network

toc::[]

As a cluster administrator, you can configure an additional secondary network for your cluster using either the `UserDefinedNetwork` (UDN) resource or `NetworkAttachmentDefinition` (NAD) resource. One thing you might consider is that UDN creates and manages NAD object for you and UDN provides flexibility when building additional networks.
