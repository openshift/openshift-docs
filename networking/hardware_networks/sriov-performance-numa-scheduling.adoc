= Optimize SR-IOV performance with NUMA-aware scheduling
include::_attributes/common-attributes.adoc[]
:context: sriov-performance-numa-scheduling

toc::[]

[role="_abstract"]
Improve SR-IOV network performance by aligning pods with the NUMA node where the SR-IOV device is located. This reduces memory latency and increases throughput for performance-critical workloads.

[IMPORTANT]
====
This is an advanced performance optimization technique. Before configuring NUMA-aware scheduling:

* Complete the xref:../../networking/hardware_networks/sriov-getting-started.adoc#sriov-getting-started[getting started guide]
* Understand your hardware's NUMA topology
* Verify that your workload benefits from NUMA alignment (not all workloads do)
====

== Prerequisites

* You have a working SR-IOV network configuration. Verify by running:
+
[source,terminal]
----
$ oc get sriovnetworknodepolicies -n openshift-sriov-network-operator
----
+
The output shows one or more SR-IOV network node policies.
* Your nodes have multiple NUMA nodes
* You installed the Topology Manager. Verify by running:
+
[source,terminal]
----
$ oc get kubeletconfigs -o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.spec.kubeletConfig.topologyManagerPolicy}{"\n"}{end}'
----
+
The output shows a topology manager policy other than `none`, such as `best-effort`, `restricted`, or `single-numa-node`.
