= Combine SR-IOV interfaces with bonding
include::_attributes/common-attributes.adoc[]
:context: using-pod-level-bonding

toc::[]

[role="_abstract"]
Increase network reliability and throughput by combining multiple Single Root I/O Virtualization (SR-IOV) virtual functions into a bonded interface within a pod.

:FeatureName: Bond Container Network Interface (CNI)

With pod-level bonding, you can create a bond interface from multiple SR-IOV virtual function interfaces in a kernel mode interface. The SR-IOV virtual functions are passed into the pod and attached to a kernel driver.

Creating a bond interface from multiple SR-IOV virtual functions on different physical functions enables workloads to maintain network connectivity even when one physical function fails, achieving high availability and increased throughput at the pod level.

[id="using-pod-level-bonding-prerequisites"]
== Prerequisites
