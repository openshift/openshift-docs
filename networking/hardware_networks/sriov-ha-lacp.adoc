= Ensure high availability with LACP monitoring
include::_attributes/common-attributes.adoc[]
:context: sriov-lacp-sriov

toc::[]

[role="_abstract"]
Prevent silent network failures by using Link Aggregation Control Protocol (LACP) to monitor SR-IOV physical function health and trigger automatic failover to backup interfaces.

For workloads using pod-level bonding with SR-IOV virtual functions (VFs), despite an upstream switch failure, an underlying physical function (PF) might still report an `up` state. This creates a silent failure, as attached VFs remain up and pods continue to send traffic to a dead endpoint, causing packet loss.

The PF Status Relay Operator solves this issue by using LACP as an active health check. In this configuration, each physical function (PF) is placed in its own single-member LACP bond with the upstream switch. When the Operator detects an LACP failure on a PF's bond, it changes the link state of the attached VFs from `auto` to `disabled`. This action triggers the pod's `active-backup` bond to fail over to its backup network path, maintaining high availability.

[id="sriov-lacp-sriov-prerequisites"]
== Prerequisites

* You configured pod-level bonding. For more information, see xref:../../networking/hardware_networks/sriov-ha-bonding.adoc#using-pod-level-bonding[Combine SR-IOV interfaces with bonding]. Verify by running:
+
[source,terminal]
----
$ oc get network-attachment-definitions -A | grep bond
----
+
The output shows the bond network attachment definitions.
