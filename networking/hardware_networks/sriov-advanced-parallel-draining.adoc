= Speed up SR-IOV configuration changes with parallel node draining
include::_attributes/common-attributes.adoc[]
:context: sriov-advanced-parallel-draining

toc::[]

[role="_abstract"]
Configure multiple nodes to drain in parallel during SR-IOV network policy updates. This speeds up configuration changes in large clusters where sequential node draining can take hours or days.

[IMPORTANT]
====
This is an advanced operational technique for large-scale clusters. Before enabling parallel draining:

* Complete the xref:../../networking/hardware_networks/sriov-getting-started.adoc#sriov-getting-started[getting started guide]
* Understand that nodes may restart during the draining process
* Ensure your cluster has enough nodes to handle workloads while nodes drain
* Test with a small `maxUnavailable` value first (for example, 2 nodes)
====

== When to use parallel node draining

Use parallel draining when:

* Your cluster has many nodes (10+) with SR-IOV devices
* You need to apply configuration changes quickly
* You have enough spare capacity to handle workloads during draining
* Sequential draining takes too long for your operational requirements

Do NOT use parallel draining when:

* Your cluster is small (fewer than 10 nodes)
* You have minimal spare capacity
* Workloads cannot tolerate multiple nodes being unavailable
* You are making your first SR-IOV configuration

== Prerequisites

* You have a working SR-IOV network configuration
* Your cluster has multiple nodes with SR-IOV devices
* You understand your cluster's capacity and workload requirements
