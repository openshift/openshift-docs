= Basic route configuration for Developers

toc::[]

Developers can create and configure a route to work with the `IngressController`. Routes can be basic or secure, with a wide variety of options to suit every environment.

//Ingress Traffic Overview
include::modules/ingress-traffic-overview.adoc[leveloffset=+1]

If you use unencrypted HTTP, you need to use a basic route option. There are four basic route options: 
* basic routes
* path-based routes
* custom host name routes
* wildcard routes

If you use encrypted HTTPS, you need to use a secure route option. There are a dozen secure route options:

//Edge-terminated TLS (oc create route edge --service=my-service)
//TLS reencrypt (oc create route reencrypt --service=my-service)
//TLS passthrough (oc create route passthrough --service=my-service; mutually exclusive with path-based routing, custom certificates, HSTS, session cookies, header actions)
//Path-based routing (oc create route edge/reencrypt  --service=my-service --path=/foo)
//Specifying target port (oc create route edge/reencrypt/passthrough  --service=my-service --port=8123)
//Custom host name (oc create route edge/reencrypt  --service=my-service --hostname=foo.example.com) 
//Sharding
//Custom certificate (oc create route edge/reencrypt  --service=my-service --cert=tls.crt --key=tls.key)
//Custom CA certificate (oc create route edge/reencrypt  --service=my-service --ca-cert=ca.crt)
//Custom destination CA cert for reencrypt routes (oc create route reencrypt  --service=my-service --dest-ca-cert=my-service-ca.crt)
//Insecure redirect policy (oc create route edge/reencrypt --service=my-service --insecure-policy=Allow/None/Redirect or oc create route passthrough --service=my-service --insecure-policy=None/Redirect)
//HSTS (must use oc edit to edit an already created route)
