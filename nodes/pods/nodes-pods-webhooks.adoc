= Authenticating pods with short-term credentials
include::_attributes/common-attributes.adoc[]

toc::[]

//Some {product-title} clusters use xref:../../authenticating/managing_cloud_provider_credentials/cco-short-term-creds.adoc#cco-short-term-creds[short-term security credentials for individual components] that are created and managed outside the cluster.
Applications in pods on these clusters can authenticate by using the short-term authentication method that the cluster uses.

To use this authentication method in your applications, you must configure a pod identity webhook.

[discrete]
== Compatibility limitations

This authentication method is only supported for {aws-first}, {gcp-first}, and {azure-full} clusters.

You can only configure a pod identity webhook that uses the same authentication method as the cluster.
Refer to the following example combinations:

* You can configure a pod identity webhook that uses {aws-short} {sts-first} on an {aws-short} cluster that uses {aws-short} {sts-short}.
* You cannot configure a pod identity webhook that uses {aws-short} {sts-first} on a {gcp-short} cluster.
* You cannot configure a pod identity webhook that uses {aws-short} {sts-first} on an {aws-short} cluster that does not use {aws-short} {sts-short}.
