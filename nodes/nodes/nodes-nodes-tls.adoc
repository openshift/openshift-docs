= Enabling TLS profiles for the kubelet
include::modules/common-attributes.adoc[]

toc::[]

You can configure a TLS profile for the kubelet when it is acting as an HTTP server to protect communication between the kubelet and the Kubernetes API server. The kubelet uses its HTTP/GRPC server to communicate with the Kubernetes API server, which sends commands to pods, gathers logs, and run exec commands on pods.

You can use a TLS profile to configure which TLS ciphers are required by the Kubelet to ensure secure HTTP communication between the kubelet and the Kubernetes API server. 

[NOTE]
====
By default, when the kubelet acts as a client with the Kubernetes API server, it automatically negotiates the TLS parameters with the API server.
====
