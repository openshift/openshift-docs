= Enable support for namespace-scoped Argo Rollouts installation
:context: enable-support-for-namespace-scoped-argo-rollouts-installation

toc::[]
The {gitops-title} enables support for two modes of Argo Rollouts installation:

* *Cluster-scoped installation* (default): The Argo Rollouts custom resources (CRs) defined in any namespace are reconciled by the Argo Rollouts instance. This allows you to use Argo Rollouts CR across any namespace on the cluster.
* *Namespace-scoped installation*: The Argo Rollouts instance is installed in a specific namespace and only handles Argo Rollouts CR within the same namespace. This installation mode includes the following benefits:
** This mode does not require cluster-wide `ClusterRole` or `ClusterRoleBinding` permissions. This means that only users with permissions that are limited to a specific namespace can use these resources.
** This mode provides security benefits by limiting the cluster scope of a single Argo Rollouts instance to a specific namespace.

[NOTE]
====
{gitops-title} does not allow both modes of Argo Rollouts installation at the same time to prevent unintended privilege escalation.
====
