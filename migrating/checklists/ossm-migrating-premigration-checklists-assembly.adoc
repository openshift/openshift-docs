.Prerequisites

* You have read and understand the differences between {SMProduct} 2 and {SMProduct} 3 in xref:../../ossm-migrating-read-me-assembly.adoc[Important information to know before migrating].
* You have read through the migration recommendations.
//migrating/ossm-migrating-from-2-to-3-assembly.html#recommendations_ossm-migrating-from-2-to-3-assembly
//exrefs for migration guides handled by OSSM-8852
* You have reviewed the " `ServiceMeshControlPlane` resource to `Istio` resource fields mapping" reference material.
//exrefs for migration guides handled by OSSM-8852
* You want to migrate from {SMProduct} 2 to {SMProduct} 3.
* You are running {SMProduct} {SMv2Version}.
* You have upgraded your `ServiceMeshControlPlane` resource to the latest version.
* If you are using the {KialiProduct}, you are running the latest version.
* You have installed the {SMProduct} Operator 3. To install the {SMProduct} 3 Operator, see: xref:../../install/ossm-installing-openshift-service-mesh.adoc[Installing OpenShift Service Mesh]

[IMPORTANT]
====
You must complete each checklist item before you can begin migrating your deployment and workloads.
====

include::modules/ossm-migrating-premigration-checklists-add-ons.adoc[leveloffset=+1]

.Next steps

* [ ] Disable Prometheus: `spec.addons.prometheus.enabled=false`
** [ ] Configure the `ServiceMeshControlPlane` with OpenShift Monitoring as the replacement. These instructions also include installing a standalone `Kiali` resource. Both can be done at the same time. link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.17/html/service_mesh/service-mesh-2-x#ossm-integrating-with-user-workload-monitoring_observability[Integration user-workload monitoring,window=_blank]
** [ ] If you are not using OpenShift monitoring, see: link:https://docs.redhat.com/en/documentation/red_hat_openshift_service_on_aws/4/html/service_mesh/service-mesh-2-x#integration-with-external-prometheus-installation[Integration with external Prometheus installation].

* [ ] Disable Tracing: `spec.tracing.type=None`
** [ ] Follow these instructions to configure the `ServiceMeshControlPlane` with OpenShift Distributed Tracing as the replacement: link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.17/html/service_mesh/service-mesh-2-x#ossm-configuring-distr-tracing-tempo_observability[Configuring Distributed Tracing Tempo].

* [ ] Disable Kiali: `spec.addons.kiali.enabled=false`
** [ ] If you haven't already created a standalone `Kiali` resource part of the previous items for Prometheus or tracing, follow these instructions: link:https://docs.redhat.com/en/documentation/red_hat_openshift_service_mesh/3.0.0tp1/html/observability/kiali-operator-provided-by-red-hat#ossm-install-kiali-operator_ossm-kiali-assembly[Kiali Operator provided by Red Hat].

* [ ] Continue to the next checklist.

== Checklist for disabling Grafana in {SMProduct} 2

Grafana is not supported in {SMProduct} 3, and must be disabled in your {SMProduct} 2 `ServiceMeshControlPlane`.

  * [ ] Disable Grafana: `spec.addons.grafana.enabled=false`.
