= oc-agentic command reference

[role="_abstract"]
The following table lists all available `oc agentic` commands.

.oc agentic commands
[cols="1,1",options="header"]
|===
| Command
| Description

| `oc agentic run create`
| Create a new agentic run. Accepts `--request`, `--agent`, and `--target-namespaces` flags.

| `oc agentic run list`
| List agentic runs. Accepts `-A`, `--phase`, and `-o wide\|json\|yaml` flags.

| `oc agentic run get <run_id>`
| Show details of a specific agentic run. Accepts `-o json\|yaml`.

| `oc agentic run approve <run_id>`
| Approve a step. Accepts `--stage`, `--option`, `--agent`, `--all`, and `--wait` flags.

| `oc agentic run deny <run_id>`
| Deny a step. Accepts `--stage` (defaults to next pending).

| `oc agentic run watch <run_id>`
| Stream phase transitions until the run reaches a terminal state.

| `oc agentic run logs <run_id>`
| Stream sandbox pod logs. Accepts `--step` and `-f` flags.

| `oc agentic run delete <run_id>`
| Delete an agentic run.

| `oc agentic status`
| Show system suspension state.

| `oc agentic suspend`
| Suspend all agentic operations. Accepts `--yes` to skip the confirmation prompt.

| `oc agentic resume`
| Resume agentic operations.

| `oc agentic version`
| Print the plugin version.
|===

[NOTE]
====
The default namespace is `openshift-lightspeed` unless overridden with `-n` or the kubeconfig context.
====
