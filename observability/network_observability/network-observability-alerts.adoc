== Configuring predefined alerts

Alerts are configured in the `FlowCollector` custom resource, through `spec.processor.metrics.alerts`.

They are organized by templates and variants. The template names are the ones listed in the preceding section, such as `PacketDropsByKernel`. For each template, you can define a list of variants, each with their own thresholds and grouping configurations.

.Example configuration
[source,yaml,subs="attributes,verbatim"]
----
spec:
  processor:
    metrics:
      alerts:
      - template: PacketDropsByKernel
        variants:
        # triggered when the whole cluster traffic (no grouping) reaches 10% of drops
        - thresholds:
            critical: "10"
        # triggered when per-node traffic reaches 5% of drops, with gradual severity
        - thresholds:
            critical: "15"
            warning: "10"
            info: "5"
          groupBy: Node
----

When you configure an alert, it overrides and replaces the default configuration for that template. To add a new alert on top of the default ones for a template, you must manually replicate the default configuration, which is described in the preceding section.

[id="network-observability-disable-alerts_{context}"]
== Disabling predefined alerts

Alert templates can be disabled in `spec.processor.metrics.disableAlerts`. This setting accepts a list of template names, as listed in the preceding section.

If a template is disabled and overridden in `spec.processor.metrics.alerts`, the disable setting takes precedence and the alert rule will not be created.
