## MAIN JOBS

preparing-gitops-install.adoc
Plan GitOps installation capacity requirements
When planning GitOps installation on an OpenShift cluster, I want to understand default resource allocations and sizing requirements for all workload components, so I can ensure adequate cluster resources are allocated to prevent installation failures and pod scheduling issues

installing-openshift-gitops.adoc
Install GitOps Operator via web console
When I need to enable GitOps capabilities on an OpenShift cluster using the web console UI, I want to install the operator from OperatorHub, so GitOps Operator is successfully installed with default Argo CD instance available in openshift-gitops namespace

installing-openshift-gitops.adoc
Install GitOps Operator via CLI
When I need to install GitOps Operator using command-line tools for automation or when web console is unavailable, I want to create namespace, OperatorGroup, and Subscription resources, so GitOps Operator is installed via CLI with all components configured correctly

installing-openshift-gitops.adoc
Access Argo CD UI after installation
When I need to log into Argo CD UI to begin managing applications, I want to authenticate using either OpenShift credentials or admin password, so I can successfully access the Argo CD dashboard

installing-argocd-gitops-cli.adoc
Install GitOps CLI tool on workstation
When I need command-line access to configure and manage GitOps and Argo CD resources, I want to install the argocd CLI on my local workstation, so I can use argocd commands from terminal or command prompt
