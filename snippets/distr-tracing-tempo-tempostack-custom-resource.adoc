.Example `TempoStack` CR for AWS S3 and MinIO storage and two tenants
[source,yaml]
----
apiVersion: tempo.grafana.com/v1alpha1
kind: TempoStack # <1>
metadata:
  name: simplest
  namespace: <project_of_tempostack_instance>
spec:
  storage: # <2>
    secret: # <3>
      name: <secret_name> # <4>
      type: <secret_provider> # <5>
  storageSize: <value>Gi # <6>
  resources:
    total:
      limits:
        memory: 2Gi
        cpu: 2000m
  tenants:
    mode: openshift # <7>
    authentication: # <8>
      - tenantName: dev # <9>
        tenantId: "1610b0c3-c509-4592-a256-a1871353dbfa" # <10>
      - tenantName: prod
        tenantId: "1610b0c3-c509-4592-a256-a1871353dbfb"
  template:
    gateway:
      enabled: true # <11>
    queryFrontend:
      jaegerQuery:
        enabled: true # <12>
----
<1> `This CR creates a TempoStack deployment with trace ingestion in the OTLP/gRPC and OTLP/HTTP protocol.` OR `This CR creates a TempoStack deployment, which is configured to receive Jaeger Thrift over the HTTP and OpenTelemetry Protocol (OTLP).` Ruben, a question for you: Is this still Jaeger Thrift? `[Yes/No]` https://pypi.org/project/opentelemetry-exporter-jaeger-thrift/
<2> Storage for storing traces.
<3> The secret you created in step 2 for the object storage that had been set up as one of the prerequisites.
<4> The value of the `name` field in the `metadata` section of the secret.
<5> The accepted values are `azure` for Azure Blob Storage; `gcs` for Google Cloud Storage; and `s3` for Amazon S3, MinIO, or {odf-full}.
<6> The size of the persistent volume claim for the Tempo Write-Ahead Logging (WAL). The default is `10Gi`.
<7> The value must be `openshift`.
<8> Lists the tenants.
<9> The tenant name from the `X-Scope-OrgId` header when ingesting the data.
<10> The unique identifier of the tenant. Must be unique throughout the lifecycle of the TempoStack deployment. This ID will be added as a prefix to the objects in the object storage. You can reuse the value of the UUID or `tempoName` field.
<11> Enables a gateway that performs authentication and authorization. The Jaeger UI is exposed at `http://<gateway_ingress>/api/traces/v1/<tenant_name>/search`.
<12> Exposes the Jaeger UI, which visualizes the data, via a route.
