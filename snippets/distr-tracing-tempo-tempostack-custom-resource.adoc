.Example `TempoStack` CR for AWS S3 and MinIO storage and two tenants
[source,yaml]
----
apiVersion: tempo.grafana.com/v1alpha1
kind: TempoStack
metadata:
  name: simplest
  namespace: <permitted_project_of_tempostack_instance>
spec:
  storage:
    secret:
      name: <secret_name>
      type: <secret_provider>
  storageSize: <value>Gi
  resources:
    total:
      limits:
        memory: 2Gi
        cpu: 2000m
  tenants:
    mode: openshift
    authentication:
      - tenantName: dev
        tenantId: "1610b0c3-c509-4592-a256-a1871353dbfa"
      - tenantName: prod
        tenantId: "1610b0c3-c509-4592-a256-a1871353dbfb"
  template:
    gateway:
      enabled: true
    queryFrontend:
      jaegerQuery:
        enabled: true
----
where:

`kind`:: This CR creates a `TempoStack` deployment, which is configured to receive Jaeger Thrift over the HTTP and OpenTelemetry Protocol (OTLP).
`namespace`:: The project that you have chosen for the `TempoStack` deployment. Project names beginning with the `openshift-` prefix are not permitted.
`spec`:: Red{nbsp}Hat supports only the custom resource options that are available in the {dt} documentation.
`storage`:: The storage for storing traces.
`secret`:: The secret you created in step 2 for the object storage that had been set up as one of the prerequisites.
`name`:: The value of the `name` field in the `metadata` section of the secret. For example: `minio`.
`type`:: The accepted values are `azure` for Azure Blob Storage; `gcs` for {gcp-full} Storage; and `s3` for Amazon S3, MinIO, or {odf-full}. For example: `s3`.
`storageSize`:: The size of the persistent volume claim for the Tempo Write-Ahead Logging (WAL). The default is `10Gi`. For example: `1Gi`.
`resources`:: Optional.
`mode`:: The value must be `openshift`.
`authentication`:: The list of tenants.
`tenantName`:: The tenant name, which is used as the value for the `X-Scope-OrgId` HTTP header.
`tenantId`:: The unique identifier of the tenant. Must be unique throughout the lifecycle of the `TempoStack` deployment. The {dt} uses this ID to prefix objects in the object storage. You can reuse the value of the UUID or `tempoName` field.
`gateway.enabled`:: Enables a gateway that performs authentication and authorization.
`jaegerQuery.enabled`:: Exposes the Jaeger UI, which visualizes the data, via a route at `+http://<gateway_ingress>/api/traces/v1/<tenant_name>/search+`.
