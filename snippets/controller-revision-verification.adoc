.Verification

* Check the controller revision reference in the updated VM `status` field by running the following command and verifying that the revision name is updated in the output:
+
[source,terminal]
----
$ oc get vms/{vm_name} -o json | jq .status.instancetypeRef
----
+
.Example output
[source,terminal]
----
{
  "controllerRevisionRef": {
    "name": "vm-cirros-csmall-csmall-3e86e367-9cd7-4426-9507-b14c27a08671-2"
  },
  "kind": "VirtualMachineInstancetype",
  "name": "csmall"
}
----

* You can also check that the VM instance is running the new configuration defined in the latest controller revision. For example, if you updated the instance type to use 2 vCPUs instead of 1, you could run the following command and check the output:
+
.Example command
[source,terminal]
----
$ oc get vmi/{vm_name} -o json | jq .spec.domain.cpu
----
+
.Example output that verifies that the revision uses 2 vCPUs
[source,terminal]
----
{
  "cores": 1,
  "model": "host-model",
  "sockets": 2
  "threads": 1
}
----
