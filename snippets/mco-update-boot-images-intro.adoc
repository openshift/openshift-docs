ifdef::openshift-enterprise,openshift-origin[]
The following table lists the platforms on which boot image management is available in {product-title}:

|===
|Platform |Worker machine sets |Control plane machine sets

|{gcp-first}
|Enabled by default
|Disabled by default

|{aws-first}
|Enabled by default
|Disabled by default

|{azure-first}
|Enabled by default 
|Disabled by default

|{vmw-first}
|Enabled by default
|Not supported

|===

For all other platforms, the MCO does not automatically update the boot image with each cluster update. For clusters where automatic updates are disabled or not supported, you can manually update the boot image on your nodes. See "Manually updating the boot image" for information. The method to update or specify the image varies by platform.  
endif::openshift-enterprise,openshift-origin[]
ifdef::openshift-dedicated[]
The following table lists the platforms on which boot image management is available in {product-dedicated}:

|===
|Platform |Worker machine sets |Control plane machine sets

|{gcp-first}
|Enabled by default
|Disabled by default

|{aws-first}
|Enabled by default
|Disabled by default

|===
For nodes where automatic updates are disabled, you can manually update the boot image on your cluster. See "Manually updating the boot image" for information. The method to update or specify the image varies by platform.  
endif::openshift-dedicated[]
ifdef::openshift-rosa[]
For {product-title}, boot image management is available by default on compute nodes and disabled by default for control plane nodes. For nodes where automatic updates are disabled, you can manually update the boot image on your nodes. See "Manually updating the boot image" for information.
endif::openshift-rosa[]
