The `shp` command line tool is a client for the Builds v2 API. You can use `shp` to manage builds on an {product-name} cluster.

== Basic syntax
----
$ shp [command or options] [arguments...]
----

[discrete]
[id="global-options"]
== Global options

The following options work with all the `shp` commands.

`-h, --help`:: Help for `shp`.
`--as <username>`:: Specify a username to impersonate for the operation.
`--as-group <stringArray>`:: Specify a group to impersonate for the operation. You can repeat this flag to specify multiple groups.
`--cache-dir <string>`:: Specify the cache directory. Otherwise, the default directory is `/Users/wyvie/.kube/cache`.)
`--certificate-authority <string>`:: Specify the path to a cert file for the certificate authority.
`--client-certificate <string>`:: Specify the path to a client certificate file for TLS.
`--client-key <string>`:: Specify the path to a client key file for TLS.
`--cluster <string>`:: Specify the name of a kubeconfig cluster to use.
`--context <string>`:: Specify the name of a kubeconfig context to use.
`--insecure-skip-tls-verify`:: Skip verifying the server's TLS certificate. Allow an insecure HTTP connection to the server instead of HTTPS.
`--kubeconfig <string>`:: Specify the path of the kubeconfig file to use for CLI requests.
`-n, --namespace <string>`:: If present (TBD), specify the namespace scope for this CLI request.
`--request-timeout <string>`:: Specify the length of time to wait before giving up on a single server request. Non-zero values should contain a corresponding time unit (e.g. `1s`, `2m`, `3h`). A value of zero means don not timeout requests. The default value is `0`.
`-s, --server <string>`:: Specify the address and port of the Kubernetes API server.
`--tls-server-name <string>`:: Specify the name of the server that has the TLS certificate. Otherwise, `shp` uses the `hostname` (s/used to contact the server is used/you specified by using `-s` or `--server`.)
`--token <string>`:: Specify the bearer token for authentication to the API server.
`--user <string>`:: Specify the name of a kubeconfig user to use.
