= About {IBMFusionSAN}

{IBMFusionSAN} is a scalable clustered file system for enterprise storage.


https://www.ibm.com/docs/en/storage-scale/5.2.2?topic=scale-improved-system-performance



Core Spectrum Scale Concepts

Fusion Access:
A singleton object that will install a specific version of the IBM CNSA software
Spectrum Cluster:
A singleton object that identifies the nodes where shared storage will be attached and utilized
LocalDisks:
Represents the shared storage LUNs
FileSystems:
Contains one or more LocalDisks that will be used to create the filesystem (“vmfs”)
StorageClasses / PVCs / PVs / StorageProfile:
StorageClass will point to a specific “FileSystem”. The other objects are OCP storage business as usual




Easy user experience
Wizard for installing configuring cluster, filesystems, storage classes
Familiar storage architecture
Leverage existing SAN infra (FC & iSCSI) via traditional methods.
Scales with workloads
Storage cluster grows with OpenShift cluster and VM workloads.
Proven performance
Tested up to ~3000 Vms on 6 bare metal hosts. More to come.
Available through IBM
Two flavors for basic storage provision and data protection.



Prereqs:

Bare Metal Worker Nodes with SAN storage attached
* All nodes need to see the same LUNs. How they are attached is environment dependent (multipath, direct-attach, FC, iSCSI, etc) and is not covered here. In general, a MachineConfig will need to take care of it



Installation and setup:
1. MachineConfig for Workers
A MachineConfig on workers installing the “kernel-devel” extension is required (*)
2. Install the upcoming Fusion Access Operator
This installs the operator in charge of UI, disk discovery and general installation of the IBM parts
3. Obtain Pull Secret from IBM
This Pull Secret will be needed for getting the containers from the IBM container registry
4. Create FusionAccess CR
This CR initializes the Spectrum Scale bits and triggers the Spectrum Scale operator installation (and starts the UI)
5. Follow UI wizard
The UI will show the user which nodes have shared disks and do walk through the necessary steps to configure and create the underlying Spectrum Objects (Cluster, LocalDisk, FileSystem) which manage the storage part. (Skipping the UI and doing it via YAML is also doable)
