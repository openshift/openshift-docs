= Configuring a cross-cluster live migration network
include::_attributes/common-attributes.adoc[]
include::_attributes/attributes-openshift-dedicated.adoc[]
:context: virt-configuring-cross-cluster-live-migration-network

toc::[]

Cross-cluster live migration requires that the clusters be connected in the same network. Specifically, virt-handler pods must be able to communicate.

Using the Dynamic Host Configuration Protocol Container Network Interface (DHCP CNI) plug-in can be advantageous in a cross-cluster live migration setting in that it removes the need to manually assign IP addresses and other network configuration settings. Instead, you are provided with a dynamic view of all network addresses across all {product-title} clusters, accommodating the movement of live workloads between clusters without interruption.

If the DHCP plug-in is not available, you can instead configure the Whereabouts plug-in.
