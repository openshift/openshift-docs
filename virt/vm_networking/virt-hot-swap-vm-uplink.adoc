= Hot swapping a virtual machine secondary network interface
include::_attributes/common-attributes.adoc[]
:context: virt-hot-swap-vm-uplink

toc::[]    

[role="_abstract"]
You can change the secondary network interface of a virtual machine (VM) without stopping your VM. The change is transparent to the guest operating system, preserving properties like the MAC address. 

By hot swapping the secondary network, you can move a running VM to a different network segment or VLAN and apply new network policies or reconfigure network topology without interrupting the workload. {VirtProductName} supports hot swapping secondary network interfaces for VMs that are connected to an OVN-Kubernetes localnet network, a Linux bridge network, and a Single Root I/O Virtualization (SR-IOV) network device. 

To hot swap a VM secondary network, you must update the `NetworkAttachmentDefinition` or `ClusterUserDefinedNetwork` custom resource (CR) for a running VM's network interface. This action triggers a live migration, re-connecting the VM to a new network without a reboot.
