== Deploying workflows in preview mode

You can deploy your local workflow on OpenShift in preview mode. This allows you to experiment and modify your workflow directly on the cluster, seeing changes almost immediately. Preview mode is used for final testing and validation before deploying to production. It also ensures that workflows will run smoothly in a production-like setting.

.Prerequisites

* {ServerlessLogicOperatorName} is installed on your cluster.
* You have created a {ServerlessLogicProductName} project or have access to a project with the appropriate roles and permissions to create applications and other workloads in OpenShift Container Platform.
* You have created your {ServerlessLogicProductName} project
* Install the OpenShift CLI `(oc)`.

To deploy a workflow in preview mode, {ServerlessLogicOperatorName} uses the build system on OpenShift. This involves creating a creating and link:https://docs.openshift.com/container-platform/4.13/cicd/builds/understanding-image-builds.html[Understanding image builds], which allows the {ServerlessLogicOperatorName} to build workflows directly on OpenShift.

This document explains how to build and deploy your workflow on a cluster using the {ServerlessLogicOperatorName} with a `SonataFlow` custom resource.
