= Configuring OpenAPI services endpoints
:context: serverless-logic-configuring-openAPI-services-endpoints
include::_attributes/common-attributes.adoc[]

toc::[]

{ServerlessLogicProductName} uses the `kogito.sw.operationIdStrategy` property to generate the REST client for invoking services defined in OpenAPI documents. This property determines how the configuration key is derived for REST client configuration. The possible values for `kogito.sw.operationIdStrategy` include `FILE_NAME`, `FULL_URI`, `FUNCTION_NAME`, and `SPEC_TITLE`.

* `FILE_NAME` (Default)
+

{ServerlessLogicProductName} uses the OpenAPI document file name to create the configuration key. The key is based on the file name, where special characters are replaced with underscores.
+
.Example configuration:
[source,sh]
----
quarkus.rest-client.stock_portfolio_svc_yaml.url=http://localhost:8282/
----
+
** OpenAPI File Path: `src/main/resources/openapi/stock-portfolio-svc.yaml`
** Generated configuration Key: `stock_portfolio_svc_yaml`
** Property example: The URL for the REST client is configured using the key `stock_portfolio_svc_yaml`.


* `FULL_URI`
+
{ServerlessLogicProductName} uses the entire URI path of the OpenAPI document as the configuration key. The full URI is sanitized to form the key.
+
.Example for Serverless Workflow
[source,json]
----
{
    "id": "myworkflow",
    "functions": [
        {
          "name": "myfunction",
          "operation": "https://my.remote.host/apicatalog/apis/123/document"
        }
    ]
    ...
}
----
+
.Example configuration:
[source,sh]
----
quarkus.rest-client.apicatalog_apis_123_document.url=http://localhost:8282/
----
+
** URI path: `https://my.remote.host/apicatalog/apis/123/document`
** Generated configuration key: `apicatalog_apis_123_document`
** Property example: The REST client URL is set using the key `apicatalog_apis_123_document`.


* `FUNCTION_NAME`
+
{ServerlessLogicProductName} combines the workflow ID and the function name referencing the OpenAPI document to generate the configuration key.
+
.Example for Serverless Workflow
[source,json]
----
{
    "id": "myworkflow",
    "functions": [
        {
          "name": "myfunction",
          "operation": "https://my.remote.host/apicatalog/apis/123/document"
        }
    ]
    ...
}
----
+
.Example configuration:
[source,sh]
----
quarkus.rest-client.myworkflow_myfunction.url=http://localhost:8282/
----
+
** Workflow ID: `myworkflow`
** Function name: `myfunction`
** Generated configuration key: `myworkflow_myfunction`
** Property example: Use the key `myworkflow_myfunction` for setting the REST client URL.

* `SPEC_TITLE`
+
{ServerlessLogicProductName} the info.title value from the OpenAPI document to create the configuration key. The title is sanitized to form the key.
+
.Example for OpenAPI document
[source,yaml]
----
openapi: 3.0.3
info:
  title: stock-service API
  version: 2.0.0-SNAPSHOT
paths:
  /stock-price/{symbol}:
...
----
+
.Example configuration:
[source,sh]
----
quarkus.rest-client.stock-service_API.url=http://localhost:8282/
----
+
** OpenAPI document title: `stock-service API`
** Generated configuration key: `stock-service_API`
** Property example: The URL for the REST client is configured using the key `stock-service_API`.
