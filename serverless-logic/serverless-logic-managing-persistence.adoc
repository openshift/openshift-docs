This section describes how to configure a SonataFlow instance to use persistence and store workflow context in a relational database.

By design, Kubernetes pods are stateless. This behavior can pose challenges for workloads that need to maintain the application state across pod restarts. In the case of{ServerlessLogicProductName}, the workflow context is lost when the pod restarts by default.

To ensure workflow recovery in such scenarios, you must configure workflow runtime persistence. Use the `SonataFlowPlatform` CR or the `SonataFlow` CR to provide this configuration. The scope of the configuration varies depending on which resource you use.
