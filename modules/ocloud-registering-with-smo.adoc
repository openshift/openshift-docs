= Register the O-Cloud Manager with an SMO

[role="_abstract"]
You can register the O-Cloud Manager with a Service Management and Orchestration (SMO) system by updating the `Inventory` custom resource with the SMO endpoint, OAuth credentials, and CA certificate configuration. Registration is a one-time operation that enables the SMO to manage infrastructure through the O2 IMS REST APIs.

.Prerequisites

* The O-Cloud Manager Operator is installed and running on the hub cluster.
* You have the SMO endpoint URL and registration endpoint path.
* You have OAuth2 client credentials (`client-id` and `client-secret`) for the O-Cloud Manager, obtained from the OAuth2 authorization server administrator.
* You have the CA certificate bundle if the SMO or OAuth2 server uses certificates signed by a non-public CA.

.Procedure

. If the SMO or OAuth2 server TLS certificates are signed by a non-public CA, create a `ConfigMap` with the CA certificate bundle and configure the cluster proxy:
+
[source,terminal]
----
$ oc create configmap -n openshift-config custom-ca-certs \
  --from-file=ca-bundle.crt=/path/to/ca-bundle.crt
$ oc patch proxy/cluster --type=merge \
  --patch='{"spec":{"trustedCA":{"name": "custom-ca-certs"}}}'
----
+
[WARNING]
====
If the cluster proxy already references a custom CA bundle, append the new certificates to the existing bundle rather than replacing it.
====

. Create a `ConfigMap` with the trust bundle injection label so that the O-Cloud Manager can access both private and public CA certificates:
+
[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  labels:
    config.openshift.io/inject-trusted-cabundle: "true"
  name: o2ims-trusted-ca-bundle
  namespace: oran-o2ims
----
+
[source,terminal]
----
$ oc apply -f <trusted_ca_bundle_filename>.yaml
----

. Create a `Secret` containing the OAuth client credentials:
+
[source,terminal]
----
$ oc create secret generic -n oran-o2ims oauth-client-secrets \
  --from-literal=client-id=<oauth_client_id> \
  --from-literal=client-secret=<oauth_client_secret>
----

. Update the `Inventory` custom resource in the `oran-o2ims` namespace to configure the SMO connection. Add the following to the `spec` section:
+
[source,yaml]
----
spec:
  smo:
    url: https://<smo_hostname> <1>
    registrationEndpoint: /mock_smo/v1/ocloud_observer <2>
    oauth:
      url: https://<oauth_server_hostname>/realms/<realm> <3>
      clientSecretName: oauth-client-secrets <4>
      tokenEndpoint: /protocol/openid-connect/token
      scopes:
        - profile
        - openid
        - role:ocloud-manager
      usernameClaim: preferred_username
      groupsClaim: roles
  caBundleName: o2ims-trusted-ca-bundle <5>
----
<1> The SMO server URL.
<2> The registration endpoint path on the SMO.
<3> The OAuth2 authorization server URL and realm.
<4> The name of the `Secret` containing OAuth client credentials.
<5> The name of the `ConfigMap` containing the trusted CA bundle. Omit if not using a private CA.

. Apply the updated `Inventory` resource:
+
[source,terminal]
----
$ oc apply -f <inventory_filename>.yaml
----

.Verification

* Verify that registration with the SMO completed successfully:
+
[source,terminal]
----
$ oc describe inventories.ocloud.openshift.io default -n oran-o2ims
----
+

[source,text]
----

  Deployment Status:
