= Using {SMProductName} with SPIRE on a single-cluster

[role="_abstract"]
Deploy SPIRE on a single {product-name} cluster to provide SPIFFE-compliant workload identities for service mesh workloads, replacing Istio's built-in certificate authority.

A single-cluster deployment provides SPIRE-issued workload identities for service mesh workloads running within one {product-name} cluster. This configuration is appropriate when:

* All your services run in a single {product-name} cluster
* You want to replace Istio's built-in certificate authority (CA) with SPIRE for enhanced security
* You need SPIFFE-compliant workload identities within a single trust domain
* You do not require cross-cluster service communication

Single-cluster SPIRE integration provides:

* One SPIRE Server acting as the CA for the cluster
* SPIRE Agents running on each node to issue certificates to workloads
* Automatic certificate rotation with approximately 1-hour TTL
* Zero trust workload authentication using SPIFFE identities
* Foundation for future multi-cluster expansion
