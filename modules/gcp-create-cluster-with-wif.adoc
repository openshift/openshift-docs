= Workload Identity Federation Overview

Workload Identity Federation (WIF) allows OSD on GCP users to replace long-lived service account keys with short-lived (and more secure) tokens. WIF  minimizes the rights granted to third parties such as Red Hat, and ensures all Google Cloud native IAM and best practices for securing Google Cloud services and infrastructure are followed.

The following list provides a basic overview of the Workload Identity Federation process:

* External applications authenticate to the identity provider.
* The external application calls Google Security Token Service to exchange the account credentials for a short-lived Google Cloud access token.
* The token can then be used to impersonate a service account and access Google Cloud resources.

.Prerequisites

* You have an GCP account with the proper permissions to create roles and share resources.
* You installed the link:https://console.redhat.com/openshift/downloads[OpenShift Cluster Manager API command-line interface (`ocm`)].

:FeatureName: OpenShift Cluster Manager API command-line interface ('ocm')
include::snippets/technology-preview.adoc[leveloffset=+1]

You installed the link:https://cloud.google.com/sdk/docs/install[Install the gcloud CLI].

.Procedure

. Register a Workload Identity Federation configuration by running the following command:

[source,terminal]
----
$ ocm create gcp-wif-config --name $WIF_NAME --project $GCP_PROJECT_ID [--auto | --manual]
----

. Confirm the following resources have been created in the GCP console:
** Deployment Service Accounts
** Operator Service Accounts
** Workload Federation Pool
** Support Access

+

. Create a cluster, specifying the name of the registered WIF configuration.

[source,terminal]
----
$ ocm create cluster --cloud gcp --wif-name $WIF_NAME [--project $GCP_PROJECT_ID]
----

[NOTE]
====
If there is more than one `wif_config` with the same name, you must specify the GCP project that the `wif_config` is part of. Multiple `wif_configs` can have the same display name as long as they do not reside in the same project.
====
