= Backing up the backend Redis database

By default, 3scale Redis uses AOF persistence. 

.Prerequisites

. You have backed up the Operator.
. You have backed up the mysql database.
. Ensure that the Redis queues have been drained before performing backups.


.Procedure

. Set the annotations on backend-redis deployment:
+
[source,yaml]
----
annotations:
post.hook.backup.velero.io/command: >-
         ["/bin/bash", "-c", "redis-cli CONFIG SET auto-aof-rewrite-percentage
         100"]
       pre.hook.backup.velero.io/command: >-
         ["/bin/bash", "-c", "redis-cli CONFIG SET auto-aof-rewrite-percentage
         0"]
----

. Backup the Redis database using YAML:
+
[source,terminal]
----
$ oc get backup redis-backup -o yaml
----
+
[source,yaml]
----
apiVersion: velero.io/v1
kind: Backup
metadata:
  labels:
    velero.io/storage-location: ts-dpa-1
  name: redis-backup
  namespace: openshift-adp
  resourceVersion: "81276"
  uid: ce6c6f5c-eb73-4ffe-b2fd-8a3a29e58659
spec:
  csiSnapshotTimeout: 10m0s
  defaultVolumesToFsBackup: true
  includedNamespaces:
  - threescale
  includedResources:
  - deploymentconfigs
  - pods
  - replicationcontrollers
  - persistentvolumes
  - persistentvolumeclaims
  itemOperationTimeout: 1h0m0s
  labelSelector:
    matchLabels:
      app: 3scale-api-management
      threescale_component: backend
      threescale_component_element: redis
  snapshotMoveData: false
  snapshotVolumes: false
  ttl: 720h0m0s
----
