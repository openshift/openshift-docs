= Control permissions with security context constraints

[role="_abstract"]
You can use security context constraints (SCCs) to control permissions for the pods in your node. These permissions determine the actions that a pod can perform and what resources the pod can access. You can use SCCs to define a set of conditions that a pod must run with to be accepted into the system.

[IMPORTANT]
====
{microshift-short} supports only `ReadWriteOnce` (RWO) volume mounts. Pods that do not operate with the SCC contexts might cause blocking of the SCC.
====
