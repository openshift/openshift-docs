= Diagnostics logs for a two-node cluster

[role="_abstract"]
You can collect diagnostic data for Two-Node with Fencing (TNF) clusters when troubleshooting node replacement failures or when opening a Red Hat Support case.

[IMPORTANT]
====
In a TNF cluster, etcd runs as a Podman container managed by pacemaker, not as a Kubernetes pod. Use `podman logs` and `journalctl` to retrieve etcd and pacemaker logs rather than `oc logs`.
====

* Retrieve etcd logs by running the following command:

[source,terminal]
----
$ sudo podman logs etcd
----

* Retrieve etcd-related lines from the pacemaker journal by running the following command:

[source,terminal]
----
$ sudo journalctl -u pacemaker -n 1000 | grep etcd
----

* Retrieve Cluster etcd Operator(CEO) logs by running the following command:

[source,terminal]
----
$ oc logs -n openshift-etcd-operator deployment/etcd-operator --all-containers=true
----

* Retrieve Pacemaker logs by running the following command:

[source,terminal]
----
$ sudo journalctl -u pacemaker -n 500
----

* Retrieve OVN-Kubernetes logs by running the following command:

[source,terminal]
----
$ oc logs -n openshift-ovn-kubernetes <ovnkube_node_pod> -c ovnkube-controller
----

* Retrieve full must-gather collection by running the following command:

[source,terminal]
----
$ oc adm must-gather --dest-dir=./must-gather
----

For TNF-specific debugging, collect pacemaker status and logs from both nodes through SSH in addition to the standard must-gather output.
