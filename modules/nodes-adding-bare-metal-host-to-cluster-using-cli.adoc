= Adding a bare metal host to the cluster using the CLI

You can add bare metal hosts to the cluster and then provision them as bare metal nodes using the CLI.

.Prerequisites

* Install a {op-system} compute machine on bare metal infrastructure for use in the cluster.
* Install the {product-title} CLI (oc).
* Log in as a user with `cluster-admin` privileges.

.Procedure

. Create a bare metal host for use in the cluster by completing the following actions:
.. Create the following `BareMetalHost` Custom Resource (CR) that defines the `Secret` credentials and the required cluster node namespace, for example `openshift-worker`. Save the YAML in the `openshift-worker-namespace.yaml` file.
+
[source,yaml]
----
apiVersion: v1
kind: Secret
metadata:
  name: openshift-worker-bmc-secret
type: Opaque
data:
  username: <secret-username>
  password: <secret-password>
---
apiVersion: metal3.io/v1alpha1
kind: BareMetalHost
metadata:
  name: openshift-worker
spec:
  online: true
  bmc:
    address: <BMC address>
    credentialsName: openshift-worker-bmc-secret
    disableCertificateVerification: True
    username: <username>
    password: <password>
  bootMACAddress: <host boot MAC address>
  hardwareProfile: unknown
  providerSpec:
    value:
      hostSelector: <1>
        matchLabels:
          key: <host label> <2>
----
<1> Optional. Used to restrict automatic `Machine` creation from `MachineSet` objects. When specified, `Machine` objects are created from `MachineSet` objects only when the same label exists on an unused `BareMetalHost` object.
<2> Key value label that denotes the bare metal host type.
.. Create the `openshift-worker-bmc-secret` and `openshift-worker` CR's in the `openshift-machine-api` namespace by running the following command:
+
[source,terminal]
----
$ oc create -f openshift-worker-namespace.yaml -n openshift-machine-api
----
.. Check the status of the new bare metal host instance and ensure it is available in the cluster:
+
[source,terminal]
----
$ oc get bmh -A
----
+
.Example output
+
[source,terminal]
----
NAMESPACE               NAME                   STATE                    CONSUMER                                  ONLINE   ERROR
openshift-machine-api   openshift-master-0-0   externally provisioned   ocp-edge-cluster-0-sfljt-master-0         true
openshift-machine-api   openshift-worker   provisioned              ocp-edge-cluster-0-sfljt-worker-0-sv86d   true
----
. Using the relevant machine set, scale the number of machine replicas to accommodate the new bare metal host and provision it as a node.
+
[source,terminal]
----
$ oc scale --replicas=<number of bare metal hosts> machineset <machine set> -n openshift-machine-api
----
+
[NOTE]
====
If specified in the `BareMetalHost` object, the `hostSelector` field must match a value in the `MachineSet` object for the machine to be created.
====
