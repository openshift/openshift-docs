= Support for Istioctl

{SMProductName} 1 and 2 did not include support for Istioctl, the command line utility for the Istio project that includes many diagnostic and debugging utilities. {SMProduct} 3 introduces support for Istioctl for select commands.

.Supported Istioctl commands
[cols="1,1,1,1"]
|===
|Command |Description |Supported |Alternative

|`admin`, `istiod` | Manage control plane (`istiod`) configuration | Yes | None
|`analyze` | Analyze Istio configuration and print validation messages | Yes | None
|`authz` | `authz` is experimental. Use `istioctl` `experimental` `authz`. |No | None
|`bug-report` | Cluster information and log capture support tool | No | Use `istio-must-gather`.
|`completion` | Generate the autocompletion script for the specified shell | Yes | None
|`create-remote-secret` | Create a secret with credentials to allow Istio to access remote Kubernetes `apiservers` | Yes | None
|`dashboard` | Access to Istio web user interfaces | No | ??
|`experimental, x, exp` | Help about any command | No | None
|`help` | Help about any command | Yes | None
|`install` | Applies an {istio} manifest, installing or reconfiguring Istio on a cluster | No | ??
|`kube-inject` | Inject Istio sidecar into Kubernetes pod resources | No | Set the `istio-injection=enabled` label
|`manifest` | Commands related to Istio manifests | No | None
|`proxy-config`, `pc` | Retrieve information about proxy configuration from Envoy (Kubernetes only) | Yes | None
|`proxy-status`, `ps` | Retrieves the synchronization status of each Envoy in the mesh | Yes | No
|`remote-clusters` | Lists the remote clusters each `istiod` instance is connected to | Yes | No
|`tag` | Command group used to interact with revision tags | No | See `RevisionTag`
|`uninstall` | Uninstall Istio from a cluster | No | ??
|`upgrade` | Upgrade Istio control plane in-place | No | ??
|`validate`, `v` | Validate Istio policy and rules files | Yes | No
|`version` | Prints out build version information | Yes | None
|`waypoint` | Manage waypoint configuration | Yes | None
|`ztunnel-config` | Update or retrieve current Ztunnel configuration. | Yes | None
|===
