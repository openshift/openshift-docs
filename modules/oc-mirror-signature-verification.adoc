= Enabling signature verification for oc-mirror plugin v2

oc-mirror plugin v2 also verifies image signatures by default for enhanced security. If a signature does not match the corresponding image, the mirroring process fails.

* You can enable this behavior for all images by running the following command:
+
[source,terminal]
----
$ oc mirror --secure-policy=true 
----

* To control verification on a per-registry or per-image basis, create a policy configuration file at one of the following locations:

** `$HOME/.config/containers/policy.json`
** `/etc/containers/policy.json`

.Example: Enable verification only for a specific image
+
[source,json]
----
{
  "default": [{"type": "reject"}],
  "transports": {
    "docker": {
      "hostname:5000/myns/sigstore-signed-image": [
        {
          "type": "sigstoreSigned",
          "keyPath": "/path/to/sigstore-pubkey.pub"
        }
      ]
    }
  }
}
----

To use a custom location, specify it with the `--policy` flag.

For more information, see the link:https://github.com/containers/image/blob/main/docs/containers-policy.json.5.md[`containers-policy.json(5)`] manual page.

[IMPORTANT]
====
Starting with {OCP} {ocp-version}, oc-mirror v2 introduces support for signature verification, that is disabled by default. You can  verify the authenticity of the container images. If a signature mismatch is detected, the mirroring workflow will fail.
====
