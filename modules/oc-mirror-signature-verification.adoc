= Enabling signature verification for oc-mirror plugin v2

oc-mirror plugin v2 also verifies image signatures enhanced security. If a signature does not match the corresponding image, the mirroring process fails.

== Default behavior of signature verification

* Signature verification is **disabled** by default in oc-mirror plugin v2. Run the following command to enable the verification for all images:

[source,terminal]
----
$ oc mirror --secure-policy=true
----

When enabled, oc-mirror verifies that container images match their signatures. If a mismatch occurs, the mirroring workflow fails.

== Specific control over signature verification

You can configure signature verification at the transport protocol, registry, namespace, or image level.

* To do so, create or edit the `policy.json` file in one of these locations:

** `$HOME/.config/containers/policy.json`
** `/etc/containers/policy.json`

.Example: Enable verification only for a specific image
+
[source,json]
----
{
  "default": [{"type": "reject"}],
  "transports": {
    "docker": {
      "hostname:5000/myns/sigstore-signed-image": [
        {
          "type": "sigstoreSigned",
          "keyPath": "/path/to/sigstore-pubkey.pub",
          "signedIdentity": {"type": "matchRepository"}
        }
      ]
    }
  }
}
----

* To specify a custom policy file, use the `--policy` flag.

For more information, see link:https://github.com/containers/image/blob/main/docs/containers-policy.json.5.md[`containers-policy.json(5)`].

[IMPORTANT]
====
Starting with {product-title}, oc-mirror plugin v2 supports signature verification, which is disabled by default. You can verify the authenticity of the container images. If a signature mismatch is detected, the mirroring workflow will fail.
