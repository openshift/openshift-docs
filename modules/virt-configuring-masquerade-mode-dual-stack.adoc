= Configuring masquerade mode with IPv6

You can use configure to use IPv6 on the default Pod Network. The IPv6 network address must be statically set in the virtual machine.

Once configured, incoming and outgoing traffic for the virtual machine is routed to the unique IP of the virt-launcher Pod.
The Pod routes the traffic to the statically set IPv6 address of the virtual machine.

This procedure can be applied to new or existing virtual machines in the cluster.

.Prerequisites

* The underlying network for the cluster must be connected with IPv6.

* The OpenShift cluster must be configured to use IPv6.

.Procedure

. Find the IP address of the DNS server by querying the `dns-default` service in the `openshift-dns` namespace:
+
----
$ oc get service -n openshift-dns dns-default -o=custom-columns=IP:.spec.clusterIP
----
+
For this example, the cluster returns an IP address of `172.30.0.10` for the DNS server.

. Edit the virtual machine configuration to add an interface with `masquerade` and configure the IPv6 IP address, default gateway, and DNS record using cloud-init.
+
[source,yaml]
----
apiVersion: kubevirt.io/v1alpha3
kind: VirtualMachine
metadata:
  name: example-vm-ipv6
...
          interfaces:
            - name: red
              masquerade: {} <1>
              ports:
                - port: 80 <2>
      networks:
      - name: red
        pod: {}
      volumes:
      - cloudInitNoCloud:
          userData: |-
            #!/bin/bash
            ip -6 addr add fd2e:f1fe:9490:a8ff::2/120 dev eth0 <3>
            sleep 5
            ip -6 route add default via fd2e:f1fe:9490:a8ff::1 src fd2e:f1fe:9490:a8ff::2 <4>
            echo "nameserver 172.30.0.10" >> /etc/resolv.conf <5>
----
<1> Connect using masquerade mode.
<2> Allows incoming traffic on port 80 to the virtual machine.
<3> This IPv6 address must be used.
<4> This default gateway and IPv6 address must be used.
<5> Adds the DNS record of the DNS server to the virtual machine. The IP address of the DNS server was discovered in the previous step.

. Apply the configuration to the namespace:
+
----
$ oc apply -f example-vm-ipv6.yaml
----
