= Working with user-defined networks

You can enable user-defined networks (UDN) in the Network Observability `FlowCollector` resource, such as shown in the following example.

//placeholder. Update when NOTE in con_user-defined-networks.adoc is also updated/refined.

.Prerequisite
* You have configured UDN in {openshift-networking}.

.Procedure
. Edit the `FlowCollector` resource by running the following command:
+
[source,terminal]
----
$ oc edit flowcollector
----

. Configure the `ebpf` section of the `FlowCollector` resource:
+
[source,yaml]
----
apiVersion: flows.netobserv.io/v1beta2
kind: FlowCollector
metadata:
  name: cluster
spec:
  agent:
    ebpf:
      sampling: 1  <1>
      privileged: true
      features:
      - UDNMapping
----
<1> Recommended so all flows are observed.

.Verify

Refresh the *Network Traffic* page. The *Traffic Flow* and *Topology* views display new information about UDNs:

. In *Network Traffic* > *Traffic flows*, UDNs are viewable under the `SrcK8S_NetworkName` and `DstK8S_NetworkName` fields.
. In the *Topology* view, you can set *Network* as *Scope* or *Group*.
