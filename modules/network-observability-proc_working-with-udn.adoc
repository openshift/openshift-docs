= Working with user-defined networks

You can enable user-defined networks (UDN) in the Network Observability `FlowCollector` resource, such as shown in the following example.

[NOTE]
====
You can enable UDN in Network Observability before you enable UDN in OpenShift Networking.
====
//placeholder. Update when NOTE in con_user-defined-networks.adoc is also updated/refined.

.Procedure
. Edit the `FlowCollector` resource by running the following command:
+
[source,terminal]
----
$ oc edit flowcollector
----

. Configure the `ebpf` section of the `FlowCollector` resource:
+
[source,yaml]
----
apiVersion: flows.netobserv.io/v1beta2
kind: FlowCollector
metadata:
  name: cluster
spec:
  agent:
    ebpf:
      sampling: 1  <1>
      privileged: true
      features:
      - UDNMapping
----
<1> Recommended so all flows are observed.

. Create a new namespace for the user-defined network with the label `k8s.ovn.org/primary-user-defined-network` set to an empty string:
+
[source,yaml]
----
apiVersion: v1
kind: Namespace <1>
metadata:
  name: 49ers
  labels:
    k8s.ovn.org/primary-user-defined-network: ""
----
<1> A new namespace is created because certain labels, like `k8s.ovn.org/primary-user-defined-network`, cannot be added to an existing namespace.

. Apply the new namespace by running the following command:
+
[source,terminal]
----
$ oc apply -f <namespace.yaml>
----

. Create a `UserDefinedNetwork` instance:
+
[source,yaml]
----
apiVersion: k8s.ovn.org/v1
kind: UserDefinedNetwork
metadata:
  name: udn-49ers
  namespace: 49ers
spec:
  topology: Layer2
  layer2:
    role: Primary
    subnets:
      - "10.0.0.0/24"
----

. Apply the `UserDefinedNetwork` instance:
+
[source,terminal]
----
$ oc apply -f <udn_instance.yaml>
----

. Add a pod into the namespace by running the following command:
[source,terminal]
----
oc apply -f <path_to_pod_definition.yaml> --namespace=<your_namespace>
----

.Verify

A secondary interface is automatically part of the UDN when you add a pod into the namespace.

. Confirm the secondary interface is part of the UDN by running the following commands:
+
[source,terminal]
----
$ oc project 49ers
pod=$(oc get --no-headers pods | awk '{ print $1;}') <1>
----
+
[source,terminal]
----
$ oc describe pod/$pod  <2>
----
<1> Get pod name.
<2> Two interfaces are mentioned in *Annotations*.

All pods in this namespace are isolated from pods in other namespaces. In essence, a namespace can be a tenant.
