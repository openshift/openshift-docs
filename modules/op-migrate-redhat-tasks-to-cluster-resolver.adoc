= Migrate Red Hat provided tasks to the cluster resolver

[role="_abstract"]
{pipelines-title} installs Red Hat provided tasks as namespace-scoped `Task` resources in the {pipelines-shortname} installation namespace, `openshift-pipelines`. To use these tasks in your pipelines, update the `taskRef` field to use the cluster resolver.

.Prerequisites

* You have identified the pipelines that reference `ClusterTask` resources.
* You have the necessary permissions to edit pipeline resources.

.Procedure

. List the available Red Hat provided tasks in the {pipelines-shortname} installation namespace by running the following command:
+
[source,terminal]
----
$ oc get tasks -n openshift-pipelines
----
+
--
*Example output:*
[source,terminal]
----
NAME                 AGE
buildah              10d
git-cli              10d
git-clone            10d
maven                10d
openshift-client     10d
s2i-java             10d
skopeo-copy          10d
tkn                  10d
----
--

. Verify that the task you want to use is available by running the following command:
+
[source,terminal]
----
$ oc get task buildah -n openshift-pipelines
----

. Update the `taskRef` definition in your pipeline to use the cluster resolver.
+
--
*Before migration:*

[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: build-and-deploy
spec:
  tasks:
  - name: build-image
    taskRef:
      name: buildah
      kind: ClusterTask
    params:
    - name: IMAGE
      value: $(params.IMAGE_NAME)
    workspaces:
    - name: source
      workspace: shared-workspace
----

*After migration:*

[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: build-and-deploy
spec:
  tasks:
  - name: build-image
    taskRef:
      resolver: cluster
      params:
      - name: kind
        value: task
      - name: name
        value: buildah
      - name: namespace
        value: openshift-pipelines
    params:
    - name: IMAGE
      value: $(params.IMAGE_NAME)
    workspaces:
    - name: source
      workspace: shared-workspace
----
--
+
--
where:

`resolver`:: Specifies the cluster resolver to access tasks from the same cluster.
`value`:: Specifies the default namespace where the task is located.
--

. Apply the updated pipeline definition by running the following command:
+
[source,terminal]
----
$ oc apply -f my-pipeline.yaml
----
+
[TIP]
====
When migrating to the cluster resolver, you only need to change the `taskRef` section. Task parameters, workspaces, and results remain unchanged.
====

.Verification

. Create a pipeline run to verify the migration by running the following command:
+
[source,terminal]
----
$ tkn pipeline start build-and-deploy --showlog
----

. Verify that the pipeline run completes successfully by running the following command:
+
[source,terminal]
----
$ tkn pipelinerun logs <pipelinerun-name> -f
----
