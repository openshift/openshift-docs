= Creating custom templates with the {sco}

Create user-defined templates with the {sco}.
This example shows how you can create templates for hosted control planes.

.Prerequisites
// Any?

.Procedure

. Create a YAML file, named `siteconfig-hcp-cluster-level.yaml`, that contains the cluster-level templates for hosted control planes:
+
--
include::snippets/siteconfig-hcp-cluster-level.adoc[]
--

. Create a YAML file, named `siteconfig-hcp-node-level.yaml`, that contains the node-level templates for hosted control planes:
+
--
include::snippets/siteconfig-hcp-node-level.adoc[]
--

. Create the cluster- and node-level templates on the hub cluster by running the following commands:
+
--
[source,terminal]
----
$ oc apply -f siteconfig-hcp-cluster-level.yaml
----

[source,terminal]
----
$ oc apply -f siteconfig-hcp-node-level.yaml
----
--

. Reference your node- and cluster-level templates in the `ClusterInstance` CR, named for example `clusterinstance-hcp.yaml`:
+
[source,yaml]
----
spec:
  [...]
  nodes:
    templateRefs: <1>
      - name: hcp-node-templates-v1.yaml
        namespace: siteconfig-system
    [...]
  templateRefs: <2>
    - name: hcp-cluster-templates-v1.yaml
      namespace: siteconfig-system
----
<1> Reference node-level templates under the `spec.nodes.templateRefs` field.
<2> Reference cluster-level templates under the `spec.templateRefs` field.

. Create the `ClusterInstance` CR by running the following command:
+
[source,terminal]
----
$ oc apply -f clusterinstance-hcp.yaml
----
