= Tune the rate limit

[role="_abstract"]
You can tune the rate limit when creating and maintaining a large number of virtual machines (VMs) in one cluster.

The `hco.kubevirt.io/tuningPolicy` annotation is used to configure the rate limit. The annotation is only taken into account if the
`annotation` tuning policy is applied to the HyperConverged CR.

.Prerequisites

* You have installed the {oc-first}.

.Procedure
. Annotate the HyperConverged CR with the `hco.kubevirt.io/tuningPolicy` annotation, and a value of a json object with the following fields:
    - `qps`: indicates the maximum QPS to the apiserver from this client.
    - `burst`: Maximum burst for throttle.

+
For example, to set the `qps` to 100 and the `burst` to 200, use the following command:

+
[source,terminal,subs="attributes+"]
----
$ oc annotate {HCOCliKind} kubevirt-hyperconverged -n {CNVNamespace} 'hco.kubevirt.io/tuningPolicy={"qps": 100, "burst": 200}'
----

. Apply the following patch to enable the `annotation` tuning policy:
+
[source,terminal,subs="attributes+"]
----
$ oc patch {HCOCliKind} kubevirt-hyperconverged -n {CNVNamespace} \
  --type=json -p='[{"op": "add", "path": "/spec/virtualization/tuningPolicy", \
  "value": "annotation"}]'
----

.Verification

* Run the following command to verify the rate limit tuning:
+
[source,terminal,subs="attributes+"]
----
$ oc get kubevirt.kubevirt.io/kubevirt-kubevirt-hyperconverged \
  -n {CNVNamespace} -o go-template --template='{{range $config, \
  $value := .spec.configuration}} {{if eq $config "apiConfiguration" \
  "webhookConfiguration" "controllerConfiguration" "handlerConfiguration"}} \
  {{"\n"}} {{$config}} = {{$value}} {{end}} {{end}} {{"\n"}}
----
