= About {ols-short} and PostgressSQL persistence 

Using PostgressSQL persistence you can save the {ols-long} conversation history to a PostgresSQL database so that the conversation history is retained in persistent volume. With PostgressSQL persistence, the conversation history persists if the PostgreSQL pod restarts or is rescheduled in your {ocp-product-title} installation. 

:FeatureName: PostgressSQL persistence
include::snippets/technology-preview.adoc[]

PostgresSQL persistence is disabled by default. To enable the functionality, define the `size` and `class` parameters in the `ols.storage` specification of the `OLSConfig` CR.
