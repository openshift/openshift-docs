= Limitations of live updating a virtual machine network

[role="_abstract"]
You must consider the following limitations before hot swapping a virtual machine (VM) secondary network interface.

* This feature only works for VMs that are configured for live migration.
* A brief network interruption can occur during the live migration process.
* If you update multiple VMs, the updates might be queued because only limited number of live migrations can run in parallel across the cluster.
* You cannot hot swap to a new network binding type or plugin. Even if it uses the same CNI, there is not guarantee that the NAD configuration will work, and so the VM's network may be broken after migration. We rely on the admin to supply a correct config, we won't automatically revert the change or detect a connectivity issue. We would however detect a major config problem, e.g. if a referenced resource is unavailable and CNI fails to attach the `virt-launcher` pod.
* This is only for secondary networks, this won't allow change from e.g. localnet to primary UDN
