= Configuring the `DataProtectionApplication` for a non-admin CSI backup

As a cluster administrator, to enable a Container Storage Interface (CSI) backup for a non-admin user, you can configure the `DataProtectionApplication` (DPA) as shown in the following example.

.Prerequisites

* You are logged in to the cluster with administrator privileges.
* You have installed the {oadp-short} Operator.

.Procedure

. To enable {oadp-short} Self-Service and CSI backup, edit the DPA and:
.. Add the `nonAdmin.enable` section.
.. Set the `nonAdmin.enable` field to `true`.
.. Add the `csi` plugin in the `velero.defaultPlugins` section.
+
.Example `DataProtectionApplication`
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: DataProtectionApplication
metadata:
  name: test-dpa
  namespace: openshift-adp
spec:
  backupLocations:
  - velero:
      credential:
        key: cloud
        name: cloud-credentials # <1>
      default: true
      objectStorage:
        bucket: <bucket_name> # <2>
        prefix: velero
      provider: gcp
  configuration:
    velero:
      defaultPlugins:
      - gcp
      - openshift
      - csi # <3>
  nonAdmin: # <4>
    enable: true # <5>
----
1. Specify the `cloud-credentials` file for your cloud provider. In this example, the cloud provider is `gcp`.
2. Specify the bucket name.
3. Add the `csi` plugin as shown in the example.
4. Add `nonAdmin.enable` section under the `spec` section of the DPA as shown in the example.
5. Set the `enable` field to `true`.
