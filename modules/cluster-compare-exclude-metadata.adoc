= Configuring the metadata.yaml file to exclude template fields

[role="_abstract"]
To prevent irrelevant data from affecting your comparison, configure the `metadata.yaml` file to exclude specific template fields. Omitting inconsequential items, such as annotations or labels, ensures that your evaluation remains focused on critical cluster configurations.

You can configure exclusions in the `metadata.yaml` file in the following ways:

* Exclude all fields in a custom resource not specified in a template.
* Exclude specific fields that you define using the `pathToKey` field.

[NOTE]
====
`pathToKey` is a dot separated path. Use quotes to escape key values featuring a period.
====

.Procedure

* To exclude all fields not specified in a template, create a `metadata.yaml` file to match your use case. Use the following structure as an example:
+
[source,yaml]
----
apiVersion: v2
parts:
  - name: Part1
    components:
      - name: Namespace
        allOf:
          - path: namespace.yaml
            config:
              ignore-unspecified-fields: true <1>
#...
----
** `ignore-unspecified-fields`: Specifies `true` to exclude from the comparison all fields in a CR that are not explicitly configured in the corresponding `namespace.yaml` template.
+
During the comparison process, the `cluster-compare` plugin renders a template by merging fields from the corresponding custom resource (CR). If you configure the `ignore-unspecified-fields` to `true`, all fields that are present in the CR, but not in the template, are excluded from the merge. Use this approach when you want to focus the comparison on the fields specified in the template only.

* To exclude specific fields by setting default exclusion fields, ]create a `metadata.yaml` file to match your use case. Use the following structure as an example:
+
[source,yaml]
----
apiVersion: v2
parts:

#...

fieldsToOmit:
   defaultOmitRef: default
   items:
      default:
         - pathToKey: a.custom.default."k8s.io"
# ...
----
** `fieldsToOmit.defaultOmitRef`: Specifies the default exclusion for all templates, unless overridden by the `config.fieldsToOmitRefs` field for a specific template.
** `fieldsToOmit.items.default.pathToKey`: Specifies the value is excluded for all templates.
+
You can exclude fields by defining a default value for `fieldsToOmitRefs` in the `defaultOmitRef` field. This default exclusion applies to all templates, unless overridden by the `config.fieldsToOmitRefs` field for a specific template.

* To exclude specific fields, create a `metadata.yaml` file to match your use case. Use the following structure as an example:
+
[source,yaml]
----
apiVersion: v2
parts:
  - name: Part1
    components:
      - name: Component1
        - path: deployment.yaml
          config:
            fieldsToOmitRefs:
              - deployments

#...

fieldsToOmit:
   items:
      deployments:
         - pathToKey: spec.selector.matchLabels.k8s-app
----
** `name.path.config.fieldsToOmitRefs.deployments`: Specifies a reference to the `fieldsToOmit.items.deployments` item for the `deployment.yaml` template.
** `fieldsToOmit.items.deployments.pathToKey`: Specifies to exclude the `spec.selector.matchLabels.k8s-app` field from the comparison. 
+
[NOTE]
====
Setting `fieldsToOmitRefs` replaces the default value.
====
+
You can specify fields to exclude by defining the path to the field, and then referencing the definition in the `config` section for a template. 

* To exclude specific fields by setting default exclusion groups, create a `metadata.yaml` file to match your use case. Use the following structure as an example:
+
[source,yaml]
----
apiVersion: v2
parts:

#...

fieldsToOmit:
   defaultOmitRef: default
   items:
    common:
      - pathToKey: metadata.annotations."kubernetes.io/metadata.name"
      - pathToKey: metadata.annotations."kubernetes.io/metadata.name"
      - pathToKey: metadata.annotations."kubectl.kubernetes.io/last-applied-configuration"
      - pathToKey: metadata.creationTimestamp
      - pathToKey: metadata.generation
      - pathToKey: spec.ownerReferences
      - pathToKey: metadata.ownerReferences
    default:
      - include: common
      - pathToKey: status
# ...
----
** `fieldsToOmit.default.include`: Specifies the `common` group that is included in the default group.
+
You can create default groups of fields to exclude. A group of exclusions can reference another group to avoid duplication when defining exclusions.
