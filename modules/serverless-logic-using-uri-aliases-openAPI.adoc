= Using URI alias

As an alternative to the `kogito.sw.operationIdStrategy`, you can assign an alias to a URI by using the `workflow-uri-definitions` custom extension. This alias simplifies the configuration process and can be used as a configuration key in REST client settings and function definitions.

The `workflow-uri-definitions` extension allows you to map a URI to an alias, which can be referenced throughout the workflow and in your configuration files. This approach provides a centralized way to manage URIs and their configurations.

.Prerequisites

* You have created a {ServerlessLogicProductName} project or have access to a project with the appropriate roles and permissions to create applications and other workloads in OpenShift Container Platform.
* You have created your {ServerlessLogicProductName} project.
* You have access to the OpenAPI specification files.

.Procedure

. Add the `workflow-uri-definitions` extension to your workflow. Within this extension, create aliases for your URIs.
+
.Example workflow
[source,json]
----
{
  "extensions": [
    {
      "extensionid": "workflow-uri-definitions",
      "definitions": {
        "remoteCatalog": "https://my.remote.host/apicatalog/apis/123/document"
      }
    }
  ],
  "functions": [
    {
      "name": "operation1",
      "operation": "remoteCatalog#operation1"
    },
    {
      "name": "operation2",
      "operation": "remoteCatalog#operation2"
    }
  ]
}
----

* Extension ID: `workflow-uri-definitions`
* Alias definition: `remoteCatalog` is mapped to `https://my.remote.host/apicatalog/apis/123/document`
* Function operations: Use the alias `remoteCatalog` with operation identifiers (`operation1`, `operation2`).

. Use the alias defined in the workflow as the configuration key in the `application.properties` file for the REST client configuration.
+
.Example property
[source,sh]
----
quarkus.rest-client.remoteCatalog.url=http://localhost:8282/
----
* Configuration key: `quarkus.rest-client.remoteCatalog.url`
* URL: Set to `http://localhost:8282/` which will be used by REST clients referring to `remoteCatalog`.

. In your workflow, use the alias when defining functions that operate on the URI.
+
.Example Workflow (Continued):
[source,json]
----
{
  "functions": [
    {
      "name": "operation1",
      "operation": "remoteCatalog#operation1"
    },
    {
      "name": "operation2",
      "operation": "remoteCatalog#operation2"
    }
  ]
}
----
