= Configure Whereabouts for IP addressing

[role="_abstract"]
Configure an SR-IOV network to use Whereabouts for automatic IP address assignment from a pool managed by {product-title}.

.Procedure

. Create an SR-IOV network with Whereabouts IPAM configuration:
+
[source,yaml]
----
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetwork
metadata:
  name: sriov-network-whereabouts
  namespace: openshift-sriov-network-operator
spec:
  resourceName: intelnics
  networkNamespace: default
  vlan: 0
  ipam: |
    {
      "type": "whereabouts",
      "range": "192.168.1.0/24",       
      "range_start": "192.168.1.10",   
      "range_end": "192.168.1.100",    
      "gateway": "192.168.1.1"         
    }
----
 IP address range in CIDR notation. Defines the overall network.
 First IP address in the allocation pool. Optional - defaults to first address in range.
 Last IP address in the allocation pool. Optional - defaults to last address in range.
 Gateway IP address. Optional.

. Create the SR-IOV network:
+
[source,terminal]
----
$ oc create -f sriov-network-whereabouts.yaml
----

. Create a pod that uses the Whereabouts network:
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: pod-whereabouts
  namespace: default
  annotations:
    k8s.v1.cni.cncf.io/networks: sriov-network-whereabouts  
spec:
  containers:
  - name: test
    image: registry.access.redhat.com/ubi9/ubi:latest
    command: ["/bin/bash", "-c", "sleep infinity"]
    resources:
      requests:
        openshift.io/intelnics: "1"
      limits:
        openshift.io/intelnics: "1"
----
 Reference to the SR-IOV network. Whereabouts automatically assigns an IP address from the configured pool.

. Create the pod:
+
[source,terminal]
----
$ oc create -f pod-whereabouts.yaml
----

. Create additional pods to see automatic IP allocation:
+
[source,terminal]
----
$ oc create -f pod-whereabouts.yaml --dry-run=client -o yaml | \
  sed 's/pod-whereabouts/pod-whereabouts-2/' | oc create -f -
----
+
Each pod receives the next available IP address from the pool.

.Verification

. Verify the pod received an IP address from Whereabouts:
+
[source,terminal]
----
$ oc exec -it pod-whereabouts -- ip addr show net1
----
+
The output should show an IP address from the range `192.168.1.10` to `192.168.1.100`.

. View the IP address allocations managed by Whereabouts:
+
[source,terminal]
----
$ oc get ippools.whereabouts.cni.cncf.io -A
----
+
This shows which IP addresses are currently allocated.

. Verify that deleting a pod returns its IP address to the pool:
+
[source,terminal]
----
$ oc delete pod pod-whereabouts
----
+
[source,terminal]
----
$ oc get ippools.whereabouts.cni.cncf.io -A
----
+
The IP address that was allocated to `pod-whereabouts` should no longer appear in the allocations.

.Troubleshooting

If pods do not receive IP addresses from Whereabouts:

* Verify the IP pool has available addresses:
+
[source,terminal]
----
$ oc get ippools.whereabouts.cni.cncf.io -A -o yaml
----

* Check that the range is large enough for your number of pods
* Ensure the range does not conflict with other IP address assignments on the network
* Verify the Multus and Whereabouts CNI plugins are running:
+
[source,terminal]
----
$ oc get pods -n openshift-multus
----

[IMPORTANT]
====
When configuring Whereabouts:

* Plan IP address ranges to avoid conflicts with other systems on the physical network
* Ensure the range is large enough for your expected number of pods
* Use non-overlapping ranges if you have multiple SR-IOV networks
* The range should not include the gateway IP address
* Reserve IP addresses for network infrastructure (switches, routers, etc.)
====

[NOTE]
====
You can configure additional Whereabouts options for advanced use cases:

.Advanced Whereabouts IPAM configuration example
[source,yaml]
----
ipam: |
  {
    "type": "whereabouts",
    "range": "192.168.1.0/24",
    "range_start": "192.168.1.10",
    "range_end": "192.168.1.100",
    "gateway": "192.168.1.1",
    "routes": [{                    
      "dst": "10.0.0.0/8",
      "gw": "192.168.1.254"
    }],
    "exclude": [                    
      "192.168.1.50/32",
      "192.168.1.60/32"
    ]
  }
----

* `routes`: Optional static routes to configure on the interface.
* `exclude`: Optional IP addresses to exclude from automatic allocation. Use this to reserve specific addresses.
====
