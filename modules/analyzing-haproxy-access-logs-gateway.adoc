= Analyzing HAProxy access logs for the Gateway API

[role="_abstract"]
Analyze HAProxy access logs to troubleshoot traffic routing and identify dropped requests or connection errors. Interpreting these logs helps you ensure that your Gateway API configuration correctly balances incoming traffic.

.Prerequisites
* You have installed the {oc-first}.
* You have access to the {product-title} cluster with `cluster-admin` privileges.

.Procedure

. Identify the router pods running in the `openshift-ingress` project by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-ingress
----

. View the HAProxy access logs for a specific router pod by running the following command:
+
[source,terminal]
----
$ oc logs <router_pod_name> -c router -n openshift-ingress
----

. Review the log output to identify HTTP status codes, connection termination states, and backend routing details.
