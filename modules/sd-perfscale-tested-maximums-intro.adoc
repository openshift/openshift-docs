= Performance and Scale tested cluster maximums

[role="_abstract"]
The following content summarizes Red{nbsp}Hat Performance and Scale testing used to publish tested cluster maximums for managed offerings.

[NOTE]
====
The following tables document *tested* limits from Red{nbsp}Hat Performance and Scale testing. They are not absolute maximums for every deployment.

* Tests used default {product-title} configurations without additional tuning unless noted.
* Recorded values reflect limits of individual resources measured with separate workloads, each stressing a subset of object types; workloads do not replicate full production traffic but are aligned with common customer patterns.
* Tests used the `kube-burner` workload orchestrator.
* Reaching or exceeding a value does not necessarily mean the cluster will fail; staying within a value does not guarantee performance.
====

Red{nbsp}Hat documents tested cluster maximums rather than a single scale ceiling for all environments. Not every combination of version, cloud platform, instance type, and workload is tested. It might not be possible to reach every maximum in the tables at the same time.
