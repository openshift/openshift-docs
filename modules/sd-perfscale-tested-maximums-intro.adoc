= Performance and scale tested cluster maximums

[role="_abstract"]
The following content summarizes Red{nbsp}Hat Performance and scale testing used to publish tested cluster maximums for managed offerings.

[NOTE]
====
The following tables document *tested* limits from Red{nbsp}Hat Performance and scale testing. They are not absolute maximums for every deployment.

* The tests use default {product-title} configurations without additional tuning unless noted.
* The tests record limits for individual resources by using separate workloads; each workload stresses a subset of object types. The workloads do not replicate full production traffic but align with common customer patterns.
* The tests use the `kube-burner` workload orchestrator.
* Reaching or exceeding a tested value does not always mean that the cluster will fail.
* Staying within a tested value does not guarantee performance.
====

Red{nbsp}Hat documents tested cluster maximums rather than a single scale ceiling for all environments. Not every combination of version, cloud platform, instance type, and workload is tested. You might not be able to reach every maximum in the tables at the same time.
