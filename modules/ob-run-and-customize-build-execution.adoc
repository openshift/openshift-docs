= Run and customize a build execution

[role="_abstract"]
Run and control individual build runs by configuring runtime parameters, service accounts, environment variables, and pod scheduling. A `BuildRun` custom resource (CR) triggers a build and overrides build-time settings for specific execution needs.

Build execution configuration includes:

* Buildrun pod configuration: Node selection and resource constraints specific to this build run
* Volumes definition for a buildrun: Custom storage mounts that replace or supplement volumes defined in the build strategy
* Parameter values definition for a BuildRun: Runtime values that supersede the defaults defined in the build strategy or `Build` resource
* BuildRun retention parameters: Automatic cleanup rules for completed build runs based on success or failure

These settings give flexibility to adapt builds to different environments, credentials, or resource requirements without modifying the base `Build` resource. The build controller merges `BuildRun` settings with `Build` definitions, giving precedence to runtime overrides.
