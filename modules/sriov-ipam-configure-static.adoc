= Configure static IP addressing

[role="_abstract"]
Configure an SR-IOV network to use static IP addresses that you specify in pod annotations.

.Procedure

. Create an SR-IOV network without IPAM configuration:
+
[source,yaml]
----
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetwork
metadata:
  name: sriov-network-static
  namespace: openshift-sriov-network-operator
spec:
  resourceName: intelnics
  networkNamespace: default
  vlan: 0
----
+
When no IPAM configuration is specified, pods must provide IP addresses in their network annotations.

. Create the SR-IOV network:
+
[source,terminal]
----
$ oc create -f sriov-network-static.yaml
----

. Create a pod with a static IP address:
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: pod-static-ip
  namespace: default
  annotations:
    k8s.v1.cni.cncf.io/networks: |
      [{
        "name": "sriov-network-static",
        "ips": ["192.168.1.10/24"],  
        "gateway": ["192.168.1.1"]    
      }]
spec:
  containers:
  - name: test
    image: registry.access.redhat.com/ubi9/ubi:latest
    command: ["/bin/bash", "-c", "sleep infinity"]
    resources:
      requests:
        openshift.io/intelnics: "1"
      limits:
        openshift.io/intelnics: "1"
----
 Static IP address and subnet for the SR-IOV interface. Replace with an appropriate IP for your network.
 Optional gateway address for the SR-IOV network.

. Create the pod:
+
[source,terminal]
----
$ oc create -f pod-static-ip.yaml
----

.Verification

. Verify the pod received the static IP address:
+
[source,terminal]
----
$ oc exec -it pod-static-ip -- ip addr show net1
----
+
The output should show `192.168.1.10/24` configured on the `net1` interface.

[IMPORTANT]
====
When using static IP addresses:

* Ensure each pod has a unique IP address to avoid conflicts
* Use IP addresses from a range that does not conflict with DHCP or other systems
* Document your IP address assignments
* Consider migrating to DHCP or Whereabouts if you have more than 10-20 pods
====
