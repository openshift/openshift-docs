= Providing secrets using workspaces

[role="_abstract"]
You can use workspaces to pass secrets for authentication with Git repositories and container repositories.

You can configure a named workspace in a task, specifying a path where {pipelines-shortname} mounts the workspace. When you run the task, use the secret as the workspace with this name. When {pipelines-shortname} executes the task, the task can access the information in the secret.

If you pass authentication secrets by using workspaces, you do not need annotations for the secrets.
