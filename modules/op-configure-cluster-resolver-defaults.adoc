= Configure cluster resolver defaults

[role="_abstract"]
You can configure default values for the cluster resolver to simplify task references in your pipelines. This is optional but can reduce repetitive configuration when most of your tasks come from a single namespace.

.Prerequisites

* You have cluster administrator permissions to edit the `ConfigMap` in the {pipelines-shortname} installation namespace.

.Procedure

. Edit the cluster resolver configuration by running the following command:
+
[source,terminal]
----
$ oc edit configmap cluster-resolver-config -n openshift-pipelines
----

. Configure default values for the cluster resolver:
+
[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: cluster-resolver-config
  namespace: openshift-pipelines
data:
  default-kind: "task"
  default-namespace: "openshift-pipelines"
----
`data.default-kind`:: Sets the default resource kind to `task`. This means you do not need to specify `kind` in every cluster resolver reference if you are referencing tasks.
`data.default-namespace`:: Sets the default namespace to `openshift-pipelines`. This means you do not need to specify the `namespace` parameter if your tasks are in this namespace.

. Save the changes to the `ConfigMap`.

. After configuring defaults, you can simplify task references in your pipelines:
+
--
*Without defaults:*

[source,yaml]
----
taskRef:
  resolver: cluster
  params:
  - name: kind
    value: task
  - name: name
    value: buildah
  - name: namespace
    value: openshift-pipelines
----

*With defaults configured:*

[source,yaml]
----
taskRef:
  resolver: cluster
  params:
  - name: name
    value: buildah
----
--

. If you need to override the defaults for a specific task reference, you can still provide explicit values:
+
[source,yaml]
----
taskRef:
  resolver: cluster
  params:
  - name: name
    value: my-custom-task
  - name: namespace
    value: my-shared-tasks
----
`taskRef.value`:: Specifies a different namespace, overriding the default.
+
[IMPORTANT]
====
Setting defaults affects all pipelines that use the cluster resolver in the cluster. Ensure that the default namespace contains the tasks that most of your pipelines use.
====

.Verification

. Create a test pipeline that uses the simplified syntax:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: test-defaults
spec:
  tasks:
  - name: test-task
    taskRef:
      resolver: cluster
      params:
      - name: name
        value: buildah
----

. Run the following command to verify that the cluster resolver correctly to apply the defaults:
+
[source,terminal]
----
$ tkn pipeline start test-defaults --showlog
----

. Check the pipeline run details to confirm the task was resolved correctly by running the following command:
+
[source,terminal]
----
$ tkn pipelinerun describe <pipelinerun_name>
----
+
`<pipelinerun_name>`:: Specifies the name of the pipeline run.
