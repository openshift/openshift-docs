= {mtc-full} 1.8.7 release notes

[id="resolved-issues-1-8-7_{context}"]
== Resolved issues

{mtc-first} 1.8.7 has the following major resolved issues:

.{mtc-short} migration stuck in `Prepare` phase on {product-title} 4.19 due to incompatible {oadp-short} version and outdated DPA specification

When running migrations using {mtc-short} 1.8.7 on {product-title} 4.19, the process becomes stuck in the `Prepare` phase with the migration plan marked as `Suspended`, despite no completed migration. 

The root cause is the deployment of an incompatible {oadp-short} version, less than 1.5.0, whose DataProtectionApplication (DPA) specification format is incompatible with {product-title}. link:https://issues.redhat.com/browse/MIG-1735[(MIG-1735)]

.Velero backup fails with a `file already closed` error when using the new {aws-short} plugin in {mtc-short}

During stage or full migrations using {mtc-short} with {oadp-short} on {product-title} clusters configured with the new {aws-first} plugin, the backup process intermittently fails with the following error in Velero logs:

[source,terminal]
----
error="read |0: file already closed"
----

The workaround is to switch to using the legacy {aws-short} plugin, which resolves the issue. This requires:

. Setting `velero_use_legacy_aws: true` in the `MigrationController` custom resource (CR).

. Restarting the {mtc-short} operator to apply changes.

. Validate the {aws-short} credentials for the cloud-credentials secret.

link:https://issues.redhat.com/browse/MIG-1738[(MIG-1738)]
