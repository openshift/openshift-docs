= Configuring network attachment definitions (NADs) for SR-IOV VFs.

After the SR-IOV Network Operator exposes the SR-IOV devices as resources, you use NetworkAttachment Definitions (NADs) to enable application pods to declaratively access the VFs.

.Prerequisites

* You have installed {microshift-short}.
* You have installed the `microshift-sriov` RPM package.
* Your physical network hardware supports SR-IOV.
* The SR-IOV Network Operator is successfully installed and running.
* Obtain the resource name exposed by the Operator (for example, `intel.com/nic_resource`).

.Procedure

. Define a NAD custom resource (CR) that specifies the SR-IOV CNI provider and the parameters for the network interface.
+
[source,yaml]
----
apiVersion: k8s.cni.cncf.io/v1
kind: NetworkAttachmentDefinition
metadata:
  name: sriov-network-attachment
spec:
  config: '{
    "cniVersion": "0.3.1",
    "type": "sriov",
    "resourceName": "vendor.com/device_resource_name",
    ...
  }'
----
. In your application deployment, annotate the pod to use the NAD that was created in the previous step and specify the required network resource in the `resources` section of the container specification.
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: my-sriov-app
  annotations:
    k8s.v1.cni.cncf.io/networks: sriov-network-attachment
spec:
  containers:
  - name: my-container
    image: my-image
    resources:
      limits:
        vendor.com/device_resource_name: "1"  # Requesting one VF
----

.Verification

Verify that the pod transitions to the `Running` state and inspect the network configuration for the pod to confirm that the expected VF interface has been successfully attached.
