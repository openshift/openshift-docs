= Ensuring that layered products are compatible with the upgrade

It is important to make sure all layered products run on the version of {product-title} that you upgrade to.
This generally includes all Operators.

.Procedure
* Before an upgrade, verify the currently installed Operators in the cluster.
For example, run the following command:
+
[source,terminal]
----
$ oc get csv -A
----
+
.Example output
[source,terminal]
----
NAMESPACE                              NAME            DISPLAY          VERSION   REPLACES                             PHASE
gitlab-operator-kubernetes.v0.17.2     GitLab                           0.17.2    gitlab-operator-kubernetes.v0.17.1   Succeeded
openshift-operator-lifecycle-manager   packageserver   Package Server   0.19.0                                         Succeeded
----

[id="telco-upgrade-check-olm-operator-compatibility-with-the-upgrade-version_{context}"]
== Check OLM Operator compatibility with the upgrade version

Operators that are installed with the Operator Lifecycle Manager (OLM) are not part of the standard cluster Operators set.

.Procedure
* Use the link:https://access.redhat.com/labs/ocpouic/?upgrade_path=4.14%20to%204.16[Operator Update Information Checker] to understand if you need to upgrade an Operator after each Y-Stream upgrade or if you can wait until you have fully upgraded to the next EUS release.
+
[TIP]
====
You can also use the link:https://access.redhat.com/labs/ocpouic/?upgrade_path=4.14%20to%204.16[Operator Update Information Checker] to see what versions of {product-title} are compatible with specific releases of an Operator.
====

[id="telco-upgrade-check-other-operators-for-compatibility-with-the-upgrade-version_{context}"]
== Check non-OLM Operators for compatibility with the upgrade version

For all OLM installed Operators that are not directly supported by Red Hat, contact the Operator vendor to ensure release compatibility.

* Some Operators are compatible with several releases of {product-title}.
You might not need to upgrade the Operators until after you complete the cluster upgrade.
See "Upgrading the worker nodes" for more information.

* See "Upgrading the OLM Operators" for information about upgrading an Operator after performing the first y-stream control plane upgrade.
