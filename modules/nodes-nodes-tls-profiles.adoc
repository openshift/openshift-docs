= About TLS profiles for Kubelet

When defining TLS profile for Kubelet, you use the `tlsSecurityProfile` parameter in a `KubeletConfig` custom resource (CR) to specify a predefined profile or define a custom TLS profile.

The default TLS policy is the  are as follows:

[source,yaml]
----
kind: KubeletConfiguration
apiVersion: kubelet.config.k8s.io/v1beta1
 ...
tlsCipherSuites:
  - TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256
  - TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
  - TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384
  - TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
  - TLS_ECDHE_ECDSA_WITH_CHACHA20_POLY1305_SHA256
  - TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305_SHA256
----

There are three TLS security profile configurations that you can specify:

* `Old`. This is a predefined profile intended for use with legacy clients or libraries. This profile is based on link:https://wiki.mozilla.org/Security/Server_Side_TLS#Recommended_configurations[recommended configurations].
+
.Sample `KubeletConfig` CR for the `Old` TLS profile
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Old
----

* `Intermediate`. This is a predefined profile that is the default profile for the Kubelet. This profile is based on link:https://wiki.mozilla.org/Security/Server_Side_TLS#Recommended_configurations[recommended configurations].
+
.Sample `KubeletConfig` CR for the `Intermediate` TLS profile
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Intermediate
----

* `Modern` Use with modern clients that support TLS 1.3 with no need for backwards compatibility.
+
.Sample `Modern` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Modern
----
////

* `Custom`. This allows you to define the TLS ciphers and versions to use.
+
[WARNING]
====
Use caution when using a `Custom` profile, because invalid configurations can cause problems.
====
+
.Sample `KubeletConfig` CR for a `Custom` TLS profile
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Custom
    custom:
      ciphers: <1>
        - ECDHE-ECDSA-AES128-GCM-SHA256
        - ECDHE-RSA-AES128-GCM-SHA256
      minTLSVersion: VersionTLS11 <2>
----
+
<1> A prioritized list of TLS ciphers.
<2> The minimum accepted TLS version:

You can see the current TLS ciphers being used by viewing the `/etc/kubernetes/kubelet.conf` file on each node.

.Sample `kubelet.conf` file with the `Intermediate` configuration

[source,json]
----
{
  "kind": "KubeletConfiguration",
  "apiVersion": "kubelet.config.k8s.io/v1beta1",
  "staticPodPath": "/etc/kubernetes/manifests",
  "syncFrequency": "0s",
  "fileCheckFrequency": "0s",
  "httpCheckFrequency": "0s",
  "tlsCipherSuites": [
    "TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256",
    "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256",
    "TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384",
    "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384",
    "TLS_ECDHE_ECDSA_WITH_CHACHA20_POLY1305_SHA256",
    "TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305_SHA256"
  ],
  "tlsMinVersion": "VersionTLS12",
 ...
