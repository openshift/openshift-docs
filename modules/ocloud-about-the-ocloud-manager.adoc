= Template-based cluster provisioning and management at scale

[role="_abstract"]
The O-Cloud Manager is an Operator that provides template-based infrastructure and cluster lifecycle management for {product-title} deployments. It runs on a hub cluster alongside {rh-rhacm-first} and orchestrates the provisioning, configuration, scaling, and upgrading of managed clusters through custom resources.

The O-Cloud Manager introduces a template-based abstraction that separates reusable cluster blueprints from site-specific deployment parameters. Platform engineers define `ClusterTemplate` resources that specify the cluster topology, hardware requirements, and configuration policies. Site operators create `ProvisioningRequest` resources that reference a template and supply site-specific values such as network addresses and cluster names. The O-Cloud Manager validates the request against the template schema, allocates hardware from managed resource pools, and orchestrates the end-to-end provisioning flow using the SiteConfig Operator, {gitops-title}, and the Topology Aware Lifecycle Manager.

All lifecycle operations, provisioning, configuration, scaling, and upgrades, are configured through the `ProvisioningRequest` custom resource, providing a single point of control for each managed cluster.

The O-Cloud Manager also implements the O-RAN Alliance O2 IMS specification, which provides standardized REST APIs for integration with Service Management and Orchestration (SMO) systems in telco environments. However, SMO integration is optional. The Operator can be used standalone for any multi-cluster bare-metal deployment that benefits from template-driven lifecycle management.
