= Anti-affinity for virtualized control planes

[role="_abstract"]
Distribute control plane VMs across different physical nodes on the hosting cluster so that a single node failure affects only one control plane VM.
Configure this placement by using one of the following methods:

* Pod anti-affinity rules in the `VirtualMachine` specification to prevent multiple control plane VMs from being scheduled on the same node.
* Topology spread constraints to evenly distribute VMs across failure domains.

A hosting cluster requires at least three physical nodes to support anti-affinity placement for the three control plane VMs of a target cluster.
