= Configuring the hub cluster for records storage

You can configure the hub cluster to use an external PostgreSQL database for storing Tekton Results records.

.Prerequisites

* You have set up an external PostgreSQL database for Tekton Results.
* You have access to the hub cluster using an account with `cluster-admin` permissions.
* You have installed the `oc` CLI.
* You have configured the hub cluster for multicluster support.

.Procedure

. Create a secret containing the PostgreSQL database credentials in the `openshift-pipelines` namespace by running the following command:
+
[source,terminal]
----
$ oc create secret generic tekton-results-postgres \
  --namespace=openshift-pipelines \
  --from-literal=POSTGRES_USER=<user> \
  --from-literal=POSTGRES_PASSWORD=<password>
----
+
Replace `<user>` and `<password>` with your PostgreSQL database credentials.
+
[NOTE]
====
If the secret already exists, you can update it by running the following command:

[source,terminal]
----
$ oc patch secret tekton-results-postgres \
  --namespace=openshift-pipelines \
  --type=merge \
  -p '{"data":{"POSTGRES_USER":"<base64_encoded_user>","POSTGRES_PASSWORD":"<base64_encoded_password>"}}'
----

Replace `<base64_encoded_user>` and `<base64_encoded_password>` with base64-encoded values of your credentials.
====

. Configure the TektonConfig custom resource to use the external PostgreSQL database by running the following command:
+
[source,terminal]
----
$ oc patch tektonconfig config --type=merge -p '
{
  "spec": {
    "result": {
      "is_external_db": true,
      "db_host": "<host>",
      "db_port": 5432,
      "db_sslmode": "prefer"
    }
  }
}'
----
+
Replace `<host>` with the hostname or IP address of your PostgreSQL database.
+
where:
+
`is_external_db`::
Set to `true` to enable the use of an external database instead of the in-cluster database.
+
`db_host`::
Specifies the hostname or IP address of the PostgreSQL database.
+
`db_port`::
Specifies the PostgreSQL database port. The default is `5432`.
+
`db_sslmode`::
Specifies the SSL mode for the database connection. Valid values are `disable`, `allow`, `prefer`, `require`, `verify-ca`, and `verify-full`. The default is `prefer`.

. Wait for the Tekton Results API pods to restart and become ready by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-pipelines -w
----
+
The pods restart automatically when the TektonConfig custom resource is updated.

.Verification

* Verify that the Tekton Results API pods are running by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-pipelines -l app.kubernetes.io/name=tekton-results-api
----
+
.Example output
[source,terminal]
----
NAME                                   READY   STATUS    RESTARTS   AGE
tekton-results-api-7d8f9c8b5d-abcde    1/1     Running   0          2m
----
