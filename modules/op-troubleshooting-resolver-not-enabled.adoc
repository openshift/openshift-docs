= Resolve cluster resolver not enabled errors

If the cluster resolver is not working, it might not be enabled in your {pipelines-title} installation. This can occur when the cluster resolver feature flag is not enabled, or when the resolver configuration is missing or incorrect.

.Procedure

. Check if the cluster resolver is enabled in the feature flags configuration by running the following command:
+
[source,terminal]
----
$ oc get configmap resolvers-feature-flags -n openshift-pipelines -o yaml
----

. Verify that `enable-cluster-resolver` is set to `"true"`:
+
[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: resolvers-feature-flags
  namespace: openshift-pipelines
data:
  enable-cluster-resolver: "true"
----
+
`data.enable-cluster-resolver`:: Must be set to `"true"` to enable the cluster resolver.

. If the setting is `"false"`, update it to `"true"` by running the following command:
+
[source,terminal]
----
$ oc patch configmap resolvers-feature-flags -n openshift-pipelines \
  --type merge -p '{"data":{"enable-cluster-resolver":"true"}}'
----

. Restart the resolver pods to apply the changes by running the following command:
+
[source,terminal]
----
$ oc delete pods -n openshift-pipelines -l app=tekton-pipelines-resolvers
----
