= {oadp-short} Self-Service namespace permissions

As a cluster administrator, ensure that a non-admin user has editor roles assigned for the following list of objects, in their namespace. These objects ensure that a non-admin user can perform the backup and restore operations in their namespace.  

* `nonadminbackups.oadp.openshift.io`
* `nonadminbackupstoragelocations.oadp.openshift.io`
* `nonadminrestores.oadp.openshift.io`
* `nonadmindownloadrequests.oadp.openshift.io`

A non-admin user cannot edit the NABSL CR that created the `BackupStorageLocation` (BSL). They can only create a new NABSL. The NAC controller does not allow a non-admin user to edit a NABSL and informs the non-admin user with the condition of the edited NABSL CR.

== Example `RBAC` YAML for backup operation

Following is an example of the `RBAC` YAML file with namespace permissions for a non-admin user to perform a backup operation.
+
.Example `RBAC`
[source,yaml]
----
...
- apiGroups:
      - oadp.openshift.io
    resources:
      - nonadminbackups
      - nonadminrestores
      - nonadminbackupstoragelocations
    verbs:
      - create
      - delete
      - get
      - list
      - patch
      - update
      - watch
  - apiGroups:
      - oadp.openshift.io
    resources:
      - nonadminbackups/status
    verbs:
      - get
----
