= Forwarding logs to an external Loki logging system

To configure log forwarding to Loki, you must create a `ClusterLogForwarder` custom resource (CR) with an output to Loki, and a pipeline that uses the output. The output to Loki can use the HTTP (insecure) or HTTPS (secure HTTP) connection.

.Prerequisites

* You have installed {clo}.
* You have administrator access to {ocp-product-title}.
* You have installed {oc-first}.
* You have a Loki logging system running at the URL you specify with the `url` field in the CR.

.Procedure

. Create or edit a YAML file that defines the `ClusterLogForwarder` CR object:
+
[source,yaml]
----
apiVersion: observability.openshift.io/v1
kind: ClusterLogForwarder
metadata:
  name: <log_forwarder_name>
  namespace: <log_forwarder_namespace>
spec:
  serviceAccount: 
    name: <service_account_name> #<1>
  outputs:
  - name: loki-output #<2>
    type: loki #<3>
    loki:
      authentication: #<4>
        token:
          from: serviceAccount
      labelKeys: #<5>
        - <label_keys>
      tenantKey: kubernetes.namespace_name #<6>
      url: https://loki.secure.com:3100 #<7>
  tls: #<8>
    ca:
      key: service-ca.crt #<9>
      configMapName: openshift-service-ca.crt #<10>
  pipelines:
  - name:  my-pipeline
    inputRefs:  #<11>
    - application
    - audit
    outputRefs: #<12>
    - loki-output
----
<1> The name of your service account. The service account is only required in multi log forwarder implementations if the log forwarder is not deployed in the `openshift-logging` namespace.
<2> Specify a name for the output.
<3> Specify the type as `loki`.
<4> Authentication 
<5> Specify which log record keys are mapped to Loki stream labels. If you do not set the `labelKeys` field, the default keys that will be set are: `log_type`, `kubernetes.container_name`, `kubernetes.namespace_name`, `kubernetes.pod_name`.
<6> Specify the tenant for the logs. The value can be a combination of static and dynamic values consisting of field paths separated by `||`. Encased dynamic values inside  single curly brackets `{}`. Dynamic values must be followed by a static fallback value.
<7> Specify the URL for Loki. If the URL scheme contains 'https', or 'tls', basic TLS is enabled for the connection.
<8> Specify the TLS settings for secure connection.
<9> Specify the name of the key used to get the value in either the referenced `ConfigMap` or `Secret` resource.
<10> Specify the name of the `ConfigMap` resource that contains the CA certificate.
<11> Specify the the names of inputs, defined in the `input.name` field for this pipeline. You can also use the built-in values `application`, `infrastructure`, `audit`.
<12> Specify the the names of outputs, defined in the `outputs.name` field for this pipeline.
+
[NOTE]
====
Because Loki requires log streams to be correctly ordered by timestamp, `labelKeys` always includes the `kubernetes_host` label set, even if you do not specify it. This inclusion ensures that each stream originates from a single host, which prevents timestamps from becoming disordered due to clock differences on different hosts.
====

. Apply the `ClusterLogForwarder` CR object by running the following command:
+
[source,terminal]
----
$ oc apply -f <filename>.yaml
----
