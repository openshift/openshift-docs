= Requirements

You must deploy an {product-title} on {GCP} into an existing VPC when using PSC. The existing VPC must contain:
* Three subnets: control plane, worker and PSC. The PSC subnet must have Purpose="Private Service Connect"

You can optionally use firewall to control egress traffic. If you choose not use a firewall to control egress traffic, you must configure Cloud Router and Cloud NAT.

== Requirements when using Workload Identity Federation authentication type

The following prerequisites are required for clusters configured with the Workload Identity Federation authentication type.

* Enable IAP API at the project level (iap.googleapis.com)
* roles/compute.admin is sufficient and is already on the osd-ccs-admin service account.
* Add IAP-Secured Tunnel User to osd-ccs-admin service account: roles/iap.tunnelResourceAccessor[*] (This role has iap.tunnelDestGroups.accessViaIAP and iap.tunnelInstances.accessViaIAP permissions required by the feature)
