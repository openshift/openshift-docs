= Prerequisites

Besides the mandatory prerequisites that you need to complete before deploying any {product-title} on {GCP} cluster, there are other specific requirements prior to deploying a private cluster using Private Service Connect. These requirements are listed below.

* A pre-created Virtual Private Cloud (VPC) with the following subnets in the same {GCP} region where your cluster will be deployed:

** A control plane subnet
** A worker subnet
** A PSC subnet with Purpose = Private Service Connect.
+
[IMPORTANT]
====
Mask lengths cannot exceed twenty-nine for subnets created with Purpose = Private Service Connect. Additionally, the PSC subnet must be within the Machine CIDR used while provisioning the {product-title} cluster.
====
+
For information on how to create a VPC on {GCP}, see link:https://cloud.google.com/vpc/docs/create-modify-vpc-networks[Create and manage VPC networks] in the Google Cloud documentation.

* An egress path to the internet for Red Hat-specific domain endpoints.

* Enabled link:https://console.cloud.google.com/marketplace/product/google/iap.googleapis.com?q=search&referrer=search&hl=en&project=openshift-gce-devel[Cloud Identity-Aware Proxy API] at the {GCP} project level.

* **Optional**: You can configure an external firewall such as Google Cloud Next Generation Firewall (NGFW), Palo Alto Networks or Fortinet firewall, to control egress traffic from {product-title} on {GCP}. You must
configure your firewall to grant access to certain domain and port combinations as shown xref:../osd_planning/gcp-ccs.adoc#osd-gcp-psc-firewall-prerequisites_gcp-ccs[here]. {product-title} on {GCP} requires this access to provide a fully-managed {product-title} service.
+
[NOTE]
====
If you configure an external firewall, configuring Private Google Access on the subnet is recommended. If Private Google Access is configured, it must be supported with private DNS records.
====
+
[IMPORTANT]
====
Even when using a firewall, An egress path to the internet for Red Hat-specific domain endpoints is required.
====

In addition to the requirements listed above, clusters configured with the **Service Account authentication type** must grant the `IAP-Secured Tunnel User` role to `osd-ccs-admin` service account.

For more information on the prerequisites that must be completed prior to deploying an {product-title} on {GCP}, see _Additional resources_.
