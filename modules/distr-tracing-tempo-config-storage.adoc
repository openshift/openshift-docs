= Storage configuration

You can configure object storage for the {TempoShortName} in the `TempoStack` custom resource under `spec.storage`. You can choose from among several storage providers that are supported.

.General storage parameters used by the {TempoOperator} to define distributed tracing storage
[options="header"]
[cols="a, a, a"]
|===
|Parameter |Description |Values

|`storage.secret.type`
|Type of object storage for the deployment.
|`azure`, `gcs`, `s3`


|`storage.secret.name`
|Name of the secret that contains the credentials for the chosen type of object storage.
|

|`storage.tls.enable`
|Optional: Enables TLS when communicating with object storage.
|

|`storage.tls.caName`
|Optional: Name of a config map that contains a `service-ca.crt` CA certificate. You must create the config map in the same project as the `TempoStack` custom resource.
|

|`storage.tls.certName`
|Optional: Name of a secret that contains a `tls.crt` certificate and `tls.key` private key. You must create the secret in the same project as the `TempoStack` custom resource.
|

|`storage.tls.minVersion`
|Optional: The oldest acceptable TLS version.
|

|===
