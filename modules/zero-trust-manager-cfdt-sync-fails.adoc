= ClusterFederatedTrustDomain fails to sync

[role="_abstract"]
When the `ClusterFederatedTrustDomain` (CFDT) fails to sync, the automatic exchange of trust bundles between clusters is interrupted. Use the following steps to identify why the sync failed.

.Procedure

. Check the `ClusterFederatedTrustDomain` by running the following command. Look for a `Ready` or `synced` status set to `False`.
+
[source,terminal]
----
$ oc describe clusterfederatedtrustdomain
----

. Check the `spire-controller-manager` logs by running the following command:
+
[source,terminal]
----
$ oc logs -n zero-trust-workload-identity-manager -l app=spire-controller-manager --tail=50
----
