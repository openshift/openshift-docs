= About Model Context Protocol (MCP) server 

The Model Context Protocol (MCP) is an open protocol that standardizes how applications provide context to large language models (LLMs). Using the protocol, an MCP server offers a standardized way for an LLM to increase context by requesting and receiving real-time information from external resources. 

:FeatureName: MCP server
include::snippets/technology-preview.adoc[]

The {ols-long} service installs an MCP server for the {ocp-product-title}. The MCP server provides the {ols-long} service with access to the {ocp-short-name} API. Through this access, the service performs read operations to gether more context for the LLM, enabling the service to answer questions about the Kubernetes objects that reside in your {ocp-short-name} cluster.

You have the option to use either a default MCP server or a custom one. For convenience and simplicity, the {ols-long} service provides access to a default MCP server, which requires little maintenance, and is ideal for rapid prototyping or initial experiments. For situations that require increased control or security, including custom integrations or access to private data, the {ols-long} service can connect to a custom MCP server.  

[NOTE]
====
You must enable MCP server.

* To deploy the default MCP server, set the `ols.introspectionEnabled` field to `enabled` in the `OLSConfig` custom resource (CR).

* To deploy a custom MCP server, set the `spec.ols.featureGate` field to `MCPServer` in the `OLSConfig` custom resource (CR).
====
