= Updating the {loki-op} by using the CLI

To update the {loki-op} to a new version, modify the update channel for the Operator subscription.

.Prerequisites
* You have checked readiness for update.
* You have installed {loki-op} v6.
* The {loki-op} version you are updating from is not more than two versions older than {logging-major-version}.{logging-minor-version}.
* You have installed the {oc-first}.

.Procedure

. Find the name of the subscription for the {loki-op} by using the following command:
+
[source,terminal]
----
$ oc get subscription -n openshift-operators-redhat
----
+
.Example output
[source,options="nowrap"]
----
NAME            PACKAGE         SOURCE             CHANNEL
loki-operator   loki-operator   redhat-operators   stable-6.<y>
----
+
Look for the value of `PACKAGE` listed for `{loki-op}` in the `NAME` column.
The subscription name in the example output is `loki-operator`.

. Patch the subscription channel by using the following command:
+
[source,terminal]
----
$ oc patch subscription loki-operator -n openshift-operators-redhat \
    --type='merge' \
    -p '{"spec":{"channel":"<channel>"}}'
----
+
Replace `<channel>` with the value `stable-{logging-major-version}`.
+
[IMPORTANT]
====
The maximum supported update channel is N+2 version, where N is your current version. For example, if you are updating from Logging 6.0, only select a maximum of `stable-6.2` version as the update channel. Updating to a version that is more than two versions newer is not supported.
====

. Check whether the `installPlanApproval` value is set to `Automatic` or `Manual`.

.. Find the install plan by using the following command:
+
[source,terminal]
----
$ oc get installplan -n openshift-operators-redhat
----
+
.Example output
[source,options="nowrap"]
----
NAME            CSV                    APPROVAL    APPROVED
install-mfvsm   loki-operator.v6.y.z   Automatic   true
----
.. If the `installPlanApproval` value is set to `Manual`, approve the plan by using the following command:
+
[source,terminal]
----
$ oc patch installplan <plan_name> -n openshift-operators-redhat --type merge -p '{"spec":{"approved":true}}'
----
+
Replace `<plan_name>` with the result obtained in the previous step.

.Verification

* Check the ClusterServiceVersion (CSV) in the namespace.
+
[source,terminal]
----
$ oc get csv -n openshift-operators-redhat | grep loki-operator
----
+
.Example output
[subs="attributes",options="nowrap"]
----
NAME                     DISPLAY                     VERSION   REPLACES                 PHASE
loki-operator.v{logging-major-version}.{logging-minor-version}     Loki Operator               {logging-major-version}.{logging-minor-version}     loki-operator.v6.y.z     Succeeded
----
