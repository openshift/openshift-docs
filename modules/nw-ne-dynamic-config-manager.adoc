= Enabling the Dynamic Configuration manager_

Dynamic Configuration Manager: Some template routers, like HAProxy, utilize a dynamic configuration manager to propagate route and endpoint changes via a dynamic API. This approach allows the underlying router to adapt to changes efficiently and ensures that the system can scale and adjust as needed.

HAProxy Manager: The HAProxy manager uses this dynamic API to modify the operational state of HAProxy backends.

Handling Endpoint Changes: Endpoint changes, such as scaling and node evictions, are managed by provisioning each backend with a pool of dynamic servers.

Dynamic Servers Pool: The size of this dynamic servers pool is controlled by the max-dynamic-servers option or the ROUTER_MAX_DYNAMIC_SERVERS environment variable.

.Procedure

To enable the dynamic configuration manager for certain template routers (like HAProxy) in Red Hat OpenShift, you need to follow these steps:

    Enable the HAProxy Config Manager:

        Set the UseHAProxyConfigManager flag to true in your TemplateRouterConfigManager.

    Set the Commit Interval:

        Define the interval at which the router commits changes using the CommitInterval option. The default value is set to 60 * 60 seconds (1 hour).

    Configure Dynamic Servers Pool:

        Adjust the size of the dynamic servers pool using the MaxDynamicServers option or the ROUTER_MAX_DYNAMIC_SERVERS environment variable.

    Blueprint Route Pool:

        Pre-allocate a pool of routes called blueprints using the BlueprintRoutePoolSize option or the ROUTER_BLUEPRINT_ROUTE_POOL_SIZE environment variable.

    Optional Customization:

        Customize blueprints using the BlueprintRouteNamespace and BlueprintRouteLabelSelector options to select routes from a specific namespace matching certain labels.

Example Configuration

Here’s an example of how you might configure these options:
[source,yaml]

apiVersion: v1
kind: ConfigMap
metadata:
  name: my-router-config
data:
  useHAProxyConfigManager: "true"
  commitInterval: "3600"
  maxDynamicServers: "20"
  blueprintRoutePoolSize: "10"
  blueprintRouteNamespace: "my-namespace"
  blueprintRouteLabelSelector: "my-label=my-value"

In your router deployment or ConfigMap, you would set these values appropriately.
Environment Variables

Alternatively, you can set these values using environment variables in your router deployment:

[source,yaml]

apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-router
spec:
  template:
    spec:
      containers:
      - name: router
        env:
        - name: ROUTER_MAX_DYNAMIC_SERVERS
          value: "20"
        - name: ROUTER_BLUEPRINT_ROUTE_POOL_SIZE
          value: "10"
        - name: USE_HAPROXY_CONFIG_MANAGER
          value: "true"
        - name: COMMIT_INTERVAL
          value: "3600"

By setting these options, you enable and configure the dynamic configuration manager for your template router, allowing it to handle route and endpoint changes efficiently.
