= VirtualMachineBackupsDiscovery API reference

[role="_abstract"]
The `VirtualMachineBackupsDiscovery` custom resource identifies which Velero backups contain a specified virtual machine.
Use this reference to configure discovery operations and interpret status information for automation and scripting.

.VirtualMachineBackupsDiscovery spec fields
[cols="1,1,2",options="header"]
|===
|Field
|Type
|Description

|`spec.virtualMachineName`
|string (required)
|Name of the virtual machine to search for in backups.

|`spec.virtualMachineNamespace`
|string (required)
|Namespace of the virtual machine.

|`spec.startTime`
|string (optional)
|RFC3339 timestamp for the earliest backup to include in discovery.
Backups created before this time are excluded.

|`spec.endTime`
|string (optional)
|RFC3339 timestamp for the latest backup to include in discovery.
Backups created after this time are excluded.

|`spec.requestedBackups`
|array of strings (optional)
|List of specific Velero backup names to check.
If specified, only these backups are searched instead of all backups in the time range.
|===

.VirtualMachineBackupsDiscovery status fields
[cols="1,1,2",options="header"]
|===
|Field
|Type
|Description

|`status.phase`
|string
|Current phase of the discovery operation.
Valid values: `New`, `InProgress`, `Completed`, `Failed`.

|`status.validBackups`
|array of objects
|List of Velero backups that contain the target VM.
Each entry includes `name` (backup name), `snapshotTimestamp` (backup creation time), and additional metadata.

|`status.backupDiscoveryProgress`
|string
|Progress percentage of the discovery operation.
Format: `"N/M"` where N is backups checked and M is total backups to check.

|`status.discoveryStats`
|object
|Statistics about the discovery operation including `totalBackups` scanned and `matchedBackups` found.

|`status.conditions`
|array of objects
|Detailed status conditions with `type`, `status`, `reason`, and `message` fields.
Use to troubleshoot failed discovery operations.

|`status.observedGeneration`
|integer
|The generation of the CR that the controller has processed.
|===

.Example VirtualMachineBackupsDiscovery CR with time range filtering
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: VirtualMachineBackupsDiscovery
metadata:
  name: discover-prod-vm-backups
  namespace: openshift-adp
spec:
  virtualMachineName: production-vm
  virtualMachineNamespace: prod-vms
  startTime: "2026-06-01T00:00:00Z"
  endTime: "2026-07-01T00:00:00Z"
----

.Example VirtualMachineBackupsDiscovery status showing completed discovery
[source,yaml]
----
status:
  phase: Completed
  validBackups:
    - name: daily-backup-20260630
      snapshotTimestamp: "2026-06-30T02:00:00Z"
    - name: daily-backup-20260629
      snapshotTimestamp: "2026-06-29T02:00:00Z"
  backupDiscoveryProgress: "150/150"
  discoveryStats:
    totalBackups: 150
    matchedBackups: 2
  conditions:
    - type: Ready
      status: "True"
      reason: DiscoveryCompleted
  observedGeneration: 1
----
