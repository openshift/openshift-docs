= Manually add World Wide Identifiers (WWIDs) for enabling multipathing on {op-system}

[role="_abstract"]
To ensure high availability for storage devices that are not automatically detected by the operating system, manually configure multipathing in {product-title}. This allows you to explicitly define device properties and path groupings, preventing I/O errors and maintaining reliable storage access.

If a node requires an update or a reboot, you must have at least two viable paths to the boot device present. In situations where the number of paths drops to a single path, you can work around this by manually adding the necessary WWIDs when you access the boot menu.

.Procedure

. At the boot menu, edit the GRUB line and append the WWID to the kernel argument:
+
[source,terminal]
----
# mpath.wwid=0x38167a80e8776b65
----

. In the emergency shell, manually add the resulting WWIDs to `/etc/multipath/wwids`. You can view the file by running the following command:
+
[source,terminal]
----
# cat /etc/multipath/wwids
----
+
.Example output
[source,text]
----
# Multipath wwids, Version: 1.0
# NOTE: This file is automatically maintained by multipath and multipathd,
# You should not need to edit this file in normal circumstances.
#
# Valid WWIDs:
/0x38167a80e8776b65/
----
