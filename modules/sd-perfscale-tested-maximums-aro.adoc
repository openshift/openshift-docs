= Tested cluster maximums for Azure Red Hat OpenShift (reference)

[role="_abstract"]
The following tables list tested resource maximums for Azure Red Hat OpenShift 4.19 without changing default cluster configuration beyond what the test describes.

Unlike some other managed offerings, control plane and infrastructure node instance types are not fixed; customers can choose supported sizes. For testing, instance types were selected to mirror the worker-count tiers used for similar managed offerings.

:fn-perfscale-aro-1: footnote:perfscale-aro-fn1[Simple pause pods staying idle on the cluster, including default control plane pods; used `node-density` workload. Actual pod counts depend on workload requirements.]
:fn-perfscale-aro-2: footnote:perfscale-aro-fn2[Workload shape can lower pods per node; for example, many PostgreSQL pods on one node with high disk IOPS, or limits on PVCs per node.]
:fn-perfscale-aro-3: footnote:perfscale-aro-fn3[Namespaces included a few idle pods plus default config maps and secrets each, including default operator namespaces.]
:fn-perfscale-aro-4: footnote:perfscale-aro-fn4[ClusterIP services only; other service types can have different limits. Used `cni-workload`.]
:fn-perfscale-aro-5: footnote:perfscale-aro-fn5[Test cluster used two routers with default configuration on infrastructure nodes; used `cluster-density` workload.]
:fn-perfscale-aro-6: footnote:perfscale-aro-fn6[ClusterIP services used a single endpoint each.]

.Test cluster sizing (ARO)
[options="header",cols="2,2"]
|===
| Node | Instance type classification

| Control plane
| 1 to 25 workers: `Standard_D8s_v3`. 26 to 100 workers: `Standard_D16s_v3`. 101+ workers: `Standard_D32s_v3`.

| Infrastructure
| 1 to 25 workers: `Standard_E4as_v5`. 26 to 100 workers: `Standard_E8as_v5`. 101+ workers: `Standard_E16as_v5`.

| Compute (worker)
| `Standard_D8s_v3`
|===
