= About the in place or recreate mode

The Vertical Pod Autoscaler Operator (VPA) contains an optional `InPlaceOrRecreate` mode. This mode instructs VPA to attempt to perform an _in-place pod resize_ when updating resources, where the VPA restarts only the containers within the node, not the entire pod. If an in-place resize is not possible, the VPA is forced to recreate the entire pod. The VPA recreates a pod in the following scenarios:

* The in-place update is considered _infeasible_ for reasons including:
** The requested resources exceed the node's total capacity.
** The pod is a static pod.
** The containers has swap enabled.
** The pod is managed by a static `cpuManagerPolicy` or `memoryManagerPolicy` parameter.
* The update is _deferred_ for more than 5 minutes. A deferred update is one that is currently not possible, but might become possible at a later time. 
* The update is in progress for more than 1 hour
* The pod QoS class would change due to the update.
* The update would downscale the Mmemory limit.

If a pod with a container has a `RestartContainer` container resize policy, which requires a restart upon a resource update, a VPA in `InPlaceOrRecreate` mode honors the resize policy and restarts the container.

For more information on in-place updates, see "Adjust pod resource levels without pod disruption".
