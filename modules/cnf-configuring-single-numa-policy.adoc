= Configuring a single NUMA node policy

The NUMA Resources Operator needs a single NUMA node policy configured on the cluster. There are two ways to achieve this as illustrated in this procedure. 

[NOTE]
====
The recommended way to do this is to use the performance profile and to create this using the Performance Profile Creator (PPC) tool. 
====

.Procedure

. The Performance Profile Creator (PPC) tool requires `must-gather` data. As a cluster administrator, run the `must-gather` command to capture information about your cluster. 

.. Go to the directory where you want to store the `must-gather` data.

. Run the `oc adm must-gather` command:
+
[source,terminal]

----
$ oc adm must-gather
----

. Create a compressed file from the `must-gather` directory that was just created in your working directory. For example, on a computer that uses a Linux
operating system, run the following command:
+
[source,terminal]
----
$ tar cvaf must-gather.tar.gz must-gather.local.5421342344627712289/ <1>
----
+
<1> Make sure to replace must-gather-local.5421342344627712289/ with the actual directory name.

. Check the machine config pool:
+
[source,terminal]
----
$ oc get mcp
----
.Example output
+
[source,terminal]
----
NAME     CONFIG                                             UPDATED   UPDATING   DEGRADED   MACHINECOUNT   READYMACHINECOUNT   UPDATEDMACHINECOUNT   DEGRADEDMACHINECOUNT   AGE
master   rendered-master-6c701a0631bc35ff35eb3947ccb57eff   True      False      False      3              3                   3                     0                      101m
worker   rendered-worker-f11f2ea0550dbc10fd91cdde7c4c93c7   True      False      False      3              3                   3                     0                      101m
----

. Use Podman to authenticate to `registry.redhat.io`:
+
[source,terminal]
----
$ podman login registry.redhat.io
----
+
[source,bash]
----
Username: <username>
Password: <password>
----

. Run `podman`:
+
[source,terminal,subs="attributes+"]
----
$ podman run --entrypoint performance-profile-creator -v /must-gather:/must-gather:z registry.redhat.io/openshift4/ose-cluster-node-tuning-rhel9-operator:v{product-version} --mcp-name=worker-cnf --reserved-cpu-count=4 --rt-kernel=true --split-reserved-cpus-across-numa=false --must-gather-dir-path /must-gather --power-consumption-mode=ultra-low-latency --topology-manager-policy=single-numa-node > my-performance-profile.yaml
----
+
[NOTE]
====
The following arguments are required:

* `reserved-cpu-count`
* `mcp-name`
* `rt-kernel`

The `mcp-name` argument in this example is set to `worker-cnf` based on the output of the command `oc get mcp`. For {sno} use `--mcp-name=master`.
====

. Review the created YAML file:
+
[source,terminal]
----
$ cat my-performance-profile.yaml
----
.Example output
+
[source,yaml]
----

----

. Apply the generated profile:
+
[source,terminal]
----
$ oc apply -f my-performance-profile.yaml
----

An alternative way if cluster is to create and apply a kubeletconfig similar to the following : 

. Create the `KubeletConfig` custom resource that configures the pod admittance policy for the machine profile:

.. Save the following YAML in the `nro-kubeletconfig.yaml` file:
+
[source,yaml]
----
apiVersion: machineconfiguration.openshift.io/v1
kind: KubeletConfig
metadata:
  name: cnf-worker-tuning
spec:
  machineConfigPoolSelector:
    matchLabels:
      cnf-worker-tuning: enabled
  kubeletConfig:
    cpuManagerPolicy: "static" <1>
    cpuManagerReconcilePeriod: "5s"
    reservedSystemCPUs: "0,1"
    memoryManagerPolicy: "Static" <2>
    evictionHard:
      memory.available: "100Mi"
    kubeReserved:
      memory: "512Mi"
    reservedMemory:
      - numaNode: 0
        limits:
          memory: "1124Mi"
    systemReserved:
      memory: "512Mi"
    topologyManagerPolicy: "single-numa-node" <3>
    topologyManagerScope: "pod"
----
<1> For `cpuManagerPolicy`, `static` must use a lowercase `s`.
<2> For `memoryManagerPolicy`, `Static` must use an uppercase `S`.
<3> `topologyManagerPolicy` must be set to `single-numa-node`.

.. Create the `KubeletConfig` custom resource (CR) by running the following command:
+
[source,terminal]
----
$ oc create -f nro-kubeletconfig.yaml
----
