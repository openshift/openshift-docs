= Restoring the mysql database

The restoration of the mysql database re-creates the Pod, ReplicationController, DeploymentConfig, additional PV, and persistent volume claim (PVC).

NOTE: Do not delete the default PV and PVC associated with the database. 

.Prerequisites

* You have restored the secrets and APIManager custom resources.

.Procedure

. Scale down the threescale operator:
+
[source,terminal]
----
$ oc scale deployment threescale-operator-controller-manager-v2 --replicas=0 -n threescale
deployment.apps/threescale-operator-controller-manager-v2 scaled
----

. Scale down all the DeploymentConfigs:
+
[source,terminal]
----
$ cat ./scaledowndc.sh

for dc in apicast-production apicast-staging backend-cron backend-listener backend-redis backend-worker system-app system-memcache system-mysql system-redis system-searchd system-sidekiq zync zync-database zync-que; do
    oc scale dc/$dc --replicas=0 -n threescale
done
----
+
[source,terminal]
----
$ ./scaledowndc.sh 

deploymentconfig.apps.openshift.io/apicast-production scaled
deploymentconfig.apps.openshift.io/apicast-staging scaled
deploymentconfig.apps.openshift.io/backend-cron scaled
deploymentconfig.apps.openshift.io/backend-listener scaled
deploymentconfig.apps.openshift.io/backend-redis scaled
deploymentconfig.apps.openshift.io/backend-worker scaled
deploymentconfig.apps.openshift.io/system-app scaled
deploymentconfig.apps.openshift.io/system-memcache scaled
deploymentconfig.apps.openshift.io/system-mysql scaled
deploymentconfig.apps.openshift.io/system-redis scaled
deploymentconfig.apps.openshift.io/system-searchd scaled
deploymentconfig.apps.openshift.io/system-sidekiq scaled
deploymentconfig.apps.openshift.io/zync scaled
deploymentconfig.apps.openshift.io/zync-database scaled
deploymentconfig.apps.openshift.io/zync-que scaled
----

. Delete the system-mysql DeploymentConfig:
+
[source,terminal]
----
$ oc delete dc system-mysql -n threescale

Warning: apps.openshift.io/v1 DeploymentConfig is deprecated in v4.14+, unavailable in v4.10000+
deploymentconfig.apps.openshift.io "system-mysql" deleted
deploymentconfig.apps.openshift.io "backend-redis" deleted
----

. Restore the mysql database using YAML:
+
[source,terminal]
----
$ oc create -f restore-backend.yaml
----
+
[source,yaml]
----
apiVersion: velero.io/v1
kind: Restore
metadata:
  name: restore-mysql
  namespace: openshift-adp
spec:
  backupName: mysql-backup
  excludedResources:
    - nodes
    - events
    - events.events.k8s.io
    - backups.velero.io
    - restores.velero.io
    - csinodes.storage.k8s.io
    - volumeattachments.storage.k8s.io
    - backuprepositories.velero.io
    - resticrepositories.velero.io
  hooks:
    resources:
      - name: restoreDB
        postHooks:
          - exec:
              command:
                - /bin/sh
                - '-c'
                - >
                  sleep 30
                  
                  mysql -h 127.0.0.1 -D system -u root
                  --password=$MYSQL_ROOT_PASSWORD <
                  /var/lib/mysqldump/data/dump.sql
              container: system-mysql
              execTimeout: 80s
              onError: Fail
              waitTimeout: 5m
  itemOperationTimeout: 1h0m0s
  restorePVs: true
----
