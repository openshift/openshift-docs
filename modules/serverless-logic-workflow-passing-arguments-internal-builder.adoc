= Passing arguments to the internal builder

You can pass build arguments to the `SonataFlowBuild` instance or set default build arguments in the `SonataFlowPlatform`, allowing for customization of the build process.

.Prerequisites

* {ServerlessLogicOperatorName} is installed on your cluster.
* You have created a {ServerlessLogicProductName} project or have access to a project with the appropriate roles and permissions to create applications and other workloads in OpenShift Container Platform.
* You have created your {ServerlessLogicProductName} project
* Install the OpenShift CLI `(oc)`.

.Procedure

. Check for existing `SonataFlowBuild` instance.
+
Use the following command to check if a SonataFlowBuild instance already exists:
+
[source,terminal]
----
$ oc get sonataflowbuild <name> -n <namespace>
----
+ 
Replace `<name>` with the name of your SonataFlowBuild instance and `<namespace>` with your namespace.

. Add build arguments to the `SonataFlowBuild` instance.
+
Use the following command to edit the SonataFlowBuild instance:
+
[source,terminal]
----
$ oc edit sonataflowbuild <name> -n <namespace>
----

. Add the desired build arguments under `.spec.buildArgs` as follows:
+
[source,yaml]
----
apiVersion: sonataflow.org/v1alpha08
kind: SonataFlowBuild
metadata:
  name: <name>
spec:
  buildArgs:
    - name: ARG1
      value: value1
    - name: ARG2
      value: value2
----

. Save the file and exit. A new build should start soon with the updated configuration.

. Set the default build arguments in `SonataFlowPlatform` instance. 
+
[source,terminal]
----
$ oc edit sonataflowplatform <name> -n <namespace>
----

. Add the desired build arguments under `.spec.buildArgs` as follows:
+
[source,yaml]
----
apiVersion: sonataflow.org/v1alpha08
kind: SonataFlowPlatform
metadata:
  name: <name>
spec:
  build:
    template:
      buildArgs:
        - name: ARG1
          value: value1
        - name: ARG2
          value: value2
----

. Save the file and exit.
