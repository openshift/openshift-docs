= Migrating VMs in a single cluster to a different storage class by using the web console

[role="_abstract"]

You can migrate the disks attached to single cluster virtual machines (VMs) from one storage class to a different storage class by using the {product-title} web console. This procedure works for both running and offline VMs. When performing this action on a running VM, the operation of the VM is not interrupted and the data on the migrated disks remains accessible.

.Procedure

. Navigate to *Virtualization* -> *VirtualMachines* in the web console.

. Click the *Virtual machines* tab.

. Select multiple VMs using the checkboxes to the left of the *Name* field. Alternatively, use the multi-select checkbox under the *Project* dropdown list to deselect the currently selected VMs (*Select none*), select all VMs on the currently displayed page (*Select page*), or select all VMs in the cluster (*Select all*).

. Click the *Actions* menu and select *Migration* -> *Storage*.

. On the *Migration details* page, perform the following actions:

.. Enter a migration plan name in the *VirtualMachine storage migration plan name* field or use the provided default.
.. Use the provided option buttons to select whether to migrate *The entire VirtualMachine* or *Selected volumes*.
+
[NOTE]
====
If you select *The entire VirtualMachine*, all disks on all selected VMs are migrated to the same target storage.
====
.. (Optional) If you selected *Selected volumes* in the previous step, use the provided checkboxes to select the volumes to migrate. Volumes that cannot be migrated have a greyed out checkbox.
.. Click *Next*.

. On the *Source and target StorageClass* page, perform the following actions:

.. Select the storage migration target with the *Select the target storage for the VirtualMachine storage migration* dropdown list.
.. The system automatically decommissions the migration source volumes once the migration completes. Select the *Keep original volumes at source after successful migration* checkbox if you want to manually verify and remove the source volumes later.
.. Click *Next*.

. On the *Review* page, confirm the storage migration options.

. (Optional) Click *Back* to move back to previous pages to change storage migration options.

. Click *Migrate VirtualMachine storage* to start the migration.

. Stay on the *Migrate VirtualMachine storage* page to watch the progress and wait for the confirmation that the migration completed successfully.

. (Optional) Click *View storage migrations* to view all storage migration plans.

.Verification

. From the *VirtualMachine details* page for each VM, navigate to *Configuration* -> *Storage*.
. Verify that all disks have the expected storage class listed in the *Storage class* column.
