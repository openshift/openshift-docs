= Using SPIRE federation with manual certificate management

[role="_abstract"]
You can use SPIRE federation for custom certificate management. This allows integration with cert-manager or other certificate providers. There is also support for externally managed TLS secrets. The certificate refersh intervals are also configurable.

.Prerequisites

* You must have three OpenShift clusters.

* {zero-trust-full} must be install on all three clusters using the Operation Lifecycle Manager (OLM).

* Access to `kubeconfig` on all three clusters.

* `oc` and `curl` must be installed.

* Ability to reach federation endpoints (`https://federation.<domain>`)

* Federation endpoints must be publicly accessible for Let's  Encrypt validation.

* cert-manager operator is available (this is installed on cluster three)

.Procedure

. Download the federation script from `setup-3-cluster-federation-serving_cert.sh`.

. Execute the script by running the following command:
+
[source,terminal]
----
$ ./setup-3-cluster-federation-serving_cert.sh \
   -k1 /path/to/cluster1/kubeconfig \
   -k2 /path/to/cluster2/kubeconfig \
   -k3 /path/to/cluster3/kubeconfig \
   -c1 cluster1-name \
   -c2 cluster2-name \
   -c3 cluster3-name
----
+
The following is an example of a configured script.
+
[source,terminal]
----
$ ./setup-3-cluster-federation-serving_cert.sh \
   -k1 /home/rausingh/Documents/aws_cluster/08DecCluster1/auth/kubeconfig \
   -k2 /home/rausingh/Documents/aws_cluster/08DecCluster2/auth/kubeconfig \
   -k3 /home/rausingh/Documents/aws_cluster/08DecCluster3/auth/kubeconfig \
   -c1 aws08dec1 \
   -c2 aws08dec2 \
   -c3 aws08dec3
----

. Install the cert-manager Operator (only on cluster 3) by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig apply -f -
----

The following is example output from running the command:
+
[source,yaml]
----
<<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: cert-manager-operator
---
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: openshift-cert-manager-operator
  namespace: cert-manager-operator
spec:
  upgradeStrategy: Default
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: openshift-cert-manager-operator
  namespace: cert-manager-operator
spec:
  source: redhat-operators
  sourceNamespace: openshift-marketplace
  name: openshift-cert-manager-operator
  channel: stable-v1
EOF>>
----

. Create an ACME Issuer (Let's Encrypt) by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig apply -f -
----
+
The following is example output from running the command:
+
[source,yaml]
----
<<EOF
apiVersion: cert-manager.io/v1
kind: Issuer
metadata:
  name: letsencrypt-http01
  namespace: zero-trust-workload-identity-manager
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    privateKeySecretRef:
      name: letsencrypt-account-key
    solvers:
      - http01:
          ingress:
            ingressClassName: openshift-default
EOF
----

. Get the federation endpoint domain by running the following command:
+
[source,terminal]
----
$ FEDERATION_DOMAIN=$(oc --kubeconfig=/path/to/cluster3/kubeconfig get route spire-server-federation -n zero-trust-workload-identity-manager -o jsonpath='{.spec.host}')
echo "Federation endpoint: $FEDERATION_DOMAIN"
----

. Export the TLS secret name by running the following command:
+
[source,terminal]
----
$ export TLS_SECRET_NAME=spire-server-federation-tls
----

. Create the certificate by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig apply -f -
----
+
The following is example output from running the command:
+
[source,yaml]
----
<<EOF
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: $TLS_SECRET_NAME
  namespace: zero-trust-workload-identity-manager
spec:
  secretName: $TLS_SECRET_NAME
  commonName: $FEDERATION_DOMAIN
  dnsNames:
    - $FEDERATION_DOMAIN
  usages:
    - server auth
  issuerRef:
    kind: Issuer
    name: letsencrypt-http01
EOF
----
+
.Example output when the certificate is ready
[source, terminal]
----
NAME                            READY      SECRET                        AGE
spire-server-federation-tls     True       spire-server-federation-tls   2m
----

. Check the certificate details by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig describe certificate $TLS_SECRET_NAME -n zero-trust-workload-identity-manager
----

. Patch the `SpireServer` with the external secret by downloading the patch script: `patch-external-tls-cert.sh`.

. Apply the patch script by running the following command:
+
[source,terminal]
----
$ ./patch-external-tls-cert.sh \
  -k /path/to/cluster3/kubeconfig \
  -s spire-server-federation-tls
----

.Verification

. Check the federation status by running the following commands:
+
[source,terminal]
----
# Cluster 1
$ oc --kubeconfig=/path/to/cluster1/kubeconfig get clusterfederatedtrustdomains
----
+
----
# Cluster 2
$ oc --kubeconfig=/path/to/cluster2/kubeconfig get clusterfederatedtrustdomains
----
+
----
# Cluster 3
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get clusterfederatedtrustdomains
----
+
.Example output
[source, terminal]
----
NAME                       TRUST DOMAIN                   AGE
cluster-12-federation      apps.cluster2.example.com      5m
cluster-13-federation      apps.cluster3.example.com      5m
----

. Check that one of the federation endpoints is available:

.. Get federation endpoint URL for cluster 3 by running the following command:
+
[source,terminal]
----
$ FEDERATION_URL=$(oc --kubeconfig=/path/to/cluster3/kubeconfig get route spire-server-federation -n zero-trust-workload-identity-manager -o jsonpath='{.spec.host}')

$ echo "Federation endpoint: https://$FEDERATION_URL"
----

.. Test the endpoint by running the following command:
+
[source,terminal]
----
$ curl -v https://$FEDERATION_URL
----

.. Fetch the trust bundle by running the following command:
+
[source,terminal]
----
$ curl https://$FEDERATION_URL > bundle.json
$ cat bundle.json | jq
----

. Check that all SPIRE resources are available.

.. Check that all SPIRE pods are available by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get pods -n zero-trust-workload-identity-manager
----

.. Check the SPIRE Server logs by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig logs -n zero-trust-workload-identity-manager spire-server-0 -c spire-server --tail=50
----

.. Check that the routes are available by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get routes -n zero-trust-workload-identity-manager
----

. Check that the certificate is available:

.. Check the certificate status by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get certificate -n zero-trust-workload-identity-manager
----

.. Check the secret details by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig describe secret spire-server-federation-tls -n zero-trust-workload-identity-manager
----

.. Check the certificate's expiry by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get secret spire-server-federation-tls -n zero-trust-workload-identity-manager -o jsonpath='{.data.tls\.crt}' | base64 -d | openssl x509 -noout -dates
----

.. Check the role-based access control (RBAC) by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get role,rolebinding -n zero-trust-workload-identity-manager | grep secret-reader
----
