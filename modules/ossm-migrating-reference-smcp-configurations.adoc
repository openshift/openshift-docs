= Configuration fields mapping between Service Mesh 2 and Service Mesh 3

Many of the `ServiceMeshControlPlane` `spec.` configuration fields in {SMProductName} 2 can be configured in the `Istio` resource in {SMProduct} 3.

The following tables provide guidance for configuring your `Istio` resource in {SMProduct} 3.

== Mode configurations

The mode configurations in the {SMProduct} 2 `ServiceMeshControlPlane` resource were:

* Multitenant
* Cluster-wide
* Federation

In {SMProduct} 3, the mode is not configured using a single field in the `Istio` resource.

By default, the {SMProduct} 3 control plane has access to all namespaces which is equivalent to cluster-wide mode in {SMProduct} 2. For a similar configuration to multitenant mode in {SMProduct} 2 in {SMProduct} 3, you need to use the `discoverySelectors` field. See "Deploying multiple service meshes on a single cluster".

== Profile configurations

The profile configuration options for {SMProduct} 3 are:

* ambient
* default
* demo
* empty
* openshift-ambient
* openshift
* preview
* stable
