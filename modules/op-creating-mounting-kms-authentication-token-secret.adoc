= Creating and mounting the KMS authentication token secret

You can provide the authentication token for the KMS server (the value of `VAULT_TOKEN`) using a secret. You must create this secret, mount it on the {tekton-chains} controller, and set the `signers.kms.auth.token-dir` parameter to the directory where the secret is mounted.

.Prerequisites

* You installed the `oc` command-line utility.
* You are logged in to your {OCP} claster with administrative rights for the `tekton-chains` namespace.

.Procedure

. Create a secret named `kms-secrets` with the `KMS_AUTH_TOKEN` file that contains the authentication token for the KMS server by entering the following command:
+
[source, terminal]
----
$ oc create secret generic kms-secret -n tekton-chains \ #
  --from-file=KMS_AUTH_TOKEN=/home/user/KMS_AUTH_TOKEN # <1>
----
<1> The full path and name of the `KMS_AUTH_TOKEN` file that contains the authentication token for the KMS server.

. In the `TektonConfig` custom resource (CR), in the `chain` section, configure mounting the secret on the {tekton-chains} controller and set the `signers.kms.auth.token-dir` parameter to the directory where the secret is mounted, as shown in the following example.
+
.Example configuration for mounting the `kms-secrets` secret
[source,yaml]
----
apiVersion: operator.tekton.dev/v1
kind: TektonConfig
metadata:
  name: config
spec:
# ...
  chain:
    disabled: false
    signers.kms.auth.token-dir: /etc/kms-secrets
    options:
      deployments:
        tekton-chains-controller:
          spec:
            template:
              spec:
                containers:
                - name: tekton-chains-controller
                  volumeMounts:
                  - mountPath: /etc/kms-secrets
                    name: kms-secrets
                volumes:
                -  name: kms-secrets
                   secret:
                    secretName: kms-secrets
# ...
----
