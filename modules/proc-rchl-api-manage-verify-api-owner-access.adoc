= Verifying the API owner access role

[role="_abstract"]
You can verify that your role-based-access controls are configured the way you intend by using the CLI.

[NOTE]
====
You can replace `--as-group` with `--as=_<username>_` or `--as=system:serviceaccount:_<namespace>_:_<name>_` as needed.
====

.Prerequisites

* You enabled the {prodname} {ocp} web console plugin.
* You have the `api-admin` role bound cluster-wide.
* You configured other role bindings as required for your use case.

.Procedure

. Verify API owner `APIProduct` creation permissions in their namespace by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc auth can-i create apiproducts \
  --as-group=_<owner_group>_ \
  -n _<owner_namespace>_
----
+
* Replace `_<owner_group>_` with the name of the API owner group.
* Replace `_<owner_namespace>_` with the API owner namespace.
+
.Example output
[source,terminal]
----
yes
----

. Verify that API owners cannot access consumer `APIKey` resources by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc auth can-i get apikeys \
  --as-group=_<owner_group>_ \
  -n _<consumer_namespace>_
----
+
* Replace `_<owner_group>_` with the name of the API owner group.
* Replace `_<consumer_namespace>_` with the API consumer namespace.
+
.Example output
[source,terminal]
----
no
----
