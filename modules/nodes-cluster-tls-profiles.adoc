= About cluster-wide TLS profiles

You can use a `KubeletConfig` object to configure a custer-wide TLS security profile by defining the schema for a TLS security profile.

The default security profiles are as follows:

[source,yaml]
----
kind: KubeletConfiguration
apiVersion: kubelet.config.k8s.io/v1beta1
 ...
tlsCipherSuites:
  - TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256
  - TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
  - TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384
  - TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
  - TLS_ECDHE_ECDSA_WITH_CHACHA20_POLY1305_SHA256
  - TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305_SHA256
----

There are three TLS security profile types that you can configure:

* `Old`. Use with very old clients or libraries. This profile is based on link:https://wiki.mozilla.org/Security/Server_Side_TLS#Recommended_configurations[recommended configurations].
+
.Sample `Old` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Old
----

* `Intermediate`. Use as a general-purpose server.  This profile is based on link:https://wiki.mozilla.org/Security/Server_Side_TLS#Recommended_configurations[recommended configurations].
+
.Sample `Intermediate` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Intermediate
----

* `Modern` Use with modern clients that support TLS 1.3 with no need for backwards compatibility.
+
.Sample `Modern` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Modern
----
////

* `Custom`. Use with specific, individual TLS security profile parameters.
+
[WARNING]
====
You must be careful using a `Custom` profile, because invalid configurations can cause problems.
====
+
.Sample `Custom` profile
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Custom
    custom:
      ciphers:
        - ECDHE-ECDSA-AES128-GCM-SHA256
        - ECDHE-RSA-AES128-GCM-SHA256
      minTLSVersion: VersionTLS11
----
+
For the `Custom` you can also set the version of TLS to allow:
+
* `VersionTLS10` to specify TLS version 1.0
* `VersionTLS11` to specify TLS version 1.1
* `VersionTLS12` to specify TLS version 1.2
* `VersionTLS13` to specify TLS version 1.3

You can see the current TLS profiles by viewing the API server custom resource (CR) or the `/etc/kubernetes/kubelet.conf` file on each node.

.Sample `kubelet.conf` file with the `Intermediate` TLS profile

[source,json]
----
{
  "kind": "KubeletConfiguration",
  "apiVersion": "kubelet.config.k8s.io/v1beta1",
  "staticPodPath": "/etc/kubernetes/manifests",
  "syncFrequency": "0s",
  "fileCheckFrequency": "0s",
  "httpCheckFrequency": "0s",
  "tlsCipherSuites": [
    "TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256",
    "TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256",
    "TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384",
    "TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384",
    "TLS_ECDHE_ECDSA_WITH_CHACHA20_POLY1305_SHA256",
    "TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305_SHA256"
  ],
  "tlsMinVersion": "VersionTLS12",
 ... 
----
