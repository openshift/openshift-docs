= Checking cluster resources before initiating the limited live migration

The following procedure shows you how to check metrics for EgressIPs resources, EgressFirewalls resources, and multicast enabled namespaces. These resources are migrated to OVN-Kubernetes, however, they might behave differently after migration. Users should check these resources before and after migration to ensure the intended behavior.

.Prerequisites

* You have access to the cluster as a user with the `cluster-admin` role.

.Procedure

. Check your cluster for EgressFirewall resources:

.. On the {product-title} web console, navigate to *Observe* -> *Metrics*.

.. In the *Expression* box, enter `sdn_controller_num_egress_firewalls`.

.. Click *Add* -> *Run queries*. Note the intended behavior of this resource, as it might change after migration.

. Check your cluster for EgressIP resources:

.. On the {product-title} web console, navigate to *Observe* -> *Metrics*.

.. In the *Expression* box on the *Metrics* page, enter `sdn_controller_num_egress_ips`.

.. Click *Add* -> *Run queries*. Note the intended behavior of this resource, as it might change after migration.

. Check your cluster for multicast enabled namespaces:

.. On the {product-title} web console, navigate to *Observe* -> *Metrics*.

.. In the *Expression* box on the *Metrics* page, enter `sdn_controller_num_multicast_enabled_namespaces`.

.. Click *Add* -> *Run queries*. Note the intended behavior of this resource, as it might change after migration.
