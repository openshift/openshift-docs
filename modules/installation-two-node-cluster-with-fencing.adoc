= Sample install-config.yaml for a two-node OpenShift cluster with fencing

You can use the following `install-config.yaml` as a template for deploying a two-node OpenShift cluster with fencing.

.Sample install-config.yaml for installing a two-node OpenShift cluster with fencing
[source,yaml]
----
apiVersion: v1
baseDomain: example.com
compute:
- name: worker
  replicas: 0 <1>
controlPlane:
  name: master
  replicas: 2 <2>
  fencing:
    credentials:
      - hostname: <control-0-hostname> <3>
        address: https://<redfish-api-url>
        username: <username>
        password: <password>
      - hostname: <control-1-hostname> <3>
        address: https://<redfish-api-url>
        username: <username>
        password: <password>
metadata:
  name: <cluster-name> <4>
featureSet: TechPreviewNoUpgrade <5>
platform:
  baremetal:
    apiVIPs:
      - <api_ip> <6>
    ingressVIPs:
      - <wildcard_ip> <6>
pullSecret: '<pull-secret>' <7>
sshKey: '<ssh-public-key>' <8>
----
<1> Set this field to `0` because a two-node fencing cluster does not include worker nodes.
<2> Set this field to `2` for a two-node fencing deployment.
<3> Provide BMC credentials for each control plane node. These credentials are required for node fencing and prevent split-brain scenarios.
<4> Cluster name is used as a prefix for hostnames and DNS records.
<5> Set this field to `TechPreviewNoUpgrade` to enable two-node OpenShift cluster deployments.
<6> Virtual IPs for the API and Ingress endpoints. Ensure they are reachable by all nodes and external clients.
<7> Contains credentials required to pull container images for the cluster components.
<8> The SSH public key for accessing cluster nodes after installation.
