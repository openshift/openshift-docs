= About identity providers in {product-title}

[role="_abstract"]
You can configure identity providers by creating a custom resource (CR) that describes the provider and adding it to the cluster. Identity providers enable user authentication in {product-title} beyond the default `kubeadmin` user.

[NOTE]
====
{product-title} user names containing `/`, `:`, and `%` are not supported.
====
