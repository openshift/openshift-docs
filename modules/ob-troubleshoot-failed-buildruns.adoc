= Troubleshoot Failed BuildRuns

[role="_abstract"]
Diagnose and resolve build failures using comprehensive error information captured in BuildRun status fields. The build controller records failure details including error messages, reasons, and the specific location where failures occurred, enabling faster troubleshooting and resolution.

Build failure troubleshooting includes:

* Failed build details: Comprehensive error information in the `status.failureDetails` field, including the container or pod where the failure occurred and specific error messages
* Build run status codes: Understanding different failure reasons such as missing strategies, parameter validation errors, timeout issues, and pod eviction
* Error location tracking: Identifying the exact step, container, or pod where a build failed to target investigation efforts
* Common failure scenarios: Recognizing patterns like missing service accounts, invalid parameter values, repository access issues, and resource constraints

These troubleshooting capabilities help platform engineers and developers quickly identify root causes of build failures and implement corrections. The build controller validates configurations before execution and captures detailed error context when builds fail.
