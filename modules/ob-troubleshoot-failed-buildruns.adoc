= Troubleshoot failed BuildRuns

[role="_abstract"]
Diagnose and resolve build failures by using comprehensive error information captured in `BuildRun` status fields. The build controller records failure details including error messages, reasons, and the specific location where failures occurred, enabling faster troubleshooting and resolution.

Failed build details:: Comprehensive error information in the `status.failureDetails` field, including the container or pod where the failure occurred and specific error messages.
Build run status codes:: Understanding different failure reasons such as missing strategies, parameter validation errors, timeout issues, and pod eviction.
Error location tracking:: Identifying the exact step, container, or pod where a build failed to target investigation efforts.
Common failure scenarios:: Recognizing patterns such as missing service accounts, invalid parameter values, repository access issues, and resource constraints.
