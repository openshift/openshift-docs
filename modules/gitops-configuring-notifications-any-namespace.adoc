= Configuring Notifications in any Namespace

By default, Argo CD manages notification configuration only within the control plane namespace. With Red Hat OpenShift GitOps, cluster administrators can enable teams to manage notification settings for their Applications from additional namespaces. This delegation allows teams to configure their own triggers, templates, and subscriptions without requiring administrator access to the control plane namespace.

To enable this functionality, you configure the target namespaces in the ArgoCD custom resource (CR). The OpenShift GitOps Operator reconciles the corresponding notification resources only for namespaces that are explicitly defined in the ArgoCD CR.

To enable notification configuration in a namespace, you must add the namespace to the following fields in the ArgoCD CR:

* `spec.sourceNamespaces`: Enables the `Apps in Any Namespace` feature for the Application controller.
* `spec.notifications.sourceNamespaces`: Allows the Notifications controller to read configuration from that namespace.

If a namespace is missing from either field, notification configuration in that namespace is not processed.

.Procedure

. List the ArgoCD CRs in the cluster:
+
[source,terminal]
----
$ oc get argocd -A
----

. Edit the target ArgoCD CR:
+
[source,terminal]
----
$ oc edit argocd <cr_name> -n <namespace>
----

. Under the `spec` section, add each target namespace to both `sourceNamespaces` and `notifications.sourceNamespaces`. Wildcards are not supported.
+
The following example enables the `example-argocd` instance to manage Applications and notification configurations in the `foo` namespace:
+
[source,yaml]
----
apiVersion: argoproj.io/v1alpha1
kind: ArgoCD
metadata:
  name: example-argocd
spec:
  sourceNamespaces:
    - foo
  notifications:
    enabled: true
    sourceNamespaces:
      - foo
----
