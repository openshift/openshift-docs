= SPIRE integration with Red Hat OpenShift Service Mesh

[role="_abstract"]
You can integrate {SMProductName} with {zero-trust-full} to have Envoy sidecars obtain their mutual TLS (mTLS) certificates from SPIRE instead of Istio's built-in certificate authority (CA).

SPIRE provides cryptographic workload identities based on the SPIFFE standard. This integration enables a zero trust security model where workload identities are cryptographically verified rather than relying on network-based authentication.

SPIRE integration provides the following benefits:

* *SPIRE-issued workload identities*: Each workload receives a unique SPIFFE ID that provides a cryptographic identity.

* *Short-lived certificates*: SPIRE issues certificates with approximately 1-hour time-to-live (TTL), which are automatically rotated. This reduces the risk window if credentials are compromised.

* *Zero trust architecture*: Workload identity is cryptographically verified, eliminating the need to trust network boundaries.
