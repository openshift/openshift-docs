= Supported installation methods

The following table shows the level of certificate authority (CA) rotation support based on the installation method you use for Central and secured clusters:

.Supported installation methods
[cols="2,2,2,2",options="header"]
|===
|Component |Installation method |CA rotation support |Notes

|Central
|Operator-installed
|Full support
|User action is required for migration.

|Secured cluster
|Operator-installed
|Full support
|Migration is fully automated.

|Secured cluster
|Helm or manifests
|Partial support
|The secured cluster can connect to Central after Central has rotated its CA, while the old CA is valid, but can not migrate to the new CA.

|===
