= Supported installation methods

The following table shows the level of certificate authority (CA) rotation support based on the installation method you use for Central and secured clusters:

.Supported installation methods
[cols="2,2,2,2",options="header"]
|===
|Component |Installation method |CA rotation support |Notes

|Central
|Operator-installed
|Full support
|Migration is fully automated.

|Central
|Helm or manifest
|Not supported
|Use the Operator for automatic CA rotation.

|Secured cluster
|Operator-installed
|Full support
|Migration is fully automated provided the connected Central instance is also Operator-installed.

|Secured cluster
|Helm or manifest
|Partial support
|Can connect to Central with a rotated CA, but can not rotate its own service certificates to the new CA. Manual re-registration is required before the old CA expires.

|===
