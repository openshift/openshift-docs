= About replacing a healthy etcd member

[role="_abstract"]
When you replace a healthy etcd member, the cluster and etcd remain operational throughout the replacement process.

The process to replace a single healthy etcd member depends on how your cluster was installed and whether the cluster uses the Machine API and a control plane machine set.

[NOTE]
====
If the etcd member is unhealthy because the machine is not running, the node is not ready, or the etcd pod is crashlooping, see Replacing an unhealthy etcd member in Additional resources instead of this procedure.

If you have lost the majority of your control plane hosts, see Restoring to a previous cluster state in Additional resources instead of this procedure.
====
