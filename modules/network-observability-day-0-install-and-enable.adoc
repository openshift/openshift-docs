= Monitoring network traffic from day zero

[role="_abstract"]
Monitor network traffic immediately after cluster creation by enabling network observability during installation.

Network observability is enabled by default on standard {product-title} 4.22 clusters. Explicitly configure this setting in the `install-config.yaml` file to ensure the Network Observability Operator is installed during cluster creation.

.Prerequisites

* You are using {product-title} 4.22.
* For Technology Preview only, you must enable the `NetworkObservabilityInstall` feature gate.
* You must have access to the `install-config.yaml` file before creating installation manifests.

.Procedure

. Edit the `install-config.yaml` file before creating installation manifests.

. Add the `networkObservability` field to the `networking` section:
+
[source,yaml]
----
apiVersion: v1
baseDomain: example.com
metadata:
  name: my-cluster
networking:
  networkObservability:
    installationPolicy: InstallAndEnable
# ...
----
+
where:
+
`networking.networkObservability.installationPolicy`:: Controls whether the Network Observability Operator is installed during cluster creation. Set to `InstallAndEnable` to install the operator automatically. Valid values are `InstallAndEnable` and `DoNotInstall`. The default value is `InstallAndEnable` for standard clusters when the `NetworkObservabilityInstall` feature gate is enabled.

. Create the installation manifests.

. Enable the `NetworkObservabilityInstall` feature gate.

. Install the cluster as documented in the installation guide for your platform.

.Verification

. Verify that the Network Observability Operator is installed and running by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-netobserv-operator
----
+
.Example output
[source,terminal]
----
NAME                                                READY   STATUS    RESTARTS   AGE
netobserv-controller-manager-6d4b8d9f8c-xk7lm      2/2     Running   0          5m
----

. Verify that the `FlowCollector` resource was created by running the following command:
+
[source,terminal]
----
$ oc get flowcollector cluster -n netobserv
----
+
.Example output
[source,terminal]
----
NAME      AGENT   SAMPLING   DEPLOYMENT MODEL   STATUS
cluster   eBPF    50         Direct             Ready
----

. Verify that network traffic data is available in the web console by navigating to *Observe* -> *Network Traffic*.
