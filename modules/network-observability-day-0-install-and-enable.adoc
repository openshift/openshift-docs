= Enable network observability during installation

[role="_abstract"]
Configure network observability to ensure installation on all cluster types, including {sno} clusters, with continuous enforcement that reinstalls the operator if deleted.

When you set the installation policy to `InstallAndEnable`, the Cluster Network Operator ensures that the Network Observability Operator remains installed throughout the cluster lifecycle. This setting overrides the default behavior that skips installation on {sno} clusters, and reinstalls the operator if users delete it.

.Prerequisites

* You must enable the `NetworkObservabilityInstall` feature gate. For more information, see _Enabling the network observability feature gate_.
* You must have created the installation manifests directory.

.Procedure

. Create a Network CR manifest file in the `manifests` directory:
+
[source,terminal]
----
$ cat <<EOF > manifests/99-network-observability.yaml
apiVersion: config.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  networkObservability:
    installationPolicy: InstallAndEnable
EOF
----
+
where:
+
`spec.networkObservability.installationPolicy`:: Controls whether the Network Observability Operator is installed and maintained during cluster operation. Set to `InstallAndEnable` to ensure continuous installation. Valid values are `InstallAndEnable` and `NoAction`. When omitted, network observability installs on multi-node clusters but not on {sno} clusters, and is not reinstalled if deleted.

. Create the cluster by running the following command:
+
[source,terminal]
----
$ openshift-install create cluster
----

.Verification

. Verify that the Network Observability Operator is installed and running by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-netobserv-operator
----
+
.Example output
[source,terminal]
----
NAME                                                READY   STATUS    RESTARTS   AGE
netobserv-controller-manager-6d4b8d9f8c-xk7lm      2/2     Running   0          5m
----

. Verify that the `FlowCollector` resource was created by running the following command:
+
[source,terminal]
----
$ oc get flowcollector cluster -n netobserv
----
+
.Example output
[source,terminal]
----
NAME      AGENT   SAMPLING   DEPLOYMENT MODEL   STATUS
cluster   eBPF    50         Direct             Ready
----

. Verify that network traffic data is available in the web console by navigating to *Observe* -> *Network Traffic*.
