= Overview

Compute Cloud@Customer (C3) lets you run applications and middleware using OCI services on high performance cloud infrastructure in your data center. You can quickly deploy applications using a rack-scale platform with optional GPU acceleration and infrastructure-as-code automation. 

You can install {product-name} on Oracle Compute Cloud@Customer by using the {ai-full}. 

.Installation process

The installation process involves building two hosts within the designated compartment of the OpenShift cluster:

* The first instance is a `bastion` host, which is used to run two Terraform scripts:
+
--
** The first script builds IAM Resources on your OCI Home Region of the C3 (2 Dynamic Groups & 1 Policy).
** The second script builds the Infrastructure Resources on the C3 System to support the OpenShift Cluster (the OpenShift VCN,  Public and Private Subnets, Load Balancers, Internet GW, NAT GW, and DNS Server). It includes all the resources needed to activate the control plane nodes and compute nodes that form a cluster. 
--
+
The bastion is installed in the designated OpenShift Compartment and configured to communicate through a designated C3 DRG Subnet or Internet GW Subnet within the C3’s parent tenancy. 

* The second instance is the `jumpserver`. The jumpserver is configured as a web server to communicate to the Openshift Cluster. The jumpserver is installed with the OpenShift Client CLI, within the OpenShift VCN built by the Terraform scripts.

Finally, three Control Plane Nodes and three Worker Nodes are also provisioned, together with the external and internal Load Balancers that form the Cluster. 

.Main steps

The procedure is broken down into the following sections:

. Installing Terraform and OCI CLI on a C3 Bastion Server 

. Terraform Script Execution – Part 1  (Run Script via Home region) 

. OpenShift Image Preparation to OCI 

. Terraform Script Execution – Part 2  (“Run Script via C3 Region”) 

. Install the Cluster using the RH Assisted Installer UI
