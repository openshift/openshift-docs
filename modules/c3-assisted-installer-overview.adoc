= Overview

{oci-c3} lets you run applications and middleware using {oci-first} services on high performance cloud infrastructure in your data center. You can quickly deploy applications using a rack-scale platform with optional GPU acceleration and infrastructure-as-code automation. 

You can install {product-title} on {oci-c3-no-rt} by using the {ai-full}. 

.Installation process

The installation process involves building two hosts within the designated compartment of the {product-title} cluster:

* The first instance is a `bastion` host, which is used to run two Terraform scripts:
+
--
** The first script builds IAM Resources in the {oci} Home Region of the {oci-c3-short} system (two Dynamic Groups and one Policy).

** The second script builds the Infrastructure Resources on the {oci-c3-short} system to support the OpenShift cluster (the OpenShift VCN,  Public and Private Subnets, Load Balancers, Internet GW, NAT GW, and DNS Server). It includes all the resources needed to activate the control plane nodes and compute nodes that form a cluster. 
--
+
The bastion host is installed in the designated OpenShift Compartment and configured to communicate through a designated {oci-c3-short} DRG Subnet or Internet GW Subnet within the {oci-c3-short} parent tenancy. 

* The second instance is the `jumpserver`. The jumpserver is configured as a web server to communicate to the Openshift cluster. The jumpserver is installed with the {product-title} client CLI, within the RHOCP VCN built by the Terraform scripts.

The installation process subsequently provisions three control plane (master) nodes and three compute (worker) noodes, together with the external and internal Load Balancers that form the cluster. 

.Main steps

The main steps of the procedure are as follows:

. Installing the Terraform script and the {oci} command line interface (CLI) on an {oci-c3-short} bastion server.

. Executing the Terraform script (part 1) - Running the script via the Home region. 

. Preparing the OpenShift image for {oci-first-no-rt}. 

. Executing the Terraform script (Part 2) - Running the rcript via the {oci-c3-short} region 

. Installing the cluster using the {ai-full} web console
