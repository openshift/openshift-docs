|Field Name |Description

|`spec.caseID`
|This field is used to specify the Red{nbsp}Hat Support case ID to which the collected diagnostic output will be uploaded. This is a crucial field for integrating with the Red Hat case management system [Conversation History].

|`spec.caseManagementAccountSecretRef.name`
|This field provides a reference to the **name of a Kubernetes secret**. This secret must contain the credentials necessary for authenticating and uploading files to the Red Hat Customer Portal support case [Conversation History].

|`spec.serviceAccountRef.name`
|This is an optional field that specifies the name of the **service account** that the operator will use to execute the `must-gather` command. It is critical to note that this service account **must have the `cluster-admin` role** due to the elevated permissions required for the diagnostic data collection operations [Conversation History].

|`spec.mustGatherImages`
|This is an optional array of strings, where each string represents the name of an **additional `must-gather` compatible image**. These images are used to collect specific diagnostic information for various OpenShift add-ons or components, such as Kubevirt or OpenShift Container Storage (OCS), beyond the standard cluster data [Conversation History].
