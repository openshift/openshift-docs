= Creating a hosted cluster and a default node pool

You can create a hosted cluster and a default node pool on OpenStack by using the `hcp` CLI tool.

.Prerequisites

* You have installed the HyperShift and HCP CLI tools.
* You have deployed the HyperShift Operator.
* You have prepared the management cluster to store etcd locally.
* Optional: You have uploaded the {op-system} image to OpenStack.
* Optional: You have created a floating IP for ingress.
* Optional: You have updated the DNS record for ingress.

.Procedure

. Create a hosted cluster and a default node pool:
+
[source,terminal]
----
hcp create cluster openstack \
  --name=<cluster_name> \
  --node-pool-replicas=<number_of_nodes> \
  --pull-secret=<pull_secret_file> \
  --openstack-cloud=<cloud_name> \
  --openstack-cloud-ca-bundle=<ca_bundle_file> \
  --openstack-cloud-config=<cloud_config_file> \
  --openstack-external-network=<external_network_id> \
  --openstack-image=<image_name> \
  --openstack-flavor=<flavor_name> \
  --openstack-network=<network_name> \
  --openstack-subnet=<subnet_name> \
  --openstack-ingress-floating-ip=<floating_ip>
----
+
where:
+
--
* `<cluster_name>` is the name of the hosted cluster.
* `<number_of_nodes>` is the number of nodes in the default node pool.
* `<pull_secret_file>` is the path to your pull secret file.
* `<cloud_name>` is the name of the OpenStack cloud configuration in `clouds.yaml`.
* `<ca_bundle_file>` is the path to the CA bundle file for the OpenStack cloud.
* `<cloud_config_file>` is the path to your `clouds.yaml` file.
* `<external_network_id>` is the ID of the external network.
* `<image_name>` is the name of the {op-system} image.
* `<flavor_name>` is the name of the OpenStack flavor to use for the nodes.
* `<network_name>` is the name of the OpenStack network.
* `<subnet_name>` is the name of the OpenStack subnet.
* `<floating_ip>` is the floating IP address for ingress.
--
+
[NOTE]
====
If you do not specify the `--openstack-image` flag, OpenStack Resource Controller (ORC) downloads the {op-system} image from the OpenShift mirror and manages its lifecycle.
====

. Monitor the hosted cluster creation:
+
[source,terminal]
----
oc get hostedclusters -n clusters
----
+
.Example output
[source,terminal]
----
NAME            VERSION   KUBECONFIG                    PROGRESS   AVAILABLE   PROGRESSING   MESSAGE
<cluster_name>  4.14.0    <cluster_name>-admin-kubeconfig   Completed   True        False         The hosted control plane is available
----

. After the hosted cluster is created, you can access it by using the generated kubeconfig file:
+
[source,terminal]
----
export KUBECONFIG=<cluster_name>-admin-kubeconfig
oc get nodes
----
