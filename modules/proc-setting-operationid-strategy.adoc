= Setting the operation ID strategy

[role="_abstract"]
The `kogito.sw.operationIdStrategy` property controls how {ServerlessLogicProductName} generates configuration keys for REST clients. You can set this property to find the naming convention that best fits your project structure and deployment requirements.

.Prerequisites

* You have access to a {ServerlessLogicProductName} project with the appropriate roles and permissions to create applications and other workloads in {ocp-product-title}.
* You have created your {ServerlessLogicProductName} project.
* You have access to the OpenAPI specification files.

.Procedure

. Open the `application.properties` file in your {ServerlessLogicProductName} project directory.

. Add the `kogito.sw.operationIdStrategy` property and set it to one of the following values:
+
* `FILE_NAME`: Use when OpenAPI files are stored locally and have descriptive names. This is the default strategy.
* `FULL_URI`: Use when OpenAPI documents are located remotely and the URI path provides meaningful context.
* `FUNCTION_NAME`: Use when you want configuration keys based on workflow and function names for better traceability.
* `SPEC_TITLE`: Use when the OpenAPI document title provides a clear and unique identifier for the service.
+
.Example configuration for `FILE_NAME` strategy
[source,properties]
----
kogito.sw.operationIdStrategy=FILE_NAME
----
+
.Example configuration for `FUNCTION_NAME` strategy
[source,properties]
----
kogito.sw.operationIdStrategy=FUNCTION_NAME
----

. Save the `application.properties` file.

. Configure the REST client URL using the configuration key that corresponds to your selected strategy:
+
--
* For `FILE_NAME`, use the OpenAPI file name as the key.
* For `FULL_URI`, use the sanitized URI path as the key.
* For `FUNCTION_NAME`, use the workflow ID and function name combination as the key.
* For `SPEC_TITLE`, use the OpenAPI document title as the key.
--
+
.Example REST client configuration for `FILE_NAME` strategy
[source,properties]
----
quarkus.rest-client.stock_portfolio_svc_yaml.url=http://localhost:8282/
----

. Deploy or restart your application to apply the configuration changes.

.Verification

* Verify that your workflow successfully invokes the OpenAPI service using the configured REST client.
