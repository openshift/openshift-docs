= S3 Compatible storage

You can create an object storage secret for S3 Compatible storage providers to store logs. This includes NetApp StorageGRID and other S3-compatible object stores.

.Prerequisites
* You have S3 Compatible storage deployed and accessible.
* You created a bucket on your S3 Compatible storage.

.Procedure

* Create an object storage secret:
+
[source,terminal]
----
$ oc create secret generic logging-loki-s3 \  # <1>
  -n openshift-logging \
  --from-literal=bucketnames="<bucket_name>" \
  --from-literal=endpoint="<s3_endpoint_url>" \
  --from-literal=access_key_id="<access_key>" \
  --from-literal=access_key_secret="<secret_key>" \
  --from-literal=forcepathstyle="true"
----
<1> `logging-loki-s3` is the name of the secret.

[NOTE]
====
S3 Compatible storage requires `forcepathstyle="true"` and does not use the `region` parameter that AWS S3 requires.
====
