= Creating an htpasswd file using Linux

[role="_abstract"]
Create an htpasswd file on {op-system-base-full} with the `htpasswd` utility so the htpasswd identity provider can authenticate users in {product-title} from locally stored credentials.

To use the htpasswd identity provider, you must generate a flat file that contains the user names and passwords for your cluster. For more information, see "htpasswd".

.Prerequisites

* Have access to the `htpasswd` utility. On {op-system-base-full},
this is available by installing the `httpd-tools` package.

.Procedure

. Create or update your flat file with a user name and hashed password by running the following command:
+
[source,terminal]
----
$ htpasswd -c -B -b </path/to/users.htpasswd> <username> <password>
----
+
The command generates a hashed version of the password.
+
For example:
+
[source,terminal]
----
$ htpasswd -c -B -b users.htpasswd <username> <password>
----
+
.Example output
[source,terminal]
----
Adding password for user user1
----

. Continue to add or update credentials to the file:
+
[source,terminal]
----
$ htpasswd -B -b </path/to/users.htpasswd> <user_name> <password>
----
