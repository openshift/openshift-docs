= Build execution controls

[role="_abstract"]
Build execution controls determine how {builds-shortname} runs container image builds in your cluster.

You can configure pods, resources, volumes, and runtime parameters to optimize build performance and manage cluster resources effectively.
Controlling execution ensures builds complete efficiently without consuming excessive cluster capacity.

When you control build execution, you specify the compute resources allocated to each build pod, including CPU and memory limits.
This prevents individual builds from overwhelming your cluster while ensuring they have enough resources to complete successfully.
You can also configure volume mounts to provide build-time data, cache dependencies, or share artifacts between build stages.

Runtime parameters control build behavior such as timeout values, parallel execution, and pod scheduling constraints.
You can use node selectors and tolerations to run builds on specific nodes, separating build workloads from production applications.
Build execution controls also include security contexts and privilege settings that determine what operations builds can perform within the cluster.
By configuring these settings appropriately, you balance build performance, cluster resource use, and security requirements.
