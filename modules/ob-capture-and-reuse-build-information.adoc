= Capture and Reuse Build Information

[role="_abstract"]
Preserve build configurations and results for reproducibility, auditing, and troubleshooting through build snapshots and retention policies. The build controller captures execution details and manages the lifecycle of build resources to balance operational needs with cluster resource constraints.

Build information capture and management includes:

* Build snapshot: Immutable record of the exact `Build` specification used for a particular execution, stored in the `BuildRun` status for reproduction and audit purposes

This mechanism ensures build configurations remain traceable and reproducible for auditing and troubleshooting purposes.
