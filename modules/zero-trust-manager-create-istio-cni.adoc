= Creating the Istio Container Network Interface custom resource

[role="_abstract"]
Deploy the Istio Container Network Interface (CNI) plugin to enable pod traffic redirection to the Istio sidecar. This plugin provides transparent traffic interception without requiring elevated privileges in application pods.

.Prerequisites

* {zero-trust-full} is installed and running. See link:https://docs.redhat.com/en/documentation/openshift_container_platform/latest/html/security_and_compliance/zero-trust-workload-identity-manager#zero-trust-manager-install-console_zero-trust-manager-install[Installing the Zero Trust Workload Identity Manager].

* The {zero-trust-full} operands are deployed and running. See link:https://docs.redhat.com/en/documentation/openshift_container_platform/latest/html-single/security_and_compliance/index#zero-trust-manager-configuration[Deploying Zero Trust Workload Identity Manager operands].

* The {zero-trust-full} is in `create-only` mode. See link:https://docs.redhat.com/en/documentation/openshift_container_platform/latest/html-single/security_and_compliance/index#zero-trust-manager-reconciliation[Enabling create-only mode for the Zero Trust Workload Identity Manager]

* Cluster administrator access to {product-title}.

* OpenShift Service Mesh Operator (Sail Operator) installed from the web console. See link:https://docs.redhat.com/en/documentation/openshift_container_platform/latest/html-single/service_mesh/index#ossm-install-ossm-operator_installing-ossm[Installing the Operators]



.Procedure

. Define the environment variables for the Istio installation by running the following commands:
+
[source,terminal]
----
$ export ZTWIM_NS=zero-trust-workload-identity-manager
----
+
[source,terminal]
----
$ export TRUST_DOMAIN=ocp.one
----
+
[source,terminal]
----
$ export JWT_ISSUER="https://oidc-discovery.$(oc get ingresses.config/cluster -o jsonpath={.spec.domain})"
----
+
[source,terminal]
----
$ export OSSM_NS=istio-system
----
+
[source,terminal]
----
$ export OSSM_CNI=istio-cni
----
+
[source,terminal]
----
$ export EXTRA_ROOT_CA="$(oc get secret oidc-serving-cert -n ${ZTWIM_NS} -o json | jq -r '.data."tls.crt"' | base64 -d | sed 's/^/        /')"
----

. Create the `istio-cni` namespace by running the following command:
+
[source,terminal]
----
$ oc new-project "${OSSM_CNI}"
----

. Create the `IstioCNI` custom resource by running the following command:
+
[source,terminal]
----
$ oc apply -f - <<EOF
apiVersion: sailoperator.io/v1
kind: IstioCNI
metadata:
  name: default
spec:
  namespace: ${OSSM_CNI}
EOF
----
+
where: 

`spec.namespace`:: Specifies the namespace for the Istio CNI. The default value is `istio-cni`.

. Wait for the Istio CNI DaemonSet to become ready by running the following commands:
+
[source,terminal]
----
$ until oc get daemonset/istio-cni-node -n "${OSSM_CNI}" &> /dev/null; do sleep 3; done
----
+
[source,terminal]
----
$ kubectl rollout status daemonset/istio-cni-node -n "${OSSM_CNI}" --timeout=300s
----
