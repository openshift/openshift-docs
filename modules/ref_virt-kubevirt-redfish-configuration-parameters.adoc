= KubeVirt Redfish configuration parameters

[role="_abstract"]
Configure KubeVirt Redfish by editing the `config.yaml` file stored in the `kubevirt-redfish-secret` Secret.
This configuration controls server settings, VM selection, authentication, and storage for VirtualMedia operations.

The configuration is structured in YAML format with the following sections:

* `server`: Server connection settings
* `chassis`: VM groupings by namespace
* `authentication`: User credentials and access control
* `kubevirt`: KubeVirt API settings
* `datavolume`: Storage settings for VirtualMedia

.Server parameters
[cols="2,1,3",options="header"]
|===
|Parameter |Type |Description

|`server.host`
|String
|IP address the server listens on. Default: `0.0.0.0`.

|`server.port`
|Integer
|Port number for the Redfish API. Default: `8443`.

|`server.tls.enabled`
|Boolean
|Enable TLS for the server. When using an {product-title} Route with edge termination, set to `false`. Default: `false`.

|`system_id_convention`
|String
|Format for Redfish system IDs. Use `enhanced` for `<namespace>.<vm-name>` format or `legacy` for `<vm-name>` only. The default value is `legacy`.
|===

.Chassis parameters
[cols="2,1,3",options="header"]
|===
|Parameter |Type |Description

|`chassis`
|List
|List of chassis definitions. Each entry maps a Redfish chassis to VMs in a specific namespace.

|`chassis[].name`
|String
|Identifier for this chassis, used in Redfish URIs.

|`chassis[].namespace`
|String
|Kubernetes namespace containing the VMs to expose under this chassis.

|`chassis[].service_account`
|String
|Service account used to access VMs in this namespace.

|`chassis[].description`
|String
|Human-readable description of the chassis.

|`chassis[].vm_selector.labels`
|Map
|Label selector to filter which VMs in the namespace are exposed. Only VMs with matching labels are visible through Redfish.
|===

.Example chassis configuration
[source,yaml]
----
chassis:
  - name: "openshift-install"
    namespace: "openshift-install"
    service_account: "kubevirt-redfish"
    description: "Control plane VMs for cluster installation"
    vm_selector:
      labels:
        redfish-enabled: "true"
----

.Authentication parameters
[cols="2,1,3",options="header"]
|===
|Parameter |Type |Description

|`authentication.users`
|List
|List of users with API access.

|`authentication.users[].username`
|String
|Username for authentication.

|`authentication.users[].password`
|String
|Password for the user. For production deployments, store credentials securely.

|`authentication.users[].chassis`
|List
|List of chassis names this user can access.
|===

.Example authentication configuration
[source,yaml]
----
authentication:
  users:
    - username: "admin"
      password: "<secure_password>"
      chassis:
        - "openshift-install"
----

.KubeVirt parameters
[cols="2,1,3",options="header"]
|===
|Parameter |Type |Description

|`kubevirt.api_version`
|String
|KubeVirt API version to use. Default: `v1`.

|`kubevirt.timeout`
|Integer
|Timeout in seconds for KubeVirt API operations. Default: `30`.

|`kubevirt.allow_insecure_tls`
|Boolean
|Allow insecure TLS connections to the KubeVirt API. Default: `true`.
|===

.DataVolume parameters
[cols="2,1,3",options="header"]
|===
|Parameter |Type |Description

|`datavolume.storage_class`
|String
|Storage class for dynamically provisioned volumes used by VirtualMedia operations. Use a storage class available on your cluster, such as `lvms-vg1` or `ocs-storagecluster-ceph-rbd`.

|`datavolume.storage_size`
|Quantity
|Default size for provisioned volumes. Must accommodate the boot media (ISO images). Default: `10Gi`.

|`datavolume.allow_insecure_tls`
|Boolean
|Allow insecure TLS when downloading ISO images. Default: `false`.

|`datavolume.vm_update_timeout`
|Duration
|Timeout for VM update operations. Default: `30s`.

|`datavolume.iso_download_timeout`
|Duration
|Timeout for ISO image downloads. Default: `30m`.

|`datavolume.helper_image`
|String
|Container image used for data import operations. The default image is `registry.access.redhat.com/ubi8/ubi-minimal:latest`
|===

.Complete configuration example
[source,yaml]
----
server:
  host: "0.0.0.0"
  port: 8443
  tls:
    enabled: false

system_id_convention: "enhanced"

chassis:
  - name: "openshift-install"
    namespace: "openshift-install"
    service_account: "kubevirt-redfish"
    description: "Control plane VMs for cluster installation"
    vm_selector:
      labels:
        redfish-enabled: "true"

authentication:
  users:
    - username: "admin"
      password: "<secure_password>"
      chassis:
        - "openshift-install"

kubevirt:
  api_version: "v1"
  timeout: 30
  allow_insecure_tls: true

datavolume:
  storage_class: "lvms-vg1"
  storage_size: "3Gi"
  allow_insecure_tls: true
  vm_update_timeout: "2m"
  iso_download_timeout: "30m"
----

To apply configuration changes, update the Secret and restart the KubeVirt Redfish pods:

[source,terminal]
----
$ oc apply -f secret.yaml
$ oc rollout restart deployment/kubevirt-redfish -n kubevirt-redfish
----
