= Place a machine on a specific Dedicated Host

[role="_abstract"]
You can create an individual machine that is placed on a specific {aws-first} Dedicated Host. Because each machine targets a single host by its ID, you must create a separate `AWSMachine` resource for each Dedicated Host.

:FeatureName: {aws-short} Dedicated Host support
include::snippets/technology-preview.adoc[]

.Prerequisites

* You have access to the cluster as a user with the `cluster-admin` role.
* You have an existing {aws-short} Dedicated Host with an instance type that is compatible with the machine.

.Procedure

. Create a YAML file that defines the `AWSMachine` resource with the Dedicated Host configuration:
+
[source,yaml]
----
apiVersion: infrastructure.cluster.x-k8s.io/v1beta2
kind: AWSMachine
metadata:
  name: <machine_name>
  namespace: openshift-cluster-api
  labels:
    cluster.x-k8s.io/cluster-name: <cluster_name>
spec:
  ami:
    id: <ami_id>
  iamInstanceProfile: <infrastructure_id>-worker-profile
  instanceType: <instance_type>
  additionalSecurityGroups:
  - filters:
    - name: tag:Name
      values:
      - <infrastructure_id>-node
  - filters:
    - name: tag:Name
      values:
      - <infrastructure_id>-lb
  additionalTags:
    kubernetes.io/cluster/<infrastructure_id>: owned
  subnet:
    filters:
    - name: tag:Name
      values:
      - <infrastructure_id>-subnet-private-<zone>
  tenancy: host
  hostAffinity: host
  hostID: <dedicated_host_id>
----
+
where:
+
--
`<machine_name>`:: Specifies a name for the `AWSMachine` resource.
`<cluster_name>`:: Specifies the name of the cluster. You can obtain the cluster name by running the following command:
+
[source,terminal]
----
$ oc get infrastructure cluster -o jsonpath='{.status.infrastructureName}{"\n"}'
----
`<ami_id>`:: Specifies the {op-system-first} Amazon Machine Image (AMI) ID for your {aws-short} region.
`<infrastructure_id>`:: Specifies the infrastructure ID that is based on the cluster ID that you set when you provisioned the cluster.
`<instance_type>`:: Specifies the {aws-short} instance type, for example `m6i.xlarge`. The instance type must be compatible with the Dedicated Host instance family.
`<zone>`:: Specifies the availability zone where the Dedicated Host is located, for example `us-east-1a`.
`<dedicated_host_id>`:: Specifies the ID of the {aws-short} Dedicated Host on which to place the machine, for example `h-0123456789abcdef0`.
--

. Create the machine by running the following command:
+
[source,terminal]
----
$ oc create -f <file_name>.yaml
----

.Verification

* Verify that the machine was created by running the following command:
+
[source,terminal]
----
$ oc get awsmachine -n openshift-cluster-api
----
