= Installing the Loki Operator by using the web console

[role="_abstract"]
Install the {loki-op} to manage the `LokiStack` log store from the OperatorHub by using the {ocp-product-title} web console.

.Prerequisites

* You have administrator permissions.
* You have access to the {ocp-product-title} web console.
* You have access to a supported object store ({aws-short} S3, {gcp-short}, {azure-short}, MinIO, {rh-storage}, or Swift).

.Procedure

. In the {ocp-product-title} web console *Administrator* perspective:
* For {ocp-product-title} 4.19 or earlier, click *Operators -> OperatorHub*.
* For {ocp-product-title} 4.20 or later, click *Ecosystem -> Software Catalog*.

. Type *Loki Operator* in the *Filter by keyword* field. Click *{loki-op}* in the list of available Operators, and then click *Install*.
+
[IMPORTANT]
====
The Community {loki-op} is not supported by Red{nbsp}Hat. Select the operator provided by Red{nbsp}Hat.
====

. Select *stable-{logging-major-version}* as the *Update channel*.
+
The {loki-op} must be deployed to the global operator group namespace `openshift-operators-redhat`, so the *Installation mode* and *Installed Namespace* are already selected. If this namespace does not already exist, it is created for you.

. Select *Enable Operator-recommended cluster monitoring on this namespace*.
+
This option sets the `openshift.io/cluster-monitoring: "true"` label in the `Namespace` object. You must select this option to ensure that cluster monitoring scrapes the `openshift-operators-redhat` namespace.

. For *Update approval* select *Automatic*, then click *Install*.
+
[NOTE]
====
An Operator might display a `Failed` status before the installation completes. If the Operator installation completes with an `InstallSucceeded` message, refresh the page.
====

.Verification

. Navigate to the *Installed Operators* page.
. Verify that the {loki-op} displays with a *Status* of *Succeeded*.
