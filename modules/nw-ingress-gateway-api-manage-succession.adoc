= Preparing for Gateway API management succession by the Ingress Operator

In {product-title} {product-version}, the Ingress Operator manages the lifecycle of any Gateway API custom resource definitions (CRDs). Upgrading from a previous version of {product-title} where this management was not present requires you to make changes or remove any Gateway API CRDs that already exist in the cluster so that they conform to the specific {product-title} specification required by the Ingress Operator.

[WARNING]
====
Updating or deleting Gateway API resources can result in downtime and loss of service or data. Be sure you understand how this will affect your cluster before performing the steps in this procedure.
====

.Prerequisites
. Install the OpenShift CLI (`oc`)
. You have access to an {product-title} account with cluster administrator access.

.Procedure

* If you want to delete the Gateway API CRDs, run the following command:
+
[source,terminal]
----
$ oc delete crd gatewayclasses.gateway.networking.k8s.io
----

* If you want to edit the Gateway API CRDs, annotate the CRDs so that they use the correct version with the following command:
+
[source,terminal]
----
$ oc annotate crds referencegrants.gateway.networking.k8s.io gateway.networking.k8s.io/bundle-version=v1.1.1 --overwrite
----
+
Any controller that was previously managing the lifecycle of the Gateway API CRDs will no longer be available. Attempting to force their use in conjunction with the Ingress Operator to manage these CRDs will cause the cluster to enter into a `"Degraded"` state and prevent the upgrade from succeeding.
