= Configure Build Resources and Execution Strategies

[role="_abstract"]
Optimize cluster resource utilization by defining multiple variants of build strategies with different resource profiles and configuring controller-level settings. These configurations enable platform engineers to match build workloads with appropriate compute, memory, and storage allocations.

Build resource and execution configuration includes:

* Strategy variants: Multiple versions of the same build strategy (such as buildah-small and buildah-medium) with different CPU, memory, and disk limits for individual steps
* Resource profiles: Predefined resource allocations that users select based on their build complexity and performance requirements
* Controller settings: Environment variables that control build controller behavior, including timeout values, container images for Git operations, and concurrency limits
* Tekton pipeline resources: Understanding how the build controller manages resources through TaskRun creation and pod scheduling

These settings provide flexibility to balance build performance with cluster capacity. Platform engineers create strategy variants for common resource profiles, while controller settings establish cluster-wide defaults for build execution behavior.
