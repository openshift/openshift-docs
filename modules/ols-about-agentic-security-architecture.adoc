= Security architecture

[role="_abstract"]
The autonomous agentic troubleshooting system is built on multiple layers of defense to ensure safe operation.

The system enforces safety through the following mechanisms:

* *Policy enforcement*: The `RemediationPolicy` CRD defines which alerts the agent can act upon and what namespaces are in scope.

* *Dynamic RBAC*: Role-based access control is dynamically scoped, particularly when one non-deterministic agent instructs another for execution. Safety is enforced by the platform, not the agent.

* *Sandboxed execution*: All remediation actions execute in a sandboxed pod with least-privilege RBAC. The agent sandbox is managed by the `kubernetes-sigs/agent-sandbox` project, using `SandboxClaim`, `Sandbox`, and `SandboxTemplate` resources.

* *Human approval*: Every remediation execution requires explicit cluster administrator approval through the Decision Queue. The AI proposes, Kubernetes enforces, and humans decide.
