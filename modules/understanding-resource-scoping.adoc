= Understanding resource scoping

[role="_abstract"]
{product-title-short} resources use global, cluster, or namespace scoping levels, which determines how you can use them in custom access scopes for multi-tenancy.

{product-title-short} includes resources that RBAC uses.
In addition to associating permissions for a resource, {product-title-short} also scopes each resource.

In {product-title-short}, resources use the following scope types:

* Global scope, where the resource does not assign to any cluster or namespace
* Cluster scope, where you assign the resource to particular clusters
* Namespace scope, where you assign the resource to particular namespaces

The scope of resources is important when creating custom access scopes.
Custom access scopes create multi-tenancy within {product-title-short}.

Only resources that use cluster or namespace scoping are applicable for scoping in access scopes. Access scopes do not scope globally scoped resources.
Therefore, you can only achieve multi-tenancy within {product-title-short} for resources that use either cluster or namespace scoping.
