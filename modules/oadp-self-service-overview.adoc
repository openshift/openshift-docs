= {oadp-short} Self-Service overview

In the earlier versions of {oadp-full}, you needed to have cluster-wide administrator privileges to perform {oadp-short} operations such as backing up and restoring an application, creating a backup storage location, and so on. 

With {oadp-short} 1.5.0 onward, you do not need to have cluster-wide administrator privileges to perform the backup and restore operations. {oadp-short} Self-Service introduces a significant change to backup and restore operations in {product-title}.

The {oadp-short} Self-Service feature provides secure self-service data protection capabilities for users without administrator privileges, while maintaining proper access controls.

As a non-admin user, you can back up, and restore applications deployed in your authorized namespace on the cluster. This means, you need to have appropriate access to your namespace.

The {oadp-short} cluster administrator creates a non-admin user and provides the necessary access to the non-admin user to perform {oadp-short} Self-Service actions.

Note that, {oadp-short} Self-service does not support cross-cluster restore.

{oadp-short} Self-Service offers the following benefits:

* As a cluster administrator:
** You allow namespace-scoped backup and restore operations to a non-admin user. This means, a non-admin user cannot access a namespace that they are not authorized to.
** You keep administrator control over non-administrator operations through templates and policies.

* As a non-administrator:
** You can create dedicated backup storage locations in your authorized namespace.
** You have secure access to backup logs and status information.


== What namespace-scoped backup and restore means

The {oadp-short} Self-Service feature ensures that non-admin users can only operate within their authorized namespace and permissions. For example, if you do not have access to a namespace, as a non-admin user, you cannot back up that namespace.

The cluster administrator enforces the access control through custom resources that securely manage the backup and restore operations.

Non-admin users can:

* Create and manage backups of their authorized namespaces.
* Restore data to their authorized namespaces.
* Configure their own backup storage locations.
* Check backup and restore status.
* Access relevant logs.
