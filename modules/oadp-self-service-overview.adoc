= {oadp-short} Self-Service overview

In the earlier versions of {oadp-full}, you need to have cluster-wide administrator privileges to perform {oadp-short} operations such as backing up and restoring an application, creating a backup storage location, and so on. With {oadp-short} 1.5.0 onward, you do not need to have cluster-wide administrator privileges to perform the backup and restore operations. {oadp-short} Self-Service introduces a significant change to backup and restore operations in {product-title}.

The {oadp-short} Self-Service feature provides secure self-service data protection capabilities while maintaining proper access controls.

As a non-admin user, you can back up, and restore applications deployed in your namespace on the cluster. This means, you need to have administrator access to your namespace created on the cluster.

The {oadp-short} cluster administrator creates a non-admin user and provides the necessary access controls to the non-admin user to perform {oadp-short} Self-Service actions.

Note that, {oadp-short} Self-service does not support cross-cluster restore.


== Benefits of {oadp-short} Self-Service

* As a cluster administrator:
** You allow namespace-scoped backup and restore operations to a non-admin user.
** You maintain cluster administrator control over non-administrator operations through templates and policies.

* As a non-administrator:
** You can create dedicated backup storage locations.
** You have secure access to backup logs and status information.


== Namespace-scoped back up and restore

With {oadp-short} Self-Service, as a non-admin user, you can perform backup and restore operations within namespaces where you have appropriate permissions. This is achieved through custom resources that securely manage the backup and restore operations while maintaining access controls and visibility.

The Self-Service feature ensures that non-admin users can only operate within their assigned namespaces and permissions. For example, if you do not have access to a namespace, as a non-admin user, you cannot back up that namespace.

Non-admin users can:

* Create and manage backups of their authorized namespaces
* Restore data to their authorized namespaces
* Configure their own backup storage locations
* Monitor backup and restore status
* Access relevant logs
