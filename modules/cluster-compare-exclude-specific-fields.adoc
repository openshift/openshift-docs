= Excluding specific fields

[role="_abstract"]
You can specify fields to exclude by defining the path to the field, and then referencing the definition in the `config` section for a template.

.Procedure

* Create a `metadata.yaml` file to match your use case. The following example shows the structure of a `metadata.yaml` file with specific field exclusions:
+
[source,yaml]
----
apiVersion: v2
parts:
  - name: Part1
    components:
      - name: Component1
        - path: deployment.yaml
          config:
            fieldsToOmitRefs:
              - <fields_to_omit_ref>

#...

fieldsToOmit:
   items:
      deployments:
         - pathToKey: <path_to_key>
----
+
where:

`<fields_to_omit_ref>`:: References the `fieldsToOmit.items.deployments` item for the `deployment.yaml` template.
`<path_to_key>`:: Excludes the specified field from the comparison.
+
[NOTE]
====
Setting `fieldsToOmitRefs` replaces the default value.
====
