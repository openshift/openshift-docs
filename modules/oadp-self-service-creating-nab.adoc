= Creating a `NonAdminBackup` CR

As a non-admin user, you can create a `NonAdminBackup` (NAB) CR to back up your application from your authorized namespace. A NAB CR is an OpenShift custom resource that securely facilitates the creation of a `Velero` backup object. The `Velero` backup object reports the status back to the NAB CR that ultimately updates the `status.phase` field.

When you create a NAB CR:

* The initial phase for the CR is `New`. 
* The CR creation request goes to the `NonAdminController` (NAC) for reconciliation and validation. 
* Upon successful validation and creation of the `Velero` Backup object, the `status.phase` field of the NAB CR is updated to the next phase, namely, `Created`.

Following are some important points to take note of, when you create a NAB CR:

* The `NonAdminBackup` CR creates the `Velero` Backup object in a secure way so that other non-admin users cannot access the CR.
* As a non-admin user, you cannot specify the namespace to back up. The namespace from which the NAB CR is created, is the namespace to be backed up.
* Apart from creating the `Velero` Backup object, the main purpose of the `NonAdminBackup` CR is to track the status of the `Velero` Backup in a secure way.

.Prerequisites

* You are logged in to the cluster as a non-admin user.
* The cluster administrator has installed the {oadp-short} Operator.
* The cluster administrator has configured the `DataProtectionApplication` to enable {oadp-short} Self-Service.
* The cluster administrator has created a namespace for you and has authorized you to operate from that namespace.
* You have installed an application in your authorized namespace. 

.Procedure

. To create a `NonAdminBackup` CR, create a YAML manifest file with the following configuration:
+
.Example `NonAdminBackup`
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: NonAdminBackup
metadata:
  name: test-nab # <1>
spec:
  backupSpec:
    includedNamespaces:
    - "test-nac-ns" # <2>
    defaultVolumesToFsBackup: true
----
1. Specify a name for the NAB CR. In this example, `test-nab` is the name.
2. Specify the namespace that is authorized for you to create a NAB CR. In this example, `test-nac-ns` is the namespace.

. To create the NAB CR, run the following command:
+
[source,terminal]
----
$ oc apply -f <nab_cr_filename> # <1>
----
1. Specify the file name containing the NAB CR configuration.

.Verfication

. To verify that the NAB CR is successfully created, run the following command:
+
[source,terminal]
----
oc get nab test-nab -o yaml
----
+
