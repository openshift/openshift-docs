[id="fixed-issues-1-20-4_{context}"]
== Fixed issues

Dex initContainer updates are correctly applied during upgrades::
Before this update, updates to the Dex `copyutil` initContainer were not always applied during upgrades from earlier {gitops-title} versions. Changes to fields such as `resources` and `volumeMounts` in the Dex Deployment could be missed, including updates to the `dexconfig` volume mount and `resource` limits. With this update, the {gitops-title} Operator correctly reconciles updates to the Dex `copyutil` initContainer during upgrades. Changes to the initContainer configuration are now applied as expected to the Dex Deployment.
+
link:https://redhat.atlassian.net/browse/GITOPS-9549[GITOPS-9549]

Argo CD server service certificate annotation is preserved during upgrades::
Before this update, upgrading the {gitops-title} Operator from version 1.12.3 to version 1.12.4 removed the `service.beta.openshift.io/serving-cert-secret-name` annotation from the Argo CD server service. The Service-CA ClusterOperator could no longer manage the service certificate, which could cause HTTP 503 errors after the certificate expired. Because the change remained on the service object, the issue could continue even after you upgraded to later Operator versions. With this update, the {gitops-title} Operator reconciles the `service.beta.openshift.io/serving-cert-secret-name` annotation during upgrades. The Service-CA ClusterOperator can continue managing certificate rotation, helping prevent HTTP 503 errors caused by expired certificates.
+
link:https://redhat.atlassian.net/browse/GITOPS-8784[GITOPS-8784]
