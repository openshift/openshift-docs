= Resolving the KubePersistentVolumeFillingUp alarm firing for Prometheus

As a cluster administrator, you can resolve the `KubePersistentVolumeFillingUp` alarm being triggered for Prometheus. 

The critical alert fires when a persistent volume (PV) claimed by a `prometheus-k8s-*` pod in the `openshift-monitoring` project has less than 3% of its total space remaining. This can cause Prometheus to not function normally.

[NOTE]
====
There are two `KubePersistentVolumeFillingUp` alerts:

* *Critical alert*: It is triggered when the mounted PV has less than 3% of its total space remaining. 
* *Warning alert*: It is triggered when the mounted PV has less than 15% of its total space remaining and is expected to fill up within four days.
====

You can resolve this issue by freeing some space for the PV by removing the three oldest Prometheus time series database (TSDB) blocks.

.Prerequisites

ifndef::openshift-dedicated,openshift-rosa[]
* You have access to the cluster as a user with the `cluster-admin` cluster role.
endif::openshift-dedicated,openshift-rosa[]
ifdef::openshift-dedicated,openshift-rosa[]
* You have access to the cluster as a user with the `dedicated-admin` role.
endif::openshift-dedicated,openshift-rosa[]
* You have installed the OpenShift CLI (`oc`).

.Procedure

. Remove the three oldest Prometheus TSDB blocks to make more space available for the PV:
+
[source,terminal]
----
$ oc debug <prometheus_k8s_pod_name> -n openshift-monitoring \// <1>
-c prometheus --image=$(oc get po -n openshift-monitoring <prometheus_k8s_pod_name> \// <1>
-o json | jq '.spec.containers[] |select(.name=="prometheus").image' -r) \
-- sh -c 'ls -latr /prometheus/ | egrep -o "[0-9|A-Z]{26}" | head -3 | \
while read BLOCK; do rm -rf /prometheus/$BLOCK; done'
----
<1> Replace `<prometheus_k8s_pod_name>` with the pod mentioned in the `KubePersistentVolumeFillingUp` alarm description.

. Verify the usage of the mounted PV and make sure there is enough space available:
+
[source,terminal]
----
$ oc debug <prometheus_k8s_pod_name> -n openshift-monitoring \// <1>
--image=$(oc get po -n openshift-monitoring <prometheus_k8s_pod_name> \// <1>
-o json | jq '.spec.containers[] |select(.name=="prometheus").image' -r) -- df -h /prometheus/
----
<1> Replace `<prometheus_k8s_pod_name>` with the pod mentioned in the `KubePersistentVolumeFillingUp` alarm description.
+
.Example command for the `prometheus-k8s-0` pod
+
[source,terminal]
----
$ oc debug prometheus-k8s-0 -n openshift-monitoring \
--image=$(oc get po -n openshift-monitoring prometheus-k8s-0 \
-o json | jq '.spec.containers[] |select(.name=="prometheus").image' -r) -- df -h /prometheus/
----
+

+
