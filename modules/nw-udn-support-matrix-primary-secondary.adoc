The following sections highlight the supported features of both user-defined networks and network attachment definitions when they are used as either the primary or secondary network. 

[id="udn-network-support-matrix_{context}"]
== User-defined networks support matrix

User-defined networks (UDN) offer {product-title} *administrators* and *developers* the ability to create customizable network configurations that provide them with the ability to define their own network topologies, ensure network isolation, manage IP addressing for the workloads, and leverage advanced networking features. User-defined networks help enhance networking flexibility, security, and performance.

User-defined networks can serve as both the primary and secondary network interface and support `layer2` and `layer3` topologies. The following table shows you the supported features and topology types available on user-defined networks when they serve as either the primary or secondary network interface.

== Network attachment definition support matrix

Network attachment definitions (NADs) offer {product-title} *administrators* the ability to define additional network interfaces to pods, which enables the pods to connect multiple networks with distinct configurations and plugins. This network type is useful for complex networking cases such as network isolation, performance tuning, and dedicated access to external resources.
