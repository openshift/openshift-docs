= Verifying a default StorageClass exists

[role="_abstract"]
`LokiStack` requires a `StorageClass` that supports dynamic provisioning of block storage (`ReadWriteOnce`). Most cloud installer-provisioned clusters have one by default. {sno-caps}, bare metal, and some edge deployments do not.

.Prerequisites
* You have cluster administrator permissions.
* You are logged in to the cluster with `oc`.

.Procedure

. List available `StorageClass` resources by running the following command:
+
[source,terminal]
----
$ oc get storageclass
----

. Identify the default `StorageClass` by running the following command. It shows `(default)` in the output or has the annotation:
+
[source,terminal]
----
$ oc get storageclass -o=jsonpath='{.items[?(@.metadata.annotations.storageclass\.kubernetes\.io/is-default-class=="true")].metadata.name}'
----

. Optional: Make a `StorageClass` the default if needed. See OpenShift storage documentation.

. Verify it supports block storage (preferred for Loki performance):
+
Look for a provisioner such as `ebs.csi.aws.com`, `csi.vsphere.vmware.com`, `openshift-storage.ceph.rbd`, or similar.
+
[NOTE]
====
If no default `StorageClass` exists, `LokiStack` PVCs remain `Pending` and pods never start. You must either configure a default `StorageClass` or specify `storageClassName` explicitly in the `LokiStack` CR. Block storage is recommended for performance.
====

.Next step
If you are on {sno} or an edge deployment without dynamic storage, follow xref:configuring-local-storage-for-sno_{context}[Configuring local storage for single-node and edge deployments].
