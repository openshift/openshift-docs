= Disable network observability during installation

[role="_abstract"]
Prevent network observability from being installed during cluster creation to reduce resource consumption when network traffic monitoring is not required.

By default on multi-node clusters, {product-title} installs the Network Observability Operator when the `NetworkObservabilityInstall` feature gate is enabled. You can disable this behavior by creating a Network CR manifest that sets the installation policy to `NoAction`. This prevents the operator installation and avoids the associated resource overhead, which typically consumes 3-5% of node CPU and memory.

.Prerequisites

* You must enable the `NetworkObservabilityInstall` feature gate. For more information, see _Enabling the network observability feature gate_.
* You must have created the installation manifests directory.

.Procedure

. Create a Network CR manifest file in the `manifests` directory:
+
[source,terminal]
----
$ cat <<EOF > manifests/99-network-observability.yaml
apiVersion: config.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  networkObservability:
    installationPolicy: NoAction
EOF
----
+
where:
+
`spec.networkObservability.installationPolicy`:: Controls whether the Network Observability Operator is installed during cluster creation. Set to `NoAction` to prevent installation. Valid values are `InstallAndEnable` and `NoAction`. When omitted, network observability installs on multi-node clusters but not on Single-Node OpenShift (SNO) clusters.

. Create the cluster by running the following command:
+
[source,terminal]
----
$ openshift-install create cluster
----

.Verification

* Verify that the Network Observability Operator is not installed by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-netobserv-operator
----
+
The output should indicate that the namespace does not exist:
+
[source,terminal]
----
Error from server (NotFound): namespaces "openshift-netobserv-operator" not found
----
