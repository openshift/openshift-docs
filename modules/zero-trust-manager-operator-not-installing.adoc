= Operator not installing

[role="_abstract"]
If the Operator fails to install or remains in a pending state, use the following steps to diagnose and resolve the issue.

.Procedure

. Check the subscription status by running the following command:
+
[source,terminal]
----
$ oc get subscription -n zero-trust-workload-identity-manager
----

. Check if the install plan has been created by running the following command. If your subscription is set to **Manual** approval, the Operator does not install until the plan is aproved.
+
[source,terminal]
----
$ oc get installplan -n zero-trust-workload-identity-manager
----

. If the subscription and install plan are correct, check the logs of the Operator pod by running the following command:
+
[source,terminal]
----
$ oc logs -n zero-trust-workload-identity-manager -l name=zero-trust-workload-identity-manager --tail=50
----
