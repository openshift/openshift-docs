= Configuring node tuning and SR-IOV Network Operator

You can configure node tuning and the SR-IOV Network Operator to optimize the performance of your hosted cluster.

.Prerequisites

* You have created a hosted cluster.
* You have access to the management cluster.
* You have configured additional ports for SR-IOV networking.

.Procedure

. Create a node pool with performance tuning:
+
[source,terminal]
----
oc create -f - <<EOF
apiVersion: hypershift.openshift.io/v1beta1
kind: NodePool
metadata:
  name: <node_pool_name>
  namespace: clusters
spec:
  clusterName: <cluster_name>
  replicas: <number_of_nodes>
  platform:
    type: OpenStack
    openstack:
      flavor: <flavor_name>
      additionalPorts:
      - network: <network_name>
        subnet: <subnet_name>
        fixedIPs:
        - subnetID: <subnet_id>
          ipAddress: <ip_address>
        securityGroups:
        - <security_group_id>
      tuning:
        cpuPartitioning: isolated
        hugepages:
          defaultHugepagesSize: 1G
          pages:
          - size: 1G
            count: 4
  release:
    image: <image_name>
EOF
----
+
where:
+
--
* `<node_pool_name>` is the name of the node pool.
* `<cluster_name>` is the name of the hosted cluster.
* `<number_of_nodes>` is the number of nodes in the node pool.
* `<flavor_name>` is the name of the OpenStack flavor to use for the nodes.
* `<network_name>` is the name of the OpenStack network for the additional port.
* `<subnet_name>` is the name of the OpenStack subnet for the additional port.
* `<subnet_id>` is the ID of the OpenStack subnet.
* `<ip_address>` is the IP address to assign to the port.
* `<security_group_id>` is the ID of the security group to apply to the port.
* `<image_name>` is the name of the {op-system} image.
--

. Install the SR-IOV Network Operator:
+
[source,terminal]
----
oc create -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: sriov-network-operator
  namespace: openshift-sriov-network-operator
spec:
  channel: stable
  name: sriov-network-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
EOF
----

. Configure the SR-IOV Network Operator:
+
[source,terminal]
----
oc create -f - <<EOF
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetworkNodePolicy
metadata:
  name: <policy_name>
  namespace: openshift-sriov-network-operator
spec:
  deviceType: vfio-pci
  isRdma: false
  nicSelector:
    pfNames:
    - <interface_name>
  nodeSelector:
    node-role.kubernetes.io/worker: ""
  numVfs: <number_of_vfs>
  priority: 10
  resourceName: <resource_name>
EOF
----
+
where:
+
--
* `<policy_name>` is the name of the SR-IOV network node policy.
* `<interface_name>` is the name of the network interface to use for SR-IOV.
* `<number_of_vfs>` is the number of virtual functions to create.
* `<resource_name>` is the name of the SR-IOV resource.
--

. Create an SR-IOV network:
+
[source,terminal]
----
oc create -f - <<EOF
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetwork
metadata:
  name: <network_name>
  namespace: openshift-sriov-network-operator
spec:
  ipam: |
    {
      "type": "host-local",
      "subnet": "<subnet_cidr>",
      "rangeStart": "<range_start>",
      "rangeEnd": "<range_end>"
    }
  networkNamespace: <namespace>
  resourceName: <resource_name>
  spoofChk: "off"
  trust: "on"
  vlan: <vlan_id>
EOF
----
+
where:
+
--
* `<network_name>` is the name of the SR-IOV network.
* `<subnet_cidr>` is the CIDR of the subnet for the SR-IOV network.
* `<range_start>` is the start of the IP range.
* `<range_end>` is the end of the IP range.
* `<namespace>` is the namespace where the SR-IOV network will be available.
* `<resource_name>` is the name of the SR-IOV resource.
* `<vlan_id>` is the VLAN ID for the SR-IOV network.
--

. Verify that the SR-IOV Network Operator is running:
+
[source,terminal]
----
oc get pods -n openshift-sriov-network-operator
----

. Verify that the SR-IOV network node policy is applied:
+
[source,terminal]
----
oc get sriovnetworknodepolicies -n openshift-sriov-network-operator
----

. Verify that the SR-IOV network is created:
+
[source,terminal]
----
oc get sriovnetworks -n openshift-sriov-network-operator
----
