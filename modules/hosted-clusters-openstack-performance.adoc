= Tuning performance for HostedCluster nodes on OpenStack
:context: hostedcluster-openstack-performance

You can tune hosted cluster node performance on {rh-openstack-first} for high-performance workloads, such as
cloud-native network functions (CNFs). It covers configuring OpenStack resources, creating a `PerformanceProfile`, deploying a tuned `NodePool`, and enabling SR-IOV device support.

CNFS are network functions designed to run in cloud-native environments. CNFs can provide network services such as routing, firewalling, and load balancing. You can configure the node pool to use high-performance computing and networking devices to run CNFs.

.Prerequisites

* An OpenStack flavor with the necessary resources to run the workload, including dedicated CPU, memory, and host aggregate information. This ensures the instance is scheduled on hardware capable of high-performance operations, such as Intel or Mellanox NICs that support SR-IOV and DPDK.
* An OpenStack network attached to SR-IOV or DPDK-capable NICs. The network must be available to the project used by the HostedCluster.

== Creating a PerformanceProfile

Before creating an additional NodePool for performance workloads, you must create a `PerformanceProfile` custom resource (CR). This CR defines node performance characteristics, such as CPU isolation, huge pages, and kernel tuning.

.Example PerformanceProfile in a ConfigMap
[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: perfprof-1
  namespace: clusters
data:
  tuning: |
    apiVersion: v1
    kind: ConfigMap
    metadata:
      name: cnf-performanceprofile
      namespace: "${HYPERSHIFT_NAMESPACE}"
    data:
      tuning: |
        apiVersion: performance.openshift.io/v2
        kind: PerformanceProfile
        metadata:
          name: cnf-performanceprofile
        spec:
          additionalKernelArgs:
            - nmi_watchdog=0
            - audit=0
            - mce=off
            - processor.max_cstate=1
            - idle=poll
            - intel_idle.max_cstate=0
            - amd_iommu=on
          cpu:
            isolated: "${CPU_ISOLATED}"
            reserved: "${CPU_RESERVED}"
          hugepages:
            defaultHugepagesSize: "1G"
            pages:
              - count: ${HUGEPAGES}
                node: 0
                size: 1G
          nodeSelector:
            node-role.kubernetes.io/worker: ''
          realTimeKernel:
            enabled: false
          globallyDisableIrqLoadBalancing: true
----

Apply the `PerformanceProfile` configuration:

[source,terminal]
----
oc apply -f perfprof.yaml
----

== Creating a NodePool

After creating the `PerformanceProfile`, you can create a `NodePool` configured to use it.

[source,terminal]
----
export NODEPOOL_NAME=$CLUSTER_NAME-cnf
export FLAVOR="m1.xlarge.nfv"

hcp create nodepool openstack \
  --cluster-name $CLUSTER_NAME \
  --name $NODEPOOL_NAME \
  --node-count 0 \
  --openstack-node-flavor $FLAVOR
----

Patch the NodePool to reference the `PerformanceProfile`:

[source,terminal]
----
oc patch nodepool -n ${HYPERSHIFT_NAMESPACE} ${CLUSTER_NAME} \
  -p '{"spec":{"tuningConfig":[{"name":"cnf-performanceprofile"}]}}' --type=merge
----

Scale the NodePool:

[source,terminal]
----
oc scale NodePool/$CLUSTER_NAME --namespace ${HYPERSHIFT_NAMESPACE} --replicas=1
----

Wait for the node to be ready:

[source,terminal]
----
oc wait --for=condition=UpdatingConfig=True nodepool -n ${HYPERSHIFT_NAMESPACE} ${CLUSTER_NAME} --timeout=5m
oc wait --for=condition=UpdatingConfig=False nodepool -n ${HYPERSHIFT_NAMESPACE} ${CLUSTER_NAME} --timeout=30m
oc wait --for=condition=AllNodesHealthy nodepool -n ${HYPERSHIFT_NAMESPACE} ${CLUSTER_NAME} --timeout=5m
----

Verify performance tuning on the node:

[source,terminal]
----
ssh core@<node-ip>
chroot /host
cat /proc/cmdline
----

== Enabling SR-IOV Network Operator

The SR-IOV Network Operator manages SR-IOV devices on cluster nodes. You must label the nodes deployed by the NodePool so the operator can manage them.

Generate a kubeconfig file for the HostedCluster:

[source,terminal]
----
hcp create kubeconfig --name $CLUSTER_NAME > $CLUSTER_NAME-kubeconfig
export KUBECONFIG=$CLUSTER_NAME-kubeconfig
----

Label each worker node:

[source,terminal]
----
oc label node "${WORKER_NODE}" feature.node.kubernetes.io/network-sriov.capable="true"
----

Install the SR-IOV Network Operator in the guest cluster by following the documentation:

link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.17/html-single/networking/index#installing-sr-iov-operator_installing-sriov-operator[Installing the SR-IOV Network Operator]

After installation, configure the workload using the same process as with a standalone OpenShift Container Platform cluster.
