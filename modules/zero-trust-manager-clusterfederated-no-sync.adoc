= ClusterFederatedTrustDomain not syncing

[role="_abstract"]
Troubleshoot `ClusterFederatedTrustDomain` synchronization issues by inspecting resource statuses, reviewing controller manager logs, and performing manual trust bundle updates to maintain identity federation.

The following symptoms indicate a breakdown in the trust propagation layer despite the presence of defined federation resources:

* ClusterFederatedTrustDomain resources exist but federation not working.

* Bundle not updating.

.Procedure

. Inspect the status of the `ClusterFederatedTrustDomain resource for reconciliation errors by running the following command:
+
[source,terminal]
----
$ oc get clusterfederatedtrustdomain <name> -o yaml
----

. Review the SPIRE controller manager logs to identify issues with fetching or applying remote bundles by running the following command:
+
[source,terminal]
----
$ oc get clusterfederatedtrustdomain <name> -o yaml
----

. If automatic synchronization fails, perform a manual bundle update:

.. Fetch the bundle from the remote cluster by running the following command:
+
[source,terminal]
----
$ curl https://federation.apps.remote.example.com > new-bundle.json
----

.. Update the resource by running the following command:
+
[source,terminal]
----
$ oc edit clusterfederatedtrustdomain <name>
----

.. Update the `trustDomainBundle` field with the new bundle content.
