= Converting a {from-api-name} resource to use the {to-api-name}

You can convert individual {from-api-name} objects to equivalent {to-api-name} objects.

:FeatureName: Converting a {from-api-name} resource to use the {to-api-name}
include::snippets/technology-preview.adoc[]

.Prerequisites

* You have deployed an {product-title} cluster on a supported infrastructure type.

ifdef::machine-to-cluster[]
* You have enabled the use of the Cluster API.
endif::[]

* You have access to the cluster using an account with `cluster-admin` permissions.

* You have installed the {oc-first}.

.Procedure

. Identify the {from-api-name} resource that you want to convert to a {to-api-name} resource by running the following command:
+
[source,terminal,subs="attributes+"]
----
$ oc get <resource_kind> -n {origin-namespace}
----
+
--
where `<resource_kind>` is one of the following values:

`machine.{from-api-group}`:: The fully qualified name of the resource kind for a compute or control plane machine.
+
[IMPORTANT]
====
The output of this command includes the control plane machines.
Converting a control plane machine in not supported.
====

`machineset.{from-api-group}`:: The fully qualified name of the resource kind for a compute machine set.
--

. Edit the resource specification by running the following command:
+
[source,terminal]
----
$ oc edit <resource_name>
----
+
where `<resource_name>` is the fully qualified name of the {from-api-name} resource that you want to convert to a {to-api-name} resource.

. In the resource specification, update the value of the `spec.authoritativeAPI` field:
+
[source,yaml,subs="attributes+"]
----
apiVersion: <api_group>/<api_version> <1>
kind: <resource_kind> <2>
metadata:
  name: <resource_name> <3>
  [...]
spec:
  authoritativeAPI: {to-api-value} <4>
  [...]
status:
  authoritativeAPI: {from-api-value} <5>
  [...]
----
<1> The API group and version vary depending on the resource kind.
<2> The resource kind varies depending on the resource kind.
<3> The name of the resource that you want to convert.
<4> Specify `{to-api-value}` to convert the {from-api-name} resource to a {to-api-name} equivalent.
<5> The unconverted value for a {from-api-name} resource.
Do not change the value in this part of the specification.

.Verification 

* #This command is incomplete, what is it?# Check the status of the conversion by running the following command:
+
[source,terminal]
----
$ status.authoritativeAPI
---- 
+
--
* While the conversion progresses, this command returns a value of `Migrating`.
* When the conversion is complete, this command returns a value of `{to-api-value}`.
--
