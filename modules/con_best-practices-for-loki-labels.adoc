= Best practices for Loki labels

Labels in Loki are both the keyspace on which Loki shards incoming data, as well as the index used for finding logs at query-time. You can optimize query performance by properly using labels. 


Here are a few qualities of good labels:

* They describe infrastructure. This could include regions, clusters, servers, applications, namespaces, or environments.

* They are long-lived. Label values should generate logs perpetually, or at least for several hours.

* They are intuitive for querying.
