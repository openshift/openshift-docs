= {pipelines-shortname} concepts

[role="_abstract"]
{pipelines-title} consists of several core concepts that work together to define and run CI/CD workflows. Tasks define individual steps of work, pipelines orchestrate multiple tasks in sequence or parallel, and task runs and pipeline runs represent the execution of those definitions. Additional concepts include workspaces for sharing data between tasks, step actions for reusable operations within tasks, and triggers for initiating pipeline runs automatically.
