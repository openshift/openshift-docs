= Controller and system settings

[role="_abstract"]
You can configure the {builds-operator} or troubleshoot build behavior affecting the Build and `BuildRun` resources.
Cluster-wide settings control controller behavior, default resource allocations, and system-level policies.

Controller configuration:: Defines controller reconciliation behavior, leader election settings, and webhook configurations.
Default resource limits:: Establishes cluster-wide defaults for CPU, memory, and ephemeral storage allocated to builds when not specified at the `Build` or `BuildRun` level.
Image registry settings:: Configures default image registries, pull secrets, and registry certificate authorities available to all builds.
Retention policies:: Sets default cleanup policies for completed `BuildRun` and their associated Pods across the cluster.
Tekton integration settings:: Configures how {builds-shortname} interacts with the underlying Tekton installation, including `TaskRun` generation and `ServiceAccount` management.
