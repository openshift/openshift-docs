= Configuring respectRBAC using {gitops-title}

The `respectRBAC` configuration feature in Argo CD controls how Argo CD watches resources on a cluster. By default, Argo CD attempts to watch all Kubernetes resources (CRDs) on a cluster at the cluster scope. You can restrict the ArgoCD controller from discovering or syncing specific resources using only controller RBAC, without manually configuring resource exclusions.

To enable this feature, set the `resource.respectRBAC` key in the `argocd-cm` config map. After you set this key, the controller automatically stops watching resources it cannot list or access. For example, if the Argo CD cluster role prevents Argo CD from watching OpenShift Routes, it returns an error during synchronization, stating it cannot watch the resource.

[NOTE]
====
You can only enable `respectRBAC` by modifying the `argocd-cm` config map. If a user does not have permission to edit the config map, they cannot enable this feature. The `respectRBAC` feature of the {gitops-shortname} Operator allows users to update `respectRBAC` configurations in the config map through the Argo CD resource.
====

You can enable the `respectRBAC` feature by creating an Argo CD instance in the CLI or the Web UI. 

.Pre-requisites

Ensure that you created a namespace and updated it in the *Subscription* resource, so it can host a cluster-scoped Argo CD instance. For more information, see Using an Argo CD instance to manage cluster-scoped resources.

[id="configuring-respectRBAC-using-the-cli_{context}"]
== Configuring respectRBAC using the CLI

You can configure `respectRBAC` in the CLI.

.Procedure

. Create a YAML object file, for example, `openshift-gitops-sub.yaml`, to configure the `respectRBAC` feature using the CLI. 
+
.Example create an Argo CD instance using the respectRBAC feature
[source,yaml]
----
apiVersion: argoproj.io/v1alpha1
kind: ArgoCD
metadata:
  name: example-argocd #<1>
spec:
  controller:
    respectRBAC: strict #<2>
----
<1> Specify the name of the Argo CD instance.
<2> Specify the value of the `respectRBAC` feature in the `argocd-cm` config map. You can set it to normal or strict. If the value is set to strict, the {gitops-shortname} Operator strictly enforces Kubernetes RBAC permissions. The {gitops-shortname} Operator discovers and syncs only the resources for which it has explicit permissions to access. If the value is set to normal, the {gitops-shortname} Operator assumes that it has complete access to all the resources, and throws an error if it cannot access the resource.

. Apply the changes to the YAML object file by running the following command.
+
[source,terminal]
----
$ oc apply -f openshift-gitops-sub.yaml \// <1>
-n openshift-gitops-operator <2>
----
<1> Specify the name of the YAML file that includes the Argo CD resource.
<2> Specify the namespace that hosts the Argo CD instance.
+
. Verify the successful installation of Argo CD by checking the status of the `argoCD.status.phase` field. The {gitops-shortname} Operator sets this field to `Available` when the installation is complete. Run the following command to check the status:
+
[source,terminal]
----
kubectl get argocd <argocd-instance-name> -o jsonpath='{.status.phase}' #<1>
----
<1> Replace <argocd-instance-name> with the name of your Argo CD instance for example, `example-argocd`.

[id="configuring-respectRBAC-using-the-web-UI_{context}"]
== Configuring respectRBAC using the Web UI

You can configure `respectRBAC` in the Web UI.

.Procedure

. Log in to the {OCP} web console. 

. In the *Administrator* perspective of the web console, click *Operators* -> *Installed Operators*.

. Create or select the project where you want to install the user-defined Argo CD instance from the *Project* list.

. Select *{gitops-title}* from the installed Operators list and click the *Argo CD* tab.
+
After successful installation, the Argo CD instance is listed under *Argo CD* tab and its Status should be set to *Available*.
+
. After the Argo CD instance is created, verify that the value in the Argo CD resource is updated in `argocd-cm` config map successfully by completing the following steps.

.. In the *Administrator* perspective, go to *Workload* -> *ConfigMaps*.
.. In the *Project* option, select *Argo CD* namespace.
.. In the `argocd-cm` config map, you can now view the `resource.respectRBAC` field.
