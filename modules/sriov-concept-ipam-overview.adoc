= What is IPAM and why does it matter?

[role="_abstract"]
When you attach a pod to an SR-IOV secondary network, that network interface needs an IP address. IP Address Management (IPAM) determines how those addresses are assigned.

Unlike your primary pod network (which {product-title} manages automatically), SR-IOV secondary networks require you to explicitly configure IP address assignment.

== Three IPAM approaches

You have three options for assigning IP addresses to SR-IOV network interfaces:

* *Static IP assignment* - You manually specify exact IP addresses in pod specifications
* *DHCP* - A DHCP server on the physical network assigns addresses automatically
* *Whereabouts* - {product-title} assigns addresses automatically from a configured pool

Each approach has different trade-offs for scalability, infrastructure requirements, and operational complexity.

== Why this matters

Choosing the wrong IPAM method can create operational problems:

* Static IPs don't scale beyond a few pods
* DHCP requires external infrastructure
* Whereabouts requires proper IP pool planning

Understanding each option helps you choose the right approach for your use case.
