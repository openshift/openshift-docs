= Build resource configuration

[role="_abstract"]
You can configure advanced resource allocation for individual build steps, build strategies, and system-wide defaults.
Resource configuration fields control CPU, memory, and storage limits at multiple levels within the build execution environment.

Step resources:: Define specific CPU and memory resources for individual build steps, enabling fine-grained control over resource consumption during multi-step builds.
Strategy parameters:: Configure global resource limits that apply to all steps within a specific build strategy, establishing baseline resource availability.
System parameters:: Define cluster-wide default resource limits and quotas that apply when builds do not specify explicit resource constraints.

Configure resources to optimize build performance, prevent resource exhaustion, and ensure fair allocation across concurrent builds in shared cluster environments.
