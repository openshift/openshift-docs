= Configuring the DPA with more than one BSL

You configure the DPA with more than one BSL and specify the credentials provided by the cloud provider.  

.Prerequisites

* You must install the {oadp-short} Operator.
* You must create the secrets by using the credentials provided by the cloud provider.

.Procedure

. Configure the DPA with more than one BSL. See the following example.
+
.Example DPA
[source, yaml]
----
#...
backupLocations:
  - name: aws # <1>
    velero:
      provider: aws
      default: true
      objectStorage:
        bucket: <bucket_name> # <2>
        prefix: <prefix> # <3>
      config:
        region: <region>> # <4>
        profile: "default"
      credential:
        key: cloud
        name: cloud-credentials # <5>
  - name: odf # <6>
    velero:
      provider: aws
      default: false
      objectStorage:
        bucket: <bucket_name> 
        prefix: <prefix> 
      config:
        profile: "default" 
        region: minio
        s3Url: <url> # <7>
        insecureSkipTLSVerify: "true"
        s3ForcePathStyle: "true"
      credential:
        key: cloud
        name: <custom_secret_name_odf> # <8>
#...
----
<1> Specify a name for the first BSL.
<2> Specify a bucket name.
<3> Specify a prefix for Velero backups, for example, `velero`, if the bucket is used for multiple purposes.
<4> Specify the AWS region for the bucket.
<5> Specify the name of the default `Secret` object that you created.
<6> Specify a name for the second BSL.
<7> Specify the URL of the S3 endpoint.
<8> Specify the correct name for the `Secret`, for example, `custom_secret_name_odf`. If you do not specify a `Secret` name, the default name is used.

. Specify the storage location to be used in the backup CR. See the following example.
+
.Example backup CR
[source,yaml]
----
# ...
spec:
  includedNamespaces:
  - <namespace> # <1>
  storageLocation: <backup_storage_location> # <2>
  defaultVolumesToFsBackup: true
----
<1> Specify the namespace to back up.
<2> Specify the storage location.
