= Configuring the Data Protection Application with more than one `BackupStorageLocation`

You configure the Data Protection Application (DPA) with more than one `BackupStorageLocation` as shown in the following example. In the example, you create two secrets by using the cloud credentials provided by the cloud provider and specify the name of the secret in the `BackupStorageLocation` configuration.

[source, yaml]
----
# ...
backupLocations:
  - name: aws # <1>
    velero:
      provider: aws
      default: true
      objectStorage:
        bucket: <bucket_name> # <2>
        prefix: <prefix> # <3>
      config:
        region: <region>> # <4>
        profile: "default"
       credential:
        key: cloud
        name: cloud-credentials # <5>
  - name: odf # <6>
    velero:
      provider: aws
      default: false
      objectStorage:
        bucket: <bucket_name> 
        prefix: <prefix> 
      config:
        profile: "default" 
        region: minio
        s3Url: <url> # <7>
        insecureSkipTLSVerify: "true"
        s3ForcePathStyle: "true"
      credential:
        key: cloud
        name: <custom_secret_name_odf> # <8>
# ...
----
<1> Specify a name for the first `BackupStorageLocation`.
<2> Specify a bucket name.
<3> Specify a prefix for Velero backups, for example, `velero`, if the bucket is used for multiple purposes.
<4> Specify the AWS region for the bucket.
<5> Specify the name of the `Secret` object that you created. If you do not specify this value, the default name, `{credentials}`, is used. If you specify a custom name, the custom name is used for the backup location.
<6> Specify a name for the second `BackupStorageLocation`.
<7> Specify the URL of the S3 endpoint.
<8> Specify the correct name for the `Secret`, for example, `custom_secret_name_odf`. If you do not specify a `Secret` name, the default name is used.
