= Complete federation configuration example

[role="_abstract"]
Use this complete configuration example to set up a SPIRE federation in your cluster. The sample YAML demonstrates how to configure bundle endpoints, define federated trust domains, and manage routes.

The following is a sample YAML configuration file.

[source,yaml]
----
apiVersion: operator.openshift.io/v1alpha1
kind: SpireServer
metadata:
  name: cluster
spec:
  # ... other configuration ...
  federation:
    #########################################################################
    # BUNDLE ENDPOINT CONFIGURATION (How we expose our bundle)
    #########################################################################
    bundleEndpoint:
      profile: "https_web"
      refreshHint: 300
      httpsWeb:
        # Option 1: ACME automatic certificates (Recommended for public access)
        acme:
          directoryUrl: "https://acme-v02.api.letsencrypt.org/directory"
          domainName: "federation.example.com"
          email: "platform-team@example.com"
          tosAccepted: "true"

        # Option 2: Manual certificates (servingCert) is omitted here for brevity

    #########################################################################
    # FEDERATED TRUST DOMAINS (Who we federate with)
    #########################################################################
    federatesWith:
      # Federating with an external partner using Web PKI
      - trustDomain: "partner-west.example.com"
        bundleEndpointUrl: "https://federation.partner-west.example.com:8443"
        bundleEndpointProfile: "https_web"

      # Federating with another internal cluster using mTLS (requires endpointSpiffeId)
      - trustDomain: "partner-east.example.com"
        bundleEndpointUrl: "https://federation.partner-east.example.com:8443"
        bundleEndpointProfile: "https_spiffe"
        endpointSpiffeId: "spiffe://partner-east.example.com/spire/server"

    #########################################################################
    # ROUTE MANAGEMENT
    #########################################################################
    managedRoute: "true"
----
