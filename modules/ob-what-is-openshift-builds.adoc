= What is OpenShift Builds?

[role="_abstract"]
{builds-shortname} is a Kubernetes-native build framework based on the Shipwright project that enables developers and platform engineers to build container images directly on OpenShift clusters without requiring external build infrastructure.

{builds-shortname} provides a standardized approach to building container images from source code, Dockerfiles, and existing container images. It supports multiple build strategies including Buildah, Source-to-Image (S2I), and Cloud Native Buildpacks, allowing you to choose the right tool for your application's requirements.

The framework integrates with OpenShift's security model, supporting features like role-based access control (RBAC), secret management for authentication, and runtime isolation through RuntimeClass configurations. Platform engineers can define reusable build strategies at the cluster level, while developers create build definitions that reference these strategies within their namespaces.

{builds-shortname} is designed for teams that need repeatable, auditable container image builds as part of their CI/CD workflows, with built-in support for monitoring build metrics, managing build retention, and tracking build execution through Tekton pipelines.
