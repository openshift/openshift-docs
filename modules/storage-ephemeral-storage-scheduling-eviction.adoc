= Ephemeral storage configuration affects pod scheduling and eviction

[role="_abstract"]
To manage resource allocation and cluster stability, review how ephemeral storage settings in the pod specification affect operations.

The settings in the pod specification affect both how the scheduler makes a decision about scheduling pods and when kubelet evicts pods to reclaim space.

* First, the scheduler ensures that the sum of the resource requests of the scheduled containers is less than the capacity of the node. The scheduler can assign the pod to a node only if the available ephemeral storage (allocatable resource) for the node is more than 4 GiB.

* Second, at the container level, because the first container sets a resource limit, kubelet eviction manager measures the disk usage of this container and evicts the pod if the storage usage of the container exceeds its limit (4 GiB). The kubelet eviction manager also marks the pod for eviction if the total usage exceeds the overall pod storage limit (8 GiB).
