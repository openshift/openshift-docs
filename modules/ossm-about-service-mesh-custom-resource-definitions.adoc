= About Service Mesh Custom Resource Definitions

Installing the {SMProductName} Operator also installs Custom Resource Definitions (CRD) that a network administrator can use to configure {istio} for {SMProductShortName} installations. There are two different categories of CRDs: {SMProduct} CRDs and {istio} CRDs.

{SMProduct} CRDs install and maintain the software that monitors the mesh. These CRDs belong to the `sailoperator` API and include the `Istio`, `IstioRevision`, `IstioCNI`, and `ZTunnel` resources. For more information about the CRD parameters, see the `sailoperator` link:https://github.com/istio-ecosystem/sail-operator/blob/main/docs/api-reference/sailoperator.io.md[API reference] documentation.

{istio} CRDs are associated with mesh configuration and workload management. These CRDs belong to different APIs and can include various resources, such as the `AuthorizationPolicy`, `DestinationRule`, and `VirtualService` resources, that network administrators use to create and manage a service mesh.
