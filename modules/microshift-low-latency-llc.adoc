= Enabling last level cache locality in {microshift-short}

[role="_abstract"]
You can align workloads with CPU cores that share the same last-level cache (LLC) to improve performance for latency-sensitive applications. Enable the Kubernetes CPU Manager option `prefer-align-cpus-by-uncorecache` in {microshift} to apply this alignment.

:FeatureName: Last Level Cache (LLC) locality
include::snippets/technology-preview.adoc[]
:FeatureName!:

.Procedure

. Add the following to `/etc/microshift/config.yaml`. If you already have a `kubelet` section from the earlier section, merge the `cpuManagerPolicyOptions` entry and ensure the feature gate is present.
+
[source,yaml]
----
apiServer:
  featureGates:
    customNoUpgrade:
      enabled:
      - "CPUManagerPolicyBetaOptions"
kubelet:
  reservedSystemCPUs: "0"
  cpuManagerPolicy: static
  cpuManagerPolicyOptions:
    prefer-align-cpus-by-uncorecache: "true"
----

. Restart {microshift-short} to apply the configuration:
+
[source,terminal,subs="+quotes"]
----
$ sudo systemctl restart microshift
----
