= Attach a pod to the SR-IOV network

[role="_abstract"]
Create a test pod that connects to your SR-IOV network using static IP addressing.

.Procedure

. Create a file named `sriov-pod.yaml` with the following content:
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: sriov-test-pod
  namespace: default
  annotations:
    k8s.v1.cni.cncf.io/networks: |  
      [{
        "name": "sriov-network-1",  
        "ips": ["192.168.1.100/24"]  
      }]
spec:
  containers:
  - name: test-container
    image: registry.access.redhat.com/ubi9/ubi:latest
    command: ["/bin/bash", "-c", "sleep infinity"]
    resources:
      requests:
        openshift.io/intelnics: "1"  
      limits:
        openshift.io/intelnics: "1"
----
 Network annotation that specifies which SR-IOV networks to attach.
 Must match the name of your SriovNetwork resource.
 Static IP address for the SR-IOV interface. Choose an IP address appropriate for your network. The pod's primary network interface will use the cluster network; this is the secondary network interface.
 Request one virtual function from the `intelnics` resource pool.

. Create the pod:
+
[source,terminal]
----
$ oc create -f sriov-pod.yaml
----

. Wait for the pod to start:
+
[source,terminal]
----
$ oc get pod sriov-test-pod -n default
----
+
Wait until the STATUS shows `Running`.

.Verification

. Verify the pod has an SR-IOV network interface:
+
[source,terminal]
----
$ oc exec -it sriov-test-pod -n default -- ip addr
----
+
.Example output
[source,terminal]
----
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
2: eth0@if123: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1450 qdisc noqueue state UP group default
    link/ether 0a:58:0a:80:02:47 brd ff:ff:ff:ff:ff:ff link-netnsid 0
    inet 10.128.2.71/23 brd 10.128.3.255 scope global eth0  
3: net1: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq state UP group default qlen 1000
    link/ether 3c:fd:fe:b8:7f:f8 brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 brd 192.168.1.255 scope global net1  
----
 `eth0` is the primary network interface on the cluster network.
 `net1` is the SR-IOV network interface with your static IP address.

. Verify the SR-IOV virtual function was allocated:
+
[source,terminal]
----
$ oc get pod sriov-test-pod -n default -o yaml | grep -A 5 "resources:"
----
+
You should see `openshift.io/intelnics: "1"` in both requests and limits.

Your pod is now connected to the SR-IOV network.
