= Using Windows containers in a disconnected cluster

The Windows Machine Config Operator (WMCO) can be installed onto a {product-title} cluster in a disconnected, or air-gapped, network. A _disconnected network_ is a restricted network without direct internet connectivity.

Because the cluster does not have access to the internet, any required images cannot be downloaded. As such, you can create a _mirror registry_, which is a container registry within the disconnected network that contains all the container images required by {product-title}. 

Using a mirror registry requires the following general steps:

* Creating the mirror registry, using a tool such as Red Hat Quay.
* Creating a container image registry credentials file.
* Copying the images in your {product-title} image repository to your mirror registry.

For information on these steps, see "About disconnected installation mirroring". 

After creating the mirror registry and mirroring the images, you can use an `ImageDigestMirrorSet` (IDMS) or `ImageTagMirrorSet` (ITMS) object to configure your cluster to pull images from the mirror registry. The IDMS and ITMS object redirect requests to pull images from a repository on a source image registry and have it resolved by a repository on a mirrored image registry. 

For WMCO nodes, the WMCO uses the mirror registry specified in the IDMS or ITMS object to configure the `hosts.toml` file on your Windows nodes. If changes are made to the IDMS or ITMS object, the WMCO automatically updates the `hosts.toml` with the new information. Note that WMCO sequentially updates each Windows node when mirror settings are changed. A such, reconfiguration times scale linearly with the number of Windows nodes in the cluster.

Also, because Windows nodes configured by the WMCO rely on containerd container runtime, the WMCO ensures that the containerd config files are up-to-date with the registry settings. For new nodes, these files are copied to the instances upon creation. For existing nodes, after activating the mirror registry, the registry controller uses SSH to access each node and copy the generated config files, replacing any existing files.

You can add machine set or Bring-Your-Own-Host (BYOH) Windows nodes to a disconnected cluster.









The image specified in MachineSets has an extra requirement of having [the OpenSSH.Server~~~~0.0.1.0 Windows capability installed.](https://learn.microsoft.com/en-us/windows-server/administration/openssh/openssh_install_firstuse?tabs=powershell#install-openssh-for-windows)
