= Pod disruption budgets for virtualized control planes

[role="_abstract"]
Create a `PodDisruptionBudget` (PDB) for the control plane VMs to prevent the hosting cluster from evicting more than one VM at a time during node drain or maintenance operations.
A PDB with `minAvailable: 2` ensures that at least two control plane VMs remain running, which preserves `etcd` quorum.
You must use `minAvailable` instead of `maxUnavailable` because the `VirtualMachineInstance` controller does not implement the Kubernetes scale subresource.
A PDB with `maxUnavailable` cannot determine the expected pod count for VMs, which causes it to block all evictions.

Without a PDB, draining multiple hosting cluster nodes concurrently could evict more than one control plane VM at a time and cause quorum loss.
