= Build output configuration

[role="_abstract"]
Build outputs determine where {builds-shortname} pushes completed container images.

Configuring output destinations ensures your images are stored in the correct registries and accessible to your deployment workflows.
Proper output configuration is essential for integrating builds into your continuous delivery pipeline.

When you configure build outputs, you specify the container registry and repository where you want to push the built image.
This can be the integrated OpenShift registry, an external registry such as Quay or Docker Hub, or a private registry in your organization.
You can also configure image tags to support versioning and promote images across environments.

Output configuration includes registry credentials, push secrets, and TLS settings that control how images are uploaded.
By setting appropriate output destinations, you ensure that builds integrate seamlessly with your deployment automation and image distribution strategy.
The output configuration also determines which teams and applications can access your built images.
