= GNR-D feature parity with SPR-EE

[role="_abstract"]
GNR-D hardware supports the same Precision Time Protocol (PTP) boundary clock capabilities as Sapphire Rapids Edge Enhanced (SPR-EE) platforms.

GNR-D provides the same boundary clock capabilities as SPR-EE platforms while removing the need for external card-to-card synchronization connections between NICs. This architecture simplifies configuration and increases `TimeTransmitter` port density within the same system form factor.

By using GNR-D hardware, you can achieve the following outcomes:

* Simplify PTP boundary clock deployment.
* Reduce hardware synchronization complexity.
* Support higher-scale TimeTransmitter configurations.
* Maintain equivalent PTP boundary clock capabilities.

The following table compares key boundary clock deployment characteristics of SPR-EE and GNR-D platforms:

[cols="3a,3a,3a",options="header"]
|===

|Capability |SPR-EE| GNR-D

|Boundary clock deployment
|Single system with external SMA synchronization between multiple NICs
|Single system without external NIC synchronization

|NIC synchronization
|External card-to-card connections required for multiple NICs
|Integrated synchronization across NAC and Carter Flats

|Configuration complexity
|Higher
|Lower

|===
