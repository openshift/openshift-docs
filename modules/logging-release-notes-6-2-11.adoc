= Logging 6.2.11 Release Notes

[role="_abstract"]
This release includes link:https://access.redhat.com/errata/RHBA-2026:14884[RHSA-2026:14884].

[id="logging-release-notes-6-2-11-bug-fixes_{context}"]
== Bug fixes

* Previously, collectors configured to use the detect_exceptions transform would create a separate instance of the state machine for every log stream. This resulted in high memory consumption when processing a large number of concurrent streams. With this release, the behavior has been modified to share a single state machine instance across log streams, significantly reducing the memory footprint of the collector.
(https://redhat.atlassian.net/browse/LOG-9348[LOG-9348])

* Previously, users who upgraded to {LoggingProductName} v6.2.10 were blocked from subsequently upgrading their ocp-product-title (OCP) cluster to version 4.20. This was a regression from earlier releases where such upgrades were permitted. This release fixes the issue by correctly setting the maximum supported OCP version to v4.20 for Logging 6.2, allowing cluster upgrades to proceed as expected.
(https://redhat.atlassian.net/browse/LOG-9365[LOG-9365])
