= Configure DNS provider credentials for AWS

[role="_abstract"]
If you want to configure AWS DNS policies in {prodname}, you must configure the DNS credentials. You must perform the steps on each {ocp} cluster that you want to use {prodname} on.

You must configure a DNS hosted zone. The credentials for your DNS provider must have permissions to update DNS records within this zone.

.Prerequisites

* You installed {prodname} on an {ocp} cluster.
* You have access to the namespace of your gateway, for example, `api-gateway`.
+
[NOTE]
====
If you already know your environment variable values, you can create the required YAML files as required for your use case.
====

.Procedure

. Optional. Set up your environment variables with the following values:

.. Assign the `AWS_ACCESS_KEY_ID` environment variable by running the following command:
+
[source,terminal]
----
$ export AWS_ACCESS_KEY_ID=xxxxxxx
----
+
`AWS_ACCESS_KEY_ID`: is the ID from AWS with Route 53 access.

.. Assign the `AWS_SECRET_ACCESS_KEY` environment variable by running the following command:
+
[source,terminal]
----
$ export AWS_SECRET_ACCESS_KEY=xxxxxxx
----
+
`AWS_SECRET_ACCESS_KEY`: is the key from AWS with Route 53 access.

.. Assign the `AWS_REGION` environment variable by running the following command:
+
[source,terminal]
----
$ export AWS_REGION=your-aws-region
----
+
`AWS_REGION`: specifies your region, for example, `us-east-2` or `eu-west-1`.

. Create a Secret resource for your credentials as follows:
+
[source,terminal]
----
$ oc create secret generic aws-credentials \
  --namespace=api-gateway \
  --type=kuadrant.io/aws \
  --from-literal=AWS_ACCESS_KEY_ID=$AWS_ACCESS_KEY_ID \
  --from-literal=AWS_SECRET_ACCESS_KEY=$AWS_SECRET_ACCESS_KEY \
  --from-literal=AWS_REGION=$AWS_REGION
----
+
[IMPORTANT]
====
You must configure the Secret in the same namespace where your `Gateway` object is applied.
====
