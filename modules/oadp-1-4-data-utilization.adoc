= OADP 1.4 data utilization

The following shows observed performance when running OADP backup and restore with different plugins in an isolated environment.  Customers' experience using OADP may vary from these results depending on the type of resources in their cluster such as cpu(cores), storage, network and content of the data that is used for backup and restores.  These numbers are not meant as a minimum expected rate but represent transfer rate when performing backup and restores in an isolated environment with optimal conditions and dedicated hardware.

.Transfer Rate

The calculated throughput rate represents the amount of non-zero data contained in a pv(s), used in the backup or restore process divided by the  time taken to process a backup or restore.

This rate allows customers to understand the performance of OADP.

== Backups for single pod and pv

Transfer Rate for FSB with Kopia and CSI Snapshot Datamovement are around 1GB/s depending on storage type

.Examples in numbers

* FSB with Kopia on Single PV on Ceph RBD
** 2T PV containing 1.5T of data transferred 1.34GB/s
** 4T PV containing 3T of data transferred 1.35GB/s

* FSB with Kopia on Single PV on Ceph RBD vs Ceph FS: RBD was faster
** 4T PV containing  3T of data backup
*** Ceph RBD: transferred 1.35 GB/s
*** CephFS: transferred  980 MB/s

* CSI Snapshot Datamovement vs FSB with Kopia Single PV on Ceph RBD
** 1T PV containing 0.5T  of data transferred 1GB/s
** 2T PV containing 1.5T  of data transferred 1.34GB/s

CSI Snapshot Data movement rates shown represent initial backup, and match rates seen on FSB with Kopia using Ceph RBD.


== Restores for single pod and pv

Restores for Single pod containing data on a single PV -  non-zero data transfer rate for restores around 230-350MB/s depending on storage type.

.Examples in numbers

* Restore from FSB with Kopia Single PV on Ceph RBD
** 2T PV containing 1.5T of data transferred 280MB/s
** 4T PV containing 3T of data transferred 240MB/s

* Restore from FSB with Kopia Single PV on Ceph RBD vs Ceph FS - rate about the same
** FSB with Kopia CephRBD: 4T PV containing  3T of data transferred 240MB/s
** FSB with Kopia CephFS: 4T PV containing  3T of data transferred 230MB/s

* Restore from CSI Snapshot Datamovement vs Restore from FSB with Kopia on Single PV on Ceph RBD
** Restore from CSI Snapshot Datamovement CephRBD:  1T PV containing 0.5T of data transferred 350MB/s
** Restore from FSB with Kopia CephRBD:  2T PV containing 1.5T of data transferred 280MB/s


== Comparing CSI & FSB with Kopia

Comparing CSI & FSB with Kopia, many pvs in single ns with 6 workers.

.Backup: CSI Snapshot VS FSB with Kopia

* 500G of data spread equally across 1,000 pods & pvs in single namespace using 6 workers:
** CSI: Data transferred 80MB/s (rate was the same for CephFS and CephRBD)
** FSB with Kopia: Data transferred 520MB/s on RBD and 420MB/s on CephFS

.Restore: CSI Snapshot VS FSB with Kopia

* 500G of data spread equally across 1,000 pods & pvs in single namespace using 6 workers:
** CSI -   data transferred 40MB/s on CephFS and 350 MB/s on CephRBD
** FSB with Kopia - data transferred 260MB/s on CephFS and 240 MB/s on CephRBD
