= About SBOMs

[role="_abstract"]
A Software Bill of Materials (SBOM) is a digital record listing software components and their origins, helping organizations locate vulnerable packages and comply with cybersecurity requirements.

A Software Bill of Materials (SBOM) is a digital record that lists the components of a piece of software and their origins. Organizations can use SBOMs to locate the presence of vulnerable software packages and components and respond more quickly to mitigate the risk. Additionally, being able to generate SBOMs helps organizations comply with "Executive Order 14028: Improving the Nations Cybersecurity".

SBOMs can contain different types of information, depending on the methods of data collection and how you generate them. The Cybersecurity & Infrastructure Security Agency (CISA) offers a document, "Types of Software Bill of Materials (SBOM)", that summarizes the types of SBOMs.

The type of SBOM that {product-title-short} generates is "Analyzed." CISA notes that analyzed SBOMs use an analysis of artifacts such as executables, packages, containers, and virtual machine images. Analyzed SBOMs offer the following benefits, as summarized by CISA:

* They can give information about software without an active development environment.
* You can generate them without access to the build process.
* You can use them to discover hidden dependencies that other tools might miss.

The SBOM generated by {product-title-short} is in "System Package Data Exchange (SPDX) 2.3" format.
