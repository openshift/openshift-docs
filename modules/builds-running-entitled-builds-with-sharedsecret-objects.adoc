= Running builds using SharedSecret objects

You can use a `SharedSecret` object to securely access the entitlement keys of a cluster in builds. The `SharedSecret` object allows you to share and synchronize secrets across namespaces without manual replication.

[IMPORTANT]
====
Starting with {OCP} 4.18, the Shared Resource CSI Driver is no longer included in the core {OCP} installation. To continue using `SharedSecret` APIs, `csi` volume mounts, or entitled `BuildConfig`-based builds that use the `etc-pki-entitlement` secret, you must install the {builds-v2title} Operator. The Shared Resource CSI Driver provided by this Operator is fully supported for `BuildConfig` builds and the Shipwright-based builds.

This is the formally supported method for sharing entitlement secrets across a cluster. Red{nbsp}Hat does not recommend or support manual secret replication across namespaces.
====
