= Running builds using SharedSecret objects

You can use a `SharedSecret` object to securely access the entitlement keys of a cluster in builds. The `SharedSecret` object allows you to share and synchronize secrets across namespaces.

[IMPORTANT]
====
In {OCP} 4.18 and later, the Shared Resource CSI Driver has been removed from the core {OCP} installation. The driver is now generally available through the {builds-v2title} Operator.

To continue using `SharedSecret` APIs, `csi` volume mounts, or entitled BuildConfig-based builds that use the `etc-pki-entitlement` secret, you must install the {builds-v2title} Operator. This operator also provides access to the Shipwright-based Build API system.

The Shared Resource CSI Driver from the {builds-v2title} Operator is the only supported method for sharing secrets across the cluster for entitlement workflows. Red Hat does not support manual replication of secrets.
