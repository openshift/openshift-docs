= Policy scope support for cluster and namespace labels

[role="_abstract"]

// JIRA: ROX-29854
Policy scopes now support regex selection of labels and names for clusters, namespaces, and deployments.

With this feature, you can complete the following actions:

* Use label selectors to include or exclude clusters, namespaces, and deployments in policy scopes
* Apply regex patterns to both label keys and values for flexible matching
* Automatically apply policies to new infrastructure based on labels rather than updating policies manually
* Specify either cluster ID or cluster label in the same scope rule
* Specify either namespace name regex or namespace label in the same scope rule

This enhancement simplifies policy management in dynamic environments where container infrastructure and workloads often change. By using labels in policies, you can apply appropriate labels to clusters or namespaces as part of your automation without modifying existing policies.

[NOTE]
====
Cluster label and namespace label fields are evaluated only on secured clusters running {product-title-short} version 4.11 or later. These fields are ignored by secured clusters running earlier versions.
====
