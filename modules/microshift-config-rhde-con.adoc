= Configuring {op-system-bundle}

{microshift-short} and {op-system-base-full} work together to bring a lighter-weight, single-node Kubernetes to the edge. This combination means that control-plane nodes and pods are not included in {microshift-short}. It also means that the operating system handles many functions. You can add and configure RPM-based features or simply opt-in to others. In many cases, you must configure the operating system or other resources in addition to the {microshift-short} service.

Bringing these pieces together is the {microshift-short} configuration file. Think of the {microshift-short} configuration file can as a facilitator for customization and added functions. For example:

* Available additional features included in {microshift-short} release packages such as ingress control settings. These settings are available by default, but must be configured using the {microshift-short} configuration file.
* Operating system functions, such as logical volume management. Persistent storage is orchestrated by an external Operator and {op-system-base}, but usage is determined by settings in the {microshift-short} configuration file.
* Advanced networking functions, such as using multiple networks. The Multus package is an installable RPM, but access is determined by the {microshift-short} configuration file. In addition, network settings are required on your networks through the host.
* Application management, such as GitOps, which can be added without using the {microshift-short} configuration file.
