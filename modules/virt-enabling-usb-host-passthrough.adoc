= Enabling USB host passthrough

You can enable USB host passthrough at the cluster level.

.Prerequisites

* Enabling the passthrough requires cluster administrator privileges.

.Procedure

. Identify the USB device vendor and product by running the following command:
+
[source,terminal]
----
$ lsusb
----

. Open the HyperConverged (HCO) custom resource (CR) by running the following commmand:
+
[source,terminal]
----
$ oc edit hyperconverged kubevirt-hyperconverged -n openshift-cnv
----

. Edit the CR by performing the following actions, as shown in the following example:

* Enable the `HostDevices` feature gate by adding it to the `spec.featureGates` stanza.
* Add the USB devices to the `permittedHostDevices` stanza.

+
.Example YAML snippet
[source,yaml]
----
apiVersion: hco.kubevirt.io/v1beta1
kind: HyperConverged
metadata:
   name: kubevirt-hyperconverged
   namespace: {CNVNamespace}
spec:
  featureGates:
    hostDevices: True <1>
# ...
  configuration:
    permittedHostDevices: <2>
      usb: <3>
        - resourceName: kubevirt.io/peripherals <4>
          selectors:
            - vendor: "045e"
              product: "07a5"
            - vendor: "062a"
              product: "4102"
            - vendor: "072f"
              product: "b100"

----
<1> The `hostDevices featureGates` is set to `True`.
<2> The host devices that have permission to be used in the cluster.
<3> The USB devices.
<4> Add multiple USB devices and assign them to the VM by using `resourceName: deviceName`.
