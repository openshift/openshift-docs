= Creating a service account for an extension

You must create a service account to install, manage, and update a cluster extension.

.Prerequisites

ifndef::openshift-rosa-hcp,openshift-rosa,openshift-dedicated[]
* Access to an {product-title} cluster using an account with `cluster-admin` permissions.
endif::[]
ifdef::openshift-rosa-hcp,openshift-rosa,openshift-dedicated[]
* Access to an {product-title} cluster using an account with `dedicated-admin` permissions.
endif::[]

.Procedure

. Create a service account, similar to the following example:
+
[source,yaml]
----
apiVersion: v1
kind: ServiceAccount
metadata:
  name: <extension>-installer
  namespace: <namespace>
----
+
.Example `extension-service-account.yaml` file
[%collapsible]
====
[source,yaml]
----
apiVersion: v1
kind: ServiceAccount
metadata:
  name: pipelines-installer
  namespace: pipelines
----
====

. Apply the service account by running the following command:
+
[source,terminal]
----
$ oc apply -f extension-service-account.yaml
----
