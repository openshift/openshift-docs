= Configuring Google Cloud Storage for LokiStack

[role="_abstract"]
You can use Google Cloud Storage (GCS) to store logs in Google Cloud.

.Prerequisites
* You have a project on Google Cloud.
* You have created a bucket in the project.
* You have created a service account with the necessary permissions.
* You have cluster administrator permissions.

.Procedure

. Obtain the service account credentials from Google Cloud and save them as `key.json`.

. Create an object storage secret with the name `logging-loki-s3`:
+
[source,terminal]
----
$ oc create secret generic logging-loki-s3 \
  -n openshift-logging \
  --from-literal=bucketname="<bucket_name>" \
  --from-file=key.json="./key.json"
----

.Verification
Verify the secret was created successfully by running the following command:
[source,terminal]
----
$ oc get secret logging-loki-s3 -n openshift-logging
----
