= Gather diagnostic information for network reconfiguration issues

[role="_abstract"]
You can gather diagnostic information to help troubleshoot network reconfiguration issues on {sno} clusters.

.Procedure

. Inspect the `IPConfig` CR status:
+
[source,terminal]
----
$ oc get ipc ipconfig -o yaml
----
+
Review the `status.conditions` field for the current state, reason, and message. Check `status.validNextStages` for allowed stage transitions, and `status.history` for timestamps of stage progression.

. View the {lcao} controller logs:
+
[source,terminal]
----
$ oc logs -n openshift-lifecycle-agent deployment/lifecycle-agent-controller-manager -c manager
----

. Access the node to inspect systemd service logs:
+
[source,terminal]
----
$ oc debug node/<node_name>
----
+
Replace `<node_name>` with the name of your {sno} node.

. After accessing the node, view the relevant service logs depending on which phase you are troubleshooting:
+
--
* For pre-pivot issues:
+
[source,terminal]
----
$ sudo journalctl -u lca-ipconfig-pre-pivot -b --no-pager
----

* For post-pivot issues:
+
[source,terminal]
----
$ sudo journalctl -u ip-configuration.service -b --no-pager
----

* For init-monitor watchdog issues:
+
[source,terminal]
----
$ sudo journalctl -u lca-init-monitor.service -b --no-pager
----

* For rollback issues:
+
[source,terminal]
----
$ sudo journalctl -u lca-ipconfig-rollback -b --no-pager
----
--
