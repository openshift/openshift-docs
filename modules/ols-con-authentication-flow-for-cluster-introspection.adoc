= Authentication flow for cluster introspection

[role="_abstract"]
{ols-long} authenticates cluster introspection requests to the Model Context Protocol (MCP) server by using secure token exchanges. This process ensures that only authorized entities can query cluster data. Production deployments support two primary authentication modes to meet information security requirements.

File-based secrets authentication::
The file-based secrets mode uses a static token loaded at service startup from a mounted {k8s} secret. Because this is a single shared credential, all users interacting with {ols-long} connect to the MCP server under the same identity. The MCP server cannot distinguish between individual users or enforce per-user access control. Administrators must use this mode only when the MCP server requires a dedicated service token and per-user authorization is not needed.

{k8s} passthrough authentication::
The {k8s} passthrough mode forwards the bearer token of the executing user directly to the MCP server. {ols-long} validates the incoming user token by using a {k8s} `TokenReview` API request before forwarding the credential. This mechanism ensures that the MCP server enforces the specific access permissions of the requesting user.
