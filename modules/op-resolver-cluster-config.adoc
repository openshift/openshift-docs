= Configuring the cluster resolver

[role="_abstract"]
You can change the default kind and namespace for the cluster resolver, or limit the namespaces that the cluster resolver can use.

.Procedure

. To edit the `TektonConfig` custom resource, enter the following command:
+
[source,terminal]
----
$ oc edit TektonConfig config
----
+
. In the `TektonConfig` custom resource, edit the `pipeline.cluster-resolver-config` spec:
+
[source,yaml]
----
apiVersion: operator.tekton.dev/v1alpha1
kind: TektonConfig
metadata:
  name: config
spec:
  pipeline:
    cluster-resolver-config:
      default-kind: pipeline
      default-namespace: namespace1
      allowed-namespaces: namespace1, namespace2
      blocked-namespaces: namespace3, namespace4
----
`default-kind`:: The default resource kind to fetch, if not specified in parameters.
`default-namespace`:: The default namespace for fetching resources, if not specified in parameters.
`allowed-namespaces`:: A comma-separated list of namespaces that the resolver is allowed to access. If this key is not defined, all namespaces are allowed.
`blocked-namespaces`:: An optional comma-separated list of namespaces that the resolver is blocked from accessing. If this key is not defined, all namespaces are allowed.
