= Checking the {microshift-short} Observability service state

After the {microshift-short} Observability service starts, you can verify the state of the service by using a `systemd` service. The {microshift-short} Observability service logs are available as `journald` logs.

.Procedure

. Check the service state by entering the following command:
+
[source,terminal]
----
$ sudo systemctl status microshift-observability
----

. Check the service logs by entering the following command:
+
[source,terminal]
----
$ sudo journalctl -u microshift-observability
----
