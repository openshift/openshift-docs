= Automatic rollback mechanisms for network reconfiguration

[role="_abstract"]
Network reconfiguration includes multiple automatic rollback safety mechanisms that help protect your cluster from failed configuration changes.

Automatic rollback on post-pivot failure::
If network configuration or certificate regeneration fails after the reboot, the system automatically triggers a rollback to the previous stateroot. You can disable this behavior by adding the `auto-rollback-on-failure.lca.openshift.io/ip-config-run: "Disabled"` annotation to the `IPConfig` CR.

Init-monitor timeout rollback::
If the network reconfiguration does not complete within the configured timeout, the system automatically triggers a rollback. The default timeout is 1800 seconds (30 minutes). You can configure this timeout using the `spec.autoRollbackOnFailure.initMonitorTimeoutSeconds` field in the `IPConfig` CR. Setting the value to `0` uses the default timeout. To disable the init-monitor rollback, add the `auto-rollback-on-failure.lca.openshift.io/init-monitor: "Disabled"` annotation to the `IPConfig` CR.

[WARNING]
====
Disabling automatic rollback mechanisms can leave your cluster in an unrecoverable state if the network reconfiguration fails. Only disable these mechanisms in controlled environments where you have alternative recovery options.
====
