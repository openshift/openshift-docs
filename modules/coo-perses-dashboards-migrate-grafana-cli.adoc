= Migrate Grafana dashboards to a Perses by using the CLI

[role="_abstract"]
You can use the `percli` CLI to convert a Grafana dashboard JSON file to Perses format and apply it as a `PersesDashboard` custom resource.

.Prerequisites

* You have access to Grafana dashboard JSON export files.
* You have edit permissions in the target namespace.
* You have created the necessary datasources in Perses.
* You have the `percli` tool installed on your local machine.

.Procedure

. Port forward the Perses service by running the following command:
+
[source,terminal]
----
$ oc -n openshift-cluster-observability-operator port-forward svc/perses 8080:8080
----

. In a separate terminal, log in to the Perses service by using the `percli` utility by running the following command:
+
[source,terminal]
----
$ percli login https://localhost:8080 --kube --insecure-skip-tls-verify
----

. Migrate your dashboard configuration to a custom resource file by running the following command:
+
[source,terminal]
----
$ percli migrate -f grafana-dashboard.json --online -o yaml --format cr --project my-namespace > perses-dashboard-cr.yaml
----

. Review the converted output and update datasource references as required.

. Apply the converted dashboard:
+
[source,terminal]
----
$ oc apply -f perses-dashboard-cr.yaml
----

.Verification

* Verify that the dashboard was created:
+
[source,terminal]
----
$ oc get persesdashboards -n <target_namespace>
----
