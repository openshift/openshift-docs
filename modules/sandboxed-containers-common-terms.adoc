= Common terms

.Sandbox
A sandbox is a controlled environment where programs can run. Within the controlled environment, you can execute untested or untrusted programs without risking harm to the host machine or the operating system. A sandbox keeps your workloads isolated from the rest of the system.

In the case of {sandboxed-containers-first}, sandboxing is achieved by running workloads in a different kernel using virtualization, providing enhanced control over the interactions between multiple workloads running on the same host.

.Pod
A pod is a kubernetes and OpenShift construct that represents resources where containers can be deployed. In other words, containers run inside pods. Pods are used for example to specify resources that can be shared between multiple containers.

In the case of {sandboxed-containers-first}, a pod is implemented as a virtual machine. Several containers can run in the same pod, and therefore in the same virtual machine.

.OpenShift sandboxed containers Operator
An operator is a software component that automates operations, automating actions that a human operator could do on the system.

The {sandboxed-containers} is tasked with managing the lifecycle of sandboxed containers on a cluster. It deals with operations such as sandboxed containers software installation and removal, and status monitoring.

.Kata Containers
Kata Containers is the name of one of the core upstream projects used in building {sandboxed-containers-first}. {sandboxed-containers-first} integrate Kata Containers with OpenShift.

.KataConfig
`KataConfig` represents a configuration of sandboxed containers. Information about the state of the cluster, for example the nodes on which the software was deployed, can be found in the `KataConfig` objects.

.Red Hat CoreOS extensions
Red Hat CoreOS (RHCOS) extensions are a mechanism to install optional OpenShift software. The {sandboxed-containers} uses this mechanism to deploy sandboxed containers on a cluster.

.Runtime class
A RuntimeClass object describes which runtime can be used to run a given workload. A runtime class named `Kata` is installed and deployed by the {sandboxed-containers}. The runtime class contains additional information about the runtime, such as the pod overhead, describing resources the runtime needs to operate.
