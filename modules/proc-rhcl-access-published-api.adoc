= Integrate with an API published by another team

[role="_abstract"]
When you need to call an API owned by another team that is protected with the `APIKey` authentication method, you can use the {ocp} web console to browse the internal API catalog and discover available services, then request access. If you are granted access, you can receive the credentials required to authenticate your application.

You can select only the API products that use the `APIKey` authentication method. APIs that use OpenID Connect (OIDC) authentication are available for access depending on your role defined in the `AuthPolicy` custom resource.

.Prerequisites

* You enabled the {prodname} {ocp} web console plugin.
* You have the `api-consumer` role bound in your namespace.
* You verified that the API you need is published in the catalog.
* You are in your own namespace.

.Procedure

. Browse the catalog to find an API that solves your problem by using the web console:
+
.. In the {ocp} web console, navigate to *{prodname}*.
.. Select the *API Products* page.
.. Look for APIs that match your use case based on their display name and description.

. Request access to the API by using the web console:
+
.. In the {ocp} web console, navigate to *{prodname} API Catalog*.
.. Select the *My API Keys* page.
.. Search for and select the API Product you want in the *API Product* field.
.. If the API offers multiple service tiers, for example, different rate limits, select the tier in the *Tier* field that matches your traffic requirements.
.. Enter a unique name for your API key resource in the *API Key Name* field.
.. Describe your use case in the *Use Case* field so that the API owner understands how you plan to use their service.
.. Click *Request*.

. Wait for approval:
+
* If the API uses automatic approval, your request is approved immediately and credentials are available.
* If the API requires manual approval, your request enters the `Pending` state. The API owner reviews your use case and approves or rejects the request.

. If you are using the API key authentication method, retrieve the credentials after approval by using the web console:
+
.. On the *My API Keys* page, click your approved API key request.
.. In the *Credentials* section, click *Show API Key*.
.. The credentials are accessible if the API key is approved. Accessible credentials are labeled with the *Active* status.

. Verify that your API key request is displayed in the *My API Keys* page with *Active* status.

. Verify that you can call the API by running the following command in your terminal window:
+
[source,terminal,subs="+quotes"]
----
$ curl -H "Authorization: Bearer _<api_key>_" \
  https://_<api_hostname>_/_<api_path>_
----
+
Replace the `_<api_key>_`, `_<api_hostname>_` and `_<api_path>_` API values with the ones required for your use case.

. If you are using either the OpenID Connect (OIDC) or JWT authentication method, you can use the web console to get the information needed to request a token and consume the API with the token:
+
.. In the web console, navigate to the *Connectivity Link* -> *APIProducts* page.
.. Find the API product that you want.
.. Click the *option icons* on the API tabs.
.. Select *Edit*.
.. On the *Edit API Product* page, navigate to the YAML View.
.. Save the information that you need to request a token.
+
.Example identity provider URL and credentials YAML snippet
[source,yaml]
----
status:
  discoveredAuthScheme:
    authentication:
      <name>:
        jwt:
          issuerUrl: https://keycloak.example.com/realms/myrealm
        credentials:
          authorizationHeader:
            prefix: Bearer
----
+
.Example token endpoint URL YAML snippet
[source,yaml]
----
status:
  oidcDiscovery:
    tokenEndpoint: https://keycloak.example.com/realms/myrealm/protocol/openid-connect/token
----

.. Configure your application to obtain JWT tokens from the identity provider by using your credentials. For example, client credentials flow or user login flow.
+
An API key is not required when your application uses JWT tokens to authenticate.

. Verify that you can call the API by running the following command in your terminal window:
+
[source,terminal,subs="+quotes"]
----
$ curl -H "Authorization: Bearer _<jwt_token>_" \
  https://_<api_hostname>_/_<api_path>_
----

.Next steps

* Integrate the API into your application. If you get errors or need help, see the API's documentation links or contact the API owner by using the information in the API product details.
