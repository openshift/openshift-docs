= Monitor cluster provisioning status

[role="_abstract"]
You can monitor the progress of a `ProvisioningRequest` by watching its status conditions and provisioning phase. The O-Cloud Manager updates the resource status as provisioning advances through hardware allocation, cluster installation, and configuration stages.

.Procedure

. Watch the `ProvisioningRequest` status in real time:
+
[source,terminal]
----
$ oc get provisioningrequests.clcm.openshift.io <uuid> -w
----

. View detailed status conditions for the request:
+
[source,terminal]
----
$ oc describe provisioningrequests.clcm.openshift.io <uuid>
----
+
The following status conditions indicate provisioning progress:
+
[cols="1,1,2",options="header"]
|===
|Condition |Status |Description

|`ProvisioningRequestValidated`
|`True`
|The provisioning request parameters passed schema validation.

|`ClusterInstanceRendered`
|`True`
|The `ClusterInstance` resource was rendered and passed dry-run validation.

|`ClusterResourcesCreated`
|`True`
|All required cluster resources (secrets, namespaces) were created on the hub.

|`HardwareProvisioned`
|`True`
|Hardware allocation, firmware updates, and BIOS configuration are complete.

|`ClusterInstalled`
|`True`
|The spoke cluster installation is complete.

|`ClusterConfigured`
|`True`
|All ACM policies are compliant on the spoke cluster.
|===

. Check the high-level provisioning phase:
+
[source,terminal]
----
$ oc get provisioningrequests.clcm.openshift.io <uuid> -o jsonpath='{.status.provisioningStatus.provisioningPhase}'
----
+
The `provisioningPhase` progresses through the following values:
+
* `pending` - The request is queued and awaiting processing.
* `progressing` - Provisioning is actively in progress.
* `fulfilled` - All stages completed successfully.
* `failed` - An error occurred during provisioning.

. If provisioning fails, inspect the status conditions to identify which stage encountered an error:
+
[source,terminal]
----
$ oc get provisioningrequests.clcm.openshift.io <uuid> -o jsonpath='{.status.conditions}' | jq .
----
+
Failed conditions include a `message` field with details about the failure cause. Common failures include hardware allocation timeout, cluster installation errors, or policy compliance failures.
