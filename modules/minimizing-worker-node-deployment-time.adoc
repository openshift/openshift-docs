= Minimizing worker node deployment time

You can use the following procedure to minimize deployment time during cluster worker node installation by applying configuration changes across nodes simultaneously.

[IMPORTANT]
====
This procedure assumes that no workloads run on the cluster worker nodes. Modifying the `maxUnavailable` value to `100%` while workloads run can violate high availability constraints.
====

.Prerequisites

*   You must log in to the cluster as a user with the `cluster-admin` role.
*   The cluster must include three separate worker Machine Config Pools (MCPs), such as `worker-1`, `worker-2`, and `worker-3`.
*   Perform this procedure while deploying the control plane nodes.

.Procedure

. Set the `maxUnavailable` parameter for each worker MCP to `100%` to ensure that all nodes update concurrently.
+
[source,terminal]
----
$ oc patch mcp/worker-1 --patch '{"spec":{"maxUnavailable":"100%"}}' --type=merge
$ oc patch mcp/worker-2 --patch '{"spec":{"maxUnavailable":"100%"}}' --type=merge
$ oc patch mcp/worker-3 --patch '{"spec":{"maxUnavailable":"100%"}}' --type=merge
----
+
.Example output
[source,terminal]
----
machineconfigpool.machineconfiguration.openshift.io/worker-1 patched
machineconfigpool.machineconfiguration.openshift.io/worker-2 patched
machineconfigpool.machineconfiguration.openshift.io/worker-3 patched
----
+
. Pause the worker MCPs by setting the `paused` field to `true`to prevent the Machine Config Operator (MCO) from applying initial configuration changes to the worker nodes while the control plane updates.
+
[source,terminal]
----
$ oc patch mcp/worker-1 --patch '{"spec":{"paused":true}}' --type=merge
$ oc patch mcp/worker-2 --patch '{"spec":{"paused":true}}' --type=merge
$ oc patch mcp/worker-3 --patch '{"spec":{"paused":true}}' --type=merge
----
+
.Example output
[source,terminal]
----
machineconfigpool.machineconfiguration.openshift.io/worker-1 patched
machineconfigpool.machineconfiguration.openshift.io/worker-2 patched
machineconfigpool.machineconfiguration.openshift.io/worker-3 patched
----
+
. Unpause all worker MCPs after the control plane nodes successfully deploy. Unpausing the MCPs triggers the MCO to apply pending configuration changes to all associated worker nodes simultaneously.
+
[source,terminal]
----
$ oc patch mcp/worker-1 --patch '{"spec":{"paused":false}}' --type=merge
$ oc patch mcp/worker-2 --patch '{"spec":{"paused":false}}' --type=merge
$ oc patch mcp/worker-3 --patch '{"spec":{"paused":false}}' --type=merge
----
+
.Example output
[source,terminal]
----
machineconfigpool.machineconfiguration.openshift.io/worker-1 patched
machineconfigpool.machineconfiguration.openshift.io/worker-2 patched
machineconfigpool.machineconfiguration.openshift.io/worker-3 patched
----
+
. Verify the progress of the worker node deployment.
+
[source,terminal]
----
$ oc get machineconfigpools
----
+
.Example output
[source,terminal]
----
NAME       CONFIG                                           UPDATED   UPDATING   DEGRADED   MACHINECOUNT   READYMACHINECOUNT   UPDATEDMACHINECOUNT   DEGRADEDMACHINECOUNT   AGE
master     rendered-master-b0bb90c4921860f2a5d8a2f8137c1867 True      False      False      3              3                   3                     0                      97m
worker-1   rendered-worker-config-new                       False     True       False      10             0                   0                     0                      5m
worker-2   rendered-worker-config-new                       False     True       False      10             0                   0                     0                      5m
worker-3   rendered-worker-config-new                       False     True       False      10             0                   0                     0                      5m
----
+
. Reset the `maxUnavailable` parameter for all worker MCPs. The recommended default value is `1`. This ensures cluster stability and high availability when users deploy workloads onto the cluster.
+
[source,terminal]
----
$ oc patch mcp/worker-1 --patch '{"spec":{"maxUnavailable":1}}' --type=merge
$ oc patch mcp/worker-2 --patch '{"spec":{"maxUnavailable":1}}' --type=merge
$ oc patch mcp/worker-3 --patch '{"spec":{"maxUnavailable":1}}' --type=merge
----
+
.Example output
[source,terminal]
----
machineconfigpool.machineconfiguration.openshift.io/worker-1 patched
machineconfigpool.machineconfiguration.openshift.io/worker-2 patched
machineconfigpool.machineconfiguration.openshift.io/worker-3 patched
----
