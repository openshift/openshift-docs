= Installing the {IBMFusionSAN} Operator

To install the {IBMFusionSAN} Operator, you must first install `kernel-devel` on nodes with shared storage. Then, install the Operator from the *OperatorHub* in the {product-title} web console.

.Prerequisites

* You installed the `oc` CLI.
* You have access to the cluster as a user with the `cluster-admin` role.

.Procedure

. Apply the MachineConfig to install kernel-devel on the nodes that have shared storage. This task is now delegated to the user and is no longer performed by the operator. ?????
. In the {product-title} web console, navigate to *Operators* -> *OperatorHub*.
. In the *Filter by keyword* field, type `IBM fusion for SAN`.
. Select the *{IBMFusionSAN}* tile and click *Install*.
. On the *Install Operator* page, select *stable* from the list of available *Update Channel* options.
. Verify that *Operator recommended Namespace* is selected for *Installed Namespace*. This installs the Operator in the `ibm-fusion-access` namespace.
+
If this namespace does not yet exist, it is automatically created.
. Verify that *Automatic* is selected for *Update Approval*. *Automatic* is the default value, and enables automatic updates when a new z-stream release is available.
. Click *Install*.

The {IBMFusionSAN} Operator is installed.

.Verification

* Once the {IBMFusionSAN} Operator installation is successful, a prompt appears to create  a `FusionAccess` custom resource (CR).

◦ On the Install Operator page, verify or select the desired options. Use the defaults for Update channel (alpha), Version (e.g., 0.0.12), Installation mode (All namespaces on the cluster), Installed Namespace (openshift-fusion-access), and Update approval (Automatic).


7. Create a Kubernetes Secret named fusion-pullsecret in the openshift-fusion-access namespace. This secret holds your IBM entitlement key, which is necessary for pulling required container images from the IBM container registry.

8. Create the FusionAccess Custom Resource (CR). This resource initializes components, reconciles the pull secret, and triggers the installation of the Spectrum Scale operator. The FusionAccess CR specification now primarily contains the version of the Fusion Software.

◦ If you are using the UI, you will see a prompt to create the required custom resource after the operator installation is successful. Click Create FusionAccess. You can accept the default name or provide a custom one.


After successfully completing these steps, the Fusion Access for SAN operator is installed and basic initial configuration is complete. You can then proceed with creating the Spectrum Cluster object and the FileSystem object to configure your storage environment. These subsequent steps can be performed either via CLI commands or through the Fusion Access for SAN UI wizard.
