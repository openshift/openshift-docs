= Starting the cluster upgrade

When upgrading from one y-stream release to the next, you need to make sure that the intermediate z-stream releases are also compatible.

[NOTE]
====
You can verify that you are upgrading to a viable release by running the `oc adm upgrade` command.
This command lists the compatible upgrade releases.
====

.Procedure
. Start the upgrade:
+
[source,terminal]
----
$ oc adm upgrade --to=4.15.33
----
+
.Example output
[source,terminal]
----
Requested update to 4.15.33
----
+
This will change depending on your particular upgrade:
+
[IMPORTANT]
====
* **EUS to EUS upgrade**: Make sure you point to the interim <y+1> release path
* **Y-stream upgrade** - Make sure you use the correct <y.z> release that follows the Kubernetes link:https://kubernetes.io/releases/version-skew-policy/[version skew policy].
* **Z-stream upgrade** - Verify that there are no problems moving to that specific release
====
