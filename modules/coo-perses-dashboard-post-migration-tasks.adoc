= Perses dashboard post-migration tasks

[role="_abstract"]
After importing a dashboard, you must complete the following tasks:

* Test all panels to verify that each panel displays data correctly.
* Update queries if the datasource configurations differ from the original Grafana instance.
* Refine visualizations to optimize panel settings for Perses.
* Confirm that all variables work with your Perses datasources.
* Document any differences from the original Grafana dashboard.

Limitations::

Not all Grafana features have direct equivalents in Perses. Review the following known limitations before migrating:

[cols="1,2",options="header"]
|===
|Feature | Limitation
|Custom plugins | Grafana-specific plugins do not have Perses equivalents. Use built-in Perses panel types instead.
|Panel types | Some advanced Grafana panel types require alternative visualizations in Perses.
|Template variables | Variable syntax might require manual adjustments after conversion.
|Alerting | Grafana alerting rules are not migrated. Configure alerting separately in OpenShift.
|===

[IMPORTANT]
====
Both the web console import tool and the `percli` CLI provide a starting point for migration. Manual adjustments might be required to achieve full functionality.
====
