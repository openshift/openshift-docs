= Long running requests

To ensure that your requests exceeding the default 600 second timeout set by Knative are not prematurely terminated, you can adjust the timeouts on the following components:

* {ocp-product-title} route
* Serving
* Load balancer (if applicable)

In order to support long running requests that exceed in duration the default 600s threshold, set by Knative, users need to adjust the timeouts at the OCP route side, the Knative side and depending on the cloud provider at the load balancer side.
