= About Support Log Gather

The Support Log Gather is an Operator that automates the uploading of the collected data to a specified Red{nbsp}Hat support case, making it easier for customers to provide the necessary data

Key Capabilities
• Automated Data Collection: Collects a wide range of debugging information, including resource definitions and service logs, across the entire cluster [3, 99, Conversation History].
• Automatic Upload to Red Hat Support: Eliminates the need for manual file transfers by directly attaching the collected data to the designated Red Hat support case [82, 100, Conversation History].
• Support for Specific Feature Data: Beyond the default quay.io/openshift/origin-must-gather:latest image (always included), administrators can specify additional mustGatherImages for OpenShift add-ons such as Kubevirt, OpenShift Container Storage (OCS), OpenShift Serverless, and more, ensuring targeted data collection for specific components.
• Namespace Scoping: The operator is designed to watch only the namespace in which it is deployed. This design assists cluster administrators in configuring limited access and properly protecting the service account, which requires cluster-admin privileges for its operations [102, Conversation History].
