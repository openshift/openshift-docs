= About Support Log Gather

The Support Log Gather is an Operator that simplifies the collection of debugging data from your cluster. It packages the collected information into a single `.tar` file and automatically uploads it to a specified Red{nbsp}Hat support case.

Key features of the {support-log-gather} include the following:

* **Automated data collection:** The Operator collects a wide range of debugging information from the entire cluster, including resource definitions and service logs.

* **Automatic upload to a support case:** The Operator directly attaches the compressed `.tar` file of collected data to the designated Red Hat support case.

* **Namespace scoping:** The Operator monitors only the namespace in which it is deployed. This design helps cluster administrators configure limited access and protect the service account, which requires `cluster-admin` privileges for its operations.

* **Targeted data collection:** In addition to the default `quay.io/openshift/origin-must-gather:latest` image, administrators can specify other `mustGatherImages` for targeted data collection. This includes add-ons such as Kubevirt, OpenShift Container Storage (OCS), and OpenShift Serverless.
