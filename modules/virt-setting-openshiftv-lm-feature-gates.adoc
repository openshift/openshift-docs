= Setting a live migration feature gate for each cluster in {VirtProductName}

To enable cross-cluster live migration, you must set a feature gate for each of the two clusters in {VirtProductName}.

.Procedure

* Set the feature gate by running the following command for each cluster:
+
[source,terminal]
----
$ oc patch hyperconverged kubevirt-hyperconverged -n openshift-cnv --type json -p '[{"op":
"replace", "path": "/spec/featureGates/decentralizedLiveMigration", "value": true}]'
----

.Verification

* To verify that the feature gate enablement is successful for each cluster, run the following command in the {VirtProductName} namespace for synchronization pods:
+
[source,terminal]
----
$ oc get -n openshift-cnv pod | grep virt-synchronization
----

* Verify that the {VirtProductName} installation is successful by provisioning and running some VMs. For more information, see link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.19/html/virtualization/creating-a-virtual-machine#virt-creating-vms-from-instance-types[Creating VMs from instance types].
