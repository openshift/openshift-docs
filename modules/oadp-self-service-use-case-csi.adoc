= Creating a non-admin backup and restore by using CSI

In this use case, as a non-admin user:

* You take a Container Storage Interface (CSI) backup of your application running in your authorized namespace. You do this by creating a `NonAdminBackup` (NAB) CR. The cluster administrator has enabled the `csi` plugin in the `DataProtectionApplication` (DPA). 
* To simulate disaster, you delete the application in your authorized namespace. 
* You then restore the application in your namespace by creating a `NonAdminRestore` (NAR) CR.

.Prerequisites

* You are logged in to the cluster as a non-admin user.
* The cluster administrator has installed the {oadp-short} Operator.
* The cluster administrator has configured the DPA to enable {oadp-short} Self-Service and the CSI plugin, as explained in section "Configuring the `DataProtectionApplication` for a non-admin CSI backup".
* The cluster administrator has created a namespace for you and has authorized you to operate from that namespace.
* You have installed an application in your authorized namespace. 

.Procedure

. To create a `NonAdminBackup` (NAB) CR for Data Mover backup of your application, configure a YAML manifest file as shown in the following example:
+
.Example `NonAdminBackup`
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: NonAdminBackup
metadata:
  name: test-nab
spec:
  backupSpec:
    includedNamespaces:
    - "test-nac" # <1>   
----
1. Specify the name of your authorized namespace in which your application is running. In this example, the namespace is `test-nac`.

. Create the NAB CR by running the following command:
+
[source,terminal]
----
$ oc apply -f <nab_cr_filename> # <1>
----
1. Specify the YAML file name containing the NAB CR configuration.

. Verify that the phase of the NAB CR is `Created` by running the following command:
+
[source,terminal]
----
$ oc get nab test-nab -o yaml
----
+

. To simulate disaster, delete the application running in your authorized namespace.

. Verify that no pods are running in your authorized namespace after you delete the application.
+
[source,terminal]
----
$ oc get pod -n test-nac # <1>
----
1. `test-nac` is your authorized namespace.

.Example output

[source,terminal]
----

----

. To create a `NonAdminRestore` (NAR) CR, configure a YAML manifest file as shown in the following example:
+
.Example `NonAdminRestore`
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: NonAdminRestore
metadata:
  name: test-nar
spec:
  restoreSpec:
    backupName: test-nab # <1>
----
1. Specify the name of the NAB CR in the `backupName` field. In this example, `test-nab` is the NAB CR.

. Create the NAR CR by running the following command:
+
[source,terminal]
----
$ oc apply -f <nar_cr_filename> # <1>
----
1. Specify the YAML file name containing the NAR CR configuration.

. Verify that the NAR CR is in the `created` phase and eventually, the `completed` phase by running the following command:
+
[source,terminal]
----
$ oc get nar test-nar -o yaml
----
+
.Example `NonAdminRestore`
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: NonAdminRestore
...
status:
  conditions:
  - lastTransitionTime: "2025-02-11T13:50:51Z"
    message: restore accepted
    reason: RestoreAccepted
    status: "True"
    type: Accepted
  - lastTransitionTime: "2025-02-11T13:50:51Z"
    message: Created Velero Restore object
    reason: RestoreScheduled
    status: "True"
    type: Queued
  phase: Created # <1>
  queueInfo:
    estimatedQueuePosition: 0
  veleroRestore:
    nacuuid: test-nac-test-nar-a1030210-73dd-427b-b94e-3ff83737948b
    name: test-nac-test-nar-a1030210-73dd-427b-b94e-3ff83737948b
    namespace: openshift-adp
    status:
      completionTimestamp: "2025-02-11T13:51:24Z"
      hookStatus: {}
      phase: Completed # <2>
      progress:
        itemsRestored: 29
        totalItems: 29
      startTimestamp: "2025-02-11T13:50:51Z"
      warnings: 6
----
1. The `status.phase` field is `Created`. 
2. After a few minutes, the `veleroRestore.status.phase` field is `Completed`, indicating a successful restore.
