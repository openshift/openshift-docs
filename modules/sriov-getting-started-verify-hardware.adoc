= Verify SR-IOV hardware compatibility

[role="_abstract"]
Before configuring SR-IOV, verify that your hardware supports SR-IOV and identify your network devices.

.Procedure

. List the worker nodes in your cluster:
+
[source,terminal]
----
$ oc get nodes
----

. Select a worker node with SR-IOV-capable hardware and open a debug session:
+
[source,terminal]
----
$ oc debug node/<node_name>
----

. Change to the host's root directory:
+
[source,terminal]
----
sh-4.4# chroot /host
----

. List network devices that support SR-IOV:
+
[source,terminal]
----
sh-4.4# lspci | grep -i ethernet
----
+
.Example output
[source,terminal]
----
19:00.0 Ethernet controller: Intel Corporation Ethernet Controller X710 for 10GbE SFP+ (rev 02)
19:00.1 Ethernet controller: Intel Corporation Ethernet Controller X710 for 10GbE SFP+ (rev 02)
----

. Verify that a device supports SR-IOV by checking for SR-IOV capabilities:
+
[source,terminal]
----
sh-4.4# lspci -v -s 19:00.0 | grep -i "Single Root"
----
+
.Example output
[source,terminal]
----
Capabilities: [160] Single Root I/O Virtualization (SR-IOV)
----
+
If you see "Single Root I/O Virtualization (SR-IOV)" in the output, the device supports SR-IOV.

. Note the PCI address of the SR-IOV device (for example, `0000:19:00.0`). You will use this in later steps.

. Exit the debug session:
+
[source,terminal]
----
sh-4.4# exit
sh-4.4# exit
----

.Verification

You have identified an SR-IOV-capable network device and its PCI address.
