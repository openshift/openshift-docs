[id="new-features-1-20-2_{context}"]
== New features

Web-based terminal issue resolved with SPDY fallback on OpenShift::
Before this update, the web-based terminal did not work on OpenShift clusters. The `WebSocket` exec handshake was rejected when the `TranslateStreamCloseWebsocketRequests` feature gate was enabled. As a result, users could not access the terminal from the web console. With this update, the system falls back to SPDY when the `WebSocket` handshake is rejected. As a result, the web-based terminal works as expected on OpenShift clusters.
+
link:https://issues.redhat.com/browse/GITOPS-9295[GITOPS-9295]

[id="fixed-issues-1-20-2_{context}"]
== Fixed issues

Init container permission issue resolved in rootless containers::
Before this update, the `init` container assumed a RHEL 8 environment. This caused permission issues on RHEL 9. As a result, the Argo CD repo server crashed when the `init` container failed. It then entered a continuous `CrashLoopBackOff` state. With this update, the `update-ca-trust` command supports an explicit output directory. This change allows correct write permissions in rootless containers. As a result, the Argo CD repo server no longer enters a `CrashLoopBackOff` state due to this issue. This improves stability and reliability.
+
link:https://issues.redhat.com/browse/GITOPS-9511[GITOPS-9511]

Pod Security label synchronization aligned with SCCs in the openshift-gitops namespace::
Before this update, the {gitops-title} Operator set static Pod Security labels on the `openshift-gitops` namespace. These labels could go out of sync with the cluster policy. With this release, the {gitops-title} Operator sets `security.openshift.io/scc.podSecurityLabelSync=true`. It no longer hard-codes `pod-security.kubernetes.io/*` labels on the namespace. OpenShift now synchronizes Pod Security labels from Security Context Constraints (SCCs). This ensures that labels stay aligned with cluster policy.
+
link:https://issues.redhat.com/browse/GITOPS-9158[GITOPS-9158]
