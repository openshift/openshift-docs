= About cluster-wide TLS profiles

You can use a `KubeletConfig` object to configure a custer-wide TLS security profile by defining the schema for a TLS security profile.

There are four TLS security profile types:

* `Old`. Use with very old clients or libraries.
* `Intermediate`. Use as a general-purpose server.
* `Modern` Use with modern clients that support TLS 1.3 with no need for backwards compatibility.
* `Custom`. Use with specific, individual TLS security profile parameters.

The `Old`, `Intermediate`, and `Modern` profiles are based on link:https://wiki.mozilla.org/Security/Server_Side_TLS#Recommended_configurations[recommended configurations]. The `Custom` profile provides the ability to specify individual TLS security profile parameters.

.Sample `Old` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Old
----

.Sample `Intermediate` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Intermediate
----

.Sample `Modern` profile configuration
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Modern
----

The `Custom` profile is a user-defined TLS security profile.

[WARNING]
====
You must be careful using a `Custom` profile, because invalid configurations can cause problems.
====

.Sample `Custom` profile
[source,yaml]
----
spec:
  tlsSecurityProfile:
    type: Custom
    custom:
      ciphers:
        - ECDHE-ECDSA-AES128-GCM-SHA256
        - ECDHE-RSA-AES128-GCM-SHA256
      minTLSVersion: VersionTLS11
----

The TLS profile being used is listed in the `/etc/kubernetes/kubelet.conf` file on each node.
