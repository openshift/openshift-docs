= Build lifecycle management

[role="_abstract"]
Build lifecycle management controls how long builds and build artifacts remain in your cluster.
You can configure retention policies to automatically clean up old builds and prevent resource exhaustion.
Managing the build lifecycle maintains cluster health and keeps build history manageable.

The build lifecycle includes all phases from build creation through completion and eventual cleanup.
Retention policies decide how many completed builds to keep and when to delete older build objects.
This is important because each build creates pods, logs, and metadata that consume cluster resources over time.

You can also manually cancel running builds when they are no longer needed or are consuming too many resources.
Lifecycle management includes configuring how failed builds are handled and whether they should be automatically retried.
By setting appropriate retention policies, you balance the need to preserve build history for troubleshooting with the need to reclaim cluster resources.
Effective lifecycle management ensures your build system remains performant and does not accumulate unnecessary data.
