= Identify pipelines that reference ClusterTask

[role="_abstract"]
Before migrating to the cluster resolver, identify all pipelines, pipeline runs, and task runs that reference `ClusterTask` resources in your cluster.

.Prerequisites

* You have access to an {OCP} cluster with {pipelines-title} installed.
* You have the necessary permissions to view pipeline resources across namespaces.

.Procedure

. Find `ClusterTask` references in `Pipeline` resources across all namespaces by running the following command:
+
[source,terminal]
----
$ oc get pipelines -A -o yaml | grep -B5 "kind: ClusterTask"
----
+
--
*Example output:*
[source,terminal]
----
namespace: my-project
  taskRef:
    name: buildah
    kind: ClusterTask
----
--

. Find `ClusterTask` references in `PipelineRun` resources by running the following command:
+
[source,terminal]
----
$ oc get pipelineruns -A -o yaml | grep -B5 "kind: ClusterTask"
----

. Find `ClusterTask` references in `TaskRun` resources by running the following command:
+
[source,terminal]
----
$ oc get taskruns -A -o yaml | grep -B5 "kind: ClusterTask"
----

. For each `ClusterTask` reference, note the following information:
+
* The task name
* The namespace where the pipeline or run exists
* Whether the task is a Red Hat-provided task or a custom task
* The pipeline or run name that references the task

.Verification

. Review your list of `ClusterTask` references to ensure completeness.

. Verify the format of a typical `ClusterTask` reference that needs migration:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: example-pipeline
spec:
  tasks:
  - name: build-image
    taskRef:
      name: buildah
      kind: ClusterTask
----
+
`ClusterTask` specifies that the reference format is no longer supported and must be migrated to use the cluster resolver.
