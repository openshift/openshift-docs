= About the {coo-full} Perses dashboard

[role="_abstract"]
You can use the Perses dashboards feature to create, manage, and view custom dashboards in the {ocp-product-title} web console. Perses is an open-source, cloud-native dashboard tool that supports metrics, traces, logs, and profiling data.

Perses provides the following capabilities:

Multiple datasource support::
Perses connects to multiple data sources, including Prometheus for metrics, Tempo for distributed tracing, and Loki for log aggregation. You can combine data from multiple sources on a single dashboard.

Kubernetes custom resource definitions (CRDs)::
Dashboards and datasources are Kubernetes resources. You manage them with `oc` or `kubectl`, integrate them into CI/CD pipelines, and apply RBAC policies using standard Kubernetes mechanisms.

Dashboards as Code::
Using the Perses Go SDK, you can create reusable and modular dashboards by using Go code.

GitOps workflow support::
Because dashboards are YAML resources, you can store them in version control, track changes, and roll back to previous versions.

Role-based access control (RBAC)::
Perses uses the standard Kubernetes RBAC model. Administrators can restrict users to specific namespaces, granting only the permissions needed to view or edit dashboards.

Panel and visualization plugins::
Perses supports time series charts, stat panels, gauge panels, bar charts, and tables. Panels are extensible through a plugin architecture.

[NOTE]
====
Perses integrates directly into the {ocp-product-title} web console under *Observe* > *Dashboards (Perses)*. You do not need a separate dashboard application. Dashboards are namespace-scoped, which aligns with the OpenShift project model and allows teams to manage their own observability data independently.
