= Version upgrades

[role="_abstract"]
Operator Lifecycle Manager (OLM) manages the {OTelOperator} on the {ocp-product-title} by default.

For version upgrades, OLM queries for available upgrades for the installed {OTelOperator}.

When OLM upgrades the {OTelOperator}, the Operator scans for running Collector instances and upgrades those instances to match the Operator's updated version.

During startup, the {OTelOperator} automatically upgrades all `OpenTelemetryCollector` custom resources and reconciles all managed instances.

If an error occurs, the Operator retries the upgrade at an exponential back-off rate. If an upgrade fails, the Operator retries the upgrade when it restarts.
