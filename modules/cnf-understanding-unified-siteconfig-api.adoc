= Understanding the unified SiteConfig API

The {sco} dynamically generates installation manifests based on user-defined cluster templates that use the data that is encapsulated in the `SiteConfig` CR.
The {sco} reconciles the `SiteConfig` CR by populating the templated fields that are referenced in the CR using the user-defined templates.
The Operator then validates and renders the installation manifests.
After rendering the manifests, the installation program consumes the manifests to deploy the clusters.

You can source the `SiteConfig` CR from your Git repository through ArgoCD, or you can manually create it on the hub cluster directly or through external tools and workflows.
As shown in Figure 1., the process involves the following steps:

. Create one or more sets of cluster templates on the hub cluster.
. Create a `SiteConfig` CR that references those cluster templates.
. After the resources are created, the {sco} validates them.
. The {sco} renders the installation manifests and performs a dry run.
. If the dry run is successful, the installation program consumes the installation manifests.
. The installation begins.
. The SiteConfig Controller continuously monitors for changes in the `ClusterDeployment` CR.
