= Configuring the Network Observability Operator FlowCollectorSlice as a project administrator

[role="_abstract"]
Implement decentralized network traffic analysis using the `FlowCollectorSlice` Custom Resource (CR). This feature allows project administrators to manage flow collection and data enrichment within their own namespaces.

As a project administrator, you can create a `FlowCollectorSlice` to customize how network flows are handled for your namespace.

.Prerequisites

* The Network Observability Operator is installed.
* You have `project-admin` permissions for the namespace.

.Procedure

. Create a YAML file named `flowCollectorSlice.yaml`:
+
[source,yaml]
----
apiVersion: flows.netobserv.io/v1alpha1
kind: FlowCollectorSlice
metadata:
  name: flowcollectorslice-sample
  namespace: netobserv
spec:
  sampling: 1
  subnetLabels:
    - name: External-Database
      cidrs:
        - 192.168.50.0/24
----

. Apply the configuration by running the following command:
+
[source,terminal]
----
$ oc apply -f flowCollectorSlice.yaml
----

.Verification

. Navigate to the *Network Traffic* view in the {product-title} web console.
. Ensure flows to `192.168.50.0/24` subnet are observed with label `External-Database`.
