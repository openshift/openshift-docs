= Configure build execution behavior

[role="_abstract"]
Control how builds execute by defining strategy parameters, system settings, and resource allocations. Build strategies encapsulate the tools and steps that transform source code into container images, while parameters customize their behavior for different use cases.

Build execution configuration includes:

* Step resource definition and Pod configuration: CPU, memory, and disk limits for individual build steps and pod-level scheduling constraints
* Strategy parameters definition: Custom variables that modify build strategy behavior, such as Dockerfile arguments, builder images, or registry settings
* System parameters definition: Reserved variables automatically populated by the build controller, providing runtime information like source paths, output images, and context directories

These settings enable platform engineers to create reusable build strategies that balance flexibility with standardization. Users select appropriate strategies and provide parameter values without modifying the underlying build logic.
