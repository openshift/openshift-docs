= Build container images based on your application requirements

[role="_abstract"]
{builds-shortname} provides build strategies to create container images from your application code.
You can build from Dockerfiles, source code, or existing OCI artifacts depending on your application's requirements and your preferred workflow.

Your application requirements decide which build strategy to use.
Applications with Dockerfiles can use the `Buildah` strategy for complete control over the build process.
Applications without Dockerfiles can use `source-to-image` (S2I) or Cloud Native Buildpacks to build directly from source code.

If the bundled build strategies do not meet your requirements, you can create custom build strategies.
By using the custom build strategies, you can implement specialized build workflows.

Each strategy supports different source inputs and customization levels.
You can use different strategies for different applications within the same cluster based on their specific needs. 
You can also use the Open Container Initiative (OCI) artifacts for copying or promoting existing images.
