= Changing the VM instance type by using the CLI

[role="_abstract"]
As a VM owner, if you want to change the instance type of a VM, you can do this by changing the `name` field in the VM spec. This triggers the update logic, ensuring a new, immutable controller revision snapshot is taken of the new resource configuration.

.Prerequisites

* You created the VM by using an instance type.
* You have installed the {oc-first}.

.Procedure

. Stop the VM.

. Run the following command, replacing `{vm-name}` with the name of your VM, and `{new-instancetype}` with the name of the new instance type you want to change to:
+
[source,terminal]
----
$ oc patch vm/{vm-name} --type merge -p '{"spec":{"instancetype":{"name": "{new-instancetype}"}}'
----

. Restart the VM.
