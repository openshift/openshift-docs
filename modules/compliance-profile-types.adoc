= Compliance Operator profile types

Compliance Operator rules are organized into profiles. Profiles can target the {product-title} Platform or Nodes, and some standards include `rhcos4` Node profiles.

Platform:: Platform profiles evaluate your {product-title} cluster components. For example, confirming APIServer configurations are using strong encryption cyphers is Platform-level rule.

Node:: Node profiles for evaluating the OpenShift configuration of each host. For example, confirming kubeconfig files have the correct permissions to meet a compliance standard is a Node-level rule.

`rhcos4` Node:: Node profiles for evaluating the {op-system-first} configuration of each host. For example, confirming the SSHD service is configured to disable password logins is an {op-system}-level rule.

[IMPORTANT]
====
For compliance standards that have Node and Platform profiles, such as PCI-DSS, you must run both profiles in your {product-title} environment.

For compliance standards that have Node, Platform, and `rhcos4` Node profiles, such as FedRAMP High, you must run all three profiles in your {product-title} environment.
====

[NOTE]
====
Both Node and `rhcos4` Node scans may take a long time to complete in a cluster with many Nodes.
====
