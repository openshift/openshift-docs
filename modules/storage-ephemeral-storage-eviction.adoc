= Ephemeral storage configuration affects pod eviction

[role="_abstract"]
To manage pod lifecycle and prevent resource exhaustion, review how ephemeral storage settings in the pod specification trigger eviction. 

The settings in the pod specification affect when kubelet evicts pods. At the container level, the kubelet eviction manager measures disk usage when the first container sets a resource limit, such as 4 GiB. The kubelet eviction manager also marks the pod for eviction if the total usage exceeds the overall pod storage limit (8 GiB).

[NOTE]
====
This policy is strictly for `emptyDir` volumes and is not applied to persistent storage. You can specify the `priorityClass` of pods to exempt the pod from eviction.
====
