= Selecting an update scenario

[role="_abstract"]
The following decision matrix helps you determine which update scenario to use:

.Decision matrix for update scenarios
[cols="1,2,2",options="header"]
|===
|Scenario
|When to use
|Key considerations

|Z-stream
|Security patches or bug fixes needed
|Apply weekly; minimal risk; minimal downtime

|Y-stream
|New features required or approaching end of support
|Quarterly cadence; review release notes; plan maintenance window

|EUS-to-EUS
|Long-term planning with staged rollouts
|18-month support; control plane and workers can update separately; larger version jumps require more validation
|===
