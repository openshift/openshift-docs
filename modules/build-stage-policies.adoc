= Build stage policies

[role="_abstract"]

{product-title-short} uses build stage policies to discover and respond to security violations before code is deployed. Build stage policies are composed entirely of image criteria, such as image registry, image content, and CVE data, including scan results and status.

Build stage policies are used when code is evaluated during the pipeline by using the {product-title-short} `roxctl image check` command. When a violation is discovered, {product-title-short} returns a `-1`. These violations are not collected or acted on by {product-title-short}, but are used with a CI/CD tool. You configure your CI/CD tool to act on those results, for example, to fail the build process if a violation is reported.
