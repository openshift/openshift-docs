= Viewing logs in the Developer Observe view

[role="_abstract"]
You can view application logs for your project in the Developer perspective Observe view. The Logs tab provides project-level log aggregation and filtering capabilities in the same context as metrics and alerts.

[NOTE]
====
The Logs tab in the Observe view provides project-level log aggregation.
This differs from viewing logs for individual pods in the Topology view, which shows logs for a specific pod only.
====

.Prerequisites

* You have access to the cluster as a developer or as a user with view permissions for the project.
* Your applications are deployed and writing logs to stdout or stderr.

.Procedure

. In the Developer perspective of the {product-title} web console, click *Observe* and go to the *Logs* tab.

. Select the resource type to view logs for:
+
--
* *Deployment*: View logs for all pods managed by a deployment
* *DeploymentConfig*: View logs for all pods managed by a deployment configuration
* *StatefulSet*: View logs for all pods in a stateful set
* *Pod*: View logs for a specific pod
* *Container*: View logs for a specific container within a pod
--

. Select a specific resource from the resource dropdown list.
+
The Logs tab displays logs for the selected resource.

. Optional: Use the log viewer controls to customize the log view:
+
--
* *Show timestamps*: Toggle to display or hide timestamps for each log line
* *Wrap lines*: Toggle to enable or disable line wrapping for long log entries
* *Download*: Click to download the current log view as a text file
* *Expand*: Click to open the log view in full-screen mode
--

. Optional: Filter logs by searching for specific text.
+
In the search field, enter a keyword or phrase to filter log entries. The log view displays only entries containing the search term.

. Optional: View real-time log streaming.
+
By default, the Logs tab streams logs in real time. New log entries appear automatically as your application generates them.

. Optional: Pause log streaming to review historical logs.
+
Click the *Pause* button in the log viewer toolbar to stop streaming new log entries.
This is useful when analyzing specific log entries without new entries scrolling the view.
+
Click the *Resume* button in the log viewer toolbar to resume streaming logs.

.Verification

* The Logs tab displays log entries from the selected resource.
* Log entries update in real time as your application generates new logs.
