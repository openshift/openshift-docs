= {SMProductName} version 3.3.3

[role="_abstract"]
This release of {SMProductName} is included with the {SMProductName} Operator 3.3.3 and is supported on {ocp-product-title} 4.18 and later. This release addresses enhancements, fixed issues, and Common Vulnerabilities and Exposures (CVEs).

For supported component versions for 3.3.3, see "Service Mesh version support tables".

[id="ossm-fixed-issues-3-3-3_{context}"]
== Fixed issues

Icons restored to OperatorHub catalog cards in the {ocp-product-title} web console::
This fix restores icons to OperatorHub catalog cards for Sail Operator and {SMProductShortName} Operator, enhancing visibility and ease of use. 
+
link:https://redhat.atlassian.net/browse/OSSM-13028[OSSM-13028]

Improved performance and proxy readiness in FIPS clusters::
Previously, in {SMProductShortName} 3.3, the mesh performed unnecessary and extended validation of `ISTIO_MUTUAL` Transport Layer Security (TLS) keys within Envoy. This occurred in clusters with Federal Information Processing Standards (FIPS) enabled. As a consequence, the readiness time for Envoy proxies increased and applying configuration changes required additional time. This fix removes the unnecessary validation and optimizes the key processing. As a result, Envoy proxies reach a ready state more quickly and {SMProductShortName} applies configuration changes more efficiently.
+
link:https://redhat.atlassian.net/browse/OSSM-12929[OSSM-12929]

Mutual TLS lock icons restored to the {ossmc-full} graph::
Before this update, an issue with internal icon font references prevented mutual TLS (mTLS) lock icons from displaying in the {ossmc-full} console. As a consequence, users could not visually identify mutual TLS connections in {ossmc-full}. This update resolves the font compatibility issue. As a result, the icons display correctly.
+
link:https://redhat.atlassian.net/browse/OSSM-13773[OSSM-13773]
