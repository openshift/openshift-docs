= Understanding tailored profiles in compliance scans

[role="_abstract"]
Tailored profiles are custom compliance profiles that you create in the Compliance Operator and use in {product-title} ({product-title-short}). Tailored profiles build on standard compliance profiles provided by the Compliance Operator, such as Center for Internet Security (CIS), National Institute of Standards and Technology (NIST), Payment Card Industry Data Security Standard (PCI-DSS), and Security Technical Implementation Guide (STIG) benchmarks. You can use tailored profiles to extend existing benchmarks and customize the combination of rules that you are using.

You create tailored profiles at the cluster level by using `TailoredProfile` custom resources (CRs), then use {product-title-short} to schedule and manage scans across multiple clusters.

[NOTE]
=====
{product-title-short} does not currently create, copy, or synchronize tailored profiles between clusters.
You must create the `TailoredProfile` CR on each cluster where you want to run the profile.
=====

[id="when-to-use-tailored-profiles_{context}"]
== When to use tailored profiles

Use tailored profiles when standard compliance profiles do not fully match your organization's requirements.
Common scenarios include:

Rule suppression:: You can exclude compliance checks that require manual verification or are not applicable to your environment.
For example, you might suppress checks that verify physical security controls or processes that cannot be automated.

Profile combination:: You can merge compliance checks from multiple standard profiles into a single tailored profile.
This approach helps you meet requirements from different regulatory frameworks or internal policies without running separate scans.

Profile extension with custom values:: You can adjust thresholds, timeouts, or other parameters in compliance checks by using the `setValues` field.
For example, you might require stricter password policies or different file permission settings than the default benchmark specifies.

Custom rules:: You can add organization-specific compliance checks by using the custom rules in Compliance Operator.
With custom rules, you can verify configurations unique to your environment that standard benchmarks do not cover. See "Adding custom labels and annotations to a `CustomRule` object".

[id="tailored-profile-integration-with-rhacs_{context}"]
== Tailored profile workflow

Tailored profiles integrate with {product-title-short} through the following workflow:

1. You create `TailoredProfile` CRs in the Compliance Operator on each target cluster.
The `TailoredProfile` resource specifies which standard profile to extend, which rules to enable or disable, and any custom values to apply.

2. {product-title-short} automatically discovers tailored profiles available on secured clusters.
Tailored profiles appear in the {product-title-short} compliance interface with standard profiles.

3. From the {product-title-short} portal, you select tailored profiles and schedule scans across one or more clusters.
{product-title-short} creates the necessary `ScanSettingBinding` resources on target clusters to start the scans.

4. {product-title-short} collects scan results from all clusters and aggregates them in the *Coverage* dashboard.
Results from tailored profiles appear in a dedicated *Tailored Profiles* tab, separate from standard profile results.

[id="tailored-profile-supported-types_{context}"]
== Supported tailored profile types

{product-title-short} supports all `TailoredProfile` CR configurations available in the Compliance Operator:

.Tailored profile types
[cols="1,2",options="header"]
|===
|Type
|Description

|Profile extension
|Profiles that extend a single standard profile by using the `extends` field

|Disabled rules
|Profiles that exclude specific compliance checks by using the `disableRules` field

|Enabled rules
|Profiles that explicitly enable compliance checks by using the `enableRules` field

|Manual rules
|Profiles that mark specific checks as requiring manual verification by using the `manualRules` field

|Modified values
|Profiles that override default values for compliance checks by using the `setValues` field

|Custom rules
|Profiles that include organization-specific checks by using the Compliance Operator `CustomRule` object

|Profile combination
|Profiles that combine checks from more than one standard profile by using `extends` entries or rule combinations
|===
