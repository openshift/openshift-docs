= Installing {clo}

After you have installed the log store, install {clo}, which collects and forwards the logs to the log store. This can be done from the {product-title} CLI.

.Prerequisites

* You have administrator permissions.
* You installed the {oc-first}.
* You installed and configured {loki-op}.

.Procedure

. Create a `Subscription` object for {clo}:
+
.Example `Subscription` object
[source,yaml]
----
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: cluster-logging
  namespace: openshift-logging # <1>
spec:
  channel: stable-6.<y> # <2>
  name: cluster-logging
  source: redhat-operators # <3>
  sourceNamespace: openshift-marketplace
----
<1> You must specify the `openshift-logging` namespace.
<2> Specify `stable-6.<y>` as the channel.
<3> Specify `redhat-operators`. If your {product-title} cluster is installed on a restricted network, also known as a disconnected cluster, specify the name of the `CatalogSource` object you created when you configured the Operator Lifecycle Manager (OLM).

. Apply the `Subscription` object by running the following command:
+
[source,terminal]
----
$ oc apply -f <filename>.yaml
----

. Create an `OperatorGroup` object:
+
.Example `OperatorGroup` object
[source,yaml]
----
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: cluster-logging
  namespace: openshift-logging # <1>
spec:
  targetNamespaces:
  - openshift-logging
----
<1> You must specify the `openshift-logging` namespace.

. Apply the `OperatorGroup` object by running the following command:
+
[source,terminal]
----
$ oc apply -f <filename>.yaml
----

. Create a `ClusterLogForwarder` CR:
+
.Example `ClusterLogForwarder` CR
[source,yaml]
----
apiVersion: observability.openshift.io/v1
kind: ClusterLogForwarder
metadata:
  name: instance # <1>
  namespace: openshift-logging # <2>
spec:
  serviceAccount:
    name: logging-collector # <3>
  outputs:
  - name: lokistack-out
    type: lokiStack # <4>
    lokiStack:
      target: # <5>
        name: logging-loki 
        namespace: openshift-logging
      authentication:
        token:
          from: serviceAccount
    tls:
      ca:
        key: service-ca.crt
        configMapName: openshift-service-ca.crt
  pipelines:
  - name: all-logs
    inputRefs: # <6>
    - application
    - infrastructure
    outputRefs:
    - lokistack-out
----
<1> Use the name `instance`.
<2> You must specify the `openshift-logging` namespace.
<3> Use the service account created before.
<4> Select the `lokiStack` output type to send logs to the LokiStack instance.
<5> Point the ClusterLogForwarder to the LokiStack instance.
<6> Select the log output types you want to send to the LokiStack instance.

. Apply the `ClusterLogForwarder CR` object by running the following command:
+
[source,terminal]
----
$ oc apply -f <filename>.yaml
----

. Verify the installation by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-logging
----
