= Configuring a BGP peer

[role="_abstract"]
Configure a `BGPPeer` custom resource to exchange routing information with network routers and advertise IP addresses for `LoadBalancer` services.

.Prerequisites

* Install the {oc-first}.
* Log in as a user with `cluster-admin` privileges.
* Configure MetalLB with a BGP advertisement.

.Procedure

. Create a file, such as `bgppeer.yaml`, with content like the following example:
+
[source,yaml]
----
apiVersion: metallb.io/v1beta2
kind: BGPPeer
metadata:
  namespace: metallb-system
  name: doc-example-peer
spec:
  peerAddress: 10.0.0.1
  peerASN: 64501
  myASN: 64500
  routerID: 10.10.10.10
# ...
----
+
.BGPPeer parameters
[cols="2,1,3",options="header"]
|===
| Parameter | Format | Description

| `spec.peerAddress`
| IP address
| The IP address of the BGP peer router.

| `spec.peerASN`
| Integer (0-4294967295)
| The Autonomous System Number (ASN) of the BGP peer router.

| `spec.myASN`
| Integer (0-4294967295)
| The local ASN that MetalLB uses when establishing the BGP session.

| `spec.routerID`
| IP address
| Optional. The router ID to use for the BGP session. If not specified, MetalLB uses the node IP.

| `spec.bfdProfile`
| String
| Optional. Name of a `BFDProfile` resource for Bidirectional Forwarding Detection.

| `spec.holdTime`
| Duration (minimum `3s`)
| Optional. The hold time for the BGP session. Default is `90s`.

| `spec.password`
| String
| Optional. MD5 password for BGP session authentication.
|===

. Apply the BGP peer configuration by entering the following command:
+
[source,terminal]
----
$ oc apply -f bgppeer.yaml
----

.Verification

. Verify the `BGPPeer` resource is created:
+
[source,terminal]
----
$ oc get bgppeer -n metallb-system
----
+
.Example output
[source,terminal]
----
NAME               AGE
doc-example-peer   10s
----

. Verify the BGP session is established by checking the FRR logs on a speaker pod:
+
[source,terminal]
----
$ oc exec -n metallb-system -c frr $(oc get pods -n metallb-system -l component=speaker -o jsonpath='{.items[0].metadata.name}') -- vtysh -c "show bgp summary"
----

.Troubleshooting
[cols="2,2,2",options="header"]
|===
| Symptom | Likely cause | Resolution

| BGP session not established
| Peer router is unreachable or not configured to accept the BGP session
| Verify network connectivity to the peer address. Confirm the peer router is configured with the matching ASN and accepts connections from the cluster nodes.

| BGP session flapping
| Hold time or keepalive mismatch between MetalLB and the peer router
| Ensure both sides use compatible hold time and keepalive values.

| Routes not advertised
| No `BGPAdvertisement` resource configured, or advertisement does not reference the correct `IPAddressPool`
| Verify: `oc get bgpadvertisement -n metallb-system -o yaml` and check `ipAddressPools` references.

| `no matches for kind "BGPPeer"`
| Operator CRDs not yet available
| Wait for the Operator CSV to reach the `Succeeded` phase.
|===
