= Resolve task resolution failed or resource not found errors

If a pipeline run fails with a "resolution failed" or "resource not found" error, the cluster resolver cannot find the specified task. This can occur when the task does not exist in the specified namespace, the task name is misspelled in the pipeline definition, or the namespace name is incorrect.

.Procedure

. Verify that the task exists in the namespace by running the following command:
+
[source,terminal]
----
$ oc get task <task_name> -n <namespace>
----
+
<task_name>:: Specifies the name of the task you are trying to reference.
<namespace>:: Specifies the namespace where the task should be located.

. If the task does not exist, check if it is available in the {pipelines-shortname} installation namespace by running the following command:
+
[source,terminal]
----
$ oc get tasks -n openshift-pipelines
----

. Verify the spelling of the task name and namespace in your pipeline definition.
