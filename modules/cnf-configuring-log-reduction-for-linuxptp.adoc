= Configuring PTP log reduction

The `linuxptp` daemon generates logs that you can use for debugging purposes. In telco or other deployment types that feature a limited storage capacity, these logs can add to the storage demand. Currently, the default logging rate is high, causing logs to rotate out in under 24 hours, which makes it difficult to track changes and identify problems.

Basic log reduction can be achieved by configuring the PtpConfig custom resource (CR) to exclude log messages that report the master offset value. The master offset log message reports the difference between the current node's clock and the master clock in nanoseconds. However, with this method, there is no summary status of filtered logs. An enhanced log reduction feature is available that allows you to configure the logging rate of PTP logs. This feature enables you to set a specific logging rate, which can help reduce the volume of logs generated by the `linuxptp` daemon while still retaining essential information for troubleshooting. An option is also available with this feature to specify a threshold where it would still display the offset logs if the offset is higher than that threshold.
