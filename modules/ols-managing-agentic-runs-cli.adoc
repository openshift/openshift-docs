= Managing agentic runs by using the CLI

[role="_abstract"]
You can manage agentic runs by using the `oc agentic` CLI plugin as an alternative to the web console.

.Prerequisites

* The `oc-agentic` CLI plugin is installed. See xref:ols-installing-oc-agentic-cli_{context}[Installing the oc-agentic CLI plugin].
* You are authenticated to the cluster.

.Procedure

* To list all agentic runs:
+
[source,terminal]
----
$ oc agentic run list
----

* To inspect a specific run:
+
[source,terminal]
----
$ oc agentic run get <run_id>
----

* To approve a stage:
+
[source,terminal]
----
$ oc agentic run approve <run_id> --stage=<stage>
----
+
Replace `<stage>` with `analysis`, `execution`, or `verification`.

* To approve all pending stages and wait for completion:
+
[source,terminal]
----
$ oc agentic run approve <run_id> --all --wait
----

* To deny a stage:
+
[source,terminal]
----
$ oc agentic run deny <run_id> --stage=<stage>
----

* To watch phase transitions in real time:
+
[source,terminal]
----
$ oc agentic run watch <run_id>
----

* To stream sandbox pod logs:
+
[source,terminal]
----
$ oc agentic run logs <run_id> --step=Execution -f
----

[NOTE]
====
The default namespace is `openshift-lightspeed` unless overridden with `-n` or the kubeconfig context.
====
