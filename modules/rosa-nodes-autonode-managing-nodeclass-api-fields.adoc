= `OpenShiftEC2NodeClass` configuration fields

[role="_abstract"]
The following example and field descriptions define the configurable fields for a non-default `OpenShiftEC2NodeClass` resource.

.Example `OpenShiftEC2NodeClass` manifest
[source,yaml,subs="+quotes"]
----
apiVersion: karpenter.hypershift.openshift.io/v1
kind: OpenshiftEC2NodeClass
metadata:
  name: <nodeclass_name>
spec:
  version: "<openshift_version>"
  metadataOptions:
    httpTokens: "<token_requirement>"
  subnetSelectorTerms:
    - tags:
        karpenter.sh/discovery: "<cluster_id>"
        <additional_tag_key>: "<additional_tag_value>"
  securityGroupSelectorTerms:
    - tags:
        karpenter.sh/discovery: "<cluster_id>"
    - id: "<security_group_id>"
  tags:
    <tag_key>: "<tag_value>"
  capacityReservationSelectorTerms:
    - tags:
        <reservation_tag_key>: "<reservation_tag_value>"
----
where:

`metadata.name`::
The name of the `OpenShiftEC2NodeClass` resource. The name `default` is reserved for the system-managed immutable resource. Non-default resources must use a different name.

`spec.version`::
Pins the {ocp-short} version for all nodes provisioned by this class. The pinned version can be up to 3 minor versions behind the hosted control plane version. For example, if the hosted control plane is version 4.25, the minimum pinned version is 4.22.
+
[IMPORTANT]
====
The 3 minor version skew for `OpenShiftEC2NodeClass` resources differs from the 2 minor version skew supported by standard {product-title} machine pools.
====

`spec.metadataOptions.httpTokens`::
Controls whether the EC2 Instance Metadata Service requires session tokens. Set to `optional` to allow IMDSv1 access without HTTP tokens. Set to `required` to enforce IMDSv2 with HTTP tokens. The default value is `required`.

`spec.subnetSelectorTerms`::
Specifies the VPC subnets where the {autonode} launches worker nodes. Use tag-based selectors to target subnets that are different from the cluster's default VPC subnets. Each entry in the list is a selector term that matches subnets by tags.

`spec.securityGroupSelectorTerms`::
Specifies the AWS security groups to attach to worker nodes. You can select security groups by tags or by ID. Add additional entries to include security groups beyond the cluster defaults.

`spec.tags`::
Specifies additional AWS tags to apply to all EC2 instances that the {autonode} provisions using this class. Tags are key-value pairs.

`spec.capacityReservationSelectorTerms`::
Specifies the EC2 Capacity Reservations that the {autonode} can use when provisioning worker nodes. Use tag-based selectors to match specific reservations.
