= Optional: Uploading the RHCOS image in OpenStack

You can specify which {op-system} image to use when deploying the node pools on OpenStack by uploading the image to the OpenStack cloud. If you do not upload the image to OpenStack, OpenStack Resource Controller (ORC) manages the {op-system} image lifecycle by downloading the image from the OpenShift mirror and deleting it when the hosted cluster is deleted.

.Prerequisites

* You have downloaded the {op-system} image from the OpenShift mirror.
* You have access to the OpenStack cloud.

.Procedure

. Upload an {op-system} image to OpenStack:
+
[source,terminal]
----
openstack image create --disk-format qcow2 --file rhcos-openstack.x86_64.qcow2 rhcos
----
+
[NOTE]
====
The `rhcos-openstack.x86_64.qcow2` file is the {op-system} image that was downloaded from the OpenShift mirror. You can download the latest {op-system} image from the Red Hat OpenShift Container Platform mirror.
====
