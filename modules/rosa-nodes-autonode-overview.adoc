= How `AutoNode` works

[role="_abstract"]
`AutoNode` is a {product-title} feature that creates and removes worker nodes based on workload scheduling. It reduces the need to size fixed machine pools for every workload pattern.

`AutoNode` builds on the open source `Karpenter` project. The `Karpenter` software performs the following work:

* Observes pods that the scheduler cannot place on existing capacity
* Evaluates resource, placement, and affinity requirements for those pods
* Creates nodes that match those requirements
* Removes or replaces nodes when they are no longer needed, based on disruption settings that you configure

You set limits on how `AutoNode` can provision capacity. Examples include acceptable instance families, zones, and capacity types.
You express workload requirements through standard Kubernetes scheduling fields on your pods and controllers.
`AutoNode` works to place pending workloads within those limits.

[NOTE]
====
`AutoNode` does not replace every use case for machine pools.
Machine pools remain the supported way to manage long-lived, explicitly sized groups of worker nodes across zones.
Choose `AutoNode` when you want dynamic scaling that follows pod scheduling behavior closely.
====
