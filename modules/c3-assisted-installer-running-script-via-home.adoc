= Running the Terraform script via the Home region

All the Identity related resources, including Dynamic Groups, Policies and Tags, are created in the {oci-first-no-rt} Home Region (for example, PHX, IAD, FRA, etc.). Other resources can be created on {oci-c3-short}.  

Run two Terraform scripts on the bastion server created in previous steps. The first script creates the 
Dynamic Group Identity resources on your {oci-c3-short} {oci} Home tenancy. The second script creates infrastructure resources on {oci-c3-short}. 

.Prerequisites

* You have tenancy privileges to create Dynamic Groups and Policies. If not, you can provision them manually during this procedure.

.Procedure

. Connect to the bastion server via SSH.

. Create `OpenShift\createResourceOnHomeRegion` folders. 

. Copy the `createInfraResources` and `terraform.tfvars` files from the C3_PCA GitHub repository into the `createResourceOnHomeRegion` folder.

. Ensure that you have access to the source environment, and that your C3 certificate has been exported.

. Run the Terraform scripts.

For the full procedure, see link:https://www.oracle.com/a/otn/docs/compute_cloud_at_customer_agent_based_installation.pdf?source=:em:nl:mt::::PCATP[Terraform Script Execution - Part 1 (Oracle documentation)].
