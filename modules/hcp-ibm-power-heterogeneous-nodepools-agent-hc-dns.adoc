= Configuring DNS

You can point `*.apps.<cluster_name>` record to either one of the worker nodes where ingress application is hosted, or if you are able to set up a load balancer on top of the worker nodes, point this record to this load balancer. When you are creating a heterogeneous nodepool, make sure the workers are reachable from each other or keep them in the same network.

.Procedure

. Create a hosted cluster for heterogeneous nodepool
+
Use a multi-architecture release image when creating the cluster to enable support for heterogeneous node pools. Find the latest multi architecture images on the link:https://multi.ocp.releases.ci.openshift.org/[Multi-arch release images] page.

. Export the cluster namepsace.
+
[source,terminal]
----
$ export CLUSTERS_NAMESPACE=<hosted_cluster_namespace>
----

. Export the machine classless inter-domain routing (CIDR) notation.
+
[source,terminal]
----
$ export MACHINE_CIDR=192.168.122.0/24
----
// Typically the namespace is created by the hypershift-operator 
// but agent cluster creation generates a capi-provider role that
// needs the namespace to already exist

. Run the following command to create the hosted control namespace:
+
[source,terminal]
----
$ oc create ns ${HOSTED_CONTROL_PLANE_NAMESPACE}
----

. Run the following command to create the cluster:
+
[source,terminal]
----
$ hypershift create cluster agent \
    --name=<hosted_cluster_name> \ <1>
    --pull-secret=<pull_secret_file> \ <2>
    --agent-namespace=<hosted_control_plane_namespace> \ <3>
    --base-domain=<basedomain> \ <4>
    --api-server-address=api.<hosted_cluster_name>.<basedomain> \
    --release-image=quay.io/openshift-release-dev/ocp-release:<ocp_release> <5>
----
+
<1> The hosted cluster name.
<2> The pull secret file path.
<3> The namespace for the hosted control plane.
<4> The basedomain for the hosted cluster.
<5> The current Openshift release version.
