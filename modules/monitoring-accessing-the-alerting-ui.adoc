= Accessing the Alerting UI

[role="_abstract"]
The Alerting UI is accessible through the Administrator perspective for cluster-wide alerting or the Developer perspective for project-scoped monitoring.

Choose the Administrator perspective if you need cluster-wide alerting capabilities or permission to view alerts across multiple namespaces.
Choose the Developer perspective if you are working within a single project and want integrated monitoring in the Observe view.

.Administrator perspective

* In the Administrator perspective of the {product-title} web console, go to menu:Observe[Alerting]. The three main pages in the Alerting UI are *Alerts*, *Silences*, and *Alerting rules*, which appear as separate pages under the Alerting menu.

The Administrator perspective Alerting UI provides cluster-wide monitoring capabilities and allows you to query alerts and metrics across all namespaces.

.Developer perspective

* In the Developer perspective of the {product-title} web console, click *Observe* to access the Observe view. The Alerting UI is available through three tabs within the Observe view: *Alerting Rules*, *Alerts*, and *Silences*.

The Developer perspective Observe view provides namespace-scoped monitoring capabilities.
Alerts, alerting rules, and silences are automatically filtered to the selected project.
