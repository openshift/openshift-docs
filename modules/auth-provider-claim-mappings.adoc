= Claim mappings

[role="_abstract"]
Claim mappings allow you to customize how {product-title-short} translates claim attributes from identity provider tokens into {product-title-short}-issued tokens.

A claim is the data an identity provider includes about a user inside the token they issue.

Using claim mappings, you can specify if {product-title-short} should customize the claim attribute it receives from an IDP to another attribute in the {product-title-short}-issued token. If you do not use the claim mapping, {product-title-short} does not include the claim attribute in the {product-title-short}-issued token.

For example, you can map from `roles` in the user identity to `groups` in the {product-title-short}-issued token using claim mapping.

{product-title-short} uses different default claim mappings for every authentication provider.
