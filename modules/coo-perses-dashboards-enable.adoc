= Enable the monitoring plugin to use Perses dashboards

[role="_abstract"]
You can enable the monitoring plugin by creating a `UIPlugin` custom resource that enables Perses integration in the {product-title} web console.

[NOTE]
====
The Perses Operator and its CRDs are installed automatically when you install the Cluster Observability Operator (COO). The Perses Operator runs continuously after COO installation. The monitoring plugin only installs the UI and creates a Perses server instance.
====

.Prerequisites

* You have access to an {product-title} cluster version 4.15 or later.
* You have installed the Cluster Observability Operator version 1.5 or later.
* You have cluster administrator permissions.

.Procedure

. In the {product-title} web console, navigate to *Operators* -> *Installed Operators*.

. Select the *Cluster Observability Operator*.

. Click the *UIPlugin* tab.

. Click *Create UIPlugin*.

. Apply the following `UIPlugin` configuration:
+
[source,yaml]
----
apiVersion: observability.openshift.io/v1alpha1
kind: UIPlugin
metadata:
  name: monitoring
spec:
  type: Monitoring
  monitoring:
    perses:
      enabled: true
----

. Click *Create*.
+
Creating the monitoring UI plugin triggers a reload of the {ocp-product-title} web console.

.Verification

* In the {product-title} web console, navigate to *Observe* -> *Dashboards (Perses)*. The Perses dashboards interface appears, confirming successful installation.

* Verify that the Perses Operator is running by entering the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-observability-operator
----
+
The operator is deployed automatically with the COO and should already be active. You will see an output similar to the following example:
+
[source,terminal]
----
NAME                                     READY   STATUS    RESTARTS   AGE
perses-operator-xxx                      1/1     Running   0          5m
----

* Verify that the Perses CRDs are installed by entering the following command:
+
[source,terminal]
----
$ oc get crds | grep perses
----
+
You will see an output similar to the following example:
+
[source,terminal]
----
persesdashboards.perses.dev
persesdatasources.perses.dev
persesglobaldatasources.perses.dev
----

[NOTE]
====
After enabling the plugin, it might take a few minutes for the *Dashboards (Perses)* menu to appear in the web console.
====
