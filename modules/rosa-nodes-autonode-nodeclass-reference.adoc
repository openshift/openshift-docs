= Node class configuration reference

[role="_abstract"]
Reference documentation for OpenShiftEC2NodeClass custom resource fields and configuration options.

Node classes define the Amazon Elastic Compute Cloud (Amazon EC2) configuration for instances that {autonode} provisions. In {product-title}, you configure node classes using the `OpenShiftEC2NodeClass` custom resource, which automatically manages the corresponding `EC2NodeClass` resource.

[IMPORTANT]
====
Always create and modify `OpenShiftEC2NodeClass` resources, not `EC2NodeClass` resources directly. The {autonode} operator manages `EC2NodeClass` resources and overwrites manual changes.
====

[id="rosa-nodes-autonode-nodeclass-structure_{context}"]
== Node class structure

The OpenShiftEC2NodeClass custom resource has the following structure:

[source,yaml]
----
apiVersion: platform.openshift.io/v1alpha1
kind: OpenShiftEC2NodeClass
metadata:
  name: <nodeclass-name>
spec:
  amiSelectorTerms: <ami-selection>
  subnetSelectorTerms: <subnet-selection>
  securityGroupSelectorTerms: <security-group-selection>
  instanceProfile: <instance-profile-name>
  userData: <custom-user-data>
  tags: <resource-tags>
  blockDeviceMappings: <storage-configuration>
  metadataOptions: <instance-metadata-options>
----

[id="rosa-nodes-autonode-nodeclass-ami_{context}"]
== Amazon Machine Image selection

`spec.amiSelectorTerms`::
Defines which Amazon Machine Images (AMIs) {autonode} can use when provisioning nodes. You can select AMIs by ID, name, tags, or by using aliases. The default configuration uses the cluster's AMI.
+
.Select AMI by ID
[source,yaml]
----
spec:
  amiSelectorTerms:
    - id: ami-0abcdef1234567890
----
+
.Select AMI by name pattern
[source,yaml]
----
spec:
  amiSelectorTerms:
    - name: "rhcos-4.19*"
----
+
.Select AMI by tags
[source,yaml]
----
spec:
  amiSelectorTerms:
    - tags:
        environment: production
        os: rhcos
----
+
You can specify multiple selector terms. {autonode} uses the first AMI that matches any selector term.
+
[WARNING]
====
Custom AMIs must be compatible with {product-title}. Using an incompatible AMI prevents nodes from joining the cluster. Only use custom AMIs based on Red Hat Enterprise Linux CoreOS (RHCOS) or that you have verified work with {product-title}.
====

[id="rosa-nodes-autonode-nodeclass-subnet_{context}"]
== Subnet selection

`spec.subnetSelectorTerms`::
Defines which Amazon Web Services (AWS) subnets {autonode} uses when provisioning nodes. By default, {autonode} uses subnets tagged with `karpenter.sh/discovery=<cluster-id>`. You can override this to use different subnets for different node classes.
+
.Select subnets by discovery tag (default)
[source,yaml]
----
spec:
  subnetSelectorTerms:
    - tags:
        karpenter.sh/discovery: "${CLUSTER_ID}"
----
+
.Select subnets by custom tags
[source,yaml]
----
spec:
  subnetSelectorTerms:
    - tags:
        environment: production
        tier: private
----
+
.Select specific subnets by ID
[source,yaml]
----
spec:
  subnetSelectorTerms:
    - id: subnet-0abcdef1234567890
    - id: subnet-0fedcba0987654321
----
+
{autonode} provisions nodes in subnets that match any selector term. Use multiple subnet IDs to provision nodes across multiple availability zones.

[id="rosa-nodes-autonode-nodeclass-security-group_{context}"]
== Security group selection

`spec.securityGroupSelectorTerms`::
Defines which AWS security groups {autonode} attaches to provisioned nodes. By default, {autonode} uses security groups tagged with `karpenter.sh/discovery=<cluster-id>`. You can override this to use different security groups for different node classes.
+
.Select security groups by discovery tag (default)
[source,yaml]
----
spec:
  securityGroupSelectorTerms:
    - tags:
        karpenter.sh/discovery: "${CLUSTER_ID}"
----
+
.Select security groups by custom tags
[source,yaml]
----
spec:
  securityGroupSelectorTerms:
    - tags:
        environment: production
        purpose: worker-nodes
----
+
.Select specific security groups by ID
[source,yaml]
----
spec:
  securityGroupSelectorTerms:
    - id: sg-0abcdef1234567890
    - id: sg-0fedcba0987654321
----
+
{autonode} attaches all security groups that match any selector term. This allows nodes to have multiple security groups for different network access requirements.

[id="rosa-nodes-autonode-nodeclass-instance-profile_{context}"]
== Instance profile

`spec.instanceProfile`::
Specifies the AWS Identity and Access Management (IAM) instance profile to attach to provisioned nodes. The instance profile grants the node permissions to AWS services. By default, {autonode} uses the worker node instance profile from your cluster.
+
.Example
[source,yaml]
----
spec:
  instanceProfile: custom-worker-instance-profile
----
+
[WARNING]
====
Custom instance profiles must include the minimum permissions required for {product-title} worker nodes. Missing permissions prevent nodes from joining the cluster or cause cluster components to fail.
====

[id="rosa-nodes-autonode-nodeclass-userdata_{context}"]
== User data

`spec.userData`::
Custom user data script that runs when nodes launch. Use user data to install additional software, configure system settings, or perform custom initialization. The user data is passed to the Amazon EC2 instance as cloud-init configuration.
+
.Example
[source,yaml]
----
spec:
  userData: |
    #!/bin/bash
    echo "Running custom initialization"
    yum install -y <package-name>
----
+
[IMPORTANT]
====
User data runs before the node joins the cluster. Errors in user data scripts can prevent nodes from becoming ready. Test user data thoroughly before deploying to production.
====

[id="rosa-nodes-autonode-nodeclass-tags_{context}"]
== Resource tags

`spec.tags`::
Key-value pairs to apply as AWS resource tags to Amazon EC2 instances and volumes provisioned by this node class. Use tags for cost allocation, resource tracking, or compliance requirements.
+
.Example
[source,yaml]
----
spec:
  tags:
    cost-center: "engineering"
    environment: "production"
    managed-by: "autonode"
----
+
Tags defined in the node class are merged with cluster-level tags. If a key exists in both, the node class tag value takes precedence.

[id="rosa-nodes-autonode-nodeclass-storage_{context}"]
== Block device mappings

`spec.blockDeviceMappings`::
Defines the storage volumes to attach to provisioned nodes. Use block device mappings to customize the root volume size, type, or encryption, or to add additional data volumes.
+
.Customize root volume
[source,yaml]
----
spec:
  blockDeviceMappings:
    - deviceName: /dev/xvda
      ebs:
        volumeSize: 100Gi
        volumeType: gp3
        encrypted: true
        deleteOnTermination: true
----
+
.Add data volume
[source,yaml]
----
spec:
  blockDeviceMappings:
    - deviceName: /dev/xvda
      ebs:
        volumeSize: 50Gi
        volumeType: gp3
    - deviceName: /dev/xvdb
      ebs:
        volumeSize: 500Gi
        volumeType: gp3
        encrypted: true
----
+
Available volume types: `gp3` (general-purpose SSD), `gp2` (general-purpose SSD previous generation), `io1` and `io2` (provisioned IOPS SSD), `st1` (throughput-optimized HDD), `sc1` (cold HDD).

[id="rosa-nodes-autonode-nodeclass-metadata_{context}"]
== Instance metadata options

`spec.metadataOptions`::
Configures the Amazon EC2 instance metadata service (IMDS) for provisioned nodes. Use metadata options to control how applications access instance metadata.
+
.Example
[source,yaml]
----
spec:
  metadataOptions:
    httpEndpoint: enabled
    httpProtocolIPv6: disabled
    httpPutResponseHopLimit: 2
    httpTokens: required
----
+
* `httpEndpoint`: Enable or disable the instance metadata service. Valid values: `enabled`, `disabled`. Default: `enabled`.
* `httpProtocolIPv6`: Enable or disable IPv6 for the metadata service. Valid values: `enabled`, `disabled`. Default: `disabled`.
* `httpPutResponseHopLimit`: The maximum number of hops for instance metadata requests. Valid range: 1-64. Default: 1.
* `httpTokens`: Require session tokens for metadata requests (IMDSv2). Valid values: `required`, `optional`. Default: `required`.
+
[NOTE]
====
Using `httpTokens: required` enforces IMDSv2, which provides enhanced security for instance metadata access. This is the recommended setting.
====

[id="rosa-nodes-autonode-nodeclass-status_{context}"]
== Node class status

After creating or modifying an OpenShiftEC2NodeClass, verify that it is ready:

[source,terminal]
----
$ oc get openshiftec2nodeclass <name>
----

.Example output
[source,terminal]
----
NAME      READY   AGE
custom    True    5m
----

If the `READY` status is `False`, describe the resource to view error messages:

[source,terminal]
----
$ oc describe openshiftec2nodeclass <name>
----

Common errors include:
* AMI not found: Verify that the AMI ID or selector terms are correct and that the AMI exists in your AWS account and region
* Subnet not found: Verify that subnet selectors match existing subnets and that the subnets are tagged correctly
* Security group not found: Verify that security group selectors match existing security groups
* Instance profile not found: Verify that the IAM instance profile exists and has the required permissions

[id="rosa-nodes-autonode-nodeclass-example_{context}"]
== Complete example

.Custom node class with multiple configuration options
[source,yaml]
----
apiVersion: platform.openshift.io/v1alpha1
kind: OpenShiftEC2NodeClass
metadata:
  name: custom-workers
spec:
  amiSelectorTerms:
    - tags:
        environment: production
        os: rhcos
  subnetSelectorTerms:
    - tags:
        karpenter.sh/discovery: "${CLUSTER_ID}"
        tier: private
  securityGroupSelectorTerms:
    - tags:
        karpenter.sh/discovery: "${CLUSTER_ID}"
  tags:
    cost-center: "engineering"
    environment: "production"
    managed-by: "autonode"
  blockDeviceMappings:
    - deviceName: /dev/xvda
      ebs:
        volumeSize: 100Gi
        volumeType: gp3
        encrypted: true
        deleteOnTermination: true
  metadataOptions:
    httpEndpoint: enabled
    httpTokens: required
    httpPutResponseHopLimit: 2
----
