= Creating a kickstart for a {microshift-short} RHEL for Edge installation

Use the Kickstart provided with the {microshift-short} RPM to provision an RHEL for Edge-based virtual machine.

.Prerequisites

* You set up the {microshift-short} Kickstart file.
* You set up the [Activation Keys](https://console.redhat.com/insights/connector/activation-keys) and organization ID to activate your Red Hat subscription.
//what else?

[IMPORTANT]
====
The subscription must include access to the `rhocp-4.x-for-rhel-9-$(uname -m)-rpms` and `fast-datapath-for-rhel-9-$(uname -m)-rpms` RPM repositories.
====

.Procedure

. Add the following variables to create an OSTree kickstart file:

* The `OSTREE_SERVER_URL` variable contains an OSTree server URL that is passed to the `ostreesetup` kickstart command.
* The `OSTREE_COMMIT_REF` variable contains an OSTree commit reference that is installed from the server.
* The `AUTH_CONFIG` contents are copied to `/etc/ostree/auth.json` at the pre-install stage to authenticate access to the `OSTREE_SERVER_URL` server. If no server authentication is required, skip this setting.
//Q: Can a user manually set the z-stream version?

.Example commands setting the variables.
+
[source,terminal]
----
$ export OSTREE_SERVER_URL="<http://my_ostree_server_url>"
----
+
[source,terminal]
----
$ export OSTREE_COMMIT_REF="myostree_commit_reference"
----
+
[source,terminal]
----
$ export AUTH_CONFIG="$(cat ~/.ostree-auth.json)"
----

. Run the following command to create the `kickstart.ks` file to be used during the virtual machine installation:
+
[source,terminal]
----
envsubst < \
    /usr/share/microshift/kickstart/kickstart-ostree.ks.template > \
    "${HOME}/kickstart.ks"
----
