= Creating templates with the {sco}

Use the {sco} to create user-defined templates.
This example shows how you can create templates for hosted control planes.

.Prerequisites
// Any?

.Procedure

. Create a YAML file that contains the cluster-level templates for hosted control planes:
+
--
include::snippets/siteconfig-hcp-cluster-level.adoc[]
--

. Create a YAML file that contains the node-level templates for hosted control planes:
+
--
include::snippets/siteconfig-hcp-node-level.adoc[]
--

. Reference your templates in the `kustomization.yaml` file:
+
[source,yaml]
----
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - hcp-cluster-templates-v1.yaml
  - hcp-node-templates-v1.yaml
[...]
----

. Reference your node- and cluster-level templates in the SiteConfig CR:
+
[source,yaml]
----
spec:
  [...]
  nodes:
    templateRefs: <1>
      - name: hcp-node-templates-v1.yaml
        namespace: siteconfig-system
    [...]
  templateRefs: <2>
    - name: hcp-cluster-templates-v1.yaml
      namespace: siteconfig-system
----
<1> Reference node-level templates under the `spec.nodes.templateRefs` field.
<2> Reference cluster-level templates under the `spec.templateRefs` field.

. Create the resources for dry run, validation, and render.
