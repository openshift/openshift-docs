= Configure build inputs and outputs

[role="_abstract"]
Configure build inputs and outputs to control where built images are stored and how build parameters are applied.
Output configuration specifies the image registry destination and authentication credentials, while parameter values customize build strategy behavior for specific requirements.
Build references provide an alternative to embedding full build specifications directly in build runs.

Output configuration determines where the build controller pushes the completed container image.
You specify the target image registry URL, authentication secrets for private registries, and optional annotations or labels that are applied to the image metadata.
The build controller validates the output configuration before starting the build to ensure authentication and registry access are properly configured.

Parameter values customize the build strategy for specific use cases.
Each build strategy defines parameters that control its behavior, such as builder image versions, additional build arguments, or platform-specific settings.
You can define parameter values in the `Build` CR as defaults, then override them in individual build runs for flexibility.

Build references allow you to reference an existing `Build` CR from a `BuildRun` instead of duplicating the full specification.
This approach maintains a single source of truth for build configuration while enabling multiple build runs from the same definition.
Using build references simplifies configuration management and ensures consistency across builds.
