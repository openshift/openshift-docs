= Replacing an etcd member manually

Without the Machine API, the machine for the node to be replaced must be manually deleted and the replacement must be manually added to the cluster.

.Prerequisites

* Install the {oc-first}.
* Log in to `oc` as a user with `cluster-admin` permission.

.Procedure

. Enter a debug session on a different healthy master node:
+
[source,terminal]
----
$ oc debug node/<healthy-node>
----

. Inside the debug shell, list the members to find the ID of the node you are replacing:
+
[source,terminal]
----
$ etcdctl member list -w table
----

. Remove the identified node:
+
[source,terminal]
----
$ etcdctl member remove <member-id>
----

. Delete the associated `Node` object:
+
[source,terminal]
----
$ oc delete node <old-node-name>
----

. Manually build your new infrastructure node. Once it joins the cluster and is labeled with `node-role.kubernetes.io/master=""`, the etcd Operator will automatically scale the etcd cluster onto the new node.
