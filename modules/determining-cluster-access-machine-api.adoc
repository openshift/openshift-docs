= Determining how to replace a healthy etcd member

[role="_abstract"]
Use the {oc-first} to determine which replacement procedure to follow.

.Prerequisites

* Install the {oc-first}.
* Log in to `oc` as a user with the `cluster-admin` role.

.Procedure

. Check whether the cluster was installed by using the {ai-full} by running the following command:
+
[source,terminal]
----
$ oc get agentclusterinstall -A
----
+
* If the command returns one or more `AgentClusterInstall` resources, follow the procedure to replace a control plane node in a healthy cluster in the {ai-full} documentation in Additional resources.
+
* If the command returns no resources, continue with the following steps.

. Check whether the cluster has a control plane machine set by running the following command:
+
[source,terminal]
----
$ oc -n openshift-machine-api get controlplanemachineset
----
+
* If the command returns a `ControlPlaneMachineSet` resource, follow Replacing a healthy etcd member with a control plane machine set in this document.
+
* If the command returns no resources, continue to the next step.

. Check whether the cluster is installed on bare-metal infrastructure by running the following command:
+
[source,terminal]
----
$ oc get infrastructure cluster -o jsonpath='{.status.platform}{"\n"}'
----
+
* If the output is `BareMetal`, follow Replacing a healthy etcd member by scaling up and scaling down in this document.
+
* If the output is not `BareMetal`, follow Replacing a healthy etcd member with the Machine API in this document.
