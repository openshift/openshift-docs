= Observability metrics of the event-based pruner

The event-based pruner exposes detailed metrics through the `tekton-pruner-controller` controller `Service` definition on port `9090` in OpenTelemetry format for monitoring, troubleshooting, and capacity planning.

Following are categories of the metrics exposed:

* Resource processing
* Performance timing
* State tracking
* Resource age analysis
* Error monitoring

Most pruner metrics use labels to provide additional context. You can use these labels in PromQL{nbsp}queries or dashboards to filter and group the metrics.

[cols="1,3"options="header"]
|===
| Label | Description

| `namespace`
| The Kubernetes namespace of the `PipelineRun` or `TaskRun`.

| `resource_type`
| The Tekton resource type.

| `status`
| The outcome of processing a resource.

| `operation`
| The pruning method that deleted a resource.

| `reason`
| Specific cause for skipping or error outcomes.
|===

== Resource processing metrics

[cols="2,3,1,3,2",options="header"]
|===
| Category | Name | Type | Description | Labels
| Resource processing | `tektoncd_pruner_resources_processed_total` | Counter | Total resources processed | namespace, resource_type, status
| Resource processing | `tektoncd_pruner_resources_deleted_total` | Counter | Total resources deleted | namespace, resource_type, operation
| Resource processing | `tektoncd_pruner_resources_errors_total` | Counter | Total processing errors | namespace, resource_type, reason
| Resource processing | `tektoncd_pruner_resources_skipped_total` | Counter | Total resources skipped | namespace, resource_type, reason
|===

== Performance timing metrics

[cols="2,3,1,3,2,2",options="header"]
|===
| Category | Name | Type | Description | Labels | Bucket 
| Performance timing | `tektoncd_pruner_reconciliation_duration_seconds` | Histogram | Time spent in reconciliation | namespace, resource_type | 0.1 to 30 seconds
| Performance timing | `tektoncd_pruner_ttl_processing_duration_seconds` | Histogram | Time spent processing TTL | namespace, resource_type | 0.1 to 30 seconds
| Performance timing | `tektoncd_pruner_history_processing_duration_seconds` | Histogram | Time spent processing history limits | namespace, resource_type | 0.1 to 30 seconds
| Performance timing | `tektoncd_pruner_resource_deletion_duration_seconds` | Histogram | Time spent deleting resources | namespace, resource_type |
|===

== State tracking metrics

[cols="2,3,1,3,2",options="header"]
|===
| Category | Name | Type | Description | Labels
| State tracking | `tektoncd_pruner_resources_queued_total` | Counter | Total resources queued | namespace, resource_type
| State tracking | `tektoncd_pruner_current_resources_queued` | UpDownCounter | Current resources in queue | namespace, resource_type
| State tracking | `tektoncd_pruner_active_resources_count` | UpDownCounter | Current active resources | namespace, resource_type
|===

== Resource age analysis

[cols="2,3,1,3,2,2",options="header"]
|===
| Category | Name | Type | Description | Labels | Bucket
| Resource age | `tektoncd_pruner_resource_age_at_deletion_seconds` | Histogram | Age of resources when deleted | namespace, resource_type | 60 to 604,800 seconds
|===

| Error monitoring  | `tektoncd_pruner_resource_update_errors_total` | Counter | Resource update errors | namespace, resource_type
|===
