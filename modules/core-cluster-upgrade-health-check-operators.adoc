= Operator and application health checks

[role="_abstract"]
You can verify OLM operator health, pod status, and pod disruption budget configuration before starting a cluster update.

OLM operators::

[source,terminal]
----
$ oc get csv -A
----

Pass criteria: All `ClusterServiceVersions` show `Succeeded` phase.

Quick check for failed operators:
[source,terminal]
----
$ oc get csv -A | grep -v Succeeded
----

Pods not in Running or Completed state::

[source,terminal]
----
$ oc get pod -A | grep -E -i -v 'complete|running'
----

Pass criteria: The command produces no output. If any pods are listed, review their status before updating.

Pod disruption budgets::

[source,terminal]
----
$ oc get pdb -A
----

Pass criteria:

* Pod disruption budgets (PDBs) exist for all critical workloads
* `ALLOWED DISRUPTIONS` is greater than 0
* `MIN AVAILABLE` is not set to 100% of replicas
