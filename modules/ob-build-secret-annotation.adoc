= How builds stay in sync with secret changes

[role="_abstract"]
Add an optional annotation to a build secret to ensure the build controller reconciles on create, update, or delete events for the referenced build secret. 

The annotation is not required for builds to use the secret. It enables the controller to detect when the secret changes and re-run build configuration validations.

Use the annotation when you want the build controller to automatically detect changes to the secret and re-run build configuration validations. Without this annotation, the build controller ignores secret update events, and you must manually trigger validation by updating the build resource.

The following example shows the usage of an annotation with a secret:

[source,yaml]
----
apiVersion: v1
data:
  .dockerconfigjson: <pull_secret>
kind: Secret
metadata:
  annotations:
    build.shipwright.io/referenced.secret: "true"
  name: secret-docker
type: kubernetes.io/dockerconfigjson
----
where:

`<pull_secret>`:: Specifies the Base64-encoded pull secret used by the build.
`build.shipwright.io/referenced.secret`:: Specifies the value of the `build.shipwright.io/referenced.secret` annotation. Set to `"true"` to enable controller reconciliation for this secret.
