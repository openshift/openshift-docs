= Delete a provisioned cluster

[role="_abstract"]
You can decommission a cluster provisioned by the O-Cloud Manager by deleting its `ProvisioningRequest` resource. The O-Cloud Manager removes the cluster, releases the allocated hardware, and returns the bare-metal servers to the resource pool for re-use.

.Prerequisites

* The cluster was provisioned using a `ProvisioningRequest`.
* You have `cluster-admin` permissions on the hub cluster.

.Procedure

. Delete the `ProvisioningRequest` resource by running the following command:
+
[source,terminal]
----
$ oc delete provisioningrequests.clcm.openshift.io <uuid>
----
+
[NOTE]
====
Deleting a `ProvisioningRequest` triggers cluster deprovisioning and resource cleanup, which typically takes several minutes. Do not force-delete the resource.
====
+
The O-Cloud Manager initiates the cleanup flow:
+
.. The `provisioningPhase` transitions to `deleting`.
.. The `ClusterInstance` resource is deleted, triggering spoke cluster decommissioning.
.. The `NodeAllocationRequest` is removed, releasing the `BareMetalHost` allocation.
.. The bare-metal server is powered off and its provisioning state returns to `Available`.

. Monitor the deletion progress:
+
[source,terminal]
----
$ oc get provisioningrequests.clcm.openshift.io <uuid>
----
+
The resource is removed from the cluster after all cleanup operations complete.

.Verification

* Verify that the `BareMetalHost` returns to the `Available` provisioning state:
+
[source,terminal]
----
$ oc get bmh -A
----
+
Servers in the `Available` state are ready to be allocated by a new `ProvisioningRequest`.
