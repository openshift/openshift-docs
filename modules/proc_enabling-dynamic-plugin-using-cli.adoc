= Enabling a dynamic plugin with the CLI

[role="_abstract"]
You can enable a dynamic plugin by using the {oc-first} after a scripted installation, such as an Operator or Helm-based install. Add the `ConsolePlugin` name to `spec.plugins` in the console Operator configuration (`console.operator.openshift.io/cluster`) so the web console loads it.

.Prerequisites
* You logged in to the cluster as a user with `cluster-admin` privileges.
* You installed the dynamic plugin using a scripted installation, such as an Operator or Helm chart.
* The installation created a `ConsolePlugin` custom resource (CR).

.Procedure
. Confirm the name of the `ConsolePlugin` resource:
+
[source,terminal]
----
$ oc get consoleplugin
----
+
Optional: View details for a specific `ConsolePlugin` resource:
+
[source,terminal]
----
PLUGIN_NAME="<plugin_name>"
oc get consoleplugin "${PLUGIN_NAME}" -o yaml
----
+
The following example shows a `ConsolePlugin` YAML with the plugin listed in `spec.plugins`:
+
[source,yaml]
----
apiVersion: operator.openshift.io/v1
kind: Console
metadata:
  name: cluster
spec:
  plugins:
    - <plugin_name> 
    # ...
----
+
Replace `<plugin_name>` with the name of your plugin. To find available plugin names, run `oc get consoleplugin`.

. Enable the dynamic plugin by adding the `ConsolePlugin` name to the console Operator configuration. 
+
[NOTE]
====
Make sure the Operator finishes installing the dynamic plugin before you run the following patch command.
====
+
[source,terminal]
----
PLUGIN_PATCH=$(cat <<EOF
[
  {
    "op": "add",
    "path": "/spec/plugins/-",
    "value": "<plugin_name>"
  }
]
EOF
)
oc patch consoles.operator.openshift.io cluster --type=json -p "${PLUGIN_PATCH}"
----

.Verification
. Confirm that the console Operator configuration includes the `ConsolePlugin` name:
+
[source,terminal]
----
$ oc get console.operator.openshift.io cluster -o jsonpath='{.spec.plugins}{"\n"}'
----

. Refresh the {product-title} web console.
+
The console can take a few minutes to apply the updated configuration.
