= Configuring Global Access parameters on GCP
As a cluster administrator with an internal load balancer on GCP, you can specify the global access option. Clients in any region within the same VPC network as the load balancer can reach the workloads running on your cluster.

For more information, please reference the GCP documentation for link:https://cloud.google.com/kubernetes-engine/docs/how-to/internal-load-balancing#global_access[Global access].

.Prerequisites

* You deployed an {product-title} cluster on GCP infrastructure.
* You created an ingress controller with internal load balancing.
* You installed the `oc` command line interface.

.Procedure

. Configure the 'clientAccess' field by specifying the appropriate level for 'clientAccess', global and local.
+
.. Edit the Ingress Controller Operator:
+
[source,terminal]
----
$ oc edit ingresses.config/cluster -o yaml
----
+
.. Edit the YAML file:
+
.Sample `clientAccess` configuration to `global`
[source,yaml]
----
apiVersion: config.openshift.io/v1
kind: Ingress
metadata:
  name: cluster
spec:
  endpointPublishingStrategy:
    type: LoadBalancerService
    loadBalancer:
      scope: Internal
  domain: apps.<domain_url>
  clientaccess: global
----
