= Prerequisites for converting a connected cluster to a disconnected cluster

[role="_abstract"]
You must meet several prerequisites before converting a connected cluster to a disconnected cluster.

The following prerequisites must be met:

* The `oc` client is installed.

* A running cluster.

* An installed mirror registry, which is a container image registry that supports Docker v2-2 in the location that will host the {product-title} cluster, such as one of the following registries:
+
--
** Red{nbsp}Hat Quay

** JFrog Artifactory

** Sonatype Nexus Repository

** Harbor
--
+
If you have a subscription to Red{nbsp}Hat Quay, see the documentation on "Deploying Red{nbsp}Hat Quay for proof-of-concept purposes" or "Deploying Red{nbsp}Hat Quay by using the Quay Operator".

* The mirror repository must be configured to share images. For example, a {quay} repository requires organizations to share images. For more information, see the "Red{nbsp}Hat Quay documentation on organizations".

* Access to the internet to obtain the necessary container images.
