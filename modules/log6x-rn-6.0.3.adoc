= Logging 6.0.3

This release includes link:https://access.redhat.com/errata/RHBA-2024:10991[{logging-uc} {for} Bug Fix Release 6.0.3].

[id="logging-release-notes-6-0-3-enhancements_{context}"]
== New features and enhancements

* With this update, {loki-op} adds support for configuring workload identity federation on {gcp-first} using the Cluster Credential Operator in {OCP-short} 4.17 and newer. (link:https://issues.redhat.com/browse/LOG-6421[LOG-6421])

[id="logging-release-notes-6-0-3-bug-fixes_{context}"]
== Bug fixes

* Before this update, the collector used default settings to collect audit logs, which did not account for back pressure from output receivers. With this update, the audit log collection process has been improved to optimize file handle usage and log reading. (link:https://issues.redhat.com/browse/LOG-6034[LOG-6034])

* Before this update, any namespace with `openshift` or `kube` in its name was classified as an infrastructure namespace. With this update, only the following namespaces are classified as infrastructure namespaces: `default`, `kube`, `openshift`, and namespaces that start with `openshift-` or `kube-`. (link:https://issues.redhat.com/browse/LOG-6204[LOG-6204])

* Before this update, an input receiver's service was repeatedly created and deleted, leading to issues with mounting TLS secrets. With this update, the service is created once and only deleted if it is not defined in the specification. (link:https://issues.redhat.com/browse/LOG-6343[LOG-6343])

* Before this update, pipeline validation could enter an infinite loop if one name was a substring of another. With this update, stricter name equality checks prevent the infinite loop. (link:https://issues.redhat.com/browse/LOG-6352[LOG-6352])

* Before this update, the collector alerting rules included summary and message fields. With this update, the collector alerting rules now include summary and description fields. (link:https://issues.redhat.com/browse/LOG-6406[LOG-6406])

* Before this update, setting up custom audit inputs in the `ClusterLogForwarder` with a `LokiStack` output caused errors due to nil pointer dereference. With this update, the operator code includes nil checks, preventing these errors. (link:https://issues.redhat.com/browse/LOG-6441[LOG-6441])

* Before this update, the condition `ValidLokistackOTLPOutputs` was shown on the `ClusterLogForwarder` even for non-`LokiStack` outputs. This update removes the condition and adds the correct validation messaging to the existing output conditions. (link:https://issues.redhat.com/browse/LOG-6476[LOG-6476])

* Before this update, the collector did not correctly mount the `/var/log/oauth-server/` path, which prevented the collection of those audit logs. With this update, the volume mount is added, and the audit logs are collected as expected. (link:https://issues.redhat.com/browse/LOG-6486[LOG-6486])
