= Replace community tasks

[role="_abstract"]
Some community tasks that were previously available as `ClusterTask` resources are no longer installed automatically by {pipelines-title}. You can install these tasks manually from the Tekton Catalog or other sources.

.Prerequisites

* You have the necessary permissions to create `Task` resources.

.Procedure

. Identify community tasks that are no longer available:
+
Examples of removed community tasks include:
+
--
* `helm-upgrade-from-repo`
* `jib-maven`
* `pull-request`
* `pytest`
--

. Obtain the task definition from the Tekton Catalog or another source:
+
[source,terminal]
----
$ curl -LO https://raw.githubusercontent.com/tektoncd/catalog/main/task/helm-upgrade-from-repo/0.3/helm-upgrade-from-repo.yaml
----

. Review the task definition to ensure it meets your requirements:
+
[source,terminal]
----
$ cat helm-upgrade-from-repo.yaml
----

. Choose a namespace for the task installation:
+
--
* *Same namespace as the pipeline*: Install the task in the same namespace as your pipeline if it is only used by that pipeline.
* *Shared namespace*: Install the task in a shared namespace if multiple pipelines in different projects use it.
--

. Apply the task to the cluster by running the following command:
+
[source,terminal]
----
$ oc apply -f helm-upgrade-from-repo.yaml -n my-namespace
----

. Update pipeline references based on where you installed the task.
+
If the task is in the *same namespace* as the pipeline:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: helm-deploy-pipeline
  namespace: my-namespace
spec:
  tasks:
  - name: upgrade-helm-chart
    taskRef:
      name: helm-upgrade-from-repo
----
For tasks in the same namespace, you can reference them by name without using the cluster resolver.
+
If the task is in a *different namespace*:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: helm-deploy-pipeline
  namespace: my-project
spec:
  tasks:
  - name: upgrade-helm-chart
    taskRef:
      resolver: cluster
      params:
      - name: kind
        value: task
      - name: name
        value: helm-upgrade-from-repo
      - name: namespace
        value: shared-tasks
----
For tasks in a different namespace, use the cluster resolver.
+
[NOTE]
====
You can also use the cluster resolver for tasks in the same namespace for consistency across all task references.
====

.Verification

. Verify that the task is available by running the following command:
+
[source,terminal]
----
$ oc get task helm-upgrade-from-repo -n my-namespace
----

. Test the task with a pipeline run or task run to ensure it works as expected.
