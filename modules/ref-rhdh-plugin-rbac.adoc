= {rhdh} plugin RBAC policy reference

[role="_abstract"]
The {prodname} {rhdh} plugin uses role-based access control (RBAC) policies to manage user permissions for API products and API keys.

[id="rhdh-plugin-default-roles_{context}"]
== Default roles

The {prodname} Backstage plugin defines three default roles with different permission levels.

api-consumer::
Users with this role can read API products and manage their own API keys. This role is suitable for developers who consume APIs.
+
* Read API products
* Create API keys
* Read own API keys
* Delete own API keys

api-owner::
Users with this role can manage API products and view API keys. This role is suitable for API product managers.
+
* Read API products
* Create API products
* Update API products
* Delete API products
* Read API keys

api-admin::
Users with this role have full administrative access to API products and API keys. This role is suitable for platform administrators.
+
* Read API products
* Create API products
* Update API products
* Delete API products
* Read API keys
* Delete API keys

[id="rhdh-plugin-rbac-policy-format_{context}"]
== RBAC policy format

The RBAC policy uses the CSV format with the following structure:

[source,csv]
----
p, role:default/<role_name>, <resource>.entity.<permission>, <action>, <effect>
g, user:default/<username>, role:default/<role_name>
----
+
* `<role_name>`: The name of the role.
* `<resource>`: The resource type, such as `APIProduct` or `APIKey`.
* `<permission>`: The permission level, `read`, `create`, `update`, or `delete`.
* `<action>`: The action type, `read`, `create`, `update`, or `delete`.
* `<effect>`: Either `allow` or `deny`.
* `<username>`: The user to assign to the role.
