= Istio configuration for SPIRE integration

[role="_abstract"]
Configuration reference for integrating Istio with SPIRE for workload identity. These configuration options enable Istio to use SPIRE as the certificate authority for service mesh mTLS.

== Sidecar injection templates

The `sidecarInjectorWebhook.templates` configuration defines how the SPIRE socket is mounted into sidecar containers.

The followinig is an example of a SPIRE template.

[source,yaml]
----
spec:
  initContainers:
  - name: istio-proxy
    volumeMounts:
    - name: workload-socket
      mountPath: /run/secrets/workload-spiffe-uds
      readOnly: true
  volumes:
    - name: workload-socket
      csi:
        driver: "csi.spiffe.io"
        readOnly: true
----

The following is an example of a `spireGateway` template.

[source,yaml]
----
spec:
  containers:
  - name: istio-proxy
    volumeMounts:
    - name: workload-socket
      mountPath: /run/secrets/workload-spiffe-uds
      readOnly: true
  volumes:
    - name: workload-socket
      csi:
        driver: "csi.spiffe.io"
        readOnly: true
----

== Mesh configuration

.Key Istio mesh configuration for SPIRE
[cols="1,1,2",options="header"]
|===
|Field |Type |Description

|`meshConfig.trustDomain`
|string
|SPIFFE trust domain matching ZTWIM configuration

|`meshConfig.defaultConfig.proxyMetadata.WORKLOAD_IDENTITY_SOCKET_FILE`
|string
|Socket filename for SPIRE Workload API (default: `spire-agent.sock`)

|`pilot.jwksResolverExtraRootCA`
|string
|Additional root CA for JWT validation

|`pilot.env.PILOT_JWT_ENABLE_REMOTE_JWKS`
|string
|Enable remote JWKS fetching for JWT validation
|===

== Pod annotations

.Required annotations for SPIRE integration
[cols="1,2",options="header"]
|===
|Annotation |Description

|`inject.istio.io/templates: "sidecar,spire"`
|Inject both standard sidecar and SPIRE volume mounts

|`inject.istio.io/templates: "gateway,spireGateway"`
|For gateway pods: inject gateway and SPIRE gateway templates

|`spiffe.io/audience`
|SPIFFE audience for JWT-SVID tokens
|===
