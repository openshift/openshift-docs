= Overriding disruption budgets

A Pod Disruption Budget (PDB) is a standard feature of Kubernetes APIs that helps limit the disruption to an application when its pods need to be rescheduled for maintenance reasons.

.Procedure

You can override the default PDB for a specific resource by modifying the `minAvailable` configuration value in the `KnativeEventing CR`.

[NOTE]
====
If you disable high-availability, for example, by changing the `high-availability.replicas` value to 1, make sure you also update the corresponding podDisruptionBudget `minAvailable` value to 0. Otherwise, the podDisruptionBudget will prevent automatic cluster or operator upgrades.
====

.An example of minAvailable set to 70% for PDB named eventing-webhook
[source,yaml]
----
apiVersion: operator.knative.dev/v1beta1
kind: KnativeEventing
metadata:
 name: knative-eventing
 namespace: knative-eventing
spec:
 podDisruptionBudgets:
 - name: eventing-webhook
   minAvailable: 70%
----
