= Operator annotations used during installation

[role="_abstract"]
{product-title-short} uses annotations attached to the custom resource (CR) so that for installations performed by using the Operator, the default value for certain configuration items are decided at runtime by the Operator based on the current environment.

For some configuration items, using a static default for installation is not ideal. {product-title-short} uses annotations to implement runtime defaults. This means that the default value for a certain configuration setting is decided by the Operator at runtime, based on the current environment.

For example, the default installation behavior is to enable Scanner V4 for new installations. However, the default installation behavior when updating is to disable Scanner V4. Beginning with {product-title-short} version 4.8, the Operator can add annotations that are attached to the CR that you apply to clusters. These annotations allow a default decision that was previously made to persist and be reapplied in the future, even if the default behavior for a fresh installation has changed.

Therefore, in the example of Scanner V4, if Scanner V4 was enabled in version 4.7, the default behavior when installing version 4.8 by using the Operator is that it will be enabled. If Scanner V4 was not enabled in version 4.7, it will not be enabled by default when installing version 4.8 by using the Operator.

Annotations are in the form `feature-defaults.platform.stackrox.io/<identifier>`. For example, `feature-defaults.platform.stackrox.io/scannerV4` is the annotation that controls the default for Scanner V4 installation.

[IMPORTANT]
====
Do not change or delete these annotations.
====
