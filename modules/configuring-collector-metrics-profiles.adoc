= Configuring metrics collection profiles to reduce cardinality

[role="_abstract"]
You can configure the cluster monitoring operator to use metrics collection profiles. These profiles control how many metrics Prometheus collects from the log collector. The minimal profile reduces metrics cardinality by collecting only the metrics needed for alerts, dashboards, and telemetry.

Metrics collection profiles provide a supported method to reduce Prometheus resource consumption without manually editing ServiceMonitor resources. The {clo} automatically reverts manual edits to ServiceMonitor resources.

.Prerequisites

* You have access to the cluster as a user with the `cluster-admin` cluster role.
* You have installed the {oc-first}.

.Procedure

. Edit the cluster monitoring configuration:
+
[source,terminal]
----
$ oc edit configmap cluster-monitoring-config -n openshift-monitoring
----

. Add or update the `collectionProfile` field to use the `minimal` profile:
+
[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: cluster-monitoring-config
  namespace: openshift-monitoring
data:
  config.yaml: |
    prometheusK8s:
      collectionProfile: minimal
----
+
[NOTE]
====
Setting the collection profile to `minimal` affects all platform services that expose metrics, not only the log collector. Any service that explicitly provides a minimal metrics set will have reduced metrics collection.
====
+
Available profiles:
+
--
`full`:: Collects all metrics exposed by platform components. This is the default profile.

`minimal`:: Collects only metrics required for platform alerts, console dashboards, telemetry, and Horizontal and Vertical Pod Autoscaling. This profile significantly reduces Prometheus resource usage.
--

. Save the ConfigMap.

. The cluster monitoring operator automatically applies the profile change. Wait approximately 2-3 minutes for the monitoring components to reconcile.

.Verification

. Verify that the minimal ServiceMonitor resources were created for the log collector:
+
[source,terminal]
----
$ oc get servicemonitor -n openshift-logging -l monitoring.openshift.io/collection-profile=minimal
----
+
.Example output
[source,terminal]
----
NAME                  AGE
minimal-instance      5m
minimal-lfme          5m
----
+
The ServiceMonitor names follow the pattern `minimal-<forwarder-name>` where `<forwarder-name>` is typically `instance` for the default ClusterLogForwarder.

. Query Prometheus to confirm that only minimal metrics are being collected:
+
[source,terminal]
----
$ oc exec -n openshift-monitoring prometheus-k8s-0 -c prometheus -- \
  /bin/sh -c "promtool tsdb analyze /prometheus" | \
  grep vector_component_received_events_count_bucket
----
+
With the minimal profile, this metric should not appear in the output, as it is excluded from the minimal metric set.

. Monitor Prometheus resource usage to verify the reduction:
+
[source,terminal]
----
$ oc adm top pod -n openshift-monitoring prometheus-k8s-0
----
+
You should observe reduced CPU and memory consumption compared to the full profile.
