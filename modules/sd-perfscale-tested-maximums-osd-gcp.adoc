= Tested cluster maximums for {product-title} on {gcp-short}

[role="_abstract"]
The following tables apply to {product-title} clusters running on {gcp-short}, tested on {product-title} 4.20 without changing default cluster configuration beyond what the test describes.

Control plane and infrastructure instance types are not fixed for all cluster sizes; the following classification mirrors the worker-count tiers used for testing. Worker nodes use `n2-standard-4` unless noted.

:fn-perfscale-osd-gcp-1: footnote:perfscale-osd-gcp-fn1[Simple pause pods staying idle on the cluster, including default control plane pods; used `node-density` workload. Actual pod counts depend on workload requirements.]
:fn-perfscale-osd-gcp-2: footnote:perfscale-osd-gcp-fn2[Workload shape can lower pods per node; for example, many PostgreSQL pods on one node with high disk IOPS, or limits on persistent volume claims (PVCs) per node.]
:fn-perfscale-osd-gcp-3: footnote:perfscale-osd-gcp-fn3[Namespaces included a few idle pods plus default config maps and secrets each, including default operator namespaces.]
:fn-perfscale-osd-gcp-4: footnote:perfscale-osd-gcp-fn4[ClusterIP services only; other service types can have different limits. Used `cni-workload`.]
:fn-perfscale-osd-gcp-5: footnote:perfscale-osd-gcp-fn5[Test cluster used two routers with default configuration on infrastructure nodes; used `cluster-density` workload.]
:fn-perfscale-osd-gcp-6: footnote:perfscale-osd-gcp-fn6[ClusterIP services used a single endpoint each.]

.Test cluster sizing by worker count (OSD on {gcp-short})
[options="header",cols="2,2"]
|===
| Node | Instance type classification

| Control plane
| 1 to 25 workers: `n2-standard-8`. 26 to 100 workers: `n2-standard-16`. 101+ workers: `n2-standard-32`.

| Infrastructure
| 1 to 25 workers: `n2-highmem-4`. 26 to 100 workers: `n2-highmem-8`. 101+ workers: `n2-highmem-16`.

| Compute (worker)
| `n2-standard-4`
|===
