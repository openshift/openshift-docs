= Customizing RBAC policies

[role="_abstract"]
You can customize RBAC policies by modifying the `ConfigMap` custom resource (CR) in the {rhdh} namespace.

.Prerequisites

* You installed the {rhdh} plugin.
* You have cluster administrator privileges to configure role-based-access (RBAC) policies and service accounts.
* You installed the {oc-first}.

.Procedure

. Add a custom role by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc edit configmap rbac-policy -n ${RHDH_NS}
----

. Add the following lines to the `rbac-policy.csv` data:
+
[source,text]
----
p, role:default/api-viewer, apiproduct.entity.read, read, allow
p, role:default/api-viewer, apikey.entity.read, read, allow
g, user:default/developer1, role:default/api-viewer
----

. After modifying the ConfigMap, restart the {rhdh} pods for the changes to take effect by running the following command:
+
[source,terminal]
----
$ oc rollout restart deployment -developer-hub -n ${RHDH_NS}
----
