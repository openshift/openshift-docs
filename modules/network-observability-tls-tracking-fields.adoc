[id="tls-tracking-fields_{context}"]
= TLS tracking fields reference

[role="_abstract"]
The following tables describe the TLS metadata fields used to analyze encryption protocols, protocol versions, and cipher suite data in network flows.

[cols="1,2,2,2",options="header"]
|===
|Field |Description |Possible values |Availability

|*TLS Version*
|Negotiated TLS protocol version.
a|* `1.0`: Deprecated
* `1.1`: Deprecated
* `1.2`: Secure
* `1.3`: Current standard
|`ClientHello`, `ServerHello`

|*TLS Cipher Suite*
|Cryptographic algorithm suite negotiated between the client and server.
a|Examples:

* `TLS_AES_256_GCM_SHA384`
* `TLS_CHACHA20_POLY1305_SHA256`
* `n/a`: Not available in this message type
|`ServerHello` only

|*TLS Curve*
|Elliptic curve used for key exchange.
a|Examples:

* `X25519`: Recommended for TLS 1.3
* `secp256r1` (P-256)
* `n/a`: Not available for TLS 1.2
|`ServerHello` (TLS 1.3 only)

|*TLS Types*
|Type of TLS handshake message captured.
a|* `ClientHello`: Initial client request
* `ServerHello`: Server response
|All TLS flows
|===

[id="field-availablity-by-message-type_{context}"]
== Field availability by message type

Use the following table to identify which TLS fields are available in each handshake message type:

[cols="1,3,3",options="header"]
|===
|Message Type |Available Fields |Notes

|*`ClientHello`*
a|* TLS Version
* TLS Types
a|* Displays the TLS version requested by the client.
* `Cipher Suite` and `Curve` fields display as `n/a`.

|*`ServerHello`*
a|* TLS Version
* TLS Cipher Suite
* TLS Curve (TLS 1.3 only)
* TLS Types
a|* Displays the negotiated TLS version selected by the server.
* `Curve` field is only populated for TLS 1.3 connections.
|===

[id="common-cipher-suites_{context}"]
== Common cipher suites

The following table lists cipher suites and their security status:

[cols="2,1,3",options="header"]
|===
|Cipher Suite |Security level |Notes

|`TLS_AES_256_GCM_SHA384`
|Secure
|TLS 1.3 suite with 256-bit AES encryption.

|`TLS_CHACHA20_POLY1305_SHA256`
|Secure
|TLS 1.3 suite optimized for mobile devices.

|`TLS_RSA_WITH_AES_256_CBC_SHA`
|Weak
|Lacks forward secrecy.

|`TLS_RSA_WITH_3DES_EDE_CBC_SHA`
|Deprecated
|Uses 3DES; replace with more secure suite.
|===

[IMPORTANT]
====
Cipher suites without forward secrecy, such as those using `RSA` key exchange without `ECDHE` or `DHE`, do not protect past communications if the server private key is compromised.
====

[id="tls-version-security-guidance_{context}"]
== TLS version security guidance

Use the following table to determine which TLS versions are secure for your deployment:

[cols="1,1,3",options="header"]
|===
|TLS Version |Status |Recommendation

|*1.0*
|Deprecated
|Use TLS 1.2 or higher.

|*1.1*
|Deprecated
|Use TLS 1.2 or higher.

|*1.2*
|Supported
|Use with strong cipher suites and forward secrecy.

|*1.3*
|Current Standard
|Recommended for all new implementations.
|===
