= Transport Security Layer tracking fields reference

[role="_abstract"]
The following tables describe the Transport Layer Security (TLS) metadata fields used to analyze encryption protocols, protocol versions, and cipher suite data in network flows.

[cols="1,2,2,2",options="header"]
|===
|Field |Description |Possible values |Availability

|*TLS Version*
|Negotiated TLS protocol version.
a|* `1.0`: Deprecated
* `1.1`: Deprecated
* `1.2`: Secure
* `1.3`: Current standard
|`ClientHello`, `ServerHello`

|*TLS Cipher Suite*
|Cryptographic algorithm suite negotiated between the client and server.
a|Examples:

* `TLS_AES_256_GCM_SHA384`
* `TLS_CHACHA20_POLY1305_SHA256`
* `n/a`: Not available in this message type
|`ServerHello` only

|*TLS Group*
|Elliptic curve used for key exchange.
a|Examples:

* `X25519`: Recommended for TLS 1.3
* `secp256r1` (P-256)
* `n/a`: Not available for TLS 1.2
|`ServerHello` (TLS 1.3 only)

|*TLS Types*
|Type of TLS handshake message captured.
a|* `ClientHello`: Initial client request
* `ServerHello`: Server response
|All TLS flows
|===

[id="field-availability-by-message-type_{context}"]
== Field availability by message type

Use the following table to identify which TLS fields are available in each handshake message type:

[cols="1,3,3",options="header"]
|===
|Message Type |Available Fields |Notes

|*`ClientHello`*
a|* TLS Version
* TLS Types
a|* Displays the TLS version requested by the client.
* `Cipher Suite` and `Group` fields display as `n/a`.

|*`ServerHello`*
a|* TLS Version
* TLS Cipher Suite
* TLS Group (TLS 1.3 only)
* TLS Types
a|* Displays the negotiated TLS version selected by the server.
* `Group` field is only populated for TLS 1.3 connections.
|===
