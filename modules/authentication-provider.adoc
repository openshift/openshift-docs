= Authentication provider

[role="_abstract"]
{product-title-managed-short} uses Red{nbsp}Hat Single Sign-On (SSO) as the default identity provider, with authentication configured during Central instance creation through the {cloud-console}.

When you create a Central instance by using the {cloud-console}, authentication for the cluster administrator is configured as part of the process.

Customers must manage all access to the Central instance as part of their integrated solution.

For more information about the available authentication methods, see "Understanding authentication providers".

The default identity provider in {product-title-managed-short} is Red{nbsp}Hat Single Sign-On (SSO). Authorization rules are set up to give administrator access to the user who created the {product-title-managed-short} and to users who are marked as organization administrators in Red{nbsp}Hat SSO. The `admin` login is disabled for {product-title-managed-short} by default and can only be enabled temporarily by SREs. For more information about authentication by using Red{nbsp}Hat SSO, see "Default access to the ACS Console".
