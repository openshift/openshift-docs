= GNR-D boundary clock without holdover

[role="_abstract"]
You can use GNR-D hardware to configure a Precision Time Protocol (PTP) boundary clock without holdover.

Use a single `TimeReceiver` interface on a Network Adapter Card (NAC) port to synchronize multiple `TimeTransmitter` interfaces. You can configure `TimeTransmitter` interfaces on Network Adapter Card (NAC) ports, Carter Flats (E830) expansion NICs, or both.

This configuration does not maintain time when the upstream reference becomes unavailable. To ensure accurate synchronization, keep a continuous connection to a PTP Grand Master. This approach is useful for telco workloads that require deterministic timing and support for multiple synchronized interfaces.


With this configuration, you can perform the following actions:

* Synchronize multiple `TimeTransmitter` interfaces from a single `TimeReceiver` interface.
* Use both NAC and Carter Flats interfaces in the same configuration.
* Use SR-IOV-enabled interfaces for time distribution.
* Align timing across NICs by using DPLL synchronization.

Before you configure this feature, consider the following limitations:

* The system does not provide holdover functionality.
* Synchronization depends on continuous availability of the upstream PTP source.
