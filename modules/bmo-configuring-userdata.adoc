= Configuring userData

The cloud image for most operating systems provides the `cloud-init` service, which can read the custom scripts that you create in the `userData` format.

The `userData` field in the `BareMetalHost` custom resource allows you to customize the bare-metal host during provisioning by providing initial configuration data, such as user credentials or system settings. For most operating systems, this is achieved by using a `cloud-init` script encapsulated in a Kubernetes secret. 

[NOTE]
====
For link:https://docs.fedoraproject.org/en-US/fedora-coreos/sysconfig-network-configuration/#_via_ignition[Fedora CoreOS (FCOS)], use Ignition.
====

Create and apply the `userData` for operating systems that use `cloud-init`.

.Procedure

. Create a file named `user-data.yaml` with the following content:
+
[source,yaml]
----
users:
- name: <name>
  sudo: ["ALL=(ALL) NOPASSWD:ALL"]
  ssh_authorized_keys:
  - <ssh_key>
----
+
<name>::
    The user name.
<ssh_key>::
    The public SSH key to use when accessing this host as the `<name>` user.

. Create a secret from the `userData` file by running the following command:
+
[source,terminal]
----
$ oc create secret generic <name>-user-data \
  --from-file=userData=user-data.yaml -n bmaas
----

. Configure the `BareMetalHost` to use the `userData` file by running the following command:
+
[source,terminal]
----
$ oc patch baremetalhost <hostname> -n bmaas \
     --type merge -p '{"spec":{"userData":{"name":"<name>-user-data"}}}'
----
+
<hostname>::
  The name of the bare-metal host.
