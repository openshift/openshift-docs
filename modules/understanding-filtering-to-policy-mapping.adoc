= Filtering criteria and policies

[role="_abstract"]
Use the filtering criteria in the *Risk* view to define the policy scope and rules when you create a policy. 

When you create new security policies from the *Risk* view, based on the filtering criteria you use, not all criteria are directly applied to the new policy.

{product-title} ({product-title-short}) converts the *Cluster*, *Namespace*, and *Deployment* filters to corresponding policy resources. 

* When converting *Risk* view filters to policy resources, {product-title-short} distinguishes between name-based and label-based filtering.
* Cluster name filters convert to cluster ID scopes, while cluster label filters convert to cluster label scopes.
* Namespace name filters convert to namespace name scopes, and namespace label filters convert to namespace label scopes.

These conversions preserve the filtering semantics when creating policies from Risk view.

[NOTE]
====
Cluster label and namespace label scopes are only created when the secured cluster runs {product-title-short} 4.11 or later.
For secured clusters running earlier versions, {product-title-short} drops label-based filters during policy creation.
====

[id="local-page-filtering_{context}"]
== Local page filtering

Local page filtering on the *Risk* view combines the search terms by using the following methods:

* {product-title-short} combines the search terms within the same category with an `OR` operator.
For example, if the search query is `Cluster:A,B`, the filter matches deployments in `cluster A` or `cluster B`.
* {product-title-short} combines the search terms from different categories with an `AND` operator.
For example, if the search query is `Cluster:A+Namespace:Z`, the filter matches deployments in `cluster A` and in `namespace Z`.
* When you add multiple scopes to a policy, the policy matches violations from any of the scopes. For example, if you search for `(Cluster A OR Cluster B) AND (Namespace Z)` it results in two policy resources, `(Cluster=A AND Namespace=Z)` OR `(Cluster=B AND Namespace=Z)`.
* {product-title-short} drops or modifies filters that do not directly map to policy criteria and reports the dropped filters.
