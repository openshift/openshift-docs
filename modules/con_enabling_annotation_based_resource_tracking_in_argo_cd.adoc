= Enabling annotation based resource tracking in Argo CD

{gitops-title} Operator enhances multi-instance support by improving annotation-based resource tracking in Argo CD. Previously, multiple Argo CD instances struggled to differentiate resources with identical application names, relying only on legacy label-based tracking. With this enhancement, users can fully leverage annotation-based tracking by assigning each instance a unique `installationID`, preventing resource conflicts, eliminating infinite sync loops, and enabling safe parallel operation.

This guide covers setting up multiple Argo CD instances, configuring annotation-based tracking, associating them with namespaces, and verifying deployments by {OCP} Web Console.

.Pre-requisites

* You have logged in to the {OCP} cluster as an administrator.
* You have installed the {gitops-title} Operator on your {OCP} cluster.

[id="configuring-annotation-based-tracking-in-multiple-argo-cd-instances_{context}"]
== Configuring annotation-based tracking in multiple Argo CD instances

. Log in to the {OCP} web console. 

. In the *Administrator* perspective of the web console, click *Operators* -> *Installed Operators*.

. Create or select the project where you want to install the user-defined Argo CD instance from the *Project* list.

. Select *{gitops-title}* from the installed Operators list and click the *Argo CD* tab.

. Click *Create ArgoCD* to create two Argo CD instances using the following YAML snippet.
+
.Example an Argo CD instance with an annotation label
[source,yaml]
----
apiVersion: argoproj.io/v1beta1
kind: ArgoCD
metadata:
  name: argocd-instance-1
  namespace: argocd-test-1
  name: argocd-instance-2
  namespace: argocd-test-2
spec:
  installationID: "instance-1"
  resourceTrackingMethod: "annotation+label"
  installationID: "instance-2"
  resourceTrackingMethod: "annotation+label"
----
+
. Configure and label target namespaces to associate namespaces with their Argo CD instances.
.. Navigate to *Administration* -> *Namespaces*.
.. Create namespaces for application deployments, `app-ns-1` and `app-ns-2`.
.. To associate each namespace with their respective Argo CD instances, run the following command:
+
[source,terminal]
----
oc label namespace app-ns-1 argocd.argoproj.io/managed-by=argocd-test-1
oc label namespace app-ns-2 argocd.argoproj.io/managed-by=argocd-test-2
----
+
. Create two applications in Argo CD.
.. In the OpenShift Web Console, go to *Operators* → *Installed Operators* → *OpenShift GitOps Operator*.
.. Select *Argo CD* and navigate to the *Applications* tab.
.. Click *Create Application*.
.. Enter the following YAML snippet to create two applications in Argo CD.
+
.Example an Argo CD instance with an annotation label
[source,yaml]
----
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: nginx-app
  namespace: argocd-test-1
  name: nginx-app
  namespace: argocd-test-2
spec:
  project: default
  source:
    repoURL: https://github.com/redhat-developer/gitops-operator
    path: test/examples/nginx
    targetRevision: HEAD
  destination:
    server: https://kubernetes.default.svc
    namespace: app-ns-1
  syncPolicy:
    automated: {}
  project: default
  source:
    repoURL: https://github.com/redhat-developer/gitops-operator
    path: test/examples/nginx
    targetRevision: HEAD
  destination:
    server: https://kubernetes.default.svc
    namespace: app-ns-2
  syncPolicy:
    automated: {}
----
+
. Next, verify the deployment of each application in the {OCP} Web console.
.. Navigate to *Workloads* → *Pods*.
.. Ensure that the pods for Argo CD instances, `argocd-instance-1` and `argocd-instance-2` are running.
.. Check the application synchronization status in the *Argo CD Applications* tab.
.. Navigate to *Logs* under each Argo CD instance and confirm if you can view the status *Running in namespace-scoped mode*.
