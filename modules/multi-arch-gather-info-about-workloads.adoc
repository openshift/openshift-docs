= Gathering information about the workloads in your multi-architecture cluster

After installing the Multiarch Tuning Operator, you can verify the multi-architecture support for workloads in your cluster. When you create a pod, the Multiarch Tuning Operator adds pod labels that allow you to quickly identify and manage pods based on their architecture compatibility. These labels are automatically set on patched pods to provide insights into their architecture support.

When a pod is created, the Multiarch Tuning Operator adds the labels described in the following table:

.Pod labels added by the Multiarch Tuning Operator when a pod is created
[%autowidth,options="header"]
|====

|Label |Description

|`multiarch.openshift.io/multi-arch: ""` |Indicates that a pod supports multiple architectures.
|`multiarch.openshift.io/single-arch: ""` |Indicates that a pod supports only a single architecture.
|`multiarch.openshift.io/arm64: ""` |Indicates that a pod supports the `arm64` architecture.
|`multiarch.openshift.io/amd64: ""` |Indicates that a pod supports the `amd64` architecture.
|`multiarch.openshift.io/ppc64le: ""` |Indicates that a pod supports the `ppc64le` architecture.
|`multiarch.openshift.io/s390x: ""` |Indicates that a pod supports the `s390x` architecture.
|`multirach.openshift.io/node-affinity: set` |Indicates that the `node-affinity` requirement is already configured.
|`multirach.openshift.io/node-affinity: not-set` |Indicates that the `node-affinity` requirement is not configured.
|`multiarch.openshift.io/scheduling-gate: gated` |Indicates that the pod is gated.
|`multiarch.openshift.io/scheduling-gate: removed` |Indicates that the pod gate has been removed.
|`multiarch.openshift.io/inspection-error: ""` |Indicates that an error has occurred while inspecting the images.

|====
