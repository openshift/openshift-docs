= Deploying roxagent from an unvalidated pipeline

[role="_abstract"]
To use {product-title} ({product-title-short}) to scan virtual machines (VMs), you must run `roxagent`, a binary executable program. It runs inside VMs, scans for installed packages, and creates index reports. If you do not want to use Podman Quadlet to install `roxagent`, you can deploy it by using an unvalidated pipeline.

.Prerequisites

* You have installed DNF in the VM and you have completed at least one DNF transaction, such as `dnf update` or `dnf install`, so that the scan agent can pull all necessary metadata.
* You have registered the VM with Red Hat Subscription Manager and you have an active {op-system-base} subscription.

.Procedure

. Perform one of the following steps to get `roxagent`:
* Download `roxagent` from `\https://mirror.openshift.com/pub/rhacs/assets/<version>/bin/linux/roxagent`. For `<version>`, use the {product-title-short} version number that you are running, for example, 4.10.0.
* Build the `roxagent` binary manually:
.. Clone the stackrox repository by entering the following command:
+
[source,terminal]
----
$ git clone git@github.com:stackrox/stackrox.git
----
.. Change to the `stackrox` directory and enter the following command:
+
[source,terminal]
----
$ GOOS=linux GOARCH=amd64 go build ./compliance/virtualmachines/roxagent
----
. Transfer the binary to the VM by using a method such as `scp` or `cloud-init`.
. Run the binary inside the VM. To perform a one-time scan, enter the following command:
+
[source,terminal]
----
$ sudo ./roxagent 
----
To perform continuous monitoring, enter the following command:
+
[source,terminal]
----
$ sudo ./roxagent --daemon
----

.Runtime options

You can customize `roxagent` behavior by using the following command-line flags:

[cols="2,3,1", options="header"]
|===
|Flag |Description |Default

|`--index-interval`
|Scan frequency when running in daemon mode.
|`3h40m`

|`--host-path`
|Root filesystem path for scanning. Use this if the VM root is mounted at a non-standard location.
|`/`

|`--port`
|Virtual socket (vsock) port for communication with the relay.
|`818`

|`--max-initial-report-delay`
|Randomized startup delay before sending the first report, used to stagger reports across VMs.
|`40m`

|`--repo-cpe-url`
|URL or local file path for repository-to-CPE mapping data. Use this in air-gapped environments where internet access is unavailable.
|Red Hat-hosted URL

|`--verbose`
|Enable debug logging for troubleshooting.
|`false`
|===

[NOTE]
====
`roxagent` enforces single-instance execution by using a lock file at `/run/lock/roxagent/roxagent.lock`. If a previous instance did not shut down cleanly, you might need to remove this file manually before restarting.
