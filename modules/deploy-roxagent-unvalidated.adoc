= Deploying roxagent from an unvalidated pipeline

[role="_abstract"]
To use {product-title} ({product-title-short}) to scan virtual machines (VMs), you must run `roxagent`, a binary executable program. It runs inside VMs, scans for installed packages, and creates index reports. If you do not want to use Podman Quadlet to install `roxagent`, you can deploy it by using an unvalidated pipeline.

.Prerequisites

* You have installed DNF in the VM and you have completed at least one DNF transaction, such as `dnf update` or `dnf install`, so that the scan agent can pull all necessary metadata.
* You have registered the VM with Red Hat Subscription Manager and you have an active {op-system-base} subscription.

.Procedure

. Perform one of the following steps to get `roxagent`:
* Download `roxagent` from `\https://mirror.openshift.com/pub/rhacs/assets/<version>/bin/linux/roxagent`. For `<version>`, use the {product-title-short} version number that you are running, for example, 4.11.0.
* Build the `roxagent` binary manually:
.. Clone the stackrox repository by entering the following command:
+
[source,terminal]
----
$ git clone git@github.com:stackrox/stackrox.git
----
.. Change to the `stackrox` directory and enter the following command:
+
[source,terminal]
----
$ GOOS=linux GOARCH=amd64 go build ./compliance/virtualmachines/roxagent
----
. Transfer the binary to the VM by using a method such as `scp` or `cloud-init`.
. Run the binary inside the VM. To perform a one-time scan, enter the following command:
+
[source,terminal]
----
$ sudo ./roxagent 
----
To perform continuous monitoring, enter the following command:
+
[source,terminal]
----
$ sudo ./roxagent --daemon
----
