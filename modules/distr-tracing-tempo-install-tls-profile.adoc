= TLS Profile

The Tempo operator adheres to the TLS configuration defined in the OpenShift `APIServer` custom resource.
The TLS settings (min version and cipher suites) from the `APIServer` are used in all TLS communication in the operator
and operands.

The TLS cluster profile adherence is configured by these environment variables:

* `FEATURE_GATE=openshift.clusterTLSPolicy`- enables reading the TLS configuration from the `APIServer` custom resource. Enabled by default.
* `TLS_PROFILE` - explicitly


When `openshift.clusterTLSPolicy` is disabled the TLS profile can be explicitly set

* `TLS_PROFILE` - TLS profile environment variable. Possible values are: `Old`, `Intermediate` and `Modern`. See https://wiki.mozilla.org/Security/Server_Side_TLS for more information.
