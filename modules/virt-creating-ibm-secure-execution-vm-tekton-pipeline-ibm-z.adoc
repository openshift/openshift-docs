= Creating an {ibm-title} Secure Execution VM by using the Tekton pipeline

[role="_abstract"]
You can automate the creation of an {ibm-name} Secure Execution virtual machine (VM) on {ibm-z-name} and {ibm-linuxone-name} by using the `secure-execution-installer` Tekton pipeline. The pipeline handles automated Linux installation, LUKS encryption, {ibm-name} Secure Execution image generation, and optional golden image creation.

.Prerequisites

* {product-title} {product-version} or later is installed.
* {pipelines-title} Operator is installed.
* Your cluster nodes run on {ibm-name} z15 or later, or {ibm-linuxone-name} III or later hardware with Secure Execution support.
* A storage class with dynamic provisioning is configured.
* You have the following certificate files:
** A host key document certificate (`host-key-doc.crt`)
** An {ibm-name} signing certificate (`ibm-sign.crt`)
** A CA certificate (`ca-cert.crt`)
* You have an SSH public key.
* The installation ISO URL for a supported operating system is available. Supported operating systems are {op-system-base} 8.x, {op-system-base} 9.x, and Fedora Server (s390x).

.Procedure

. Deploy the role-based access control (RBAC) resources for the pipeline by running the following command:
+
[source,terminal]
----
$ oc apply -f https://raw.githubusercontent.com/kubevirt/kubevirt-tekton-tasks/main/templates-pipelines/secure-execution-installer/manifests/se-pipeline-rbac.yaml
----

. Encode the required certificate files and your SSH public key in Base64 format by running the following commands:
+
[source,terminal]
----
$ HOST_DOC=$(base64 -w0 < host-key-doc.crt)
----
+
[source,terminal]
----
$ IBM_SIGN=$(base64 -w0 < ibm-sign.crt)
----
+
[source,terminal]
----
$ CA_CERT=$(base64 -w0 < ca-cert.crt)
----
+
[source,terminal]
----
$ SSH_KEY=$(cat ~/.ssh/id_rsa.pub)
----

. Create a `PipelineRun` manifest file named `se-installer-run.yaml` with the following content:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: PipelineRun
metadata:
  generateName: se-installer-run-
  labels:
    pipelinerun: se-installer-run
spec:
  taskRunTemplate:
    serviceAccountName: pipeline
  params:
    - name: vmName
      value: "<vm_name>"
    - name: namespace
      value: "<namespace>"
    - name: memory
      value: "<memory>"
    - name: diskSize
      value: "<disk_size>"
    - name: isoUrl
      value: "<iso_url>"
    - name: bootImage
      value: "<boot_image>"
    - name: hostDoc
      value: "<host_doc_base64>"
    - name: ibmSign
      value: "<ibm_sign_base64>"
    - name: caCert
      value: "<ca_cert_base64>"
    - name: sshKey
      value: "<ssh_public_key>"
    - name: storageClass
      value: "<storage_class>"
    - name: createGoldenImage
      value: "false"
    - name: seTemplatesConfigMapURL
      value: "https://raw.githubusercontent.com/kubevirt/kubevirt-tekton-tasks/main/templates-pipelines/secure-execution-installer/configmaps/se-templates-configmaps.yaml"
    - name: cliImage
      value: "<cli_image>"
    - name: kickstartServerImage
      value: "docker.io/httpd:2.4-alpine"
  pipelineRef:
    name: secure-execution-installer
  timeouts:
    pipeline: 20m0s
----
+
where:

`<vm_name>`:: Specifies the name of the VM to create.
`<namespace>`:: Specifies the namespace in which to create the VM.
`<memory>`:: Specifies the memory allocation for the VM, for example `4Gi`.
`<disk_size>`:: Specifies the disk size for the VM, for example `30Gi`.
`<iso_url>`:: Specifies the URL of the operating system installation ISO image.
`<boot_image>`:: Specifies the container image to use for booting, for example `quay.io/<repo>/<image>:latest`.
`<host_doc_base64>`:: Specifies the Base64-encoded host key document certificate. Use the value of `$HOST_DOC` from the earlier step.
`<ibm_sign_base64>`:: Specifies the Base64-encoded {ibm-name} signing certificate. Use the value of `$IBM_SIGN` from the earlier step.
`<ca_cert_base64>`:: Specifies the Base64-encoded CA certificate. Use the value of `$CA_CERT` from the earlier step.
`<ssh_public_key>`:: Specifies the SSH public key for accessing the VM. Use the value of `$SSH_KEY` from the earlier step.
`<storage_class>`:: Specifies the name of the storage class to use for the VM disk.
`<cli_image>`:: Specifies the container image that provides the `kubectl` CLI, for example `quay.io/<repo>/kubectl:latest`.

. Start the pipeline run by running the following command:
+
[source,terminal]
----
$ oc create -f se-installer-run.yaml
----

. Watch the pipeline run progress by running the following command:
+
[source,terminal]
----
$ oc get pipelinerun -w
----

.Verification

* Verify that the VM is running by running the following command:
+
[source,terminal]
----
$ oc get vm
----

[NOTE]
====
To create a golden image from the completed VM, set `createGoldenImage` to `"true"` in the `PipelineRun` manifest and optionally specify a `goldenImageName` parameter value. The pipeline creates a `DataSource` from the completed VM that you can reuse for rapid deployment of additional {ibm-name} Secure Execution VMs with the same configuration.
====
