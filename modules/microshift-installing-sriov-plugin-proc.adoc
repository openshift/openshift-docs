= Installing the SR-IOV plugin and CNI provider

Install the necessary SR-IOV components to enable {microshift-short} to discover SR-IOV devices and expose them as resources for scheduling.

.Prerequisites

* You have the required RPM package or deployment manifests containing the SR-IOV device plugin and the CNI provider.
* You have configured the underlying network interface controllers (NICs) to expose VFs.

.Procedure

. If provided as an optional RPM, install the required `microshift-sriov` RPM package.

. Restart the {microshift-short} service to deploy the SR-IOV resources in the `sriov-network-operator` namespace.

. Deploy the `NetworkNodePolicy` custom resource definition (CRD) to specify the desired VF configuration based on the available hardware.

. Deploy an `SriovNetwork` custom resource (CR) which references the `NetworkNodePolicy`. The operator generates a `NetworkAttachmentDefinition` CR and the VFs become available to the pods.


[source,bash]
----
sudo yum install microshift-sriov-plugins  # Placeholder command
sudo systemctl restart microshift.service
----

.Verification

. Verify that the SR-IOV device plugin resources are deployed.

. Verify that the device plugin DaemonSet or required resources are running, which enables the devices to be exposed as resources within the {microshift-short} environment.

. Verify that the SR-IOV CNI provider is available by checking system logs or {microshift-short} components.
+
[NOTE]
====
The SR-IOV CNI provider is required for using Network Attachment Definitions (NADs) to access the VFs.
====
+
. Verify that the new resource types that represent SR-IOV devices are listed as allocatable resources on your {microshift-short} node.
