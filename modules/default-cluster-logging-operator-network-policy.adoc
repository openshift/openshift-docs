= Default cluster logging operator network policy

When a `ClusterLogForwarder` resource is deployed, the {clo} creates a permissive `NetworkPolicy` resource by default. The network policy that the resource defines allows all ingress and egress traffic for the collector pods. The resource is automatically created and removed along with the collector deployment lifecycle.

The default `NetworkPolicy` resource ensures that all ingress and egress traffic for the collector pods is allowed even when the following restrictions are in place:

* Restrictive default `NetworkPolicies` in the cluster.
* `AdminNetworkPolicy` configuration that limits pod communications.
* Namespace-level network restrictions.

The {clo} creates the a `NetworkPolicy` resource with the following content:

[source,yaml]
----
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: <COLLECTOR-INSTANCE-NAME>
  namespace: <COLLECTOR-NAMESPACE>
  labels:
    app.kubernetes.io/name: vector
    app.kubernetes.io/instance: <COLLECTOR-INSTANCE-NAME>
    app.kubernetes.io/component: collector
    app.kubernetes.io/part-of: cluster-logging
    app.kubernetes.io/managed-by: cluster-logging-operator
    app.kubernetes.io/version: <CLO-VERSION>
spec:
  podSelector:
    matchLabels:
      app.kubernetes.io/name: vector
      app.kubernetes.io/instance: <COLLECTOR-INSTANCE-NAME>
      app.kubernetes.io/component: collector
      app.kubernetes.io/part-of: cluster-logging
      app.kubernetes.io/managed-by: cluster-logging-operator
  policyTypes:
    - Ingress
    - Egress
  ingress:
    - {}  # Allow all ingress traffic
  egress:
    - {}  # Allow all egress traffic
----

As a cluster administrator, you can edit the resource and your changes will not be reconciled by the operator upon updates.
