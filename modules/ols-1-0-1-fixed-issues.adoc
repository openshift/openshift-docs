= Fixed issues

The following issues are fixed with {ols-official} 1.0.1:

* Previously, if the {ocp-product-title} cluster had a cluster-wide proxy and the `no_proxy` environment variable was set, the {ols-long} service still tried to connect to the large language model (LLM) provider through the proxy. Now, when you set the `no_proxy` environment variable, the service does not try to connect to the LLM provider. link:https://issues.redhat.com/browse/OLS-1861[OLS-1861].
