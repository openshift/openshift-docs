= Configure DHCP for IP addressing

[role="_abstract"]
Configure an SR-IOV network to obtain IP addresses from a DHCP server on the physical network.

.Prerequisites

* A DHCP server is running on the physical network connected to the SR-IOV devices
* The DHCP server has IP address pools configured
* The DHCP server allows DHCP requests from SR-IOV virtual function MAC addresses

.Procedure

. Create an SR-IOV network with DHCP IPAM configuration:
+
[source,yaml]
----
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetwork
metadata:
  name: sriov-network-dhcp
  namespace: openshift-sriov-network-operator
spec:
  resourceName: intelnics
  networkNamespace: default
  vlan: 0
  ipam: |  
    {
      "type": "dhcp"
    }
----
 IPAM configuration specifying DHCP as the IP address assignment method.

. Create the SR-IOV network:
+
[source,terminal]
----
$ oc create -f sriov-network-dhcp.yaml
----

. Create a pod that uses the DHCP network:
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: pod-dhcp
  namespace: default
  annotations:
    k8s.v1.cni.cncf.io/networks: sriov-network-dhcp  
spec:
  containers:
  - name: test
    image: registry.access.redhat.com/ubi9/ubi:latest
    command: ["/bin/bash", "-c", "sleep infinity"]
    resources:
      requests:
        openshift.io/intelnics: "1"
      limits:
        openshift.io/intelnics: "1"
----
 Reference to the SR-IOV network. No IP address needed - DHCP provides it automatically.

. Create the pod:
+
[source,terminal]
----
$ oc create -f pod-dhcp.yaml
----

.Verification

. Verify the pod received an IP address from DHCP:
+
[source,terminal]
----
$ oc exec -it pod-dhcp -- ip addr show net1
----
+
The output should show an IP address assigned by your DHCP server.

. Verify the pod can reach the default gateway provided by DHCP:
+
[source,terminal]
----
$ oc exec -it pod-dhcp -- ip route show dev net1
----
+
You should see a default route if your DHCP server provided one.

.Troubleshooting

If the pod does not receive an IP address from DHCP:

* Verify the DHCP server is reachable on the physical network
* Check DHCP server logs for requests from the pod's MAC address
* Verify the DHCP server has available IP addresses in its pool
* Ensure the SR-IOV virtual function MAC address is not blocked by the DHCP server
* Check that the correct VLAN is configured if using VLANs

[NOTE]
====
DHCP lease times are controlled by the DHCP server. Ensure lease times are appropriate for your pod lifecycle. Long-running pods should have longer lease times to avoid renewal issues.
====
