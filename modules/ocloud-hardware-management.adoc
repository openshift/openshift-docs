= Declarative hardware management for bare-metal servers

[role="_abstract"]
The O-Cloud Manager provides declarative hardware management for bare-metal servers used in cluster provisioning. It manages an infrastructure hierarchy of locations, sites, and resource pools, and automates hardware allocation, firmware preparation, and BIOS configuration as part of the provisioning workflow.

The infrastructure hierarchy consists of four levels:

Location:: A geographic place where infrastructure is deployed, represented by a `Location` custom resource.
OCloudSite:: A logical site at a given location, represented by an `OCloudSite` custom resource that references a `Location`.
ResourcePool:: A group of bare-metal resources at a site, represented by a `ResourcePool` custom resource that references an `OCloudSite`.
Resource:: An individual bare-metal server, represented by a `BareMetalHost` custom resource with labels that associate it with a `ResourcePool`.

Servers are registered in the hub cluster as `BareMetalHost` resources with metadata labels that identify the resource pool membership, server type, and other characteristics. The Bare Metal Operator connects to each host's BMC, inspects the physical hardware, and populates a `HardwareData` resource with CPU, memory, NIC, and storage details.

The `HardwareProfile` custom resource defines the desired hardware state for a class of servers. It specifies target firmware levels and BIOS settings that the hardware manager applies during initial provisioning (Day-0) or as Day-2 updates. A `HardwareProfile` is optional. Servers can be provisioned without one if firmware and BIOS management is not required.

When processing a `ProvisioningRequest`, the O-Cloud Manager selects servers from resource pools using a two-stage filtering process defined in the `ClusterTemplate`:

Label-based filtering:: Selects `BareMetalHost` resources by labels such as server type, location identifiers, or custom resource selector labels.
Hardware data filtering:: Further filters hosts based on their `HardwareData` custom resource, matching criteria such as CPU architecture, thread count, memory size, NIC speed and count, and storage type and capacity.

All criteria in the resource selector must match for a host to be selected. Only hosts in the `Available` provisioning state are evaluated for allocation. After selection, the O-Cloud Manager creates an `AllocatedNode` resource to track the assignment and, if a `HardwareProfile` is specified, triggers firmware and BIOS preparation before cluster installation begins.
