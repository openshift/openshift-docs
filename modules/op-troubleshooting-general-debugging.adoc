= General debugging tips for cluster resolver migration

If you encounter other issues during migration, use the following troubleshooting steps.

.Procedure

. View the pipeline run events for detailed error messages by running the following command:
+
[source,terminal]
----
$ oc get events --sort-by='.lastTimestamp' | grep <pipelinerun_name>
----
+
<pipelinerun_name>:: The name of the pipeline run to search for in the events.

. Check the resolver logs for resolution errors by running the following command:
+
[source,terminal]
----
$ oc logs -n openshift-pipelines -l app=tekton-pipelines-resolvers --tail=100
----

. Verify the service account has the necessary permissions by running the following command:
+
[source,terminal]
----
$ oc auth can-i get tasks --as=system:serviceaccount:my-project:pipeline -n openshift-pipelines
----

. Ensure that the {pipelines-title} Operator is running the correct version by running the following command:
+
[source,terminal]
----
$ oc get csv -n openshift-pipelines
----
