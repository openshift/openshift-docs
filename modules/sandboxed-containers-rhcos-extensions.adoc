= Red Hat CoreOS extensions

The {sandboxed-containers} uses the {op-system-first} extensions concept. The Kata binaries, RPMs, and QEMU are provided in the form of extensions. They are easily enabled via the `MachineConfig` resources provided by the Machine Config Operator.

For more information on how the extension concept works in practice, see link:https://docs.openshift.com/container-platform/latest/post_installation_configuration/machine-configuration-tasks.html#rhcos-add-extensions_post-install-machine-configuration-tasks[adding extensions to RHCOS].
