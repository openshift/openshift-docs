= Configuring revision timeout

You can configure the default number of seconds to use for the per-request revision timeout.

.Prerequisites

* You have installed the {ServerlessOperatorName} and Knative Serving.
* You have cluster administrator permissions on {ocp-product-title}, or you have cluster or dedicated administrator permissions on {rosa-product-title} or {dedicated-product-title}.

.Procedure

* To configure the revision timeout globally, set the `revision-timeout-seconds` field in the `KnativeServing` custom resource (CR):
+
[source,yaml]
----
apiVersion: operator.knative.dev/v1beta1
kind: KnativeServing
metadata:
  name: knative-serving
  namespace: knative-serving
data:
  revision-timeout-seconds: "300"
----
+
Alternatively, you can configure the timeout per revision by setting the `timeoutSeconds` field in your service definition:
+
[source,yaml]
----
apiVersion: serving.knative.dev/v1
kind: Service
metadata:
  namespace: my-ns
spec:
  template:
    spec:
      timeoutSeconds: 300
      containers:
      - image: ghcr.io/knative/helloworld-go:latest
----
