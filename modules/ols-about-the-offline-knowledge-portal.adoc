= About the Offline Knowledge Portal

[role="_abstract"]
The Offline Knowledge Portal (OKP) is the primary retrieval-augmented generation (RAG) mechanism that {ols-long} uses to retrieve {ocp-product-title} product documentation. OKP serves documentation through Apache Solr and is enabled by default.

The {ols-long} Operator deploys an RHOKP sidecar container alongside the lightspeed-service app server in the same pod. The sidecar is a self-contained appliance that includes the {ocp-product-title} documentation corpus and an Apache Solr search engine.

{ols-long} uses tool-based retrieval. During its reasoning loop, the large language model (LLM) decides when to invoke the documentation search tool. The search uses hybrid retrieval, combining lexical queries with vector reranking. Results are deduplicated by parent document, scored, filtered by threshold, and optionally expanded with neighboring chunks.

OKP requires approximately 75 GiB of ephemeral storage for Solr data, along with 2 CPU and 2 GiB of memory by default. Cluster administrators can tune these resource allocations by using `spec.ols.deployment.rhokp` on the `OLSConfig` custom resource (CR).

On {product-rosa} (ROSA) clusters, OKP automatically includes ROSA-specific documentation alongside {ocp-product-title} documentation, so that answers on ROSA clusters can reference guidance tailored to the managed-service environment.

OKP is enabled by default. To use only custom documentation that you provide through the BYO Knowledge tool, set `spec.ols.byokRAGOnly` to `true` in the `OLSConfig` CR. When this field is `true`, the operator does not deploy the RHOKP sidecar.
