= Exposing isolated {mcpg} instances externally

[role="_abstract"]
After you deploy isolated {mcpg} instances, you can expose each `Gateway` object externally by creating {ocp} routes with TLS termination.

.Prerequisites

* You installed {ocp}.
* You installed {oc-first}.
* You deployed isolated {mcpg} instances for team A and team B.
* You set the `TEAM_A_HOST` and `TEAM_B_HOST` environment variables.

.Procedure

. Create an {ocp} route for the team A gateway by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc apply -f - <<EOF
apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: team-a-gateway
  namespace: gateway-system
spec:
  host: $TEAM_A_HOST
  tls:
    insecureEdgeTerminationPolicy: Redirect
    termination: edge
  port:
    targetPort: mcp
  to:
    kind: Service
    name: team-a-gateway-istio
    weight: 100
  wildcardPolicy: None
EOF
----

. Create an {ocp} route for the team B gateway by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc apply -f - <<EOF
apiVersion: route.openshift.io/v1
kind: Route
metadata:
  name: team-b-gateway
  namespace: gateway-system
spec:
  host: $TEAM_B_HOST
  tls:
    insecureEdgeTerminationPolicy: Redirect
    termination: edge
  port:
    targetPort: mcp
  to:
    kind: Service
    name: team-b-gateway-istio
    weight: 100
  wildcardPolicy: None
EOF
----

. Verify the routes are created by running the following command:
+
[source,terminal]
----
$ oc get routes -n gateway-system
----

.Next steps

* Register MCP servers with each gateway instance. When registering servers, ensure the `HTTPRoute` CR `parentRef` targets the correct `Gateway` object:
** Team A instance: `team-a-gateway` in `gateway-system`
** Team B instance: `team-b-gateway` in `gateway-system`
