= Analyzing TLS traffic data

[role="_abstract"]
View and filter TLS metadata to identify deprecated configurations and verify encryption compliance in the cluster.

.Prerequisites

* TLS tracking is enabled in the `FlowCollector` custom resource (CR).
* TLS-encrypted traffic is present in the cluster.
* Access to the {product-title} web console.

.Procedure

. Navigate to *Observe* → *Network Traffic* in the {product-title} web console.

. Add TLS-specific columns to the traffic table:

.. Click *Manage columns*.

.. Select the following TLS columns:
* *TLS Version*
* *TLS Cipher Suite*
* *TLS Curve*
* *TLS Types*

.. Click *Save*.
+
[NOTE]
====
If the TLS columns are not visible after enabling TLS tracking, click *Restore default columns* to refresh the list.
====

. Filter traffic to view TLS metadata:

.. In the filter bar, select *TLS Types*.

.. Select *ServerHello* from the dropdown menu.
+
This filters the view to show only `ServerHello` messages, which contain negotiated TLS metadata such as cipher suite and curve information.

. Analyze TLS metrics in the overview panel:

.. Scroll to the *Overview* section.

.. Review the following TLS metrics:
* *TLS Version distribution*: Displays the breakdown of TLS versions in use in the cluster.
* *Top cipher suites*: Displays the most frequently used cipher suites.
* *Insecure TLS connections*: Highlights flows using TLS versions earlier than 1.2.

. View TLS metrics in dashboard charts:

.. Click *View dashboard*.

.. Review the *TLS Security* dashboard for the following information:
* TLS version trends over time.
* Cipher suite usage patterns.
* Workloads using deprecated TLS configurations.

. Identify secure connections in the topology view:

.. Navigate to *Observe* → *Network Traffic*.

.. Click the *Topology* tab.
+
Connections secured with TLS are marked with a lock icon. Hover over a connection to view the TLS version and cipher suite.

. Filter by specific TLS versions to identify deprecated configurations:

.. In the filter bar, select *TLS Version*.

.. Select the versions to filter:
* *1.0*: Deprecated.
* *1.1*: Deprecated.
* *1.2*: Secure.
* *1.3*: Current standard.

.. To identify all deprecated connections, filter for TLS versions 1.0 and 1.1.

.Verification

* Confirm that TLS fields are populated in the traffic table:
+
[source,terminal]
----
$ oc get flows -o jsonpath='{.items[0].spec.tls}'
----
+
.Example output
[source,json]
----
{
  "version": "1.3",
  "cipherSuite": "TLS_AES_256_GCM_SHA384",
  "curve": "X25519",
  "type": "ServerHello"
}
----
