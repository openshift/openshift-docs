= About {IBMFusionSAN}

{IBMFusionSAN} is a scalable clustered file system for enterprise storage, providing an easy user experience for storage management with {VirtProductName}. It also lets users leverage their existing SAN infrastructure (FC, iSCSI) using traditional methods (???? such as???).

Key components include {Fusion Access for SAN} Operator installation, Spectrum Cluster for identifying shared storage nodes, LocalDisks representing storage LUNs, and FileSystems built upon these disks.

Standard {product-title} (OCP) storage objects like StorageClasses, PVCs, and PVs are then layered on top, with StorageClasses specifically pointing to defined FileSystems. This architecture leverages existing SAN infrastructure and scales with both OCP clusters and virtual machine (VM) workloads.

{IBMFusionSAN} has tested up to ~3000 VMs on 6 bare-metal hosts, and can currently beused for basic storage provision and data protection.

The installation and setup process involves several prerequisites and steps. Bare-metal worker nodes with attached SAN storage, where all nodes can see the same LUNs, are required. The installation then proceeds with a MachineConfig for workers, the installation of the Fusion Access Operator, obtaining a pull secret from IBM, and creating a FusionAccess Custom Resource (CR) to initialize Spectrum Scale. Finally, users can follow a UI wizard to configure the underlying Spectrum objects (Cluster, LocalDisk, FileSystem), although YAML-based configuration is also an option. This streamlined approach aims to simplify the deployment and management of shared storage for OpenShift workloads.

Prereqs:

Bare-metal Worker Nodes with SAN storage attached
* All nodes need to see the same LUNs. How they are attached is environment dependent (multipath, direct-attach, FC, iSCSI, etc) and is not covered here. In general, a MachineConfig will need to take care of it



Installation and setup:
1. MachineConfig for Workers
A MachineConfig on workers installing the “kernel-devel” extension is required (*)
2. Install the upcoming Fusion Access Operator
This installs the operator in charge of UI, disk discovery and general installation of the IBM parts
3. Obtain Pull Secret from IBM
This Pull Secret will be needed for getting the containers from the IBM container registry
4. Create FusionAccess CR
This CR initializes the Spectrum Scale bits and triggers the Spectrum Scale operator installation (and starts the UI)
5. Follow UI wizard
The UI will show the user which nodes have shared disks and do walk through the necessary steps to configure and create the underlying Spectrum Objects (Cluster, LocalDisk, FileSystem) which manage the storage part. (Skipping the UI and doing it via YAML is also doable)
