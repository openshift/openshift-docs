== Understanding the PromQL expression for alerts

[role="_abstract"]
Learn about the base query for Prometheus Query Language (PromQL), and how to customize it so you can configure network observability alerts for your specific needs.

The base query pattern for PromQL is:

`sum(rate(netobserv_namespace_ingress_bytes_total{SrcK8S_Namespace="openshift-ingress"}[30m])) by (DstK8S_Namespace)`

This is the bytes rate coming from "openshift-ingress" to any of your workload's namespaces over the last 30 minutes. Depending on your configuration, you may need to use `netobserv_workload_ingress_bytes_total` instead of `netobserv_namespace_ingress_bytes_total`.

Appending `> 1000` to this query keeps only the rates observed that are greater than `1 KB/s`, which eliminates noise from low-bandwidth consumers.

`(sum(rate(netobserv_namespace_ingress_bytes_total{SrcK8S_Namespace="openshift-ingress"}[30m])) by (DstK8S_Namespace) > 1000)`

The bytes rate is relative to the sampling ratio defined in the `FlowCollector` custom resource configuration. If you have a sampling ratio of `1:100`, the actual traffic might be approximately 100 times higher than what is reported by the metrics.

You can run the same query for a particular period of time. For example, `offset 1d` runs the query for one day earlier, and `offset 5h` runs the query for five hours ago.

`sum(rate(netobserv_namespace_ingress_bytes_total{SrcK8S_Namespace="openshift-ingress"}[30m] offset 1d)) by (DstK8S_Namespace))`

This gives you the formula `100 * (<query now> - <query yesterday>) / <query yesterday>`, which is the percentage of increase compared to the previous day. This value can be negative if the bytes rate today is lower than yesterday.

Finally, you can eliminate increases that are lower than you want so you won't get alerted for them. For example, `> 100` eliminates increases that are lower than 100%.

Together, the expression looks something like this:

[source,yaml]
----
...
      expr: |-
        (100 *
          (
            (sum(rate(netobserv_namespace_ingress_bytes_total{SrcK8S_Namespace="openshift-ingress"}[30m])) by (DstK8S_Namespace) > 1000)
            - sum(rate(netobserv_namespace_ingress_bytes_total{SrcK8S_Namespace="openshift-ingress"}[30m] offset 1d)) by (DstK8S_Namespace)
          )
          / sum(rate(netobserv_namespace_ingress_bytes_total{SrcK8S_Namespace="openshift-ingress"}[30m] offset 1d)) by (DstK8S_Namespace))
        > 100
----
