= Unexpected resource deletion behavior

On clusters that support migrating resources between the Machine API and the Cluster API, users might experience unexpected behavior when deleting resources that do not use the current authoritative API.
This behavior occurs when deleting resources directly and when performing scale-down operations.

Cause::

For any resource that uses the authoritative API, the two-way synchronization controller creates and maintains corresponding unprovisioned resources that do not use the current authoritative API.
+
When you delete a resource directly or by performing a scale-down operation, the synchronization controller deletes any corresponding resources.

Consequence::

When you delete an unprovisioned resource that does not use the current authoritative API, the synchronization controller deletes the corresponding resource that does use the current authoritative API.

Workaround::

Do not delete any unprovisioned resource that does not use the current authoritative API unless you want to delete the corresponding resource that does use the current authoritative API.
