= Configuring {aws-short} Bedrock

[role="_abstract"]
To activate {aws-short} Bedrock as your model provider, you must create an infrastructure secret containing your authentication credentials and modify the cluster `OLSConfig` custom resource file.

.Prerequisites

* You have logged in to the {ocp-product-title} web console as a user with the `cluster-admin` role.
* You have installed the {ols-long} Operator.
* You have an {aws-short} infrastructure account with active permission to access your required models.

.Procedure

. Create a `Secret` resource file within the `openshift-lightspeed` namespace. Ensure that the secret aligns with your preferred authentication configuration.
+
* Example credentials secret using a Bearer token:
+
[source,yaml]
----
apiVersion: v1
kind: Secret
metadata:
  name: bedrock-credentials
  namespace: openshift-lightspeed
type: Opaque
stringData:
  apitoken: <mantle-bearer-token>
----

* Example credentials secret using AWS IAM keys:
+
[source,yaml]
----
apiVersion: v1
kind: Secret
metadata:
  name: bedrock-credentials
  namespace: openshift-lightspeed
type: Opaque
stringData:
  aws_access_key_id: AKIA...
  aws_secret_access_key: ...
----

* Example credentials secret using IAM keys with role assumption (STS):
+
[source,yaml]
----
apiVersion: v1
kind: Secret
metadata:
  name: bedrock-credentials
  namespace: openshift-lightspeed
type: Opaque
stringData:
  aws_access_key_id: AKIA...
  aws_secret_access_key: ...
  role_arn: arn:aws:iam::123456789012:role/BedrockAccessRole
----

. Apply the configuration file to create your secret by running the following command:
+
[source,terminal]
----
$ oc create -f <secret_filename>.yaml
----

. Open and edit your existing `OLSConfig` custom resource definition to define the new provider architecture:
+
[source,terminal]
----
$ oc edit olsconfig cluster
----

. Update the `spec` section to include the `bedrock` provider configuration and declare your target foundational models:
+
[source,yaml]
----
spec:
  ols:
    defaultProvider: bedrock
    defaultModel: anthropic.claude-opus-4-7
  llm:
    providers:
    - name: bedrock
      type: bedrock
      url: https://bedrock-mantle.us-east-1.api.aws
      credentialsSecretRef:
        name: bedrock-credentials
      models:
      - name: anthropic.claude-opus-4-7
      - name: deepseek.v3.1
----

. Save your modifications and exit the text editor. The operator automatically reconciles the changes, manages the service endpoints, and provisions the necessary access credentials.
