= Creating a cluster-scoped user-defined network with a localnet topology

[role="_abstract"]
You can connect multiple namespaces to a cluster scoped physical network through a `ClusterUserDefinedNetwork` custom resource in the {product-title} web console.

.Prerequisites
* You have access to the OpenShift Container Platform web console as a user with `cluster-admin` permissions.

.Procedure

. From the *Virtualization* perspective, click *Networking* â†’ *VirtualMachine networks*.

. Click the *Create* button. The *Create virtual machine network* wizard is displayed.

. Provide details about the network on the *Network definition* page.
+
[NOTE]
====
The Maximum Transmission Unit (MTU) is the largest size of a data packet, in bytes, that is transmitted. Ensure that all underlying physical network equipment supports this MTU, or higher.
====

. Click *Next*.

. Select the projects that the network should be available to on the *Project mapping* page. The default is that all projects will have access to the network.

. Click *Create*.

.Verification

* Navigate to the *Virtualization* -> *VirtualMachine networks* page.
* Click the *OVN localnet* tab.
* Verify that your new network appears in the list.
