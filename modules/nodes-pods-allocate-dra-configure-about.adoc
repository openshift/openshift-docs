= About configuring device allocation by using device attributes

// Taken from https://kubernetes.io/docs/concepts/scheduling-eviction/dynamic-resource-allocation/#terminology
{attribute-based-full} uses the following Kubernetes API kinds to provide the core allocation functionality. All of these API kinds are included in the resource.k8s.io/v1 API group.

Device class::
A device class is a category of devices that can be claimed and how to select specific device attributes in claims. Some device drivers contain their own device class, such as gpu.nvidia.com. Alternatively, an administrator can create device classes. To claim devices from a device class, use a `ResourceClaims` to select specific device attributes.

.Example device class
[source,yaml]
----
apiVersion: resource.k8s.io/v1
kind: DeviceClass
metadata:
  name: example-device-class
spec:
  selectors:
  - cel:
      expression: |-
        device.driver == "driver.example.com"
----

Resource slice::
A resource slice represents one or more GPU resources that are attached to nodes. The DRA device driver on each node creates and manages resource slices in the cluster. When a resource claim is created and used in a pod, {product-title} uses these resource slices to find nodes that have access to the claimed resources. {product-title} allocates resources to the resource claim and schedules the pod onto a node that can access the resources.

The following example resource slice is managed by the `resource-driver.example.com` driver in the `black-cat-pool` pool. The `allNodes: true` field indicates that any node in the cluster can access the devices. There is one device in the resource slice, named `large-black-cat`, with the following attributes:

* color: black
* size: large
* cat: true

A device class could select this resource slice by using these attributes and a resource claim could filter for specific devices in that device class.

.Example resource slice
[source,yaml]
----
apiVersion: resource.k8s.io/v1
kind: ResourceSlice
metadata:
  name: cat-slice
spec:
  driver: "driver.example.com" <1>
  pool: <2>
    generation: 1
    name: "black-cat-pool"
    resourceSliceCount: 1
  allNodes: true <3>
  devices: <4>
  - name: "large-black-cat"
    attributes:
      color:
        string: "black"
      size:
        string: "large"
      cat:
        boolean: true
----
<1> Specifies a device driver, which manages resource slices.
<2> Specifies a group of one or more GPU resources that the driver manages. The pool can span more than one resource slice. Changes to the resources in a pool must be propagated across all of the resource slices in that pool.
<3> Specifies whether any node in the cluster can access the device.
<4> Specifies the devices in this pool and the device attributes. 

resourceSliceCount Specifies the total number of resource slices that belong to the resource pool.

Resource claim::
A resource claim is a request for access to attached cluster resources, such as devices. Resource claims provide pods with access to a specific resource. A `ResourceClaims` object can be created by admin or non-admin users or generated by {product-title} based on a resource claim template.

The following example resource claim requests devices in the `example-device-class` device class that match both of the following parameters:

* Devices that have a `driver.example.com/type` attribute with a value of `gpu`.
* Devices that have 64Gi of capacity.

.Example resource claim object
[source,yaml]
----
apiVersion: resource.k8s.io/v1beta1
kind: ResourceClaim
metadata:
  namespace: gpu-test3
  name: single-gpu
spec:
  devices:
    requests:
    - name: gpu
      deviceClassName: gpu.nvidia.com
----

Resource claim template::
A resource claim template defines a template that {product-title} uses to create resource claims for a workload. Resource claim templates provide pods with access to separate, similar resources. Each resource claim that {product-title} generates from the template is bound to a specific pod. When the pod terminates, {product-title} deletes the corresponding resource claim.

.Example resource claim template object
[source,yaml]
----
apiVersion: resource.k8s.io/v1beta1
kind: ResourceClaimTemplate
metadata:
  namespace: gpu-test1
  name: single-gpu-template
spec:
  spec:
    devices:
      requests:
      - name: gpu
        deviceClassName: gpu.nvidia.com
----
