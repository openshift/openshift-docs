= Create a node pool with a non-default `OpenShiftEC2NodeClass`

[role="_abstract"]
After creating a non-default `OpenShiftEC2NodeClass` resource, create a `NodePool` that references it. This binds the node pool's provisioned nodes to the custom configuration defined in the non-default `OpenShiftEC2NodeClass`.

.Prerequisites

* A non-default `OpenShiftEC2NodeClass` resource exists and is in a `Ready` state.

.Procedure

. Create a node pool manifest that references the non-default `OpenShiftEC2NodeClass`:
+
[source,terminal]
----
$ cat > nodepool.yaml <<'EOF'
apiVersion: karpenter.sh/v1
kind: NodePool
metadata:
  name: custom-np
spec:
  template:
    spec:
      requirements:
      - key: node.kubernetes.io/instance-type
        operator: In
        values:
        - c5.xlarge
      - key: karpenter.sh/capacity-type
        operator: In
        values: ["on-demand"]
      nodeClassRef:
        group: karpenter.k8s.aws
        kind: EC2NodeClass
        name: custom-nodeclass
EOF
----
where:
`nodeClassRef.name`::
The name of the non-default `OpenShiftEC2NodeClass` resource. This value must match the `metadata.name` of an existing `OpenShiftEC2NodeClass`.

. Apply the node pool:
+
[source,terminal]
----
$ oc apply -f nodepool.yaml
----

.Verification

* Verify that the node pool references the non-default `OpenShiftEC2NodeClass`:
+
[source,terminal]
----
$ oc get nodepool
----
+
.Example output
[source,terminal]
----
NAME        NODECLASS          NODES   READY   AGE
custom-np   custom-nodeclass   0       True    3s
----
