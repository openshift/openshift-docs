= Viewing logs

[role="_abstract"]
You can view the logs of a build run to identify and troubleshoot runtime errors.

.Prerequisites

* You have installed the `oc` CLI.
* Optional: You have installed the link:https://console.redhat.com/openshift/downloads[`shp` CLI].

.Procedure

* View logs of a `Buildrun` resource by using the `shp` CLI or the `oc` CLI.
** Using `shp` CLI:
+
[source,terminal]
----
$ shp buildrun logs <buildrun_name>
----
+
--
where:

`<buildrun_resource_name>`:: Specifies the name of the `Buildrun` resource for which you want to view the logs.
--
+
The following example shows the command and its output:
+
[source,terminal]
----
$ shp buildrun logs buildah-dockerfile-buildrun
----
+

** Using `oc` CLI:
. Run the following command to view the logs of a `Buildrun` resource for all containers:
+
[source,terminal]
----
oc logs -l build.shipwright.io/name=<buildrun_name> --all-containers
----
+
--
where:

`<buildrun_resource_name>`:: Specifies the name of the `Buildrun` resource for which you want to view the logs.
--
+
The following example shows the command and its output:
+
[source,terminal]
----
oc logs -l build.shipwright.io/name=buildah-dockerfile-build --all-containers
----
+

. Run the following command to view the logs of a `Buildrun` resource of a specific container or build step:
+
[source,terminal]
----
oc logs -l build.shipwright.io/name=<buildrun_resource_name> -c <container_name>
----
+
--
where:

`<buildrun_resource_name>`:: Specifies the name of the `Buildrun` resource for which you want to view the logs.
`<container-name>`:: Specifies the name of the container.
--
+
The following example shows the command and its output:
+
[source,terminal]
----
oc logs -l build.shipwright.io/name=buildah-dockerfile-build -c step-build-and-push
----
+
