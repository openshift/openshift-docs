= Create the Backstage custom resource

[role="_abstract"]
To apply your configurations in {rhdh}, you must create a `Backstage` custom resource (CR) that brings all of your instructions and permissions together.

.Prerequisites

* You installed {prodname} 1.4 or later on an {ocp} cluster.
* You installed {rhdh} on the same {ocp} cluster.
* You have cluster administrator privileges to configure role-based-access (RBAC) policies and service accounts.
* You installed the {oc-first}.
* Your organization has APIs published through {prodname} gateways that you want to make discoverable to development teams.

.Procedure

. Link the dynamic plugin `ConfigMap` object by adding the following content:
+
[source,yaml]
----
spec:
  application:
    dynamicPluginsConfigMapName: dynamic-plugins-rhdh
----

. Link the main `rhdh-app-config` `ConfigMap` object by adding the following content:
+
[source,yaml]
----
spec:
  application:
    appConfig:
      mountPath: /opt/app-root/src
      configMaps:
         - name: rhdh-app-config
----

. Link the RBAC policies from the `rbac-policies` `ConfigMap` object by adding the following content:
+
[source,yaml]
----
spec:
  application:
    extraFiles:
      mountPath: /opt/app-root/etc
      configMaps:
         - name: rbac-policies
----
+
The mounting path is referenced from `app-config.yaml`, RBAC for the kuadrant functionality section. Ensure they match.

. Optional. Enable service account token auto mount for `in-cluster` mode by adding the following content:
+
[source,yaml]
----
spec:
  deployment:
    patch:
      spec:
        template:
          spec:
            automountServiceAccountToken: true
----

.Results

. Your final YAML should include the following basic information and format:
+
[source,yaml]
----
apiVersion: rhdh.redhat.com/v1alpha3
kind: Backstage
metadata:
  name: rhdh
  namespace: $RHDH_NS
spec:
  application:
    appConfig:
      mountPath: /opt/app-root/src
      configMaps:
         - name: rhdh-app-config
    extraFiles:
      mountPath: /opt/app-root/etc
      configMaps:
         - name: rbac-policies
    route:
      enabled: true
      subdomain: frontend
    dynamicPluginsConfigMapName: dynamic-plugins-rhdh
  database:
    enableLocalDb: true
  deployment:
    patch:
      spec:
        template:
          spec:
            automountServiceAccountToken: true
----
