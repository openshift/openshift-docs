= Enabling network observability during installation

[role="_abstract"]
Configure network observability to be installed automatically during cluster creation for immediate network traffic monitoring capabilities.

Network observability is enabled by default on standard {product-title} clusters when the `NetworkObservabilityInstall` feature gate is enabled. You can explicitly configure this setting in the `install-config.yaml` file to ensure the Network Observability Operator is installed during cluster creation.

.Prerequisites

* You must enable the `NetworkObservabilityInstall` feature gate. For more information, see _Enabling the network observability feature gate_.
* You must have access to the `install-config.yaml` file before creating installation manifests.

.Procedure

. Edit the `install-config.yaml` file before generating installation manifests.

. Add the `networkObservability` field to the `networking` section:
+
[source,yaml]
----
apiVersion: v1
baseDomain: example.com
metadata:
  name: my-cluster
networking:
  networkObservability:
    installationPolicy: Enabled
# ...
----
+
where:
+
`networking.networkObservability.installationPolicy`:: Controls whether the Network Observability Operator is installed during cluster creation. Set to `Enabled` to install the operator automatically. Valid values are `Enabled` and `Disabled`. The default value is `Enabled` for standard clusters when the `NetworkObservabilityInstall` feature gate is enabled.

. Continue creating the installation manifests and enabling the feature gate before installing the cluster.

.Verification

* After cluster installation completes, verify that the Network Observability Operator is installed and running:
+
[source,terminal]
----
$ oc get pods -n openshift-netobserv-operator
----
+
.Example output
[source,terminal]
----
NAME                                                READY   STATUS    RESTARTS   AGE
netobserv-controller-manager-6d4b8d9f8c-xk7lm      2/2     Running   0          5m
----

* Verify that the FlowCollector resource was created:
+
[source,terminal]
----
$ oc get flowcollector cluster -n netobserv
----
+
.Example output
[source,terminal]
----
NAME      AGENT   SAMPLING   DEPLOYMENT MODEL   STATUS
cluster   eBPF    50         Direct             Ready
----

* Verify that network traffic data is available in the web console by navigating to *Observe* -> *Network Traffic*.
