= O-Cloud Manager architecture

[role="_abstract"]
The O-Cloud Manager Operator runs on a hub cluster and integrates with several {product-title} and {rh-rhacm-first} components to manage the lifecycle of spoke clusters. The Operator acts as a central orchestration point that coordinates hardware allocation, cluster provisioning, configuration, and upgrades.

The following diagram shows the O-Cloud Manager architecture and the flow from a `ProvisioningRequest` to a running spoke cluster:

[source,text]
----
SMO (optional, third-party)
    |
    | O2 IMS REST APIs
    v
O-Cloud Manager Operator (hub cluster, oran-o2ims namespace)
    |
    | Creates and reconciles CRs
    v
+------------------------------------------------------+
| ClusterTemplate    | ProvisioningRequest              |
| HardwareProfile    | NodeAllocationRequest            |
+------------------------------------------------------+
    |                    |                    |
    v                    v                    v
SiteConfig           Bare Metal              Topology Aware
Operator             Operator                Lifecycle Manager
(ClusterInstance)    (hardware mgmt)         (upgrades)
    |
    v
Spoke / Workload Clusters
----
