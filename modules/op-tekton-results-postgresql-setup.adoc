= Setting up PostgreSQL for Tekton Results

You can configure an external PostgreSQL database to store Tekton Results records across your multicluster environment. Both hub and spoke clusters connect to this shared database.

.Prerequisites

* You have access to create a PostgreSQL database instance.
* You have `cluster-admin` permissions on both hub and spoke clusters.

.Procedure

. Create a PostgreSQL database instance.
+
You can use a managed PostgreSQL service such as Amazon RDS, Google Cloud SQL, or Azure Database for PostgreSQL.
+

Ensure the database is configured with the following:
+
* Database name: `tekton-results`
* Network access from both hub and spoke cluster networks
* Appropriate security groups or firewall rules

. Verify database connectivity from a client machine.
+
Use the `psql` command-line tool to test the connection:
+
[source,terminal]
----
$ export PGPASSWORD='<password>'
$ psql -h <host> -U <user> -d tekton-results
----
+
Replace `<password>`, `<host>`, and `<user>` with your database credentials.
+
If the connection succeeds, you can proceed with configuring the hub and spoke clusters.

.Verification

* The `psql` command successfully connects to the database and displays a prompt:
+
[source,terminal]
----
psql (13.x)
Type "help" for help.

tekton-results=>
----
