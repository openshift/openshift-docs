= Track build progress and manage running builds

[role="_abstract"]
Track build progress and manage running builds by monitoring `BuildRun` status conditions and performing operations on active builds.
The `BuildRun` custom resource (CR) provides status information that indicates whether a build is pending, running, succeeded, or failed.
You can cancel running builds when needed and access system-defined results that provide build metadata and output information.

Build status information helps you understand the current state of your builds and diagnose issues.
The `BuildRun` CR stores status conditions in the `status.conditions` field, which includes condition types such as `Succeeded`, along with reasons and messages that explain the current state.
A build progresses through different status values as it moves from scheduling through execution to completion.
You can query build status using standard Kubernetes commands to monitor progress or integrate builds into automation workflows.

Canceling builds provides control over long-running or incorrect build executions.
When you cancel a running build, the build controller stops the build pod and updates the `BuildRun` status to indicate cancellation.
This operation is useful for stopping builds that encounter errors, consume excessive resources, or were started with incorrect configuration.
Canceled builds remain in the cluster until cleaned up by retention policies or manual deletion.

System results capture information generated during build execution.
After a build completes, the build controller populates the `status.output` and `status.conditions` fields with details such as the built image digest, source revision used, and build duration.
These results provide traceability and enable verification that builds used the expected source code and produced the correct image artifacts.
