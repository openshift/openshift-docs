= Federation endpoint not accessible

[role="_abstract"]
Troubleshoot and resolve connectivity issues with the SPIRE federation endpoint by verifying routes, service status, and SPIRE server logs in OpenShift.

The following error logs and timeout behaviors were observed during the connection attempt to the federation service:

* `curl -v https://federation.apps.example.com` fails

* Script times out waiting for federation endpoint

.Procedure

. Check if the federation route exists by running the following command:
+
[source,terminal]
----
$ oc get route spire-server-federation -n zero-trust-workload-identity-manager
----

. Verify the status of the route by running the following command:
+
[source,terminal]
----
$ oc describe route spire-server-federation -n zero-trust-workload-identity-manager
----

. Check the status and logs of the SPIRE server pod by running the following commands:
+
[source,terminal]
----
$ oc get pod spire-server-0 -n zero-trust-workload-identity-manager
----
+
[source,terminal]
----
$ oc logs spire-server-0 -c spire-server -n zero-trust-workload-identity-manager
----

. Confirm the SPIRE service is running by running the following command:
+
[source,terminal]
----
$ oc logs spire-server-0 -c spire-server -n zero-trust-workload-identity-manager
----
