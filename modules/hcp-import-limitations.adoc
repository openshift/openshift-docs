= Limitations of managing imported hosted clusters

Hosted clusters are automatically imported into the local {mce}, unlike a standalone {product-title} or third party clusters. Hosted clusters run some of their agents in the _hosted mode_ so that the agents do not use the resources of your cluster. If you choose to automatically import hosted clusters, you can update hosted clusters by using the `HostedCluster` resource on the management cluster.

You can import hosted clusters into a location other than the local {mce-short} by using the {mce-short} or {rh-rhacm-first}. You must use the following options to update hosted clusters:

* The console or the command-line interface of the management cluster, if you use the {mce} to import a hosted cluster.
* A policy, if you use the {rh-rhacm} hub cluster to import a hosted cluster.

The {mce} console can directly access the `HostedCluster` and `NodePool` resources. {rh-rhacm} agent runs inside the managed cluster. In {hcp}, the control plane handles the updates, which is present outside the hosted clusters. Unlike hosted clusters, you can update standalone {product-title} clusters by using the `ClusterVersion` resource, which is present inside an {product-title} cluster.

{rh-rhacm} manages the {mce} cluster. Therefore, as a workaround for managing imported hosted clusters limitations, you can use the `Policy` and `ManifestWork` objects to access and update the `HostedCluster` and `NodePool` resources.
