= Limitations

[role="_abstract"]
Autonomous agentic troubleshooting has the following limitations that you must consider before enabling the feature.

* *Risk of incorrect diagnoses*: As with any AI-driven system, the agent can produce hallucinated diagnoses or propose incorrect remediation actions. Always review the proposed diagnosis and remediation details before approving execution.

* *Model-dependent accuracy*: The quality of diagnoses and remediation proposals depends on the large language model (LLM) provider that you configure. Performance varies by LLM provider.

* *No guaranteed rollback*: If you stop an execution while it is in progress, rollback is not guaranteed. A partially executed remediation can leave your cluster in an inconsistent state. See xref:ols-rollback-and-emergency-stop_{context}[Rollback and emergency stop] for guidance.

* *No post-execution monitoring*: The agent does not monitor cluster state after executing a remediation. You must verify the cluster health independently after any remediation is executed.

[IMPORTANT]
====
Review all proposed remediation actions carefully before approving execution. The AI agent proposes actions based on its analysis, but the responsibility for approving and executing changes to your cluster remains with you.
====
