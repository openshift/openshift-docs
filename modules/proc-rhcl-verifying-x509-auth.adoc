= Verifying X.509 authentication

[role="_abstract"]
To test that your X.509 authentication is working as intended, you can run a test scenario and check for various certificate issues to ensure that connections are rejected.

.Prerequisites

* You are logged into {ocp} as a cluster administrator.
* You installed {prodname} using Istio as your `Gateway` object controller.
* You created your CAs and X.509-formatted certificates.
* You created your `Gateway` object.
* You created your route.
* You installed the `httpbin` sample application.

.Procedure

. Start the test scenario by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ GATEWAY_IP=$(oc get gateway mtls-gateway -n gateway-system -o jsonpath='{.status.addresses[0].value}')
----

. Check for a valid certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.$GATEWAY_IP.nip.io/get \
  --cert /tmp/client.crt \
  --key /tmp/client.key
----
+
.Expected output
[source,text]
----
HTTP 200
----

. Verify that the TLS handshake fails when there is no certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.$GATEWAY_IP.nip.io/get
----
+
.Expected output
[source,text]
----
TLS handshake fails
----
//Q: what is the error?

. Verify that the TLS handshake fails when there is an untrusted certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.$GATEWAY_IP.nip.io/get \
  --cert /tmp/untrusted.crt \
  --key /tmp/untrusted.key
----
//Q: probably would not load key into cluster?
+
.Expected output
[source,text]
----
TLS handshake fails
----
//Q: what is the error?

. Verify connection failure when there is a valid certificate, but wrong attributes by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.$GATEWAY_IP.nip.io/get \
  --cert /tmp/unauthorized-client.crt \
  --key /tmp/unauthorized-client.key
----
//Q: probably would not load key into cluster?
+
.Expected output
[source,text]
----
HTTP 403
----
