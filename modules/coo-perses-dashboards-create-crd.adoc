= Create a dashboard by using a custom resource

[role="_abstract"]
You can create Perses dashboards by defining `PersesDashboard` custom resources by enabling _dashboard-as-code_ practices and GitOps workflows.

.Prerequisites

* You have enabled the monitoring plugin.
* You have at least one datasource exists in your target namespace.
* You have edit permissions in the target namespace.
* You are familiar with YAML syntax and Kubernetes custom resources.

.Procedure

. Create a `PersesDashboard` custom resource file. The following example creates a dashboard with a single time series panel:
+
[source,yaml]
----
apiVersion: perses.dev/v1alpha1
kind: PersesDashboard
metadata:
  name: application-overview
  namespace: <target_namespace>
  annotations:
    perses.dev/tags: "application,overview"
spec:
  display:
    name: "Application Overview"
    description: "Overview of application metrics"
  variables:
    - kind: ListVariable
      spec:
        name: interval
        allowAllValue: false
        allowMultiple: false
        plugin:
          kind: StaticListVariable
          spec:
            values:
              - value: 1m
                label: 1m
              - value: 5m
                label: 5m
        defaultValue: 1m
  panels:
    requestRate:
      kind: Panel
      spec:
        display:
          name: "HTTP Request Rate"
        plugin:
          kind: TimeSeriesChart
          spec: {}
        queries:
          - kind: TimeSeriesQuery
            spec:
              plugin:
                kind: PrometheusTimeSeriesQuery
                spec:
                  query: rate(http_requests_total[$interval])
  layouts:
    - kind: Grid
      spec:
        display:
          title: "Application Metrics"
          collapse:
            open: true
        items:
          - x: 0
            y: 0
            width: 24
            height: 6
            content:
              $ref: "#/spec/panels/requestRate"
  duration: 1h
----
+
where:
+
`<target_namespace>`:: Replace with the namespace where you want to create the dashboard.

. Apply the custom resource by running the following command:
+
[source,terminal]
----
$ oc apply -f application-overview-dashboard.yaml
----

.Verification

* Verify that the dashboard was created successfully by running the following command:
+
[source,terminal]
----
$ oc get persesdashboards -n <target_namespace>
----
+
You should see an output similar to the following example:
+
[source,terminal]
----
NAME                   AGE
application-overview   1m
----

* In the {product-title} web console, navigate to *Observe* -> *Dashboards (Perses)*, select your namespace, and confirm that the dashboard appears and displays data.
