= Removing {external-secrets-operator} resources

Once you have uninstalled the {external-secrets-operator}, you have the option to eliminate its associated resources from your cluster.

.Prerequisites

* You have access to the cluster with `cluster-admin` privileges.
* You have access to the {product-title} web console.

.Procedure

. Log in to the {product-title} web console.

. Remove the deployments of the external-secrets components, such as `external-secrets` in the `external-secrets` namespace.

.. Click the *Project* drop-down menu to see a list of all available projects, and select the *external-secrets* project.

.. Navigate to *Workloads* -> *Deployments*.

.. Select the deployment that you want to delete.

.. Click the *Actions* drop-down menu, and select *Delete Deployment* to see a confirmation dialog box.

.. Click *Delete* to delete the deployment.

.. Alternatively, delete deployments of the external-secrets components in the `external-secrets` namespace by using the command-line interface (CLI).
+
[source,terminal]
----
$ oc delete deployment -n external-secrets -l app.kubernetes.io/instance=external-secrets
----
