= Acknowledging the y-stream release upgrade

When moving between y-stream releases you need to run a patch command to explicitly acknowledge the upgrade.
In the output of the `oc adm upgrade` command a URL is provided that shows the specific command to run.

.Procedure
. Acknowledge the y-stream release upgrade by patching the `openshift-config` resource.
For example, run the following command:
+
[source,terminal]
----
$ oc -n openshift-config patch cm admin-acks --patch '{"data":{"ack-4.15-kube-1.29-api-removals-in-4.16":"true"}}' --type=merge
----
+
.Example output
[source,terminal]
----
configmap/admin-acks patched
----
