= Build container images based on your application requirements

[role="_abstract"]
{builds-shortname} provides many build strategies to support different application development workflows.
Choose a build strategy based on your application source code format and team expertise.

You can build container images by using the following strategies:

* `Buildah` strategy: Use when you have a Dockerfile that defines your application's container image.
This strategy provides the most control over the build process and supports advanced Dockerfile features.

* `Source-to-Image` (S2I) strategy: Use when you want to build from application source code without writing a Dockerfile.
S2I combines your source code with a builder image that contains the runtime, dependencies, and build tools.

* Buildpacks strategy: Use when you want to build from application source code with automatic dependency detection.
Buildpacks detect your application framework and dependencies, then build and package the application automatically.

Each build strategy creates a `Build` resource that defines the build configuration, including the source location, build parameters, and output image location.
After creating the build, you run it by creating a `BuildRun` resource that executes the build steps and pushes the resulting container image to your target registry.
