= Protecting low-latency workloads

In high-performance Telco environments (such as RAN DU or 5G Core), operators need to perform troubleshooting or monitoring tasks for example running `oc exec`without compromising the deterministic performance of the primary application.

Prior to {product-title} {product-version}, executing these commands introduced a risk that the new process could be scheduled on any CPU core, causing micro-interruptions and latency spikes that violated the strict timing requirements of the primary workload.

The `ExecCPUAffinity capability allows you to execute these operations safely by isolating secondary processes. By default, this feature pins the "initial" affinity of any new command to the first CPU (the housekeeping or shared core) in the container's set. This ensures that the exclusive cores dedicated to your time-sensitive application remain undisturbed, preserving the integrity of your high-performance workload.
