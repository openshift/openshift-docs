= PlanPolicy custom resource parameters

[role="_abstract"]
This section provides a detailed reference for the configuration parameters available for the `PlanPolicy` custom resource:

.PlanPolicy custom resource configuration fields definitions table
[cols="1,2"]
|===
|Parameter|Description

|`spec.targetRef`
|A reference to an existing Gateway API resource. A `PlanPolicy` object can target a Gateway, HTTPRoute, or GRPCRoute. The target resource must be in the same namespace as the `PlanPolicy` object.

|`spec.plans`
|An ordered list of plan definitions. Plans are evaluated in sequence and the first plan whose predicate evaluates to `true` is applied. More specific predicates must appear before more general ones.

|`spec.plans.tier`
|A unique string identifier for the plan tier, for example `gold`, `silver`, or `bronze`.

|`spec.plans.predicate`
|Specifies a CEL expression evaluated against the authenticated request context. 

|`spec.plans.limits`
|Specifies the rate limiting configuration for the tier. 

|===
