= Creating a Kickstart for a {microshift-short} image mode for RHEL installation

You can use the Kickstart provided with {microshift-short} to provision an image mode for RHEL virtual machine.

.Prerequisites

* You set up the {microshift-short} Kickstart file.
* You set up the [Activation Keys](https://console.redhat.com/insights/connector/activation-keys) and organization ID to activate your Red Hat subscription.
* You have the information needed to set required and optional variables.

[IMPORTANT]
====
The subscription must include access to the `rhocp-4.x-for-rhel-9-$(uname -m)-rpms` and `fast-datapath-for-rhel-9-$(uname -m)-rpms` RPM repositories.
====

.Procedure

. Add the required `BOOTC_IMAGE_URL` variable and value to create an image mode for RHEL Kickstart file by using the following command:
+
[source,terminal]
----
$ export BOOTC_IMAGE_URL=quay.io/<myorg>/<mypath>/microshift-image:tag # <1>
----
<1> Replace _<myorg>_ and _<mypath>_ with your information.

* The `BOOTC_IMAGE_URL` variable contains a reference to the image that is installed with the `ostreecontainer` Kickstart command. If needed, you can specify a z-stream release other than the latest using the tag.

. Optional. Add variables and values for registry authentication and configuration by using the following commands:
+
.Example commands setting optional variables.
+
* The `AUTH_CONFIG` variable contents are copied to `/etc/ostree/auth.json` to authenticate access to the `BOOTC_IMAGE_URL` image.
+
[source,terminal]
----
$ export AUTH_CONFIG="$(cat ~/.quay-auth.json)"
----
+
* The `REGISTRY_CONFIG` contents are copied to `/etc/containers/registries.conf.d/999-microshift-registry.conf` to configure access to the registry containing the `BOOTC_IMAGE_URL` image.
+
[source,terminal]
----
$ export REGISTRY_CONFIG="$(cat ~/.quay-config.conf)"
----

. Run the following command to create the `kickstart.ks` file to be used during the virtual machine installation:
+
[source,terminal]
----
envsubst < \
    /usr/share/microshift/kickstart/kickstart-bootc.ks.template > \
    "${HOME}/kickstart.ks"
----
