= Application Management Across Namespaces

The Argo CD Agent architecture uses the *Applications in any namespace* feature to organize Applications per workload cluster on the control plane.

Each workload cluster corresponds to a unique namespace on the control plane, for example:

* `argocd-w1`
* `argocd-w2`
* `argocd-w3`

Applications for each cluster reside only within their respective namespaces.

Example configuration for the control plane Argo CD instance:
[source,yaml]
----
apiVersion: argoproj.io/v1beta1
kind: ArgoCD
metadata:
  name: argo-cd
spec:
  sourceNamespaces:
    - argocd-w1
    - argocd-w2
    - argocd-w3
----

This configuration enables the control plane Argo CD instance to reconcile and monitor applications across these namespaces while maintaining logical separation for each workload cluster.
