= Bundle fetching fails

[role="_abstract"]
If you are unable to retrieve the trust bundle from a cluster, use the following steps to isolate the problem.

.Procedure

. Vierify that the federation route is not missing or misconfigured by running the following command:
+
[source,terminal]
----
$ oc get route -n zero-trust-workload-identity-manager
----

. Test the network's connection by running the following command:
+
[source,terminal]
----
$ curl -vk https://federation.<APPS_DOMAIN>
----

. Check the `spire-server` pod logs by running the following command:
+
[source,terminal]
----
$ oc logs -n zero-trust-workload-identity-manager spire-server-0 -c spire-server --tail=50
----
