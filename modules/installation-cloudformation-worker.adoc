= CloudFormation template for worker machines

You can use the following CloudFormation template to deploy the worker machines that you need for your {product-title} cluster.

.CloudFormation template for worker machines
[%collapsible]
====
[source,yaml]
----
include::https://raw.githubusercontent.com/openshift/installer/release-4.18/upi/aws/cloudformation/06_cluster_worker_node.yaml[]
----
====
[IMPORTANT]
====
When you use the CloudFormation template for the master machines, it provisions all three master machines with a single stack. When you use the CloudFormation template to deploy the worker machines, however, you must create the number of stacks based on your requirements, that is, the number you defined in the `install-config.yaml` file. Each stack is provisioned one time only for each machine. To provision a new worker machine, you must change the stack name.
====

[source,terminal]
----
$ aws cloudformation create-stack --stack-name <name> <1>
     --template-body file://<template>.yaml \ <2>
     --parameters file://<parameters>.json <3>
----
<1> `<name>` is the name for the CloudFormation stack, such as `cluster-worker-1`. You need the name of this stack if you remove the cluster.
<2> `<template>` is the relative path to and name of the CloudFormation template YAML file that you saved.
<3> `<parameters>` is the relative path to and name of the CloudFormation
parameters JSON file.

.Example output

----
