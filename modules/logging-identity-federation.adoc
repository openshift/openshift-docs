= Identity Federation
With {logging} 5.9 on {product-title} 4.14 and higher, if you use the {product-title} web console to install, STS clusters are automatically detected. You will be prompted to create roles and supply the data required for the Loki Operator to create a `CredentialsRequest` object, which will then populate a secret.

For installations using the CLI, create a subscription object using the appropriate template for your storage provider.

.Azure sample subscription
[source,yaml]
----
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: loki-operator
  namespace: openshift-operators-redhat
spec:
  channel: "stable-5.9"
  installPlanApproval: Manual
  name: loki-operator
  source: qe-app-registry
  sourceNamespace: openshift-marketplace
  config:
    env:
      - name: CLIENT_ID
        value: <your_client_id>
      - name: TENANT_ID
        value: <your_tenant_id>
      - name: SUBSCRIPTION_ID
        value: <your_subscription_id>
      - name: REGION
        value: <your_region>
----

.AWS sample subscription
[source,yaml]
----
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: loki-operator
  namespace: openshift-operators-redhat
spec:
  channel: "stable-5.9"
  installPlanApproval: Manual
  name: loki-operator
  source: qe-app-registry
  sourceNamespace: openshift-marketplace
  config:
    env:
    - name: ROLE_ARN
      value: "<role_ARN>"
----
