= Configuring {microshift-short} Observability

You can collect information from a MicroShift system using an optional MicroShift observability service. This service transmits data to remote endpoints for monitoring and analysis. This service is disabled by default. See [Setting MicroShift Observability configurations] for information about configurations.

[IMPORTANT]
====
You must specify an OpenTelemetry Protocol (OTLP)-compatible endpoint for each configuration, such as Prometheus, before starting {microshift-short}. If an endpoint is not specified, the observability service will not begin.
====

.Procedure

. Update the `/etc/microshift/observability/opentelemetry-collector.yaml` file to specify an OTLP-compatible endpoint with the following information. The endpoint must link to an IP address or host name, and port number of an OTLP service.
+
.OTLP-compatible endpoint configuration
[source,yaml]
----
exporters:
  otlp:
    sending_queue:
      storage: file_storage
    endpoint: ${env:K8S_NODE_NAME}:4317 <1>
    tls:
      insecure: true
(...)
service:
(...)
  telemetry:
    metrics:
      readers:
        - periodic:
            exporter:
              otlp:
                protocol: http/protobuf
                endpoint: http://${env:K8S_NODE_NAME}:4318
----  
<1> Replace `${env:K8S_NODE_NAME}` with the IP address or host name of the remote back end. This IP address will resolve to the local node's host name. An unreachable endpoint is reported in the {microshift-short} service logs.    

. Restart the {microshift-short} observability service. A restart is required before the configuration updates are applied.
