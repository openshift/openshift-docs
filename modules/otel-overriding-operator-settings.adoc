= Overriding the Operator settings

[role="_abstract"]
Since RHOSDT 3.9 the OpenTelemetry Operator itself can now be configured and override the default settings via the OLM Subscription CRD.

.Prerequisites

* RHOSDT(OTEL or Operator???) 3.9 or later. SEE THE SCREENSHOT.

.Procedure

* Do what?
+
.Example (allowed example syntax?)
[source,yaml]
----
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: opentelemetry-operator
spec:
  channel: stable
  name: opentelemetry-operator
  source: operatorhubio-catalog
  sourceNamespace: olm
  config:
    env:
  - name: OPENSHIFT_CREATE_DASHBOARD <1>
    value: "true"
  - name: ENABLE_CR_METRICS <2>
    value: "true"
  - name: CREATE_SM_OPERATOR_METRICS <3>
    value: "true"
  - name: FEATURE_GATES <4>
    value: operator.networkpolicy,operand.networkpolicy
----
<1> OPENSHIFT_CREATE_DASHBOARD (default: "true"): Creates OpenShift monitoring dashboards for visualizing OpenTelemetryCollector instance metrics and health.
<2> ENABLE_CR_METRICS (default: "true"): Enables exposing Custom Resource metrics for OpenTelemetryCollector, Instrumentation, and other CRDs managed by the operator.
<3> CREATE_SM_OPERATOR_METRICS (default: "true"): Automatically creates a ServiceMonitor resource to scrape the operator's own internal metrics using Prometheus Operator.
<4> FEATURE_GATES (default: "operator.networkpolicy,operand.networkpolicy"): Enables experimental features; OpenShift enables NetworkPolicy creation for both the operator itself and managed collector operands.

CHECK
The values can be configured on the OpenShift Console itself:
https://openshift.github.io/openshift-origin-design/designs/administrator/olm/env-toleration-config-on-olm-subscription/
