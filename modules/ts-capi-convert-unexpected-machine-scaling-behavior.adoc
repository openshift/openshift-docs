= Unexpected machine scaling behavior

On clusters that support converting resources between the Machine API and the Cluster API, users might experience unexpected behavior when scaling the number of compute machines.

#It's not clear to me how much of this behavior is resolved for 4.19, so I am not able to complete this (if it is even needed any more).#

Cause::

When you configure a Machine API compute machine set to deploy Cluster API compute machines, it deploys unprovisioned Machine API compute machines that use the Cluster API as authoritative.
The two-way synchronization controller then creates corresponding Cluster API machines.
+
When you configure a compute machine set to deploy machines that use a different authoritative API, scaling up the
number of replicas in the machine set creates enough new machines to satisfy the replica requirement.
The compute machines that it creates use a different authoritative API than the machine set that deploys them.
Any existing machines that use the same authoritative API as the machine set also count towards satisfying the replica requirement.
+
The following example changes a the value of `spec.template.spec.authoritativeAPI` set to `ClusterAPI` in a Machine API compute machine set and increases the value of the `replicas` parameter from `1` to `2`.
This configuration change causes the machine set to create one Cluster API machine to satisfy the replica requirement.
+
.Example configuration change
[source,yaml]
----
apiVersion: machine.openshift.io/v1beta1
kind: MachineSet
[...]
spec:
  authoritativeAPI: MachineAPI <1>
  replicas: 2 <2>
  [...]
  template:
    [...]
    spec:
      authoritativeAPI: ClusterAPI <3>
status:
  authoritativeAPI: MachineAPI <4>
  [...]
----
<1> The original value for the Machine API compute machine set.
This machine set uses the Machine API as its authoritative API.
<2> The updated value for the number of replicas.
This value originated as `1`.
<3> The updated value for the authoritative API for any new compute machines.
This value originated as `MachineAPI`.
Specifying `ClusterAPI` configures this compute machine set to deploy Cluster API compute machines.
<4> The current value for the Machine API compute machine set.
This machine set maintains the Machine API as its authoritative API, so this value matches the value of the `spec.authoritativeAPI` parameter.
