= Configure MetalLB adddress pool for VLAN

Once the VLAN interface is configured, you can set up MetalLB to assign IPs from a range that corresponds to your VLAN.

.Prerequisites

* Install the OpenShift CLI (oc).

* Separate VLAN configured.

* Log in as a user with `cluster-admin` privileges.

.Procedure

. Create a file, such as `ipaddresspool-vlan.yaml`, with content such as the following example:
+
[source,yaml]
----
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  namespace: metallb-system
  name: doc-example-vlan
spec:
  addresses:
  - 192.168.100.1-192.168.100.254 <1>
  - 203.0.113.65-203.0.113.75
----
<1> This IP range must match the subnet assigned to the VLAN on your network.

. Apply the configuration for the IP address pool:
+
[source,terminal]
----
$ oc apply -f ipaddresspool-vlan.yaml
----

. To support this configuration you need to set the `gatewayConfig` `ipForwarding` to `Global`. 

.. Run the following command to edit the network configuration custom resource (CR):
+
[source,terminal]
----
$ oc edit network.config.openshift/cluster
----

.. Within the editor, update the `spec.defaultNetwork.ovnKubernetesConfig` section to include the `gatewayConfig.ipForwarding` set to `Global`. It should look something like this:
+
[source,yaml]
----
...
spec:
  clusterNetwork:
    - cidr: 10.128.0.0/14
      hostPrefix: 23
  defaultNetwork:
    type: OVNKubernetes
    ovnKubernetesConfig:
      gatewayConfig:
        ipForwarding: Global
...
----
