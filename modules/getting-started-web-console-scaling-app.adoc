= Scaling up the application

In Kubernetes, a `Deployment` object defines how an application deploys. In most cases, users use `Pod`, `Service`, `ReplicaSet`, and `Deployment` resources together. In most cases, {product-title} creates the resources for you.

When you deploy the `parksmap` image, a deployment resource is created. In this example, only one pod is deployed. You might want to scale up your application to keep up with user demand or to ensure that your application is always running even if one pod is down.

The following procedure scales the `parksmap` deployment to use two instances.

.Prerequisites

// TODO: Developer perspective going away - check this
* You are in the *Developer* perspective of the web console.
* You have deployed the `parksmap` front-end application.

.Procedure

. In the *Topology* view, click the `parksmap` application.
. Click the *Details* tab.
. Use the up arrow to scale the pod to two instances.
