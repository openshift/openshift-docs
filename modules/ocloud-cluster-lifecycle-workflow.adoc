= O-Cloud Manager cluster lifecycle workflow

[role="_abstract"]
The O-Cloud Manager manages bare-metal clusters from initial environment preparation through ongoing operations. All post-provisioning operations are driven through updates to the `ProvisioningRequest` custom resource.

Prepare:: Install the O-Cloud Manager Operator on the hub cluster, configure the GitOps repository, and define `ClusterTemplate` resources with cluster installation defaults, configuration policies, and hardware management settings. Optionally, define `HardwareProfile` resources for firmware and BIOS management.

Onboard:: Register bare-metal servers in resource pools by creating `BareMetalHost` resources with resource selector labels that identify server type, location, and hardware characteristics. The Bare Metal Operator connects to each host's BMC, inspects the physical hardware, and populates a `HardwareData` resource used for resource selection during provisioning.

Provision:: Create a `ProvisioningRequest` that references a validated `ClusterTemplate` and supplies site-specific parameters. The Operator allocates matching hardware from the resource pool, applies any specified firmware and BIOS settings, renders a `ClusterInstance` resource, and orchestrates the cluster installation through the SiteConfig Operator. For single-node {product-title} clusters, image-based installation (IBI) is supported as an alternative provisioning method. After installation, configuration policies are applied and monitored for compliance.

Operate:: Update the `ProvisioningRequest` to drive Day-2 operations on the running cluster:

* Modify policy parameters to apply configuration changes.
* Adjust node group sizes to scale worker nodes in or out.
* Update the referenced release version to trigger z-stream, y-stream, or EUS-to-EUS upgrades through the {cgu-operator-full}.
* For single-node {product-title} clusters, image-based upgrades (IBU) are supported as an alternative upgrade path.
