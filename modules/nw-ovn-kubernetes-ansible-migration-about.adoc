= Using the Ansible playbook to migrate to the OVN-Kubernetes network plugin

As a cluster administrator, you can use an Ansible playbook to migrate from the OpenShift SDN Container Network Interface (CNI) network plugin to the OVN-Kubernetes plugin for your cluster.

The Ansible playbook includes the following set of playbooks that each represent a stage in the migration process:

. Prechecks
. Migrate
. Reboot
. Post-migration

After you download the `openshift` Ansible collection and save the collection to your preferred path, you can go to `roles.<playbook_name>.tasks` subdirectory, ensuring that you replace `<playbook_name>` with the name of a playbook, to view the list of tasks associated with that playbook.

If a playbook fails, check the playbook output for the source of the error. Resolve the error and then rerun the Ansible playbook.

[IMPORTANT]
====
If you run a playbook as a non-cluster administrator, the playbook fails.
====

.Prerequisites

Before you run a playbook, you must manually complete the following prerequisites tasks: 

* You set a bastion server.
* You installed `ansible`, `jq`, `pip3`, and `python3` packages.
** To install the `ansible` package, see "Getting started with playbook" in the Additional resources section.
* You set the `KUBECONFIG` environment variable.
* Optional: See "Migrating to the OVN-Kubernetes network plugin by using the offline migration method" for a list of other prerequisites. Although the Prechecks playbook checks these prerequsities, you can meet them in advance of the playbook check.
* You logged in to the {hybrid-console} and opened the link:https://console.redhat.com/ansible/ansible-dashboard[Ansible Automation Platfor] web console.
* From the *Automation Hub > Token management* page of the Ansible Automation Platform web console, you created an API token and configured Red{nbsp}Hat Automation Hub as the primary source for content. See link:https://docs.redhat.com/en/documentation/red_hat_ansible_automation_platform/1.2/html/getting_started_with_red_hat_ansible_automation_hub/proc-create-api-token[Creating the Red{nbsp}Hat Automation Hub API token].
// System dependencies might be a complex prereq for customers if they have a complex system configuration.

.Procedure

. Download the `openshift` Ansible collection from the link:https://console.redhat.com/ansible/automation-hub[Ansible automation hub] web console page.

////
. Enter the following command in the `ansible-galaxy` command-line interface to install the downloaded `openshift` Ansible collection file:
+
[source,terminal]
----
$ ansible-galaxy collection install openshift
----
////

. In the directory where you downloaded the `openshift` Ansible collection, run the following `make` command to install the collection:
+
[source,terminal]
----
$ make install
----

// anislbe-galaxy up for discussion
. Verify that `openshift` Ansible collection was successfully installed by entering the following command:
+
[source,terminal]
----
$ ansible-galaxy collection list openshift
----
+
.Example output
+
[source,terminal]
----
# /home/<user>/.ansible/collections/ansible_collections <1>
Collection               Version
------------------------ -------
openshift                4.0.1 
----
<1> Where `<user>` is the user home directory on your system.

. Optional: You can configure the `playbook/playbook-migration.yml` file to meet your needs. The following example `playbook-migration.yml` file shows specific parameters and their value:
+
[source,terminal]
----
# ...
    migration_disable_auto_migration: true <1>
    migration_egress_ip: false
    migration_egress_firewall: false
    migration_multicast: false
    migration_mtu: 1400 <2>
    migration_geneve_port: 6081
    migration_ipv4_subnet: "100.64.0.0/16" <3>
    migration_interface_name: eth0 <4>
# ...
----
<1> Disables the auto-migration of OpenShift SDN CNI plug-in features to the OVN-Kubernetes plugin. If you need to enable auto-migration of these features, set the parameter to `false`.
<2> Set a specific maximum transmission unit (MTU) that is applied to your cluster network after the migration process.
<3> Set an IPv4 address range for internal use by OVN-Kubernetes. The default value for the OVN-Kubernetes plug-in is `100.64.0.0/16`.
<4> If you use a `NodeNetworkConfigurationPolicy` (NNCP) object to specify the primary network for your network, set the interface name in the `migration_interface_name` parameter.

. To start each Ansible playbook in a sequenced pattern, enter the following command in the `oc` CLI:
+
[source,terminal]
----
$ ansible-playbook -v playbook/playbook-migration.yml
----
+
The command runs the playbooks in the Prechecks, Migrate, Reboot, and Post-migration playbooks sequence. The output of each task completed by the playbook must state `ok` or `changed`. The final Post-migration playbook output, summarizes the results under a `PLAY RECAP` heading as shown in the following example:
+
.Example output
[source,terminal]
----
localhost : ok=38 changed=11 unreachable=0 faild=0 skipped=9 rescued=0 ignored=0
----

.Verification

* To confirm that the network plugin is OVN-Kubernetes, enter the following command.  The value of `status.networkType` must be `OVNKubernetes`.
+
[source,terminal]
----
$ oc get network.config/cluster -o jsonpath='{.status.networkType}{"\n"}'
----
