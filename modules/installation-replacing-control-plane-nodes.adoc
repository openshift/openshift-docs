= Replacing control plane nodes in a two-node OpenShift cluster

You can replace a failed control plane node in a two-node cluster. The replacement node must use the same host name and IP address as the failed node.

.Prerequisites

* You have a functioning survivor control plane node.  
* You have verified that either the machine is not running or the node is not ready.
* You have access to the cluster as a user with the `cluster-admin` role.
* You know the host name and IP address of the failed node.  
* You have taken an etcd backup.
+
[IMPORTANT]
====
You must take an etcd backup before performing this procedure so that your cluster can be restored if you encounter any issues.
====
* You have taken a back up of `BareMetalHost` (BMH) and machine objects for the failed node, if applicable. The BMH resource represents the physical host, while the Machine resource represents the node in the cluster.

.Procedure

. Check the quorum state:
.. If both nodes lost power, restore quorum manually on the survivor node by running the following command:
+
[source,terminal]
----
$ sudo pcs quorum expected-votes 1
$ sudo pcs cluster start
----
.. If only one node failed, verify that etcd is running on the survivor node by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-etcd -o wide
----

. Delete etcd secrets for the failed node by running the following command:
+
[source,terminal]
----
$ oc project openshift-etcd
$ oc delete secret etcd-peer-<node_name>
$ oc delete secret etcd-serving-<node_name>
$ oc delete secret etcd-serving-metrics-<node_name>
----

. Delete resources for the failed node:

.. Delete the BMH object by running the following command:
+
[source,terminal]
----
$ oc project openshift-machine-api
$ oc delete bmh/<node_name>
----
.. Delete the machine object that corresponds to the failed node by running the following command:
+
[source,terminal]
----
$ oc delete machine/<machine_name>
----

. Recreate the failed host by using the same name and IP address:

.. Redefine the virtual machine or provision replacement hardware by using the saved domain XML or hardware profile:
+
[source,terminal]
----
$ TBD
----
.. Create the Baseboard Management Controller (BMC) secret for the replacement host by running the following command: 
+
[source,terminal]
----
$ oc create -f backup/bmc-secret.yaml
----
.. Update the BMH manifest with the correct MAC address and UUID.

.. Create the BMH object by running the following command:
+
[source,terminal]
----
$ oc create -f backup/bmh.yaml
----

.. Create the machine object for the replacement node by running the following command:
+
[source,terminal]
----
$ oc create -f backup/machine.yaml
----

. Approve pending Certificate Signing Requests (CSRs) so the replacement node can join the cluster by running the following command:
+
[source,terminal]
----
$ oc get csr -ojson | jq -r '.items[] | select(.status == {} ) | .metadata.name' | \
  xargs oc adm certificate approve
----

.. Place the certificates on the replacement node by running the following command:
+

----

----

.. Force a redeployment of the etcd operator by running the following command:
+
[source,terminal]
----
$ oc patch etcd cluster -p='{"spec": {"forceRedeploymentReason": "recovery-'"$(date --rfc-3339=ns)"'"}}' --type=merge
----

. Rejoin the replacement node to the pacemaker cluster by running the following command:
+
[source,terminal]
----
$ sudo pcs cluster node remove <NODE_NAME>
$ sudo pcs cluster node add <NODE_NAME> addr=<NODE_IP> --start --enable
----

. Delete stale jobs for the failed node by running the following command:
+
[source,terminal]
----
$ oc project openshift-etcd
$ oc delete job tnf-auth-job-<node_name>
$ oc delete job tnf-after-setup-job-<node_name>
----

.Verification

* Verify that the replacement node is in the `Ready` state by running the following command:
+
[source,terminal]
----
$ oc get nodes
----

* Verify that etcd members are healthy by running the following command:
+
[source,terminal]
----
$ oc rsh -n openshift-etcd <etcd-pod> etcdctl member list -w table
----

* Verify that pacemaker lists both nodes by running the following command:
+
[source,terminal]
----
$ sudo pcs status
----
