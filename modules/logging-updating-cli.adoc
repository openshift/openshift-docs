= Updating logging using the CLI

If the approval strategy in the subscription is set to Automatic, the update process initiates as soon as a new Operator version is available in the selected channel. If the approval strategy is set to Manual, you must manually approve pending updates.

You can use the {product-title} CLI to change the Red Hat OpenShift Logging Operator subscription channel.

include::snippets/logging-stable-updates-snip.adoc[]

Prerequisites

* The {product-title} version is 4.9 or later.

* The Logging status is healthy:
+
** All pods are `ready`.

.Procedure

. Find the name of your `Subscription` object YAML file.
+
[source,terminal]
----
$ oc get subscription | grep logging
----
+
. Edit your `Subscription` object YAML file.
+
[source, terminal]
----
$ oc edit <your_subscription_file>
----
