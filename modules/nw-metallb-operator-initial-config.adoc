= Starting MetalLB on your cluster

[role="_abstract"]
To start MetalLB on your cluster after installing the MetalLB Operator in {product-title}, you create a single MetalLB custom resource.

.Prerequisites

* Install the OpenShift CLI (`oc`).
+
Verify: `oc version`

* Log in as a user with `cluster-admin` privileges.
+
Verify: `oc auth can-i create namespace --all-namespaces`

* Install the MetalLB Operator.
+
Verify: `oc get csv -n metallb-system -o custom-columns=Name:.metadata.name,Phase:.status.phase`

.Procedure

. Create a single instance of a MetalLB custom resource:
+
.MetalLB instance parameters
[cols="2,1,3",options="header"]
|===
| Parameter | Value | Description

| `name`
| `metallb`
| The name of the MetalLB instance. Only one instance is supported per cluster.

| `namespace`
| `metallb-system`
| The namespace where the Operator is installed. Use `openshift-operators` if you installed the Operator by using the web console.
|===
+
[source,terminal]
----
$ cat << EOF | oc apply -f -
apiVersion: metallb.io/v1beta1
kind: MetalLB
metadata:
  name: metallb
  namespace: metallb-system
EOF
----

.Verification

[NOTE]
====
The controller deployment and speaker daemon set typically reach a ready state within 60 seconds of creating the `MetalLB` resource.
====

Confirm that the deployment for the MetalLB controller and the daemon set for the MetalLB speaker are running.

. Verify that the controller deployment is available:
+
[source,terminal]
----
$ oc wait deployment/controller -n metallb-system --for=condition=Available --timeout=60s
----

. Verify that the deployment for the controller is running:
+
[source,terminal]
----
$ oc get deployment -n metallb-system controller
----
+
.Example output
[source,terminal]
----
NAME         READY   UP-TO-DATE   AVAILABLE   AGE
controller   1/1     1            1           11m
----

. Verify that the speaker daemon set is running on all nodes:
+
[source,terminal]
----
$ oc get daemonset -n metallb-system speaker
----
+
.Example output
[source,terminal]
----
NAME      DESIRED   CURRENT   READY   UP-TO-DATE   AVAILABLE   NODE SELECTOR            AGE
speaker   6         6         6       6            6           kubernetes.io/os=linux   18m
----
+
The `READY` count must match the `DESIRED` count. The number of speaker pods equals the number of nodes in your cluster.

.Troubleshooting
[cols="2,2,2",options="header"]
|===
| Symptom | Likely cause | Resolution

| `no matches for kind "MetalLB"`
| Operator CRDs are not yet registered
| Wait for the Operator CSV to reach the `Succeeded` phase before creating the `MetalLB` instance.

| Controller pod in `Pending` state
| Insufficient cluster resources or scheduling constraints
| Check pod events: `oc describe pod -n metallb-system -l component=controller`.

| Speaker pods in `CrashLoopBackOff`
| Node kernel does not support required networking features
| Check speaker pod logs: `oc logs -n metallb-system -l component=speaker`.

| Speaker `READY` count does not match `DESIRED`
| Some nodes are not schedulable or have taints
| Check node status: `oc get nodes` and verify no unexpected taints or cordons.
|===
