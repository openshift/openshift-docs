= Handling YAML Parsing Errors in {pac}

[role="_abstract"]

{pac} can detect and report errors in the YAML definition of a `PipelineRun`. When a YAML formatting issue is encountered in the repository, the system logs the error in the user namespace events log and the {pac} controller log.

These errors do not halt the execution of valid pipeline runs because the system continues to process correctly formatted and matched pipeline runs, preventing a single formatting problem from disrupting the entire automation workflow.

When a pull request (PR) triggers the event, {pac} creates a comment on the PR that describes the error. If the PR is updated and new YAML errors are detected, the comment is updated with the latest information. If no new errors are found, the existing comment remains unchanged and is not deleted.
