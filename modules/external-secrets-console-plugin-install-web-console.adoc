= Installing the {external-secrets-console-plugin}

[role="_abstract"]
You can use the web console to install the {external-secrets-console-plugin}.

.Prerequisites

* You have access to the cluster with `cluster-admin` privileges.

* You have access to the {product-title} web console.

.Procedure

. Log in to the {product-title} web console.

. Navigate to *Ecosystem* -> *Software Catalog*.

. Enter *External Secrets Management Console* into the filter box.

. Select *External Secrets Management Console*.

. Select the {external-secrets-console-plugin} version from the *Version* drop-down list, and click *Install*.

. On the *Install Operator* page:

.. Update the *Update channel*, if necessary. The channel defaults to *tech-preview-v1*.

.. Select an *Update approval* strategy.
+
* The *Automatic* strategy allows the Operator Lifecycle Manager (OLM) to automatically update the Operator when a new version is available.
+
* The *Manual* strategy requires a user with appropriate credentials to approve the Operator update.

.. Click *Install*.

.Verification

. Verify that *{external-secrets-console-plugin}* is available under *Plugins*.

. Verify that all the installed secrets management Operators are listed when you click *{external-secrets-console-plugin}*.
