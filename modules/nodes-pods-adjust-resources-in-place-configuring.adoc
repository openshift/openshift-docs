= Configuring in-place pod resizing

In-place pod resize allows you to changing the CPU and memory resources for containers within a running pod restarting or re-creating the pod.

.Procedure

. Create a pod spec with a resize policy:

.. Create a YAML file similar to the following example:
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: resize-pod
spec:
  containers:
  - name: pause
    image: registry.k8s.io/pause:3.8
    resizePolicy: <1>
    - resourceName: cpu
      restartPolicy: NotRequired
    - resourceName: memory
      restartPolicy: RestartContainer
    resources: <2>
      limits:
        cpu: "700m"
        memory: "200Mi"
      requests:
        cpu: "700m"
        memory: "200Mi"
# ...
----
<1> Specifies a resize policy. For CPU and/or memory resources specify one of the following values:
+
* `NotRequired`: Apply any resource changes without restarting the pod. This is the default.
* `RestartContainer`: Apply any resource changes and restart the pod.
<2> Specifies CPU and/or memory resources and limits. For more information, see "Understanding resource requests and limits".

.. Create the `Pod` object by running the following command:
+
[source,terminal]
----
$ oc create -f <file_name>.yaml
----

. Optional: After the pod is created, modify the CPU resource and limits for the pod by using the following command:
+
[source,terminal]
----
$ oc edit pod <pod_name>  --subresource resize
----
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: resize-pod
spec:
# ...
    resources: <1>
      limits:
        cpu: "800m"
        memory: "200Mi"
      requests:
        cpu: "800m"
        memory: "200Mi"
# ...
----
<1> For example, change the CPU resources to `800m`.
+
After the change is applied, the pod is updated without a restart because in this example, the resize policy for CPU is `NotRequired`.
+
[source,terminal]
----
$oc get pods
----
+
.Example output
[source,terminal]
----
NAME                         READY   STATUS    RESTARTS   AGE
resize-pod                   1/1     Running   0          3s
----

. Optional: After the pod is created, modify the memory resource and limits for the pod by using the following command:
+
[source,terminal]
----
$ oc edit pod <pod_name>  --subresource resize
----
+
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: resize-pod
spec:
# ...
    resources: <1>
      limits:
        cpu: "800m"
        memory: "300Mi"
      requests:
        cpu: "800m"
        memory: "300Mi"
# ...
----
<1> For example, change the memory resources to `300Mi`.
+
After the change is applied, the pod is restarted because in this example, the resize policy for memory is `RestartContainer`.
+
[source,terminal]
----
$oc get pods
----
+
.Example output
[source,terminal]
----
NAME                         READY   STATUS    RESTARTS   AGE
resize-pod                   1/1     Running   1          5s
----

.Verification

. Check that the modified resources are applied to the container in the pod by using a command similar to the following:
+
[source,terminal]
----
apiVersion: v1
kind: Pod
metadata:
  name: resize-pod
# ...
spec:
  containers:
# ...
    resources: <1>
      limits:
        cpu: 800m
        memory: 300Mi
      requests:
        cpu: 800m
        memory: 300Mi
# ...
status:
# ...
 containerStatuses:
    resources: <2>
      limits:
        cpu: 700m
        memory: 200Mi
      requests:
        cpu: 700m
        memory: 200Mi
# ...
----
<1> Shows the resources that were requested by the changes to the pod spec. 
<2> Shows the actual resources allocated to the container on the pod.
