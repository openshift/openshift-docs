= Configuring security context for event listeners

You can configure a custom security context directly in your `EventListener` definition to ensure that containers run as a non-root user and with restricted access to the file system. 

This configuration improves the security of your pipeline and helps ensure compliance with {OCP} security context constraints (SCCs).

.Procedure

. In your `EventListener` definition, add a `securityContext` field under the `resources.kubernetesResource.spec.template.spec` section.

. Set `runAsNonRoot: true`.

. Add a container-level `securityContext` field and set `readOnlyRootFilesystem: true`.

.Example `EventListener` definition with configured security context
[source,yaml,subs="attributes+"]
----
apiVersion: triggers.tekton.dev/v1beta1
kind: EventListener
metadata:
#...
spec:
  serviceAccountName: tekton-triggers-sa
  resources:
    kubernetesResource:
      spec:
        template:
          spec:
            securityContext:
              runAsNonRoot: true
            containers:
              - resources:
                  requests:
                    memory: "64Mi"
                    cpu: "250m"
                  limits:
                    memory: "128Mi"
                    cpu: "500m"
                securityContext:
                  readOnlyRootFilesystem: true
#...
----
