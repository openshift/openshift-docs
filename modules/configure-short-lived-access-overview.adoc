= Short-lived access overview

[role="_abstract"]
You can configure short-lived access by exchanging OpenID Connect (OIDC) identity tokens for {product-title-short}-issued tokens, enabling secure machine-to-machine authentication.

Red{nbsp}Hat recommends this especially for Continuous Integration (CI) usage, where short-lived access is preferable over long-lived API tokens.

The following steps outline the high-level workflow on how to configure short-lived access to the user interface and API calls:

. Configure {product-title-short} to trust OIDC identity token issuers for exchanging short-lived {product-title-short}-issued tokens.
. Exchange an OIDC identity token for a short-lived {product-title-short}-issued token by calling the API.
