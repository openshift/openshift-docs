= Deploying roxagent

[role="_abstract"]
To use {product-title} ({product-title-short}) to scan virtual machines (VMs), you must run `roxagent`, a binary executable. It runs inside VMs, scans for installed packages, and creates index reports.

.Prerequisites

* DNF is installed in the VM and you have completed at least one DNF transaction, such as `dnf update` or `dnf install`, so that the scan agent can pull all necessary metadata.
* The virtual machine is registered with Red Hat Subscription Manager and you have an active {op-system-base} subscription.

.Procedure

. Perform one of the following steps to get `roxagent`:
* Download `roxagent` from https://mirror.openshift.com/pub/rhacs/assets/4.9.0/bin/linux/roxagent.
* Build the `roxagent` binary manually:
.. Clone the stackrox repo by entering the following command:
+
[source,terminal]
----
$ git clone git@github.com:stackrox/stackrox.git
----
.. Change to the stackrox directory and enter the following command:
+
[source,terminal]
----
$ GOOS=linux GOARCH=amd64 go build ./compliance/virtualmachines/roxagent
----
. Transfer the binary to the VM by using a method such as `scp` or `cloud-init`.
. Run the binary inside the virtual machine. To perform a one-time scan, enter the following command:
+
[source,terminal]
----
$ sudo ./roxagent 
----
To perform continuous monitoring, enter the following command:
+
[source,terminal]
----
$ sudo ./roxagent --daemon
----
