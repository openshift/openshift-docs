= Kickstart files for embedding {microshift-short} with a {op-system-base} installation

By using a Kickstart, you create a single file containing the information needed for a typical {op-system-base-full} installation. Thus, a Kickstart file automates the installation. Using a Kickstart can also automate your {microshift-short} installation, including it with the {op-system-base} type that you choose. You can use a Kickstart to provision virtual machines, or complete a regular {op-system-base} installation.

For {microshift-short}, your Kickstart must include information to provision the {op-system-base} system so that it meets the following requirements:

* The VM you are provisioning must meet the system requirements for installing {microshift-short}.
* The file system must have a logical volume manager (LVM) volume group (VG) with sufficient capacity for the persistent volumes (PVs) of your workload.
* A pull secret from the https://console.redhat.com/openshift/install/pull-secret[Red Hat Hybrid Cloud Console] must be present as `/etc/crio/openshift-pull-secret` and have root user-only read/write permissions.
* Firewall configurations with the settings you require for cluster access.
