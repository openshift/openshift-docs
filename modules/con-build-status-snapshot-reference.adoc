= Build status and snapshot

[role="_abstract"]
You can monitor build health, diagnose failures, and trace `BuildRun` results to their Build configuration.
Status snapshots preserve the Build configuration used for a specific `BuildRun`, ensuring traceability between builds and their results.

Completion status fields:: Track whether a build succeeded, failed, or is in progress, including detailed conditions and reason codes.
Snapshot fields:: Preserve the exact Build specification used for a `BuildRun`, allowing you to decide which configuration produced specific results.
Tekton integration fields:: Reference the underlying Tekton `TaskRun` that executes the build, enabling access to detailed execution logs and step-level results.
Step result fields:: Expose individual step outcomes from the Tekton `TaskRun`, including source digests, image digests, and build artifact metadata.
Timing fields:: Record start time, completion time, and duration for build execution and individual steps.
