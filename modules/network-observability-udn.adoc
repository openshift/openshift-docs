= User-defined networks

User-defined networks (UDN) improve the flexibility and segmentation capabilities of the default Layer 3 topology for a Kubernetes pod network by enabling custom Layer 2 and Layer 3 network segments, where all these segments are isolated by default. These segments act as either primary or secondary networks for container pods and virtual machines that use the default OVN-Kubernetes CNI plugin.

UDNs enable a wide range of network architectures and topologies, enhancing network flexibility, security, and performance.

You can enable this feature in the network observability `FlowCollector` resource, such as outline in the following example. When enabled, the *Traffic* flow table has a UDN labels column. You can filter on *Source Network Name* and *Destination Network Name*.

.Procedure
. Run the following command:
[source,terminal]
----
$oc edit flowcollector
----

. Configure the `ebpf` section of the `FlowCollector` resource:
+
[source,yaml]
----
apiVersion: flows.netobserv.io/v1beta2
kind: FlowCollector
metadata:
  name: cluster
spec:
  agent:
    ebpf:
      sampling: 1  <1>
      privileged: true
      features:
      - UDNMapping
----
<1> Recommended so all flows are observed.

. Create a new namespace for a user-defined network with the label `k8s.ovn.org/primary-user-defined-network` set to an empty string:
+
[source,yaml]
----
apiVersion: v1
kind: Namespace <1>
metadata:
  name: 49ers
  labels:
    k8s.ovn.org/primary-user-defined-network: ""
----
<1> A new namespace is created because certain labels, like `k8s.ovn.org/primary-user-defined-network`, cannot be added to an existing namespace.

. Apply the new namespace by running the following command:
+
[source,terminal]
----
$oc apply
----

. Create a `UserDefinedNetwork` instance:
+
[source,yaml]
----
apiVersion: k8s.ovn.org/v1
kind: UserDefinedNetwork
metadata:
  name: udn-49ers
  namespace: 49ers
spec:
  topology: Layer2
  layer2:
    role: Primary
    subnets:
      - "10.0.0.0/24"
----

. Apply the `UserDefinedNetwork` instance:
+
[source,terminal]
----
$ oc apply
----

. Add a pod into the namespace.

.Verify

. Confirm the secondary interface is part of the UDN by running the following commands:
+
[source,terminal]
----
$ oc project 49ers
pod=$(oc get --no-headers pods | awk '{ print $1;}') <1>
----
+
[source,terminal]
----
$ oc describe pod/$pod  <2>
----
<1> Get pod name.
<2> Two interfaces are mentioned in *Annotations*.

All pods in this namespace are isolated from pods in other namespaces. In essence, a namespace can be a tenant. The UDN feature also has a `ClusterUserDefinedNetwork` resource that allows a UDN to span across multiple namespaces.
