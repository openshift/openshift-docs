= Configure hardware profiles for bare-metal nodes

[role="_abstract"]
You can define a `HardwareProfile` custom resource to specify the desired firmware levels and BIOS settings for a class of bare-metal servers. The O-Cloud Manager applies the hardware profile during provisioning to ensure servers meet the required hardware state before cluster installation begins.

[NOTE]
====
Hardware profiles are optional. If you do not specify a hardware profile, the O-Cloud Manager skips firmware and BIOS configuration during provisioning and uses the server in its current hardware state.
====

.Prerequisites

* The O-Cloud Manager Operator is installed on the hub cluster.
* You know the target firmware versions and BIOS settings for your server hardware.

.Procedure

. Create a `HardwareProfile` custom resource that defines the desired hardware state:
+
[source,yaml]
----
apiVersion: clcm.openshift.io/v1alpha1
kind: HardwareProfile
metadata:
  name: rh-profile-xr8620t-bios-settings
  namespace: <template_namespace>
spec:
  bios:
    attributes:
      BootMode: Uefi
      LogicalProc: Enabled
      ProcVirtualization: Enabled
      WorkloadProfile: HpcProfile
  bmcFirmware:
    version: "7.20.30.50"
  biosFirmware:
    version: "2.6.3"
----
+
* `spec.bios.attributes` defines the desired BIOS settings as key-value pairs. The available settings depend on your server hardware.
* `spec.bmcFirmware.version` specifies the target firmware version for the BMC controller (for example, iDRAC). You can optionally include a `url` field pointing to the firmware binary.
* `spec.biosFirmware.version` specifies the target BIOS firmware version. The hardware manager applies these levels during provisioning.

. Reference the hardware profile in a `ClusterTemplate` by adding the profile name to the `hwMgmtDefaults.nodeGroupData` section:
+
[source,yaml]
----
spec:
  templateDefaults:
    hwMgmtDefaults:
      nodeGroupData:
        - name: controller
          role: master
          hwProfile: rh-profile-xr8620t-bios-settings
          resourceSelector:
            resourceselector.clcm.openshift.io/server-type: XR8620t
----
+
Alternatively, you can override the hardware profile for a specific deployment by setting `hwMgmtParameters.nodeGroupData[].hwProfile` in the `ProvisioningRequest`. When both the `ClusterTemplate` and the `ProvisioningRequest` specify a hardware profile, the `ProvisioningRequest` value takes precedence.

. Apply the `HardwareProfile` resource to the hub cluster, or commit it to the `hardwareprofiles/` directory in the GitOps repository for ArgoCD synchronization.

.Verification

* Verify that the `HardwareProfile` resource exists on the hub cluster:
+
[source,terminal]
----
$ oc get hardwareprofiles.clcm.openshift.io -A
----
