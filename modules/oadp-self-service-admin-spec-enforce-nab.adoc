= Self-Service administrator spec enforcement for NAB

As a cluster administrator, you can enforce the following fields for a `NonAdminBackup` (NAB) CR:

* `csiSnapshotTimeout`
* `itemOperationTimeout`
* `resourcePolicy`
* `includedResources`
* `excludedResources`
* `orderedResources`
* `includeClusterResources`
* `excludedClusterScopedResources`
* `excludedNamespaceScopedResources`
* `includedNamespaceScopedResources`
* `labelSelector`
* `orLabelSelectors`
* `snapshotVolumes`
* `ttl`
* `snapshotMoveData`
* `uploaderConfig.parallelFilesUpload`

For example, if you want to enforce a `ttl` value of a backup, and want to enforce Data Mover for a namespace admin user, you can set up the `DataProtectionApplication` (DPA) CR as following:

.Example `DataProtectionApplication` CR
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: DataProtectionApplication
...
spec:
  nonAdmin:
    enable: true
    enforceBackupSpec: # <1>
      snapshotMoveData: true # <2>
      ttl: 158h0m0s # <3>
----

In the example DPA, you have:

* Added the `enforceBackupSpec` section as shown in annotation *1*.
* Enforced the Data Mover by setting the `snapshotMoveData` field to `true`, as shown in annotation *2*. 
* Enforced the `ttl` value by setting the field to 158h0m0s, as shown in annotation *3*.

When a namespace admin user creates a NAB CR, they must follow the template set up in the DPA. Otherwise, the `status.phase` field on the NAB CR is set to `BackingOff` and the NAB CR fails to create.
