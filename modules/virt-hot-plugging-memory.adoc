= Hot plugging memory on a virtual machine

You can add or remove the amount of memory allocated to a virtual machine (VM) without having to restart the VM by using the {product-title} web console.

.Procedure

. Navigate to *Virtualization* -> *VirtualMachines*.
. Select the required VM to open the *VirtualMachine details* page.
. On the *Configuration* tab, click *Edit CPU|Memory*.
. Enter the desired amount of memory and click *Save*.

The system applies these changes immediately. If the VM is migratable, a live migration is triggered. If not, or if the changes cannot be live-updated, a `RestartRequired` condition is added to the VM.

[NOTE]
====
Memory hotplug for virtual machines requires guest operating systems to support the `virtio-mem` driver. Instead of relying on specific upstream kernel versions, support depends on whether the driver is included and enabled in the guest OS.
For {op-system-base-full} guests, refer to the official support matrix to determine which RHEL versions and architectures include support for memory hotplug, see link:https://docs.google.com/document/d/1LG9Vqm6Q3TKX5X7__RpsnN9K13innr_9H0XVLUELT_M/edit?usp=sharing[RHEL guests supported kernel versions].
Avoid assuming support based on kernel version numbers, as RHEL uses backports and may not align with upstream kernel feature availability.
