= Admission control and runtime enforcement

{product-title-short} includes an admission controller that provides policy enforcement by rejecting a deployment attempt during the deploy phase. This is also known as “hard enforcement”. {product-title-short} also includes a backup enforcement mechanism, known as “soft enforcement”, which attempts to scale deployments down to zero replicas.

Runtime enforcement includes the following actions:

* Shutting down an offending pod, which results in creation of another healthy pod in its place
* Intercepting and preventing certain Kubernetes API calls
