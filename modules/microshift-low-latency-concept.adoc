= Lowering latency in {microshift-short} applications

Latency is defined as the the time from an event to the response to that event. You can use low latency configurations and tuning in a {microshift-short} cluster running in an operational or software-defined control system where an edge controller has to respond quickly to an external event. {microshift-short} low latency settings provide predictable latency.

[id="microshift-low-latency-workflow_{context}"]
== Workflow for configuring low latency for {microshift-short} applications
To configure low latency for applications running in a {microshift-short} cluster, you must complete the following tasks:

* Install the `microshift-low-latency` RPM.
* Configure the `kubelet` section of the `config.yaml` file in the `/etc/microshift/` directory.
//does a kubeletconfiguration need to already exist? are we passing this config there, or do these parameters go the other way?
* Configure and activate a suitable TuneD profile.
* Configure workload partitioning for the best performance.
* If you are using the x86_64 architecture, you can optionally install the link:https://docs.redhat.com/en/documentation/red_hat_enterprise_linux_for_real_time/9[Red Hat Enterprise Linux for Real Time 9].

[NOTE]
====
The CPU set for management applications, such as the {microshift-short} service, OVS, CRI-O, {microshift-short} pods, and isolated cores, must contain all-online CPUs.
