= Tailored profile requirements and support

[role="_abstract"]
Tailored profiles in {product-title} ({product-title-short}) have specific version requirements and naming conventions.
Understanding these requirements helps you plan your compliance scanning strategy across multiple clusters and {ocp} versions.

Tailored profiles are supported in {product-title-short} version 4.11 and later. For information about Compliance Operator and {ocp} version support, see the Compliance Operator release notes.

Keep the following conditions in mind:

* Compliance benchmark content can change between Compliance Operator versions. To check whether the underlying compliance checks have changed, compare the `ProfileBundle` resources on each cluster. If the rule definitions differ, the same tailored profile might produce different results or fail validation.

* If compliance benchmarks such as Center for Internet Security (CIS) change between {ocp} versions, the same tailored profile can produce different check descriptions or remediation steps on clusters running different versions. This is a known limitation related to upstream compliance content changes, not a {product-title-short} limitation.

For details about compliance content changes, see the Compliance Operator release notes.
