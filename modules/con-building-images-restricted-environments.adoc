= Build container images in restricted environments

[role="_abstract"]
Network-restricted environments limit direct access to external registries and repositories, which affects container image builds.
{builds-shortname} supports building images in air-gapped networks, clusters with firewall, and networks that require proxy configuration.

Network-restricted environments include air-gapped clusters with no internet access, networks with firewalls that block external registries, and networks that require HTTP proxies for outbound connections.
Building container images in these environments presents challenges such as accessing base images, pulling build dependencies, and connecting to package repositories.

You can configure builds to use internal mirror registries for base images and dependencies, eliminating the need for external network access during builds.
