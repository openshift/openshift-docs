= Scaling down workflows with the `gitOps` profile

Before upgrading, you must scale down workflows running with the `gitops` profile before upgrading and scale them up again after the upgrade.

.Procedure

. Modify the `my-workflow.yaml` CRD and scale down the each workflow to `zero` before upgrading as shown in the following example:
+
[source,yaml]
----
spec:
  podTemplate:
    replicas: 0
----

. Apply the updated CRD by executing the following command:
+
[source,terminal]
----
$ oc apply -f <my-workflow.yaml> -n <target_namespace>
----
