= Prevent automatic network observability installation

[role="_abstract"]
Prevent automatic network observability installation during cluster creation when network traffic monitoring is not required.

By default, {product-title} installs the Network Observability Operator on new clusters when the `NetworkObservabilityInstall` feature gate is enabled. You can disable this behavior by configuring the `install-config.yaml` file before creating the cluster. Disabling network observability prevents the operator installation and avoids the associated resource consumption, which typically uses 3-5% of node CPU and memory.

.Prerequisites

* For Technology Preview only, you must enable the `NetworkObservabilityInstall` feature gate.
* You must have access to the `install-config.yaml` file before creating installation manifests.

.Procedure

. Edit the `install-config.yaml` file before creating installation manifests.

. Add the `networkObservability` field to the `networking` section:
+
[source,yaml]
----
apiVersion: v1
baseDomain: example.com
metadata:
  name: my-cluster
networking:
  networkObservability:
    installationPolicy: DoNotInstall
# ...
----
+
where:
+
`networking.networkObservability.installationPolicy`:: Controls whether the Network Observability Operator is installed during cluster creation. Set to `DoNotInstall` to prevent installation. Valid values are `InstallAndEnable` and `DoNotInstall`. The default value is `InstallAndEnable` for standard clusters when the `NetworkObservabilityInstall` feature gate is enabled.

. Create the installation manifests, enable the feature gate, and install the cluster.

.Verification

* Verify that the Network Observability Operator is not installed by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-netobserv-operator
----
+
The output should indicate that the namespace does not exist:
+
.Example output
[source,terminal]
----
Error from server (NotFound): namespaces "openshift-netobserv-operator" not found
----
