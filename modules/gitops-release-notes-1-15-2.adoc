[id="new-features-1-15-2_{context}"]
== New features

* With this update, a new option, `.spec.SyncPolicy.Automated`, has been added to the Argo CD Application resource to explicitly enable or disable `automatedSync`. This option provides the following benefits:
+
--
** Enables or disables auto-sync behavior independently of `prune` and `selfHeal`.
** When set to `true`, auto-sync is enabled.
** When set to `false`, auto-sync is explicitly disabled even if the `automated` block exists.
** Defaults to `true` for backward compatibility.
--
+
.Example
[source,YAML]
----
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: my-app
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/my-org/my-app.git
    targetRevision: HEAD
    path: manifests
  destination:
    server: https://kubernetes.default.svc
    namespace: my-app-namespace
  syncPolicy:
    automated:
      enabled: false #<1>
      prune: true
      selfHeal: true
----
<1> This option enables or disables `autoSync`. Setting the option to `false` explicitly disables automated sync, even when automated sync is enabled.
+
--

[id="fixed-issues-1-15-2_{context}"]
== Fixed issues

* Before this update, Argo CD components, such as, `server`, `repo-server`, and `application-controller`, could crash when accessing the Redis instance due to network or DNS instabilities within the cluster. This issue stemmed from a race condition in the `go-redis` client library when multiple connections in a connection pool call the `dial hook` function. With this update, this issue is fixed. This update resolves the issue by integrating an updated `go-redis` client library that eliminates race conditions during `dial hook` function calls. It also improves the handling and recovery from network and DNS errors, ensuring greater stability for Argo CD components. link:https://issues.redhat.com/browse/GITOPS-6287[GITOPS-6287]

* Before this update, upgrading the {gitops-title} Operator to v1.15.1 raised a health check error that prevented Red Hat Advanced Cluster Management (ACM) policies from syncing. This update fixes the issue by adding a missing nil check to `.status.placement` for *Policy*. link:https://issues.redhat.com/browse/GITOPS-6664[GITOPS-6664]

* Before this update, when `.spec.source.targetRevision` was set to an annotated Git tag, Argo CD treated resource drift as needing reconciliation. This caused manual changes on the cluster to be reverted, despite self-healing being disabled. This behavior was inconsistent with that of lightweight tags. With this update, Argo CD now handles annotated and lightweight tags consistently. When `selfHeal` is set to `false`, manual changes to live resources are no longer reverted, even if you are using an annotated Git tag as the source. link:https://issues.redhat.com/browse/GITOPS-6150[GITOPS-6150]
