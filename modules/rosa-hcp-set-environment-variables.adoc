== Set Environment Variables

You may set following environment variables to streamline resource creation.

.Procedure

. Set your environment variable by using the following command:
+
[source,terminal]
----
$ export <variable_name>=<variable_value>
----
+
ifdef::rosa-disconnected-install[]
.Suggested variables for disconnected {product-title} clusters
[cols="3",options="header"]
|===
|Variable name
|Variable value
|Notes

|`AWS_ACCOUNT_ID`
|$(aws sts get-caller-identity --query Account --output text)
|You must be logged into your AWS account with `rosa login`.

|`CLUSTER_NAME``
|The name you want for your cluster
|Your cluster name cannot exceed 26 characters.

|`OIDC_ID`
|The 32-digit ID for you OIDC configuration.
|You generate this ID by running `rosa create oidc-config`.

|`OPERATOR_ROLES_PREFIX`
|Role prefix
| -

|`PRIVATE_SUBNET`
|The ID of your private subnets.
|This value should be enclosed in quotation marks (") and each subnet ID should be comma-separated.

|`PUBLIC_SUBNET`
|The ID of your public subnets.
|This value should be enclosed in quotation marks (") and each subnet ID should be comma-separated.

|`REGION`
|Your desired AWS region.
| -

|`SUBNET_IDS`
|The IDs of all your subnets.
|This value should be enclosed in quotation marks (") and each subnet ID should be comma-separated.

|
|
| -
|===
endif::rosa-disconnected-install[]
