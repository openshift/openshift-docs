= Volumes definition for a build

[role="_abstract"]
Define volumes in your `Build` CR. The defined volumes override the volumes specified in the `BuildStrategy` resource. The build run fails if a volume is not explicitly overridden.

The following example shows the usage of the `volumes` field in a `Build` CR:

[source,yaml]
----
apiVersion: shipwright.io/v1beta1
kind: Build
metadata:
  name: <build_name>
spec:
  source:
    git:
      url: https://github.com/example/url
  strategy:
    name: buildah
    kind: ClusterBuildStrategy
  paramValues:
  - name: dockerfile
    value: Dockerfile
  output:
    image: registry/namespace/image:latest
  volumes:
    - name: <volume_name>
      configMap:
        name: <configmap_name>
----
+
where:

<build_name>:: Specifies the name of the `build`.
<volume_name>:: Specifies the the name of the volume.
<configmap_name>:: Specifies the name of the `ConfigMap`.
