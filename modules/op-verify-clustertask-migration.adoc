= Verify the ClusterTask migration

[role="_abstract"]
After updating your pipelines to use the cluster resolver, verify that the migration is complete and that your pipelines work correctly.

.Prerequisites

* You updated all `taskRef` definitions to use the cluster resolver.
* You have access to the namespaces where your pipelines are defined.

.Procedure

. Verify that no `ClusterTask` references remain in your cluster by running the following command:
+
[source,terminal]
----
$ oc get pipelines -A -o yaml | grep -c "kind: ClusterTask"
----
+
--
*Example output:*
[source,terminal]
----
0
----
--
+
A count of `0` indicates that no `ClusterTask` references remain.

. Check pipeline runs for any remaining `ClusterTask` references by running the following command:
+
[source,terminal]
----
$ oc get pipelineruns -A -o yaml | grep -c "kind: ClusterTask"
----

. Apply one of your updated pipelines to the cluster by running the following command:
+
[source,terminal]
----
$ oc apply -f my-updated-pipeline.yaml
----

. Start a pipeline run to test the migrated pipeline by running the following command:
+
[source,terminal]
----
$ tkn pipeline start my-updated-pipeline --showlog
----

. Monitor the pipeline run to ensure all tasks resolve correctly by running the following command:
+
[source,terminal]
----
$ tkn pipelinerun logs <pipelinerun_name> -f
----
+
<pipelinerun_name>:: Specifies the name of the pipeline run to monitor.

. Verify that the task was resolved from the correct namespace by running the following command:
+
[source,terminal]
----
$ tkn pipelinerun describe <pipelinerun_name>
----
+
--
<pipelinerun_name>:: Specifies the name of the pipeline run.
--
+
--
.Example output
[source,terminal]
----
Name:           my-pipeline-run-xxxxx
Namespace:      my-project
Pipeline Ref:   my-updated-pipeline
TASKRUNS
NAME                             TASK NAME      STARTED         DURATION   STATUS
my-pipeline-run-xxxxx-build      buildah        2 minutes ago   1 minute   Succeeded
----
`my-pipelines-run-xxxxx-build`:: Indicates the task ran successfully using the cluster resolver
--

. Review the resolved task reference in the pipeline run status:
+
[source,terminal]
----
$ oc get pipelinerun <pipelinerun_name> -o jsonpath='{.status.childReferences[*].pipelineTaskName}'
----
+
<pipelinerun_name>:: Specifies the name of the pipeline run to be inspected.

.Verification

. Confirm that all your critical pipelines have been migrated and tested.

. Document any custom tasks or community tasks that you installed for future reference.

. Create a test plan to verify all common pipeline scenarios:
+
--
* Pipelines with multiple tasks
* Pipelines using tasks from different namespaces
* Pipelines with conditional tasks
* Pipelines using `when` expressions
--

[TIP]
====
After verifying the migration, consider removing any old `ClusterTask` resources if they still exist in your cluster to prevent confusion.
====
