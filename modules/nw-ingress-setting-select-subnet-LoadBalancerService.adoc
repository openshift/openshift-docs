= Choosing subnets while creating a LoadBalancerService ingress controller
Users may have dedicated subnets for their load balancers due to security reasons or infrastructure constraints. A cluster administrator can manually set the subnets when creating an ingress controller of type LoadBalancer Service.

Previously, when creating an Ingress Controller, AWS automatically selected the subnets. However, the AWS LoadBalancer Service allows specifying the target subnet using the `service.beta.kubernetes.io/aws-load-balancer-subnets` annotation.

[NOTE]
====
This feature and procedure applies only to AWS. 
====

.Prerequisites
* The following assumes that you already created an Ingress Controller

.Procedure
To create an Ingress Controller of type **LoadBalancerService** in Red Hat OpenShift Container Platform 4.17, you can follow these steps:

* Create a custom resource (CR) file.

** Create a YAML file (e.g., `sample-ingress.yaml`) with the following content:
+
[source,yaml]
----
apiVersion: operator.openshift.io/v1
    kind: IngressController
    metadata:
      namespace: openshift-ingress-operator
      name: <name>
    spec:
      domain: <domain>
      endpointPublishingStrategy:
        type: LoadBalancerService
        loadBalancer:
          scope: External
      dnsManagementPolicy: Managed
----
+
Replace `<name>` with a name for the Ingress Controller and `<domain>` with the DNS name serviced by the Ingress Controller.
+
** Apply the CR file:

Save the file and apply it using the OpenShift CLI:
+
[source,terminal]
----
$  oc apply -f sample-ingress.yaml
----
+
* Choose the subnets.

[NOTE]
====
When configuring subnets in OpenShift Container Platform 4.17, you can use the Cluster Network Operator (CNO).
====

** View your current cluster network configuration using the following command:
+
[source,terminal]
----
$   oc describe network.config/cluster
----
+
** Modify the subnet configuration in the `Spec` field. This field allows you to define the cluster CIDR and host prefix. For example:
+
[source,yaml]
----
apiVersion: config.openshift.io/v1
    kind: Network
    metadata:
      name: cluster
    spec:
      clusterNetwork:
        cidr: 10.128.0.0/14
        hostPrefix: 23
      serviceNetwork: 172.30.0.0/16
----
+
** Save the changes and apply them using the OpenShift CLI:
+
[source,terminal]
----
$   oc apply -f network-config.yaml
----
+
