= Configuring the Generic Device Plugin

The Generic Device Plugin is disabled by default in {microshift-short}. To use it, you must enable it and specify which host devices your KubeOncernetes applications can access. This involves modifying the {microshift-short} `config.yaml` file.

.Prerequisites
* You have installed {microshift-short}.
* You have `sudo` privileges on the {microshift-short} host.
* You have identified the specific host devices (for example, `/dev/video0`, `/dev/ttyUSB*`, or USB Vendor/Product IDs) that you want to expose to your {microshift-short} cluster.

.Procedure

. Access the {microshift-short} configuration file.
+
The main {microshift-short} configuration is typically located at `/etc/microshift/config.yaml`.
+
If this file does not exist, you can generate a default one by running `microshift show-config -> /etc/microshift/config.yaml` and then modify it.

. Enable and configure the Generic Device Plugin.
+
.. Open `/etc/microshift/config.yaml` in a text editor with `sudo` privileges.
+
.. Locate the `genericDevicePlugin` section. If it is not present, add it.
+
.. Set the `status` to `Enabled` and define the `devices` that should be exposed. Each device definition needs a `name` and one or more `groups`. Each group can specify devices using `paths` (for file-based devices, including glob patterns) or `usb` (for USB devices using Vendor/Product IDs). You cannot mix `paths` and `usb` within the same device group.
+
Here is an example configuration that enables the GDP and exposes a serial device, a video device, and allocates the FUSE device multiple times:
+
[source,yaml]
----
genericDevicePlugin:
  status: Enabled
  domain: device.microshift.io # This is the default domain for MicroShift GDP
  devices:
    - name: serial
      groups:
        - paths:
            - path: /dev/ttyUSB*  # Exposes all USB serial devices matched by this glob
            - path: /dev/ttyACM*  # Exposes all ACM serial devices matched by this glob
    - name: video
      groups:
        - paths:
            - path: /dev/video0    # Exposes a specific video device
    - name: fuse
      groups:
        - count: 10              # Allows /dev/fuse to be allocated up to 10 times concurrently
          paths:
            - path: /dev/fuse
    - name: zigbee
      groups:
        - usb: # Exposes a specific USB device by Vendor and Product ID
            - vendor: "1a86"  # Example: Vendor ID for CH340 serial converter
              product: "7523" # Example: Product ID for CH340 serial converter
----
+
[IMPORTANT]
====
* For consistency and precise device targeting, especially when dealing with multiple similar devices, consider using stable device paths like `/dev/serial/by-id/` or specific USB Vendor/Product/Serial IDs instead of broad glob patterns like `/dev/ttyUSB*`.
* The `count` parameter in a device group allows a single device (or a set of devices matched by a glob) to be allocated multiple times concurrently to different pods. If omitted, `count` defaults to `1`.
* For a comprehensive list of configuration parameters and their descriptions, refer to the "Generic Device Plugin Configuration Reference" module.
====
+
. Save the configuration file.
. Restart the {microshift-short} service to apply the changes:
+
[source,bash]
----
sudo systemctl restart microshift
----
+
Allow some time for {microshift-short} to restart and for the Generic Device Plugin to register its devices with the Kubelet.

. Verify the device plugin status (Optional):
+
You can check the available devices in your cluster by querying the node's allocatable resources:
+
[source,bash]
----
kubectl describe node <microshift-node-name> | grep "device.microshift.io"
----
+
You should see output similar to this, depending on your configuration:
+
[source,text]
----
  device.microshift.io/serial:  1
  device.microshift.io/video:   1
  device.microshift.io/fuse:    10
----
+
This output indicates that the devices are now discoverable and schedulable within your {microshift-short} cluster.
