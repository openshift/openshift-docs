= Configuring the Generic Device Plugin
:FeatureName: The Generic Device Plugin for {microshift-short}

include::snippets/technology-preview.adoc[]

The Generic Device Plugin (GDP) is disabled by default in {microshift-short}. To use the GDP, you must enable it and specify which host devices your Kubernetes applications can access. To enable the GDP, you must modify the {microshift-short} `config.yaml` file or create a configuration drop-in file (for example, `/etc/microshift/config.d/10-gdp.yaml`).

.Prerequisites

* You installed {microshift-short}.
* You created a custom `config.yaml` file in the `/etc/microshift` directory.
* You installed the {oc-first}.
* You have `sudo` privileges on the {microshift-short} host.
* You have identified the specific host devices that you want to expose to your {microshift-short} cluster (for example, `/dev/video0`, `/dev/ttyUSB*`, or USB Vendor/Product IDs).

.Procedure

. Enable and configure the GDP.
+
.. Open `/etc/microshift/config.yaml` in a text editor with `sudo` privileges.
+
.. Locate the `genericDevicePlugin` section. If it is not present, add it.
+
.. Set the `status` parameter to `Enabled` and define the `devices` that should be exposed. Each device definition needs a `name` and one or more `groups`. Each group can specify devices using `paths` (for file-based devices, including glob patterns) or `usbs` (for USB devices using Vendor/Product IDs). You cannot mix `paths` and `usbs` within the same device group.
+
.GDP fields with default values
[source,yaml]
----
apiServer:
# ...
genericDevicePlugin:
  status: Enabled
  domain: device.microshift.io # This is the default domain for MicroShift GDP
  devices:
    - name: serial
      groups:
        - paths:
            - path: /dev/ttyUSB*  # Exposes all USB serial devices matched by this glob
            - path: /dev/ttyACM*  # Exposes all ACM serial devices matched by this glob
    - name: fuse
      groups:
            - path: /dev/fuse
    - name: converter
      groups:
        - usbs: # Exposes a specific USB device by Vendor and Product ID
            - vendor: "0x1a86"  # Example: Vendor ID for CH340 serial converter
              product: "0x7523" # Example: Product ID for CH340 serial converter
----
+
[IMPORTANT]
====
* The output of the `microshift show-config` parameter might include pre-configured default paths for serial, video, and audio devices even if you have not explicitly configured them in `config.yaml`. These paths represent the default discovery settings if the Generic Device Plugin is enabled without specific user configuration.
* For consistency and precise device targeting, especially when dealing with multiple similar devices, consider using stable device paths like `/dev/serial/by-id/` or specific USB Vendor/Product/Serial IDs instead of broad glob patterns like `/dev/ttyUSB*`.
* The `count` parameter in a device group allows a single device (or a set of devices matched by a glob) to be allocated multiple times concurrently to different pods. If omitted, `count` defaults to `1`.
====
+
. Save the `config.yaml` file.
. Restart the {microshift-short} service to apply the changes:
+
[source,bash]
----
sudo systemctl restart microshift
----
+
Allow some time for {microshift-short} to restart and for the GDP to register its devices with the Kubelet.

. Verify the device plugin status (Optional):
+
You can check the available devices in your cluster by querying the node's allocatable resources:
+
[source,bash]
----
oc describe node <microshift-node-name> | grep "device.microshift.io"
----
+
You should see output similar to this, depending on your configuration:
+
[source,text]
----
  device.microshift.io/serial:  1
  device.microshift.io/video:   1
  device.microshift.io/fuse:    10
----
+
This output indicates that the devices are now discoverable and schedulable within your {microshift-short} cluster.
