== Resolved issues

*Self-signed certificate for internal image backup should not break other BSLs*

In this update, the Velero instance previously applied a certificate globally rather than combining it with system-trusted certificates. This led to Velero instances with AWS backup failing due to certificate validation issues, impacting backup functionality. In this release, Backup Service Location (BSL) certificates are now combined with system-trusted certificates, enabling the use of several BSLs with custom CA certificates concurrently. As a result, numerous BSLs with custom CA certificates can operate simultaneously, enhancing compatibility for users with various storage providers.
