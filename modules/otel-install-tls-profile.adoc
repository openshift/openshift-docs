= Adhere to the cluster TLS profile

The OpenTelemetry operator adheres to the TLS configuration defined in the OpenShift `APIServer` custom resource.
The TLS settings (min version and cipher suites) from the `APIServer` are used in all TLS communication in the operator
and operands.

The TLS cluster profile adherence is configured by these environment variables:

* `TLS_CLUSTER_PROFILE` - enables reading the TLS configuration from the `APIServer` custom resource. Enabled by default.
* `TLS_CONFIGURE_OPERANDS` - enables TLS configuration in the operands (e.g. receivers). Enabled by default.

When TLS_CLUSTER_PROFILE is disabled the TLS min version and ciphers can be configured directly:

* `TLS_MIN_VERSION` - min TLS version. Defaults to `VersionTLS12`.
* `TLS_CIPHER_SUITES` - TLS cipher suites. Defaults to the default Golang cipher suites for the given version.
