== Preparing the git repository for migration

The directory structure in Git remains the same when defining the configuration for your clusters with `ClusterInstance` CRs. For demonstration purposes, these steps use a dedicated `clusters` folder within the project structure. Within this `clusters` folder, you can create individual subfolders for each cluster, enabling logical grouping of related resources.

To prepare the git repository for the migration, create a new folder `ClusterInstance` CR similar to the following example.  

[source,text]
----
clusters/sno3
├── clusterinstance.yaml
├── extra-manifests
│   └── master-core-password.yaml
├── kustomization.yaml
├── ns.yaml
└── secrets.yaml
----
