= Configure RBAC for Perses dashboards

[role="_abstract"]
You can grant users the ability to view or edit Perses dashboards and datasources by binding the appropriate `ClusterRole` resources.

== Granting namespace-scoped viewer access

Use a `RoleBinding` to grant a user read-only access to dashboards and datasources in a specific namespace:

[source,yaml]
----
kind: RoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: <user>-viewer-dashboard
  namespace: <target_namespace>
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: <username>
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: persesdashboard-viewer-role
---
kind: RoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: <user>-viewer-datasource
  namespace: <target_namespace>
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: <username>
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: persesdatasource-viewer-role
----
where:

`<target_namespace>`:: Replace with the namespace to which you are granting access.

`<username>`:: Replace with the name of the user.

== Granting cluster-wide editor access

Use a `ClusterRoleBinding` to grant a user editor access to dashboards and datasources across all namespaces:

[source,yaml]
----
kind: ClusterRoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: <user>-editor-dashboard
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: <username>
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: persesdashboard-editor-role
---
kind: ClusterRoleBinding
apiVersion: rbac.authorization.k8s.io/v1
metadata:
  name: <user>-editor-datasource
subjects:
  - kind: User
    apiGroup: rbac.authorization.k8s.io
    name: <username>
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: persesdatasource-editor-role
----

* Apply the role binding manifest by running the following command:
+
[source,terminal]
----
$ oc apply -f rolebinding.yaml
----

* Verify that the binding was created by running the following command:
+
[source,terminal]
----
$ oc get rolebindings -n <target_namespace> | grep perses
----

* Ask the user to navigate to *Observe* -> *Dashboards (Perses)* in the {product-title} web console and confirm that the namespace and dashboards are visible.

== Revoking access

To revoke access, delete the role binding:

[source,terminal]
----
$ oc delete rolebinding <binding_name> -n <target_namespace>
----
