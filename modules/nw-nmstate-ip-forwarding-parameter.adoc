= nmstate IP forwarding configuration reference

[role="_abstract"]
The nmstate API provides the `forwarding` parameter under the `ipv4` section of an interface definition to control IPv4 packet forwarding on a per-interface basis. You can use this parameter in a `NodeNetworkConfigurationPolicy` custom resource to enable or disable forwarding on specific network interfaces.

== IP forwarding parameter

The following table describes the `forwarding` parameter:

.nmstate IPv4 forwarding parameter
[cols="1,1,3",options="header"]
|===
|Parameter |Type |Description

|`forwarding`
|Boolean
|Controls whether IPv4 packet forwarding is enabled on the interface. Set to `true` to enable forwarding, or `false` to disable it. When not specified, the interface inherits the system default behavior.
|===

== Example configurations

.Enable IPv4 forwarding on an Ethernet interface
[source,yaml]
----
apiVersion: nmstate.io/v1
kind: NodeNetworkConfigurationPolicy
metadata:
  name: enable-forwarding-eth1
spec:
  desiredState:
    interfaces:
    - name: eth1
      type: ethernet
      state: up
      ipv4:
        enabled: true
        dhcp: true
        forwarding: true
----

.Enable IPv4 forwarding on a VLAN interface
[source,yaml]
----
apiVersion: nmstate.io/v1
kind: NodeNetworkConfigurationPolicy
metadata:
  name: enable-forwarding-vlan100
spec:
  desiredState:
    interfaces:
    - name: eth1.100
      type: vlan
      state: up
      vlan:
        base-iface: eth1
        id: 100
      ipv4:
        enabled: true
        address:
        - ip: 192.168.100.10
          prefix-length: 24
        forwarding: true
----

.Enable IPv4 forwarding on a bond interface
[source,yaml]
----
apiVersion: nmstate.io/v1
kind: NodeNetworkConfigurationPolicy
metadata:
  name: enable-forwarding-bond0
spec:
  desiredState:
    interfaces:
    - name: bond0
      type: bond
      state: up
      link-aggregation:
        mode: balance-rr
        port:
        - eth1
        - eth2
      ipv4:
        enabled: true
        dhcp: true
        forwarding: true
----

.Enable IPv4 forwarding on a Linux bridge interface
[source,yaml]
----
apiVersion: nmstate.io/v1
kind: NodeNetworkConfigurationPolicy
metadata:
  name: enable-forwarding-br1
spec:
  desiredState:
    interfaces:
    - name: br1
      type: linux-bridge
      state: up
      bridge:
        port:
        - name: eth1
      ipv4:
        enabled: true
        dhcp: true
        forwarding: true
----
