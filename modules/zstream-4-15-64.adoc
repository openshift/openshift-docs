== Fixed issues

* Before this update, the `osDiskImage.source.id` property in the user-provisioned infrastructure (UPI) ARM template was incorrectly placed. As a consequence, it lead to installation failures. With this release, the `storageProfile.osDiskImage.source.id` property location in the UPI ARM template is corrected. As a result, the UPI ARM template installation on Azure no longer encounters invalid parameter errors. (link:https://issues.redhat.com/browse/OCPBUGS-79036[OCPBUGS-79036])
