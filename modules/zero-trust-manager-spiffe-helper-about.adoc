= SPIFFE Helper container image

[role="_abstract"]
SPIFFE Helper is a utility that connects to the SPIFFE Workload API, fetches identity credentials, writes them to files on disk, and optionally notifies a workload when X.509 material is renewed. Use it with the {zero-trust-full} when your application cannot call the Workload API directly but can read TLS certificates from a shared volume.

The {zero-trust-full} provides a supported SPIFFE Helper container image based on https://github.com/spiffe/spiffe-helper[upstream SPIFFE Helper]. The configuration file format, command-line flags, and Workload API behavior are compatible with upstream.

SPIFFE Helper performs the following tasks:

. Connects to the SPIFFE Workload API, usually through the SPIRE Agent socket exposed by the SPIFFE CSI driver.
. Fetches X.509 SVIDs, JWT SVIDs, and/or JWT bundles from SPIRE.
. Writes credentials to files under a configured directory (`cert_dir`).
. Optionally notifies a workload when X.509 material is renewed in daemon mode.
