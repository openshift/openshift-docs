= Configuring OTLP output

The OpenTelemetry Protocol (OTLP) allows the cluster administrator to collect and forward logs to the OTLP receivers. The OTLP sends data over HTTP using JSON encoding by adhering to the https://opentelemetry.io/docs/specs/otlp/[OpenTelemetry Observability Framework].

:FeatureName: The OTLP output log forwarder
include::snippets/technology-preview.adoc[]

.Procedure

* To enable OTLP, create or edit the `ClusterLogForwarder` CR using the template below:
+
.Example ClusterLogForwarder CR
[source,yaml]
----
apiVersion: observability.openshift.io/v1
kind: ClusterLogForwarder
metadata:
    name: my-logforwarder
    namespace: my-app-namespace
    annotations: # <1>
        observability.openshift.io/tech-preview-otlp-output: "enabled"
spec:
    outputs:
    - type: otlp # <2>
      name: otel-collector
      tls:
        insecureSkipVerify: true # <3>
      otlp:
        url: 'http://my-otel-receiver-service:4318/v1/logs' # <4>
        tuning:
            compression: none # <5>
        authentication: # <6>
            token:
                from: serviceAccount
    pipelines:
    - name: my-pipeline
      inputRefs:
      - application
      - infrastructure
      outputRefs:
      - otel-collector
serviceAccount:
    name: logger-admin
----
+
<1> Annotations for the `ClusterLogForwarder` metadata. 
** This annotation specifies that the OTLP output is enabled as a Technical Preview feature.
<2> Specifies the output type as `otlp` for OpenTelemetry.
<3> Setting the field `insecureSkipVerify` value to `true` disables TLS verification for the OTLP connection.
+
[WARNING]
====
This setting is not recommended for the production configurations.
====
+
<4> Defines the URL of the OpenTelemetry receiver endpoint. This URL must be absolute and is where the logs are sent.
<5> Configures the tuning options for the OTLP output. The `compression`  type default value is `gzip`. Here, `compression` is set to `none`, which means no compression is applied to the logs.
<6> The OTLP `authentication` is optional and specifies a `token` with `from` and a value of `serviceAccount`. The token can also be read from a secret.
** Specifies the token-based authentication method.
** Specifies that the token is sourced from a service account. The token can also be sourced from a secret.

[NOTE]
====
The OTLP output uses the OpenTelemetry data model, which is different from other log output formats. The OTLP output does not send ViaQ logs. Instead, it adheres to the https://opentelemetry.io/docs/specs/semconv/[OpenTelemetry Semantic Conventions] and https://opentelemetry.io/docs/specs/otel/logs/data-model/[Logs Data Model].
====
