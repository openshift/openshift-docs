= Restart the {mcpg} broker component

[role="_abstract"]
While you are troubleshooting your on-premise MCP server, you might make changes to your `MCPServerRegistration` custom resource (CR). To ensure that your changes are applied, you must restart the {mcpg} broker component.

.Prerequisites

* You installed {mcpg}.
* You installed {prodname}.
* You configured a `Gateway` object.
* You configured an `HTTPRoute` object for the gateway.
* You installed the {oc-first}.
* You registered an MCP server.

.Procedure

* Restart the {mcpg} broker component after making `MCPServerRegistration` CR changes by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc rollout restart deployment/_<mcp_gateway>_ -n _<mcp_system>_
----
+
** Replace `_<mcp_gateway>_` with the name of your {mcpg} deployment.
** Replace `_<mcp_system>_` with the namespace of your {mcpg} deployment.
