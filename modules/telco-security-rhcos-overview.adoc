= Red Hat Enterprise Linux CoreOS (RHCOS)

Red Hat Enterprise Linux CoreOS (RHCOS) is different from Red Hat Enterprise Linux (RHEL) in key areas. For more information, see "About RHCOS".

From a telco perspective, a major distinction is the control of `rpm-ostree`, which is updated through the Machine Config Operator. 

RHCOS follows the same immutable design used for pods in {product-title}. This ensures that the operating system remains consistent across the cluster. For information about RHCOS architecture, see "Red Hat Enterprise Linux CoreOS (RHCOS)".

To manage hosts effectively while maintaining security, avoid direct access whenever possible. It is best to avoid direct access to hosts. Instead, you can use methods for host management:

* Debug pod
* Direct SSHs
* Console access

RHCOS was built with a strong emphasis on security. The following security mechanisms are integral to maintaining host security:

Linux namespaces:: Provide isolation for processes and resources. Each container's processes and files are kept within its own namespace. If a user escapes from the container’s namespace, they could gain access to the host operating system, which can compromise security.

Security-Enhanced Linux (SELinux):: Enforces mandatory access controls to restrict access to files and directories by processes. It adds an extra layer of security by preventing unauthorized access to files if a process tries to break its confinement.
+
SELinux follows the security policy of denying everything unless explicitly allowed. If a process attempts to modify or access a file without permission, SELinux denies access. For more information, see link:https://docs.redhat.com/en/documentation/red_hat_enterprise_linux/9/html-single/using_selinux/index#introduction-to-selinux_getting-started-with-selinux[Introduction to SELinux].

Linux capabilities:: Assigns specific privileges to processes at a granular level, minimizing the need for full root permissions. For more information, see "Linux capabilities".

Control group (cgroup):: Allocates and manages system resources (CPU, memory, and so on) for processes and containers, ensuring efficient usage. it ensures fair allocation of resources across processes and containers. As of {product-title} 4.16, there are two versions of cgroups. Cgroup v2 is now configured by default.

CRI-O:: Serves as a lightweight container runtime that enforces security boundaries and manages container workloads.
