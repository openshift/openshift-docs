= {aws-short} Bedrock authentication types

[role="_abstract"]
When you integrate {aws-short} Bedrock with {ols-long}, you can authenticate through a single configuration secret by using one of the following example credentials secret:

[NOTE]
====
For {aws-short} Bedrock, the only tested models are Anthropic, OpenAI, and DeepSeek. If you require additional models, you can test them. If they do not work, you must open a request for enhancement (RFE) for {ols-long}.
====

Bearer token::
This method applies when you connect to an external provider gateway, such as a Bedrock Mantle gateway. You map your connection credentials to the `apitoken` key.

AWS Identity and Access Management (IAM) keys::
This method uses an explicit access key pair. You map these parameters to the `aws_access_key_id` and `aws_secret_access_key` keys.

IAM keys with AWS Security Token Service (STS) role assumption::
This method applies when your primary access keys lack permissions to call {aws-short} Bedrock directly and must assume a specific role first. You define the target identity by adding the `role_arn` parameter to your credentials secret alongside your access key pair. Use this option for cross-account model access or security models that enforce short-lived, low-privilege sessions.
+
[NOTE] 
====
Bearer token configurations do not support the `role_arn` parameter.
====
