= FlexVolume driver requirements

Before you install a FlexVolume driver, ensure that the driver implements the following operations.

.Required FlexVolume driver operations
[cols="1,2,2,1,1"]
|===
|Operation |Description |Arguments |Executed on |Expected output

|`init`
|Initializes the driver. Called during initialization of all nodes.
|None
|Node
|Default JSON

|`mount`
|Mounts a volume to a directory. This can include finding the device and then mounting it.
|`<mount-dir>` `<json>`
|Node
|Default JSON

|`unmount`
|Unmounts a volume from a directory. This can include cleaning up the volume after unmounting.
|`<mount-dir>`
|Node
|Default JSON

|`mountdevice`
|Mounts a volume device to a directory where individual pods can then bind-mount.
|`<mount-dir>` `<json>`
|Node
|Default JSON

|`unmountdevice`
|Unmounts a volume device from a directory.
|`<mount-dir>`
|Node
|Default JSON
|===

[NOTE]
====
The `mountdevice` call-out does not pass secrets that are specified in the FlexVolume specification. If your driver requires secrets, do not implement this call-out.
====

All other operations must return JSON with `{"status": "Not supported"}` and exit code `1`.
