= Verifying the replaced control plane node in TNF
 
[role="_abstract"]
After replacing a failed control plane node in a two-node fencing (TNF) cluster, verify that the cluster is fully operational.
 
.Prerequisites
 
* The replacement node procedure is complete and the node shows as `Ready`.
 
.Procedure
 
. Verify that the pacemaker cluster is online. SSH to the replacement node and check the status by running the following command:
+
[source,terminal]
----
$ sudo pcs status
----
+
Both nodes must show as online, with etcd and kubelet resources started.
 
** If needed, clean the pacemaker fencing state by running the following command:
+
[source,terminal]
----
$ sudo pcs resource cleanup
----

** If needed, clean the pacemaker fencing state by running the following command:
+
[source,terminal]
----
$ sudo pcs stonith cleanup
----

. Verify all cluster Operators are available:
+
[source,terminal]
----
$ oc get co
----
+
All Operators must show `Available=True`, `Degraded=False`, and `Progressing=False`.
 
. Check the etcd Operator and `PacemakerCluster` status:
+
[source,terminal]
----
$ oc get co etcd -o yaml
$ oc get pacemakercluster cluster -o yaml
----
+
Verify the following conditions in the `PacemakerCluster` resource:
+
--
* `status.conditions` shows `Healthy`, `InService`, and `NodeCountAsExpected`.
* `status.nodes[].conditions` shows per-node conditions such as `Online` and `FencingAvailable`.
--
+
The etcd Operator condition `PacemakerHealthCheckDegraded` reflects the overall `PacemakerCluster` status.
 
. Verify the etcd member list by running the following command:
+
[NOTE]
====
Because etcd runs as a Podman container in a TNF cluster, you must SSH to the surviving node to run this check.
====
+
[source,terminal]
----
$ ssh core@<survivor_ip>
$ sudo podman exec etcd etcdctl member list -w table
----
+
Both members must show the status as `started`.
 
. SSH to both nodes and verify pacemaker resource status by running the following command:
+
[source,terminal]
----
$ sudo pcs status
----
+
. Verify that the system now meets the following conditions:
+
* Both nodes must show the status as `online`.
* etcd and kubelet resources must show the status as `Started` on both nodes.
* Fencing devices are successfully configured and are operational.
