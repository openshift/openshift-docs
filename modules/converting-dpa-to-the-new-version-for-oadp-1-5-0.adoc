= Converting DPA to the new version for OADP 1.5.0

The {oadp-first} 1.4 is not supported on {OCP-short} 4.19. You can convert Data Protection Application (DPA) to the new {oadp-short} 1.5 version by using the new `spec.configuration.nodeAgent` field and its new fields.

.Procedure

* Use the `spec.configuration.nodeAgent` parameter in DPA for configuring `nodeAgent` `daemonset`:
+
.Example spec:
[source,yaml]
----
 spec:
   configuration:
     nodeAgent:
       enable: true
       uploaderType: restic
----

* Use the new fields such as `backupPVC`, `loadConcurrency`, `podResources`, and `restorePVC` if you are using the ConfigMap resource named `node-agent-config`` for configuring Node Agent DaemonSet:
+
.Example spec:
[source,yaml]
----
 spec:
   configuration:
     nodeAgent:
       backupPVC:
         ...
       loadConcurrency:
         ...
       podResources:
         ...
       restorePVC:
        ...
----
