= Enabling {tekton-chains} to operate only in selected namespaces

[role="_abstract"]
By default, the {tekton-chains} controller watches resources in all namespaces. You can configure {tekton-chains} to run only in specific namespaces. This gives you granular control over its actions.

.Prerequisites
* You have logged in to your {OCP} cluster with `cluster-admin` privileges.

.Procedure

* In the `TektonConfig` CR, in the `chain` section, add the `--namespace=` argument to list the namespaces that the controller should watch.
+
The following example shows the configuration for the {tekton-chains} controller to only watch resources within the `dev` and `test` namespaces:
+
[source,yaml]
----
apiVersion: operator.tekton.dev/v1alpha1
kind: TektonConfig
metadata:
  name: config
spec:
  chain:
    disabled: false
    options:
      deployments:
        tekton-chains-controller:
          spec:
            template:
              spec:
                containers:
                - args:
                  - --namespace=dev, test
                  name: tekton-chains-controller
----
+
If you do not provide the `--namespace` argument or leave it empty, the controller watches all namespaces by default.
