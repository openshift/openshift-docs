= Creating a kickstart for a {microshift-short} RPM installation

Use the Kickstart provided with the {microshift-short} RPM to provision an RPM-based virtual machine.

.Prerequisites

* You set up the {microshift-short} Kickstart file.
* You set up the [Activation Keys](https://console.redhat.com/insights/connector/activation-keys) and organization ID to activate your Red Hat subscription.
//what else?

[IMPORTANT]
====
The subscription must include access to the `rhocp-4.x-for-rhel-9-$(uname -m)-rpms` and `fast-datapath-for-rhel-9-$(uname -m)-rpms` RPM repositories.
====

.Procedure

. Add the following variables to create an RPM kickstart file:

* The `RHSM_ORG` variable contains a Red Hat Subscription Manager organization ID for the subscription registration command in kickstart.
* The `RHSM_KEY` variable contains a Red Hat Subscription Manager activation key for the subscription registration command in kickstart.
* The `MICROSHIFT_VER` variable references the {microshift-short} version to install using the `4.y` format. The latest available `.z` version is automatically installed.
//Q: Can a user manually set the z-stream version?

.Example commands setting the variables.
+
[source,terminal]
----
$ export RHSM_ORG="$(cat ~/.rhsm-activation-org)"
----
+
[source,terminal]
----
$ export RHSM_KEY="$(cat ~/.rhsm-activation-key)"
----
+
[source,terminal]
----
$ export MICROSHIFT_VER=4.17 <1>
----
<1> For every minor version you set, the latest z-stream of that version is installed.

. Run the following command to create the `kickstart.ks` file to be used during the virtual machine installation:
+
[source,terminal]
----
envsubst < \
    /usr/share/microshift/kickstart/kickstart-rpm.ks.template > \
    "${HOME}/kickstart.ks"
----
