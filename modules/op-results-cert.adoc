= Preparing a secret with an SSL certificate

{tekton-results} provides a REST API using the HTTPS protocol, which requires an SSL certificate. By default, {tekton-results} creates a self-signed certificate.

Optionally, if you have an existing certificate provided by a certificate authority (CA), you can configure {tekton-results} to use this certificate.

.Prerequisites

* You have an SSL certificate provided by a CA.

.Procedure

* Create a transport security layer (TLS) secret from your certificate by entering the following command:
+
[source,terminal]
----
$ oc create secret tls -n openshift-pipelines tekton-results-tls --cert=cert.pem --key=key.pem
----
+
Replace `cert.pem` with the name of the file containing the certificate and `key.pem` with the name of the file containing the key for the certificate.

.Result

The {tekton-results} REST API endpoint uses the SSL certificate specified in the `tekton-results-tls` secret.
