= Configuring spoke clusters for records storage

You can configure spoke clusters to use the same external PostgreSQL database as the hub cluster for storing Tekton Results records.

.Prerequisites

* You have set up an external PostgreSQL database for Tekton Results.
* You have configured the hub cluster for records storage.
* You have access to each spoke cluster using an account with `cluster-admin` permissions.
* You have installed the `oc` CLI.
* You have configured the spoke clusters for multicluster support.

.Procedure

Perform the following steps on each spoke cluster.

. Create a secret containing the PostgreSQL database credentials in the `openshift-pipelines` namespace by running the following command:
+
[source,terminal]
----
$ oc create secret generic tekton-results-postgres \
  --namespace=openshift-pipelines \
  --from-literal=POSTGRES_USER=<user> \
  --from-literal=POSTGRES_PASSWORD=<password>
----
+
Replace `<user>` and `<password>` with the same PostgreSQL database credentials used on the hub cluster.
+
[NOTE]
====
The database credentials must be identical on all clusters to connect to the same PostgreSQL instance.
====

. Configure the TektonConfig custom resource to use the external PostgreSQL database by running the following command:
+
[source,terminal]
----
$ oc patch tektonconfig config --type=merge -p '
{
  "spec": {
    "result": {
      "is_external_db": true,
      "db_host": "<host>",
      "db_port": 5432,
      "db_sslmode": "prefer"
    }
  }
}'
----
+
Replace `<host>` with the hostname or IP address of your PostgreSQL database. This must be the same database used by the hub cluster.

. Wait for the Tekton Results pods to restart and become ready by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-pipelines -w
----
+
The pods restart automatically when the TektonConfig custom resource is updated.

.Verification

* Verify that the Tekton Results pods are running by running the following command:
+
[source,terminal]
----
$ oc get pods -n openshift-pipelines -l app.kubernetes.io/part-of=tekton-results
----
+
.Example output
[source,terminal]
----

tekton-results-api-7d8f9c8b5d-abcde            1/1     Running   0          2m
