= Preparing the {oci} Bastian server

A bastion host, also referred to as a jump server, enables access to a private network from 
an external network, like the internet. For the {product-title} deployment on {oci-c3-no-rt}, a bastion 
host hosts the Terraform scripts used to configure infrastructure and build {product-title} clusters. It is also used for reaching resources and instances within a subnet in your {oci-c3-short} environment.

Currently {oci-c3-short} does not support {oci} Resource Manager Stack (RMS). To run the 
Terraform script, install a Bastion Server within the targeted {product-title} cluster deployment. 
Then load the Terraform and {oci} CLI on to the bastion server for execution.

.Procedure

. Obtain the Private Cloud Appliance external silo CA chain before you can run commands. For details, see link:https://docs.oracle.com/en/engineered-systems/private-cloud-appliance/3.0-latest/user/user-usr-ce-cli.html#usr-cli-obtain-cabundle[Obtaining the Certificate Authority Bundle].

. Install the {oci} command line interface on your bastion host. For details, see link:https://app.slack.com/client/E030G10V24F/D05CCFA8UVB[Bastian Server - Prerequisites (Oracle documentation)]. *This link isn't working and also they have installation instructions for the CLI further down. Maybe we don't need this step. It's on page 6 of the oracle document*. 

. Prepare the bastion server by running the following installations:

.. Install the bastion server. For details, see link:https://app.slack.com/client/E030G10V24F/D05CCFA8UVB[Bastion Installation].

.. Install the Terraform script. For details, see link:https://app.slack.com/client/E030G10V24F/D05CCFA8UVB[Terraform installation].

.. Install and configure the {oci} command line interface (CLI). For details, see link:https://app.slack.com/client/E030G10V24F/D05CCFA8UVB[Installing and Configuring the {oci} CLI].
