= Enabling user-managed DNS

You can install a cluster with a DNS solution that you manage. As a result, you can manage the API and Ingress DNS records in your own system rather than managing the records in the cluster. For example, your organization's security policies might not allow the use of public DNS services such as Google Cloud DNS. In such scenarios, you can use your own DNS service to bypass the public DNS service. You can manage your DNS only for the IP addresses of the API and Ingress servers.

[NOTE]
====
If you enable user-managed DNS during installation, the installation program will not provision DNS records for the API and Ingress servers. To ensure that you can log in to your cluster, you must provision the DNS records for the API and Ingress servers after installation.
====

.Prerequisites

* You installed the `jq` package.
* You installed the `gcloud` CLI tool.

.Procedure
. Use a text editor to edit the `install-config.yaml` file before you deploy your cluster and add the following stanza:
.. To enable your own DNS for control plane machines, compute machines, or both:
+
[source,yaml]
----
platform:
  gcp:
    userProvisionedDNS: Enabled <1>
----
<1> Enable DNS management.

. Use the `gcloud` CLI to find the IP address of the API server and then provision the corresponding DNS record.
.. Run the following command:
+
[source,terminal]
----
$ gcloud compute forwarding-rules describe --global "${infra_id}-apiserver" --format json | jq -r .IPAddress
----

.. Use the IP address to provision your own DNS record with the hostname `api.<cluster_name>.<base_domain>.`, using your cluster name and base cluster domain.

. Use the `gcloud` CLI to find the IP address of the ingress service and then provision the corresponding DNS record.
.. To find the forwarding rule for the ingress service, run the following command:
+
[source,terminal]
----
$ ingress_forwarding_rule=$(gcloud compute target-pools list --format=json --filter="instances[]~${infra_id}" | jq -r .[].name)
----
.. To use the forwarding rule value to find the IP address of the ingress service, run the following command:
+
[source,terminal]
----
$ ingress_ip_address=$(gcloud compute forwarding-rules describe --region "${region}" "${ingress_forwarding_rule}" --format json | jq -r .IPAddress)
----
.. Use the IP address to provision your own DNS record with the hostname `*.apps.<cluster_name>.<base_domain>.`, using your cluster name and base cluster domain.
