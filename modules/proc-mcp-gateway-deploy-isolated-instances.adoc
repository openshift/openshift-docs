= Deploying isolated {mcpg} instances

[role="_abstract"]
You can deploy two isolated {mcpg} instances, each with its own `Gateway` object, `MCPGatewayExtension` custom resource (CR), and `ReferenceGrant` CR. This procedure creates isolated instances for two teams: team A and team B.

.Prerequisites

* You installed {ocp}.
* You installed {oc-first}.
* You installed the {mcpg} Operator.
* You configured at least one listener on your `Gateway` object.

.Procedure

. Set environment variables for the hostnames that each team uses:
+
[source,terminal]
----
$ export TEAM_A_HOST="team-a.apps.$(oc get dns cluster -o jsonpath='{.spec.baseDomain}')"
----
+
[source,terminal]
----
$ export TEAM_B_HOST="team-b.apps.$(oc get dns cluster -o jsonpath='{.spec.baseDomain}')"
----

. Create a namespace for team A by running the following command:
+
[source,terminal]
----
$ oc create namespace team-a
----

. Create a `Gateway` object for team A by applying the following CR:
+
[source,yaml,subs="+quotes"]
----
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: team-a-gateway
  namespace: gateway-system
spec:
  gatewayClassName: openshift-default
  listeners:
    - name: mcp
      hostname: $TEAM_A_HOST
      port: 443
      protocol: HTTPS
      allowedRoutes:
        namespaces:
          from: All
      tls:
        certificateRefs:
          - group: ''
            kind: Secret
            name: default-ingress-cert
        mode: Terminate
----

. Create a `ReferenceGrant` CR in the `Gateway` object namespace to allow the cross-namespace reference from team A:
+
[source,yaml]
----
apiVersion: gateway.networking.k8s.io/v1beta1
kind: ReferenceGrant
metadata:
  name: allow-team-a
  namespace: gateway-system
spec:
  from:
    - group: mcp.kuadrant.io
      kind: MCPGatewayExtension
      namespace: team-a
  to:
    - group: gateway.networking.k8s.io
      kind: Gateway
----

. Create an `MCPGatewayExtension` CR for team A to associate the namespace with the `Gateway` object:
+
[source,yaml]
----
apiVersion: mcp.kuadrant.io/v1alpha1
kind: MCPGatewayExtension
metadata:
  name: team-a-mcp-gateway
  namespace: team-a
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: Gateway
    name: team-a-gateway
    namespace: gateway-system
    sectionName: mcp
----

. Verify the team A deployment by running the following commands:
+
[source,terminal]
----
$ oc get gateway team-a-gateway -n gateway-system
----
+
[source,terminal]
----
$ oc wait --for=condition=Ready mcpgatewayextension/team-a-mcp-gateway -n team-a --timeout=60s
----
+
[source,terminal]
----
$ oc get pods -n team-a
----

. Repeat steps 2 through 6 for team B by creating the namespace, `Gateway` object, `ReferenceGrant` CR, and `MCPGatewayExtension` CR with the following values:
+
--
* Namespace: `team-b`
* `Gateway` name: `team-b-gateway`
* `Gateway` hostname: `$TEAM_B_HOST`
* `ReferenceGrant` name: `allow-team-b` with `namespace: team-b`
* `MCPGatewayExtension` name: `team-b-mcp-gateway` with `targetRef.name: team-b-gateway`
--
+
Verify the team B deployment by running the following commands:
+
[source,terminal]
----
$ oc get gateway team-b-gateway -n gateway-system
----
+
[source,terminal]
----
$ oc wait --for=condition=Ready mcpgatewayextension/team-b-mcp-gateway -n team-b --timeout=60s
----
+
[source,terminal]
----
$ oc get pods -n team-b
----
