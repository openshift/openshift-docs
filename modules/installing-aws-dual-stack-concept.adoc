= Dual-stack networking on AWS

[role="_abstract"]
You can deploy an {product-title} cluster on {aws-full} with dual-stack networking, which provides both IPv4 and IPv6 addresses to pods and services. Dual-stack networking on {aws-short} enables you to leverage existing IPv6 infrastructure while maintaining IPv4 connectivity.

[id="installing-aws-dual-stack-requirements_{context}"]
== Requirements for dual-stack networking on AWS

To deploy a cluster with dual-stack networking on {aws-short}, you must meet the following requirements:

Instance types:: Control plane and compute machine pool instance types must be Nitro-based and support IPv6 addressing. Generally, Nitro-based instance types already support IPv6. The default `m6i.xlarge` instance type supports dual-stack networking.

Load balancers:: All API and Ingress load balancers must be Network Load Balancers (NLBs). Classic Load Balancers do not support IPv6 addressing.

Subnets:: If you are using existing subnets by specifying the `platform.aws.vpc.subnets` parameter, all provided subnets must be dual-stack with both IPv4 and IPv6 address ranges assigned.

Networking configuration:: You must configure both IPv4 and IPv6 CIDR ranges for the `networking.clusterNetwork` and `networking.serviceNetwork` parameters in the `install-config.yaml` file.

[id="installing-aws-dual-stack-supported-regions_{context}"]
== Supported regions

Dual-stack networking is supported in the following {aws-short} region types:

* Standard commercial regions
* {aws-short} GovCloud (US) regions
* European Sovereign Cloud (EUSC) regions

[id="installing-aws-dual-stack-limitations_{context}"]
== Limitations

Consider the following limitations when deploying a dual-stack cluster on {aws-short}:

* *Local Zones:* The installer cannot automatically provision dual-stack subnets in Local Zones. To use Local Zones with dual-stack networking, you must provide existing dual-stack subnets. Additionally, only a limited number of Local Zones support IPv6. For more information about Local Zone IPv6 support, see the link:https://docs.aws.amazon.com/wavelength/latest/developerguide/wavelength-quotas.html[AWS documentation].

* *Wavelength Zones:* IPv6 addressing is not currently supported in Wavelength Zones.
