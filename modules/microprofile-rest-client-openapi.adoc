= MicroProfile REST Client for OpenAPI services

[role="_abstract"]
{ServerlessLogicProductName} uses the MicroProfile REST Client specification to call `OpenAPI` services from workflows. This approach provides a standardized way to configure and interact with REST services.

The MicroProfile REST Client is a type-safe REST client API that enables declarative REST service invocation. When you define `OpenAPI` services in your workflows, {ServerlessLogicProductName} generates REST clients based on the `OpenAPI` specifications. These clients handle the HTTP communication with external services.

{ServerlessLogicProductName} follows the MicroProfile Config specification to configure REST clients. This specification defines a standard way to externalize configuration, which allows you to manage service endpoints and other REST client properties without modifying the workflow code.

The `kogito.sw.operationIdStrategy` property determines how {ServerlessLogicProductName} generates the configuration keys for REST clients. By selecting an appropriate strategy, you can control how the REST client configuration keys are derived from the `OpenAPI` document, workflow definition, or custom aliases.
