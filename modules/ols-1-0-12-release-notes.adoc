= {ols-long} version 1.0.12

[role="_abstract"]
{ols-official} 1.0.12 provides maintenance updates, security fixes, and feature enhancements. This release ensures continued support and stability for clusters running {ocp-product-title} 4.16 and later.

[id="ols-1-0-12-enhancements_{context}"]
== Enhancements

[role="_abstract"]
{ols-official} 1.0.12 introduces security enhancements for secret handling, updated component management, and support for modern AI software development kits.

{ols-official} 1.0.12 includes the following enhancements:

//OLS-2927

* Previously, editing an attached alert created a duplicate attachment instead of replacing the original. With this update, editing a Prometheus alert that is attached to a prompt correctly updates the original attachment.

//OLS-2925

* With this update, re-attaching a resource that you have already edited in the preview modal preserves your changes. Previously, selecting the same resource from the attachment menu again replaced the edited version with the original content and discarded the edits.

//OLS-2786

* With this update, {ols-long} automatically includes the context of the current {ocp-product-title} web console page with your prompt. When you view resource list or details pages, {ols-long} uses this page information to provide more accurate and relevant responses to context-specific queries, such as "What is wrong with this deployment?"

//OLS-2724

* With this update, {ols-long} displays a visual indicator in the user interface when it performs automatic history compression. This indicator provides visibility into the compression process and its impact on response duration.
