= Sample YAML for a Cluster API infrastructure resource on {rh-openstack}

The infrastructure resource is provider-specific and defines properties that are shared by all the compute machine sets in the cluster, such as the region and subnets.
The compute machine set references this resource when creating machines.

[IMPORTANT]
====
Only the parameters in the following example are validated to be compatible with {product-title}.
Other parameters, such as those documented in the upstream link:https://cluster-api-openstack.sigs.k8s.io/clusteropenstack/configuration#optional-configuration({cap-openstack-first}) book, might cause undesired behavior.
====

[source,yaml]
----
apiVersion: infrastructure.cluster.x-k8s.io/v1beta1
kind: OpenStackCluster # <1>
metadata:
  name: <cluster_name> # <2>
  namespace: openshift-cluster-api
  labels:
    cluster.x-k8s.io/cluster-name: <cluster_name>
spec:
  controlPlaneEndpoint: <control_plane_endpoint_address> # <3>
  disableAPIServerFloatingIP: true
  tags:
    - openshiftClusterID=<cluster_name>
  network:
    id: <api_service_network_id> # <4>
  externalNetwork:
    id: <floating_network_id> # <5>
  identityRef:
    cloudName: openstack
    name: openstack-cloud-credentials
----
<1> Specify the infrastructure kind for the cluster.
This value must match the value for your platform.
<2> Specify the cluster ID as the name of the cluster. #IIUC this value is also used in the tag `openshiftClusterID=<cluster_name>` but not 100% sure.#
<3> Specify the IP address of the control plane endpoint and the port used to access it.
<4> Specify the UUID of the network for the subnet of the `APIServerInternalIP` object. #There is probably a better way to describe this. What does `APIServerInternalIP` do and how would a user know what subnet it's on?#
<5> Specify the UUID of the external network used to create floating IP addresses for load balancer virtual IP addresses (VIPs). #Stolen from other RHOSP docs, is this correct in this context?#
