= Transcript collection overview 

[role="_abstract"]

{ols-long} sends chat transcripts to {red-hat} on a set schedule. The Service uses a redaction layer to filter data before the Service shares or logs it.

By default, {ols-long} sends these transcripts every two hours. {red-hat} cannot see your original data. {ols-long} hides sensitive data before it reaches any logs.

{ols-long} saves conversation transcripts for a short time. This includes:

* Queries from the user.

* The complete message sent to the configured large language model (LLM) provider, which includes system instructions, referenced documentation, and the user question.

* The complete response from the LLM provider.

Transcripts come from your cluster and stay linked to it. {red-hat} can match these clusters to specific customer accounts. These transcripts do not contain any user data.
