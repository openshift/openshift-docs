= Supported load balancers

{product-title} supports the following load balancers:

Application Load Balancer (ALB): ALBs are designed to handle HTTP and HTTPS traffic. They operate at the application layer and provide advanced routing capabilities.

Classic Load Balancer (CLB): CLBs operate at both the transport layer and the application layer. CLBs are suitable for applications that were built within the Amazon EC2-Classic network.

Elastic Load Balancer (ELB): ELB is a general term for AWS load balancers. ELBs automatically distribute incoming application traffic across multiple targets, such as EC2 instances.

Network Load Balancer (NLB): NLBs operate at the transport layer and are designed to handle high volumes of TCP traffic with ultra-low latency.

By default, AWS routers support ELBs. Self-managed systems support CLBs. {product-title} ROSA supports NLBs. Openshift Enterprise supports CLBs, NLBs, and ALBs.

Note: ALBs are intended to be used directly in front on an application and not in front of the router. ALBs require the AWS Load Balancer Operator and are not supported for all AWS regions and OpenShift profiles
