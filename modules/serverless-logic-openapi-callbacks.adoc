= OpenAPI callbacks for asynchronous service interactions

[role="_abstract"]
OpenAPI callbacks enable asynchronous service interactions where a workflow initiates a request and receives a response later through a callback URL. The external service invokes the callback URL to send a `CloudEvent` back to the workflow, resuming execution.

An OpenAPI callback is an asynchronous, out-of-band request that a service sends to another service in response to specific events. When the workflow makes an asynchronous service call, you must give the callback URL in the request body. The external service invokes the callback URL and sends a `CloudEvent` to the workflow.

== OpenAPI callback workflow

The following process describes how OpenAPI callbacks function in {ServerlessLogicProductName}:

. The workflow initiates an asynchronous service call.
. The workflow provides a callback URL in the request body.
. Control returns to the caller immediately, as this is an asynchronous operation.
. The external service processes the request.
. The external service invokes the callback URL with a `CloudEvent` to resume the workflow.

== Event identification and correlation

For the workflow to identify and route the correct `CloudEvent`, one of the following conditions must be met:

* The external service includes the workflow instance ID in the `CloudEvent` header.
* Event correlation mechanisms are configured.

Each workflow instance has a unique identifier that is generated automatically. This identifier enables proper event routing to the correct workflow instance when multiple instances are running.

== OpenAPI callbacks compared to Callback states

Both OpenAPI callbacks and Callback states support fire-and-wait-for-result operations. However, they differ in their approach:

OpenAPI callbacks:: The workflow provides a callback endpoint in the initial request. The external service uses this endpoint to send a `CloudEvent` back to the workflow after completing the operation.

Callback states:: External services independently publish `CloudEvent` without receiving a callback endpoint from the workflow. The workflow waits for matching events based on correlation criteria.

The choice between OpenAPI callbacks and Callback states depends on whether the external service can accept and use a callback URL provided by the workflow.
