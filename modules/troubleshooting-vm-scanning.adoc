= Troubleshooting virtual machine scanning

[role="_abstract"]
If you have problems when viewing vulnerability results for VMs, you can try some common troubleshooting steps. You can try different steps depending on the type of issue you are having, for example, connectivity or performance issues.

.Procedure

* If `roxagent` cannot connect to the host, try these steps:
** Verify `vsock` is enabled in the VM configuration.
** Check that the relay is running on the host node.
** Ensure `vsock` kernel modules are loaded.
** Verify port configuration matches between the `roxagent` and relay.
* If no packages are found, try these steps:
** Verify that `--host-path` points to the correct filesystem location.
** Verify rpm and dnf databases exist and are readable by `roxagent`.
** Use the `--verbose` flag to examine index reports.
** Confirm that the roxagent has sudo privileges.
* If vulnerability data does not appear in {product-title-short}, try these steps:
** Verify the feature flag is enabled on all components.
** Check Central logs for enrichment processing.
** Ensure the VM has a valid {op-system-base} subscription.
** Confirm repository-to-CPE mapping downloads are successful.
* If there is a network connectivity problem:
** Verify the VM has internet access or connectivity to Sensor.
** Check the firewall rules for repository access.
** Validate DNS resolution from within the VM.
* If you observe performance issues, such as large memory issues, try these steps:
** Adjust ROX_VIRTUAL_MACHINES_VSOCK_CONN_MAX_SIZE_KB to a lower number.
** Monitor relay resource consumption.
** Consider reducing scan frequency in daemon mode.
* If you notice slow scan times, try these steps:
** Verify metal nodes are used for VM hosting.
** Check network latency for repository downloads.
** Monitor disk I/O during package database reads.
