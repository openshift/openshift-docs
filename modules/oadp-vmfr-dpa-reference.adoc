= DataProtectionApplication vmFileRestore reference

[role="_abstract"]
The `vmFileRestore` section in the `DataProtectionApplication` custom resource enables {oadp-short} virtual machine file restore controllers and custom resource definitions.
Understanding this configuration helps administrators activate VMFR capabilities and verify proper deployment.

.vmFileRestore configuration fields
[cols="1,1,2",options="header"]
|===
|Field
|Type
|Description

|`spec.vmFileRestore.enable`
|boolean
|Enables or disables the VMFR feature.
Set to `true` to deploy VMFR controllers and CRDs.
Default value is `false`.
|===

.Controller deployment details
When `vmFileRestore.enable` is set to `true`, the {oadp-short} Operator deploys the following components:

* `oadp-vm-file-restore-controller-manager` pod in the `openshift-adp` namespace
* `VirtualMachineBackupsDiscovery` CRD
* `VirtualMachineFileRestore` CRD
* Cluster roles and role bindings for VMFR controllers

.Prerequisites for vmFileRestore
The following DPA configuration is required before enabling VMFR:

* The `kubevirt` plugin must be in the `spec.configuration.velero.defaultPlugins` list
* Node Agent must be enabled with `spec.configuration.nodeAgent.enable: true`
* Node Agent uploader type must be `kopia` with `spec.configuration.nodeAgent.uploaderType: kopia`
* File system backup must be enabled with `spec.configuration.velero.disableFsBackup: false`

.Example DataProtectionApplication CR with VMFR enabled
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: DataProtectionApplication
metadata:
  name: oadp-backup
  namespace: openshift-adp
spec:
  configuration:
    nodeAgent:
      enable: true
      uploaderType: kopia
    velero:
      defaultPlugins:
        - kubevirt
        - csi
        - openshift
      disableFsBackup: false
  vmFileRestore:
    enable: true <1>
  backupLocations:
    - velero:
        provider: aws
        default: true
        objectStorage:
          bucket: my-backup-bucket
----
<1> Enables VMFR controllers and CRDs.

.Verification
After enabling VMFR, verify deployment by checking:

* DPA status includes condition `type: VMFileRestoreReady` with `status: "True"`
* Pod `oadp-vm-file-restore-controller-manager` is running in `openshift-adp` namespace
* CRDs `virtualmachinebackupsdiscoveries.oadp.openshift.io` and `virtualmachinefilerestores.oadp.openshift.io` are registered
