= Binding PCI devices to the VFIO driver
To bind PCI devices to the VFIO driver, obtain the values for `vendor-ID` and `device-ID` from each device and create a list with the values. Add this list to the `MachineConfig` object file. The `MachineConfig` Operator generates the `/etc/modprobe.d/vfio.conf` on the nodes with the PCI devices, and binds the PCI devices to the VFIO driver.

.Prerequisites
* Adding kernel arguments to enable IOMMU for the CPU.

.Procedure
. Create the `MachineConfig` object file to enable the binding of PCI devices to the VFIO driver.
+
.Example
[source,yaml]
----
apiVersion: machineconfiguration.openshift.io/v1
kind: MachineConfig <1>
metadata:
  labels:
    machineconfiguration.openshift.io/role: worker <2>
  name: 100-worker-vfiopci-configuration <3>
spec:
  config:
    ignition:
      version: 3.2.0
    storage:
      files:
      - contents:
          source: data:text/plain;charset=utf-8;base64,b3B0aW9ucyB2ZmlvLXBjaSBpZHM9MTBkZToxZWI4Cg== <4>
        mode: 420 <5>
        overwrite: true <6>
        path: /etc/modprobe.d/vfio.conf <7>
      - contents:
          source: data:text/plain;charset=utf-8;base64,dmZpby1wY2kK <8>
        mode: 420
        overwrite: true
        path: /etc/modules-load.d/vfio-pci.conf <9>

----
<1> The type of object being configured is `MachineConfig`.
<2> Applies the new kernel argument only to worker nodes.
<3> Named to identify where this `MachineConfig` object fits among the other `MachineConfig` objects.
<3> The number (100) indicates where this `MachineConfig` object fits among all `MachineConfig` objects. In this example, the name indicates the purpose of the `MachineConfig` object.
<4> The `echo "options vfio-pci ids=10de:1eb8" | base64` command creates the encoded text. The `vendor-ID` value (10de) and the `device-ID` value (1eb8) applies to a single device that binds to the VFIO driver. You can add a list of multiple devices for `source` with their vendor and device information.
<5> The values specified for `mode` determine the permissions for the file.
<6> Description needed. When `mode` is set to `true`, is there a default file that is overwritten by the `/etc/modprobe.d/vfio.conf` file? If that's not correct, please provide details.
<7> The `/etc/modprobe.d/vfio.conf` was created by the `echo "options vfio-pci ids=10de:1eb8 > /etc/modprobe.d/vfio/conf"` command.
<8> The `echo "dmZpby1wY2kK" | base64 -d` command and the  `vfio-pci` commands create the encoded text for `source` as `dmZpby1wY2kK`. The encoded text translates to `vfio-pci`.
<9> The file that loads the vfio-pci kernel module on the worker nodes.

. Create a `MachineConfig` object file that identifies the kernel argument for the IOMMU configuration:
+
[source,terminal]
----
$ oc create -f 100-worker-kernel-arg-iommu.yaml
----

. Verify that the `MachineConfig` object file was added.
+
[source,terminal]
----
$ oc get MachineConfig
----
+
.Example output

. Use the `lspci` command to obtain the `vendor-ID` and the `device-ID` for the PCI device.
+
[source, terminal]
----
$ lspci -nnv | grep -i nvidia
----
.Example output
[source, terminal]
02:01.0 3D controller [0302]: NVIDIA Corporation GV100GL [Tesla V100 PCIe 32GB] [10de:1eb8] (rev a1)

. Specify the values for `vendor-ID` and `device-ID` and convert the `/etc/modprobe.d/vfio.conf` file to base64 format:
+
[source,terminal]
----
$ echo "options vfio-pci ids=10de:1eb8 | base64
----
These values are converted into the `base64 format` and the output is encoded.
+
.Example output
[source,terminal]
----
b3B0aW9ucyB2ZmlvLXBjaSBpZHM9MTBkZToxZWI4Cg==
----

. Add the encoded text to `source` in the `MachineConfig` object file.

. Convert `vfio-pci` into base64 format and encode the text:
+
.Example output
[source,terminal]
----
$ echo vfio-pci | base64
----
These values are converted into the `base64 format` and the output is encoded.
+
.Example output
[source,terminal]
----
dmZpby1wY2kK
----
. Add the encoded text to `source` in the `MachineConfig` object file.

.Verification
* Verify that the VFIO driver is loaded.
+
[source,terminal]
----
$ lspci -nnk -d 10de:
----
The output confirms that the VFIO driver is being used.
+
.Example output
----
04:00.0 3D controller [0302]: NVIDIA Corporation GP102GL [Tesla P40] [10de:1eb8] (rev a1)
        Subsystem: NVIDIA Corporation Device [10de:1eb8]
        Kernel driver in use: vfio-pci
        Kernel modules: nouveau
----
