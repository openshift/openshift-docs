= Configuring a GNR-D boundary clock without holdover

[role="_abstract"]
Configure a Precision Time Protocol (PTP) boundary clock without holdover on GNR-D hardware by identifying the required interfaces and applying a `PtpConfig` custom resource (CR).

.Prerequisites

* You have a GNR-D system with a timing module installed.
* You have one or more Carter Flats (E830) NICs installed.
* You have installed {product-title} 4.20.15 or later.
* You have installed the PTP Operator.
* You have applied the ZTP RAN profile.
* You have configured the GNR-D interface mapping MachineConfig.
* You have access to a PTP Grand Master time source.

.Procedure

. Identify the PTP-capable interfaces on the node by running the following command:
+
[source,terminal]
----
oc describe -n openshift-ptp nodeptpdevices.ptp.openshift.io
----
+
Use the command output to determine the interface naming:
+
* NAC interfaces use names such as `eno*`
* Carter Flats interfaces use names such as `enp*s0f*`

. Identify the leading interface associated with the PHC by determining which NAC interface maps to the PHC device on the node.
+
Locate the NAC interface that is associated with the PHC device and record this interface name as the leading interface.

. Select the interfaces for synchronization based on your hardware topology.
+
Assign roles to the interfaces as follows:
+
* Select one NAC interface as the `TimeReceiver` (upstream port).
* Select NAC interfaces for `TimeTransmitter` roles.
* Select Carter Flats interfaces for `TimeTransmitter` roles.

. Create the `PtpConfig` custom resource by updating the configuration with your interface names.
+
Ensure that you define the `TimeReceiver` and `TimeTransmitter` interfaces correctly based on your topology.
+
Use the full configuration provided by your hardware setup or internal examples.

. Apply the configuration to the cluster by running the following command:
+
[source,terminal]
----
oc apply -f gnrd-bc-ptpconfig.yaml
----

. Verify that the configuration is applied successfully by running the following command:
+
[source,terminal]
----
oc get ptpconfig -n openshift-ptp
----

.Verification

. Verify that the `PtpConfig` resource is created successfully by running the following command:
+
[source,terminal]
----
oc get ptpconfig -n openshift-ptp
----
+
Ensure that the `gnrd-bc` resource is listed.

. Verify that the PTP pods are running by running the following command:
+
[source,terminal]
----
oc get pods -n openshift-ptp
----
+
Ensure that the PTP Operator and related pods are in the `Running` state.

. Verify that the node is synchronized with the PTP Grand Master by checking the PTP metrics or logs.
+
Check that the reported clock class matches the Grand Master clock class.

. Validate synchronization on `TimeTransmitter` interfaces by connecting a downstream device.
+
Confirm that the downstream device synchronizes successfully with the transmitted time.
