= Creating a route for OIDC

The Zero Trust Workload Identity Manager controller can automatically create OIDC routes that enable external access to the SPIRE OIDC Discovery Provider endpoints through OpenShift Routes. When a `SpireOIDCDiscoveryProvider` is deployed, the enhanced controller automatically creates routes to provide secure, externally accessible OIDC endpoints for workload identity verification and JWT token validation in multi-cluster and hybrid cloud environments.

.Prerequisites

* Either the Vault or Entra ID OIDC Provider is installed.

* An externally reachable hostname that is used for the route.

* A Transport Layer Socket (TLS) and its corresponding private key need to be installed if creating an HTTPS route.

.Procedure

. Create the `SpireOIDCDiscoveryProvider.yaml` Custom Resource (CR) file.
+
[source,yaml]
----
apiVersion: operator.openshift.io/v1alpha1
kind: SpireOIDCDiscoveryProvider
metadata:
 name: cluster
spec:
 trustDomain: cluster.local
 managedRoute: "true" <1>
 jwtIssuer: <custom-issuer> <2>

----
+
<1> Defaults to `true`. If set to `false`, external access must be configured manually.
<2> Custom JWT issuer domain. If not set, defaults to `https://oidc-discovery.$trustDomain`.

. Apply the configuration by running the following command:
+
[source, terminal]
----
$ oc apply -f SpireOIDCDiscoveryProvider.yaml
----

. Access the endpoint URL at `https://<route-hostname>/.well-known/openid_configuration`. If you set the `jwtIssuer` parameter, the hostname matches the issuer host.

. If you did not set the `jwtIssuer` parameter, you can determine the route hostname by running the following command:
+
[source, terminal]
----
$ oc get route spire-oidc-discovery-provider -o jsonpath='{.spec.host}'
----

.Verification

. Get the route certificate information by running the following command:
+
[source, terminal]
----
$ oc get route spire-oidc-discovery-provider -o jsonpath='{.spec.tls}' | jq
----

. Get the service Certificate Authority (CA) certificate to verify the internal trust chain by running the following command:
+
[source, terminal]
----
$ oc get secret oidc-serving-cert -o jsonpath='{.data.tls\.crt}' | base64 -d
----

. Make sure that the route hostname is available by running the following command:
+
[source, terminal]
----
oc get route spire-oidc-discovery-provider -o jsonpath='{.spec.host}'
----
