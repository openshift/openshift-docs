= Enable API catalog integration in {rhdh}

[role="_abstract"]
When you want developers to discover and request access to {prodname}-protected APIs directly from {rhdh}, integrate the {prodname} plugin. The plugin synchronizes API products into the {rhdh} catalog and enables self-service API key management.

{rhdh} pulls the {rhcl} plugin directly from `registry.redhat.io` as OCI artifacts. This method requires a registry authentication secret in the same {ocp} project as your {rhdh} deployment.

[IMPORTANT]
====
Replace API Key features in the example dynamic plugin configuration YAML and enforce security, such as OpenID Connect (OIDC) or JSON web token (JWT) `AuthPolicy` objects, for production environments.
====

.Prerequisites

* You installed {prodname} 1.4 or later on an {ocp} cluster.
* You installed {rhdh} on the same {ocp} cluster.
* You have a Red Hat account with access to `registry.redhat.io`.
* You have a registry service account token from the Red Hat Customer Portal.
* You have cluster privileges to configure role-based-access (RBAC) policies and service accounts.
* You installed the {oc-first}.
* Your organization has APIs published through {prodname} gateways that you want to make discoverable to development teams.

.Procedure

. Create an `auth.json` file on your local machine with your `registry.redhat.io credentials` by using the following example:
+
[source,json]
----
{
  "auths": {
    "registry.redhat.io": {
      "auth": "<base64-encoded-username:password>"
    }
  }
}
----
+
To generate the `base64-encoded` value, use the `printf '%s' '<username>:<password>' | base64` command.

. Plan the `metadata.name` value for your planned `Backstage` custom resource (CR). You must use this value when creating Secrets.
+
[NOTE]
====
By default, standard {rhdh} deployments use `developer-hub` as the `Backstage` CR name. The {rhdh} Operator automatically looks for an authentication secret named `<backstage_cr_name>-dynamic-plugins-registry-auth` during plugin installation.
====

. Create the authentication Secret in the {ocp} project where you installed {rhdh} using one of the two following methods:
+
.. For a default Operator installation with name `developer-hub`, run the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc create secret generic developer-hub-dynamic-plugins-registry-auth \
  --from-file=auth.json=./auth.json
----
+
.. If you already have Podman credentials configured locally, run the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc create secret generic <backstage_cr_name>-dynamic-plugins-registry-auth \
  --from-file=auth.json=${XDG_RUNTIME_DIR}/containers/auth.json
----
+
Replace `<backstage_cr_name>` with the `metadata.name` value you plan to assign to your `Backstage` CR.
+
[IMPORTANT]
====
The Secret name must follow the pattern `<backstage_cr_name>-dynamic-plugins-registry-auth`, where `<backstage_cr_name>` matches the `metadata.name` value of your `Backstage` custom resource (CR). You must create the Secret before you configure your plugin.
====

. Verify that the Secret exists in your {ocp} project by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc get secret <backstage_cr_name>-dynamic-plugins-registry-auth
----
+
Replace `<backstage_cr_name>` with the `metadata.name` value you plan to assign to your `Backstage` CR.

. Set the environment variable for your {rhdh} namespace by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ export RHDH_NS=rhdh
----

. Set the environment variable for your cluster base hostname by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ export CLUSTER_HOSTNAME=$(oc get ingress.config.openshift.io cluster -o jsonpath='{.spec.domain}')
----

. Create a namespace for your {rhdh} instance by running the following command:
+
[source,terminal]
----
$ echo "apiVersion: v1
kind: Namespace
metadata:
  name: $RHDH_NS" | envsubst | oc apply -f -
----

. Create the dynamic plugin configuration files, for example, `_<dynamic_plugins_rhdh.yaml>_`, by using the following example:
+
.Example dynamic plugin configuration YAML
[source,yaml,subs="+quotes"]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: _<dynamic_plugins_rhdh>_
  namespace: $RHDH_NS
data:
  dynamic-plugins.yaml: |
includes:
  - dynamic-plugins.default.yaml
plugins:
   - package: oci://registry.redhat.io/rhdh/kuadrant-backstage-plugin-backend-dynamic-rhel9:bs_1.4__v0.2.1!kuadrant-backstage-plugin-backend-dynamic
     disabled: false
   - package: oci://registry.redhat.io/rhdh/kuadrant-backstage-plugin-frontend-dynamic-rhel9:bs_1.4__v0.2.1!kuadrant-backstage-plugin-frontend-dynamic
     disabled: false
     pluginConfig:
     dynamicPlugins:
       frontend:
         kuadrant.kuadrant-backstage-plugin-frontend:
           apiFactories:
             - importName: kuadrantApiFactory
           appIcons:
             - name: kuadrantIcon
               importName: KuadrantIcon
             - name: apiIcon
               importName: ApiIcon
             - name: keyIcon
               importName: KeyIcon
             - name: approvalIcon
               importName: ApprovalIcon
           menuItems:
             kuadrant.api-products:
               parent: kuadrant
             kuadrant.my-api-keys:
               parent: kuadrant
             kuadrant.api-key-approval:
               parent: kuadrant
           dynamicRoutes:
             - path: /kuadrant
               importName: KuadrantPage
               menuItem:
                 icon: kuadrantIcon
                 text: Kuadrant
             - path: /kuadrant/api-products
               importName: ApiProductsPage
               menuItem:
                 icon: apiIcon
                 text: API Products
             - path: /kuadrant/my-api-keys
               importName: MyApiKeysPage
               menuItem:
                 icon: keyIcon
                 text: My API Keys
             - path: /kuadrant/api-key-approval
               importName: ApiKeyApprovalPage
               menuItem:
                 icon: approvalIcon
                 text: API Key Approval
             - path: /kuadrant/api-products/:namespace/:name
               importName: ApiProductDetailPage
             - path: /kuadrant/api-keys/:namespace/:name
               importName: ApiKeyDetailPage
           entityTabs:
             - mountPoint: entity.page.api-keys
               path: /api-keys
               title: API Keys
             - mountPoint: entity.page.api-product-info
               path: /api-product-info
               title: API Product Info
           mountPoints:
             - mountPoint: entity.page.api-keys/cards
               importName: EntityKuadrantApiKeyManagementTab
               config:
                 layout:
                   gridColumn: "1 / -1"
                 if:
                   allOf:
                     - isKind: api
             - mountPoint: entity.page.api-product-info/cards
               importName: EntityKuadrantApiProductInfoContent
               config:
                 layout:
                   gridColumn: "1 / -1"
                 if:
                   allOf:
                     - isKind: api
----
+
* Replace the value of `metadata.name` with the name you used.
* Replace the environment variables with actual values.
+
[IMPORTANT]
====
Enter your {rhdh} version of Backstage and the plugin version, in the format `bs_<backstage-version>__<plugin-version>`; use the double underscore delimiter.

* Find your Backstage version in the link:https://docs.redhat.com/en/documentation/red_hat_developer_hub/{rhdh-v}/html/red_hat_developer_hub_release_notes/pr01[Red Hat Developer Hub release notes preface].
* Locate the plugin version in the link:https://docs.redhat.com/en/documentation/red_hat_developer_hub/{rhdh-v}/html-single/dynamic_plugins_reference/index[Dynamic Plugins Reference].
====

. Create the `ConfigMap` CR by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ oc create configmap dynamic-plugins-rhdh --from-file=_<dynamic_plugins_rhdh>_ --namespace=$RHDH_NS
----
+
Replace `_<dynamic_plugins_rhdh>_` with the filename you used.
