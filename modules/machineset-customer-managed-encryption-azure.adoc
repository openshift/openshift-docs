= Enabling customer-managed encryption keys for a machine set

Azure allows users to supply an encryption key to encrypt data on disks at rest. You can enable encryption with a customer-managed key by using the Machine API.

A disk encryption set, a key vault, and an encryption key are required to use a customer-managed key. It is assumed that the disk encryption set presides in a resource group where the Cloud Credential Operator (CCO) has granted permissions. If not, an additional reader role is required to be granted on the disk encryption set.

If you do not already have a Microsoft Azure key vault, you must first link:https://docs.microsoft.com/en-us/azure/virtual-machines/windows/disk-encryption-key-vault#create-a-key-vault[create a key vault] and set the key vault advanced access policies. You can also link:https://docs.microsoft.com/en-us/azure/virtual-machines/windows/disk-encryption-key-vault#set-up-a-key-encryption-key-kek[set up a key encryption key (KEK)] for an additional layer of security for encryption keys.
