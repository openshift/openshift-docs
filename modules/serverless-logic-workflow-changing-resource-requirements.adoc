= Changing resource requirements

You can create or edit a `SonataFlowPlatform` in the workflow namespace to specify resource requirements for the internal builder pods.

.Example of SonataFlowPlatform
[source,yaml]
----
apiVersion: sonataflow.org/v1alpha08
kind: SonataFlowPlatform
metadata:
  name: sonataflow-platform
spec:
  build:
    template:
      resources:
        requests:
          memory: "64Mi"
          cpu: "250m"
        limits:
          memory: "128Mi"
          cpu: "500m"
----

[NOTE]
====
Only one `SonataFlowPlatform` is allowed per namespace. Fetch and edit the one the {ServerlessLogicOperatorName} created for you instead of trying to create another one.
====

You can fine-tune the resource requirements for a particular workflow. Each workflow instance will have a `SonataFlowBuild` instance created with the same name as the workflow. You can edit the `SonataFlowBuild` custom resource and specify the parameters as follows:

.Example of SonataFlowBuild
[source,yaml]
----
apiVersion: sonataflow.org/v1alpha08
kind: SonataFlowBuild
metadata:
  name: my-workflow
spec:
  resources:
    requests:
      memory: "64Mi"
      cpu: "250m"
    limits:
      memory: "128Mi"
      cpu: "500m"
----

These parameters will apply only to new build instances.
