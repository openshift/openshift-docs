= Configuring a hard concurrency limit

[role="_abstract"]
A hard concurrency limit sets a strict upper bound on the number of requests. When concurrency reaches this limit, the system buffers additional requests until capacity becomes available. You can specify a hard concurrency limit for your Knative service by modifying the `containerConcurrency` spec, or by using the `kn service` command with the correct flags.

.Procedure

* Optional: Set the `containerConcurrency` spec for your Knative service in the spec of the `Service` custom resource:
+
.Example service spec
[source,yaml]
----
apiVersion: serving.knative.dev/v1
kind: Service
metadata:
  name: showcase
  namespace: default
spec:
  template:
    spec:
      containerConcurrency: 50
----
+
The default value is `0`, which means that the system does not limit the number of simultaneous requests that can reach a single service replica.
+
A value greater than `0` sets the exact number of requests that can reach a single service replica at one time. In this example, the system limits concurrency to 50 requests.

* Optional: Use the `kn service` command to specify the `--concurrency-limit` flag:
+
[source,terminal]
----
$ kn service create <service_name> --image <image_uri> --concurrency-limit <integer>
----
+
.Example command to create a service with a concurrency limit of 50 requests
[source,terminal]
----
$ kn service create showcase --image quay.io/openshift-knative/showcase --concurrency-limit 50
----
