= Configuring the SPIRE Vault UpstreamAuthority plugin

[role="_abstract"]
To automate certificate provisioning and integrate SPIRE with your existing Vault PKI infrastructure, use this procedure. This procedure configures SPIRE Server to obtain intermediate signing certificates from HashiCorp Vault by using the Vault UpstreamAuthority plugin with token authentication.

.Prerequisites

* You have completed the preparation steps, including Vault PKI Engine setup and policy configuration.

* You have a running HashiCorp Vault instance with the PKI Engine enabled at the specified mount point.

* You have configured one of the supported authentication methods (token, AppRole, Kubernetes, or client certificate).

* Your authentication credentials have the required Vault policy attached with `update` capability for the `pki/root/sign-intermediate` path.

* SPIRE Server is installed in your environment.

* You have access to edit the SPIRE Server configuration file or ConfigMap.

.Procedure

. Obtain the CA certificate from your Vault server if you are using a custom certificate authority by running the following command:
+
[source,terminal]
----
$ curl -o /path/to/vault-ca-cert.pem https://vault.example.com:8200/v1/pki/ca/pem
----

. Edit the SPIRE Server configuration file to add the Vault UpstreamAuthority plugin configuration by running the following command:
+
[source,terminal]
----
plugins {
  UpstreamAuthority "vault" {
    plugin_cmd = "/path/to/vault-plugin"
    plugin_checksum = "sha256-checksum-of-plugin"
    plugin_data {
      vault_addr = "https://vault.example.com:8200"
      pki_mount_point = "pki"
      ca_cert_path = "/path/to/vault-ca-cert.pem"
      token_auth {
        token = "${VAULT_TOKEN}"
      }
    }
  }
}
----
+
where

plugin_data.vault_addr:: Specifies the URL of the Vault server. Defaults to the `VAULT_ADDR` environment variable if not specified. Example: `https://vault.example.com:8200`

plugin_data.pki_mount_point:: Specifies the path where the PKI secrets engine is mounted in Vault. The default is `pki`.

plugin_data.ca_cert_path:: Specifies the path to a CA certificate file used to verify the Vault server certificate. Replace `/path/to/vault-ca-cert.pem` with the actual path to your Vault CA certificate. The `${VAULT_TOKEN}` environment variable is substituted at runtime. Must be in PEM format.

plugin_data.token_auth.token:: Specifies the Vault authentication token. Supports environment variable substitution by using `${VAULT_TOKEN}`.

. Set the Vault token as an environment variable for the SPIRE Server process by running the following command:
+
[source,terminal]
----
$ export VAULT_TOKEN="your-vault-token"
----

. Configure the SPIRE Server's `ca_ttl` setting to ensure it does not exceed the Vault PKI engine's maximum lease TTL by running the following command:
+
[source,hcl]
----
server {
  ca_ttl = "24h"
  ca_subject {
    country = ["US"]
    organization = ["EXAMPLE"]
    common_name = "SPIRE Server"
  }
}
----

. Restart the SPIRE Server to apply the configuration changes by running the following command:
+
[source,terminal]
----
$ systemctl restart spire-server
----

.Verification

. Verify that the SPIRE Server started successfully and connected to Vault by running the following command:
+
[source,terminal]
----
$ journalctl -u spire-server -n 50
----
+
.Example output
[source,terminal]
----
time="2026-04-07T10:15:30Z" level=info msg="Upstream authority loaded" subsystem_name=ca
time="2026-04-07T10:15:31Z" level=info msg="Server CA activated" certificate_fingerprint="ABC123..."
----

. Verify that SPIRE Server successfully obtained a signed intermediate certificate from Vault by running the following command:
+
[source,terminal]
----
$ /opt/spire/bin/spire-server healthcheck
----
+
.Example output
[source,terminal]
----
Server is healthy.
----

. Inspect the SPIRE Server CA bundle to confirm the Vault-signed certificate is present by running the following command:
+
[source,terminal]
----
$ /opt/spire/bin/spire-server bundle show
----
