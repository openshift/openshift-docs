= About Service and Deployment objects

[role="_abstract"]
Before you can create a route to host your application at a public URL, you must create a `Service` custom resource (CR) as a routing rule. As a best practice, you must also create a `Deployment` object that is the application pod.

The following is an example of both `Deployment` and `Service` CRs for the Toystore example application that you can use as reference in creating your own.

.Example application Deployment and Service CRs
[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: toystore
  labels:
    app: toystore
spec:
  selector:
    matchLabels:
      app: toystore
  template:
    metadata:
      labels:
        app: toystore
    spec:
      containers:
        - name: toystore
          image: quay.io/kuadrant/authorino-examples:api
          env:
            - name: LOG_LEVEL
              value: "debug"
            - name: PORT
              value: "3000"
          ports:
            - containerPort: 3000
              name: http
  replicas: 1
---
apiVersion: v1
kind: Service
metadata:
  name: toystore
spec:
  selector:
    app: toystore
  ports:
    - name: http
      port: 80
      protocol: TCP
      targetPort: 3000
----
