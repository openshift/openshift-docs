= Resolve build pod failures with RuntimeClass not found error

[role="_abstract"]
If events report that the `runtimeClass` is missing, the operator setup might be incomplete.

.Prerequisites

* You have configured RuntimeClass isolation for your builds.
* You have OpenShift Sandboxed Containers installed.

.Procedure

. Confirm the operator is active:
+
[source,terminal]
----
$ oc get csv -n openshift-sandboxed-containers-operator
----

. Check the `KataConfig` installation status:
+
[source,terminal]
----
$ oc get kataconfig -o jsonpath='{.items[0].status.installationStatus}'
----

. Ensure all worker nodes have completed the mandatory reboot process.

.Verification

* Run the following command to verify the `RuntimeClass` resource exists:
+
[source,terminal]
----
$ oc get runtimeclass kata
----
+
Example output:
[source,terminal]
----
NAME   HANDLER   AGE
kata   kata      5m
----
