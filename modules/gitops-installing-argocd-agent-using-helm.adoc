= Installing the Argo CD Agent by using a Helm chart

Use the Helm chart provided by the Argo CD Agent project to install the Agent component on the workload cluster.

.Procedure

. Create the Agent secret on the Principal cluster.

.. To create a secret on the Principal cluster that contains configuration details for the Agent, run the following command:
+
[source,terminal]
----
$ argocd-agentctl agent create "<AGENT_NAME>" \
  --principal-context "<PRINCIPAL_CONTEXT>" \
  --principal-namespace "<PRINCIPAL_NAMESPACE>" \
  --resource-proxy-server "<PRINCIPAL_RESOURCE_PROXY_URL>"
----
+
where:

`<resource-proxy-server>`:: Specifies the address of the Principal's `resource-proxy` service that the Agent connects to.
`<label, l>`:: Adds optional metadata labels for the Agent.
`<principal-namespace>`:: Specifies the namespace where the Principal component is installed.
`<principal-context>`:: Specifies the cluster context of the Principal component.
+
This command generates a client certificate signed by the Principal CA, creates an Argo CD cluster secret with the certificate data, and sets up credentials for resource proxy access.

. Copy the CA certificate to the Agent cluster.

.. To copy the Principal CA certificate to the Agent cluster for validation, run the following command:
+
[source,terminal]
----
$ argocd-agentctl pki propagate \
    --agent-context "<AGENT_CONTEXT>" \
    --principal-context "<PRINCIPAL_CONTEXT>" \
    --principal-namespace "<PRINCIPAL_NAMESPACE>" \
    --agent-namespace "<AGENT_NAMESPACE>"
----
+
where:

`<force, -f>`:: Forces regeneration of the PKI if it already exists on the target cluster.
`<principal-namespace>`:: Specifies the namespace where the Principal component is installed.
`<agent-namespace>`:: Specifies the namespace where the Agent component is installed.
`<principal-context>`:: Specifies the kubeconfig context of the Principal (hub) cluster.
`<agent-context>`:: Specifies the kubeconfig context of the Agent (workload) cluster.
+
This command copies the CA certificate from the Principal cluster and creates the `argocd-agent-ca` secret in the Agent cluster.

. Generate a client certificate for the Agent.

.. To generate a client certificate and private key for the Agent, run the following command:
+
[source,terminal]
----
$ argocd-agentctl pki issue agent agent-managed \
    --principal-context "<PRINCIPAL_CONTEXT>" \
    --agent-context "<AGENT_CONTEXT>" \
    --agent-namespace "<AGENT_NAMESPACE>"
----
+
where:

`<upsert, -u>`:: Updates the existing JWT key if one already exists in the Agent namespace.
`<principal-context>`:: Specifies the kubeconfig context of the Principal (hub) cluster.
`<agent-context>`:: Specifies the kubeconfig context of the Agent (workload) cluster.
`<agent-namespace>`:: Specifies the namespace where the Agent component is installed.
+
This command creates the `argocd-agent-client-tls` secret, generates the Agent's client certificate and private key, and signs the certificate using the Principal CA.

. Verify the required secrets.

.. To verify that all required secrets are created in the Agent namespace, run the following command:
+
[source,terminal]
----
$ oc get secrets -n "<AGENT_NAMESPACE>" --context "<AGENT_CONTEXT>" | grep agent
----

... As a result, the following output is displayed:
+
[source,terminal]
----
NAME                              TYPE
argocd-agent-ca                   kubernetes.io/tls
argocd-agent-principal-tls        kubernetes.io/tls
argocd-agent-resource-proxy-tls   kubernetes.io/tls
argocd-agent-jwt                  Opaque
----

. Deploy the Argo CD instance

.. To deploy an Argo CD instance on the Agent cluster, create an Argo CD custom resource similar to the following example:
+
[source,yaml]
----
apiVersion: argoproj.io/v1beta1
kind: ArgoCD
metadata:
  name: openshift-gitops
spec:
  server:
    enabled: false
  sourceNamespaces:
    - "<APPLICATION_NAMESPACE>"
----

.. Apply the configuration by running the following command:
+
[source,terminal]
----
$ oc apply -f argocd.yaml -n "<AGENT_NAMESPACE>"
----

.. To verify that the Argo CD instance is created, run the following command:
+
[source,terminal]
----
$ oc get pod -n "<AGENT_NAMESPACE>" --context "<AGENT_CONTEXT>"
----

... As a result, the following output is displayed:
+
[source,terminal]
----
NAME                                           READY   STATUS
openshift-gitops-application-controller-0      1/1     Running
openshift-gitops-redis-5b6668544-sxtth         1/1     Running
openshift-gitops-repo-server-8cb87b698-cnfgl   1/1     Running
----

. Deploy the Agent by using the Helm chart

.. To deploy the Agent using the provided Helm chart, run the following command:
+
[source,terminal]
----
$ helm install argocd-agent ghcr.io/argoproj-labs/argocd-agent/argocd-agent-agent-helm \
  --version 0.1.0 \
  --set namespaceOverride=<AGENT_NAMESPACE> \
  --kube-context <AGENT_CONTEXT>
----

.. To verify that the Agent deployment is created, run the following command:
+
[source,terminal]
----
$ oc get pod -n "<AGENT_NAMESPACE>" --context "<AGENT_CONTEXT>"
----

... As a result, the following output is displayed:
+
[source,terminal]
----
NAME                                           READY   STATUS
argocd-agent-agent-6489fc5dd-48whw             1/1     Running
----

.Verification

Verify that all required Agent pods are in the `Running` state and that the secrets are successfully created before proceeding with synchronization.
