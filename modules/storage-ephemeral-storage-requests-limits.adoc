= Ephemeral storage requests and limits example

[role="_abstract"]
To properly allocate resources for multi-container workloads, use the configuration example to define ephemeral storage requests and limits. The example manifest demonstrates how to specify storage capacity for a pod with two containers to ensure predictable performance.

The example configuration file shows a pod with two containers:

* Each container requests 2 GiB of local ephemeral storage.
* Each container has a limit of 4 GiB of local ephemeral storage.
* At the pod level, kubelet calculates an overall pod storage limit by summing the limits of all the containers in that pod.
** In this case, the total storage usage at the pod level is the sum of the disk usage from all containers plus the pod's `emptyDir` volumes.
** Therefore, the pod has a request of 4 GiB of local ephemeral storage, and a limit of 8 GiB of local ephemeral storage.

.Example ephemeral storage configuration with quotas and limits
[source, yaml]
----
apiVersion: v1
kind: Pod
metadata:
  name: frontend
spec:
  containers:
  - name: app
    image: images.my-company.example/app:v4
    resources:
      requests:
        ephemeral-storage: "2Gi"
      limits:
        ephemeral-storage: "4Gi"
    volumeMounts:
    - name: ephemeral
      mountPath: "/tmp"
  - name: log-aggregator
    image: images.my-company.example/log-aggregator:v6
    resources:
      requests:
        ephemeral-storage: "2Gi"
      limits:
        ephemeral-storage: "4Gi"
    volumeMounts:
    - name: ephemeral
      mountPath: "/tmp"
  volumes:
    - name: ephemeral
      emptyDir: {}
----

where:

`requests.ephemeral-storage`:: Specifies the container request for local ephemeral storage.

`limits.ephemeral-storage`:: Specifies the container limit for local ephemeral storage.
