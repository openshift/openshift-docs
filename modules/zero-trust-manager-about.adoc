== SPIRE server

A SPIRE server is composed of one or more SPIRE agents. The server manages and issues SPIFFE identities within a trust domain. It stores registration entries (selectors that determine under what conditions a SPIFFE ID should be issued) and signing keys. The server also performs node and workload attestation to verify the identity of the node or workload requesting a {svid-full} based on predefined conditions.

== SPIRE agent

The SPIRE agent acts as a secure intermediary between the SPIRE server and the workloads running on a node. It performs node attestation against the server as well as workload attestation againt applications running on the node. The SPIRE agent is also responsible for retrieving, delivering, and renewing SVIDs. A SPIRE agent runs on every node on which an identified workload runs.

== SPIFFE

{spiffe-full} provides a standardized way to establish trust between software workloads in distributed systems. SPIFFE assigns unique IDs called SPIFFE IDs. These IDs are Uniform Resource Identifiers (URI) that include a trust domain and a workload identifier.

The SPIFFE IDs are contained in the {svid-full}. SVIDs are used by workloads to verify their identity to other workloads so the workloads can communicate with each other. The two main SVID formats are:

* X.509-SVIDs: X.509 certificates where the SPIFFE ID is embedded in the Subject Alternative Name (SAN) field.
* JWT-SVIDs: JSON Web Tokens (JWTs) where the SPIFFE ID is included as the `sub` claim.

== Attestation

Attestation is the process by which the SPIRE server verifies the identity of both nodes and workloads before issuing them SPIFFE IDs and SVIDs. The SPIRE server gathers attributes of both the workload and node that the SPIRE Agent runs on and then compares them to a set of selectors defined when the workload was registered. If the comparison is successful, the entities are providedThis ensures that only legitimate and expected entities within the trust domain receive cryptographic identities. The two main types of attestation in SPIFFE/SPIRE are:

* Node attestation: verifies the identity of a machine or a node on a system, before a SPIRE agent running on that node can be trusted to request identities for workloads.
* Workload attestation: verifies the identity of an application or service running on an attested node before the SPIRE agent on that node can provide it with a SPIFFE ID and SVID.
