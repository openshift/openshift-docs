= Workload not getting the correct SPIFFE Verifiable Identity Document

[role="_abstract"]
When a workload fails to receive its SVID, the application cannot prove its identity to other services. Use these steps to determine the issue.

.Procedure

. Verify that the `ClusterSPIFFEID` matches the workload's labels or namespace by running the following command:
+
[source,terminal]
----
$ oc get clusterspiffeid
----

. Check the SPIRFE Agent logs to see if the SPIRE Agent is running on every node and delivering the SVID by running the following command:
+
[source,terminal]
----
$ oc logs -n zero-trust-workload-identity-manager -l app=spire-agent --tail=50
----

. Verify that the Container Storage Interface (CSI) driver is working by running the following command:
+
[source,terminal]
----
$ oc get pods -n zero-trust-workload-identity-manager | grep csi
----
