= Deploying quantum secure gateways

[role="_abstract"]

Configure and deploy a service mesh gateway by using the {k8s} Gateway API to enable post-quantum cryptographic protection for ingress traffic.

.Prerequisites

* You have logged in to the {ocp-product-title} web console 4.19+ as a user with the `cluster-admin` role.

* You have installed the {SMProductName} Operator 3.2.1+

* You have created an `{istio}` resource with PQC enabled, and the Operator has deployed {istio}. For more information, see "Installing service mesh with {pqc-short}".


* You have installed the following CLI tools locally:
** `oc`
** `podman`
** `curl`

.Procedure

. Deploy a gateway using the {k8s} Gateway API by running the following command:
+
[source,terminal]
----
$ oc apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1beta1
kind: Gateway
metadata:
 name: pqc-gateway
 namespace: istio-system
spec:
 gatewayClassName: istio
 listeners:
 - name: https
   port: 443
   protocol: HTTPS
   tls:
     mode: Terminate
     certificateRefs:
     - name: httpbin-credential
       namespace: istio-system
   allowedRoutes:
     namespaces:
       from: All
---
apiVersion: gateway.networking.k8s.io/v1beta1
kind: HTTPRoute
metadata:
 name: httpbin-route
 namespace: default
spec:
 parentRefs:
 - name: pqc-gateway
   namespace: istio-system
 hostnames:
 - "httpbin.example.com"
 rules:
 - matches:
   - path:
       type: PathPrefix
       value: /
   backendRefs:
   - name: httpbin
     port: 8000
EOF
----

. Add the `istio-injection=enabled` label to the `default` namespace by running the following command:
+
[source,terminal]
----
$ oc label ns default istio-injection=enabled
----

. Deploy the backend server by running the following command:
+
[source,terminal]
----
$ oc apply -n default -f \
https://raw.githubusercontent.com/openshift-service-mesh/istio/master/samples/httpbin/httpbin.yaml
----
