= Configuring dual-stack networking for a cluster on AWS

[role="_abstract"]
You can configure dual-stack networking for an {product-title} cluster on {aws-full} by specifying both IPv4 and IPv6 address families in the `install-config.yaml` file.

:FeatureName: Dual-stack networking for {product-title} on {aws-full}
include::snippets/technology-preview.adoc[]
:!FeatureName:

.Prerequisites

* You have configured an {aws-short} account with the required permissions.
* You have downloaded the {product-title} installation program and the pull secret for your cluster.
* You have generated the `install-config.yaml` file for your cluster.
* The instance types for control plane and compute machine pools are Nitro-based and support IPv6 addressing. The default `m6i.xlarge` instance type meets this requirement.
* If you are installing into an existing VPC, all provided subnets are configured with both IPv4 and IPv6 CIDR blocks.

.Procedure

. Edit the `install-config.yaml` file to set the `platform.aws.ipFamily` parameter to either `DualStackIPv4Primary` or `DualStackIPv6Primary`:
+
--
* `DualStackIPv4Primary`: IPv4 is the primary address family. Specify IPv4 addresses first in all network configuration lists.
* `DualStackIPv6Primary`: IPv6 is the primary address family. Specify IPv6 addresses first in all network configuration lists.
--

. Configure the `networking.clusterNetwork`, `networking.serviceNetwork`, and `networking.machineNetwork` parameters with both IPv4 and IPv6 CIDR blocks. The order of address families must match the `platform.aws.ipFamily` value.
+
.Example dual-stack networking configuration with `DualStackIPv4Primary`
[source,yaml]
----
networking:
  clusterNetwork:
  - cidr: 10.128.0.0/14
    hostPrefix: 23
  - cidr: fd01::/48
    hostPrefix: 64
  machineNetwork:
  - cidr: 10.0.0.0/16
  - cidr: fd2e:6f44:5dd8::/64
  serviceNetwork:
  - 172.30.0.0/16
  - fd02::/112
platform:
  aws:
    region: us-east-1
    ipFamily: DualStackIPv4Primary
    lbType: NLB
----
+
* `platform.aws.ipFamily` -- Set to `DualStackIPv4Primary` or `DualStackIPv6Primary` to enable dual-stack networking.
* `platform.aws.lbType` -- Defaults to `NLB` for dual-stack configurations. Classic Load Balancers do not support IPv6.
* `networking.clusterNetwork`, `networking.machineNetwork`, `networking.serviceNetwork` -- Each must include both IPv4 and IPv6 CIDR blocks. List the primary address family first to match the `ipFamily` value.

. If you are installing into an existing VPC, specify the dual-stack subnets in the `platform.aws.vpc.subnets` parameter:
+
[source,yaml]
----
platform:
  aws:
    region: us-east-1
    ipFamily: DualStackIPv4Primary
    lbType: NLB
    vpc:
      subnets:
      - id: subnet-1
      - id: subnet-2
      - id: subnet-3
----
+
All subnets must be configured with both IPv4 and IPv6 address ranges.

. Proceed with the remaining installation steps for your cluster.

.Verification

* After the cluster is deployed, verify that the cluster network is configured for dual-stack:
+
[source,terminal]
----
$ oc get network.config cluster -o jsonpath='{.status.clusterNetwork}' | jq .
----
+
The output shows both IPv4 and IPv6 CIDR blocks.
