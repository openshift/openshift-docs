= Monitor BuildRun status and results

[role="_abstract"]
Track build execution progress and capture build outputs through BuildRun status fields and step results. The build controller continuously updates status information to reflect the current state of image construction, providing visibility into build lifecycle and outcomes.

BuildRun monitoring includes:

* Build run statuses: Real-time status indicators showing whether a build is pending, running, succeeded, or failed, with specific reason codes and error messages
* Tekton relationship: Understanding how BuildRun resources create and monitor TaskRun resources that execute build strategy steps
* Step results: Outputs captured from build execution, including Git commit information, image digest, image size, and error details
* Status transitions: Tracking builds through lifecycle stages from validation to execution to completion or failure

These monitoring capabilities enable platform engineers and developers to observe build progress, diagnose failures, and verify build outputs. The build controller maintains accurate status even when underlying Tekton tasks change state or when builds are canceled.
