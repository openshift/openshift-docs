= Define build inputs and source code

[role="_abstract"]
Configure build inputs and source code to control how your container images are built in {builds-shortname}.
Build inputs specify the source code location, build strategy, container file, and configuration options that determine how the build process executes.
These settings define the foundation of your build workflow before any build runs.

Build inputs include the source code repository location and authentication, the build strategy that transforms source into an image, and the container file that provides build instructions.
You configure these inputs in the `Build` custom resource (CR) specification, which acts as a template for creating build runs.
The build controller uses these inputs to prepare the build environment and execute the build process.

Configuration options extend beyond basic source and strategy settings.
You can specify pods configuration to control resource allocation and scheduling, define service accounts for authentication, set environment variables for build-time configuration, and define volumes for persistent or ephemeral storage during builds.
Resource management settings in Tekton pipelines ensure builds run efficiently within cluster constraints.
