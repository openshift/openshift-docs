= Monitoring dynamic path file activity (Technology Preview)

:FeatureName: Dynamic path support for file activity monitoring
include::snippets/technology-preview.adoc[]

[role="_abstract"]
File activity monitoring supports user-defined path specifications with wildcard patterns, enhanced filtering options, and file rename operation detection.

// JIRA: ROX-32633

With this feature, you can take the following actions:

* Define custom paths to monitor using wildcard patterns
* Filter file activity based on process name, ancestor, arguments, and unique identifier (UID)
* Create policies for file rename operations
* Combine process criteria with file access criteria in deployment and node policies
* Use a unified file path criterion that replaces *Effective Path* and *Actual Path* fields

Node violations now display cluster name, cluster ID, node name, and node ID in the dashboard of the {product-title-short} portal.

The file operation terminology has been updated for clarity. The *Delete* operation is now called *Delete (unlink)*.

This release provides a new default policy category, *File Activity Monitoring*, to help organize related policies.

The following limitations apply:

* File activity deployment violations for files on the `overlayFS` of the container do not contain an *Actual Path* value. The *Effective Path* provides the primary value for monitoring.
* Support for ARM, {ibm-power}, and {ibm-z} architecture is not available.
