= Configure Build Inputs and Outputs

[role="_abstract"]
Control how your build accesses input sources and delivers output images by configuring parameters, volumes, environment variables, and resource constraints. These settings decide what data enters the build process and where the resulting container images are stored.

Build input and output configuration includes:

* Output definition: Registry location where the build pushes the completed container image, including authentication credentials and image metadata
* Parameter values definition for a build: Strategy-specific settings that control build behavior, such as build arguments for Dockerfiles or builder images for Source-to-Image builds
* Build reference definition: Reference to an existing build specification or inline build definition

These configurations ensure builds have the necessary inputs to produce container images and deliver them to the correct destinations. The build controller validates these settings before starting a build and enforces them throughout the build lifecycle.
