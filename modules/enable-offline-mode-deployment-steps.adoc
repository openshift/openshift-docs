= Deployment steps for offline mode

[role="_abstract"]
To deploy and operate {product-title} in offline mode, complete the deployment steps and update procedures.

To deploy and operate {product-title} in offline mode:

. Download {product-title-short} images and install them in your clusters. If you are using {ocp}, you can use {olm-first} and the Software Catalog to download images to a workstation that connects to the internet. The workstation then pushes images to a mirror registry that also connects to your secured cluster. For other platforms, you can use a program such as Skopeo or Docker to pull the images from the remote registry and push them to your own private registry, as described in "Downloading images for offline use".
. Enable offline mode during installation.
. Update the vulnerability list for Scanner by uploading a new definitions file at least once per day.

[IMPORTANT]
====
You can only enable offline mode during the installation, and not during an upgrade.
====
