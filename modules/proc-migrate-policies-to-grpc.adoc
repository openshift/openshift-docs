= Migrating policies to `GRPCRoute` support

:FeatureName: `GRPCRoute` policy attachment support

include::snippets/technology-preview.adoc[]

[role="_abstract"]
When you want to replace your `HTTPRoute` custom resources (CRs) with `GRPCRoute` CRs, you must update your policies to reference a `GRPCRoute` CR, and configure other parameters in those policies as required for your use case.

[IMPORTANT]
====
A `GRPCRoute` CR cannot coexist with an `HTTPRoute` CR on the same hostname. If both gRPC and HTTP traffic must share a hostname, use an `HTTPRoute` CR. Otherwise, use separate hostnames for HTTP and gRPC traffic.
====

.Prerequisites

* You installed {ocp}.
* You installed {oc-first}.
* You installed the {mcpg} Operator.
* You created a Gateway object.
* You have an existing `HTTPRoute` CR routing traffic to a gRPC service.

.Procedure

. Delete your `HTTPRoute` CRs. The `HTTPRoute` CR must be removed before the `GRPCRoute` CR can be used on the same hostname.

. Update the `spec.targetRef.kind` parameter value in your `AuthPolicy` and `RateLimitPolicy` CRs to `GRPCRoute` by using the following example:
+
.Example `AuthPolicy` CR updated to gRPC
[source,yaml]
----
#...
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: GRPCRoute
    name: grpcstore
#...
----
