= {ols-long} version 1.0.11

[role="_abstract"]
{ols-official} 1.0.11 provides maintenance updates, security fixes, and feature enhancements. This release ensures continued support and stability for clusters running {ocp-product-title} 4.16 and later.

[id="ols-1-0-11-enhancements_{context}"]
== Enhancements

[role="_abstract"]
{ols-official} 1.0.11 introduces security enhancements for secret handling, updated component management, and support for modern AI software development kits.

{ols-official} 1.0.11 includes the following enhancements:

//OLS-2746

* With this update, you can configure the maximum number of tool call iterations to ensure {ols-long} completes all required tool interactions before responding to a query. To set this limit, you must specify the `maxIterations` field in the `OLSConfig` custom resource (CR). This field defines the number of rounds {ols-long} executes when invoking the LLM with tools. Omit this field to use the default value of `5`.

//OLS-2692 - internal

//OLS-2575

* With this update, {ols-long} restricts access to {k8s} Secret resources at the Model Context Protocol (MCP) server level. This enhancement replaces the previous keyword-based filtering, which often blocked legitimate operations, such as deployments, that referenced secrets by name. As a result, {ols-long} ensures that sensitive data does not reach the LLM while improving the accuracy and reliability of tool calls. If you use a custom MCP server, you must configure it to secure sensitive resources.

//OLS-2396

* With this update, {ols-long} uses dedicated service accounts for the PostgreSQL server and the console user interface (UI) plugin. This enhancement improves security by ensuring these components run with minimal privileges and prevents them from accidentally inheriting permissions assigned to the default service account.

//OLS-2777 - internal

//OLS-2836

* With this update, {ols-long} introduces the `toolBudgetRatio` parameter in the `OLSConfig` custom resource (CR), allowing you to define the percentage of the `contextWindowSize` allocated for tool outputs. By specifying a value between 0.1 and 0.5, you can prevent the truncation of large data sets, such as logs and metrics. This enhancement ensures that {ols-long} has sufficient context to perform troubleshooting tasks effectively without exceeding message window limits.

//OLS-2816

* With this update, the `useOpenOLS` hook supports an optional flag to hide the initial prompt in the {ols-long} panel. This enhancement enables other console pages and plugins to trigger {ols-long} with an immediate response for contextual actions, such as `ask about this resource`. As a result, developers can integrate AI-driven insights more seamlessly across the user interface without displaying the initial prompt.

//OLS-2747

* With this update, {ols-long} provides backend support for OpenAI reasoning models, such as the o-series and GPT-5, by preserving reasoning tokens during multi-round tool calls. This enhancement allows the model to maintain its chain of thought within a single request, which prevents repetitive responses and potential timeouts. As a result, {ols-long} delivers more accurate and efficient results during complex troubleshooting and tool-calling scenarios.

//OLS-2700

* With this update, {ols-long} on {ocp-product-title} version 4.19 and later supports *Ask* and *Troubleshooting* modes. You can select a mode to receive responses tailored to your specific intent, such as general information gathering or technical problem resolution. As a result, the Model provides more relevant answers based on your selected context. The *Ask* mode is the default setting.

//OLS-2686

* With this update, {ols-long} supports custom certificate authority (CA) certificates for Model Context Protocol (MCP) servers. You can provide a CA bundle by specifying the `additionalCAConfigMapRef` field in the `OLSConfig` custom resource (CR). This enhancement ensures that {ols-long} correctly validates self-signed certificates for MCP servers, preventing connection failures caused by missing certificate validation.

//OLS-2826

With this update, the `useOpenOLS` hook includes an optional flag to hide the initial prompt in the {ols-long} panel. This enhancement enables other console pages and plugins to trigger {ols-long} with an immediate response for contextual actions, such as `ask about this resource`. As a result, you can provide a more integrated and direct experience when interacting with specific console resources.

//OLS-2722

With this update, {ols-long} introduces the `ols.tool-ui` extension, which allows {ocp-product-title} console plugins to map React components to specific tools. This enhancement enables the console to render custom user interfaces, such as Perses dashboards—directly within the {ols-long} panel when a tool is called. As a result, developers can provide a more integrated and visual experience for specialized monitoring and troubleshooting tools.

//OLS-2836
With this update, {ols-long} introduces the `toolBudgetRatio` parameter in the `OLSConfig` custom resource (CR). This parameter defines the percentage of the `contextWindowSize` used for tool outputs, with a supported range of 0.1 to 0.5. By adjusting this ratio, you can prevent the truncation of large logs and metrics. As a result, {ols-long} maintains sufficient context for accurate troubleshooting in data-intensive scenarios.

//OLS-2665
With this update, you can configure the timeout value for the built-in {k8s} Model Context Protocol (MCP) server when introspection is enabled. To customize this value, specify the timeout field in the `mcpKubeServerConfig` section of the `OLSConfig` custom resource (CR). This enhancement ensures that complex or long-running queries, such as Prometheus searches, do not fail due to the default 60-second limit. You must specify a value of at least 5 seconds.
