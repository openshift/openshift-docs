= Creating RBAC roles for virtual machine backup

[role="_abstract"]
You can create a custom role-based access control (RBAC) role that grants non-administrator users permission to back up and restore virtual machines. You assign the role to users or groups by creating a `RoleBinding`.

.Prerequisites
* You have access to the cluster as a user with the `cluster-admin` role.
* You have installed the OpenShift CLI (`oc`).

.Procedure

. Create a `Role` that grants permissions for VM backup operations:
+
.Example `vm-backup-operator` Role
[source,yaml]
----
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: vm-backup-operator
  namespace: production-vms <1>
rules:
- apiGroups: ["kubevirt.io"]
  resources: ["virtualmachines", "virtualmachineinstances"] <2>
  verbs: ["get", "list", "watch"]
- apiGroups: ["snapshot.storage.k8s.io"]
  resources: ["volumesnapshots", "volumesnapshotcontents"] <3>
  verbs: ["get", "list", "watch", "create", "delete"]
- apiGroups: [""]
  resources: ["persistentvolumeclaims"] <4>
  verbs: ["get", "list", "watch"]
- apiGroups: ["cdi.kubevirt.io"]
  resources: ["datavolumes"] <5>
  verbs: ["get", "list", "watch"]
- apiGroups: ["velero.io"]
  resources: ["backups", "restores"] <6>
  verbs: ["get", "list", "watch", "create", "delete"]
----
<1> The namespace containing VMs to back up.
<2> Grants permission to view VMs and VM instances.
<3> Grants permission to create and manage volume snapshots.
<4> Grants permission to view persistent volume claims used by VMs.
<5> Grants permission to view data volumes.
<6> Grants permission to create and manage OADP backups and restores. Remove these rules if not using OADP.

. Apply the `Role` by running the following command:
+
[source,terminal]
----
$ oc apply -f vm-backup-operator-role.yaml
----

. Create a `RoleBinding` to grant the `vm-backup-operator` role to the backup team:
+
.Example `RoleBinding` for backup operators
[source,yaml]
----
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: backup-team-vm-backup-operator
  namespace: production-vms
subjects:
- kind: Group
  name: backup-team
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: vm-backup-operator
  apiGroup: rbac.authorization.k8s.io
----

. Apply the `RoleBinding` by running the following command:
+
[source,terminal]
----
$ oc apply -f backup-team-vm-backup-operator-rolebinding.yaml
----

. Optional: If you are using OADP, grant additional cluster-level permissions for backup storage locations by creating the following `ClusterRole`:
+
.Example `ClusterRole` for OADP backup storage access
[source,yaml]
----
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: oadp-backup-storage-viewer
rules:
- apiGroups: ["oadp.openshift.io"]
  resources: ["backupstoragelocations", "volumesnapshotlocations"]
  verbs: ["get", "list", "watch"]
----

.Verification

* Verify that the role and role binding were created successfully by running the following commands:
+
[source,terminal]
----
$ oc get role vm-backup-operator -n production-vms
----
+
[source,terminal]
----
$ oc get rolebinding backup-team-vm-backup-operator -n production-vms
----
