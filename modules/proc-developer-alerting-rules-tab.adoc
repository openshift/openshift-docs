= Viewing alerting rules for your project

[role="_abstract"]
You can view alerting rules defined for your project in the Developer perspective. Alerting rules are created using PrometheusRule custom resources and define conditions that trigger alerts.

.Prerequisites

* You have access to the cluster as a developer or as a user with view permissions for the project.
* User-defined monitoring is enabled in the cluster.
* You have the `monitoring-rules-view` or `monitoring-rules-edit` role for the project.

[NOTE]
====
If no PrometheusRule resources exist in your project, the Alerting Rules tab displays a message indicating no rules are defined.
To create alerting rules, see xref:../observability/monitoring/managing-alerts.adoc#monitoring-about-creating-alerting-rules-for-user-defined-projects_managing-alerts[Creating alerting rules for user-defined projects].
====

.Procedure

. In the Developer perspective of the {product-title} web console, click *Observe* and go to the *Alerting Rules* tab.

. View the list of alerting rules defined for the selected project.
+
Each alerting rule displays the following information:
+
--
* *Name*: The alerting rule name
* *Severity*: The severity level (critical, warning, info, or none)
* *Alert state*: The current state (firing, pending, or inactive)
* *Notifications*: Whether alert notifications are enabled or silenced
--

. Optional: Filter the alerting rules by state or severity.
+
Click the *Filter* dropdown and select one or more of the following options:
+
--
* *Alert state*: Show only rules in a specific state (firing, pending, inactive)
* *Severity*: Show only rules with a specific severity level
--

. Click an alerting rule name to view its details.
+
The rule details page displays:
+
--
* *Expression*: The PromQL expression that defines the alert condition
* *Labels*: Key-value pairs attached to the alert
* *Annotations*: Additional metadata about the alert, such as description and summary
* *Active alerts*: The list of currently firing or pending alerts for this rule
--

. Optional: View the metrics for an alerting rule.
+
On the rule details page, click *View metrics* to see the underlying metric data for this alerting rule in the Metrics tab.
The Metrics tab displays the PromQL query from the rule's Expression field with current metric values.
This helps you understand why an alert is firing or pending.

.Verification

* The Alerting Rules tab displays the alerting rules defined in PrometheusRule custom resources for your project.
* You can view rule expressions, labels, and current alert states.
