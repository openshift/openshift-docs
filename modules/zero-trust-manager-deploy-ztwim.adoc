= Deploying the {zero-trust-full}

[role="_abstract"]
Deploy {zero-trust-full} and configure it to work with Istio. This Operator automates the deployment and management of SPIRE components for issuing workload identities.

.Prerequisites

* Cluster administrator access to the {openshift-title}.

* Access to the OpenShift Container Platform web console.

.Procedure

. Install the {zero-trust-full} from the web console. See link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.20/html/security_and_compliance/zero-trust-workload-identity-manager#zero-trust-manager-install-console_zero-trust-manager-install[Installing the Zero Trust Workload Identity Manager].
+
[NOTE]
====
Install only the Operator. Do not deploy any operands.
====

. Patch the {zero-trust-full} Subscription to configure the controller to work in `CREATE_ONLY` mode by running the following command:
+
[source,terminal]
----
$ oc -n zero-trust-workload-identity-manager patch subscription openshift-zero-trust-workload-identity-manager --type='merge' -p '{"spec":{"config":{"env":[{"name":"CREATE_ONLY_MODE","value":"true"}]}}}'
----
+
This prevents the Operator from overwriting manual changes to the SPIRE Agent configuration.
