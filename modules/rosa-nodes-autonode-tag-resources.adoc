= Tagging Amazon Web Services resources for {autonode} discovery

[role="_abstract"]
Tag your cluster's private subnets and security groups so that {autonode} can discover the network resources to use when provisioning nodes.

{autonode} uses Amazon Web Services (AWS) resource tags to discover the subnets and security groups that it should use when provisioning compute nodes for your cluster. You must apply the Karpenter discovery tag to your cluster's private subnets and security groups before enabling {autonode}.

.Prerequisites

* You have created a {product-title} cluster configured for {autonode}.
* You have the cluster ID stored in the `$CLUSTER_ID` environment variable.
* You have installed and configured the latest AWS command-line interface (CLI) on your installation host.
* Your AWS CLI session is configured to use the same AWS region where your cluster is deployed.

.Procedure

. Set your AWS region to match your cluster's region:
+
[source,terminal]
----
$ export AWS_REGION=us-east-1
----
+
For the {autonode} private preview, the region must be `us-east-1`.

. Find the security group for your cluster:
+
[source,terminal]
----
$ SECURITY_GROUP_ID=$(aws ec2 describe-security-groups \
  --filters "Name=tag:Name,Values=$CLUSTER_ID-default-sg" \
  --query 'SecurityGroups[0].GroupId' \
  --output text)
----
+
This command locates the default security group that was created for your {product-title} cluster.

. Verify that the security group was found:
+
[source,terminal]
----
$ echo $SECURITY_GROUP_ID
----
+
.Example output
[source,terminal]
----
sg-0a1b2c3d4e5f67890
----

. Find the private subnet for your cluster:
+
[source,terminal]
----
$ PRIVATE_SUBNET_ID=$(aws ec2 describe-subnets \
  --filters "Name=tag:kubernetes.io/role/internal-elb,Values=*" \
            "Name=tag:kubernetes.io/cluster/$CLUSTER_ID,Values=shared" \
  --query 'Subnets[0].SubnetId' \
  --output text)
----
+
This command locates a private subnet tagged for internal load balancers that is associated with your cluster.

. Verify that the private subnet was found:
+
[source,terminal]
----
$ echo $PRIVATE_SUBNET_ID
----
+
.Example output
[source,terminal]
----
subnet-0a1b2c3d4e5f67890
----

. Tag the private subnet with the Karpenter discovery tag:
+
[source,terminal]
----
$ aws ec2 create-tags \
  --resources "$PRIVATE_SUBNET_ID" \
  --tags Key="karpenter.sh/discovery",Value="$CLUSTER_ID"
----
+
This tag enables {autonode} to discover and use the subnet when provisioning nodes.

. Tag the security group with the Karpenter discovery tag:
+
[source,terminal]
----
$ aws ec2 create-tags \
  --resources "$SECURITY_GROUP_ID" \
  --tags Key="karpenter.sh/discovery",Value="$CLUSTER_ID"
----
+
This tag enables {autonode} to discover and use the security group when provisioning nodes.

.Verification

* Verify that the tags were applied to the private subnet:
+
[source,terminal]
----
$ aws ec2 describe-subnets \
  --subnet-ids $PRIVATE_SUBNET_ID \
  --query 'Subnets[0].Tags[?Key==`karpenter.sh/discovery`]'
----
+
.Example output
[source,json]
----
[
    {
        "Key": "karpenter.sh/discovery",
        "Value": "abc123-xyz456"
    }
]
----

* Verify that the tags were applied to the security group:
+
[source,terminal]
----
$ aws ec2 describe-security-groups \
  --group-ids $SECURITY_GROUP_ID \
  --query 'SecurityGroups[0].Tags[?Key==`karpenter.sh/discovery`]'
----
+
.Example output
[source,json]
----
[
    {
        "Key": "karpenter.sh/discovery",
        "Value": "abc123-xyz456"
    }
]
----

.Additional resources

* For more information about tagging additional subnets in different availability zones, see the link:https://karpenter.sh/docs/concepts/nodeclasses/[Karpenter NodeClass documentation].

.Next steps

* Enable {autonode} on your cluster by using the `rosa edit cluster` command.
