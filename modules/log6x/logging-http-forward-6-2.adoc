= Forwarding logs over HTTP

To enable forwarding logs over HTTP, specify `http` as the output type in the `ClusterLogForwarder` custom resource (CR).

.Procedure

* Create or edit the `ClusterLogForwarder` CR using the template below:
+
.Example ClusterLogForwarder CR
[source,yaml]
----
apiVersion: observability.openshift.io/v1
kind: ClusterLogForwarder
metadata:
  name: <log_forwarder_name> <1>
  namespace: <log_forwarder_namespace> <2>
spec:
  managementState: Managed
  outputs:
    - http:
        headers: <3>
          h1: v1
          h2: v2
        method: POST
        proxyURL: <proxy_url> <4>
        url: <url>   <5>
      name: httpout-app
      type: http
      tls:
        insecureSkipVerify: <6>
        ca:
          secretName: <secret_name>  <7>
          key: <secret_key>
  pipelines:
    - inputRefs:
        - application
      name: <pipeline_name> 
      outputRefs:
        - httpout-app <8>
  serviceAccount:
    name: <service_account_name> <9>
----
<1> In legacy implementations, the CR name must be `instance`. In multi log forwarder implementations, you can use any name.
<2> In legacy implementations, the CR namespace must be `openshift-logging`. In multi log forwarder implementations, you can use any namespace.
<3> Additional headers to send with the log record.
<4> Optional: URL of the HTTP proxy that should be used to forward logs over http or https from this output. This setting overrides any default proxy settings for the cluster or the node.
<5> Destination address for logs.
<6> Values are either `true` or `false`.
<7> Secret name for destination credentials.
<8> This value should be the same as the output name.
<9> The name of your service account. The service account is only required in multi log forwarder implementations if the log forwarder is not deployed in the `openshift-logging` namespace.
