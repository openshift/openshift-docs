= Verify recovered file integrity

[role="_abstract"]
Verify file integrity using checksums to confirm recovered data is uncorrupted and matches the backup source.
Checksum verification is essential for compliance requirements and validating critical file recovery.

.Prerequisites

* You downloaded files from a VM backup using web browser or SSH access.
* You have access to the original VM or a baseline checksum from before the backup was created.

.Procedure

. Generate a checksum for the original file in the running VM before creating backups:
+
[source,terminal]
----
$ oc exec <vm_pod> -- md5sum /path/to/file
----
+
Replace `<vm_pod>` with the pod name for the running VM.
+
Save the checksum output as a baseline for future verification.

. After recovering the file from a backup, generate a checksum for the recovered file:
+
[source,terminal]
----
$ md5sum /local/path/to/recovered/file
----

. Compare the checksums to verify data integrity:
+
* If the checksums match, the file was recovered successfully without corruption.
* If the checksums differ, the recovered file may be corrupted or from a different version.

. For more secure verification, use SHA256 instead of MD5:
+
[source,terminal]
----
$ sha256sum /local/path/to/recovered/file
----
+
SHA256 provides better protection against hash collisions and is recommended for security-sensitive files.

. To verify multiple files in a directory, create a checksum manifest:
+
[source,terminal]
----
$ find /local/path/to/recovered/directory -type f -exec sha256sum {} \; > checksums.txt
----

. Compare checksums across multiple backup versions to identify when a file changed:
+
[source,bash]
----
#!/bin/bash
SSH_KEY="/tmp/vmfr-key"
SSH_ADDRESS="<address>"
USERNAME="<username>"
VM_NAME="<vm_name>"
REMOTE_FILE="/etc/myapp/config.yaml"
BACKUPS=("daily-backup-20260630" "daily-backup-20260629" "daily-backup-20260628")

for backup in "${BACKUPS[@]}"; do
    echo "Checksum for $backup:"
    ssh -p 2222 -i "$SSH_KEY" "${USERNAME}@${SSH_ADDRESS}" \
        "md5sum /restores/*/${backup}/${VM_NAME}${REMOTE_FILE}"
done
----
+
Different checksums indicate the file changed between backup versions.

.Verification

. Document checksums for critical files in a secure location for future comparison.

. For compliance requirements, store checksum manifests alongside backup metadata to prove data integrity.
