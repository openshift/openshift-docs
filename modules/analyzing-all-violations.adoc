= Analyzing all violations

By viewing the *Violations* page, you can analyze all violations and take corrective action.

.Procedure

. In the {product-title-short} portal, click *Violations*.
. Click the tab to view violations for the following categories:
** *User Workloads*: Displays violations for user-managed workloads.
** *Platform*: Displays violations for workloads used by {ocp} and layered services.
** *All violations*: Displays violations for user workload and platform components, and audit log violations for cluster resources.
. To view different types of violations, click one of the following tabs:
** *Active*: Displays violations in the build and deploy stages or unresolved runtime violations.
** *Resolved*: Displays violations in the build and deploy stages for workloads that were removed or modified to be compliant, manually resolved runtime violations, and violations in all stages that were generated before a policy exclusion was added.
** *Attempted*: Displays violations for deployment actions that were attempted but blocked by a pre-check of {product-title-short} policies by the {product-title-short} admission controller. For example, the deployment action that was attempted would have triggered a violation if it had been allowed to succeed, but the policy enforcement behavior did not allow it to succeed.
. Optional: Choose the appropriate method to reorganize or filter information in the *Violations* page:
** To sort the violations in ascending or descending order, select a column heading.
** To filter violations, use the filter bar.
** To see more details about the violation, select a violation in the *Violations* page.
. Optional: Choose the appropriate method to exclude a deployment from the policy:
** If you selected a single deployment, click the overflow menu {kebab} and then select *Exclude deployment from policy*.
** If you selected multiple deployments, from the *Row actions* drop-down list, select *Exclude deployments from policy*.
