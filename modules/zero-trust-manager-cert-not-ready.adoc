= Certificate not ready (Configuration 2)

[role="_abstract"]
Troubleshoot cert-manager certificate issuance failures by inspecting ACME challenge statuses, analyzing controller logs, and verifying network requirements for DNS or HTTP validation.

The 'not ready' status of the certificate is characterized by these specific validation failures:

* Certificate shows `READY=False`

* ACME challenges fail

.Procedure

. Inspect the certificate status and events for error messages by running the following command:
+
[source,terminal]
----
$ oc get route spire-server-federation -n zero-trust-workload-identity-manager
----

. Review the `cert-manager`` controller logs for API communication or provider errors by running the following command:
+
[source,terminal]
----
$ oc logs -n cert-manager -l app=cert-manager --tail=100
----

. Identify and describe failing challenges by running the following commands:
+
[source,terminal]
----
$ oc get challenges -n zero-trust-workload-identity-manager
----
+
[source,terminal]
----
$ oc describe challenge <challenge-name> -n zero-trust-workload-identity-manager
----

Resolve certificate issuance::

To resolve a stalled certificate issuance, address these common validation and infrastructure constraints:

* *DNS not resolving*: Ensure domain is publicly accessible.

* *HTTP-01 validation fails*: Check ingress is reachable from internet.

* *Rate limits*: Use Let's Encrypt staging for testing.
