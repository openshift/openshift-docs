== New Features and Enhancements


=== Log Collection

* With this update, HTTP outputs have a `proxy` field, which allows log data to be sent via an HTTP proxy. (link:https://issues.redhat.com/browse/LOG-6069[LOG-6069]).

* This enhancement adds OpenTelemetry (OTel) semantic stream labels to the loki and lokistack outputs so that logs can be queried using both ViaQ and OTel stream labels. (link:https://issues.redhat.com/browse/LOG-6508[LOG-6508]).

=== Log Storage

* With this update, time-based stream sharding in Loki is now enabled by the Loki Operator. This reolves the issue of not being able to save historic data into Loki.  (link:https://issues.redhat.com/browse/LOG-6757[LOG-6757])

[id="logging-release-notes-6-2-0-technology-preview-features_{context}"]
== Technology Preview

:FeatureName: The OpenTelemetry Protocol (OTLP) output log forwarder
include::snippets/technology-preview.adoc[]

* A new output type, `OTLP`, allows sending JSON-encoded log records using the OpenTelemetry data model and resource semantic conventions. (link:https://issues.redhat.com/browse/LOG-4225[LOG-4225]). 

* With this update, Loki Operator allows the administrator to configure LokiStack to drop OpenTelemetry attributes when receiving data through OpenTelemetry protocol (OTLP). (link:https://issues.redhat.com/browse/LOG-6507[LOG-6507]).
