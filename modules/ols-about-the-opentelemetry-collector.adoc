= About the OpenTelemetry Collector

[role="_abstract"]
{ols-long} includes a custom OpenTelemetry (OTel) Collector that serves as the centralized telemetry hub for the platform, handling audit logging and distributed tracing.

The OTel Collector has two primary roles:

* *Audit log persistence* -- Receives OTLP audit logs from {ols-long} components and stores them when audit logging is enabled.
* *Distributed trace forwarding* -- When configured, forwards traces to an external backend, such as Jaeger, by using OTLP/gRPC.

The {ols-long} Operator always deploys the OTel Collector as a {k8s} Deployment. All endpoints are secured with TLS by using service-ca certificates.

You can configure the OTel Collector by using the following fields on the `OLSConfig` custom resource (CR):

* `spec.audit.logging` -- Enables audit log storage. The default value is `true`.
* `spec.audit.tracingEndpoint` -- Specifies the URL of an external trace backend. When set, the collector forwards distributed traces to this endpoint.
