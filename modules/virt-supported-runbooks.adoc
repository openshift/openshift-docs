= Alerts

// Use the virt-runbooks claude skill to automatically update the content in this topic
[role="_abstract"]
{VirtProductName} generates alerts to indicate issues with virtualization components. Each alert includes a description of the condition that triggered it.

[id="virt-runbooks-supported-table"]
[cols="1m,1"]
.{VirtProductName} alerts
|===
| Alert | Description

| `CDIDataImportCronOutdated`
| This alert fires when the `DataImportCron` custom resource (CR) cannot poll or import the latest disk image versions. The `DataImportCron` CR polls disk images, checking for the latest versions, and imports the images into persistent volume claims persistent volume claims (PVCs) or `VolumeSnapshot` objects.

| `CDIDataVolumeUnusualRestartCount`
| This alert fires when a `DataVolume` object restarts more than three times.

| `CDIDefaultStorageClassDegraded`
| This alert fires if the default storage class does not support CSI or snapshot-based smart cloning, or the `ReadWriteMany` access mode.

| `CDIMultipleDefaultVirtStorageClasses`
| This alert fires when more than one default virtualization storage class exists. A default virtualization storage class has precedence over a default {product-title} storage class for creating a `VirtualMachine` disk image.

| `CDINoDefaultStorageClass`
| This alert fires when a data volume is in the `Pending` state because there is no default storage class. A default virtualization storage class has precedence over a default {product-title} storage class for creating a `VirtualMachine` disk image.

| `CDINotReady`
| This alert fires when the Containerized Data Importer (CDI) is in a degraded state and is not progressing or available to use.

| `CDIOperatorDown`
| This alert fires when the CDI Operator is down. The CDI Operator deploys and manages the CDI infrastructure components, such as data volume and PVC controllers.

| `CDIStorageProfilesIncomplete`
| This alert fires when a CDI storage profile is incomplete. If a storage profile is incomplete, the CDI cannot infer PVC fields, such as `volumeMode` and `accessModes`, which are required to create a virtual machine (VM) disk.

| `CnaoDown`
| This alert fires when the CNAO is down. The CNAO deploys additional networking components on top of the cluster.

| `CnaoNMstateMigration`
| This alert fires when a `kubernetes-nmstate` deployment is detected and the NMState Operator is not installed.

| `DeprecatedMachineType`
| This alert triggers when one or more VMs are using machine types that are no longer supported.

| `GuestFilesystemAlmostOutOfSpace`
| This alert triggers when a file system in a VM is running out of available disk space.

| `GuestVCPUQueueHighCritical`
| A `VirtualMachineInstance` object virtual machine instance (VMI) reported a `guest CPU run‑queue length greater than 20` runnable or uninterruptible threads within the last scrape window of 120 seconds.

| `GuestVCPUQueueHighWarning`
| Within the most recent scrape window of 120 seconds, a VMI reported a `guest CPU run‑queue length of more than 10` runnable or uninterruptible threads.

| `HAControlPlaneDown`
| A control plane node has been detected as not ready for longer than 5 minutes.

| `HCOGoldenImageWithNoArchitectureAnnotation`
| When running on a heterogeneous cluster, the `DataImportCronTemplate` objects, or golden images, in the HyperConverged Cluster Operator (HCO) must be annotated with the `ssp.kubevirt.io/dict.architectures` annotation.

| `HCOGoldenImageWithNoSupportedArchitecture`
| When running on a heterogeneous cluster, the golden images in the HCO must be annotated with the `ssp.kubevirt.io/dict.architectures` annotation.

| `HCOInstallationIncomplete`
| This alert fires when the HCO runs for more than an hour without a `HyperConverged` CR.

| `HCOMisconfiguredDescheduler`
| A descheduler is a {product-title} application that causes the control plane to re-arrange the workloads in a more optimized way.

| `HCOMultiArchGoldenImagesDisabled`
| `DataImportCron` objects, also known as golden images, are used to create boot images for VMs.

| `HCOOperatorConditionsUnhealthy`
| This alert triggers when the HCO conditions or its secondary resources are in an error or warning state.

| `HighNodeCPUFrequency`
| This alert triggers when the CPU frequency on a node exceeds 80% of the maximum frequency for more than 5 minutes.

| `HPPNotReady`
| This alert fires when a hostpath provisioner (Hostpath Provisioner (HPP)) installation is in a degraded state. The HPP dynamically provisions hostpath volumes to provide storage for PVCs.

| `HPPOperatorDown`
| This alert fires when the HPP Operator is down. The HPP Operator deploys and manages the HPP infrastructure components, such as the daemon set that provisions hostpath volumes.

| `HPPSharingPoolPathWithOS`
| This alert fires when the HPP shares a file system with other critical components, such as `kubelet` or the operating system (OS).

| `HighCPUWorkload`
| This alert fires when a node's CPU utilization exceeds 90% for more than 5 minutes.

| `KubemacpoolDown`
| `KubeMacPool` is down. `KubeMacPool` is responsible for allocating MAC addresses and preventing MAC address conflicts.

| `KubeVirtCRModified`
| This alert fires when an operand of the HCO is changed by someone or something other than the HCO.

| `KubeVirtDeprecatedAPIRequested`
| This alert fires when a deprecated `KubeVirt` API is used.

| `KubeVirtVMGuestMemoryAvailableLow`
| The VM guest OS has very low available memory of less than 3% headroom for an extended period with no meaningful swap I/O, approaching OOM conditions.

| `KubeVirtVMGuestMemoryPressure`
| The VM guest OS is under sustained memory pressure, having low usable memory with elevated major page faults and/or swap I/O.

| `KubeVirtNoAvailableNodesToRunVMs`
| This alert is triggered when all nodes in the {product-title} cluster are missing hardware virtualization or CPU virtualization extensions.

| `KubeVirtVMIExcessiveMigrations`
| This alert fires when a VMI live migrates more than 12 times over a period of 24 hours.

| `LowKVMNodesCount`
| This alert fires when fewer than two nodes in the cluster have KVM resources.

| `LowReadyVirtControllersCount`
| This alert fires when one or more `virt-controller` pods are running, but not all of them have been in a `Ready` state for the last 10 minutes.

| `LowReadyVirtOperatorsCount`
| This alert fires when one or more `virt-operator` pods are running, but not all of them have been in a `Ready` state for the last 10 minutes.

| `LowVirtAPICount`
| This alert fires when only one available `virt-api` pod is detected during a 60-minute period, although at least two nodes are available for scheduling.

| `LowVirtControllersCount`
| This alert fires when a low number of `virt-controller` pods is detected. At least one `virt-controller` pod must be available in order to ensure high availability.

| `LowVirtOperatorCount`
| This alert fires when only one `virt-operator` pod in a `Ready` state has been running for the last 60 minutes.

| `NetworkAddonsConfigNotReady`
| This alert fires when the `NetworkAddonsConfig` CR of the CNAO is not ready.

| `NoLeadingVirtOperator`
| This alert fires when no `virt-operator` pod with a leader lease has been detected for 10 minutes, although the `virt-operator` pods are in a `Ready` state.

| `NoReadyVirtController`
| This alert fires when no available `virt-controller` devices have been detected for 5 minutes. The `virt-controller` devices monitor the custom resource definitions (CRDs) of VMIs and manage the associated pods.

| `NoReadyVirtOperator`
| This alert fires when no `virt-operator` pod in a `Ready` state has been detected for 10 minutes. The `virt-operator` is the first Operator to start in a cluster.

| `NodeNetworkInterfaceDown`
| This alert fires when one or more network interfaces on a node have been down for more than 5 minutes.

| `OrphanedVirtualMachineInstances`
| This alert fires when a VMI or `virt-launcher` pod runs on a node that does not have a running `virt-handler` pod.

| `OutdatedVirtualMachineInstanceWorkloads`
| This alert fires when running VMIs in outdated `virt-launcher` pods are detected 24 hours after the {VirtProductName} control plane has been updated.

| `PersistentVolumeFillingUp`
| A persistent volume associated with your cluster has crossed the warning threshold of 90% used capacity and is expected to fill up within 4 days.

| `SSPCommonTemplatesModificationReverted`
| This alert fires when the Scheduling, Scale, and Performance (SSP) Operator reverts changes to common templates as part of its reconciliation procedure.

| `SSPDown`
| This alert fires when all the SSP Operator pods are down. The SSP Operator is responsible for deploying and reconciling the common templates and the Template Validator.

| `SSPFailingToReconcile`
| This alert fires when the reconcile cycle of the SSP Operator fails repeatedly, although the SSP Operator is running.

| `SSPHighRateRejectedVms`
| This alert fires when a user or script attempts to create or modify a large number of VMs, using an invalid configuration.

| `SSPTemplateValidatorDown`
| This alert fires when all the Template Validator pods are down. The Template Validator checks VMs to ensure that they do not violate their templates.

| `UnsupportedHCOModification`
| This alert fires when a JSON Patch annotation is used to change an operand of the HCO.

| `VirtAPIDown`
| No running `virt-api` pod has been detected for 10 minutes.

| `VirtApiRESTErrorsBurst`
| For the last 10 minutes or longer, more than 80% of the REST calls made to `virt-api` pods have failed.

| `VirtControllerDown`
| No running `virt-controller` pod has been detected for 10 minutes.

| `VirtControllerRESTErrorsBurst`
| For the last 10 minutes or longer, more than 80% of the REST calls made to `virt-controller` pods have failed.

| `VirtHandlerDaemonSetRolloutFailing`
| The `virt-handler` daemon set has failed to deploy on one or more worker nodes after 15 minutes.

| `VirtHandlerRESTErrorsBurst`
| For the last 10 minutes or longer, more than 80% of the REST calls made to `virt-handler` pods have failed.

| `VirtLauncherPodsStuckFailed`
| A large number of `virt-launcher` pods remain in the `Failed` phase. The condition `cluster:kubevirt_virt_launcher_failed:count >= 200` appears for 10 minutes.

| `VirtOperatorDown`
| This alert fires when no `virt-operator` pod in the `Running` state has been detected for 10 minutes.

| `VirtOperatorRESTErrorsBurst`
| For the last 10 minutes or longer, more than 80% of the REST calls made to `virt-operator` pods have failed.

| `VirtualMachineInstanceHasEphemeralHotplugVolume`
| The `VirtualMachineInstanceHasEphemeralHotplugVolume` alert is triggered when a VMI contains an ephemeral hot plug volume.

| `VirtualMachineStuckInUnhealthyState`
| This alert triggers when a VM is in an unhealthy state for more than 10 minutes and does not have an associated `VirtualMachineInstance` object.

| `VirtualMachineStuckOnNode`
| This alert triggers when a VM with an associated VMI has been stuck in an unhealthy state for more than 5 minutes on a specific node.

| `VMCannotBeEvicted`
| This alert fires when the eviction strategy of a VM is set to `LiveMigration` but the VM is not migratable.

| `VMStorageClassWarning`
| When running VMs using ODF storage with `rbd` mounter or `rbd.csi.ceph.com` provisioner, Windows VMs may cause reports of bad crc/signature errors due to certain I/O patterns.
|===
