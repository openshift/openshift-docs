= Troubleshooting the Cluster Network Operator

The Cluster Network Operator is responsible for deploying the networking components. It runs very early in the installation process, after the control plane nodes have come up but before the installation program removes bootstrap control plane. Issues with this Operator can indicate installation program issues.

.Procedure

. Ensure the network configuration exists by executing the following command:
+
[source,terminal]
----
$ oc get network -o yaml cluster
----
+
If it doesn't exist, the installation program didn't create it. To determine why, execute the following command:
+
[source,terminal]
----
$ openshift-install create manifests
----
+
Review the manifests to determine why the installation program didn't create the network configuration.

. Ensure the network is running by executing the following command:
+
[source,terminal]
----
$ oc get po -n openshift-network-operator
----
