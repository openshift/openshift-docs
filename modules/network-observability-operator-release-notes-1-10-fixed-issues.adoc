= Network Observability Operator 1.10 fixed issues

The following issues are fixed in the Network Observability Operator 1.10 release.

[id="special-character-not-allowed-flowmetrics_{context}"]
== Special characters are not allowed in `FlowMetrics`
Before this update, special characters were not validated in the `FlowMetrics` data fields, leading to user experience issues due to invalid Prometheus names containing special characters. With this release, `FlowMetrics` names now disallow invalid characters, improving naming consistency and preventing naming conflicts. This enhancement aims to increase data accuracy for users.
