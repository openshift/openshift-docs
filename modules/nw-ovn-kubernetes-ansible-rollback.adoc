= Rolling back to the OpenShift SDN network plugin by using an Ansible playbook

Cluster administrators can roll back to the OpenShift SDN Container Network Interface (CNI) network plugin by using the Ansible playbook offline migration method. 

The Ansible playbook for rolling back to includes the OpenShift SDN plugin involves the following set of playbooks that each represent a stage in the migration process:

. Prechecks
. Rollback
. Reboot
. Post-rollback

After you download the `openshift` Ansible collection and save the collection to your preferred path, you can go to `roles.<playbook_name>.tasks` subdirectory, ensuring that you replace `<playbook_name>` with the name of a playbook, to view the list of tasks associated with that playbook.

.Prerequisites

. You used the Ansible playbook to migrate from the OpenShift SDN Container Network Interface (CNI) network plugin to the the OVN-Kubernetes plugin for your cluster.

.Procedure

. If you want rollback the plug-in migration, you must disable automatic migration. To do this, edit the `migration-playbook.yml` file and set the following values for parameters:
+
[source,yaml]
----
# ...
checks:
 disable_auto_migration: true
 egress_ip: false
 egress_firewall: false
 multicast: false
 mtu: 1400
 geneve_port: 6081
 ipv4_subnet: "192.168.1.0/24
# ...
----
+
The playbook generates a `patch` command.

. Run the `patch` command. The following example `patch` command runs each playbook with a specific configuration:
+
[source,terminal]
----
$ oc patch network.operator.openshift.io cluster --type='merge' --patch '{ "spec": { "migration": { "networkType": "OpenShiftSDN", "features": { "egressIP": <bool>, "egressFirewall": <bool>, "multicast": <bool> } } } }'
----
