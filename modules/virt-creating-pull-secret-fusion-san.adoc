= Creating a kubernetes pull secret

You must create a secret object in the `ibm-fusion-access` namespace you created when installing the {IBMFusionSAN} Operator. This secret holds the IBM entitlement key you need for pulling the required container images from the IBM container registry.

.Prerequisites

* You installed the `oc` CLI.
* You have access to the cluster as a user with the `cluster-admin` role.

.Procedure

. Log in to the *IBM Container software library* with your {IBMFusionSAN} *IBMid* and *password*.
. If you already have an entitlement key, click *Copy*.
. If you do not have an entitlement key yet, click *Get entitlement key* or *Add new key*, and then click *Copy*.
. Save the entitlement key in a safe place.
. Using the `oc` CLI, create the secret object:
+
[source,terminal]
----
$ oc create secret -n ibm-fusion-access generic fusion-pullsecret \
--from-literal=ibm-entitlement-key=<ibm-entitlement-key> <1>
----
<1> This is the entitlement key you copied from the *IBM Container software library*.
+
The `oc create` command creates the secret object.
