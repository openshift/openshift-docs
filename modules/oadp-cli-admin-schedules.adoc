= Managing schedules

You can use the {oadp-short} CLI to create, view, describe, and delete backup schedules. Schedules automate the creation of backups at specified intervals using a cron expression.

.Prerequisites

* The {oadp-short} CLI plugin is installed and configured in admin mode.
* You are logged in to the {product-title} cluster as a user with `cluster-admin` privileges.

.Creating a schedule

* Create a backup schedule:
+
[source,terminal]
----
$ oc oadp schedule create <schedule_name> [flags]
----
+
.Flags for `oc oadp schedule create`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--schedule`
|Cron expression for the schedule. For example, `0 1 * * *` for daily at 1 AM.

|`--include-namespaces`
|Namespaces to include in scheduled backups. Default: `*` (all namespaces).

|`--exclude-namespaces`
|Namespaces to exclude from scheduled backups.

|`--include-resources`
|Resources to include in scheduled backups. Accepts simple kind names (for example, `deployments,services`) or `resource.group` format (for example, `deployments.apps`) for disambiguation. Default: `*` (all resources).

|`--exclude-resources`
|Resources to exclude from scheduled backups. Same format as `--include-resources`.

|`--storage-location`
|Name of the backup storage location to use.

|`--volume-snapshot-locations`
|Volume snapshot location or locations to use.

|`--selector`, `-l`
|Label selector to filter resources.

|`--snapshot-volumes`
|Take PersistentVolume snapshots. Default: `true`.

|`--snapshot-move-data`
|Move snapshot data to the backup storage location.

|`--default-volumes-to-fs-backup`
|Use filesystem backup for all volumes.

|`--include-cluster-resources`
|Include cluster-scoped resources.

|`--ttl`
|Backup retention period. Default: `720h`.

|`--request-timeout`
|Timeout for the request to the Kubernetes API server.
|===
+
.Example
[source,terminal]
----
$ oc oadp schedule create daily-backup \
    --schedule "0 1 * * *" \
    --include-namespaces my-namespace \
    --ttl 720h
----

.Listing schedules

* List all schedules:
+
[source,terminal]
----
$ oc oadp schedule get [<schedule_name>] [flags]
----
+
.Flags for `oc oadp schedule get`
[cols="1,3",options="header"]
|===
|Flag |Description

|`-o`
|Output format. Supported values: `json`, `yaml`.
|===

.Describing a schedule

* Describe a schedule to view its details:
+
[source,terminal]
----
$ oc oadp schedule describe <schedule_name>
----

.Deleting a schedule

* Delete a schedule:
+
[source,terminal]
----
$ oc oadp schedule delete <schedule_name> [flags]
----
+
.Flags for `oc oadp schedule delete`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--confirm`
|Confirm deletion without prompting.
|===
