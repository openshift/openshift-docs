= About gateway injection

[role="_abstract"]

You can install a gateway by applying labels and annotations to {k8s} `Deployment` and `Service` resources, triggering the {istio} control plane to inject and configure the gateway proxy.

Gateway injection relies upon the same mechanism as sidecar injection to inject the Envoy proxy into gateway pods. To install a gateway by using gateway injection, you create a {k8s} `Deployment` object and an associated {k8s} `Service` object in a namespace that is visible to the {istio} control plane. When creating the `Deployment` object, you apply labels and annotations so the {istio} control plane injects and configures a proxy to act as a gateway. After installing the gateway, you configure it to control ingress and egress traffic by using the {istio} `Gateway` and `VirtualService` resources.
