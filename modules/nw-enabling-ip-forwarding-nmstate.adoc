= Enabling per-interface IP forwarding by using the nmstate API

[role="_abstract"]
You can enable IPv4 forwarding on a specific network interface by using the Kubernetes NMState Operator to apply a `NodeNetworkConfigurationPolicy` (NNCP) custom resource. This method provides a declarative approach to manage interface-level forwarding settings across your cluster nodes.

.Prerequisites

* The Kubernetes NMState Operator is installed. For more information, see xref:../../networking/networking_operators/k8s-nmstate-about-the-k8s-nmstate-operator.adoc#k8s-nmstate-about-the-k8s-nmstate-operator[Kubernetes NMState Operator].
* You have access to the cluster as a user with the `cluster-admin` role.
* You have identified the network interfaces that require IP forwarding enabled.
* You have verified that the nodes are running {op-system-first} with NetworkManager 1.30 or later.

.Procedure

. Create a `NodeNetworkConfigurationPolicy` manifest file that enables IP forwarding on the target interface.
+
The following example enables IPv4 forwarding on the `eth1` interface for all worker nodes:
+
[source,yaml]
----
apiVersion: nmstate.io/v1
kind: NodeNetworkConfigurationPolicy
metadata:
  name: enable-forwarding-eth1 <1>
spec:
  nodeSelector: <2>
    node-role.kubernetes.io/worker: ""
  desiredState:
    interfaces:
    - name: eth1 <3>
      type: ethernet
      state: up
      ipv4:
        enabled: true
        forwarding: true <4>
----
<1> The name of the policy. Use a descriptive name that indicates the purpose and target interface.
<2> Optional: A node selector to apply the policy to specific nodes. If omitted, the policy applies to all nodes in the cluster.
<3> The name of the network interface to configure. Replace `eth1` with the name of your target interface.
<4> Set `forwarding` to `true` to enable IPv4 forwarding on this interface.

. Apply the configuration by running the following command:
+
[source,terminal]
----
$ oc apply -f enable-forwarding-eth1.yaml
----

. Monitor the policy application status by running the following command:
+
[source,terminal]
----
$ oc get nncp enable-forwarding-eth1 -o yaml
----
+
Wait until the policy status shows `Available` and the conditions indicate successful application.

.Verification

. Verify that the policy was successfully applied by checking the `NodeNetworkConfigurationEnactment` resources:
+
[source,terminal]
----
$ oc get nnce
----
+
Each node should have an enactment resource showing the status of the policy application.

. Access a node to verify that IP forwarding is enabled on the interface by running the following commands:
+
[source,terminal]
----
$ oc debug node/<node_name>
----
+
[source,terminal]
----
sh-5.1# chroot /host
----
+
[source,terminal]
----
sh-5.1# cat /proc/sys/net/ipv4/conf/eth1/forwarding
----
+
.Example output
[source,terminal]
----
1
----
+
A value of `1` indicates that IPv4 forwarding is enabled on the `eth1` interface.
