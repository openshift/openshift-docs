= API catalog integration in {rhdh}

[role="_abstract"]
By integrating {prodname} with {rhdh}, you can create a self-service API catalog that makes API consumption easier and enables teams to work together more effectively.

You can create a centralized portal where development teams can discover available APIs and understand their capabilities. Developers can request access to the APIs they want and get credentials without leaving their development workflow. You can centrally manage APIs spread across many teams and namespaces.

To achieve these goals, you must add the {prodname} plugin on a {rhdh} deployment in an {ocp} cluster. You must then inject your {rhdh} configurations into your {ocp} `ConfigMap` object.
