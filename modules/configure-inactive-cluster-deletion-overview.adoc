= Overview of automatic cluster removal

[role="_abstract"]
{product-title-short} provides the option to automatically remove nonactive clusters so that you can monitor active clusters only. You can configure the decommissioned cluster age and add labels to preserve specific clusters from removal.

Automatic removal of nonactive clusters from {product-title-short} is disabled by default. To enable this setting, enter a nonzero number in the *Decommissioned cluster age* field, as described in the following procedure. The *Decommissioned cluster age* field indicates the number of days that a cluster can remain unreachable before it becomes nonactive. When a cluster is nonactive, the *Clusters* page displays the status of the cluster. Nonactive clusters show the `unhealthy` label and the window shows the number of days after which {product-title-short} will remove the cluster if it continues to remain nonactive. After {product-title-short} removes a cluster, an `info` log entry documents the action in the Central logs.

[NOTE]
====
There is a 24-hour grace period after you enable this setting before {product-title-short} removes any clusters. {product-title-short} never removes the cluster that hosts Central.
====
