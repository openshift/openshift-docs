= Setting up a trust relationship between Vault and Authorino

[role="_abstract"]
To ensure that the Authorino service can act on an `AuthPolicy` custom resource (CR), you must set up a secure, identity-based trust relationship between the Authorino service and your HashiCorp Vault (Vault) database. Your goal is to allow the Authorino service to dynamically retrieve sensitive credentials, such as API keys or OIDC client secrets, from Vault.

[NOTE]
====
In production environments, using a tool such as Terraform or GitOps to manage policies is recommended. Automation ensures that if a cluster is deleted, then your security policies are automatically recreated.
====

.Prerequisites

* You completed the installation for {mcpg}.
* You created a `Gateway` object.
* You have a Vault database.
* You created a namespace for your MCP server secrets.

.Procedure

. Enable JWT authentication in your Vault server by following the instructions in the link:https://developer.hashicorp.com/vault/api-docs/auth/jwt#configure[Vault documentation].

. Create a Vault policy that saves the Access Control List (ACL) that defines the `read` and `list` capabilities for Authorino by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -H "X-Vault-Token: $VAULT_TOKEN" -H 'Content-Type: application/json' -X POST \
  --data '{
    "policy": "path \_<"secret/data/mcp_gateway/*\">_ {\n  capabilities = [\"read\", \"list\"]\n}"
  }' \
  _<vault_service_hostname:port>_/v1/sys/policies/acl/authorino
----
+
* Replace `_<"secret/data/mcp_gateway/*\">_` with the path to the namespace for your MCP server secrets.
* Replace `_<vault_service_hostname:port>_` with the Vault service address.

. Create a Vault role that stores a set of rules for Authorino by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -H "X-Vault-Token: $VAULT_TOKEN" \
  -H 'Content-Type: application/json' \
  -X POST \
  --data '{
    "role_type": "jwt",
    "bound_audiences": ["authorino"],
    "user_claim": "sub",
    "policies": ["authorino"],
    "ttl": "1h"
  }' \
  _<vault_service_url:port>_/v1/auth/jwt/role/authorino-role
----
+
Replace `_<vault_service_url:port>_` with the Vault service address.
