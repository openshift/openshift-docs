= How {rhoai-full} works in {microshift-short}

Edge deployments are where data happens and decisions need to be made. You can use {rhoai} to integrate a fleet of {microshift-short}-driven edge devices into the artificial intelligence and machine learning (AI/ML) operations cycle.

{rhoai-full} ({rhoia}) is a platform for data scientists and developers of AI/ML applications. Develop, train, and test an AI model in the cloud or data center using {rhoai}, then run your model in your edge deployments on {microshift-short}. After you deploy your model, application data can be sent to the model. In turn, the model learns to make highly accurate predictions. Over time, your model can make data-driven decisions without a human user. This is an ideal scenario for edge applications where interaction with an administrator is naturally limited.

[id="microshift-limits-rhoai_{context}"]
== Limitations of {rhoai} with {microshift-short} deployments

As an edge-optimized Kubernetes deployment, {microshift-short} has the following limitations when using {rhoai}:

* As a single-node Kubernetes, {microshift-short} does not support multi-model deployments.

* Using {microshift-short} as a development platform for models is also not advised.

* Use the models you develop elsewhere, such as on {OCP}.

* In addition, additional RAM and storage capacity might be required to serve your model.

* To administer model serving on {microshift-short}, you must use the CLI.
