= SPIRE federation key features

[role="_abstract"]
Review the SPIRE federation key features to understand how to securely connect and manage multiple trust domains. You can use these capabilities to automate TLS certificate management, configure authentication profiles, and synchronize trust bundles.

Federation bundle endpoint:: Each SPIRE server can expose a federation bundle endpoint that allows other trust domains to fetch and verify trust bundles. The Operator automatically:

* Configures the SPIRE server to expose a federation endpoint on port 8443

* Creates and manages an OpenShift Route for external access

* Handles TLS certificate provisioning and renewal

* Supports both automatic and manual certificate management

Multiple authentication profiles:: The implementation supports two SPIRE-standard authentication profiles:

* **https_spiffe** profile (default)

** Uses SPIFFE-based TLS authentication

** SPIRE server presents its own SVID for authentication

** Uses self-signed certificates managed by SPIRE

** Requires `endpointSpiffeId` to identify the remote endpoint

* **https_web** profile

** Uses Web PKI (X.509 certificates from public Certificate Authorities)

** Supports automated certificate management via ACME (Let's Encrypt)

** Supports manual certificate management via ServingCert configuration

Automatic Certificate Management (ACME):: For the `https_web` profile, built-in ACME support enables:

* Automatic certificate provisioning from Let's Encrypt

* Automatic certificate renewal before expiration

Manual certificate management:: For organizations requiring custom certificate management:

* Integration with cert-manager or other certificate providers

* Support for externally managed TLS secrets

* Configurable certificate refresh intervals

Dynamic federation relationships:: The Operator integrates with the SPIRE Controller Manager to support:

* `ClusterFederatedTrustDomain` custom resources for declarative federation management

* Automatic trust bundle synchronization

* Dynamic addition/removal of federated trust domains

* Support for up to 50 federated trust domains per cluster

Managed routes:: The Operator provides flexible route management:

* Automatic Route Creation: Set `managedRoute: "true"` for operator-managed routes

* Manual Route Control: Set `managedRoute: "false"` to manage routes manually

* Route naming convention: `federation.<trust-domain>`

* Automatic TLS configuration based on authentication profile
