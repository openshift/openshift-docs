= Network object configuration for tracking network flows

The following YAML shows the configuration fields for network flows collectors in the Cluster Network Operator configuration.

[source,yaml]
----
apiVersion: operator.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  exportNetworkFlows: <.>
    netFlow:
      collectors:
        - <netflow-collector-1>:2056
        - <netflow-collector-2>:2056
        ...
    sFlow:
      collectors:
        - <sflow-collector-1>:6343
        - <sflow-collector-2>:6343
        ...
    ipfix:
      collectors:
        - <ipfix-collector-1>:4739
        - <ipfix-collector-2>:4739
        ...
----
<.> The `exportNetworkFlows` field identifies the type of network flow collector, `netFlow`, `sFlow`, or `ipfix`, and the IP address and port for each collector. You can specify up to 10 collectors of each type.

.Example configuration for tracking network flows

After applying the following manifest to the Cluster Network Operator, the Operator configures Open vSwitch (OVS) on each node in the cluster to send network flows records to the NetFlow collector that is listening at `192.168.1.99:2056`.

[source,yaml]
----
apiVersion: operator.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  exportNetworkFlows:
    netFlow:
      collectors:
        - 192.168.1.99:2056
----
