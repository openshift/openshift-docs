= Generic Device Plugin Configuration Reference

This section provides a detailed reference for the configuration parameters available for the Generic Device Plugin within the {microshift-short} `config.yaml` file.

.Parameters for `genericDevicePlugin`
[cols="1,1,2"]
|===
| Parameter | Type | Description

| `status`
| `Enabled`, `Disabled`
| Controls the deployment and activation of the Generic Device Plugin. The default is `Disabled`.

| `domain`
| `string`
| The domain prefix with which devices will be advertised and present in the cluster, for example, `device.microshift.io/serial`. This should be set to `device.microshift.io` for MicroShift.

| `devices`
| `Device`
| A list of device definitions to be exposed by the plugin. Each `Device` entry contains a `name` and a list of `groups`.
|===

.Parameters for `Device` (within `devices`)
[cols="1,1,2"]
|===
| Parameter | Type | Description

| `name`
| `string`
| A unique string representing the kind of device this specification describes (for example, `serial`, `video`, `fuse`). This name will be used in pod resource requests (for example, `device.microshift.io/serial`).

| `groups`
| `DeviceGroup`
| A list of device groups. Devices within a group are allocated and mounted together into a container.
|===

.Parameters for `DeviceGroup` (within `groups`)
[cols="1,1,2"]
|===
| Parameter | Type | Description

| `count`
| `int`
| Specifies how many times this group of devices can be mounted concurrently to different pods. If unspecified, `Count` defaults to `1`. Setting a high count (for example, `1000` for `/dev/fuse`) is possible as there are no inherent limits, but performance might be impacted depending on the host's capabilities and the nature of the device.

| `paths`
| `PathSpec`
| A list of host device file paths. Paths can be glob patterns (for example, `/dev/ttyUSB*`), in which case each matched device will be schedulable `Count` times. This field is **exclusive** with `usbs`; you cannot define both in the same device group.

| `usbs`
| `USBSpec`
| A list of USB specifications to match devices by their Vendor ID and Product ID. This field is **exclusive** with `paths`.
|===

.Parameters for `PathSpec` (within `paths`)
[cols="1,1,2"]
|===
| Parameter | Type | Description

| `path`
| `string`
| The file path of a device on the host (for example, `/dev/video0`, `/dev/ttyUSB*`).

| `mountPath`
| `string`
| The file path at which the host device should be mounted within the container. When unspecified, `MountPath` defaults to the `Path`.

| `permissions`
| `string`
| The file-system permissions given to the mounted device. Applies only to mounts of type `Device`. Can be one or more of:
* `r` - allows the container to read from the specified device.
* `w` - allows the container to write to the specified device.
* `m` - allows the container to create device files that do not yet exist.
When unspecified, `Permissions` defaults to `mrw`.

| `readOnly`
| `boolean`
| Specifies whether the path should be mounted read-only (`true` or `false`). Applies only to mounts of type `Mount`.

| `type`
| `string` (`Device`, `Mount`)
| Describes what type of file-system node this `Path` represents and thus how it should be mounted. When unspecified, `Type` defaults to `Device`.
|===

.Parameters for `USBSpec` (within `usbs`)
[cols="1,1,2"]
|===
| Parameter | Type | Description

| `vendor`
| `string` (hexadecimal representation of `uint16`)
| The USB Vendor ID of the device to match on (for example, `"1a86"`).

| `product`
| `string` (hexadecimal representation of `uint16`)
| The USB Product ID of the device to match on (for example, `"7523"`).

| `serial`
| `string`
| The serial number of the device to match on. If unspecified, any serial number will match.
|===

.Troubleshooting Configuration Issues

Be aware that the output of `microshift show-config` might include `type: ""` and `usb: null` for path-based configurations. These values do not impact runtime validity but can make the output appear less clean.
