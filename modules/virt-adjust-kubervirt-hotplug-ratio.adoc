= Disabling hotplugging for all VMs on a cluster

If you are a cluster administrator and want to adjust or disable hotplugging for an entire cluster, you can modify the `spec.configuration.liveUpdateConfiguration.maxHotplugRatio` settings in the `KubeVirt` custom resource (CR).

[WARNING]
====
Modifying the `KubeVirt` CR spec can impact multiple VMs and other workloads, and is not recommended for large deployments.
====

.Prerequisites

* You have installed the {oc-first}.

.Procedure

. Modify the `KubeVirt` CR and set the `maxHotplugRatio` value to `1`:
+
[source,yaml]
----
apiVersion: kubevirt.io/v1
kind: KubeVirt
metadata:
  name: kubevirt
  namespace: kubevirt
spec:
  configuration:
    liveUpdateConfiguration:
      maxHotplugRatio: 1
----

. Apply the changes to the `KubeVirt` CR by running the following command:
+
[source,terminal]
----
$ oc apply -f <filename>.yaml
----

.Verification

. Check that you have configured the `maxHotplugRatio` value correctly, by running the following command:
+
[source,terminal]
----
$ oc get hyperconverged -n openshift-cnv -o jsonpath='{.spec.liveUpdateConfiguration.maxHotplugRatio}'
----
+
If the configuration was successful, the output is the `maxHotplugRatio` value that you set in the previous procedure:
+
*Example output*
+
[source,terminal]
----
1
----
