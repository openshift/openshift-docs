= Preinstallation considerations

Before setting up {product-title} on {oci-c3-no-rt}, make sure you have all the necessary prerequisites ready. These include the following:

* Access to {oci-c3-short}: Make sure you have an active account with the appropriate 
permissions to create and manage resources on {oci-c3-no-rt}. 

* Access to the {oci-first-no-rt} region tenancy: Make sure you have an active account in the {oci} 
region’s tenancy that will be connected to {oci-c3-short} using a Federated Identity Provider. 
This {oci} tenancy will be used to manage and maintain all IAM Services for {oci-c3-short}.  

* Access to {ai-full} clusters portal: During the step-by-step deployment process, you will be using the {hybrid-console} portal: https://console.redhat.com/openshift/assisted-installer/clusters.  

* Access to credentials: You will need a bastion/jump host within your {oci-c3-short} target compartment’s tenancy to 
connect with an external Terraform website to download and install Terraform. You will also use this bastion 
host for the {oci} CLI, for running the Terraform scripts, and for preparing the {product-title} clusters.  

* Networking considerations: Ensure that your network setup allows for communication, which includes setting up a Virtual Cloud Network (VCN), configuring subnets, and managing security lists. If your {oci-c3-short} system is disconnected, you will need to access external networks via a DRG.  

* Infrastructure planning: Develop a strategy outlining the resources needed capacity projections and design 
aspects customized to suit your organizations requirements. 

* Bastion host configuration: Prepare a bastion host to host Terraform scripts that configure infrastructure and build {product-title} clusters. Follow the instructions below to install the {oci} CLI in your bastion host.
