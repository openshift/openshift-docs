= Configuring node pools for Nova availability zones

You can distribute node pools across multiple Nova availability zones to improve the high availability of your hosted cluster.

.Prerequisites

* You have created a hosted cluster.
* You have access to the management cluster.

.Procedure

. Create a node pool that uses a specific availability zone:
+
[source,terminal]
----
oc create -f - <<EOF
apiVersion: hypershift.openshift.io/v1beta1
kind: NodePool
metadata:
  name: <node_pool_name>
  namespace: clusters
spec:
  clusterName: <cluster_name>
  replicas: <number_of_nodes>
  platform:
    type: OpenStack
    openstack:
      flavor: <flavor_name>
      availabilityZone: <availability_zone>
  release:
    image: <image_name>
EOF
----
+
where:
+
--
* `<node_pool_name>` is the name of the node pool.
* `<cluster_name>` is the name of the hosted cluster.
* `<number_of_nodes>` is the number of nodes in the node pool.
* `<flavor_name>` is the name of the OpenStack flavor to use for the nodes.
* `<availability_zone>` is the name of the Nova availability zone.
* `<image_name>` is the name of the {op-system} image.
--

. Verify that the node pool is created:
+
[source,terminal]
----
oc get nodepools -n clusters
----
+
.Example output
[source,terminal]
----
NAME              CLUSTER         DESIRED   CURRENT   AVAILABLE   PROGRESSING   MESSAGE
<node_pool_name>  <cluster_name>  2         2         2           False         All replicas are available
----

. Verify that the nodes are created in the specified availability zone:
+
[source,terminal]
----
oc get nodes -o custom-columns=NAME:.metadata.name,ZONE:.metadata.labels.topology\\.kubernetes\\.io/zone
----
+
.Example output
[source,terminal]
----
NAME                                                ZONE
<node_pool_name>-<cluster_name>-<random_string>    <availability_zone>
<node_pool_name>-<cluster_name>-<random_string>    <availability_zone>
----
