= Securing {osp} clusters by using the Operator

[role="_abstract"]
You can secure {osp} clusters by installing the {product-title-short} Operator and using it to deploy secured cluster resources.

.Procedure

. Verify that the clusters you want to secure meet the "default requirements".
. In the {cloud-console}, create an "ACS Instance".
. On each {osp} cluster you want to secure, create a "project named `stackrox`". This project will contain the resources for {product-title-managed-short} secured clusters.
. Generate a cluster registration secret (CRS) or an init bundle, which contains secrets that are used to establish initial trust between Central and the secured clusters. Using a CRS is the preferred method. Complete only one of the following actions to generate the CRS:
* In the ACS Console, generate a "CRS".
* Log in to Central and use the `roxctl` CLI to generate a "CRS".
. On each {osp} cluster, apply the "CRS".
. On each {osp} cluster, install the "{product-title-short} Operator".
. On each {osp} cluster, install "secured cluster resources in the `stackrox` project" by using the Operator.
. Verify the "installation" by ensuring that your secured clusters can communicate with the ACS instance.
