= Deploying the database application

The following procedure deploys `mongodb-nationalparks`, which is a MongoDB database for holding the national park location information.

.Prerequisites

// TODO: Check now
* You are in the *Developer* perspective of the web console.
* You have deployed the `parksmap` front-end application.
* You have deployed the `nationalparks` back-end application.

.Procedure

// TODO: Check
. From the *+Add* view in the *Developer* perspective, click *Container images* to open a dialog.
. In the *Image Name* field, enter `registry.redhat.io/rhmap47/mongodb`.
. In the *Runtime icon* field, search for `mongodb`.
. Scroll down to the *General* section.
. Ensure that you have the current values for the following:
.. Application: `national-parks-app`
.. Name: `mongodb-nationalparks`
. Select *Deployment* as the *Resource*.
. Unselect the checkbox next to *Create route to the application*.
+
// TODO: Explain why
. In the *Advanced Options* section, click *Deployment* to add the following environment variables:
+
.Environment variable names and values
[cols="1,1"]
|===
|Name |Value

|`MONGODB_USER`|`mongodb`
|`MONGODB_PASSWORD`|`mongodb`
|`MONGODB_DATABASE`|`mongodb`
|`MONGODB_ADMIN_PASSWORD`|`mongodb`
|===

. Click *Create*.
