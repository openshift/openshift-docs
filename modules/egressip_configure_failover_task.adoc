= Configuring the EgressIP failover time limit

Follow this procedure to configure the `reachabilityTotalTimeoutSeconds` parameter and control how quickly the system detects a failing `egressIP` node and initiates a failover.

.Prerequisites

* Install the OpenShift CLI (`oc`).
* Log in to the cluster as a cluster administrator.

.Procedure

. Edit the `Network` custom resource by running the following command:
+
[source,bash]
----
$ oc edit network cluster
----

. Locate the `ovnKubernetesConfig` section under `spec:defaultNetwork:`

. Include the `egressIPConfig` object and set the desired value for `reachabilityTotalTimeoutSeconds`.
+
[NOTE]
====
The value must be an integer between 0 and 60. For details on value implications, see the "EgressIP failover settings" section.
====

The following example configures the system to wait a maximum of 5 seconds to detect a node failure before initiating a failover:

[source,yaml]
----
apiVersion: operator.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  clusterNetwork:
  - cidr: 10.128.0.0/14
    hostPrefix: 23
  defaultNetwork:
    ovnKubernetesConfig:
      egressIPConfig: 
        reachabilityTotalTimeoutSeconds: 5  
      gatewayConfig:
        routingViaHost: false
      genevePort: 6081
----
