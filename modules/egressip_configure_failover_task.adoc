= Configuring the EgressIP Failover Time Limit

As a cluster administrator, you can configure the `reachabilityTotalTimeoutSeconds` parameter to control how quickly the system detects a failing EgressIP node and initiates a failover. This parameter directly determines the **maximum time** the platform will wait before declaring a node unreachable and triggering service continuity measures.

.Prerequisites
* You understand the failover time trade-offs (see the related concept).
* You have access to edit the `Network` custom resource.

.Procedure
. Edit the `Network` custom resource:
+
[source,bash]
----
oc edit network cluster
----
. Locate the `ovnKubernetesConfig` section under `spec:defaultNetwork:`
. Include the `egressIPConfig` object and set the desired value for `reachabilityTotalTimeoutSeconds`.

[NOTE]
====
The value must be an integer between 0 and 60. For details on value implications, see "EgressIP failover reference".
====

.Example Configuration
The following example configures the system to wait a maximum of **5 seconds** to detect a node failure before initiating a failover:

[source,yaml]
----
apiVersion: operator.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  clusterNetwork:
  - cidr: 10.128.0.0/14
    hostPrefix: 23
  defaultNetwork:
    ovnKubernetesConfig:
      egressIPConfig: 
        reachabilityTotalTimeoutSeconds: 5  # Limits failover detection time to 5 seconds
      gatewayConfig:
        routingViaHost: false
      genevePort: 6081
----
