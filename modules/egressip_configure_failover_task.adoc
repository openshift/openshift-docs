= Configure the EgressIP failover time limit

[role="_abstract"]
Follow this procedure to configure the `reachabilityTotalTimeoutSeconds` parameter and control how quickly the system detects a failing `egressIP` node and initiates a failover.

.Prerequisites

* Install the OpenShift CLI (`oc`).
* Log in to the cluster as a cluster administrator.

.Procedure

. Edit the `Network` custom resource by running the following command:
+
[source,bash]
----
$ oc edit network.operator cluster
----

. Navigate to the `egressIPConfig: {}` section under `spec:defaultNetwork:ovnKubernetesConfig:`
+
[NOTE]
====
If the `egressIPConfig` field shows as an empty object (`egressIPConfig: {}`), this is a valid configuration that uses default values for all EgressIP settings.
The default value for `reachabilityTotalTimeoutSeconds` is 1 second when not explicitly specified.

You might also see this field serialized with an explicit default value:

[source,yaml]
----
egressIPConfig:
  reachabilityTotalTimeoutSeconds: 1
----

Both forms are semantically identical and represent the same configuration.
The empty object form is commonly returned by `oc get network.operator cluster -o yaml`.
====

. Modify the block to include the `reachabilityTotalTimeoutSeconds` parameter with your chosen value, 5 seconds for example. Make sure to use the correct indentation:
+
[source,yaml]
----
  defaultNetwork:
    ovnKubernetesConfig:
      egressIPConfig:
        reachabilityTotalTimeoutSeconds: 5
----
+
[NOTE]
====
The value must be an integer between 0 and 60.
For details on possible values, see xref:egressip_failover_reference_{context}[EgressIP failover settings].
====

. Save and exit the editor. The operator automatically applies the changes.

.Verification

. Verify that the system correctly accepted the `reachabilityTotalTimeoutSeconds` parameter by running the following command:
+
[source,terminal]
----
$ oc get network.operator cluster -o yaml
----

. Inspect the output and confirm that the `reachabilityTotalTimeoutSeconds` parameter is correctly nested under `spec:defaultNetwork:ovnKubernetesConfig:egressIPConfig:` with your intended value:
+
[source,yaml]
----
 # ...
  spec:
    # ...
    defaultNetwork:
      ovnKubernetesConfig:
        egressIPConfig:
          reachabilityTotalTimeoutSeconds: 5
        gatewayConfig:
  # ...
----
