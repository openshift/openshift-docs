= Capture and reuse build information

[role="_abstract"]
Capture and reuse build information through build snapshots that preserve the exact configuration used for each build execution.
Build snapshots enable verification of build settings, support reproducibility requirements, and provide an audit trail for troubleshooting and compliance purposes.
The build controller automatically creates snapshots when reconciling build runs, storing complete build specifications in the `BuildRun` status.

Build snapshots record the state of a `Build` custom resource (CR) at the moment a build run starts.
When the build controller processes a `BuildRun`, it copies the entire build specification to the `status.buildSpec` field in the `BuildRun` CR.
This snapshot captures all build settings including source configuration, strategy selection, parameter values, and output destinations exactly as they existed when the build executed.

Snapshot preservation ensures build reproducibility even when the original `Build` CR changes.
After a build run completes, you can examine the snapshot to verify which source revision, build strategy version, or parameter values were used for that specific image build.
This capability is critical for troubleshooting build failures, auditing production images, and ensuring compliance with build policies.
The snapshot remains available if the `BuildRun` CR exists, independent of changes to the referenced `Build` CR.

Build snapshots support verification workflows by providing a complete record of build inputs.
You can compare snapshots across multiple build runs to identify configuration drift, validate that builds use approved settings, or reproduce builds from historical configurations.
Automated systems can extract snapshot data to generate software bills of materials, track dependency versions, or enforce governance policies on build configurations.
