= Network observability automatic installation process

[role="_abstract"]
The cluster installation process automatically deploys network observability capabilities when configured in the `install-config.yaml` file.

[id="installation-flow_{context}"]
== Installation flow

During cluster installation, the following sequence occurs:

. You configure the `networking.networkObservability.installationPolicy` field in the `install-config.yaml` file.

. The `openshift-install` binary reads the `install-config.yaml` file and passes the network observability configuration to the cluster.

. The Cluster Network Operator (CNO) receives the configuration and sets the `installationPolicy` field in the Network custom resource.

. If `installationPolicy` is set to `InstallAndEnable` or is not specified when the feature gate is enabled, the CNO:
   - Installs the Network Observability Operator
   - Creates a default `FlowCollector` configuration
   - Starts collecting network flow data

. If `installationPolicy` is set to `DoNotInstall`, the CNO skips Network Observability installation.

[id="configuration-inheritance_{context}"]
== Configuration inheritance

The `install-config.yaml` file contains the initial network observability configuration:

[source,yaml]
----
apiVersion: v1
baseDomain: example.com
networking:
  networkObservability:
    installationPolicy: InstallAndEnable
----

This configuration is inherited by the Network custom resource during cluster creation:

[source,yaml]
----
apiVersion: operator.openshift.io/v1
kind: Network
metadata:
  name: cluster
spec:
  networkObservability:
    installationPolicy: InstallAndEnable
----

The CNO uses this Network custom resource to determine whether to install the Network Observability Operator.

[id="default-behavior_{context}"]
== Default behavior

When the `NetworkObservabilityInstall` feature gate is enabled:

* If the `installationPolicy` field is not specified in the `install-config.yaml` file, it defaults to `InstallAndEnable` for standard multi-node clusters and Hosted Control Plane clusters.
* {sno} clusters default to `DoNotInstall` due to resource constraints.
* The `installationPolicy` value cannot be changed after cluster installation. To enable or disable network observability postinstallation, you must use the standard operator installation or removal procedures.
