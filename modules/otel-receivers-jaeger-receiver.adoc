= Jaeger Receiver

[role="_abstract"]
The Jaeger Receiver ingests traces in the Jaeger formats.

The following `OpenTelemetryCollector` custom resource configures the Jaeger Receiver:

[source,yaml]
----
# ...
  config:
    receivers:
      jaeger:
        protocols:
          grpc:
            endpoint: 0.0.0.0:14250
            tls:
              cert_file: cert.pem
              key_file: key.pem
              client_ca_file: client.pem
              client_ca_file_reload: false
              reload_interval: 1h
              min_version: "1.2"
              max_version: "1.3"
          thrift_http:
            endpoint: 0.0.0.0:14268
          thrift_compact:
            endpoint: 0.0.0.0:6831
          thrift_binary:
            endpoint: 0.0.0.0:6832

    service:
      pipelines:
        traces:
          receivers: [jaeger]
# ...
----
where:

`grpc.endpoint`:: Jaeger gRPC endpoint. If omitted, the default `+0.0.0.0:14250+` is used.
`tls`:: Server-side TLS configuration. Defines paths to TLS certificates. TLS is configured per protocol and is only supported for `grpc` and `thrift_http` protocols. UDP-based protocols, such as `thrift_compact` and `thrift_binary`, do not support TLS.
`client_ca_file`:: Path to the TLS certificate that the server uses to verify a client certificate. This sets the value of `ClientCAs` and `ClientAuth` to `RequireAndVerifyClientCert` in the `TLSConfig`. For more information, see the link:https://godoc.org/crypto/tls#Config[`Config` of the Golang TLS package].
`client_ca_file_reload`:: When set to `true`, reloads the `client_ca_file` certificate when it is modified. The default is `false`.
`reload_interval`:: Specifies the time interval at which the certificate is reloaded. If the value is not set, the certificate is never reloaded. The `reload_interval` field accepts a string containing valid units of time such as `ns`, `us`, `ms`, `s`, `m`, `h`. Set this value to an interval shorter than your certificate lifetime if you use automated certificate rotation.
`min_version`:: Sets the minimum supported TLS protocol version. This parameter is optional. The default is `1.2`.
`max_version`:: Sets the maximum supported TLS protocol version. This parameter is optional. The default is `1.3`.
`thrift_http.endpoint`:: Jaeger Thrift HTTP endpoint. If omitted, the default `+0.0.0.0:14268+` is used.
`thrift_compact.endpoint`:: Jaeger Thrift Compact endpoint. If omitted, the default `+0.0.0.0:6831+` is used.
`thrift_binary.endpoint`:: Jaeger Thrift Binary endpoint. If omitted, the default `+0.0.0.0:6832+` is used.
