= Jaeger Receiver

[role="_abstract"]
The Jaeger Receiver ingests traces in the Jaeger formats.

The following `OpenTelemetryCollector` custom resource configures the Jaeger Receiver:

[source,yaml]
----
# ...
  config:
    receivers:
      jaeger:
        protocols:
          grpc:
            endpoint: 0.0.0.0:14250
            tls:
              cert_file: cert.pem
              key_file: key.pem
          thrift_http:
            endpoint: 0.0.0.0:14268
          thrift_compact:
            endpoint: 0.0.0.0:6831
          thrift_binary:
            endpoint: 0.0.0.0:6832

    service:
      pipelines:
        traces:
          receivers: [jaeger]
# ...
----
where:

`grpc.endpoint`:: Jaeger gRPC endpoint. If omitted, the default `+0.0.0.0:14250+` is used.
`tls`:: Server-side TLS configuration. Defines paths to TLS certificates. TLS is configured per protocol and is only supported for `grpc` and `thrift_http` protocols. UDP-based protocols, such as `thrift_compact` and `thrift_binary`, do not support TLS.
`thrift_http.endpoint`:: Jaeger Thrift HTTP endpoint. If omitted, the default `+0.0.0.0:14268+` is used.
`thrift_compact.endpoint`:: Jaeger Thrift Compact endpoint. If omitted, the default `+0.0.0.0:6831+` is used.
`thrift_binary.endpoint`:: Jaeger Thrift Binary endpoint. If omitted, the default `+0.0.0.0:6832+` is used.
