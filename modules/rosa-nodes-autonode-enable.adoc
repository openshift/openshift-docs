= Enabling {autonode} on your cluster

[role="_abstract"]
Enable the {autonode} feature on your {product-title} cluster and verify that the Karpenter controllers are installed correctly.

After you create your cluster and tag the required Amazon Web Services (AWS) resources, you can enable {autonode} by using the {rosa-cli}. This activates the Karpenter controllers and creates the default node classes that {autonode} uses for provisioning compute nodes.

.Prerequisites

* You have created a {product-title} cluster configured for {autonode}.
* You have captured the {autonode} role Amazon Resource Name (ARN) in the `$ROLE_ARN` environment variable.
* You have captured your cluster ID in the `$CLUSTER_ID` environment variable.
* You have tagged your cluster's private subnets and security groups with the Karpenter discovery tag.
* Your cluster is in the `ready` state.

.Procedure

. Enable {autonode} on your cluster:
+
[source,terminal]
----
$ rosa edit cluster -c $CLUSTER_ID --autonode=enabled --autonode-iam-role-arn=$ROLE_ARN
----
+
This command enables the {autonode} feature and configures the Karpenter controller to use the IAM role you created. The Karpenter controllers are deployed automatically to your cluster.

. Optional: Create a cluster administrator user for testing:
+
[source,terminal]
----
$ rosa create admin -c $CLUSTER_ID
----
+
This command creates a temporary cluster administrator account. The output includes the login credentials.
+
[NOTE]
====
This step creates a cluster administrator account for testing purposes only.
====

. Log in to your cluster by using the credentials from the previous step:
+
[source,terminal]
----
$ oc login <api_url> --username <username> --password <password>
----
+
Replace the placeholders with the values from the `rosa create admin` command output.

. Verify that the Karpenter custom resource definitions (CRDs) are present and ready:
+
[source,terminal]
----
$ oc get ec2nodeclass
----
+
.Example output
[source,terminal]
----
NAME      READY   AGE
default   True    19m
----

. Verify that the OpenShift EC2 node class is present and ready:
+
[source,terminal]
----
$ oc get openshiftec2nodeclass
----
+
.Example output
[source,terminal]
----
NAME      READY   AGE
default   True    19m
----

.Verification

* Both the `ec2nodeclass` and `openshiftec2nodeclass` resources named `default` display `READY` as `True`.
* If either resource shows `READY` as `False`, the Karpenter controllers encountered an issue during initialization. Check the controller logs for errors:
+
[source,terminal]
----
$ oc logs -n kube-system -l app.kubernetes.io/name=karpenter
----

.Next steps

* Create a NodePool to define the compute capacity limits and instance requirements for {autonode}.
