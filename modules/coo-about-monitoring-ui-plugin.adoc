= About the {coo-full} monitoring UI plugin

[role="_abstract"]
The monitoring UI plugin extends the {ocp-product-title} web console with advanced monitoring capabilities, including integration with {rh-rhacm} for centralized alert management and incident detection for streamlined alert correlation and troubleshooting.

* **{rh-rhacm}:** The monitoring plugin in {coo-first} allows it to function in {rh-rhacm-first} environments, providing {rh-rhacm} with the same alerting capabilities as {ocp-product-title}. You can configure the plugin to fetch alerts from the {rh-rhacm} Alertmanager and Thanos backends. This enables seamless integration and user experience by aligning {rh-rhacm} and {ocp-product-title} monitoring workflows.

* **Incident detection:** The incident detection feature groups related alerts into incidents, to help you identify the root causes of alert bursts, instead of being overwhelmed by individual alerts. It presents a timeline of incidents, color-coded by severity, and you can drill down into the individual alerts within an incident. The system also categorizes alerts by affected component, grouped by severity. This helps you focus on the most critical areas first.
+
The incident detection feature is available in the {ocp-product-title} web console by clicking on **Observe** -> **Alerting** and choosing the **Incidents** tab.
