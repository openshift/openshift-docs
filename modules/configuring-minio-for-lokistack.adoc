= Configuring MinIO for LokiStack

[role="_abstract"]
You can use MinIO to store logs, which is common for on-site or development environments.

.Prerequisites
* You have MinIO deployed and accessible.
* You have created a bucket on MinIO.
* You have cluster administrator permissions.

.Procedure

. Create an object storage secret with the name `logging-loki-s3`:
+
[source,terminal]
----
$ oc create secret generic logging-loki-s3 \
  -n openshift-logging \
  --from-literal=bucketnames="<bucket_name>" \
  --from-literal=endpoint="<minio_endpoint>" \
  --from-literal=access_key_id="<minio_access_key_id>" \
  --from-literal=access_key_secret="<minio_access_key_secret>" \
  --from-literal=forcepathstyle="true"
----

. Optional: If your MinIO installation uses self-signed certificates, follow xref:configuring-tls-ca-bundle-for-object-storage_{context}[Configuring a TLS CA bundle for object storage].

.Verification
Verify the secret was created successfully by running the following command:
[source,terminal]
----
$ oc get secret logging-loki-s3 -n openshift-logging
----
