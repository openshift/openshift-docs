= Improving threat detection

When you create the log analytics table, you need to create a schema to improve threat detection.

.Prerequisites

* You have set up a data collection rule, log analytics workspace, service principal and sentinel on Microsoft Azure.
* You have configured the notifier supports secret or client certificate authentication.

.Procedure

* To automatically create a schema, upload the following example content to sentinel:
+
[source,json]
----
{
 "TimeGenerated": "2024-09-03T10:56:58.5010069Z",
 "msg": {
   "id": "1abe30d1-fa3a-4f85-a0eb-781f0a12228a",
   "policy": {
     "id": "cecf3916-d9ce-40da-abb4-c7906472bb9d",
     "name": "Latest tag (COPY)",
     "description": "Alert on deployments with images using tag 'latest'",
     "rationale": "Example.",
     "remediation": "Example.",
   }
 }
}
----
