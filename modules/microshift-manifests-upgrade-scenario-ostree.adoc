== Manifest upgrade scenario for OSTree systems
Use the following procedure in the upgrade scenario for OSTree systems where some resources need to be removed while retaining others to preserve data.

[IMPORTANT]
====
For OSTree systems, the `delete` subdirectories are read only.
====

.Procedure

. Identify the manifest that needs updating.
. Create new manifest for resource deletion that needs to be placed in the `delete` subdirectories.
. Create new manifest that needs to be applied in the manifest path. link:https://access.redhat.com/documentation/en-us/red_hat_build_of_microshift/{ocp-version}/html/running_applications/applications-with-microshift#microshift-applying-manifests-example_applications-microshift[Using manifests example] to create new manifests using the example.
. Use the procedure in "Manifest removal scenario for OSTree systems" to create `delete` subdirectories and place the manifests created for resource deletion in this path.
