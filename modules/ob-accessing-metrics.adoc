= Access build controller metrics

[role="_abstract"]
You can access the build controller metrics by creating a route to expose the metrics endpoint.

.Prerequisites

* You have installed the `oc` CLI.
* You have cluster administrator privileges.

.Procedure

. Create a Service resource to expose the metrics port:
+
[source,yaml]
----
apiVersion: v1
kind: Service
metadata:
  name: <service_name>
  namespace: openshift-builds
  labels:
    app: shipwright-build-controller
spec:
  selector:
    app: shipwright-build-controller
  ports:
  - name: metrics
    port: 8383
    protocol: TCP
    targetPort: 8383
----
+
where:
+
`<service_name>`:: Specifies a name for the Service resource, such as `shipwright-build-controller-metrics`.

. Create a Route to expose the metrics service:
+
[source,terminal]
----
$ oc create route edge <route_name> \
  --service=<service_name> \
  --port=8383 \
  -n openshift-builds
----
+
where:
+
`<route_name>`:: Specifies a name for the Route resource, such as `shipwright-metrics`.
`<service_name>`:: Specifies the name of the Service resource from the previous step.
+
This creates an HTTPS route with edge TLS termination.

. Run the following command to get the route URL:
+
[source,terminal]
----
$ oc get route <route_name> -n openshift-builds -o jsonpath='{.spec.host}'
----

. Access the metrics endpoint using the route:
+
[source,terminal]
----
$ curl -k https://<route_host>/metrics
----
+
Replace `<route_host>` with the hostname from the previous step.
+
The output displays Prometheus-formatted metrics from the build controller.
