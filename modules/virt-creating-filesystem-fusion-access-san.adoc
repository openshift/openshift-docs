= Creating a file system with {IBMFusionSAN}

Create a file system to represent your required storage. The file system is based on the storage available in the nodes you selected when creating the storage cluster.

.Prerequisites

* You have created an {IBMFusionSAN} storage cluster.

.Procedure

. In the {product-title} web console, navigate to *Storage* -> *{IBMFusionSAN}*.

. In the *File systems* tab, click *Create file system*.

. Enter a *Name* for the new file system.

. Select the LUNs you want to use as the storage volumes for your file system.

. Click *Create file system*. 

The *{IBMFusionSAN}* page reloads, and the new file system appears in the *File systems* tab.

Repeat these steps for each file system you want to create.

.Verification

. Track the *Status* of the file system in the *File systems* tab until it is marked as *Healthy*.

. Click on the *StorageClass* for the file system.

. In the *YAML* tab, verify the following:
+
* The `name` is the name of the file system you created.
* The `provisioner` is `spectrumscale.csi.ibm.com`.
* The `volBackendFs` matches the name of the file system you created.

+
.Example YAML
[source,yaml,subs="attributes+"]
----
kind: StorageClass
apiVersion: storage.k8s.io/v1
metadata:
  name: filesystem1
  uid: eb410309-a043-a89b-9bb05483872a
  resourceVersion: `87746`
  creationTimestamp: `2025-05-14T12:30:08Z`
  managedFields:
provisioner: spectrumscale.csi.ibm.com
parameters:
  volBackendFs: filesystem1
reclaimPolicy: Delete
allowVolumeExpansion: true
volumeBindingMode: Immediate
----
