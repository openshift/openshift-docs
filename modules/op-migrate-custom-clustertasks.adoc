= Migrate custom ClusterTask resources

[role="_abstract"]
If your organization created custom `ClusterTask` resources, you must convert them to namespace-scoped `Task` resources before upgrading to {pipelines-title} 1.17 or later.

.Prerequisites

* You have identified the custom `ClusterTask` resources in your cluster.
* You have the necessary permissions to create `Task` resources.

.Procedure

. Export the existing `ClusterTask` resource to a YAML file by running the following command:
+
[source,terminal]
----
$ oc get clustertask my-custom-task -o yaml > my-custom-task.yaml
----

. Edit the YAML file to convert it to a namespace-scoped `Task`:
+
--
.. Change `kind: ClusterTask` to `kind: Task`.
.. Update `apiVersion` from `tekton.dev/v1beta1` to `tekton.dev/v1` if necessary.
.. Add a `namespace` field under `metadata`.
.. Remove generated fields such as `creationTimestamp`, `generation`, `resourceVersion`, `selfLink`, and `uid`.
.. Remove the `status` section if present.
--
+
--
*Example of a converted task:*
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Task
metadata:
  name: my-custom-task
  namespace: my-shared-tasks
spec:
  params:
  - name: MESSAGE
    type: string
    default: "Hello from custom task"
  steps:
  - name: echo-message
    image: registry.access.redhat.com/ubi9/ubi-minimal:latest
    script: |
      #!/bin/sh
      echo "$(params.MESSAGE)"
----
`metadata.namespace`:: Specifies the namespace where the task is created.
--

. Create the namespace if it does not exist by running the following command:
+
[source,terminal]
----
$ oc create namespace my-shared-tasks
----

. Apply the converted task to the cluster by running the following command:
+
[source,terminal]
----
$ oc apply -f my-custom-task.yaml
----

. Update all pipeline references to use the cluster resolver by running the following command:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: Pipeline
metadata:
  name: example-pipeline
spec:
  tasks:
  - name: run-custom-task
    taskRef:
      resolver: cluster
      params:
      - name: kind
        value: task
      - name: name
        value: my-custom-task
      - name: namespace
        value: my-shared-tasks 
----
`spec.taskRef.params.value`:: Specifies the namespace where you have created the converted task.

. If you want to share tasks across multiple projects, consider creating a dedicated namespace for shared tasks by running the following command:
+
[source,terminal]
----
$ oc create namespace shared-pipeline-tasks
----
+
Then, grant access to the namespace by creating a `RoleBinding` for each project that needs to use the tasks:
+
[source,yaml]
----
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: pipeline-tasks-viewer
  namespace: shared-pipeline-tasks
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: view
subjects:
- kind: ServiceAccount
  name: pipeline
  namespace: my-project
----
`subjects.namespace`:: Specifies the namespace where your pipelines run.

.Verification

. Verify that the task was created successfully by running the following command:
+
[source,terminal]
----
$ oc get task my-custom-task -n my-shared-tasks
----

. Create a `TaskRun` to test the converted task:
+
[source,yaml]
----
apiVersion: tekton.dev/v1
kind: TaskRun
metadata:
  name: test-custom-task
spec:
  taskRef:
    resolver: cluster
    params:
    - name: kind
      value: task
    - name: name
      value: my-custom-task
    - name: namespace
      value: my-shared-tasks
----

. View the task run logs to confirm it works correctly by running the following command:
+
[source,terminal]
----
$ tkn taskrun logs test-custom-task -f
----
