= Backup verification

[role="_abstract"]
You can verify that a recent etcd backup exists before starting a cluster update.

Recent etcd backup check::

Verify that an etcd backup exists from within the past 24 hours.

For automated backups, check your backup storage location (file system, S3 bucket, or other) for recent backup files with timestamps from the past 24 hours.

To create a manual backup:
[source,terminal]
----
$ oc debug node/<control_plane_node> -- chroot /host /usr/local/bin/cluster-backup.sh /home/core/assets/backup
----

Document the backup location and timestamp for potential disaster recovery scenarios.
