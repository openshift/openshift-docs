= Setting up {ibm-cloud-title} Object Storage

You can set up {ibm-cloud-title} Object Storage on the command line.

.Prerequisites

* You have installed the latest version of the OpenShift CLI. For more information, see "Getting started with the OpenShift CLI" in _CLI tools_.

* You have installed the latest version of the {ibm-cloud-title} Command Line Interface (AWS CLI). For more information, see "Getting started with the {ibm-cloud-title} CLI" in _{ibm-cloud-title} Docs_.

.Procedure

. Set the {ibm-cloud-title} credential. In this case, the credential is fetched from a Cluster deployed on {ibm-cloud-title}:
+
[source,terminal]
----
export IBMCLOUD_API_KEY=$(oc get secrets ibmcloud-creds -n kube-system -o json | jq -r '.data.apiKey' | base64 -d)
----
+
. Log in to {ibm-cloud-title} by using {ibm-cloud-title} CLI:
+
[source,terminal]
----
$ ibmcloud login -r us-east 
----
+
. Install the `cloud-object-storage` and `infrastructure-service` plugins:
+
[source,terminal]
----
$ ibmcloud plugin install cloud-object-storage \
  && \
  ibmcloud plugin install infrastructure-service
----
+
. Create a resource group:
+
[source,terminal]
----
$ ibmcloud resource group-create tempo-tracing \
  && \
  ibmcloud target -g tempo-tracing
----
+
. Create a service instance for the object store:
+
[source,terminal]
----
$ ibmcloud resource service-instance-create -d premium-global-deployment \
  <tempo_bucket> cloud-object-storage standard global \
  && \
  export IBM_SERVICE_INSTANCE_ID=$(ibmcloud resource service-instance <tempo_bucket> -o json | jq -r '.[0].crn')
----
+
. Create an object store bucket:
+
[source,terminal]
----
$ ibmcloud cos bucket-create --bucket <tempo_bucket> --ibm-service-instance-id "$IBM_SERVICE_INSTANCE_ID"
----
+
. Create a service key for connecting to the object store bucket:
+
[source,terminal]
----
$ ibmcloud resource service-key-create <tempo_bucket> Writer --instance-name <tempo_bucket> --parameters '{"HMAC":true}'
----
+
. Create a secret in the {product-title} cluster with the {ibm-cloud-title} bucket credentials:
+
[source,terminal]
----
$ oc new-project tracing-system
----
+
. Extract the access and secret keys to be used on secret creation:
+
[source,terminal]
----
$ json_output=$(ibmcloud resource service-key <tempo_bucket> -o json) \
  && \
  export IBM_BUCKET_ACCESS_KEY=$(echo $json_output | jq -r '.[0].credentials.cos_hmac_keys.access_key_id') \
  && \
  export IBM_BUCKET_SECRET_KEY=$(echo $json_output | jq -r '.[0].credentials.cos_hmac_keys.secret_access_key') \
  && \
  export IBM_BUCKET_ENDPOINT="https://s3.us-east.cloud-object-storage.appdomain.cloud"
----
+
. Create a secret on {ibm-cloud-title}:
+
[source,terminal]
----
$ oc -n tracing-system create secret generic ibm-cos-secret \
  --from-literal=bucket="<tempo_bucket>" \
  --from-literal=endpoint="$IBM_BUCKET_ENDPOINT" \
  --from-literal=access_key_id="$IBM_BUCKET_ACCESS_KEY" \
  --from-literal=access_key_secret="$IBM_BUCKET_SECRET_KEY"
----
+
. Set the storage section in the TempoStack custom resource:
+
[source,yaml]
----
apiVersion: tempo.grafana.com/v1alpha1
kind: TempoStack
# ...
spec:
# ...
  storage:
    secret:
      name: ibm-cos-secret # <1>
      type: s3 # <2>
# ...
----
<1> Name of the secret that contains {ibm-cloud-title} Storage access and secret keys.
<2> This should be set to S3.
