= Scale considerations

[role="_abstract"]
In large environments with thousands of namespaces, user scopes covering large numbers of namespaces can cause performance issues such as slow page rendering or timeouts.

In large environments, for example, with thousands of namespaces, the application of user scopes that cover large numbers of namespaces can cause some pages to render slowly or time out. For example, you might experience timeouts when opening the main *Dashboard* page.

Red{nbsp}Hat tested performance in a single-cluster context on an environment with 20,000 deployments across 5,000 namespaces and 80,000 alerts in one test cluster. In that testing, the load time of the main *Dashboard* page for scopes including up to 2,000 namespaces on that cluster was under a few seconds.

Red{nbsp}Hat tested performance in a multi-cluster context on an environment with 200,000 deployments across 50,000 namespaces and 800,000 alerts across 10 test clusters. In that testing, the load time of the main *Dashboard* page was under a few seconds for scopes including up to 500 namespaces across the 10 clusters and for scopes including up to 2,000 namespaces on a single cluster.

Red{nbsp}Hat cannot guarantee that the load time of {product-title-short} portal pages for user scopes with more namespaces than the tested numbers will stay under the timeout limit.
