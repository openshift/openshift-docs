= Creating a {product-title} cluster for {autonode}

[role="_abstract"]
Create a {product-title} cluster configured for the {autonode} private preview program.

Because the {autonode} private preview requires a new cluster in a specific provisioning shard, you must create your cluster with additional parameters. The cluster must be created in the `us-east-1` Amazon Web Services (AWS) region and must use OpenShift version 4.19 or later.

.Prerequisites

* You have completed the procedures to create AWS Identity and Access Management (IAM) resources, the {autonode} IAM policy, and the {autonode} IAM role.
* You have captured the {autonode} role Amazon Resource Name (ARN) in the `$ROLE_ARN` environment variable.
* You have the required AWS quota for creating a {product-title} cluster in the `us-east-1` region.
* You have network requirements prepared, including Virtual Private Cloud (VPC), subnets, and Network Address Translation (NAT) gateways if using a custom VPC.

.Procedure

. Create your {product-title} cluster with the required {autonode} parameters:
+
[source,terminal]
----
$ rosa create cluster \
  --properties=provision_shard_id:9f11dd2b-98c1-11f0-8fe5-0a580a830a08 \
  --region=us-east-1 \
  --hosted-cp \
  --version 4.19.15 \
  --cluster-name <cluster_name> \
  --sts \
  --mode auto \
  --operator-roles-prefix $PREFIX \
  --oidc-config-id $OIDC_CONFIG_ID \
  -y
----
+
Replace `<cluster_name>` with your preferred cluster name.
+
[IMPORTANT]
====
* The `--properties=provision_shard_id:9f11dd2b-98c1-11f0-8fe5-0a580a830a08` parameter is required for the {autonode} private preview and directs the cluster to the correct provisioning shard.
* The `--region=us-east-1` parameter is required for the {autonode} private preview. Other regions are not currently supported.
* The `--version 4.19.15` parameter specifies OpenShift version 4.19 or later, which is required for {autonode} support.
====
+
This command creates a {product-title} cluster configured for {autonode}. Additional cluster creation parameters can be specified as needed for your environment.

. Capture your cluster ID for use in subsequent steps:
+
[source,terminal]
----
$ export CLUSTER_ID=$(rosa describe cluster -c <cluster_name> | grep "^ID:" | awk '{print $2}')
----
+
Replace `<cluster_name>` with the name of your cluster.

. Verify that the cluster enters the installing state:
+
[source,terminal]
----
$ rosa describe cluster -c $CLUSTER_ID | grep -i State
----
+
.Example output
[source,terminal]
----
State:                      installing
----

. Wait for the cluster to reach the ready state:
+
[source,terminal]
----
$ rosa describe cluster -c $CLUSTER_ID --watch
----
+
This command monitors the cluster status until it reaches the `ready` state. This process typically takes 10-15 minutes.

.Verification

* Verify that your cluster is in the `ready` state:
+
[source,terminal]
----
$ rosa describe cluster -c $CLUSTER_ID | grep -i State
----
+
.Example output
[source,terminal]
----
State:                      ready
----

* Verify that the cluster was created with the correct provisioning shard:
+
[source,terminal]
----
$ rosa describe cluster -c $CLUSTER_ID | grep -i properties
----
+
.Example output
[source,terminal]
----
Properties:                 provision_shard_id:9f11dd2b-98c1-11f0-8fe5-0a580a830a08
----

.Next steps
