= {oadp-short} Self-Service prerequisites

Before you start using {oadp-short} Self-Service as a non-admin user, ensure you meet the following prerequisites:

* The cluster administrator has configured the {oadp-short} `DataProtectionApplication` (DPA) CR to enable Self-Service. 
* The cluster administrator has:
** Created a non-admin user account. See "Configuring an htpasswd identity provider", for more details.
** Created a namespace for the non-admin user.
** Assigned appropriate privileges for the non-admin user's namespace. This ensures that the non-admin user is authorized to access and perform backup and restore operations in their assigned namespace.
* Optionally, the cluster administrator can create a `NonAdminBackupStorageLocation` (NABSL) for the non-admin user.
