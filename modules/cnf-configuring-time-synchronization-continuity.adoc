= Configuring GNSS/NTP failover

Telco operators require time source redundancy to ensure time synchronization continuity and system stability.

{product-title} provides automatic failover capabilities to maintain synchronization. The system utilizes GNSS (delivered by `phc2sys`) as the Primary Time Source. To protect against primary signal loss, such as jamming or antenna failure, the system automatically transitions to the Secondary Time Source, NTP delivered by `chronyd`. Upon signal recovery, the system automatically switches back to and resumes synchronization with `phc2sys`.

You can control the resilience of the time synchronization by setting the `ts2phc.holdover` parameter in seconds. This value dictates the maximum time the internal control algorithm that disciplines the clock is allowed to continue synchronizing the PHC after the main time of day (ToD) source—like a GNSS receiver—is lost. This continuation is only permitted as long as the control algorithm remains in a stable state (`SERVO_LOCKED_STABLE`). When the process exits this configured holdover period, it signifies an unrecoverable primary signal loss, which then allows for failover to a secondary source like NTP.
