= Configuring time synchronization continuity

Telco Operators and SREs require tools to ensure time synchronization continuity and system stability in environments that demand extremely high time accuracy, typically within 1.5 microseconds. To meet these requirements, {product-title} provides redundancy against potential GNSS signal loss (the primary time source) by automatically transitioning to the Network Time Protocol (NTP) as a secondary source.

This capability provides redundancy against potential GNSS signal loss, which could be caused by issues like jamming or antenna failure. The mechanism for achieving this continuity involves the system automatically transitioning from the Primary Time Source, GNSS (delivered via phc2sys), to the Secondary Time Source, NTP (delivered via chronyd), upon detection of signal loss. Once the GNSS signal becomes available again, the system automatically recovers by switching back and resuming synchronization with `phc2sys6`.

The crucial metric that allows customers to control this synchronization process is the `ts2phc.holdover` parameter. The automatic failover from GNSS to NTP is initiated specifically when the `ts2phc` process exits its configured holdover state. The `ts2phc.holdover` value, defined in seconds, dictates the maximum duration the system will rely on its internal clock before declaring the GNSS signal lost and initiating the failover to NTP. By configuring this single value, you directly control the system's resilience and how quickly it reacts to primary signal loss, making it the central quantifiable metric for success.
