= Configuring worker node batching by using machine config pools

[role="_abstract"]
You can configure the `MachineConfigPool` resource to control how many worker nodes update simultaneously during a cluster update.
Adjusting the `maxUnavailable` setting balances update speed against workload availability.

.Prerequisites

* Access to the target cluster with cluster-admin privileges.
* The `oc` CLI tool is installed and configured.

.Procedure

. Check the current `maxUnavailable` setting for the worker `MachineConfigPool` resource by running the following command:
+
[source,terminal]
----
$ oc get mcp worker -o jsonpath='{.spec.maxUnavailable}'
----

. Patch the worker `MachineConfigPool` resource to set the required `maxUnavailable` value:
+
[source,terminal]
----
$ oc patch mcp worker --type merge -p '{"spec":{"maxUnavailable":"<max_unavailable>"}}'
----
+
Replace `<max_unavailable>` with one of the following values:
+
--
* An integer, for example `1`, to specify the exact number of nodes that can be unavailable during the update.
* A percentage, for example `"50%"`, to specify the proportion of nodes that can be unavailable.
--
+
The following example sets `maxUnavailable` to `1`, which updates worker nodes one at a time:
+
[source,yaml]
----
apiVersion: machineconfiguration.openshift.io/v1
kind: MachineConfigPool
metadata:
  name: worker
spec:
  maxUnavailable: 1
----
+
The following example sets `maxUnavailable` to `100%`, which updates all worker nodes simultaneously for faster updates:
+
[source,yaml]
----
apiVersion: machineconfiguration.openshift.io/v1
kind: MachineConfigPool
metadata:
  name: worker
spec:
  maxUnavailable: "100%"
----
+
NOTE: The default `maxUnavailable` value is 1 worker node or 33% of worker nodes, whichever is smaller. Setting a higher value accelerates updates but increases the number of nodes unavailable at any given time. Consult your application teams to find the appropriate value for your workloads.

. Verify the updated setting by running the following command:
+
[source,terminal]
----
$ oc get mcp worker -o yaml
----

.Verification

* Verify the `MachineConfigPool` resource reflects the updated `maxUnavailable` value by running the following command:
+
[source,terminal]
----
$ oc get mcp worker -o jsonpath='{.spec.maxUnavailable}'
----
