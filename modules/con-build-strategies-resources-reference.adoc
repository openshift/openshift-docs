= Build strategies and resources

[role="_abstract"]
You can optimize resource allocation and configure strategy-specific features for your builds.
Each build strategy (Buildpacks, Buildah, Source-to-Image) has unique resource consumption patterns and specialized parameters.

Resource consumption patterns:: Documents typical CPU, memory, and storage requirements for each build strategy based on common workload characteristics.
Strategy-specific parameters:: Details unique configuration options available for each strategy, including builder images, platform targets, and optimization flags.
Cache configuration:: Explains how each strategy implements layer caching to improve build performance and reduce resource consumption.
Security context requirements:: Documents the security constraints and ServiceAccount permissions needed for each strategy to run successfully.
Multi-platform support:: Describes platform architecture targeting capabilities and cross-compilation options available for each strategy.

Strategy-specific resource patterns help you set appropriate resource limits, choose the right strategy for your workload, and configure builds to balance performance with cluster resource availability.
