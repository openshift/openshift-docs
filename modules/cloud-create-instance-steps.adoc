= Creating an instance in the console

[role="_abstract"]
In the {cloud-console}, create an *ACS instance* to connect to your secured clusters.

.Procedure

. Log in to the {cloud-console}.
. From the navigation menu, select *Advanced Cluster Security* -> *ACS Instances*.
. Select *Create ACS instance* and enter information into the displayed fields or select the appropriate option from the drop-down list:
* *Name*: Enter the name of your *ACS instance*. An *ACS instance* contains the {product-title-short} Central component, also referred to as "Central", which includes the {product-title-managed-short} management interface and services that Red{nbsp}Hat configures and manages for you. You manage your secured clusters that communicate with Central. You can connect many secured clusters to one instance.
* *Cloud provider*: The cloud provider that hosts Central. Select *AWS*.
* *Cloud region*: The region for your cloud provider that hosts Central. Select one of the following regions:
** US-East, N. Virginia
** Europe, Ireland
* *Availability zones*: Use the default value (*Multi*).
. Click *Create instance*.
