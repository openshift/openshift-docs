= Minimal metrics collected for the log collector

[role="_abstract"]
When you use the `minimal` metrics collection profile, Prometheus collects only the metrics required for platform functionality: alerts, console dashboards, telemetry, and Horizontal and Vertical Pod Autoscaling. This significantly reduces metrics cardinality compared to the `full` profile.

[NOTE]
====
The minimal profile excludes high-cardinality histogram metrics, such as `vector_component_received_events_count_bucket` and `vector_buffer_send_duration_seconds_bucket`. These metrics create thousands of time series in multitenant environments but are not required for platform functionality.

Additionally, the minimal profile drops the following metrics specifically for transform components (`component_kind=transform`): `vector_component_received_bytes_total`, `vector_component_received_event_bytes_total`, `vector_component_received_events_total`, and `vector_component_sent_bytes_total`. This further reduces cardinality while preserving metrics for source and sink components where they are most valuable.
