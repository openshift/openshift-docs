= Diagnosing traffic balancing issues

[role="_abstract"]
Diagnose traffic balancing issues to identify why requests are unevenly distributed or failing to reach backend services. Resolving these routing issues ensures high availability and reliable performance for your applications.

.Prerequisites
* You have installed the {oc-first}.
* You have access to the {product-title} cluster with `cluster-admin` privileges.

.Procedure

. Verify that your `Gateway` resource is correctly configured and accepted by the controller by running the following command:
+
[source,terminal]
----
$ oc get gateway <gateway_name> -n <namespace> -o yaml
----
+
Ensure that the conditions in the `status` section report `Accepted: True` and `Programmed: True`.

. Check the routing configuration in your `HTTPRoute` resource to ensure it is successfully attached to the Gateway and targeting the correct backend services:
+
[source,terminal]
----
$ oc describe httproute <route_name> -n <namespace>
----

. Verify that your backend service has healthy, active endpoints that correspond to your application pods by running the following command:
+
[source,terminal]
----
$ oc get endpoints <service_name> -n <namespace>
----
+
If the endpoints list is empty, the Gateway API cannot balance traffic because the backend pods are either failing their readiness probes or the service selector is misconfigured.

. Check the logs of your backend application pods to confirm whether they are receiving the balanced requests:
+
[source,terminal]
----
$ oc logs -l <app_label_selector> -n <namespace>
----
