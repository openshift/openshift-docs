= Enable network observability feature gate on existing clusters

[role="_abstract"]
Enable the `NetworkObservabilityInstall` feature gate on a running cluster to configure automated network observability installation through the Network CR.

You can enable the `NetworkObservabilityInstall` feature gate on an existing cluster by editing the cluster-wide `FeatureGate` resource. After enabling the feature gate, you can configure the `installationPolicy` field in the Network CR to control whether network observability is installed. This is useful for testing Technology Preview features on development or test clusters.

[NOTE]
====
Enabling the feature gate alone does not install Network Observability. You must also configure the Network CR `spec.networkObservability.installationPolicy` field to `InstallAndEnable` to trigger installation on an existing cluster.
====

.Prerequisites

* You have access to the cluster as a user with the `cluster-admin` role.

.Procedure

. Edit the cluster-wide `FeatureGate` resource by running the following command:
+
[source,terminal]
----
$ oc edit featuregate cluster
----

. Add the `NetworkObservabilityInstall` feature gate to the configuration:
+
[source,yaml]
----
apiVersion: config.openshift.io/v1
kind: FeatureGate
metadata:
  name: cluster
spec:
  featureSet: CustomNoUpgrade
  customNoUpgrade:
    enabled:
    - NetworkObservabilityInstall
----
+
where:
+
`spec.featureSet`:: Set to `CustomNoUpgrade` to enable a custom set of Technology Preview features.
`spec.customNoUpgrade.enabled`:: Lists the specific feature gates to enable. Add `NetworkObservabilityInstall` to the list of enabled features.

. Save and close the file.

. Wait for the Machine Config Operator to apply the changes to all nodes. This process can take several minutes as nodes are updated sequentially.
+
[IMPORTANT]
====
Enabling the `CustomNoUpgrade` feature set cannot be undone and prevents minor version updates. Use this feature set only on test clusters where you can fully test Technology Preview features.
====

.Verification

* Verify that the feature gate is enabled by running the following command:
+
[source,terminal]
----
$ oc get featuregate cluster -o jsonpath='{.spec.customNoUpgrade.enabled}'
----
+
The output should include `NetworkObservabilityInstall`:
+
[source,terminal]
----
["NetworkObservabilityInstall"]
----

* Monitor the Machine Config Operator status to ensure all nodes are updated by running the following command:
+
[source,terminal]
----
$ oc get mcp
----
+
Wait until all machine config pools show `UPDATED=True` and `UPDATING=False`.
