DPA no longer reports an error when the `podConfig` value is empty and `nodeAgent` affinity is specified::
Before this update, the `DataProtectionApplication` (DPA) custom resource reported an error due to an empty `podConfig` with specified `nodeAgent` affinity. This caused DPA to fail to reconcile. With this release, DPA reports an error only when `nodeAgent.podConfig.nodeSelector` does not match `spec.configuration.nodeAgent.LoadAffinityConfig` when `podConfig` is explicitly set.
+
link:https://issues.redhat.com/browse/OADP-7017[OADP-7017]

Backup deletion no longer hangs when CSI snapshots are missing::
Before this update, the Velero `DeleteBackupRequest` queue got stuck when attempting to delete backups that referenced non-existent Amazon Web Services (AWS) EBS snapshots. This occurred after removing schedules in the granular schedules-per-hosted-cluster model. With this release, backup deletion handles missing CSI snapshots correctly and completes without getting stuck.
+
link:https://issues.redhat.com/browse/OADP-7537[OADP-7537]

Windows VM backups no longer fail with `QuiesceFailed` errors::
Before this update, Windows virtual machine (VM) snapshot operations failed because the guest agent became unresponsive during the freeze operation, causing `QuiesceFailed` errors. With this release, CSI snapshot early frequent polling is enabled for all snapshots, which resolves the timing issue with the guest agent during the freeze operation.
+
link:https://issues.redhat.com/browse/OADP-7543[OADP-7543]

Wildcard handling for `excludedClusterScopedResources` no longer causes validation failures::
Before this update, the {oadp-short} Operator incorrectly appended default resources to `excludedClusterScopedResources` in the `NonAdminBackup` CR when a wildcard (`*`) was present. This caused `FailedValidation` errors when excluding all cluster-scoped resources. With this release, the Operator correctly handles the wildcard without appending default resources.
+
link:https://issues.redhat.com/browse/OADP-7562[OADP-7562]

CSI snapshot early frequent polling is configurable in the `DataProtectionApplication` CR::
With this release, you can disable or configure the Container Storage Interface (CSI) snapshot early frequent polling interval in the {oadp-short} `DataProtectionApplication` (DPA) custom resource (CR). This allows you to tune snapshot polling behavior to better suit your environment.
+
link:https://issues.redhat.com/browse/OADP-7883[OADP-7883]
