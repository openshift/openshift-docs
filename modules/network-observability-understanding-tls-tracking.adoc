= TLS tracking

[role="_abstract"]
Monitor TLS traffic metrics from handshake messages to identify insecure cipher suites and deprecated protocol versions without decrypting traffic.

The TLS tracking feature in Network Observability analyzes the security posture of encrypted traffic by capturing metadata from TLS handshake messages. This feature identifies the TLS versions and cipher suites in use, detects deprecated configurations, and assists in maintaining compliance without decrypting traffic data.

[id="how-tls-tracking-works_{context}"]
== How TLS tracking works

The eBPF agent captures TLS metadata from two primary handshake messages:

* `ClientHello`: Initiates the TLS handshake and contains the TLS version requested by the client.
* `ServerHello`: Responds to the client and contains the negotiated TLS version, cipher suite, and elliptic curve information for TLS 1.3.

The agent does not decrypt traffic or access the content of encrypted communications. It captures only the protocol metadata visible during the handshake phase.



[id="tls-field-availability_{context}"]
== TLS field availability

TLS field population depends on the message type and the TLS version:

[cols="2,2,4",options="header"]
|===
|Field |Message type |Description

|TLS Version
|`ClientHello`, `ServerHello`
|The negotiated protocol version, such as 1.2 or 1.3.

|TLS Cipher Suite
|`ServerHello` only
|The negotiated cryptographic algorithm suite. Displays as `n/a` in `ClientHello` messages.

|TLS Curve
|`ServerHello` only (TLS 1.3)
|The elliptic curve used for key exchange. Displays as `n/a` for TLS 1.2 or in `ClientHello` messages.
|===

[IMPORTANT]
====
To view complete TLS metadata for a flow, filter for `ServerHello` message types. `ClientHello` messages contain limited metadata.
====

[id="tls-use-cases_{context}"]
== Use cases

TLS tracking provides the following capabilities:

* Protocol identification: Detects workloads using deprecated TLS versions, such as 1.0 or 1.1.
* Cipher suite monitoring: Verifies the use of strong cryptographic algorithms.
* Compliance auditing: Audits TLS configurations to meet regulatory standards.
* Traffic visualization: Displays encrypted connections in the topology view.
* Alerting: Reports insecure TLS configurations through Prometheus alerts.
