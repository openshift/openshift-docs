= Storage requirements

[role="_abstract"]
The hosting cluster must have a storage solution that supports VM disk images and meets the latency requirements of `etcd`.

image::vcp_storage_arch.png[Virtualized control plane storage architecture]

Use {lvms-first} backed by NVMe drives for virtualized control plane storage.
{lvms} writes directly to local disks, providing the consistent low latency that `etcd` requires.
Use dedicated NVMe drives for control plane VM disks to ensure predictable performance.

[IMPORTANT]
====
Do not use {odf-first} for virtualized control plane storage.
{odf-short} replicates data across multiple storage nodes, which can cause unpredictable latency spikes during synchronization that affect `etcd` performance.
====
