= Configuring an address pool

[role="_abstract"]
Configure an `IPAddressPool` to define the IP address ranges that MetalLB can assign to `LoadBalancer` services.

.Prerequisites

* Install the {oc-first}.
+
Verify: `oc version`
* Log in as a user with `cluster-admin` privileges.
+
Verify: `oc auth can-i create namespace --all-namespaces`
* Install and start MetalLB on your cluster.
+
Verify: `oc get metallb -n metallb-system`

.Procedure

. Create a file, such as `ipaddresspool.yaml`, with content like the following example:
+
.IPAddressPool parameters
[cols="2,1,3",options="header"]
|===
| Parameter | Format | Description

| `addresses`
| CIDR or range
| One or more IP address ranges for `LoadBalancer` services. Use CIDR notation (`203.0.113.0/24`) or hyphenated ranges (`203.0.113.1-203.0.113.10`). These addresses must not be assigned by DHCP. For L2 mode, the addresses must be on the same Layer 2 subnet as the cluster node interfaces so that MetalLB can respond to ARP requests. For BGP mode, the addresses must be routable by the BGP peer router.

| `name`
| DNS-compatible string
| Name for the `IPAddressPool` resource. Referenced by `L2Advertisement` or `BGPAdvertisement` resources.

| `autoAssign`
| Boolean
| If `true` (default), MetalLB automatically assigns addresses from this pool to new `LoadBalancer` services. Set to `false` to require explicit pool selection by using the `metallb.universe.tf/address-pool` annotation.

| `labels`
| Key-value pairs
| Optional. Labels that can be referenced by `ipAddressPoolSelectors` in advertisement resources to associate this pool with an advertisement.
|===
+
[source,yaml]
----
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  namespace: metallb-system
  name: doc-example
  labels:
    zone: east
spec:
  addresses:
  - 203.0.113.1-203.0.113.10
  - 203.0.113.65-203.0.113.75
# ...
----

. Apply the configuration for the IP address pool by entering the following command:
+
[source,terminal]
----
$ oc apply -f ipaddresspool.yaml
----

.Verification

. View the address pool by entering the following command:
+
[source,terminal]
----
$ oc describe -n metallb-system IPAddressPool doc-example
----
+
.Example output
[source,terminal]
----
Name:         doc-example
Namespace:    metallb-system
Labels:       zone=east
Annotations:  <none>
API Version:  metallb.io/v1beta1
Kind:         IPAddressPool
Metadata:
  ...
Spec:
  Addresses:
    203.0.113.1-203.0.113.10
    203.0.113.65-203.0.113.75
  Auto Assign:  true
Events:         <none>
----

. Confirm that the address pool name, such as `doc-example`, and the IP address ranges exist in the output.

.Troubleshooting
[cols="2,2,2",options="header"]
|===
| Symptom | Likely cause | Resolution

| `IPAddressPool` not created
| `MetalLB` instance not running
| Verify that the `MetalLB` custom resource exists: `oc get metallb -n metallb-system`.

| `LoadBalancer` service stuck at `<pending>`
| No `IPAddressPool` configured, pool exhausted, or no advertisement configured
| Check available pools: `oc get ipaddresspool -n metallb-system`. Verify an `L2Advertisement` or `BGPAdvertisement` references the pool.

| IP address conflict on the network
| Pool addresses overlap with DHCP or other static assignments
| Verify that the addresses in the pool are not in use on the network.
|===
