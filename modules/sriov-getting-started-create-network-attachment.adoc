= Create an SR-IOV network attachment

[role="_abstract"]
Create an SR-IOV network attachment definition that pods can use to connect to the SR-IOV network.

.Procedure

. Create a file named `sriov-network.yaml` with the following content:
+
[source,yaml]
----
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetwork
metadata:
  name: sriov-network-1  
  namespace: openshift-sriov-network-operator
spec:
  resourceName: intelnics  
  networkNamespace: default  
  vlan: 0  
----
 Name for this network attachment. Pods will reference this name.
 Must match the `resourceName` from your SR-IOV network node policy.
 Namespace where pods can use this network. Start with `default` namespace for testing.
 VLAN ID. Use `0` for untagged traffic on the native VLAN.
+
[NOTE]
====
This configuration does not include IPAM (IP address management). Pods using this network will assign IP addresses using static configuration in the pod specification. For dynamic IP assignment with DHCP or Whereabouts, see the "Configure IP address management for SR-IOV networks" guide.
====

. Create the SR-IOV network:
+
[source,terminal]
----
$ oc create -f sriov-network.yaml
----

.Verification

. Verify the network attachment definition was created:
+
[source,terminal]
----
$ oc get network-attachment-definitions -n default
----
+
.Example output
[source,terminal]
----
NAME              AGE
sriov-network-1   10s
----

. Verify the SR-IOV network resource:
+
[source,terminal]
----
$ oc get sriovnetwork -n openshift-sriov-network-operator sriov-network-1
----
+
.Example output
[source,terminal]
----
NAME              AGE
sriov-network-1   20s
----

The network attachment is now ready for pods to use.
