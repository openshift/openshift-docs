= Storage and runtime dependencies for builds

[role="_abstract"]
Configure storage for builds by defining volumes and volume mounts. These configurations enable builds to access persistent storage, configuration data, and temporary workspace needed during image construction.

Storage configuration includes:

* Volumes definition for a build: Persistent or ephemeral storage declared in the build specification
* Volumes definition for a buildrun: Runtime volume overrides that supplement or replace build-level volumes
* Volumes and volume mounts definition: How volumes connect to specific paths in build steps, giving build processes access to shared data
