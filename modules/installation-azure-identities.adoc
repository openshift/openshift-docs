= Supported identities to access {azure-short} resources

[role="_abstract"]
To install an {product-title} cluster on {azure-short}, you can use a service principal, a system-assigned managed identity, or a user-assigned managed identity to create and manage {azure-short} resources.

* A service principal
* A system-assigned managed identity
* A user-assigned managed identity

For more information on Azure identities, see "Managed identity types".
