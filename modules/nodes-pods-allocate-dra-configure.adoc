= Requesting devices in workloads by using device attributes

{attribute-based-full}

.Prerequisites

* The Nvidia DRA driver is installed. For more information see "NVIDIA DRA Driver for GPUs" in the "Additional Resources" section.
* The Nvidia GPU Operator is installed. For more information see "Adding Operators to a cluster" in the "Additional Resources" section.
* A resource slice and a resource claim or resource claim template has been created. For more information, see "About configuring device allocation by using device attributes" in the "Additional Resources" section.

.Procedure

. Create a pod by creating a YAML file similar to the following:

.Example pod that is requesting resources
[source,yaml]
----
apiVersion: v1
kind: Pod
metadata:
  namespace: gpu-test1
  name: pod1
  labels:
    app: pod
spec:
  containers:
  - name: ctr
    image: ubuntu:22.04
    command: ["bash", "-c"]
    args: ["nvidia-smi -L; trap 'exit 0' TERM; sleep 9999 & wait"]
    resources:
      claims: <1>
      - name: shared-gpu-template
      - name: shared-gpu
  - name: ctr1
    image: nvidia/cuda
    command: ["nvidia-smi" "-L"]
    resources:
      claims:
      - name: gpu
  resourceClaims: <2>
  - name: shared-gpu-template
    source:
      resourceClaimTemplate: single-gpu-template
  - name: shared-gpu
    source:
      resourceClaim: single-gpu
  tolerations:
  - key: "nvidia.com/gpu"
    operator: "Exists"
    effect: "NoSchedule"
----
<1> Specifies one or more resource claims to use with this container.
<2> Specifies the resource claims that are required for the containers to start. Include an arbitrary name for the resource claim request and the resource claim and/or resource claim template.
