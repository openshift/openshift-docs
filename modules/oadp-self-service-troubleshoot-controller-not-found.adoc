= Non-admin controller not found

If you get a `control-plane=non-admin-controller"` error, it means, the cluster administrator has not enabled {oadp-short} Self-Service in the `DataProtectionApplication` (DPA) custom resource (CR).

.Error

[source, terminal]
----
$ oc get pod -n openshift-adp -l  control-plane=non-admin-controller
Error from server (NotFound): pods "control-plane=non-admin-controller" not found
----

.Solution

The cluster administrator must enable the {oadp-short} Self-Service in the DPA as shown in the following example. For more details, see "Enabling {oadp-short} Self-Service".

.Example
[source, yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: DataProtectionApplication
metadata:
  name: oadp-backup
  namespace: openshift-adp
spec:
  configuration:
  ...
  nonAdmin:
    enable: true
----
