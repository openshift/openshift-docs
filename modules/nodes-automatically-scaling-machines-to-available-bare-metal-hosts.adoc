= Automatically scaling machines to the number of available bare metal hosts

You can configure `Machine` objects to be automatically created to match the number of available `BareMetalHost` objects. This is done by adding a `metal3.io/autoscale-to-hosts` annotation to the `MachineSet` object that you want to be scaled automatically.

.Prerequisites

* Install {op-system} bare metal compute machines for use in the cluster, and create corresponding `BareMetalHost` objects.
* Install the {product-title} CLI (oc).
* Log in as a user with `cluster-admin` privileges.

.Procedure

. Edit the machine set that you want to configure for automatic scaling by running the following command, replacing `<machineset>` with the name of the machine set to be changed:
+
[source,terminal]
----
$ oc edit machineset <machineset> -n openshift-machine-api
----

. Add the `metal3.io/autoscale-to-hosts` annotation, and set the number of machines that you want to scale to, for example:
+
[source,yaml]
----
apiVersion: machine.openshift.io/v1beta1
kind: MachineSet
metadata:
  annotations:
    metal3.io/autoscale-to-hosts: '<number to scale to>'
...
----
+
Wait for the new scaled machines to start.

[NOTE]
====
When a `BareMetalHost` object has been used to create by a machine in the cluster and labels or selectors are subsequently changed on the `BareMetalHost`, the `BareMetalHost` object will continue be counted against the `MachineSet` that the `Machine` object was created from.
====
