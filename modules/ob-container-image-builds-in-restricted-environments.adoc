= Container image builds in restricted environments

[role="_abstract"]
In network-restricted or air-gapped environments, you can build container images by configuring your cluster to use mirrored registries and proxy settings that comply with security policies.

Restricted environments often prevent direct access to external registries and repositories. To enable successful image builds, configure your cluster to retrieve base images, dependencies, and build resources from approved internal registries or mirrored repositories. 

You can also configure cluster-wide proxy settings to allow controlled access to external resources when required.
These configurations help maintain secure, consistent, and reliable container image builds while complying with organizational network and security requirements.
