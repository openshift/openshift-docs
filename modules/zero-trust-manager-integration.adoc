= SPIRE {zero-trust-full} integration with Istio

[role="_abstract"]
{SMProductName} integrates with SPIRE through {zero-trust-full} to provide flexible workload identity attestation and cryptographic identity management. This integration enables enhanced node and workload attestation beyond the default Kubernetes namespace and service account attestation offered by Istio.

== SPIFFE and SPIRE

SPIRE is a production-ready implementation of the SPIFFE specification that performs node and workload attestation to securely issue cryptographic identities to workloads running in heterogeneous environments.

SPIRE integrates with Istio through Envoy's Secret Discovery Service (SDS) API. Istio detects the presence of a UNIX Domain Socket that implements the Envoy SDS API on a defined socket path, allowing Envoy to communicate and fetch identities directly from SPIRE.

This integration provides flexible attestation options not available with default Istio identity management:

* **Diverse workload attestation**: SPIRE's plugin architecture enables attestation options beyond the Kubernetes namespace and service account attestation offered by Istio.

* **Node attestation**: SPIRE extends attestation to the physical or virtual hardware on which workloads run.

* **Service mesh integration**: SPIRE harnesses Istio's service management capabilities while providing enhanced identity security.

== Integration architecture

The integration between {SMProductName}, SPIRE, and {zero-trust-full} works as follows:

. {zero-trust-full} deploys and manages SPIRE Server and SPIRE Agent components.

. The SPIFFE Container Storage Interface (CSI) driver mounts the SPIRE Agent socket into workload pods.

. Custom Istio sidecar injection templates mount the socket into Envoy containers

. Istio proxies communicate with SPIRE through the SDS API to obtain certificates

. SPIRE issues X.509 SVIDs based on workload attestation

. Istio uses SPIRE-issued certificates for mTLS between services

This architecture avoids insecure host mounts by using the CSI driver to securely inject the Workload API socket, while maintaining compatibility with Istio's service mesh features.
