= Enabling the {prodname} dynamic plug-in in {ocp}

[role="_abstract"]
The {prodname} Operator automatically installs and configures the {prodname} dynamic plug-in for the {ocp} web console. You can use the {prodname} dynamic plug-in to view and manage your gateways and policies in the {ocp} web console.

NOTE: You must perform these steps on each {ocp} cluster that you want to use the {prodname} dynamic plug-in.

.Prerequisites

* See xref:install-prerequisites_{context}[].
//TODO
* You have access to the {ocp} web console.

.Procedure

. In the {ocp} web console, log in with `cluster-admin` privileges.

. In the left navigation menu, select the *Administrator* perspective.

. Click *Home > Overview*.

. In the *Status* panel, click *Dynamic Plugins* > *View all*.

. On the *Console plugins* tab, find the *kuadrant-console-plugin* entry in the table, which should be listed but disabled.

. In the *kuadrant-console-plugin* row, click *Disabled*.

. Select the *Enable* option, and click *Save*.

. Wait for the plug-in status to change to *Loaded*.

.Verification

After the plug-in is enabled, refresh the console. A new *{prodname}* menu item in the navigation sidebar.

You can click  *{prodname} > Overview* to explore the available resources and to get started with creating a Gateway and configuring policies in the {ocp} web console.
