= Configuring the workflow base builder image

If your scenario requires strict policies for image usage, such as security or hardening constraints, you can replace the default image used by the {ServerlessLogicOperatorName} to build the final workflow container image.

By default, the {ServerlessLogicOperatorName} uses the image distributed upstream to build workflows. To change this image, edit the SonataFlowPlatform custom resource in the namespace where you deployed your workflows:

.Prerequisites

* {ServerlessLogicOperatorName} is installed on your cluster.
* You have created a {ServerlessLogicProductName} project or have access to a project with the appropriate roles and permissions to create applications and other workloads in OpenShift Container Platform.
* You have created your {ServerlessLogicProductName} project
* Install the OpenShift CLI `(oc)`.

.Procedure

. Get the `SonataFlowPlatform` Name.
+
Use the following command to list all SonataFlowPlatform resources in your namespace:
+
[source,terminal]
----
$ oc get sonataflowplatform -n <your_namespace>
----
+ Replace `<your_namespace>` with the name of your namespace.

. Patch the SonataFlowPlatform with the new builder image.
+
Use the following command to patch the `SonataFlowPlatform` custom resource with the new builder image:
+
[source,terminal]
----
$ oc patch sonataflowplatform <name> --patch 'spec:\n  build:\n    config:\n      baseImage: <your_new_image_full_name_with_tag>' -n <your_namespace>
----

. Verify that the `SonataFlowPlatform` custom resource has been patched correctly by describing the resource:
+
[source,terminal]
----
$ oc describe sonataflowplatform <name> -n <your_namespace>
----
+
Ensure that the `baseImage` field under `spec.build.config` reflects the new image.
