= Verifying X.509 authentication layer 1

[role="_abstract"]
To test your layer 1 X.509 authentication, you can run test scenarios to check that connections are rejected.

.Prerequisites

* You are logged into {ocp} as a cluster administrator.
* You installed {prodname} using Istio as your `Gateway` object controller.
* You created your CAs and X.509-formatted certificates.
* You created your `Gateway` object.
* You created your route.
* You installed the `httpbin` sample application.

.Procedure

. Retrieve the `Gateway` object hostname by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ export GATEWAY_HOSTNAME=$(oc get gateway mtls-gateway -n _<gateway_system>_ -o jsonpath='{.status.addresses[0].value}')
----
+
Replace `_<gateway_system>_` with the namespace where your Gateway object is applied.

. Test with a valid certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.${GATEWAY_HOSTNAME}/get \
  --cert /tmp/client.crt \
  --key /tmp/client.key
----
+
.Expected output
[source,text]
----
HTTP 200
----

. Verify that the TLS handshake fails when there is no certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.${GATEWAY_HOSTNAME}/get
----
+
.Expected output
[source,text]
----
curl: (56) OpenSSL SSL_read: OpenSSL/3.6.2: error:0A00045C:SSL routines::tlsv13 alert certificate required, errno 0
----

. Verify that the TLS handshake fails when there is an untrusted certificate by running the following commands:
+
.. Generate an untrusted certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ openssl req -x509 -sha512 -nodes \
  -days 365 \
  -newkey rsa:4096 \
  -subj "/CN=Untrusted CA/O=Untrusted Org/C=US" \
  -keyout /tmp/untrusted-ca.key \
  -out /tmp/untrusted-ca.crt
----

.. Generate an untrusted private key by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ openssl genrsa -out /tmp/untrusted-client.key 4096
----

.. Create a Certificate Signing Request (CSR) for the untrusted client by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ openssl req -new \
  -subj "/CN=untrusted-client/O=Untrusted Org/C=US" \
  -key /tmp/untrusted-client.key \
  -out /tmp/untrusted-client.csr
----

.. Sign the request using your untrusted authority to issue the client certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ openssl x509 -req -sha512 \
  -days 365 \
  -CA /tmp/untrusted-ca.crt \
  -CAkey /tmp/untrusted-ca.key \
  -CAcreateserial \
  -extfile /tmp/x509v3.ext \
  -in /tmp/untrusted-client.csr \
  -out /tmp/untrusted-client.crt
----

.. Test the untrusted certificate by running the following command:
+
[source,terminal,subs="+quotes"]
----
$ curl -ik https://httpbin.${GATEWAY_HOSTNAME}/get \
  --cert /tmp/untrusted-client.crt \
  --key /tmp/untrusted-client.key
----
+
.Expected output
[source,text]
----
curl: (56) OpenSSL SSL_read: OpenSSL/3.6.2: error:0A000418:SSL routines::tlsv1 alert unknown ca, errno 0
----
