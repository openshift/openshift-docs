= About ClusterTask removal

[role="_abstract"]
The cluster resolver replaces `ClusterTask` functionality. The cluster resolver accesses namespace-scoped `Task` resources from any namespace in the cluster. Tasks that were previously installed as `ClusterTask` resources are now installed as `Task` resources in the {pipelines-shortname} `openshift-pipelines` namespace.

[IMPORTANT]
====
Before upgrading to {pipelines-title} 1.17 or later, update all pipeline definitions that reference `ClusterTask` resources. Pipelines that continue to use `kind: ClusterTask` fail after the upgrade.
====

The cluster resolver provides the following benefits over `ClusterTask`:

* *Namespace isolation*: Tasks are scoped to specific namespaces, providing better security and organization.
* *Fine-grained access control*: You can control access to tasks using namespace-level RBAC policies.
* *Version management*: Multiple versions of the same task can exist in different namespaces.
* *Flexibility*: You can reference tasks from any namespace on the cluster, not only cluster-scoped resources.

[NOTE]
====
Tasks are available in the `openshift-pipelines` namespace. You can use the cluster resolver to access these tasks from any namespace in your cluster.
====
