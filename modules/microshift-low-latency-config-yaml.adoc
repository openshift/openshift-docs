= Setting kubelet parameters and values in the {microshift-short} config.yaml

The first part of the low latency workflow is to add configurations to the {microshift-short} `config.yaml` file.

.Prerequisites

* You installed the OpenShift CLI (`oc`).
* You have root access to the cluster.
* You made a copy of the provided `config.yaml.default` file in the `/etc/microshift/` directory, renaming it `config.yaml`.
//anything else? do we need to create the KubeletConfiguration first? or does this config populate that for us?

.Procedure

. Add the kubelet configuration to the {microshift-short} `config.yaml` file

.Example `kubelet` configuration
[source,yaml]
----
# ...
kubelet:
  cpuManagerPolicy: static
  cpuManagerPolicyOptions:
    full-pcpus-only: "true"
  cpuManagerReconcilePeriod: 5s
  memoryManagerPolicy: Static
  topologyManagerPolicy: single-numa-node
  reservedSystemCPUs: 0-1
  reservedMemory:
  - limits:
      memory: 1100Mi <1>
    numaNode: 0
  kubeReserved:
    memory: 500Mi
  systemReserved:
    memory: 500Mi
  evictionHard:
    imagefs.available: 15%
    memory.available: 100Mi
    nodefs.available: 10%
    nodefs.inodesFree: 5%
  evictionPressureTransitionPeriod: 0s
# ...
----
//need footnotes
<1> The value in `reservedMemory[0].limits.memory`, 1100 Mi in this example, is equal to `kubeReserved.memory` + `systemReserved.memory` + `evictionHard.memory.available`.
