= Configuring machine pool AutoRepair

ROSA with HCP supports an automatic repair process for machine pools. AutoRepair is useful when you would want the ROSA service to detect certain unhealthy machines, drain the unhealthy nodes, and recreate the nodes. You can disable auto-repair if the unhealthy machines should not be replaced in cases where the nodes or the unhealthy instances should be preserved. AutoRepair is enabled by default on Machine Pools. 

The AutoRepair process deems a node unhealthy when the State of the node is either `NotReady` or Unknown for predefined amount of time (typically 8 minutes). Whenever two or more nodes become unhealthy simultaneously, the AutoRepair process will stop repairing the nodes.
Similarly, when a new node is created unhealthy even after a predefined amount of time (typically 20 minutes), the service will auto-repair.
