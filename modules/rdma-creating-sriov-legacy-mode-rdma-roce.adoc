= Creating a SRIOV Legacy Mode RDMA on RoCE

This procedure describes how to configure a SRIOV Legacy Mode host device RDMA on RoCE.

.Procedure

. Generate a new host device `NicClusterPolicy` custom resource (CR), as shown below.
+
[source,yaml]
----
$ cat <<EOF > network-sriovleg-nic-cluster-policy.yaml
apiVersion: mellanox.com/v1alpha1
kind: NicClusterPolicy
metadata:
  name: nic-cluster-policy
spec:
  ofedDriver:
    image: doca-driver
    repository: nvcr.io/nvidia/mellanox
    version: 24.10-0.7.0.0-0
    startupProbe:
      initialDelaySeconds: 10
      periodSeconds: 20
    livenessProbe:
      initialDelaySeconds: 30
      periodSeconds: 30
    readinessProbe:
      initialDelaySeconds: 10
      periodSeconds: 30
    env:
    - name: UNLOAD_STORAGE_MODULES
      value: "true"
    - name: RESTORE_DRIVER_ON_POD_TERMINATION
      value: "true"
    - name: CREATE_IFNAMES_UDEV
      value: "true"
EOF
----

. Create the policy on the cluster, using the following command:
+
[source,terminal]
----
$ oc create -f network-sriovleg-nic-cluster-policy.yaml
----
+
.Example output
+
[source,terminal]
----
nicclusterpolicy.mellanox.com/nic-cluster-policy created
----

. Validate the pods by running the following command in the DOCA/MOFED container:
+
[source,terminal]
----
$ oc get pods -n nvidia-network-operator
----

. Create a `SriovNetworkNodePolicy` that generates the VFs for the device you want to operate in SRIOV legacy mode, as shown in the following custom resource file:
+
[source,yaml]
----
$ cat <<EOF > sriov-network-node-policy.yaml
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetworkNodePolicy
metadata:
  name: sriov-legacy-policy
  namespace:  openshift-sriov-network-operator
spec:
  deviceType: netdevice
  mtu: 1500
  nicSelector:
    vendor: "15b3"
    pfNames: ["ens8f0np0#0-7"]
  nodeSelector:
    feature.node.kubernetes.io/pci-15b3.present: "true"
  numVfs: 8
  priority: 90
  isRdma: true
  resourceName: sriovlegacy
EOF
----

. Create the custom resource on the cluster using the following command.
+
[NOTE]
====
Make sure that SR-IOV Global Enable is enabled as shown in the link:https://access.redhat.com/solutions/37376[Red{nbsp}Hat Knowledgebase]. 
====
+
[source,terminal]
----
$ oc create -f sriov-network-node-policy.yaml
----
+
.Example output
+
[source,terminal]
----
sriovnetworknodepolicy.sriovnetwork.openshift.io/sriov-legacy-policy created
----

. After the VFs for SRIOV legacy mode are in place, you can generate the `SriovNetwork` custom resource file, as shown in the following example:
+
[source,yaml]
----
$ cat <<EOF > sriov-network.yaml
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetwork
metadata:
  name: sriov-network
  namespace:  openshift-sriov-network-operator
spec:
  vlan: 0
  networkNamespace: "default"
  resourceName: "sriovlegacy"
  ipam: |
    {
      "type": "whereabouts",
      "range": "192.168.3.225/28",
      "exclude": [
       "192.168.3.229/30",
       "192.168.3.236/32"
      ]
    }
EOF
----

. Create the custom resource on the cluster, using the following command:
+
[source,terminal]
----
$ oc create -f sriov-network.yaml
----
+
.Example output
+
[source,terminal]
----
sriovnetwork.sriovnetwork.openshift.io/sriov-network created
----
