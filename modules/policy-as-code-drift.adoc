= About policy drift

[role="_abstract"]
Because you can edit, delete, and create policies in the {product-title-short} portal and also externally, sometimes _policy drift_ can occur. Drift occurs when the version of a policy in Central in {product-title-short} does not match the version of the policy in Kubernetes.

Drift can occur when you apply a change to an externally-managed policy by using the {product-title-short} portal or the API instead of by modifying its Kubernetes custom resource. {product-title-short} does not prevent drift, but Red{nbsp}Hat does not recommend it. {product-title-short} automatically resolves drift within ten hours after you introduce it.
