= OpenAPI service endpoint configuration strategies

[role="_abstract"]
The `kogito.sw.operationIdStrategy` property determines how {ServerlessLogicProductName} generates REST client configuration keys for `OpenAPI` services. The property supports four values that control the naming convention for configuration keys.

`FILE_NAME`:: {ServerlessLogicProductName} uses the `OpenAPI` document file name to create the configuration key. The key is based on the file name, where special characters are replaced with underscores.
+
You get an output similar to the following example:
+
[source,sh]
----
quarkus.rest-client.stock_portfolio_svc_yaml.url=http://localhost:8282/
----
+
The OpenAPI file path is `<project_application_dir>/specs/stock-portfolio-svc.yaml`. The generated key that configures the URL for the REST client is `stock_portfolio_svc_yaml`.

`FULL_URI`:: {ServerlessLogicProductName} uses the complete URI path of the OpenAPI document as the configuration key. The full URI is sanitized to form the key.
+
You get an output similar to the following example:
+
[source,json]
----
{
    "id": "myworkflow",
    "functions": [
        {
          "name": "myfunction",
          "operation": "https://my.remote.host/apicatalog/apis/123/document"
        }
    ]
}
----
+
You get an output similar to the following example:
+

[source,sh]
----
quarkus.rest-client.apicatalog_apis_123_document.url=http://localhost:8282/
----
+
The URI path is `https://my.remote.host/apicatalog/apis/123/document`. The generated key that configures the URL for the REST client is `apicatalog_apis_123_document`.

`FUNCTION_NAME`:: {ServerlessLogicProductName} combines the workflow ID and the function name referencing the OpenAPI document to generate the configuration key.
+
You get an output similar to the following example:
+
[source,json]
----
{
    "id": "myworkflow",
    "functions": [
        {
          "name": "myfunction",
          "operation": "https://my.remote.host/apicatalog/apis/123/document"
        }
    ]
}
----
+
You get an output similar to the following example configuration:
+

[source,sh]
----
quarkus.rest-client.myworkflow_myfunction.url=http://localhost:8282/
----
+
The workflow ID is `myworkflow`. The function name is `myfunction`. The generated key that configures the URL for the REST client is `myworkflow_myfunction`.

`SPEC_TITLE`:: {ServerlessLogicProductName} uses the `info.title` value from the OpenAPI document to create the configuration key. The title is sanitized to form the key.
+
The following example displays OpenAPI document:
+
[source,yaml]
----
openapi: 3.0.3
info:
  title: stock-service API
  version: 2.0.0-SNAPSHOT
paths:
  /stock-price/{symbol}:
...
----
+
You get an output similar to the following example command:
+

[source,sh]
----
quarkus.rest-client.stock-service_API.url=http://localhost:8282/
----
+
The OpenAPI document title is `stock-service API`. The generated key that configures the URL for the REST client is `stock-service_API`.
