= About the ClusterLogging custom resource

[role="_abstract"]
To make changes to your {logging} environment, create and modify the `ClusterLogging` custom resource (CR).

.Sample `ClusterLogging` custom resource (CR)
[source,yaml]
----
apiVersion: logging.openshift.io/v1
kind: ClusterLogging
metadata:
  name: instance
  namespace: openshift-logging
spec:
  managementState: Managed
# ...
----

metadata.name::
  The CR name must be `instance`.

metadata.namespace::
  The CR must be installed to the `openshift-logging` namespace.

spec.managementState::
  The {clo} management state. When the state is set to `unmanaged`, the Operator is in an unsupported state and does not receive updates.
