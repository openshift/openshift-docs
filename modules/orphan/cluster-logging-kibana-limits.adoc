= Configure the CPU and memory limits for the log visualizer

[role="_abstract"]
You can adjust both the CPU and memory limits for the pod that hosts the log visualizer. 

.Procedure

. Edit the `ClusterLogging` custom resource (CR) in the `openshift-logging` project: 
+
[source,terminal]
----
$ oc edit ClusterLogging instance
----
+
[source,yaml]
----
apiVersion: "logging.openshift.io/v1"
kind: "ClusterLogging"
metadata:
  name: "instance"

....

spec:
    visualization:
      type: "kibana"
      kibana:
        replicas:
      resources:
        limits:
          memory: 1Gi
        requests:
          cpu: 500m
          memory: 1Gi
      proxy:
        resources:
          limits:
            memory: 100Mi
          requests:
            cpu: 100m
            memory: 100Mi
----

spec.visualization.kibana.resources::
  Specify the CPU and memory limits to allocate for each node.

spec.visualization.kibana.proxy::
  Specify the CPU and memory limits to allocate to the Kibana proxy.
