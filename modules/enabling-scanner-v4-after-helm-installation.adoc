= Enabling RHACS Scanner V4 after Helm installation

[role="_abstract"]
Scanner V4 is not enabled by default, but you can enable Scanner V4 after installation. Scanner V4 also requires that the StackRox Scanner remain enabled. Do not disable the StackRox Scanner even if Scanner V4 is enabled.

If you installed {product-title-short} by using Helm charts, you must first enable Scanner V4 on the cluster where Central is installed.

.Procedure

. Create a `values-public.yaml` file that contains the following lines:
+
[source,yaml]
----
scannerV4:
    disable: false
----
. Run the command to update {product-title-short} with customizations:
+
[source,terminal]
----
helm upgrade -n stackrox --create-namespace \
  stackrox-central-services rhacs/central-services \
  -f <path_to_values_public.yaml> -f <path_to_values_private.yaml>
----

For more information, see "Changing configuration options after deploying the central-services Helm chart".

After you enable Scanner V4 on the cluster where Central is installed and configure your init bundle or cluster registration secret (CRS) to allow communication between Central and the secured cluster, you must enable Scanner V4 on the secured cluster.

.Procedure
. Create a `values-public.yaml` file that contains the following lines:
+
[source,yaml]
----
scannerV4:
    scannerComponent: AutoSense
----
. Run the command to update {product-title-short} with customizations:
+
[source,terminal]
----
helm upgrade -n stackrox --create-namespace \
  stackrox-secured-cluster-services rhacs/secured-cluster-services \

helm install -n stackrox --create-namespace \
    stackrox-secured-cluster-services rhacs/secured-cluster-services \
    --set-file crs.file=<crs_file_name.yaml> \
    -f <path_to_pull_secret.yaml> \
        -f <path_to_values_public.yaml> -f <path_to_values_private.yaml> \
    --set clusterName=<name_of_the_secured_cluster> \
    --set centralEndpoint=<endpoint_of_central_service> \
    --set scanner.disable=false
----
+
For more information, see "Changing configuration options after deploying the secured-cluster-services Helm chart".
