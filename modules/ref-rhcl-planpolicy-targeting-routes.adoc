= Targeting an HTTPRoute or GRPCRoute

[role="_abstract"]
. When a `PlanPolicy` object targets an HTTPRoute or GRPCRoute, you can use the following example to enforce the plan-based rate limits on all traffic flowing through that specific route:
+
.Example enforcement of plan-based rate limits on route traffic 

[source,yaml]
----
apiVersion: extensions.kuadrant.io/v1alpha1
kind: PlanPolicy
metadata:
  name: _<my-route-plan>_
  namespace: _<my-namespace>_
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: _<my-route>_
  plans:
    - tier: gold
      predicate: |
        has(auth.identity) && auth.identity.metadata.annotations["secret.kuadrant.io/plan-id"] == "gold"
      limits:
        daily: 500
        weekly: 2000
    - tier: silver
      predicate: |
        has(auth.identity) && auth.identity.metadata.annotations["secret.kuadrant.io/plan-id"] == "silver"
      limits:
        daily: 500
        weekly: 2000
    - tier: bronze
      predicate: |
        has(auth.identity) && auth.identity.metadata.annotations["secret.kuadrant.io/plan-id"] == "bronze"
      limits:
        daily: 100
        weekly: 500
----
* Replace _<my-route-plan>_ with the name of your route plan.
* Replace _<my-namespace>_ with your namespace.
* Replace _<my-route>_ with the name of the HTTProute or GRPCRoute.

[NOTE]
====
The CEL expressions can be anything that applies to your use case. You can use an expression different from the `secret.kuadrant.io/plan-id` expression.
====
