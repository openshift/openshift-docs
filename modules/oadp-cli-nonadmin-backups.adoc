= Managing non-admin backups

You can use the {oadp-short} CLI to create, view, describe, and delete non-admin backups in your namespace.

.Prerequisites

* The {oadp-short} CLI plugin is installed and configured in non-admin mode.
* You are logged in to the {product-title} cluster and your current namespace context is set to the namespace you want to back up.
* You have editor roles for `nonadminbackups.oadp.openshift.io` in your namespace.
* A `NonAdminBackupStorageLocation` exists in your namespace, or a default has been configured by running `oc oadp client config set default-nabsl=<name>`.

.Creating a non-admin backup

* Create a backup of resources in your current namespace:
+
[source,terminal]
----
$ oc oadp nonadmin backup create <backup_name> [flags]
----
+
.Flags for `oc oadp nonadmin backup create`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--storage-location`
|Name of the `NonAdminBackupStorageLocation` to use. Required unless a default is configured.

|`--include-resources`
|Resources to include in the backup. Accepts simple kind names (for example, `deployments,services`) or `resource.group` format (for example, `deployments.apps`) for disambiguation. Default: `*` (all resources).

|`--exclude-resources`
|Resources to exclude from the backup. Same format as `--include-resources`.

|`--selector`, `-l`
|Only back up resources matching this label selector.

|`--or-selector`
|Back up resources matching at least one of the label selectors, separated by ` or `.

|`--ttl`
|How long before the backup can be garbage collected. Default: `720h`.

|`--csi-snapshot-timeout`
|Timeout for CSI snapshot creation.

|`--item-operation-timeout`
|Timeout for asynchronous plugin operations.

|`--snapshot-volumes`
|Take snapshots of PersistentVolumes as part of the backup.

|`--snapshot-move-data`
|Move snapshot data to the backup storage location.

|`--default-volumes-to-fs-backup`
|Use pod volume file system backup by default for volumes.
|===
+
.Example
[source,terminal]
----
$ oc oadp nonadmin backup create my-backup \
    --storage-location my-nabsl \
    --include-resources deployments,services \
    --selector app=myapp \
    --snapshot-volumes \
    --ttl 720h
----

[TIP]
====
To avoid specifying the storage location on each backup, run the following command to set a default:

[source,terminal]
----
$ oc oadp client config set default-nabsl=<nabsl_name>
----
====

.Listing non-admin backups

* List all backups in your current namespace:
+
[source,terminal]
----
$ oc oadp nonadmin backup get [<backup_name>] [flags]
----
+
.Flags for `oc oadp nonadmin backup get`
[cols="1,3",options="header"]
|===
|Flag |Description

|`-o`
|Output format. Supported values: `json`, `yaml`.
|===

.Describing a non-admin backup

* Describe a backup to view its details:
+
[source,terminal]
----
$ oc oadp nonadmin backup describe <backup_name> [flags]
----
+
.Flags for `oc oadp nonadmin backup describe`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--details`
|Display additional backup details, including volume snapshots, resource lists, and item operations.

|`--request-timeout`
|Timeout for fetching backup details from the server.
|===

.Viewing non-admin backup logs

* View the logs for a backup:
+
[source,terminal]
----
$ oc oadp nonadmin backup logs <backup_name> [flags]
----
+
.Flags for `oc oadp nonadmin backup logs`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--request-timeout`
|Timeout for fetching logs from the server.
|===

[NOTE]
====
Backup logs are only available when using a `NonAdminBackupStorageLocation`. Logs are not available for backups that use the default cluster BSL.
====

.Deleting non-admin backups

* Delete one or more backups:
+
[source,terminal]
----
$ oc oadp nonadmin backup delete [<backup_name>...] [flags]
----
+
.Flags for `oc oadp nonadmin backup delete`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--confirm`
|Skip the confirmation prompt and delete immediately.

|`--all`
|Delete all backups in the current namespace.
|===
+
Backup deletion is performed asynchronously by the {oadp-short} non-admin controller.
+
.Example
[source,terminal]
----
$ oc oadp nonadmin backup delete my-backup --confirm
----
