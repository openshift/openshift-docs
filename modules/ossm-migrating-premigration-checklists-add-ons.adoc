= Checklist for configuring add on replacements in Service Mesh 2

If you did not disable and configure add on replacements when you upgraded from {SMProduct} 2.5 to {SMProduct} 2.6, you must disable them in your `ServiceMeshControlPlane` resource.

.Prerequisite

* {SMProduct} Operator 3 is installed.

.Procedure

* [ ] Disable Prometheus: `spec.addons.prometheus.enabled=false`
** [ ] Configure the `ServiceMeshControlPlane` with OpenShift Monitoring as the replacement. These instructions also include installing a standalone `Kiali` resource. Both can be done at the same time. link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.17/html/service_mesh/service-mesh-2-x#ossm-integrating-with-user-workload-monitoring_observability[Integration user-workload monitoring,window=_blank]
** [ ] If you are not using OpenShift monitoring, see: link:https://docs.redhat.com/en/documentation/red_hat_openshift_service_on_aws/4/html/service_mesh/service-mesh-2-x#integration-with-external-prometheus-installation[Integration with external Prometheus installation].

* [ ] Disable Tracing: `spec.tracing.type=None`
** [ ] Follow these instructions to configure the `ServiceMeshControlPlane` with OpenShift Distributed Tracing as the replacement: link:https://docs.redhat.com/en/documentation/openshift_container_platform/4.17/html/service_mesh/service-mesh-2-x#ossm-configuring-distr-tracing-tempo_observability[Configuring Distributed Tracing Tempo].

* [ ] Disable Kiali: `spec.addons.kiali.enabled=false`
** [ ] If you didn't create a standalone `Kiali` resource as part of Prometheus or tracing, see: "Using Kiali Operator provided by Red Hat".
