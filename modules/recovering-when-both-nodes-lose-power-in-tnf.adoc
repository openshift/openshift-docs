= Recovering when both nodes lose power in TNF

If both nodes lose power and only one can restart, for example due to a power outage or hardware failure, you must manually unblock the pacemaker quorum before continuing with the standard node replacement procedure.

.Procedure

. On the surviving node after it boots, unblock the pacemaker quorum to allow pacemaker to start with fewer nodes than expected:
+
[source,terminal]
----
$ sudo pcs quorum unblock
----

. Confirm that the failed node has been fenced. This is required because actual fencing may fail if the node was already powered off before fencing was triggered:
+
[source,terminal]
----
$ sudo pcs stonith confirm --force
----

. Wait for etcd to start. The CEO detects that quorum is restored and continues the recovery process automatically.

. Resume the node replacement procedure starting from xref:tnf-phase3-remove-failed-node_{context}[Phase 3: Remove failed node references].
