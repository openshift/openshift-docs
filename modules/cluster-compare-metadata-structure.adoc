= Structure of the metadata.yaml file

[role="_abstract"]
To centrally define and configure the templates in a reference configuration, use the `metadata.yaml` file. By configuring this central file, you can securely establish template dependencies, set validation rules, and apply descriptive metadata.

The file features a hierarchy of `parts` and `components`, where `parts` are groups of `components` and `components` are groups of templates.

.Example metadata.yaml file
[source,yaml]
----
apiVersion: v2
parts:
  - name: Part1
    components:
      - name: Component1
        <component1_configuration>
  - name: Part2
      - name: Component2
        <component2_configuration>
# ...
----

where:

`parts`:: Specifies a workload or a set of workloads for a part.

`parts.name`:: Specifies a `part` name.

`parts.name.components.name`::  Specifies a `component` name.

`components.name.<component1_configuration>`:: Specifies the configuration for a template. For example, define template relationships or configure what fields to use in a comparison.
