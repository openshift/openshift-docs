= Custom memory bump-up after OOMKill

If your cluster experiences an OOM (out of memory) event, which indicates the pod ran out of available memory, the  Vertical Pod Autoscaler Operator (VPA) increases the memory recommendation based on the memory consumption observed during the OOM event and a specified multiplier value.

[source,text]
----
`recommendation = max(memory-usage-in-oomkill-event + oom-min-bump-up-bytes, memory-usage-in-oomkill-event * oom-bump-up-ratio)`.
----

You can configure the memory increase by specifying the following values for the recommender:

* `oom-min-bump-up-bytes`. This value, in bytes, specifies the minimum increase in memory after an OOM (out of memory) event occurs. The default is `100MiB`.
* `oom-bump-up-ratio`. This value specifies the memory increase ratio when the OOM kill occurred. The default value is `1.2`. The default value means that memory will be increased by 20% after an OOM kill event.

.Example recommender deployment object

[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: vpa-recommender-default
  namespace: openshift-vertical-pod-autoscaler
# ...
spec:
# ...
  template:
# ...
    spec
      containers:
      - name: recommender
        args:
        - --oom-bump-up-ratio=2.0
        - --oom-min-bump-up-bytes=524288000
# ...
----
