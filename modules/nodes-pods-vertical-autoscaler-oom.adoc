= Custom memory bump-up after OOMKill

As a cluster administrator, you can 

After an OOM (out of memory) event is observed, which indicates the pod ran out of available memory, the  Vertical Pod Autoscaler Operator (VPA) increases the memory recommendation based on the previous memory usage, the OOM event's memory consumption, and the oom-bump-up-ratio.


on the the OOM event's memory consumption, the observed memory usage in the event according to the following formula: 

[source,text]
----
`recommendation = max(memory-usage-in-oomkill-event + oom-min-bump-up-bytes, memory-usage-in-oomkill-event * oom-bump-up-ratio)`.
----

You can configure the minimum memory increase and ratio by specifying the following startup arguments for the recommender:

* `oom-min-bump-up-bytes`. This value, in bytes, specifies the minimum increase in memory after an OOM (out of memory) event occurs. The default is `100MiB`.
* `oom-bump-up-ratio`. This value specifies the memory increase ratio when the OOM kill occurred. The default value is `1.2`. The default value means that memory will be increased by 20% after an OOM kill event.

Usage in recommender deployment

[source,yaml]
----
apiVersion: apps/v1
kind: Deployment
metadata:
  name: vpa-recommender-default
  namespace: openshift-vertical-pod-autoscaler
# ...
spec:
# ...
  template:
# ...
    spec
      containers:
      - name: recommender
        args:
        - --oom-bump-up-ratio=2.0 <1>
        - --oom-min-bump-up-bytes=524288000 <2>
# ...
----
