= Build volumes and storage

[role="_abstract"]
You can configure persistent and ephemeral volumes that provide build-time access to configuration files, credentials, and shared data.
Volume definitions and mount configurations enable builds to access external storage resources during execution.

Volume definitions:: Declare volumes sourced from ConfigMaps, Secrets, `PersistentVolumeClaims`, or ephemeral storage that builds can mount during execution.
Volume mounts:: Configure mount paths, read-only access, and `subPath` mappings that control how volumes appear within the build container filesystem.
Storage configuration:: Define ephemeral volume size limits, cache volume persistence, and temporary workspace storage parameters.

Configure volumes to inject build-time configuration, securely provide credentials, share artifacts between builds, and optimize build performance through persistent caching.
