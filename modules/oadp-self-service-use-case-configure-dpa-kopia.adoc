= Configuring the `DataProtectionApplication` for a non-admin filesystem backup

As a cluster administrator, configure the `DataProtectionApplication` (DPA) as shown in the following example, to enable filesystem backup for a non-admin user.

.Prerequisites

* You are logged in to the cluster with administrator privileges.
* You have installed the {oadp-short} Operator.

.Procedure

.Procedure

. To enable {oadp-short} Self-Service, edit the DPA and set the `nonAdmin.enable` field to `true` as shown in the following example. Ensure that you have enabled the `nodeAgent`.
+
.Example `DataProtectionApplication`
[source,yaml]
----
apiVersion: oadp.openshift.io/v1alpha1
kind: DataProtectionApplication
metadata:
  name: test-dpa
  namespace: openshift-adp
spec:
  backupLocations:
  - velero:
      credential:
        key: cloud
        name: cloud-credentials # <1>
      default: true
      objectStorage:
        bucket: <bucket_name> # <2>
        prefix: velero
      provider: gcp
  configuration:
    nodeAgent:
      enable: true
      uploaderType: kopia # <3>
    velero:
      defaultPlugins:
      - gcp
      - openshift
  nonAdmin: # <4>
    enable: true # <5>
----
1. Specify the `cloud-credentials` file for your cloud provider. In this example, the cloud provider is `gcp`.
2. Specify the bucket name.
3. Ensure that the `nodeAgent` is enabled and `uploaderType` is set to `kopia`.
4. Add `nonAdmin.enable` section under the `spec` section of the DPA as shown in the example.
5. Set the `enable` field to `true`.
