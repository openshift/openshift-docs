= API management custom resource definitions

[role="_abstract"]
You can use custom resource definitions (CRDs) to request, grant or remove access to APIs with the {prodname} {ocp} web console plugin. The following list details available CRDs.

`APIProduct`:: The `APIProduct` CRD wraps an existing `HTTPRoute` custom resource (CR) with the business context needed for consumption, including a human-readable name, documentation links, contact information, and access policies. When an API owner creates an `APIProduct` resource and sets the `publishStatus` field to `Published`, the API becomes discoverable in the catalog.

`APIKey`:: The `APIKey` CRD represents the actual API access credentials in the API consumer's namespace. It is created by the controller when an access request is approved.

`APIKeyRequest`:: An `APIKeyRequest` CRD is a shadow resource in the API owner namespace that enables discovery without exposing API key values. These resources are managed by the {prodname} controller and must not be created or modified manually.

`APIKeyApproval`:: An `APIKeyApproval` CRD represents an approval or rejection action on an `APIKeyRequest` resource. API owners and administrators create `APIKeyApproval` resources to approve or reject pending requests.
