= Build resource fields

[role="_abstract"]
You can look up field properties, accepted values, and usage patterns when creating or modifying Build resources.
Build resources define how source code is transformed into container images by using configurable source, strategy, output, and parameter fields.

Source fields:: Configure where your application source code originates, including Git repositories, local paths, and credentials.
Strategy fields:: Select and configure the build strategy with strategy-specific parameters.
Output fields:: Specify where the resulting container image is pushed, including registry credentials and image annotations.
Parameter fields:: Define environment variables, resource limits, and runtime configuration that affects build execution.
Timeout fields:: Control build execution time limits and retry behavior.
