= BuildRun resource fields

[role="_abstract"]
You can configure individual build runs with custom settings while preserving the base Build definition.
BuildRun fields control runtime behavior, override Build defaults, and specify execution-specific parameters.

Build reference fields:: Link the `BuildRun` to a specific `Build` resource and control which Build version is executed.
Runtime override fields:: Override Build-level settings such as timeout values, service account, and output destinations for this specific execution.
Parameter override fields:: Override environment variables and build arguments defined in the Build resource.
Retention fields:: Configure how long completed `BuildRun` resources and their associated Pods are retained in the cluster.
State fields:: Control `BuildRun` lifecycle states including cancellation and deletion policies.

Each field includes its data type, override precedence rules, and examples showing how runtime configuration differs from Build-level defaults.
