= Collecting diagnostic data

You can use the {oadp-short} CLI to collect diagnostic information for {oadp-short} installations. The `must-gather` command runs the {oadp-short} must-gather tool to collect logs and cluster state information needed for troubleshooting and support cases.

.Prerequisites

* The {oadp-short} CLI plugin is installed and configured in admin mode.
* You are logged in to the {product-title} cluster as a user with `cluster-admin` privileges.
* The `oc` CLI is installed and available on your `PATH`.

.Procedure

* Collect {oadp-short} diagnostic information:
+
[source,terminal]
----
$ oc oadp must-gather [flags]
----
+
.Flags for `oc oadp must-gather`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--dest-dir`
|Directory where must-gather output is stored. Default: `./must-gather`.

|`--request-timeout`
|Timeout for the gather script. For example, `30s` or `1m`.

|`--skip-tls`
|Skip TLS verification.
|===
+
.Example
[source,terminal]
----
$ oc oadp must-gather --dest-dir=/tmp/oadp-diagnostics --request-timeout=1m
----
