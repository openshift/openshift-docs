= OLSConfig field reference for AWS Bedrock

[role="_abstract"]
The following reference tables describe the configuration schema for AWS Bedrock providers.

.Provider fields (`spec.llm.providers[]`)
[cols="2,1,1,4", options="header"]
|===
| Field | Type | Required | Description

| `name`
| `string`
| Yes
| Logical name for the provider. Referenced by `spec.ols.defaultProvider`.

| `type`
| `string`
| Yes
| Must be set to `bedrock`.

| `credentialsSecretRef.name`
| `string`
| Yes
| Name of the Secret in the operator namespace that contains provider credentials.

| `url`
| `string`
| Yes
| The provider API endpoint URL. Specifies your active Bedrock connection path or Bedrock Mantle gateway regional endpoint.

| `models`
| `array`
| Yes
| List of models available from the provider.
|===

.Model fields (`spec.llm.providers[].models[]`)
[cols="2,1,1,4", options="header"]
|===
| Field | Type | Required | Description

| `name`
| `string`
| Yes
| Explicit model ID published by your Mantle infrastructure or regional AWS console (such as `anthropic.claude-opus-4-7`). Referenced by `spec.ols.defaultModel`. Do not use generic names.

| `url`
| `string`
| No
| The model-specific API endpoint URL.

| `contextWindowSize`
| `integer`
| No
| Context window size in tokens. Minimum value: 1024.

| `parameters.maxTokensForResponse`
| `integer`
| No
| Maximum tokens allowed for responses. Default value: 2048.

| `parameters.toolBudgetRatio`
| `float`
| No
| Ratio of the context window allocated for the tool token budget. Range: 0.1 to 0.5. Default value: 0.5.
|===


.Examples of supported model IDs:
* *Anthropic:* `anthropic.claude-opus-4-7`
* *Meta:* `meta.llama3-1-70b-instruct-v1:0`
* *Amazon:* `amazon.nova-pro-v1:0`
* *DeepSeek:* `deepseek.v3.1`
* *Mistral:* `mistral.mistral-large-2407-v1:0`
+
[NOTE] 
====
Certain models demand cross-region execution paths. In these environments, you must declare the cross-region inference profile format, such as `us.meta.llama3-1-70b-instruct-v1:0`.
====
