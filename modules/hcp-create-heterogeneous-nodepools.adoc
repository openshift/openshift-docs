= Creating heterogeneous node pools

You can create heterogeneous node pools by using the `NodePool` custom resource (CR).

.Procedure

* To create heterogeneous node pools, create a YAML file that defines the `NodePool` CR:
+
[source,yaml]
----
apiVersion: hypershift.openshift.io/v1beta1
kind: NodePool
metadata:
  name: <hosted_cluster_name>
  namespace: <clusters_namespace>
spec:
  arch: <arch_ppc64le>
  clusterName: <hosted_cluster_name>
  management:
    autoRepair: false
    upgradeType: InPlace
  nodeDrainTimeout: 0s
  nodeVolumeDetachTimeout: 0s
  platform:
    agent:
      agentLabelSelector:
        matchLabels:
          inventory.agent-install.openshift.io/cpu-architecture: <arch_ppc64le>
    type: Agent
  release:
    image: quay.io/openshift-release-dev/ocp-release:<ocp_release>
  replicas: 0
----
+
. To create a node pool of architecture `ppc64le` with zero replicas, see the following configuration:
+
[source,yaml]
----
# ...
agentLabelSelector:
    matchLabels:
        inventory.agent-install.openshift.io/cpu-architecture: ppc64le
# ...
----
+
This selector block is used to select the agents which are matching the given label. Which will ensure it will select only the agents from `ppc64le` architecture when it is scaled.
