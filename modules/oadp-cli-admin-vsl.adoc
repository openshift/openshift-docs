= Managing volume snapshot locations

You can use the {oadp-short} CLI to create, view, set, and delete volume snapshot locations (VSLs). Volume snapshot locations define where PersistentVolume snapshots are stored.

.Prerequisites

* The {oadp-short} CLI plugin is installed and configured in admin mode.
* You are logged in to the {product-title} cluster as a user with `cluster-admin` privileges.

.Creating a volume snapshot location

* Create a volume snapshot location:
+
[source,terminal]
----
$ oc oadp snapshot-location create <vsl_name> [flags]
----
+
.Flags for `oc oadp snapshot-location create`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--provider`
|Name of the cloud provider. For example, `aws`, `gcp`, or `azure`.

|`--config`
|Provider-specific configuration as `key=value` pairs.

|`--request-timeout`
|Timeout for the request to the Kubernetes API server.
|===
+
.Example
[source,terminal]
----
$ oc oadp snapshot-location create my-vsl \
    --provider aws \
    --config region=us-east-1
----

.Listing volume snapshot locations

* List all volume snapshot locations:
+
[source,terminal]
----
$ oc oadp snapshot-location get [<vsl_name>] [flags]
----
+
.Flags for `oc oadp snapshot-location get`
[cols="1,3",options="header"]
|===
|Flag |Description

|`-o`
|Output format. Supported values: `json`, `yaml`.
|===

.Setting a default volume snapshot location

* Set the default volume snapshot location:
+
[source,terminal]
----
$ oc oadp snapshot-location set <vsl_name>
----

.Deleting a volume snapshot location

* Delete a volume snapshot location:
+
[source,terminal]
----
$ oc oadp snapshot-location delete <vsl_name> [flags]
----
+
.Flags for `oc oadp snapshot-location delete`
[cols="1,3",options="header"]
|===
|Flag |Description

|`--confirm`
|Confirm deletion without prompting.
|===
