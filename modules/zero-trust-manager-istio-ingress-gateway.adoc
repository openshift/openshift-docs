= Deploying Istio Ingress Gateway with SPIFFE

[role="_abstract"]
Deploy an Istio Ingress Gateway with SPIFFE integration and configure the routing to backend services. This enables external traffic to enter the service mesh while maintaining SPIRE-based identity authentication to backend workloads.

.Prerequisites

* Istio deployed with SPIRE integration.

* Test workloads deployed with mTLS configured.

* Helm CLI installed.

* `istioctl` CLI installed.

.Procedure

. Grant the `anyuid` security context constraint to the Istio Gateway service account by running the following command:
+
[source,terminal]
----
$ oc adm policy add-scc-to-user anyuid system:serviceaccount:istio-system:istio-gateway
----

. Add the Istio Helm repository by running the following commands:
+
[source,terminal]
----
$ helm repo add istio https://istio-release.storage.googleapis.com/charts
----
+
[source,terminal]
----
$ helm repo update
----

. Install the Istio Gateway using Helm by running the following command:
+
[source,terminal]
----
$ helm install istio-gateway -n "${OSSM_NS}" istio/gateway --set-json 'podAnnotations={"inject.istio.io/templates":"gateway,spireGateway"}'
----
+
[NOTE]
====
The `inject.istio.io/templates` annotation includes both `gateway` and `spireGateway` templates to ensure the SPIRE Agent socket is mounted to the gateway pod.
====

. Wait for the Istio Gateway deployment to become available by running the following commands:
+
[source,terminal]
----
$ until oc get deployment istio-gateway -n "${OSSM_NS}" &> /dev/null; do sleep 3; done
----
+
[source,terminal]
----
$ oc wait --for=condition=Available deployment/istio-gateway -n "${OSSM_NS}" --timeout=300s
----


. Create an Istio `Gateway` custom resource for the httpbin service by running the following command:
+
[source,terminal]
----
$ cat <<EOF | oc apply -f -
apiVersion: networking.istio.io/v1
kind: Gateway
metadata:
  name: httpbin-gateway
  namespace: ${TPJ}
spec:
  selector:
    istio: gateway
  servers:
    - port:
        number: 80
        name: http
        protocol: HTTP
      hosts:
        - "*"
EOF
----

. Create an Istio `VirtualService` to route traffic to the httpbin service by running the following command:
+
[source,terminal]
----
$ cat <<EOF | oc apply -f -
apiVersion: networking.istio.io/v1
kind: VirtualService
metadata:
  name: httpbin
  namespace: ${TPJ}
spec:
  hosts:
    - "*"
  gateways:
    - httpbin-gateway
  http:
    - route:
      - destination:
          host: httpbin.${TPJ}.svc.cluster.local
          port:
            number: 80
EOF
----

. Get the Istio Gateway external IP address by running the following command:
+
[source,terminal]
----
$ export GATEWAY_IP="$(oc get svc istio-gateway -n istio-system  -o jsonpath={.status.loadBalancer.ingress[].ip})"
----

. Test access to the httpbin service through the Ingress Gateway by running the following command:
+
[source,terminal]
----
$ curl -H "Host: httpbin" "${GATEWAY_IP}" -s -o /dev/null -w "%{http_code}"
----
+
.Example output
[source,terminal]
----
200
----
+
An HTTP 200 response confirms that traffic is being securely routed from the Istio Gateway to the httpbin service using SPIFFE mTLS.
