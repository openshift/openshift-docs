= Restoring a hosted cluster by using {oadp-short}

Before you start restoring a hosted cluster, ensure the following:

* No pods and persistent volume claims (PVCs) are present in the hosted control plane namespace.
* If you are restoring a hosted cluster within the same management cluster, you deleted objects of the following components:

** Hosted cluster
** Node pools
** PVCs

* If you're using an _in-place_ update, InfraEnv does not need spare nodes. You need to re-provision the worker nodes from the new management cluster.
* If you're using a _replace_ update, you need some spare nodes for InfraEnv to deploy the worker nodes.

[IMPORTANT]
====
After you back up your hosted cluster, you must destroy it to initiate the restoring process. To initiate node provisioning, you must back up workloads in the data Plane before deleting the hosted cluster.
====

You can restore the hosted cluster by creating the `Restore` custom resource (CR).

. Create a YAML file that defines the `Restore` CR:
+
.Example `restore-hosted-cluster.yaml` file
[source,yaml]
----
apiVersion: velero.io/v1
kind: Restore
metadata:
  name: hc-clusters-hosted-restore
  namespace: openshift-adp
spec:
  backupName: hc-clusters-hosted-backup
  restorePVs: true <1>
  existingResourcePolicy: update <2>
  excludedResources:
  - nodes
  - events
  - events.events.k8s.io
  - backups.velero.io
  - restores.velero.io
  - resticrepositories.velero.io
----

<1> Initiates the recovery of PVs and its pods.
<2> Ensures that the existing objects are overwritten with the backed up content.

[IMPORTANT]
====
You must create the `infraenv` resource in a separate namespace. Do not delete the `infraenv` resource during the restore process.
====

. Apply the `Restore` CR by running the following command:
+
[source,terminal]
----
$ oc apply -f `restore-hosted-cluster.yaml`
----
