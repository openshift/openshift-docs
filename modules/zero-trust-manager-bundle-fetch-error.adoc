= Bundle fetch failures

[role="_abstract"]
Troubleshoot trust bundle fetch failures by verifying federation endpoint connectivity, validating DNS and TLS route configurations, and ensuring correct RBAC permissions for secret access in OpenShift.

The following indicators confirm a breakdown in the trust bundle retrieval process and endpoint accessibility:

* Script fails at the "Fetching trust bundles" step.

* The `curl` command fails.

.Procedure

. Manually test the federation endpoint to bypass certificate validation for initial connectivity testing by running the following command:
+
[source,terminal]
----
$ curl -kv https://federation.apps.example.com
----

. Verify that the federation domain resolves to the correct IP address by running the following command:
+
[source,terminal]
----
$ nslookup federation.apps.example.com
----

. Inspect the OpenShift route to ensure TLS termination is correctly configured by running the following command:
+
[source,terminal]
----
$ oc get route spire-server-federation -n zero-trust-workload-identity-manager -o yaml | grep -A10 tls
----

. For deployments using specialized secret management (Configuration 2), verify that the required Role-Based Access Control (RBAC) permissions exist by running the following command:

+
[source,terminal]
----
$ oc get role,rolebinding -n zero-trust-workload-identity-manager | grep secret-reader
----
