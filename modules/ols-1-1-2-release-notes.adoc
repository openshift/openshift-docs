= {ols-long} version 1.1.2

[role="_abstract"]
{ols-official} 1.1.2 addresses security vulnerabilities, fixes software bugs, and improves stability. This release helps ensure continued support and stability for clusters running {ocp-product-title} 4.16 and later.

[id="ols-1-1-2-enhancements_{context}"]
== Enhancements

[role="_abstract"]
{ols-official} 1.1.2 introduces the support for {aws-first} Bedrock model provider, replaces the custom RAG implementation with the Offline Knowledge Portal (OKP) for {ocp-product-title} documentation retrieval, adds support for {ocp-product-title} 4.22, and adds ROSA-aware documentation retrieval.

{ols-official} 1.1.2 includes the following enhancements:

//OLS-2605

* With this update, cluster administrators can configure {aws-first} Bedrock as a large language model (LLM) provider in the `OLSConfig` custom resource (CR).

//OLS-3310

* With this update, {ols-long} replaces the previous custom OLS RAG implementation with OKP (Offline Knowledge Portal) for serving {ocp-product-title} product documentation. The operator deploys an RHOKP sidecar on the app-server pod and configures Solr hybrid retrieval for the cluster's {ocp-product-title} version.
+
Cluster administrators can tune RHOKP sidecar CPU, memory, and ephemeral storage (disk) by using `spec.ols.deployment.rhokp` on the `OLSConfig` custom resource (CR). RHOKP requires substantial disk space for Solr data; the operator's recommended default request is 75 GiB ephemeral storage, along with 2 CPU and 2 GiB memory.

//OLS-2456

* With this update, {ols-long} adds support for {ocp-product-title} 4.22.

//OLS-1894

* With this update, {ols-long} automatically detects whether the cluster is a {product-rosa} (ROSA) cluster. On ROSA, the operator determines the ROSA deployment type based on control plane topology and configures the appropriate ROSA documentation product for OKP-based RAG retrieval. ROSA-specific documentation is included alongside {ocp-product-title} documentation, so answers on ROSA clusters can reference guidance tailored to the managed-service environment. On non-ROSA clusters, behavior is unchanged.
