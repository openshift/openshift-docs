= Define Build Inputs and Source Code

[role="_abstract"]
Configure your build by defining input sources, execution parameters, and runtime settings. A `Build` custom resource (CR) specifies how the build controller transforms source code into container images using a selected build strategy.

The build input configuration includes the following key elements:

* Build source configuration: Git repository URL, branch, or OCI artifact containing your application code
* Build strategy selection: The method used to build your image, such as Buildah, Source-to-Image, or Buildpacks
* Builder or Dockerfile definition: Container file path and build context configuration
* Build specification definition: Core build resource specification and parameters
* Build pod configuration: Resource limits, node selection, and scheduling constraints
* Service account definition: Identity and permissions for accessing protected resources
* Environment variables definition: Configuration passed to the build container
* Resource management in Tekton pipelines: Tekton-specific resource configuration

These configurations work together to define a reproducible build process. The build controller validates your configuration, creates the necessary Kubernetes resources, and executes the build using Tekton pipelines.
