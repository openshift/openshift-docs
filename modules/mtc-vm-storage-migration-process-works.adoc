= How the virtual machine storage migration process works

During the migration, a `MigMigration` resource is created indicating what type of migration is happening:

* stage
* rollback
* cutover

* The status of the `MigMigration` contains progress information about any live storage migrations.

Any offline migrations will have a `DirectVolumeMigrationProgress` status that shows the progress of the offline migration.

Each `MigMigration` creates a `DirectVolumeMigration` if the migration plan is a direct volume migration plan. To perform a storage live migration, a direct volume migration is required. The `DirectVolumeMigration` resource status indicates the progress and status of any ongoing migrations, either live migrations or offline migrations.
