= Configuring the Cloud Credential Operator utility

To create and manage cloud credentials from outside of the cluster when the Cloud Credential Operator (CCO) is operating in manual mode, use the CCO utility (`ccoctl`).
//For all cases of manual mode, or just when also using STS?

.Global flags

By default, `ccoctl` outputs to the directory in which the commands are run. To specify a directory, use the `--target-dir` flag.

Some `ccoctl` commands make AWS API calls to create or modify AWS resources. To place JSON files on the local file system instead, use the `--generate-files-only` flag. These JSON files can be reviewed or modified and then applied with the AWS CLI tool using the `--cli-input-json` parameters.

.Prerequisites

. **have the release image ready**

. Generate the public and private RSA key files are used to set up the OpenID Connect provider for the cluster:
+
[source,terminal]
----
$ ccoctl create key-pair
----
+
.Example output:
+
[source,terminal,subs="+quotes"]
----
2021/04/13 11:01:02 Generating RSA keypair
2021/04/13 11:01:03 Writing private key to /__<path_to_dir>__/ocp-install/ocp48/serviceaccount-signer.private
2021/04/13 11:01:03 Writing public key to /__<path_to_dir>__/ocp-install/ocp48/serviceaccount-signer.public
2021/04/13 11:01:03 Copying signing key for use by installer
----
+
Where `serviceaccount-signer.private` and `serviceaccount-signer.public` are the generated key files.

. Create an OpenID Connect identity provider in the cloud:
//"in the cloud"?
+
[source,terminal,subs="+quotes"]
----
$ ccoctl create identity-provider --name-prefix=__<name_prefix>__ --region=__<aws_region>__ --public-key=__<path_to_public_key_file>__
----
+
Where:

** `_<name-prefix>_` is the name used to name and tag any account cloud resources that are created.
//What are "account cloud resources"? Where are they tagged - in OCP, AWS, or both?
** `_<aws-region>_` is the AWS region in which cloud resources will be created.
** `_<public-key>_` is the path to a public key file generated using the `ccoctl create key-pair` command.
+
//Better to just have output here?
The command creates a discovery document file named `02-openid-configuration` and a JSON web key set file named `03-keys.json`.

. Create IAM roles for each component in the cluster.

.. Extract the list of `CredentialsRequest` objects from the {product-title} release image:
+
[source,terminal,subs="+quotes"]
----
$ oc adm release extract --credentials-requests --cloud=aws --to=./credrequests quay.io/__<path_to>__/openshift-release:__<version>__
----

.. Process each `CredentialsRequest` object in the `./credrequests` directory with `ccoctl`:
+
[source,terminal,subs="+quotes"]
----
$ ccoctl create iam-roles --name-prefix=__<name_prefix>__ --region=__<aws_region>__ --credentials-requests-dir=__<path_to_directory_with_list_of_credentials_requests>__ --identity-provider-arn=__<arn_of_identity_provider_created_in_previous_step>__
----
+
For each `CredentialsRequest` object, the Cloud Credential Operator utility creates an IAM role with a trust policy that is tied to the specified OIDC identity provider, and a permissions policy as defined in each `CredentialsRequest` object from the {product-title} release image.
