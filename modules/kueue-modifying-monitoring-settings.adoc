= Modifying monitoring settings

*NOTE TO REVIEWERS: LET'S DISCUSS THIS.* 

You can modify the `nominalConcurrencyShares` field in the PriorityLevelConfiguration asset by modifying the value of the `annotation` `kueue.openshift.io/allow-nominal-concurrency-shares-update` annotation to `true`. 

The possible values you can specify for `nominalConcurrencyShares` are `0`, `2` (the default) and `5`. If you specify a value that is not acceptable (any value above `5`), the default value `2`, is enforced.

See the following example:

[source,yaml]
----
apiVersion: flowcontrol.apiserver.k8s.io/v1
kind: PriorityLevelConfiguration
metadata:
  name: kueue-visibility
  annotations:
    kueue.openshift.io/allow-nominal-concurrency-shares-update: "true"
spec:
  limited:
    borrowingLimitPercent: 0
    lendablePercent: 90
    limitResponse:
      queuing:
        handSize: 4
        queueLengthLimit: 50
        queues: 16
      type: Queue
    nominalConcurrencyShares: 2
  type: Limited
----
