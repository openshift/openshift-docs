= About signature mirroring and verification in oc-mirror plugin v2

Starting with {product-title} 4.19, the oc-mirror plugin v2 includes new default behavior for mirroring and verifying container image signatures.

== Signature mirroring

When mirroring container images, oc-mirror plugin v2 now mirrors `Sigstore` tag-based signatures for the following images by default:

* {OCP} release images
* Operator images
* Additional images
* Helm charts

== Signature verification

With oc-mirror plugin v2 in OpenShift 4.19, signature verification is available by default, ensuring that container images match their corresponding signatures. It verifies the authenticity of the images, confirming that they have not been modified and come from a trusted source.
