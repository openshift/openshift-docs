= Configure your first SR-IOV device

[role="_abstract"]
Create a simple SR-IOV network node policy to configure virtual functions on your SR-IOV network device.

.Procedure

. Create a file named `sriov-node-policy.yaml` with the following content:
+
[source,yaml]
----
apiVersion: sriovnetwork.openshift.io/v1
kind: SriovNetworkNodePolicy
metadata:
  name: policy-intel-net-1  
  namespace: openshift-sriov-network-operator
spec:
  nodeSelector:
    feature.node.kubernetes.io/network-sriov.capable: "true"  
  resourceName: intelnics  
  numVfs: 8  
  nicSelector:
    pfNames: ["ens3f0"]  
  deviceType: netdevice  
----
 Name for this policy. You can choose any name.
 Select nodes that have SR-IOV-capable hardware. You can customize this selector to target specific nodes.
 Resource name that will be used to request these virtual functions. You will use this name when creating network attachments.
 Number of virtual functions to create. Start with 8 for testing.
 Physical function name from your hardware verification step. Replace `ens3f0` with your actual device name (for example, if PCI address was `0000:19:00.0`, the device name might be `ens3f0` - use `ip link show` to verify).
 Use `netdevice` for standard kernel networking. Use `vfio-pci` only for DPDK applications.

. Create the SR-IOV network node policy:
+
[source,terminal]
----
$ oc create -f sriov-node-policy.yaml
----

. Monitor the configuration progress. The SR-IOV Network Operator will configure the device and may reboot the node:
+
[source,terminal]
----
$ oc get sriovnetworknodestates -n openshift-sriov-network-operator
----
+
Wait until the `SYNC STATUS` shows `Succeeded` for your node. This may take 5-10 minutes and may require a node reboot.

.Verification

. Verify the virtual functions were created:
+
[source,terminal]
----
$ oc get sriovnetworknodestates -n openshift-sriov-network-operator <node_name> -o yaml
----
+
Look for the `status.interfaces` section. You should see 8 virtual functions listed under your physical function.

. Verify the network device pool was created:
+
[source,terminal]
----
$ oc get node <node_name> -o json | jq '.status.allocatable'
----
+
You should see `openshift.io/intelnics: "8"` in the output, confirming that 8 virtual functions are available.

[NOTE]
====
This basic configuration uses simple settings. For advanced options like VF partitioning, MTU configuration, and NUMA alignment, see the "Configure SR-IOV network devices" guide.
====
