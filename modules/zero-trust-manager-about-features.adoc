= About the available features

The following features will be available as part of the intial release of {zero-trust-full}.

[id="spiffe-csi-driver_{context}"]
== SPIFFE CSI Driver

The SPIFFE Container Storage Interface (CSI) is a plugin that helps pods securely obtain their {svid-full} by delivering the Workload API socket into the pod. The SPIFFE CSI driver is deployed as a daemonset on the cluster ensuring that a driver instance runs on each node. The driver uses the ephemeral inline volume capability of Kubernetes allowing pods to request volumes directly provided by the SPIFFE CSI driver simplifying their use by applications that need temporary storage.

When the pod starts, the Kubelet calls the SPIFFE CSI driver to provision and mount a volume into the pod's containers. The SPIFFE CSI driver mounts a directory that contains the SPIFFE Workload API into the pod. Applications in the pod then communicate with the Workload API to obtain their SVIDs. The driver guarantees that each SVID is unique.

[id="spire-oidc-federation_{context}"]
== SPIRE OpenID Connect Federation

SPIRE OpenId Connect Federation (OIDC) capability allows workloads to authenticate to external systems using short-lived OIDC credentials. An OIDC Discovery Provider servers as a trusted source for OIDC documents, enabling other OIDC providers to validate tokens issued by SPIRE. These documents contain all the essential URLs, supported features, and configuration details allowing the client applications to be dynamically configured to work with other OIDC Providers.

[id="spire-agent-telemetry_{context}"]
== SPIRE server and agent telemetry

SPIRE server and agent telemetry provide insight into the health of the SPIRE deployment. The metrics are in the format provided by the Prometheus Operator. The following are some of the metrics that can be displayed by the SPIRE server:

* monitoring

* debugging

* performance analysis

* alerting

The SPIRE agent telemetry refers to the metrics and logs provided by the SPIRE agent. The following are some of the metrics available in the SPIRE agent:

* agent startup time

* node attestation

* workload attestation

* SVID requests and issuance

[id="spire-controller-manager_{context}"]
== SPIRE Controller Manager

The SPIRE Controller Manager uses custom resource definitions (CRDs) to facilitate the registration of workloads and establish federation relationships. To facilitate workload registration, the SPIRE Controller Manager registers controllers against pods and CRDs. When changes are detected on these resources, a workload reconciliation process is triggered. This process determines which SPIRE entries should exist based on the existing pods and CRDs. The reconciliation process creates, updates, and deletes entries on the SPIRE server as appropriate.

To facilitate federation, the SPIRE Controller Manager registers controllers against the `ClusterFederatedTrustDomain` CRD. When changes are detected on this resource, the federation reconciliation process begins. This process determines which SPIRE federation relationships should exist based on the existing `ClusterFederatedTrustDomain` resources. It creates, updates, and deleted federation relationships as appropriate.

The SPIRE Controller Manager is designed to be deployed on the same pod as the SPIRE server. The manager communicates with the SPIRE server API using a private Unix Domain Socket within a shared volume.


[id="key-manager_{context}"]
== Key Manager

The Key Manager is a plugin that is responsible for managing the cryptographic private keys for signing and verifying SVIDs. The Key Manager `disk` plugin generates a key pair for the agent's identity and stores it on a disk. This ensures that the keys are not lost if the system goes down or if the system is restarted. Each time the agent is restarted, the key is loaded from the disk.

The Key Manager `memory` plugin stores the keys in memory, rather than on a disk. This plugin is often used where the keys are emphemeral and do not need to persist. If a system goes down or restarts, the keys are lost.
