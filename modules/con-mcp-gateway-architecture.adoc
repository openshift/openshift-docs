= {mcpg} architecture

[role="_abstract"]
The architecture of the Model Context Protocol (MCP) gateway builds on the routing capabilities of the Envoy proxy, adding the capabilities to handle the MCP. Because the {mcpg} using the Envoy proxy, you can use {prodname} for policies and avoid complex configuration.

The design includes the following high-level goals:

* The MCP gateway works with the Gateway API as a routing configuration.
* The Envoy proxy controls routing and traffic as the implementation of the Gateway API.
* The MCP gateway focuses on the MCP Protocol.
* Uses Istio as the gateway control plane in {ocp} with the Gateway API.
* Uses {prodname} for policies, such as `AuthPolicy` and rate-limiting custom resources.
