= Checklist for migrating {SMProduct} 2 routing

Remember that automatic route creation, also known as Istio OpenShift Routing (IOR), is a deprecated feature that is disabled by default for any `ServiceMeshControlPlane` resource created using {SMProduct} 2.5 and later. To move from {SMProduct} 2 to {SMProduct} 3, you need to migrate from IOR to explicitly-managed routes.

If you already moved to explicitly-managed routes in {SMProduct} 2, then continue to the next checklist.
