= Network flows advanced filtering and query reference

[role="_abstract"]
The Network Observability Operator uses a Query Expression Language to filter vast amounts of `NetFlow` data. You can use *Quick Filters* for common tasks or write custom expressions for deep forensic analysis.

[id="default-quick-filters_{context}"]
== Default quick filters
[cols="1,3",options="header"]
|===
|Filter |Usage
|*External ingress* |Isolates traffic originating outside the cluster entering the SDN.
|*External egress* |Isolates traffic leaving the cluster for external destinations.
|*Dropped* |Shows traffic blocked by network policies.
|===

[id="advanced-query-syntax_{context}"]
== Advanced query syntax
You can enter expressions directly into the filter bar using the following attributes:

[cols="1,2",options="header"]
|===
|Attribute |Description
|`SrcK8S_Type` |Resource type (for example, `VirtualMachine` or `Pod`).
|`FlowStatus` |Status of the flow (for example, `Succeeded` or `Dropped`).
|`RTT` |Round Trip Time in milliseconds.
|===

.Example Queries
[source,text]
----
# Audit all traffic for SDN-managed Virtual Machines
SrcK8S_Type = 'VirtualMachine'

# Identify high-latency database connections
RTT > 100ms AND DstPort = 5432
----
