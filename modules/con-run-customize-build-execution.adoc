= Run and customize a build execution

[role="_abstract"]
Run and customize build executions by creating `BuildRun` custom resources (CRs) that reference existing `Build` configurations or include standalone specifications.
Customize individual build runs by overriding pod configuration, volumes, parameter values, and retention settings without modifying the base `Build` CR.
This approach provides flexibility for one-time builds or testing while maintaining reusable build templates.

A `BuildRun` CR triggers a single image build execution.
You can create a `BuildRun` that references an existing `Build` CR by name, or define a complete build specification directly in the `BuildRun` using the `spec.build.spec` field.
The build controller processes the `BuildRun` and creates the necessary pods and resources to execute the build.

Customization options in `BuildRun` CRs allow you to override settings from the base `Build` CR for specific execution requirements.
You can configure pod-level settings such as resource limits, node selectors, and tolerations to control where and how the build pod runs.
Volume configurations can be added or modified to provide build-specific storage or inject configuration data.
Parameter value overrides enable testing different builder image versions or build arguments without changing the `Build` CR.

Retention parameters control the automatic cleanup of completed build runs.
You can configure retention policies based on success or failure status, set time-to-live durations, or limit the number of build runs to keep.
These settings help manage cluster resources by automatically removing old build runs while preserving recent build history for troubleshooting.
