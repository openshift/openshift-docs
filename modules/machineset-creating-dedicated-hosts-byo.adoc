= Place a machine on a specific Dedicated Host

[role="_abstract"]
You can create an individual machine that is placed on a specific {aws-first} Dedicated Host. Because each machine targets a single host by its ID, you must create a separate `Machine` resource for each Dedicated Host.

:FeatureName: {aws-short} Dedicated Host support
include::snippets/technology-preview.adoc[]

.Prerequisites

* You have access to the cluster as a user with the `cluster-admin` role.
* You have an existing {aws-short} Dedicated Host with an instance type that is compatible with the machine.

.Procedure

. Create a YAML file that defines the `Machine` resource with the Dedicated Host configuration:
+
[source,yaml]
----
apiVersion: machine.openshift.io/v1beta1
kind: Machine
metadata:
  labels:
    machine.openshift.io/cluster-api-cluster: <infrastructure_id>
    machine.openshift.io/cluster-api-machine-role: worker
    machine.openshift.io/cluster-api-machine-type: worker
  generateName: <infrastructure_id>-dh-
  namespace: openshift-machine-api
spec:
  lifecycleHooks: {}
  metadata: {}
  providerSpec:
    value:
      ami:
        id: <ami_id>
      apiVersion: machine.openshift.io/v1beta1
      blockDevices:
      - ebs:
          encrypted: true
          iops: 0
          volumeSize: 120
          volumeType: gp3
      credentialsSecret:
        name: aws-cloud-credentials
      deviceIndex: 0
      iamInstanceProfile:
        id: <infrastructure_id>-worker-profile
      instanceType: <instance_type>
      kind: AWSMachineProviderConfig
      metadata:
        creationTimestamp: null
      metadataServiceOptions: {}
      placement:
        availabilityZone: <zone>
        region: <region>
        tenancy: host
        host:
          affinity: DedicatedHost
          dedicatedHost:
            id: <dedicated_host_id>
      securityGroups:
      - filters:
        - name: tag:Name
          values:
          - <infrastructure_id>-node
      - filters:
        - name: tag:Name
          values:
          - <infrastructure_id>-lb
      subnet:
        filters:
        - name: tag:Name
          values:
          - <infrastructure_id>-subnet-private-<zone>
      tags:
      - name: kubernetes.io/cluster/<infrastructure_id>
        value: owned
      userDataSecret:
        name: worker-user-data
----
+
where:
+
--
`<infrastructure_id>`:: Specifies the infrastructure ID that is based on the cluster ID that you set when you provisioned the cluster. You can obtain the infrastructure ID by running the following command:
+
[source,terminal]
----
$ oc get -o jsonpath='{.status.infrastructureName}{"\n"}' infrastructure cluster
----
`<ami_id>`:: Specifies the {op-system-first} Amazon Machine Image (AMI) ID for your {aws-short} region. You can obtain the AMI ID from an existing machine in your cluster by running the following command:
+
[source,terminal]
----
$ oc -n openshift-machine-api -o jsonpath='{.spec.template.spec.providerSpec.value.ami.id}{"\n"}' get machineset/<infrastructure_id>-worker-<zone>
----
`<instance_type>`:: Specifies the {aws-short} instance type, for example `m6i.xlarge`. The instance type must be compatible with the Dedicated Host instance family.
`<zone>`:: Specifies the availability zone where the Dedicated Host is located, for example `us-east-1a`.
`<region>`:: Specifies the {aws-short} region, for example `us-east-1`.
`<dedicated_host_id>`:: Specifies the ID of the {aws-short} Dedicated Host on which to place the machine, for example `h-0123456789abcdef0`.
--

. Create the machine by running the following command:
+
[source,terminal]
----
$ oc create -f <file_name>.yaml
----

.Verification

* Verify that the machine was created by running the following command:
+
[source,terminal]
----
$ oc get machine -n openshift-machine-api
----
