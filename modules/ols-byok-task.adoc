= BYOK task


.Prerequisites

* You are logged in to the {ocp-product-title} web console as a user with the `cluster-admin` role. Alternatively, you are logged in to a user account that has permission to create a cluster-scoped CR file.

* You have configured the LLM provider.

* You have installed the {ols-long} Operator.

* The information you want to provide resides in a Markdown - no other file format is supported at this time.

* You have an account for an image registry repository, such as Quay.

.Procedure 

. Load the image that contains the tool by running the following command:
+
[source,terminal]
----
$ podman load < <dir_for_image_tar>/my-byok-image.tar
----

. Instantiate the object by running the following command:

Load the image that contains the tool by running the following command:
+
[source,terminal]
----
$ podman images
----
+
The command displays information about the image.
+
.Example output
[source,terminal]
----
REPOSITORY                      TAG                IMAGE ID      CREATED         SIZE
localhost/my-byok-image         latest             33f09213a608  23 seconds ago  103 MB
...
----

. Tag the image you want to use:
+
[source,terminal]
----
$ podman tag localhost/my-byok-image:latest quay.io/$USENAME/my-byok-image:latest
----

. Push the image to the container image registry
+
[source,terminal]
----
$ podman push quay.io/$USENAME/my-byok-image:latest
----
