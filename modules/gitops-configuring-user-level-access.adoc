= Configuring user level access

[role="_abstract"]
To manage and change the user level access, configure the role-based access control (RBAC) section in the Argo CD custom resource (CR).

.Procedure

. Edit the `argocd` CR:
+
[source,terminal]
----
$ oc edit argocd <argocd-instance-name> -n <namespace>
----
Example Output:
+
[source,yaml]
----
metadata
...
...
  rbac:
    policy: 'g, rbacsystem:cluster-admins, role:admin'
    scopes: '[groups]'
----
+
. Add the `policy` configuration to the `rbac` section and add the `name` and the required `role` to be applied to the user:
+
[source,yaml]
----
metadata
...
...
rbac:
    policy: g, <name>, role:<role>
    scopes: '[groups]'
----

[NOTE]
====
Currently, Red Hat SSO (RH SSO) cannot read the group information of the {gitops-title} users. Therefore, configure the RBAC at the user level.
====
