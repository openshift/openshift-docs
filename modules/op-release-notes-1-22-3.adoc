= Release notes for {pipelines-title} 1.22.3

[role="_abstract"]
With this update, {pipelines-title} General Availability (GA) 1.22.3 is available on {OCP} 4.14, 4.16, and later versions.

[id="pipelines-fixed-issues-1-22-3"]
== Fixed issues

.{pac}

.User Interface

Current status column rendering::
Before this update, the *Current status* column had overlapping text. This made it hard to read pipeline task status. With this update, the column renders correctly. You can now read and verify task status easily.
+
link:https://redhat.atlassian.net/browse/SRVKP-9448[SRVKP-9448]

Webhook request validation for GitHub App credentials::
Before this update, a forged HTTP header in a webhook request could trick {pac} into sending GitHub App credentials to an attacker-controlled server. All {pac} installations were affected regardless of whether GitHub Enterprise was in use. With this update, {pac} validates webhook requests to prevent GitHub App credentials from being sent to untrusted servers. Users are encouraged to upgrade.
+
link:https://redhat.atlassian.net/browse/SRVKP-12216[SRVKP-12216]

CEL expression evaluation on Bitbucket Cloud::
Before this update, CEL expressions did not evaluate correctly on Bitbucket Cloud. As a result, PipelineRuns that used the `pipelinesascode.tekton.dev/on-cel-expression` annotation were skipped, and the {pac} controller logged a parsing error.
+
link:https://redhat.atlassian.net/browse/SRVKP-9635[SRVKP-9635]

GitHub App installation token scoping and remote resource cache fix::
Before this update, when a PipelineRun referenced remote tasks or pipelines via annotations, the token could access any private repository under the same GitHub App installation. With this update, GitHub App installation tokens are scoped to only the triggering repository by default. Unless additional repositories are explicitly configured using `secret-github-app-scope-extra-repos` in the `pipelines-as-code` config map, the token is restricted to the repository that triggered the event.
+
This also applies to incoming webhooks. The token is scoped using the repository name from the Repository CR URL.
+
Additionally, this update fixes a bug where cached remote Pipeline and Task objects could be mutated during inlining. This caused subsequent PipelineRuns referencing the same remote resource to receive corrupted definitions. Cached objects are now deep-copied before use to prevent mutation.
+
[IMPORTANT]
====
If your PipelineRuns fetch remote tasks or pipelines from other private repositories in the same installation, add those repositories to `secret-github-app-scope-extra-repos` in your `pipelines-as-code` ConfigMap using comma-separated `owner/repo` format.
====
+
link:https://redhat.atlassian.net/browse/SRVKP-12241[SRVKP-12241]
