= Manage build retention and cleanup

[role="_abstract"]
Control cluster resource usage by automatically deleting completed build runs based on time-to-live settings or count limits. Retention policies prevent build history from accumulating indefinitely while preserving recent builds for troubleshooting and auditing.

Build retention and cleanup includes:

* Retention parameters for a build and buildrun: Time-to-live settings or count limits for automatic deletion of completed build runs
* Automatic build run deletion: Background cleanup process that removes expired build runs according to configured policies

These policies balance operational needs for build history with cluster resource constraints. Platform engineers configure retention at the `Build` level to establish defaults, while individual `BuildRun` resources can override settings for special cases requiring extended preservation.
