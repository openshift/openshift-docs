= Etcd quorum restoration on the surviving node in TNF

[role="_abstract"]
After node failure, etcd on the surviving node stops when it loses quorum. Monitor the CEO conditions and pacemaker status while waiting for etcd to stop. The CEO automatically detects the failed member and runs the quorum restore process. Pacemaker brings etcd online on the surviving node in single-member mode. No manual intervention is needed in a normal node replacement flow.
+
[NOTE]
====
If a temporary Redfish outage occurs during the fencing-job window, the CEO may remain in a degraded state. The expected behavior is that the cluster retries the fencing job after Redfish connectivity is restored. If the cluster does not recover automatically, manually clear the degraded condition or restart the CEO.
====
