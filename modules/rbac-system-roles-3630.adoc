= System roles

[role="_abstract"]
{rh-rhacs-first} includes some default system roles that you can apply to users when you create rules.
You can also create custom roles as required.

[cols="1,3"]
|===
| System role | Description

| *Admin*
| Use this role for administrators. It grants read and write access to all resources.

| *Analyst*
| Use this role for a user who cannot make any changes, but can view everything. It grants read-only access for all resources.

| *Continuous Integration*
| Use this role for CI (continuous integration) systems. It includes the permission set required to enforce deployment policies.

| *Network Graph Viewer*
| Use this role for users who need to view the network graph.

| *None*
| This role has no read and write access to any resource.
You can set this role as the minimum access role for all users.

| *Sensor Creator*
| {product-title-short} uses this role to automate new cluster setups. It includes the permission set to create Sensors in secured clusters.

| *Vulnerability Management Approver*
| Use this role to give access to approve vulnerability deferrals or false positive requests.

| *Vulnerability Management Requester*
| Use this role to give access to request vulnerability deferrals or false positives.

| *Vulnerability Report Creator*
| Use this role to create and manage vulnerability reporting configurations for scheduled vulnerability reports.

|===
