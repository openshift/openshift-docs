= Installing the {ServerlessLogicOperatorName} using the Knative workflow plugin from the CLI

OpenShift Serverless Logic Workflow provides a plugin named `kn-plugin-workflow` for the Knative CLI, enabling you to set up a local workflow project using the command line.

.Prerequisites

* {ServerlessLogicOperatorName} is installed on your cluster.
* You have installed the Knative (`kn`) CLI.
* You have created a project or have access to a project with the appropriate roles and permissions to create applications and other workloads in {ocp-product-title}.

.Procedure

. Download the latest binary file.

. Install the `kn workflow` command as a plug-in of the Knative CLI using the following steps:

.. Copy the `kn-workflow` binary file to a directory in your PATH, such as `/usr/local/bin`, and ensure the file name is `kn-workflow`:
+
[source,terminal]
----
cp path/to/downloaded/kn-workflow /usr/local/bin/kn-workflow
----

.. Make the binary file executable as follows:
+
[source,terminal]
----
chmod +x /usr/local/bin/kn-workflow
----

.. Run the following command to verify that the `kn-workflow` plugin is installed successfully:
+
[source,terminal]
----
kn plugin list
----
+
After installing the plug-in, you can use `kn workflow` to run the related subcommands.

. Use the workflow subcommand in Knative CLI as follows:
+
.Aliases to use workflow subcommand
[source,terminal]
----
kn workflow
kn-workflow
----
+
.Example output
[source,terminal]
----
Manage OpenShift Serverless Logic Workflow projects

Usage:
  kn workflow [command]

Aliases:
  kn workflow, kn-workflow

Available Commands:
  completion  Generate the autocompletion script for the specified shell
  create      Creates a new OpenShift Serverless Logic Workflow project
  deploy      Deploy an OpenShift Serverless Logic Workflow project on Kubernetes via SonataFlow Operator
  help        Help about any command
  quarkus     Manage OpenShift Serverless Logic Workflow projects built in Quarkus
  run         Run an OpenShift Serverless Logic Workflow project in development mode
  undeploy    Undeploy an OpenShift Serverless Logic Workflow project on Kubernetes via SonataFlow Operator
  version     Show the version

Flags:
  -h, --help      help for kn
  -v, --version   version for kn

Use "kn [command] --help" for more information about a command.
