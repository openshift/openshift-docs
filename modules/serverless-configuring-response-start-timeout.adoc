= Configuring revision response start timeout

By setting the revision response start timeout, you can specify the maximum duration in seconds that Serving will wait for a revision to start sending network traffic after a request has been routed to it. The default is 300 seconds.

.Prerequisites

* You have installed the {ServerlessOperatorName} and Knative Serving.
* You have cluster administrator permissions on {ocp-product-title}, or cluster or dedicated administrator permissions on {rosa-product-title} or {dedicated-product-title}.

.Procedure

* To configure the revision response start timeout, set the `revision-response-start-timeout-seconds` field in your `KnativeServing` custom resource (CR):
+
.Example of revision response start timeout set to 600 seconds (10 minutes)
[source,yaml]
----
apiVersion: operator.knative.dev/v1beta1
kind: KnativeServing
metadata:
  name: knative-serving
  namespace: knative-serving
spec:
  config:
    defaults:
      revision-response-start-timeout-seconds: "600"
----
