= Troubleshooting KubeVirt Redfish

[role="_abstract"]
Diagnose issues with KubeVirt Redfish by examining logs, verifying configuration, and testing API connectivity.

.Prerequisites

* Access to the cluster with `oc` CLI and permissions to read pods and logs in the KubeVirt Redfish namespace.
* Familiarity with the configuration in xref:../../virt/post_installation_configuration/virt-kubevirt-redfish.adoc#ref_virt-kubevirt-redfish-configuration-parameters_{context}[KubeVirt Redfish configuration parameters].

.Procedure

. Verify the KubeVirt Redfish pods are running:
+
[source,terminal]
----
$ oc get pods -n <kubevirt_redfish_namespace>
----
+
All pods should show `Running` status.
If pods are in `CrashLoopBackOff` or `Error` state, check the logs.

. View the application logs:
+
[source,terminal]
----
$ oc logs deployment/<kubevirt_redfish_deployment> -n <kubevirt_redfish_namespace> --tail=100
----
+
Look for error messages related to authentication, Kubernetes API access, or configuration issues.

. If API requests are failing, locate the correlation ID in the logs:
+
[source,terminal]
----
$ oc logs deployment/<kubevirt_redfish_deployment> -n <kubevirt_redfish_namespace> --since=5m | grep -i "<correlation_id>"
----
+
The correlation ID appears in response headers and helps trace requests through the logs.

. Verify the chassis configuration matches your VM namespaces:
+
[source,terminal]
----
$ oc get vm -n <vm_namespace>
----
+
Ensure the VMs exist in the namespace specified in your chassis configuration.

. If VMs do not appear in the Systems collection:
+
--
* Verify the namespace is included in the chassis configuration
* Check that the user has access to that chassis
* Verify any label selectors match the VM labels
--

. Test authentication directly:
+
[source,terminal]
----
$ curl -sk -u "<user>:<password>" \
    https://<kubevirt_redfish_route>/redfish/v1/Systems
----
+
A `401 Unauthorized` response indicates incorrect credentials or the user does not have access to any chassis.

. For boot media issues:
+
--
* Verify the VirtualMedia actions are supported by your KubeVirt Redfish version
* Check that the storage class in your configuration supports the required access modes
* Review the VM events for disk attachment errors:
+
[source,terminal]
----
$ oc describe vm <vm_name> -n <vm_namespace>
----
--

. For VM power state issues:
+
--
* Verify the VM exists and is in the expected state:
+
[source,terminal]
----
$ oc get vmi <vm_name> -n <vm_namespace>
----
* Check if the guest agent is installed for graceful shutdown support
* Review recent events:
+
[source,terminal]
----
$ oc get events -n <vm_namespace> --sort-by='.lastTimestamp'
----
--

. Collect diagnostic information for support cases:
+
[source,terminal]
----
$ oc describe pod -l app=kubevirt-redfish -n <kubevirt_redfish_namespace>
$ oc logs deployment/<kubevirt_redfish_deployment> -n <kubevirt_redfish_namespace> > kubevirt-redfish-logs.txt
$ oc get events -n <kubevirt_redfish_namespace> > kubevirt-redfish-events.txt
----
+
[IMPORTANT]
====
Remove or redact credentials and sensitive information before sharing diagnostic files.
====

.Common issues
[cols="1,2",options="header"]
|===
|Symptom |Possible cause and resolution

|VMs not appearing in Systems collection
|Chassis configuration does not include the VM namespace, or label selector does not match. Verify `chassis[].namespace` and `chassis[].labelSelector` in values.yaml.

|401 Unauthorized responses
|Incorrect credentials or user does not have access to the requested chassis. Verify `authentication.users` configuration.

|Power actions have no effect
|VM may already be in the requested state, or the VirtualMachine CR does not exist. Verify VM state with `oc get vm`.

|Graceful shutdown times out
|Guest agent not installed or not responding. Use `ForceOff` as a fallback, or install the QEMU guest agent in the VM.

|Boot media not attaching
|Storage class does not support required access mode, or VirtualMedia actions are not implemented. Check storage class configuration and KubeVirt Redfish release notes.
|===
