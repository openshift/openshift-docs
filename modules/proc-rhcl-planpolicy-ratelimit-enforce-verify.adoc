= Verifying the PlanPolicy enforcement of ratelimiting

[role="_abstract"]
You can verify that ratelimiting is enforced by the `PlanPolicy` object by running several commands.

.Procedure

. Confirm that the AuthConfig was updated with plan data from the `PlanPolicy` object:
+
[source,terminal,subs="+quotes"]
----
$ oc get authconfig -n _<kuadrant-system>_ -o yaml
----
+
* Replace `_<kuadrant-system>_` with the namespace for the operator.
* The output should include plan information in the response section of the AuthConfig.
+ 
. Send a request without an API key and confirm that it is rejected:
+
[source,terminal,subs="+quotes"]
----
$ curl _<gateway_hostname>_/toy -i
----
+
* Replace `_<gateway_hostname>_` with the hostname for your API.
+
.Expected output
[source,terminal,subs="+quotes"]
----
HTTP/1.1 401 Unauthorized
----
+
. Send a request using the `bronze-tier` API key and confirm that it succeeds:
+
[source,terminal,subs="+quotes"]
----
$ curl -H 'Authorization: _<bronze_api_key_value>_' _<gateway_hostname>_/toy -i
----
+
* Replace `_<bronze_api_key_value>_` with the actual API key string.
* Replace `_<gateway_hostname>_` with the hostname for your API.
+
.Expected output
[source,terminal,subs="+quotes"]
----
HTTP/1.1 200 OK
----
+ 
. Send a second request using the `bronze-tier` API key and confirm that it is ratelimited:
+
[source,terminal,subs="+quotes"]
----
$ curl -H 'Authorization: _<bronze_api_key_value>_' _<gateway_hostname>_/toy -i
----
+
* Replace `_<bronze_api_key_value>_` with the actual API key string.
* Replace `_<gateway_hostname>_` with the hostname for your API.
+
.Expected output
[source,terminal,subs="+quotes"]
----
HTTP/1.1 429 Too Many Requests
----
 
. Confirm that the `gold-tier` key allows up to five requests per day before the limit is enforced.
