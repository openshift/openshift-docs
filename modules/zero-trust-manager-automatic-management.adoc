= Using SPIRE federation with Automatice Certificate Management (ACME)

[role="_abstract"]
Using SPIRE federation with ACME provides automatic certificate provisioning using Let's Encrypt. This also allows automatic certificate renewal before the certificate expires.

.Prerequisites

* You must have three OpenShift clusters.

* {zero-trust-full} must be install on all three clusters using the Operation Lifecycle Manager (OLM).

* Access to `kubeconfig` on all three clusters.

* `oc` and `curl` must be installed.

* Ability to reach federation endpoints (`https://federation.<domain>`)

* Federation endpoints must be publicly accessible for Let's  Encrypt validation.

.Procedure

. Download the federation script from `setup-3-cluster-federation-acme.sh`.

. Execute the script by running the following command:
+
[source,terminal]
----
$ ./setup-3-cluster-federation-acme.sh \
   -k1 /path/to/cluster1/kubeconfig \
   -k2 /path/to/cluster2/kubeconfig \
   -k3 /path/to/cluster3/kubeconfig \
   -c1 cluster1-name \
   -c2 cluster2-name \
   -c3 cluster3-name \
   -e your-email@example.com
----
+
The following is an example of a configured script.
+
[source,terminal]
----
$ ./setup-3-cluster-federation-acme.sh \
   -k1 /home/rausingh/Documents/aws_cluster/08DecCluster1/auth/kubeconfig \
   -k2 /home/rausingh/Documents/aws_cluster/08DecCluster2/auth/kubeconfig \
   -k3 /home/rausingh/Documents/aws_cluster/08DecCluster3/auth/kubeconfig \
   -c1 aws08dec1 \
   -c2 aws08dec2 \
   -c3 aws08dec3
----

.Verification

. Check the federation status by running the following commands:
+
[source,terminal]
----
# Cluster 1
$ oc --kubeconfig=/path/to/cluster1/kubeconfig get clusterfederatedtrustdomains
----
+
----
# Cluster 2
$ oc --kubeconfig=/path/to/cluster2/kubeconfig get clusterfederatedtrustdomains
----
+
----
# Cluster 3
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get clusterfederatedtrustdomains
----
+
.Example output
[source, terminal]
----
NAME                       TRUST DOMAIN                   AGE
cluster-12-federation      apps.cluster2.example.com      5m
cluster-13-federation      apps.cluster3.example.com      5m
----

. Check that one of the federation endpoints is available:

.. Get federation endpoint URL for cluster 3 by running the following command:
+
[source,terminal]
----
$ FEDERATION_URL=$(oc --kubeconfig=/path/to/cluster3/kubeconfig get route spire-server-federation -n zero-trust-workload-identity-manager -o jsonpath='{.spec.host}')

$ echo "Federation endpoint: https://$FEDERATION_URL"
----

.. Test the endpoint by running the following command:
+
[source,terminal]
----
$ curl -v https://$FEDERATION_URL
----

.. Fetch the trust bundle by running the following command:
+
[source,terminal]
----
$ curl https://$FEDERATION_URL > bundle.json
$ cat bundle.json | jq
----

. Check that all SPIRE resources are available.

.. Check that all SPIRE pods are available by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get pods -n zero-trust-workload-identity-manager
----

.. Check the SPIRE Server logs by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig logs -n zero-trust-workload-identity-manager spire-server-0 -c spire-server --tail=50
----

.. Check that the routes are available by running the following command:
+
[source,terminal]
----
$ oc --kubeconfig=/path/to/cluster3/kubeconfig get routes -n zero-trust-workload-identity-manager
----
