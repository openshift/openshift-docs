= Understanding the {sco}
include::_attributes/common-attributes.adoc[]
:context: siteconfig-operator

toc::[]

With the new {sco}, you can deploy clusters with all available installation methods.
The {sco} uses the unified `SiteConfig` API, which decouples parameters that define a cluster from the manner in which the cluster is deployed.
This separation removes certain limitations that are presented by the `SiteConfig` kustomize plugin in the current xref:../../edge_computing/ztp-deploying-far-edge-sites.adoc#ztp-deploying-far-edge-sites[GitOps ZTP flow], such as agent cluster installations and scalability constraints posed by ArgoCD.
Thanks to the unified `SiteConfig` API, the {sco} offers the following improvements:

Consistency:: Whether you deploy your cluster with the Assisted Installer, the Image-based Installer, or any other flows based on your custom templates, the API is consistent with all installation methods.
Scalability:: If you are using {ztp-first}, you can synchronize deployments with hundreds of clusters from a single `ClusterInstance` CR.
Flexibility:: You can define your own set of manifest templates.
Control:: You can declaratively specify sequenced operations, such as node deletion, in the `ClusterInstance` CR.
