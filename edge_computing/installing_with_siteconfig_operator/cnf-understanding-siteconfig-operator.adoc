= Understanding the {sco}
include::_attributes/common-attributes.adoc[]
:context: siteconfig-operator

toc::[]

With the new {sco}, you can deploy clusters with all available installation methods.
The Operator decouples parameters defining a cluster from the manner in which the cluster is deployed.
This separation removes certain limitations that are presented by the SiteConfig kustomize plugin in the current GitOps ZTP flow, such as agent cluster installations and scalability constraints posed by ArgoCD.
The SiteConfig Operator offers the following improvements:

Consistency:: Whether you deploy your cluster with the Assisted Installer, the Image-based Installer, or any other flows based on your custom templates, the API is consistent with all installation methods.
Scalability:: If you are using {ztp-first}, you can synchronize deployments with hundreds of clusters from a single `SiteConfig` CR.
Flexibility:: You can define your own set of manifest templates.
Control:: You can declaratively specify sequenced operations, such as node deletion, in the `SiteConfig` CR.
