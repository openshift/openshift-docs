= Configuring application pods before updating your OpenShift cluster
include::_attributes/common-attributes.adoc[]
:context: update-cnf-update-prep

toc::[]

It is very important to have a standard set of guidelines for your applications that run in cloud-native environments. In the telecommunications industry most CNF (containerized network function) vendors follow the guidance in link:https://redhat-best-practices-for-k8s.github.io/guide/[Red Hat best practices for Kubernetes] to ensure that the cluster can schedule pods properly during an upgrade.

[IMPORTANT]
====
Always deploy pods in groups by using `Deployment` resources.
`Deployment` resources spread the workload across all of the available pods ensuring there is no single point of failure.
When a pod that is managed by a `Deployment` resource is deleted, a new pod takes its place automatically.
====
