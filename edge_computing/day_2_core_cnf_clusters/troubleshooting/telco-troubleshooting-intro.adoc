= Troubleshooting and maintaining telco core CNF clusters
include::_attributes/common-attributes.adoc[]
:context: telco-troubleshooting-intro

toc::[]

Troubleshooting and maintenance are weekly tasks that can be a challenge if you do not have all the tools at your disposal to reach your goal, whether that is updating a component or investigating an issue.
Part of the challenge is knowing where and how to search for tools and answers.

The aim of this chapter is to provide quick guides, tips and tricks, best practices, and additional notes to maintain and troubleshoot a bare-metal environment where high-bandwidth network throughput is required.

[IMPORTANT]
====
This document is not a reference for configuring {product-title} or developing cloud-native network function (CNF) applications. 

For more information about best practices for developing cloud native functions (CNFs) for telco, see link:https://redhat-best-practices-for-k8s.github.io/guide/[Red Hat Best Practices for Kubernetes].
====
